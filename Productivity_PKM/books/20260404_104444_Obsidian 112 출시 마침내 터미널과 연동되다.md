# Obsidian 1.12 출시 — 마침내 터미널과 연동되다

새롭게 출시된 Obsidian 1.12 버전은 CLI(명령줄 인터페이스), 더 스마트해진 파일 처리 방식, 그리고 학술 연구 과정에서 발생하는 수많은 번거로움을 해결해 줄 미세한 워크플로우 업그레이드를 선보였습니다. Obsidian을 주력 도구로 사용하는 사용자들에게 이번 업데이트는 단순한 버전 판올림 이상의 의미가 있습니다.

### Obsidian CLI: 터미널과 지식 체계의 연결

1.12 버전의 핵심 기능은 새로운 Obsidian CLI(명령줄 인터페이스)입니다. 이제 터미널에서 직접 Obsidian을 제어할 수 있게 되었습니다.

특히 Python 스크립트, Bash 터미널, Git 저장소 등을 자주 사용하는 연구자들에게 이는 매우 큰 변화입니다. 이전에는 연구 결과를 수동으로 복사해 Obsidian에 옮겨야 했지만, 이제는 CLI를 통해 Obsidian을 스크립트로 제어할 수 있습니다.

*   시뮬레이션 실행 후 자동으로 일일 실험 로그 생성
*   스크립트를 통한 구조화된 노트 템플릿 제작
*   자동화 도구에서 보관함(Vault) 전체 검색 실행
*   태스크 러너(Task runner) 또는 CI 파이프라인과의 통합

이러한 변화는 사고 환경과 계산 환경의 경계를 허물어 줍니다. 이제 문서화 작업이 별도의 과정이 아닌, 워크플로우의 일부로 자연스럽게 통합됩니다.

### 라이브 프리뷰 이미지 크기 조절: 사소하지만 강력한 변화

이제 라이브 프리뷰(Live Preview) 모드에서 이미지 모서리를 드래그하는 것만으로 크기를 조절할 수 있습니다. 모서리를 더블 클릭하면 원래 크기로 복원됩니다.

연구자들은 종종 Python으로 생성한 그래프나 논문 다이어그램을 삽입하는데, 매번 마크다운(Markdown) 구문을 수정하며 크기를 맞추는 번거로움이 사라졌습니다. 시각적으로 즉시 크기를 조정할 수 있게 되면서 맥락 전환(Context switch) 없이 글쓰기에만 집중할 수 있게 되었습니다.

### 자동 첨부 파일 정리: 디지털 쓰레기 관리

노트를 삭제할 때 해당 노트에 연결된 첨부 파일도 함께 삭제할지 묻는 기능이 추가되었습니다. 설정의 '파일 및 링크' 항목에서 다음과 같이 선택할 수 있습니다.

*   항상 삭제
*   매번 확인
*   삭제 안 함

장기 프로젝트를 진행하다 보면 사용하지 않는 이미지와 PDF가 보관함에 쌓이기 마련인데, 이 기능은 지식 시스템을 깔끔하게 유지하는 데 도움을 줍니다.

### Bases 개선: 엑셀 없이 구조화된 사고하기

Bases 기능의 업데이트로 Obsidian 내부의 구조화된 데이터를 더욱 편리하게 다룰 수 있게 되었습니다.

*   쿼리 결과 필터링을 위한 검색 툴바 버튼 추가
*   Bases로의 드래그 앤 드롭 파일 가져오기 지원
*   표 보기(Table view)에서 개별 행에 대한 컨텍스트 메뉴 지원

이를 통해 문헌 데이터베이스, 실험 로그, 프로젝트 마일스톤 등을 도구 전환 없이 Obsidian 내부에서 관리할 수 있어 인지적 연속성을 유지하기가 훨씬 쉬워졌습니다.

### 파일 탐색기 및 편집기 개선

*   **파일 탐색기 복사 및 붙여넣기:** 이제 파일 탐색기에서 직접 단축키(Ctrl+C/V 또는 Cmd+C/V)를 사용하여 파일을 관리할 수 있습니다.
*   **편집기 우클릭 메뉴:** 편집기 빈 공간을 우클릭하여 줄 번호나 인라인 제목 표시 여부를 즉시 전환할 수 있습니다.
*   **시각적 디테일 수정:** 인용구(Blockquotes) 뒤의 공백 누락 문제와 굵은 글씨 링크의 스타일 문제 등이 해결되어 시각적 불편함을 줄였습니다.

### 캔버스 백링크 통합

캔버스(Canvas) 파일이 백링크를 올바르게 감지하여 백링크 뷰와 그래프 뷰에 표시됩니다. 개념 지도나 연구 프레임워크를 캔버스로 작성할 때, 이를 지식 그래프의 일부로 완벽하게 통합할 수 있게 되었습니다.

### 안전한 보관함 관리 및 전환

'보관함 변경(Change vault)' 메뉴가 '보관함 관리(Manage vaults)'로 이름이 변경되었으며, 새로운 명령어를 통해 키보드만으로 보관함을 빠르게 전환하거나 현재 보관함을 연 상태에서 다른 보관함을 열 수 있습니다. 연구용, 개인용, 프로젝트용 보관함을 분리해서 사용하는 사용자에게 매우 유용합니다.

### 기타 주요 업데이트

*   **이미지 및 HTML 복사:** 이미지를 우클릭하여 직접 복사하거나, 텍스트를 HTML 형식으로 복사할 수 있어 외부 도구(구글 문서 등)로의 이동이 매끄러워졌습니다.
*   **외부 앱 실행 안전성:** 외부 애플리케이션으로 파일을 열 때 확인 대화 상자가 표시되며, 실행 파일 실행 시 경고가 표시됩니다.
*   **동기화 및 검색 개선:** Obsidian Sync에서 용량 제한으로 건너뛴 파일에 대한 로그를 제공하며, 공백이 포함된 검색어에 대한 퍼지 검색(Fuzzy search) 결과가 개선되었습니다.
*   **안정성 향상:** 레이아웃 저장 문제, 아웃라인 뷰 깜빡임, 스크롤 위치 오류 등 다수의 버그가 수정되어 시스템 안정성이 높아졌습니다.

### 개발자 업데이트

*   Electron v39.7.0 버전으로 업데이트되었습니다.
*   번역 파일 처리 방식이 개선되었습니다.
*   `corner-shape` CSS 속성을 통한 새로운 테마 지원이 추가되었습니다.
*   `appendBinary` 메서드 등 새로운 API가 추가되었습니다.

### 결론

Obsidian 1.12는 화려한 신기능보다는 '통합'과 '다듬기'에 집중한 업데이트입니다. CLI를 통해 외부 기술 생태계와 연결되고, 이미지 조절과 파일 관리 기능으로 미세한 불편함을 해소했습니다. 장기적인 연구 프로젝트를 진행하는 사용자에게 안정적이고 신뢰할 수 있는 도구로서 한 단계 더 진화한 모습입니다.
