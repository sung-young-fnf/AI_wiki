최근 커맨드라인(command-line) 기반 코딩 도우미를 연구하던 중, 이전에 들어보지 못했던 도구를 접하게 되었습니다. 이 도구는 **Tmux**라고 불리며, **Terminal Multiplexer**의 약자입니다.

가장 간단히 말해, Tmux는 하나의 터미널 창을 여러 개의 개별 창으로 분할할 수 있게 해주며, 각 창은 그 자체로 완전한 터미널 기능을 수행합니다. Tmux와 같은 도구의 채택이 급증한 주된 이유는 Claude Code와 같은 커맨드라인 코딩 에이전트 제품에서 발견되는 에이전트 팀 워크플로우(agent team workflows) 때문입니다. 여기서 Claude Code를 언급했지만, Google의 Gemini CLI, OpenAI의 Codex와 같은 주요 경쟁 제품들도 이미 자체적인 유사 기능을 제공하고 있거나 개발 중입니다.

핵심은 Claude Code와 같은 도구에서 여러 에이전트가 작업을 수행할 때, 각 에이전트에게 고유한 윈도우 페인(window pane)을 할당할 수 있다는 점입니다. 이러한 시각적 단서는 에이전트 기반 작업(agentic workloads)이 진행됨에 따라 (사람이) 상황을 훨씬 더 쉽게 추적할 수 있도록 돕습니다.

Claude Code는 필요에 따라 여러 개의 개별 터미널 윈도우 페인을 생성하고, 각 에이전트에 페인을 할당하며, 에이전트가 작업을 완료하면 자동으로 해당 페인을 닫습니다. 그리고 많은 경우, 터미널 관리를 위해 Tmux를 사용합니다.

미래에는 점점 더 많은 작업이 다수의 에이전트를 통해 수행될 가능성이 높으므로, Tmux를 더 자주 접하고 사용하게 될 것입니다. 따라서 Tmux가 무엇이며 코딩 플랫폼 외에 어떤 기능을 할 수 있는지 더 자세히 알아보는 것이 합리적이라고 생각합니다. 이 글의 나머지 부분에서는 Tmux를 다운로드하는 방법과 Tmux로 할 수 있는 몇 가지 일반적인 작업을 설명할 것입니다. 가장 중요한 개념들만 다룰 것이며, 글 말미에 Tmux 홈페이지 링크를 추가할 예정입니다.

**참고:** 저는 이 글에 언급된 제품이나 회사와 아무런 관련이나 제휴가 없으며, 단지 사용자일 뿐입니다.

## Tmux 라이선스

Tmux는 완전히 무료로 사용할 수 있으며 오픈 소스입니다. 저장소(repo)의 각 소스 코드 파일에는 다음 라이선스가 첨부되어 있습니다.

본 소프트웨어를 유료 또는 무료로 어떤 목적으로든 사용, 복사, 수정 및 배포하는 것을 허가합니다. 단, 위의 저작권 고지 및 이 허가 고지가 모든 복사본에 표시되어야 합니다.

본 소프트웨어는 "있는 그대로(AS IS)" 제공되며, 저작자는 상업성 및 특정 목적에의 적합성에 대한 모든 묵시적 보증을 포함하여 이 소프트웨어에 대한 모든 보증을 부인합니다. 어떠한 경우에도 저작자는 본 소프트웨어의 사용 또는 성능과 관련하여 발생하는 정신적 손실, 사용 손실, 데이터 손실 또는 이익 손실로 인해 발생하는 모든 종류의 특별, 직접, 간접, 또는 결과적 손해 또는 어떠한 손해에 대해서도 계약, 과실 또는 기타 불법 행위 여부를 불문하고 책임을 지지 않습니다. 본 소프트웨어의 사용 또는 성능과 관련하여 발생하는 경우를 포함합니다.

## Tmux 설치하기

운영 체제에 따라 다음 방법 중 하나로 Tmux를 설치할 수 있습니다. Tmux는 완전히 무료로 사용할 수 있습니다.

### macOS

macOS에 Tmux를 설치하는 가장 쉬운 방법은 Homebrew를 이용하는 것입니다. 터미널을 열고 다음을 실행하세요:

```bash
brew install tmux
```

### Ubuntu / Debian / Linux Mint

데비안(Debian) 기반 배포판에서는 `apt` 패키지 관리자를 사용하세요:

```bash
sudo apt update
sudo apt install tmux
```

### Fedora / CentOS / RHEL

`dnf` 또는 `yum`을 사용하는 배포판의 경우:

```bash
sudo dnf install tmux
```

### Windows

저는 Windows 사용자이지만, 아쉽게도 Windows는 Tmux를 기본적으로 지원하지 않습니다. 하지만 **Windows Subsystem for Linux (WSL)**를 통해 실행할 수 있습니다. 그렇게 하려면 다음 단계를 따르세요:

1.  Microsoft Store에서 WSL 리눅스 배포판(예: Ubuntu)을 설치합니다.
2.  WSL 터미널을 엽니다.
3.  Ubuntu 명령어를 사용하여 `sudo apt install tmux`를 실행합니다.

## 알아야 할 주요 Tmux 용어

다음은 Tmux의 기본적인 개념이며, 거의 모든 다른 기능들이 이를 기반으로 합니다.

*   **세션(Sessions)**: 연결이 끊어져도 유지되는 영구적인 작업 공간입니다. 나중에 연결을 해제(detach)했다가 다시 연결(reattach)할 수 있습니다.
*   **윈도우(Windows)**: 세션 내의 탭(tab)과 같습니다. 각 윈도우는 완전한 터미널 환경을 가집니다.
*   **페인(Panes)**: 윈도우 내의 서브 윈도우(sub-window)와 같습니다. 수직 또는 수평으로 나눌 수 있습니다. 각 페인은 그 자체로 작은 커맨드라인 터미널입니다.

Tmux의 주요 장점 중 하나는 세션을 연결(attach)하고 해제(detach)할 수 있다는 점입니다. 세션을 해제하면 실행 중인 모든 프로세스가 백그라운드에서 계속 실행되며, 나중에 세션에 다시 연결하면 이미 진행 중이던 작업의 업데이트된 상태를 확인할 수 있습니다.

## Tmux 세션 시작하기

Tmux 사용의 "Hello World"는 단순히 세션을 시작하는 것입니다.

```bash
$ tmux
```

큰 변화는 없을 것이지만, 터미널 창 하단에 녹색 상태 바(status bar)가 나타나며, 이는 현재 Tmux 세션 안에 있음을 나타냅니다. 여기서 일반 운영 체제 명령을 입력할 수 있습니다. 예를 들어,

![Image by Nano Banana](https://cdn-images-1.medium.com/max/1600/1*986rQ42K2KjY4QvK4c-H4Q.png)

다음 명령어를 사용하여 모든 세션을 나열할 수 있습니다:

```bash
$ tmux ls
```

```
0: 3 windows (created Tue Feb 10 18:22:12 2026)
2: 1 windows (created Tue Feb 10 19:11:12 2026)
3: 1 windows (created Wed Feb 11 10:42:16 2026) (attached)
session2: 2 windows (created Tue Feb 10 18:29:23 2026)
```

세션을 종료하려면 Tmux 터미널에 다음 중 하나를 입력하세요:

```bash
# 특정 세션을 종료하려면:
tmux kill-session -t [session_name_or_number]

# 모든 세션을 종료하려면 ("Nuke" 옵션):
tmux kill-server
```

## Tmux 내장 명령 실행하기

Tmux는 명령을 실행하기 위해 "접두사(prefix)" 키 조합에 의존합니다. 기본적으로 이는 `Ctrl + b` (즉, `Control` 키를 누른 상태에서 `b` 문자를 누른 다음 둘 다 놓는 것)이며, 일반적으로 그 뒤에 단일 문자가 따라옵니다. 다음은 자주 사용하게 될 몇 가지 일반적인 명령어입니다.

| 액션                     | 명령                       |
| :----------------------- | :------------------------- |
| 화면 수직 분할             | `Ctrl + b` 다음 `%`        |
| 화면 수평 분할             | `Ctrl + b` 다음 `"`        |
| 다른 페인으로 이동         | `Ctrl + b` 다음 화살표 키 |
| 현재 윈도우 종료           | `Ctrl + b` 다음 `&`        |
| Tmux 종료                | `Ctrl + d`                 |

### 예시 1 - 새 페인 생성하기

위와 같이 Tmux 세션을 시작한 후, `Ctrl+b`를 누르고 이어서 퍼센트 문자(`%`)를 입력하여 원래 윈도우를 수직으로 두 개의 페인으로 분할해 보세요. 다음과 같은 화면이 나타날 것입니다.

![Image by Nano Banana](https://cdn-images-1.medium.com/max/1600/1*bJ5xR3F3o0C7fH5V6rW54Q.png)

이제 `Ctrl+b`를 누르고 이어서 쌍따옴표(`"`) 문자를 입력하면, 새 페인이 다음과 같이 수평으로 분할된 것을 볼 수 있을 것입니다.

![Image by Nano Banana](https://cdn-images-1.medium.com/max/1600/1*N0pM7Kx-6O4i2m6P0x0TLA.png)

현재 윈도우의 "이름"은 항상 화면 하단의 상태 바(status bar)에 표시됩니다. 기본적으로 이들은 0부터 순차적으로 번호가 매겨지지만, 원한다면 이름을 변경할 수 있습니다.

활성 페인(active pane)을 닫으려면 `Ctrl+b`를 누르고 이어서 `x` 문자를 입력하세요. 상태 바에 작업 확인을 요청하는 프롬프트가 나타날 것입니다.

### 예시 2 - 새 윈도우 생성하기

새 윈도우를 생성하려면 `Ctrl+b`를 누르고 이어서 `c` 문자를 입력하세요. 새 윈도우는 현재 표시된 기존 Tmux 윈도우를 덮어 가릴 것입니다. 하지만 상태 바를 보면 모든 윈도우 이름이 표시되는 것을 볼 수 있습니다. 다른 윈도우 간에 전환하려면 `Ctrl+b`를 누르고 이어서 `n` (다음 윈도우) 또는 `p` (이전 윈도우)를 사용하여 키보드 단축키를 사용하세요. `n` 또는 `p` 대신 특정 윈도우 번호를 입력하여 해당 윈도우로 전환할 수도 있습니다.

윈도우를 닫으려면 `Ctrl+b` 명령을 누르고 이어서 앰퍼샌드(`&`) 문자를 입력하세요. 상태 바에 작업 확인을 요청하는 프롬프트가 나타날 것입니다.

### 예시 3 - 페인 간 이동 및 마우스 활성화

자, 윈도우 안에 여러 개의 페인을 생성했습니다. 이제 페인들 사이를 이동해야 한다고 가정해 봅시다. 참고로 현재 활성화된 페인은 녹색 테두리로 표시되어 있어 항상 어느 페인에 초점이 맞춰져 있는지 알 수 있습니다.

두 가지 방법이 있습니다. 키보드를 통해 `Ctrl+b`를 누르고 적절한 화살표 키를 누를 수 있습니다. 페인을 이동하고 싶을 때마다 이 순서를 입력해야 합니다.

이처럼 항상 키보드를 사용하여 이동하는 것은 번거로울 수 있지만, 대신 마우스를 사용하는 방법이 있습니다. 마우스를 활성화하려면 `Ctrl+b`를 누르고 이어서 콜론(`:`) 문자를 입력하세요. 화면 하단 바의 색깔이 바뀌고 콜론 프롬프트가 나타날 것입니다. 다음을 입력하세요:

```bash
: set -g mouse on
```

그 후에는 마우스 왼쪽 클릭만으로 페인들 사이를 전환할 수 있게 될 것입니다. 여러 윈도우가 열려 있다면, 상태 바에서 해당 윈도우 이름을 왼쪽 클릭하여 윈도우 간을 이동할 수도 있습니다.

### 예시 4 - Tmux 설정 파일

이전 예시에서는 세션별로 마우스 설정을 했습니다. 이는 Tmux를 닫았다가 다시 열면 마우스 설정이 사라진다는 의미입니다. Tmux를 시작할 때마다 이렇게 설정해야 하는 번거로움을 피하려면, 홈 디렉터리(home directory)에 있는 `.tmux.conf`와 같은 설정 파일(configuration file)에 해당 설정을 넣을 수 있습니다. 다음과 같이요.

```bash
echo "set -g mouse on" > ~/.tmux.conf
```

Tmux를 다시 시작하지 않고 변경 사항을 즉시 적용하려면 Tmux 터미널 내에서 다음을 실행하세요:

`Ctrl + b` 다음 `:` 다음 `source-file ~/.tmux.conf`를 입력합니다.

그렇지 않으면, Tmux가 다음에 시작될 때 자동으로 설정 파일을 찾아 존재하면 사용합니다.

설정 파일에 추가할 수 있는 다른 많은 설정들이 있으니, 전체 목록은 글 말미에 있는 Tmux 문서 링크를 확인해 보시기 바랍니다.

### 예시 5 - 세션 분리(Detaching)

이것이 바로 Tmux의 초능력입니다. 프로세스(예: 대용량 다운로드)를 시작하고, 거기서 "분리(detach)"한 다음, 터미널 앱을 닫아도 프로세스는 백그라운드에서 계속 실행됩니다. 세션을 분리하려면 `Ctrl+b`를 누르고 이어서 `d` 문자를 입력하세요. 세션이 분리되면, 일반 터미널 명령 창에 다음을 입력하여 다시 연결(re-attach)할 수 있습니다:

```bash
$ tmux attach
```

### 예시 6 - 나만의 `Ctrl+b` 명령어 생성하기

사용자 지정 `Ctrl+b` 명령을 생성하려면 Tmux의 `bind` 명령을 사용하세요. 예를 들어, `Ctrl+b` 다음에 `m` 문자를 입력할 때마다 마우스 지원을 활성화하는 단축키 시퀀스를 만들고 싶다고 가정해 봅시다. 다음 명령어를 입력하세요:

```bash
# 먼저 상태 바의 명령줄로 이동합니다.
Ctrl+b :
# 이제 Ctrl+b m을 눌러 마우스를 ON으로 설정하도록 매핑합니다.
bind m set -g mouse on \; display 'Mouse: ON'
```

다시 말하지만, 이것은 현재 세션에만 마우스 ON 키 매핑을 활성화할 것입니다. 모든 Tmux 세션에서 사용할 수 있도록 하려면 `bind` 명령을 `.tmux.conf` 파일에 배치하세요.

```bash
echo "bind m set -g mouse on \; display 'Mouse: ON'" >> .tmux.conf
```

### 예시 7 - 기타 팁

*   **페인 크기 조정(Resizing a pane).** 마우스가 활성화되어 있다면, 다른 페인과의 경계선을 왼쪽 클릭하여 좌우 또는 위아래로 드래그하여 페인 크기를 조정할 수 있습니다. 마우스가 활성화되어 있지 않다면, `Ctrl+b`를 누른 다음 `ALT` 키를 누른 상태에서 화살표 키를 누르세요.
*   **모든 `Ctrl-b` 명령 표시(Display all Ctrl-b commands).** `Ctrl+b`를 누르고 이어서 물음표(`?`) 문자를 입력하세요. 이 표시를 종료하려면 `c` 또는 `ESC` 키를 누르세요.
*   **확대(Zooming in).** 여러 페인이 있는 경우, `Ctrl+b`를 누르고 이어서 `z`를 누르면 현재 활성 페인이 전체 윈도우를 채웁니다. 동일한 키 조합을 다시 누르면 원래대로 돌아갑니다.
*   **"내가 어디에 있지?" 플래시(The "Where Am I?" Flash).** 화면에 페인이 가득하여 어떤 페인이 무엇인지 놓쳤을 때, `Ctrl + b`를 누르고 이어서 `q`를 누르세요. 각 페인 위에 큰 숫자가 깜박이며 나타날 것입니다. 이는 각 페인의 번호를 보여줄 뿐만 아니라, 숫자가 표시되는 동안 해당 번호를 입력하면 즉시 그 페인으로 이동합니다.
*   **페인 위치 전환(Switching pane positions).** 페인 위치를 바꾸려면 `Ctrl+b`를 누르고 이어서 여는 중괄호(`{`) 또는 닫는 중괄호(`}`) 문자를 입력하세요. 이렇게 하면 활성 페인이 윈도우 내의 다른 페인과 교환됩니다.

## 요약

Tmux와 같은 유틸리티를 익히고 사용하는 것은 개발자 도구 상자(developer toolbox)에 좋은 추가 사항이라고 생각합니다. Claude Code와 같은 많은 코드 도우미 도구들이 에이전트 기반 프로세스(agentic processes)의 진행 상황을 표시하기 위해 Tmux와 같은 도구를 적극적으로 활용하기 시작했다는 사실이 이를 뒷받침합니다. 커맨드라인 기반 에이전트 개발 워크플로우(command-line agentic development workflows)가 번성함에 따라 이러한 유형의 사용 사례를 더 많이 보게 될 것입니다.

이 글은 Tmux 유틸리티가 할 수 있는 것의 극히 일부만을 다루었습니다. 이와 같은 많은 도구들은 진정으로 능숙해지기까지 상당한 시간이 걸릴 수 있지만, 끈기를 가지고 노력할 가치가 있다고 생각합니다. 그렇게 한다면, 나중에 Tmux 없이 어떻게 작업했는지 궁금해할 수도 있을 것입니다.
