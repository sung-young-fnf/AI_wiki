저는 예전에는 터미널(terminal)을 피했습니다. 하지만 이제는 Mac 앱보다 명령줄 인터페이스(CLI) 도구를 더 많이 설치합니다.

15년 전 대학 시절에는 터미널을 많이 사용했습니다. 그러다 사용을 멈췄죠. 10년 넘게 그래픽 사용자 인터페이스(GUI)만 사용했습니다. Finder, 드래그 앤 드롭(drag-and-drop), App Store 등 일반적인 방식이었죠.

그러다 Claude Code가 등장했습니다.

일단 매일 터미널을 사용하기 시작하면, 그 안에서 더 많은 것을 하고 싶어집니다.

올해 제가 발견한 최고의 Mac 도구 중 일부는 창이 없습니다. Dock에 아이콘도 없죠. 명령어를 입력하고 엔터를 누르면 작업이 실행됩니다.

지금 제가 많이 사용하는 도구들을 소개합니다.

**Homebrew**

이 목록에서 단 하나만 설치한다면, Homebrew를 선택하세요. macOS용 패키지 관리자(package manager)입니다. 기본적으로 터미널용 App Store라고 생각하면 됩니다. 하지만 더 빠르고, 훌륭한 소프트웨어가 많으며, Apple ID로 로그인할 필요도 없습니다.

`brew install something`이라고 입력하면 끝입니다.

저는 Homebrew가 어떻게 Mac App Store를 대체했는지에 대한 글을 쓴 적이 있습니다. 그때와 달라진 점은 없습니다. 오히려 더 심해졌다고 할까요. 아니, 어떻게 보느냐에 따라 더 나아졌다고 할 수도 있습니다. 이제 거의 모든 것을 Homebrew를 통해 설치합니다. `brew install --cask`를 사용하면 GUI 앱도 설치할 수 있습니다.

Rectangle, Maccy, Espanso는 물론 Obsidian과 Notion까지. 모두 Homebrew를 통해 설치됩니다.

**Claude Code**

당연히 언급해야 할 도구입니다.

이것이 저를 터미널로 다시 이끈 장본인입니다. 저는 Claude Code에 대해 전에 글을 쓴 적이 있습니다. 터미널 내에서 동작하는 AI 코딩 어시스턴트(AI coding assistant)입니다(제 경우). 브라우저도 없고, 채팅창도 없습니다. 오직 당신, 프롬프트(prompt), 그리고 커서(cursor)만 있을 뿐입니다.

하지만…

이것은 단순히 코드를 작성하는 데 그치지 않습니다. 코드를 실행합니다. 파일을 생성하고, 스크립트를 실행하며, 패키지를 설치하고, 오류를 디버깅(debugging)합니다. 저는 이것으로 전체 Flutter 앱을 만들었습니다. 파일 관리를 자동화하고, 이미지를 일괄 처리하며, 수백 개의 HTML 파일에서 깨진 링크를 수정했습니다.

이제는 제 글쓰기 워크플로(workflow)의 많은 부분을 자동화합니다.

**Mole**

이 도구는 (저에게는) 상당히 새로운 것입니다.

Mole은 Mac을 딥 클리닝(deep clean)하고 최적화(optimize)하는 터미널 도구입니다. CleanMyMac과 같다고 생각하면 되지만, 무료이고 오픈 소스(open source)이며 단 한 번의 명령어로 실행됩니다.

`mole`

이것이 전부입니다. 불필요한 파일, 캐시(cache), 로그(log), 깨진 심볼릭 링크(symlink) 등 macOS가 시간이 지남에 따라 쌓는 모든 것을 스캔합니다. 찾은 내용을 보여주고, 무엇을 제거할지 결정하게 합니다.

구독료도 없고, 추가 판매도 없으며, "전체 클리닝을 위해 Pro로 업그레이드하세요" 같은 문구도 없습니다. 그저 깨끗한 Mac을 만들어줍니다.

최근에 발견했는데, 이미 열렬한 팬이 되었습니다.

**ripgrep**

혹시 파일에서 텍스트를 검색하기 위해 `grep`을 사용해 본 적이 있다면, ripgrep (rg)은 그보다 훨씬 빠릅니다. `.gitignore` 파일을 존중하고, 기본적으로 숨겨진 디렉터리(directory)를 건너뛰며, 출력 결과가 읽기 쉽습니다.

`rg "something" .`

저는 이것을 끊임없이 사용합니다. 수백 개의 발행된 글에서 특정 문구를 찾을 때, ripgrep이 빠르게 찾아줍니다.

참고로 Claude Code도 내부적으로 ripgrep을 사용합니다.

**fd**

ripgrep과 같은 아이디어이지만, 텍스트 대신 파일을 찾는 도구입니다.

Mac의 기본 `find` 명령어는 괜찮지만, 구문(syntax)이 엉망입니다. fd는 더 간단하고 빠릅니다.

`fd "*.html" Published/`

이 명령어는 제 `Published` 폴더에 있는 모든 HTML 파일을 찾습니다. 내장된 `find`를 사용하려면 `find Published/ -name "*.html"`와 같이 입력해야 합니다. 나쁘지는 않지만, fd가 훨씬 깔끔하고, 다시 말하지만 더 빠릅니다.

**zoxide**

한 번 사용하면 다시 돌아갈 수 없습니다.

Zoxide는 `cd` 명령어를 대체합니다. 자주 방문하는 디렉터리를 학습하여 부분적인 이름만으로도 해당 디렉터리로 이동할 수 있게 해줍니다.

`cd ~/Library/Mobile\ Documents/com~apple~CloudDocs/Blogging` 대신, 저는 다음과 같이 입력합니다.

`z blogging`

끝입니다. 제가 이전에 그곳에 가본 적이 있기 때문에, Zoxide는 제가 어디로 가고 싶은지 압니다. 더 많이 사용할수록 더 똑똑해집니다.

사소한 것이지만, 많은 시간을 절약해 줍니다.

**jq**

JSON 파일로 작업한다면, jq는 아주 유용합니다. 터미널에서 JSON을 쿼리(query), 필터링(filtering), 변환(transform)할 수 있게 해줍니다.

저는 이것을 사용하여 제 블로그의 기사 데이터베이스를 확인하고, 내보내기 데이터를 필터링하며, API 응답에서 특정 필드를 추출합니다. jq가 없다면 텍스트 편집기에서 파일을 열고 스크롤해야 할 것입니다. jq를 사용하면 하나의 명령어로 정확히 필요한 것을 얻을 수 있습니다.

`cat articles.json | jq '.[] | .title'`

모든 제목을 한 줄로 얻을 수 있습니다.

**ffmpeg**

미디어 변환 도구입니다. 비디오, 오디오, 이미지 등 거의 모든 것을 다른 거의 모든 것으로 변환할 수 있습니다.

동영상을 GIF로 바꾸고 싶다면 ffmpeg을 사용하세요. YouTube 다운로드에서 오디오를 추출하는 것도 ffmpeg으로 가능합니다. 200MB 스크린 레코딩을 15MB로 압축하는 것도 ffmpeg이 해냅니다.

구문은 예쁘지 않습니다. 하지만 그것을 위해 Claude Code가 있습니다. 제가 원하는 것을 설명하면, Claude Code가 ffmpeg 명령어를 작성하고 실행해 줍니다.

**Ollama**

Mac에서 로컬로(locally) 실행되는 AI입니다. 인터넷 연결이 필요 없고, API 비용도 들지 않으며, 데이터가 기기를 벗어나지 않습니다.

`ollama run llama3`

이 명령어를 입력하면, 하드웨어(hardware)에서 전적으로 실행되는 대규모 언어 모델(large language model, LLM)과 대화할 수 있습니다. Apple Silicon은 이 작업에 능합니다. M 시리즈 칩은 로컬 AI 모델을 잘 처리합니다.

저는 Claude의 모든 기능이 필요하지 않은 간단한 질문을 할 때 사용합니다. 또는 데이터를 클라우드(cloud)로 보내지 않고 무언가를 테스트하고 싶을 때 사용합니다. 제가 Linux에 대해 글을 쓸 때 "구조에 의한 프라이버시(Privacy by structure)"라고 불렀던 것과 같습니다.

**pandoc**

범용 문서 변환기(universal document converter)입니다. Markdown을 HTML로. HTML을 PDF로. Word를 Markdown으로. 거의 모든 텍스트 형식을 다른 텍스트 형식으로 변환할 수 있습니다.

`pandoc article.md -o article.html`

여러 플랫폼에서 여러 형식으로 글을 쓰는 사람에게는 이것이 매우 유용합니다. 저는 더 이상 "수동으로" 아무것도 변환하지 않습니다.

**gh**

GitHub의 공식 명령줄 인터페이스(CLI)입니다. 저는 이것을 사용하여 레포지토리(repository)를 생성하고, 풀 리퀘스트(pull request)를 열고, CI(Continuous Integration) 상태를 확인하며, 이슈(issue)를 관리합니다.

`gh repo create my-new-project --public`

더 이상 할 말이 없습니다.

**패턴**

이 도구들 대부분은 한 가지 일을 합니다. 그리고 그 한 가지 일을 아주 잘 해냅니다. 설정 패널(settings panel)도 없고, 온보딩(onboarding) 과정도 없으며, 계정 생성도, 구독료도 없습니다.

명령어를 입력하고 결과를 얻습니다.

그것이 전부입니다.

터미널 도구는 개발자만을 위한 것이 아닙니다. 훌륭한 일상 도구입니다. 더 빠르죠. 그리고 명령어를 한 번 알게 되면, 아마도 GUI 버전으로는 다시 돌아가지 않을 것입니다.

음, 대부분의 경우에는 그렇습니다. 저는 터미널에서 사진을 편집하지는 않습니다. 아직은…

**결론**

Mac 터미널에는 많은 사람이 전혀 알지 못하는 놀라운 소프트웨어가 가득합니다. Homebrew는 앱 스토어와 같습니다. 일단 Homebrew를 사용하기 시작하면, Mole, ripgrep, zoxide, Claude Code와 같은 도구들을 쉽게 만날 수 있습니다.

저는 터미널의 부활을 정말 좋아합니다.

작가는 독자 없이는 아무것도 아닙니다. 이 글이 도움이 되셨다면, 저의 소중한 이메일 구독자가 되어주세요. 그 어떤 것도 저를 더 행복하게 만들지 못할 것입니다.
