# 소프트웨어 엔지니어로서 삶을 더 쉽게 만들어 준 6가지 도구
당신의 개발 환경이 당신을 방해하는 것이 아니라 당신에게 이롭게 작동하도록 만드세요.

소프트웨어 엔지니어라면 매일 얼마나 많은 일들이 당신의 주의를 요구하는지 이미 알고 있을 것입니다. 코드 작성은 업무의 일부에 불과합니다. 해결해야 할 버그, 구성해야 할 도구, 관리해야 할 환경, 그리고 당신의 시간을 조용히 잡아먹는 수많은 작은 작업들이 있습니다.

바로 그렇기 때문에 개발 환경이 매우 중요합니다.

좋은 설정은 당신을 지원하고 작업을 원활하게 만듭니다. 나쁜 설정은 그 반대입니다. 당신의 속도를 늦추고, 흐름을 방해하며, 간단한 작업을 짜증 나는 일로 바꿉니다.

시간이 지나면서 저는 생산성을 높이는 가장 좋은 방법 중 하나는 환경이 당신을 방해하는 것이 아니라 당신을 위해 작동하도록 만드는 것임을 알게 되었습니다. 이는 터미널을 적절하게 설정하고, 반복적인 워크플로우를 자동화하며, 일상적인 작업을 단순화하고, 의존하는 도구를 개선하는 것을 의미합니다.

결국, 시간은 우리의 가장 소중한 자원이며, 워크플로우의 모든 작은 개선이 축적되기 때문입니다.

이 글에서 저는 제가 개인적으로 거의 매일 사용하는 6가지 도구를 공유하고자 합니다. 이 도구들은 저의 일상을 단순화하고, 일반적인 작업을 간소화하며, 개발 워크플로우를 훨씬 더 즐겁게 만드는 데 도움이 됩니다.

## 1. Refined GitHub

Refined GitHub는 GitHub를 훨씬 더 쾌적하게 사용할 수 있도록 조용히 돕는 작은 도구 중 하나입니다. 간단한 브라우저 확장 프로그램(browser extension)이지만, 일단 사용하기 시작하면 기본 인터페이스로 돌아가기 어렵습니다.

이 확장 프로그램은 일상적인 작업을 더 쉽게 만들어주는 많은 신중한 개선 사항으로 GitHub UI를 향상시킵니다. 제가 개인적으로 가장 높이 평가하는 기능 중 일부는 다음과 같습니다.

*   공백 문자(whitespace characters)를 표시하여 코드 변경 사항을 검토할 때 매우 유용합니다.
*   워크플로우를 트리거한 풀 리퀘스트(pull request)로 직접 연결되는 링크를 추가합니다.
*   반응 아바타(reaction avatars)를 표시하여 누가 댓글에 반응했는지 즉시 확인할 수 있습니다.
*   PR 제목을 기본 스쿼시 커밋 메시지(squash commit message)로 자동 사용하고, 커밋 제목이 변경되면 PR 제목을 업데이트합니다.

그리고 이것은 제공하는 기능의 작은 부분에 불과합니다. GitHub 탐색을 더 원활하고 효율적으로 만드는 다른 미묘한 개선 사항들도 많이 있습니다.

이 확장 프로그램은 Chrome, Firefox, Safari에서 사용할 수 있으며, 설치하는 데는 몇 초밖에 걸리지 않습니다.

또한 오픈 소스(open source)라는 점도 더욱 좋습니다. 궁금하다면 GitHub에서 직접 프로젝트와 그 기능을 탐색할 수 있습니다.

추신: 이 기능 중 일부는 현재 GitHub에 이미 있을 수도 있습니다. 저는 Refined GitHub를 꽤 오랫동안 사용해왔기 때문에, 이 시점에서 어떤 개선 사항이 GitHub 자체에서 온 것이고 어떤 것이 확장 프로그램에서 온 것인지 구별하기 어렵습니다.

## 2. Amphetamine

Mac, 특히 다중 모니터(multi-monitor) 설정에서 작업하는 경우 이러한 불편함을 겪어본 적이 있을 것입니다. 잠시 자리를 비우면 Mac이 잠자기 모드(sleep mode)로 전환되고, 돌아오면 모든 것이 재배치됩니다. 터미널이 이동하고, 창이 바뀌며, 레이아웃이 깨집니다.

이는 당신의 워크플로우를 완전히 방해할 수 있는 작은 성가신 일 중 하나입니다.

바로 여기에 Amphetamine이 유용합니다.

Amphetamine은 필요할 때마다 Mac을 깨어 있게 유지하는 간단한 macOS 유틸리티입니다. 단 하나의 켜기/끄기 스위치로 작업하는 동안 컴퓨터와 선택적으로 디스플레이가 잠자기 모드로 전환되는 것을 방지할 수 있습니다.

더욱 좋은 점은 트리거(trigger)를 사용하여 이러한 동작을 자동화할 수 있다는 것입니다. 예를 들어, 다음과 같은 경우 자동으로 활성화되도록 구성할 수 있습니다.

*   특정 Wi-Fi 네트워크에 연결할 때
*   특정 애플리케이션이 실행 중일 때
*   디스플레이가 연결되었을 때
*   또는 Mac이 전원에 연결되었을 때

일단 구성하면 다시 생각할 필요 없이 백그라운드에서 조용히 작동합니다.

Amphetamine에 대해 더 자세히 알아보고 Mac App Store에서 직접 다운로드할 수 있습니다.

참고: Amphetamine은 macOS에서만 사용할 수 있지만, Windows 및 Linux용으로도 동일한 기능을 제공하는 유사한 도구들이 있습니다.

## 3. Lunar

Mac에서 외부 모니터와 함께 작업하는 경우, 내장된 디스플레이 제어 기능이 얼마나 제한적인지 깨달았을 것입니다. 외부 화면의 밝기나 다른 설정을 조정하려면 종종 모니터의 물리적 버튼을 조작해야 하는데, 이는 빠르게 성가신 일이 됩니다.

바로 이럴 때 Lunar가 엄청나게 유용해집니다.

Lunar는 macOS에서 외부 모니터의 밝기, 색상, 볼륨 및 입력 설정을 직접 제어할 수 있는 경량 도구입니다. 조명이 바뀔 때마다 모니터를 수동으로 조정하는 대신, Mac에서 모든 것을 처리할 수 있습니다.

제가 특히 높이 평가하는 기능 중 하나는 주변 조명에 따라 밝기를 얼마나 쉽게 조정할 수 있는지입니다. 강한 햇빛 아래에서 낮에 작업하든, 밤늦게 코딩하든, Lunar는 디스플레이를 편안하게 볼 수 있도록 도와줍니다.

작은 도구이지만, 특히 화면 앞에서 오랜 시간을 보내는 경우 눈에 띄는 차이를 만듭니다.

솔직히, 눈의 피로를 줄이는 데 도움이 되는 모든 것은 가질 가치가 있습니다.

Lunar에 대해 더 자세히 알아보고 GitHub 저장소(repository) 또는 공식 웹사이트에서 다운로드할 수 있습니다.

참고: Lunar는 현재 macOS에서 사용할 수 있습니다. 그러나 Windows 및 Linux용으로도 유사한 모니터 제어 기능을 제공하는 대안이 있습니다.

## 4. Rectangle

소프트웨어 엔지니어로 일할 때, 화면은 일반적으로 여러 창(터미널, 편집기, 브라우저, 문서, 그리고 몇 가지 디버깅 도구)으로 가득 차 있습니다. 이 모든 것을 수동으로 관리하는 것은 빠르게 혼란스러워질 수 있습니다.

바로 이럴 때 Rectangle이 엄청나게 도움이 됩니다.

Rectangle은 간단한 키보드 단축키(keyboard shortcuts) 또는 스냅 영역(snap areas)을 사용하여 창을 이동하고 크기를 조절할 수 있는 macOS 유틸리티입니다. 마우스로 창을 이리저리 끄는 대신, 원하는 위치에 정확히 즉시 배치할 수 있습니다.

예를 들어, 창을 빠르게 배치할 수 있습니다.

*   화면의 왼쪽 또는 오른쪽 절반에
*   상단 또는 하단 섹션에
*   다양한 그리드 레이아웃(grid layouts)과 크기로

몇 가지 단축키만으로 전체 작업 공간(workspace)을 몇 초 만에 정리할 수 있습니다. 일단 익숙해지면, 이는 당신의 근육 기억(muscle memory)의 일부가 되어 멀티태스킹(multitasking)을 훨씬 더 원활하게 만듭니다.

터미널, 코드 편집기, 브라우저 사이를 끊임없이 전환할 때, 이와 같은 도구는 당신의 워크플로우에 놀랍도록 좋은 생산성 향상을 가져다줄 수 있습니다.

Rectangle에 대해 더 자세히 알아보고 GitHub 저장소(repository) 또는 공식 웹사이트에서 다운로드할 수 있습니다.

참고: Rectangle은 macOS용으로 설계되었지만, Windows 및 Linux용으로도 유사한 창 관리 도구(window management tools)가 존재합니다.

## 5. Maccy

Maccy는 macOS용으로 간단하지만 엄청나게 유용한 클립보드 관리자(clipboard manager)입니다. 마지막으로 복사한 것만 기억하는 대신, 복사한 모든 항목의 기록을 유지하여 이전 클립보드 항목을 쉽게 찾고 재사용할 수 있게 합니다.

여러 항목을 연속으로 복사한 다음 이전 항목이 필요하다는 것을 깨달은 적이 있다면, 그것이 얼마나 답답할 수 있는지 알 것입니다. Maccy를 사용하면 그 문제는 기본적으로 사라집니다.

Maccy를 언제든지 열어 클립보드 기록(clipboard history)에서 항목을 빠르게 찾아보고, 검색하고, 재사용할 수 있습니다. 저는 개인적으로 링크, 데이터베이스 ID, 텍스트 조각 또는 이전에 복사한 다른 어떤 것을 검색할 때 거의 매일 사용합니다.

이것은 하루 종일 당신의 시간을 조용히 절약해 주는 작은 도구 중 하나입니다.

Maccy에 대해 더 자세히 알아보고 GitHub 저장소(repository) 또는 공식 웹사이트에서 다운로드할 수 있습니다.

참고: Maccy는 macOS에서 사용할 수 있습니다. Windows 또는 Linux를 사용하는 경우, 유사한 기능을 제공하는 클립보드 관리자 도구들이 있습니다.

## 6. Grammarly

Grammarly는 글을 쓰는 동안 문법, 철자 및 전반적인 명확성을 확인하는 데 도움이 되는 무료 AI 기반 작성 지원 도구(AI-powered writing assistant)입니다.

소프트웨어 엔지니어링이라고 하면 글쓰기를 가장 먼저 떠올리지 않을 수도 있지만, 실제로는 매일 많은 글을 씁니다. 문서, 풀 리퀘스트 설명, 커밋 메시지, 댓글, 이메일, 기술 토론 등입니다.

명확한 의사소통은 깔끔한 코드만큼이나 중요합니다.

비원어민 영어 사용자로서, 저는 개인적으로 Grammarly가 매우 유용한 도구라고 생각했습니다. 사소한 실수를 잡아내고, 문장 구조를 개선하며, 글을 더 명확하고 전문적으로 만드는 데 도움이 됩니다.

저는 종종 문서, 메시지 또는 이와 같은 글을 쓸 때 이 도구를 사용합니다. 마치 작은 편집자가 백그라운드에서 조용히 도와주는 것과 같습니다.

Grammarly에 대해 더 자세히 알아보고 웹사이트에서 다운로드할 수 있습니다. 또한 Chrome 확장 프로그램(Chrome extension)을 제공하여 온라인으로 글을 쓰는 동안 브라우저 내에서 직접 쉽게 사용할 수 있습니다.

이 글을 끝까지 읽어주셔서 감사합니다. 개발자로서 우리는 끊임없이 새로운 도구를 배우고 실험하며, 때로는 가장 작은 유틸리티가 일상적인 워크플로우에 가장 큰 변화를 가져올 수 있습니다.

제가 여기서 공유한 여섯 가지 도구는 혁신적이거나 복잡하지 않지만, 일상적인 작업에서 마찰을 조용히 제거하고 제가 진정으로 중요한 것, 즉 소프트웨어를 구축하고 문제를 해결하는 데 더 집중할 수 있도록 돕습니다.

편집자 참고: 이 글에는 제휴 링크나 유료 프로모션이 포함되어 있지 않습니다. 여기에 언급된 도구들은 전적으로 저의 개인적인 경험, 연구 및 일상적인 사용을 기반으로 합니다. 저는 소프트웨어 엔지니어로서 생산성을 유지하는 데 진정으로 도움이 되는 도구들을 공유하고 있을 뿐입니다.
