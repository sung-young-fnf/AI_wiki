# 삶을 더 편리하게 만드는 셸(Shell) 트릭

53P GN⁺ 2일 전 | ★ 즐겨찾기 | 댓글 1개

**[기사 요약]**
*   다양한 키보드 단축키와 명령어 조합을 활용하여 셸(Shell) 환경에서 작업 속도와 효율성을 높이는 방법 정리
*   `CTRL + W`, `U`, `K`, `Y` 등 기본 단축키로 줄 편집, `CTRL + A/E`, `ALT + B/F`로 커서 이동, `reset`으로 터미널(Terminal) 복구 가능
*   Bash와 Zsh에서는 `CTRL + R`로 명령 히스토리(History) 검색, `!!`·`!$`·`ESC + .` 등으로 이전 명령 재활용, 브레이스 확장(Brace Expansion)과 글롭스타(Globstar)로 반복 작업 자동화
*   프로세스 대체(Process Substitution), `tee`, `bg/disown` 등 고급 기능으로 프로세스(Process) 제어와 로그(Log) 관리 효율화
*   셸(Shell)을 도구 상자(Toolbox)처럼 익숙하게 다루면 생산성(Productivity)과 집중력을 높일 수 있으며, 단축키를 하나씩 습관화하는 것이 핵심임

### 거의 모든 셸(Shell)에서 작동하는 기본 단축키 및 기능

*   `CTRL + W`, `U`, `K`, `Y` 조합은 커서 위치를 기준으로 단어 또는 줄의 앞뒤를 빠르게 삭제하거나 복원하는 라인 편집(Line Editing) 단축키입니다.
    *   `CTRL + W`: 커서 앞의 단어 삭제
    *   `CTRL + U`: 커서부터 줄의 시작까지 잘라내기(Cut), `CTRL + Y`로 다시 붙여넣기(Paste) 가능
    *   `CTRL + K`: 커서부터 줄 끝까지 잘라내기(Cut)
*   `CTRL + A / E`: 줄의 처음과 끝으로 이동, `ALT + B / F`: 단어 단위로 앞뒤 이동
    *   대부분의 터미널(Terminal)에서 기본 활성화되어 있으며, Mac에서는 Option 키를 Meta 키로 설정해야 작동합니다.
*   `reset` 또는 `stty sane` 명령은 바이너리 파일(Binary File)을 잘못 출력하여 터미널이 손상되었을 때 복구합니다.
    *   화면이 손상되어도 입력은 가능하므로, `reset` 입력 후 엔터를 누르면 복원됩니다.
*   `CTRL + C / D`: 각각 명령 중단(Interrupt)과 EOF(End-Of-File) 신호 전송
    *   `CTRL + D`: 빈 프롬프트(Prompt)에서 누르면 셸(Shell)이 종료됩니다.
*   `CTRL + L`: `clear` 명령과 동일하게 화면을 즉시 정리합니다.
    *   현재 입력 중인 명령을 유지한 채 프롬프트(Prompt)를 화면 상단으로 이동시킵니다.
*   `cd -`: 이전 디렉터리(Directory)로 즉시 이동, `pushd / popd`: 디렉터리 스택(Directory Stack)을 이용해 여러 위치를 오가며 작업 가능
*   `> file.txt`: 파일을 비우되 삭제하지 않아 권한(Permission)과 소유권(Ownership)을 유지합니다.
*   `$_` 변수(Variable): 이전 명령의 마지막 인자(Argument)를 재사용합니다.
    *   예: `mkdir -p /path/newdir && cd "$_"`
*   `set -e`, `set -u`: 스크립트(Script) 안전성을 높이는 옵션(Option)입니다.
    *   `set -e`: 오류 발생 시 즉시 종료
    *   `set -u`: 정의되지 않은 변수를 참조할 경우 오류 처리
    *   `set -euo pipefail` 조합은 보다 강력한 보호 기능을 제공합니다.

### Bash와 Zsh에서의 고급 단축키 및 기능

*   `CTRL + R`: 명령어 히스토리(History)에서 역방향 증분 검색(Reverse Incremental Search)을 수행합니다.
    *   키워드 입력 시 과거 명령을 즉시 찾아 재사용할 수 있습니다.
*   `!!`: 이전 명령 전체를 재실행합니다.
    *   `sudo !!`: 직전 명령을 관리자 권한(Administrator Privileges)으로 재실행할 수 있습니다.
*   `CTRL + X, CTRL + E`: 현재 입력 중인 명령을 기본 편집기(Default Editor, Vim, Nano 등)에서 열어 수정한 후 실행합니다.
    *   Zsh에서는 별도 설정이 필요합니다.
*   `fc` 명령: 이전 명령을 `$EDITOR`로 열어 수정하는 전통적인 방식입니다.
*   `ESC + .` 또는 `ALT + .`: 직전 명령의 마지막 인자(Argument)를 커서 위치에 삽입합니다.
    *   반복 입력 시 더 이전 명령의 인자까지 순환시킬 수 있습니다.
*   `!$`: 같은 기능을 비대화형(Non-interactive)으로 수행합니다.
*   **브레이스 확장(Brace Expansion)**: 반복 입력을 줄이는 중괄호 확장 기능입니다.
    *   예: `cp pf.conf{,.bak} → cp pf.conf pf.conf.bak`
    *   예: `mv filename.{txt,md} → mv filename.txt filename.md`
    *   예: `mkdir -p project/{src,tests,docs}`로 여러 디렉터리를 생성할 수 있습니다.
*   **프로세스 대체(Process Substitution)** `<(command)`: 명령어 출력(Command Output)을 파일처럼 다룹니다.
    *   예: `diff <(sort file1.txt) <(sort file2.txt)`로 임시 파일(Temporary File) 없이 비교할 수 있습니다.
*   **글롭스타(Globstar)** (`**`): 하위 디렉터리(Subdirectory)까지 재귀적으로(Recursively) 파일을 탐색합니다.
    *   Bash에서는 `shopt -s globstar`로 활성화해야 하며, Zsh에서는 기본 활성화되어 있습니다.
    *   예: `ls **/*.js`로 모든 하위 폴더의 JS 파일(JavaScript File)을 탐색합니다.
*   `CTRL + Z`, `bg`, `disown` 조합: 프로세스(Process)를 백그라운드(Background)에서 실행하고 셸(Shell)과 분리합니다.
    *   `CTRL + Z`로 일시 중단(Suspend) → `bg`로 백그라운드 실행 → `disown`으로 셸과 분리
    *   SSH 종료 후에도 프로세스를 유지할 수 있습니다.
*   `command |& tee file.log`: 표준 출력(Standard Output, `stdout`)과 표준 오류(Standard Error, `stderr`)를 동시에 파이프(Pipe)로 전달합니다.
    *   `|&`는 `2>&1 |`의 축약형(Shorthand)입니다.
    *   `tee`를 사용해 화면 출력과 로그(Log) 저장을 동시에 수행합니다.

### 셸(Shell) 활용의 핵심 조언

*   셸(Shell)은 도구 상자(Toolbox)이며, 익숙하게 다루면 생산성(Productivity)을 크게 높일 수 있습니다.
*   모든 단축키를 한 번에 익히기보다 하나씩 습관화하는 것이 효과적입니다.
*   반복적인 입력과 불필요한 타이핑을 줄여 작업 흐름(Workflow)을 단순화할 수 있습니다.
*   익숙해지면 터미널(Terminal)은 더 이상 장애물이 아니라 자신만의 작업 공간이 됩니다.

위쪽 화살표 키를 다시 매핑(Remapping)한 순간, 제 인생이 바뀌었다고 느꼈습니다. 이제는 모든 명령어를 순회하지 않고, 이미 입력한 문자로 시작하는 명령어만 탐색합니다. 예를 들어 `tar -` 입력 후 위쪽 화살표를 누르면 이전에 사용한 `tar` 옵션이 바로 나옵니다.
Zsh에서는 다음과 같이 설정합니다.
```
bindkey "^[OA" up-line-or-beginning-search # Up  
bindkey "^[OB" down-line-or-beginning-search # Down
```

`CTRL + R`을 사용하기 시작하면 위쪽 화살표를 다시는 사용하지 않게 됩니다. 이것은 정말 혁신적(Game Changer)이라고 생각합니다. Bash에서도 비슷하게 하려면 `.inputrc`에 다음을 추가합니다.
```
"\e[A":history-search-backward  
"\e[B":history-search-forward
```

Fish 셸(Shell)에서는 이것이 기본 동작입니다. 입력한 단어를 기준으로 위/아래 화살표로 명령어를 탐색하고, `ALT + 위/아래`로 인자(Argument)만 탐색할 수도 있습니다. `.inputrc`에서도 동일하게 구현 가능합니다. 저는 위/아래 화살표는 그대로 두고, 대신 `CTRL + P`와 `CTRL + N`을 이 방식으로 설정했습니다.

터미널에서 Vim 모드(Vim Mode)를 켜면 훨씬 편안해집니다. 세 단어 전에서 실수했을 때? `3bcw`면 끝입니다. 전체 삭제는 `cc`, 복잡한 수정은 `v`로 (Neo)Vim에서 바로 열 수 있습니다. 이미 (Neo)Vim을 사용하고 있다면 새로운 단축키를 외울 필요가 없어서 최고입니다.

예전 Emacs 매뉴얼의 한 구절이 떠오릅니다. 복잡한 명령 대신 마우스 설정(Mouse Configuration)을 추천하고 싶습니다. 20년 넘게 (N)Vim을 써왔지만 셸(Shell)의 Vi 모드(Vi Mode)는 싫습니다. 복잡한 명령이 필요할 때는 `CTRL + X, CTRL + E`로 Neovim에서 열어 쓰는 것이 좋은 절충안(Compromise)입니다. 저도 Vim을 자주 쓰지만 셸(Shell)에서는 사용하지 않습니다. 대신 Emacs 스타일 단축키(Emacs-style Shortcuts, `CTRL + A`, `CTRL + E` 등)를 macOS 전역에서 사용합니다. Vim이 삽입 모드(Insert Mode)에서 Emacs 바인딩(Emacs Bindings)을 지원했으면 좋겠다고 생각합니다. "Readline Vi 모드(Readline Vi Mode)"라는 기능 이름을 처음 알게 되었습니다. 더 찾아봐야겠습니다. 왜 Emacs 모드(Emacs Mode)가 기본인지 이해가 안 됩니다. 새 셸(Shell)을 열면 제일 먼저 `set -o vi`를 입력합니다.

어떤 사람이 `#`라는 스크립트(Script)를 `PATH`에 넣어서 파이프라인(Pipeline) 일부를 주석 처리할 수 있게 했다는 점이 인상 깊었습니다.
```sh
#!/bin/sh
cat
```
저도 비슷한 트릭(Trick)을 씁니다. `~/bin/noglob` 파일에
```sh
#!/bin/sh
$*
```
이렇게 해서 Bash에서 Zsh 스크립트(Script)를 호출할 때 `noglob` 충돌을 피합니다. 저는 괄호 주석(Parenthetical Comments)을 씁니다. 스크립트(Script) 문서화에는 좋지만 대화형 셸(Interactive Shell)에서는 번거롭습니다. 그래도 창의적인 주석 방식이라 재미있습니다. `mycmd1 #| mycmd2`와 비교했을 때 어떤 장점이 있는지 궁금합니다. 이건 정말 유용해서 제 `$PATH`에 추가할 예정입니다.

`CTRL + W`는 보통 이전 공백까지 삭제하므로 `/var/log/nginx/` 전체를 지웁니다. `ALT + Backspace`는 비알파벳 문자(Non-alphabetic Characters) 전까지 삭제합니다. 단, 브라우저 탭(Browser Tab)을 닫는 단축키와 겹칠 수 있으니 주의해야 합니다.
Firefox v147부터 단축키 재정의(Remapping) 기능이 추가되었습니다. [관련 링크](https://www.mozilla.org/en-US/firefox/127.0/releasenotes/)
macOS에서는 GUI(Graphic User Interface) 단축키와 터미널 단축키가 구분되어 있어서 이런 문제가 없습니다. `⌘C`, `⌘W` 등이 터미널에서도 동일하게 작동합니다.
제 환경(Fish + Alacritty)에서는 반대로 동작합니다. 그래도 `CTRL + SHIFT + T`로 닫힌 탭(Tab)을 복구할 수 있어서 다행입니다.
`$WORDCHARS` 설정에서 `/`를 제거하면 원하는 동작을 얻을 수 있습니다 ([참고 링크](https://github.com/olets/zsh-git-prompt/issues/72#issuecomment-680459523)).
`CTRL + SHIFT + T`로 탭(Tab)을 바로 복구할 수 있습니다.

FZF 셸(Shell) 통합(Integration)으로 히스토리 검색(History Search)을 업그레이드(Upgrade)하길 추천합니다.
[시연 영상](https://www.youtube.com/watch?v=F_f0h9uFq14) / [공식 문서](https://github.com/junegunn/fzf#shell-integration)

자주 쓰는 트릭(Trick) 하나가 있습니다. 긴 명령을 입력하다가 다른 것을 먼저 해야 할 때, `CTRL + C`로 취소하지 않고 주석 처리 후 실행해서 히스토리(History)에 남깁니다.
```sh
$ long_command  
$ #long_command  
$ stuff_1  
$ stuff_2  
$ #long_command  
$ long_command
```
Zsh에서는 "push-line-or-edit" 바인딩(Binding)으로 가능하고, Bash에서는 `CTRL + U` 후 `CTRL + Y`로 비슷하게 구현 가능합니다. 더 쉬운 방법은 `CTRL + U`로 현재 줄을 저장하고 비운 뒤, `CTRL + Y`로 다시 붙이는 것입니다. Zsh에서는 `ALT + Q`로 이 과정을 자동화할 수 있습니다. Bash에서는 `ALT + SHIFT + 3`으로 현재 명령 앞에 `#`을 붙이고 새 줄로 이동할 수 있습니다.

LLM(Large Language Model) 스타일의 제목은 별로지만, 몇몇 팁(Tip)은 유용해서 시도해볼 예정입니다. 브레이스 확장(Brace Expansion)이 탭 완성(Tab Completion)과 잘 맞지 않는 점이 아쉽습니다. 마지막 토큰(Token) 복제나 파일 확장자 삭제 같은 기능이 더 있었으면 좋겠습니다.

글의 문체 자체도 LLM(Large Language Model) 느낌이 납니다. 예를 들어 "The shell is a toolbox, not an obstacle course." 같은 문장 말이죠. PowerShell에도 Readline 모드(Readline Mode)가 있어서 WSL(Windows Subsystem for Linux)과 병행할 때 전환 부담이 줄어듭니다. Readline은 Bash의 일부처럼 작동하므로, 이런 팁(Tip)들을 셸(Shell) 기능이라고 불러도 무방하다고 생각합니다.

Zsh 설정에서 특정 명령을 히스토리(History)에서 제외하는 함수(Function)를 사용합니다. 예를 들어 `--force` 같은 위험한 명령이 히스토리(History)에 남지 않게 합니다.
```sh
function zshaddhistory() {
  emulate -L zsh
  if ! [[ "$1" =~ "(^ |--force|whatever)" ]] ; then
    print -sr -- "${1%%$'\n'}"
    fc -p
  else
    return 1
  fi
}
```
저도 비슷하게, 기록하지 않을 명령에 공백을 앞에 붙이는 별칭(Alias)을 만들어 두었습니다.
```sh
unhist () { alias $1=" $1"; }
unhist unhist
unhist fzf
unhist rghist
```

Zsh에서 글로벌 별칭(Global Alias) 자동 확장(Auto Expansion)을 구현한 스니펫(Snippet)을 사용합니다. 예를 들어 `alias -G G='rg -s'`로 설정하면, `command | G` 입력 시 자동으로 `command | rg -s`로 확장됩니다. 스페이스(Space) 입력 시 자동으로 확장되며, `\alias`로 입력하면 확장을 건너뛸 수 있습니다.

`.bashrc`에 만든 "Deep CD" 함수(Function)가 있습니다. 인자(Argument)로 받은 문자열을 포함하는 첫 번째 디렉터리(Directory)로 자동 이동합니다.
```sh
dcd() {
  [ -z "$1" ] && return
  local dir
  dir=$(find . -type d -path "*$1*" -print -quit 2>/dev/null)
  [ -n "$dir" ] && cd "$dir"
}
```
처음엔 느릴 줄 알았는데 의외로 자주 사용하게 됩니다.

`autojump`의 `jc`나 `z`, `fzf` 같은 스마트 CD 도구(Smart CD Tool)도 함께 살펴보면 좋습니다.
