# GUI 앱을 대체하는 7가지 Homebrew 도구: 미니멀리스트의 비밀 스택

### Mac이 느리게 느껴지는 이유 (실제로는 그렇지 않더라도)

대부분의 사람들은 성능 문제가 없습니다.
그들은 소프트웨어 문제가 있습니다.

지금 바로 시스템을 살펴보세요:

*   여러 `Electron` 앱 실행 중
*   백그라운드 헬퍼 프로세스
*   메모리를 잡아먹는 탭
*   `GUI` 애니메이션이 모든 것을 느리게 만듭니다.

`활성 상태 보기` (`Activity Monitor`)를 열어보면 더 많은 리소스가 소모되고 있는 것을 발견할 뿐입니다.
이것은 반복되는 악순환입니다.

진정한 해결책은 하드웨어 업그레이드가 아닙니다.
불필요한 레이어를 제거하는 것입니다.

바로 여기서 명령줄(`Command Line`)의 힘이 발휘됩니다.

`Homebrew`를 사용하면 무거운 `GUI` 앱을 가볍고 빠르며 스크립트 가능한 `CLI` 도구로 대체할 수 있습니다.

다음은 제가 시스템을 사용하는 방식을 완전히 바꾼 7가지 도구입니다.

### 1. `활성 상태 보기` (`Activity Monitor`)를 `btop`으로 대체하기

시스템이 느려지면 `활성 상태 보기`를 여는 것이 첫 번째 본능입니다.
하지만 `활성 상태 보기`는 느리고 클릭 위주이며, 아이러니하게도 자체적으로 리소스를 소모합니다.

`btop`은 빠르고 키보드 기반의 인터페이스로 이 문제를 해결합니다.

`btop`은 다음을 보여줍니다:

*   코어별 `CPU` 사용량
*   메모리 및 스왑
*   디스크 `I/O`
*   네트워크 활동

모든 것을 실시간으로 보여줍니다.
프로세스를 즉시 필터링하고 단일 키로 종료할 수 있습니다.

```bash
brew install btop
```

**사용법**

```bash
btop
```

**주요 단축키:**

*   `f` → 프로세스 필터링
*   `k` → 프로세스 종료
*   `q` → 종료

**더 나은 이유:** 빠르고 시각적이며 마우스가 필요 없음
**최적의 사용 사례:** 실시간 시스템 모니터링

### 2. `Finder`를 `yazi`로 대체하기

`Finder`는 가벼운 용도로는 작동합니다.
하지만 진지한 파일 관리에는 너무 느려집니다.

`yazi`는 속도를 위해 구축된 최신 터미널 파일 관리자입니다.

`yazi`는 다음 기능을 제공합니다:

*   다중 창 탐색
*   즉각적인 미리 보기 (이미지, `PDF`, 텍스트)
*   퍼지 검색 (`Fuzzy search`)
*   키보드 우선 워크플로

폴더를 클릭하여 이동하는 대신, 원하는 위치로 바로 이동할 수 있습니다.

```bash
brew install yazi
```

**사용법**

```bash
yazi
```

**기본 제어:**

*   화살표 키 / `j k` → 탐색
*   `Enter` → 폴더/파일 열기
*   `q` → 종료
*   `z` → 디렉토리로 이동

**더 나은 이유:** 더 빠른 탐색 + 즉각적인 미리 보기
**최적의 사용 사례:** 프로젝트 및 설정 파일 관리

### 3. `미리보기` (`Preview`) / `ImageOptim`을 `imagemagick`으로 대체하기

간단한 편집을 위해 `GUI` 도구를 여는 대신 `imagemagick`을 사용하세요.

**설치**

```bash
brew install imagemagick
```

**사용법**

모든 `PNG`를 `JPG`로 변환:

```bash
mogrify -format jpg *.png
```

이미지 크기 조정:

```bash
convert input.png -resize 800x600 output.png
```

이미지 압축:

```bash
convert input.png -quality 70 output.jpg
```

**더 나은 이유:** 몇 초 만에 일괄 작업 (`Batch operations`) 가능
**최적의 사용 사례:** 이미지 자동화 워크플로

### 4. `VS Code` (속도를 위해)를 `neovim`으로 대체하기

`VS Code`는 강력하지만 무겁습니다.
`neovim`은 가볍고 매우 빠릅니다.

**설치**

```bash
brew install neovim
```

**사용법**

파일 열기:

```bash
nvim file.txt
```

**기본 제어:**

*   `i` → 입력 모드 (`insert mode`)
*   `Esc` → 일반 모드 (`normal mode`)
*   `:w` → 저장
*   `:q` → 종료
*   `:wq` → 저장 및 종료

**전문가 팁:** 완전한 `IDE` 경험을 위해서는 `LazyVim` 또는 `NvChad`를 설치하세요.

**더 나은 이유:** 즉각적인 시작 + 키보드 기반
**최적의 사용 사례:** 빠른 코딩 및 편집

### 5. `Spotify 앱`을 `ncspot`으로 대체하기

`Spotify` 데스크톱 앱은 리소스 소모가 많습니다.
`ncspot`은 터미널에서 최소한의 리소스만으로 실행됩니다.

**설치**

```bash
brew install ncspot
```

**사용법**

```bash
ncspot
```

**기본 제어:**

*   `/` → 검색
*   `Enter` → 재생
*   `n` → 다음 트랙
*   `Space` → 일시 정지/재생

**참고:** `Spotify Premium`이 필요합니다.

**더 나은 이유:** 가볍고 + 방해 요소 없음
**최적의 사용 사례:** 백그라운드 음악 감상

### 6. 빠른 검색을 `ddgr`로 대체하기

간단한 검색을 위해 브라우저를 열면 다음으로 이어집니다:

*   여러 탭
*   주의 산만
*   메모리 사용량

`ddgr`을 사용하면 터미널에서 `DuckDuckGo`를 직접 검색할 수 있습니다.

```bash
ddgr "how to install docker"
```

깔끔하게 번호가 매겨진 결과를 즉시 얻을 수 있습니다.
필요할 때만 브라우저를 여세요.

```bash
brew install ddgr
```

**더 나은 이유:** 광고 없음, 추적 없음, 탭 과부하 없음
**최적의 사용 사례:** 빠른 문서 조회

### 7. `Postman`을 `curlie`로 대체하기

`Postman` 및 유사 도구들은 비대해졌습니다.
계정, 동기화, `UI` 오버헤드 — 간단한 `API` 호출을 위해서 말이죠.

`curlie`는 터미널에서 깔끔하고 형식화된 `API` 응답을 직접 제공합니다.

```bash
brew install curlie
```

**예시:**

```bash
curlie https://api.github.com/users/octocat
```

**더 나은 이유:** 빠르고, 스크립트 가능하며, `UI` 오버헤드 없음
**최적의 사용 사례:** `API` 테스트 및 자동화

### 이 스택이 뛰어난 이유

이러한 변화는 단순히 "미니멀리스트"가 되는 것에 관한 것이 아닙니다.
이는 지연 시간(`latency`)과 제어(`control`)에 관한 것입니다.

`GUI` 앱은 다음 레이어를 추가합니다:

*   렌더링
*   애니메이션
*   이벤트 처리

`CLI` 도구는 이 모든 것을 제거합니다.

그 결과:

*   더 빠른 실행
*   더 낮은 메모리 사용량
*   더 나은 자동화
*   더 높은 집중도

더 이상 소프트웨어를 기다릴 필요가 없습니다.
소프트웨어는 즉시 당신에게 반응하기 시작합니다.

### 시작하는 방법

`Homebrew`가 설치되어 있지 않다면:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

그다음에는 작게 시작하세요.
모든 것을 한 번에 대체하지 마세요.

다음 중 하나로 시작해 보세요:

*   모니터링을 위한 `btop`
*   탐색을 위한 `yazi`

속도 차이를 느끼면 나머지는 자연스럽게 따라올 것입니다.

### 마지막 생각

대부분의 사람들은 더 많은 것을 함으로써 생산성을 향상시키려고 합니다.
더 나은 도구, 더 많은 앱, 더 많은 시스템.

하지만 진정한 향상은 마찰을 제거하는 데서 옵니다.

이러한 도구들은 복잡성을 추가하지 않습니다.
복잡성을 제거합니다.
