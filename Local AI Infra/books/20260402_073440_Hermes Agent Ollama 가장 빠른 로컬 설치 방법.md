# Hermes Agent + Ollama: 가장 빠른 로컬 설치 방법

OpenClaw 출시 이후 거의 매주 새로운 에이전트가 등장하고 있어 이를 모두 확인하는 것은 불가능에 가깝습니다. 하지만 최근 Nous Research에서 개발한 **Hermes Agent**는 많은 이들의 주목을 받고 있습니다.

Hermes Agent는 메모리 점유율 경쟁보다는 성능에 초점을 맞춘 오픈 소스 에이전트 프레임워크로, 이 분야에서 OpenClaw의 실질적인 유일한 경쟁자로 평가받습니다. 특히 AI의 고질적인 문제인 '기억상실' 문제를 근본적으로 해결하는 데 집중하고 있습니다.

## Hermes Agent의 핵심 특징

### 1. 자동 기술 학습 (Skill-Learning Automated)
Hermes의 가장 강력한 무기는 사용하면 할수록 똑똑해진다는 점입니다. 예를 들어 사용자가 "Docker를 사용하여 서비스를 배포해 줘"라고 요청하면, Hermes는 작업을 수행한 후 그 단계를 '기술(Skill)'로 자동 저장합니다. 다음에 유사한 요청을 받으면 이전 경험을 바탕으로 더 빠르고 정확하게 수행합니다. 

사람이 직접 기술을 작성하고 유지 관리해야 하는 기존 방식과 달리, Hermes는 스스로 학습하고 오류를 수정하며 발전합니다.

### 2. 3계층 메모리 구조 (Three-Layer Structure of Memory)
대부분의 AI는 대화가 끝나면 내용을 잊어버리지만, Hermes는 세 가지 계층으로 기억을 관리합니다.
*   **MEMORY.md:** 에이전트가 작성하는 개인 메모로, 환경, 프로젝트 정보, 사용자 선호도를 기록합니다.
*   **USER.md:** 사용자의 이름, 시간대, 소통 스타일 등 프로필 정보를 담습니다.
*   **세션 검색 (FTS5):** 모든 과거 대화를 SQLite에 저장하여, 몇 주 전의 대화 내용도 검색하고 답변할 수 있습니다.

### 3. 12개 채널의 광범위한 플랫폼 지원
텔레그램(Telegram), 디스코드(Discord), 슬랙(Slack), 왓츠앱(WhatsApp), CLI 등 총 12개 채널을 지원합니다. 아침에는 텔레그램으로 지시하고 밤에는 디스코드에서 진행 상황을 확인하는 식의 끊김 없는(Seamless) 상호작용이 가능하며, 모든 플랫폼에서 동일한 메모리를 공유합니다.

## OpenClaw vs. Hermes Agent

*   **Hermes Agent가 유리한 경우:** 개인용 비서로 폭넓게 사용하고 싶을 때, 반복적인 작업을 자동화하고 싶을 때, 음성 기능이 필요할 때, 가볍고 매끄러운 실행(20MB vs 200MB 이상)을 원할 때 적합합니다.
*   **OpenClaw가 유리한 경우:** 여러 AI 에이전트를 연동할 때, 대규모 커뮤니티의 지원이 필요할 때, 고도화된 브라우저 자동화나 다양한 플러그인이 필요할 때 유리합니다.

결론적으로 두 에이전트는 공존이 가능합니다. OpenClaw를 전체 시스템을 관리하는 사령관으로 사용하고, Hermes Agent를 1:1로 소통하며 학습시키는 개인 맞춤형 조언자로 활용하는 전략이 효과적입니다.

## 설치 및 설정 방법 (Ollama 활용)

Hermes Agent는 단일 명령어로 간편하게 설치할 수 있습니다.

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

설치 후 셋업 위저드(Setup Wizard)를 통해 다음 과정을 진행합니다.

1.  **추론 제공자(Inference Provider) 설정:** API 비용 없이 완전 로컬 환경을 구축하기 위해 `Custom OpenAI-compatible endpoint`를 선택합니다.
2.  **Ollama 연동:** 베이스 URL을 `http://localhost:11434/v1`로 설정합니다. API 키는 비워두고, Ollama에 이미 로드된 모델(예: `gpt-oss:20b`)을 지정합니다.
3.  **에이전트 설정 (Agent Settings):** 최대 반복 횟수(Max iterations), 도구 진행 상황 표시 여부, 컨텍스트 압축(Context Compression) 비율 등을 설정합니다.
4.  **세션 초기화 정책:** 대화가 너무 길어지거나 비활성 상태가 지속될 때 자동으로 대화를 정리하도록 설정할 수 있습니다. (기본값: 24시간 비활성 시 초기화)
5.  **도구 구성 (Tools Configuration):** 웹 검색, 브라우저 제어, 터미널 액세스, 파일 처리, 코드 실행 기능 등을 활성화합니다. 브라우저 자동화에는 별도 설정이 필요 없는 `Local Browser(Headless Chromium)`를 추천합니다.

설치가 완료된 후 터미널에 `hermes`를 입력하여 실행할 수 있습니다. 만약 기본 모델이 Claude로 설정되어 있다면, `hermes model` 명령어를 통해 로컬 Ollama 엔드포인트로 빠르게 변경할 수 있습니다.

## 총평

현재 Hermes Agent는 v0.2.0 단계로, 메모리 정확도나 기술 생성 측면에서 아직 다듬어지지 않은 부분이 있습니다. 그러나 기존 AI 도구들의 고질적인 문제인 '맥락의 단절'을 정면으로 해결하려는 시도는 매우 가치 있습니다.

매번 대화를 시작할 때마다 프로젝트 배경이나 규칙을 설명할 필요 없이, 나를 이해하고 함께 성장하는 '기억'을 가진 AI 어시스턴트라는 점에서 차세대 AI의 방향성을 잘 보여주고 있습니다. 특히 오픈 소스로서 데이터 주권을 사용자가 가질 수 있다는 점이 큰 장점입니다.
