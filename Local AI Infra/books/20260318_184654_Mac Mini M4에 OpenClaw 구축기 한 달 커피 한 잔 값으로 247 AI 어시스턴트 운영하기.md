저는 값비싼 클라우드 서버(cloud server) 없이 24시간 내내 작동하는 AI 어시스턴트를 원했습니다. 텔레그램(Telegram), 왓츠앱(WhatsApp), 디스코드(Discord)와 같은 실제 메시징 앱과 연동되고, 각 메시지에 가장 적합한 AI 모델을 자동으로 선택하며, 큰 비용 부담 없이 이미지 이해(image understanding) 기능을 지원하는 어시스턴트 말이죠.

그 해답은 Mac Mini M4에서 OpenRouter의 자동 라우터(auto router)로 구동되는 OpenClaw였습니다.

수 주간의 튜닝, 디버깅, 최적화 끝에 제가 배운 모든 것을 여기에 정리했습니다.

## OpenClaw란 무엇인가요?

OpenClaw (이전 명칭: Clawdbot)는 오픈 소스(open-source) 개인 AI 어시스턴트 게이트웨이입니다. 이를 메시징 앱과 AI 모델 사이의 다리라고 생각하시면 됩니다.

작동 방식은 간단합니다. 텔레그램(Telegram), 왓츠앱(WhatsApp), 디스코드(Discord), 슬랙(Slack), 시그널(Signal) 등 10개 이상의 지원 플랫폼 중 하나에서 메시지를 보내면, OpenClaw가 Mac Mini에서 메시지를 수신하고 OpenRouter를 통해 AI 제공업체로 전달한 다음, 응답을 다시 채팅으로 보냅니다.

이 시스템은 이미지, 오디오, 파일, 비디오 입력을 지원합니다. 대화 간의 맥락(context)을 기억하는 메모리 시스템(memory system)이 있으며, 웹 검색, 이미지 생성, 파일 처리(file handling)를 위한 내장 도구(built-in tools)를 제공합니다. 구글 플레이스 검색(Google Places search), 위스퍼(Whisper)를 통한 음성 전사(voice transcription), 로컬 텍스트-투-스피치(local text-to-speech)와 같은 사용자 정의 기능(custom capabilities)을 위한 스킬 시스템(skills system)도 갖추고 있습니다. 심지어 macOS, iOS, 안드로이드(Android)용 네이티브 앱(native apps)도 있습니다.

## Mac Mini M4를 선택한 이유

제 설정은 기본 모델로, 16GB 통합 메모리(unified memory), 256GB SSD, 약 500달러의 일회성 비용이 들었습니다.

제가 라즈베리 파이(Raspberry Pi)나 클라우드 서버(cloud server) 대신 Mac Mini를 선택한 이유는 몇 가지가 있습니다. 유휴(idle) 상태에서 약 5와트(watt)만 소모하여 월 전기 요금이 약 0.50달러에 불과합니다. 이는 야간등보다도 적은 비용이죠. 애플 실리콘(Apple Silicon) 칩은 `sherpa-onnx`를 통해 로컬 텍스트-투-스피치를 실행하고 미디어 파일을 처리하기에 충분히 빠릅니다. macOS는 잠자기(sleep) 및 깨우기(wake) 기능을 원활하게 처리하며, OpenClaw는 문제가 발생해도 자동으로 복구됩니다.

또한 개인용 무료 VPN(Virtual Private Network)인 테일스케일(Tailscale)을 설치하여 어디서든 Mac Mini에 안전하게 접근할 수 있습니다. 월별 서버 비용도 들지 않습니다. 하드웨어를 한 번 구매하면 영원히 제 소유가 됩니다.

AI 모델을 로컬(local)에서 직접 실행하는 것이 아니라 클라우드 제공업체로 요청을 라우팅(routing)하는 것이기 때문에 16GB 모델로도 충분합니다.

## OpenRouter란 무엇이며 왜 중요한가요?

이것이 전체 설정에서 가장 중요한 부분이므로 제대로 설명해 드리겠습니다.

OpenRouter는 앤스로픽(Anthropic), OpenAI, 구글(Google), 딥시크(DeepSeek), 미스트랄(Mistral), xAI, 문샷(Moonshot) 등 모든 주요 제공업체의 300개 이상의 AI 모델에 단일 API 키(API key)로 접근할 수 있도록 해주는 통합 API 게이트웨이(unified API gateway)입니다. 각 제공업체에 개별 계정을 만들고 여러 API 키를 관리하는 대신, 모든 모델에 사용할 수 있는 하나의 키를 얻게 됩니다.

하지만 진짜 마법은 자동 라우터(auto router)에 있습니다. 모델을 `openrouter/auto`로 설정하면, NotDiamond가 제공하는 메타 모델(meta-model)을 사용하게 되는데, 이 모델은 모든 들어오는 프롬프트(prompt)를 분석하여 최적의 모델로 라우팅합니다. 프롬프트의 복잡성, 작업 유형, 비전 지원(vision support)과 같은 필수 기능, 각 모델의 강점을 고려합니다.

날씨에 대한 간단한 질문에는 작고 저렴한 모델을 선택합니다. 코딩 작업이라면 코드스트랄(Codestral)로 라우팅할 수 있습니다. 복잡한 추론에는 딥시크 R1(DeepSeek R1)을 사용하고, 사진을 보내면 자동으로 시각(vision) 처리 능력이 있는 모델을 선택합니다.

선택된 모델의 가격만 지불하며, 자동 라우팅(auto routing) 자체에 대한 추가 라우팅 수수료는 없습니다. 그리고 여기에 똑똑한 부분이 있습니다. 자동 라우터는 기본 균형 모드(default balanced mode)에서 역제곱 가격 책정 알고리즘(inverse-square pricing algorithm)을 사용합니다. 이는 백만 토큰(token)당 1달러인 제공업체가 3달러인 제공업체보다 9배 더 선택될 가능성이 높다는 의미입니다. 시스템은 작업을 잘 처리할 수 있는 더 저렴한 모델을 자연스럽게 선호합니다.

이것이 전체 설정을 매우 비용 효율적으로 만드는 이유입니다. 하나의 프리미엄 모델에 대해 고정 요금을 지불하는 것이 아니라, 라우터가 각 질문에 대해 적절하게 답변할 수 있는 가장 저렴한 모델을 찾아주기 때문에 메시지당 몇 센트의 비용만 지불하게 됩니다.

## 엄선된 9개 모델 풀(Pool) 구축

기본적으로 자동 라우터는 클로드 오푸스 4.6(Claude Opus 4.6, 백만 토큰당 5/25달러) 또는 GPT-5.2 프로(GPT-5.2 Pro, 백만 토큰당 21/168달러)와 같은 비싼 모델을 포함하여 약 25개의 모델 중에서 선택할 수 있습니다. 라우터가 간단한 질문에 이러한 모델을 선택하면 비용이 폭증하게 됩니다.

해결책은 OpenRouter 설정(Settings)으로 이동하여 자동 라우터 플러그인(Auto Router plugin)을 찾고, `allowed_models` 설정을 비용 효율적인 모델만 포함하도록 지정하는 것입니다.

저는 9개의 모델로 압축했습니다.

다음 6개는 시각(vision) 처리 능력이 있습니다.
*   클로드 하이쿠 4.5(Claude Haiku 4.5, 백만 토큰당 1.00/5.00달러)
*   제미니 2.5 플래시 라이트(Gemini 2.5 Flash Lite, 0.08/0.30달러)
*   GPT-5 미니(GPT-5 Mini, 0.40/1.60달러)
*   GPT-5 나노(GPT-5 Nano, 0.15/0.60달러)
*   키미 K2.5(Kimi K2.5, 0.23/3.00달러)
*   미스트랄 라지(Mistral Large, 0.50/1.50달러)

다음 3개는 텍스트 전용 전문가 모델입니다.
*   코딩을 위한 코드스트랄 2508(Codestral 2508, 0.15/0.45달러)
*   추론을 위한 딥시크 R1(DeepSeek R1, 0.70/2.50달러)
*   저렴한 Q&A를 위한 그록 3 미니(Grok 3 Mini, 0.30/0.50달러)

시각 처리 능력을 갖춘 모델의 비율 67%는 매우 중요합니다. 이미지를 보낼 때 자동 라우터는 시각 처리 능력이 있는 모델을 필요로 합니다. 만약 모델 풀(pool)의 대부분이 텍스트 전용이라면, "404 No endpoints found that support image input"이라는 치명적인 오류가 발생할 것입니다. 모델의 3분의 2를 시각 처리 가능하도록 유지하면 라우터가 사진과 스크린샷에 적합한 모델을 거의 항상 찾을 수 있습니다.

## 텔레그램(Telegram), 보안 및 세션 관리

텔레그램은 설정하기 가장 쉬운 채널이었습니다. 봇파더(BotFather)를 통해 봇을 생성하고 토큰(token)을 받아 설정(config)에 추가했습니다. 보안을 위해 `dmPolicy`를 "pairing"으로 설정하고, 제가 승인한 그룹에서만 봇이 응답하도록 `groupPolicy`를 "allowlist"로 설정했습니다.

보안을 위해 `elevatedDefault`를 "ask"로 설정했습니다. 즉, AI는 특권 명령(privileged commands)을 실행하기 전에 승인을 요청해야 합니다. 처음에는 "full"(제한 없음)로 시작했지만, 위험성을 빠르게 깨달았습니다. 절대 그렇게 하지 마세요. 또한 `node`, `git`, `ls`, `cat`, `mkdir`, `open`과 같은 신뢰할 수 있는 명령들을 포함하는 명령 허용 목록(command allowlist)을 설정했습니다. 알 수 없는 명령은 텔레그램에서 저의 승인 프롬프트(prompt)를 띄웁니다.

세션(session)은 2시간 동안 활동이 없으면 자동으로 재설정되지만, 메모리 시스템(memory system)은 중요한 맥락을 세션 간에 저장합니다. 대화가 길어지면 4,000 토큰(token) 임계값(threshold)에서 압축(compaction) 기능이 기록을 자동으로 요약합니다.

## 솔직한 비용 분석 — 실제 수치

저는 "무료 AI!"를 약속하면서 실제 비용은 숨기는 블로그 게시물을 싫어합니다. 여기 제 실제 비용 내역이 있습니다.

고정 월별 비용은 거의 없습니다. Mac Mini 전기 요금은 약 0.50달러입니다 (유휴 시 5W, 월 3.6 kWh, kWh당 0.14달러 기준). 테일스케일(Tailscale)은 개인용으로 무료입니다. 텔레그램 봇 API(Telegram Bot API)도 무료입니다. 브레이브 검색 API(Brave Search API)는 월 2,000쿼리(query)의 무료 티어(free tier)를 제공합니다. 총 고정 비용은 월 약 0.50달러입니다.

변동 비용은 API 사용량이며, 여기서 자동 라우터가 진가를 발휘합니다. 제 설정에서 각 작업에 실제로 드는 비용은 다음과 같습니다.

*   "오늘 날씨는?"과 같은 간단한 질문은 GPT-5 나노(GPT-5 Nano)로 라우팅되어 약 0.0005달러가 듭니다.
*   그록 3 미니(Grok 3 Mini)를 통해 단락을 한국어로 번역하는 데 약 0.0008달러가 듭니다.
*   코드스트랄(Codestral)을 통해 파이썬(Python) 함수를 작성하는 데 약 0.001달러가 듭니다.
*   클로드 하이쿠 4.5(Claude Haiku 4.5)를 통해 스크린샷에 대해 질문하는 데 약 0.006달러가 듭니다.
*   딥시크 R1(DeepSeek R1)을 통한 긴 디버깅(debugging) 세션은 약 0.008달러가 듭니다.

월별 총 비용은 사용량에 전적으로 달려있습니다. 월 약 100개 메시지의 가벼운 사용은 총 1~2달러가 들고, 월 약 300개 메시지의 보통 일일 사용은 3~6달러가 듭니다. 월 500~800개 메시지의 헤비 파워 유저(power user) 수준은 6~16달러, 월 1,000개 이상의 메시지를 사용하는 극단적인 개발 어시스턴트(dev-assistant) 사용은 16~31달러가 듭니다.

단일 모델을 사용하는 것과 비교했을 때 자동 라우터가 비용을 크게 절감하는 이유는 다음과 같습니다. 제 트래픽(traffic)의 약 60%는 GPT-5 나노(GPT-5 Nano) 또는 제미니 플래시 라이트(Gemini Flash Lite)와 같은 저렴한 모델로 라우팅되는 간단한 메시지이며, 각각 약 0.0005달러가 듭니다. 약 25%는 그록 3 미니(Grok 3 Mini) 또는 코드스트랄(Codestral)로 라우팅되는 중간 난이도 메시지이며, 각각 약 0.001달러가 듭니다. 클로드 하이쿠(Claude Haiku) 또는 딥시크 R1(DeepSeek R1)이 필요한 복잡한 메시지는 약 15%에 불과하며, 각각 약 0.005달러가 듭니다.

자동 라우팅(auto routing)이 없었다면, 클로드 소네트 4.5(Claude Sonnet 4.5)를 모든 메시지에 사용했을 경우 300개 메시지에 대한 API 수수료만 약 2.70달러가 들었을 것입니다. 엄선된 9개 모델 풀과 자동 라우팅을 사용한 결과, 동일한 300개 메시지에 약 0.50~2.00달러가 들었습니다. 이는 탁월한 응답 품질을 유지하면서 50~80%의 비용 절감을 달성한 것입니다.

## 경험에서 얻은 팁

모델 풀(pool)의 최소 60~70%는 시각(vision) 처리 능력을 갖추도록 유지하세요. 기본 대체 모델(default fallback model)은 제미니 3 플래시 프리뷰(Gemini 3 Flash Preview)와 같이 안정적이고 저렴한 시각 모델로 설정하세요. "기본 (균형)"(`Default (balanced)`) 제공업체 정렬을 사용하세요. 이 방식의 역제곱 알고리즘(inverse-square algorithm)은 자연스럽게 비용을 최적화합니다.

`elevatedDefault: "full"`로 시작하지 마세요. 몇 초의 승인 시간은 보안을 위해 충분히 가치 있습니다. 설정(config) 변경 후 텔레그램 채팅에서 `/new`를 실행하세요. OpenRouter의 활동 대시보드(Activity dashboard)를 주간 단위로 확인하여 어떤 모델이 요청을 처리하는지, 각 모델의 비용은 얼마인지 모니터링(monitoring)하세요.

`~/.openclaw/media/`에 있는 2분 미디어 정리 기능은 버그(bug)가 아니라 의도된 기능(feature)입니다. 스크린샷과 같은 임시 파일 공유에 완벽합니다. 중요한 파일은 다른 곳에 저장하세요.

## 수 주간의 튜닝을 통해 얻은 것
