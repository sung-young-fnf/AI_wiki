# 로컬에서 Gemma 4 미세 조정하기: 상세 가이드

최근 Google이 Gemma 4를 출시했으며, 그 벤치마크 결과는 인상적입니다. AIME 2026 수학 점수는 20.8%에서 89.2%로, LiveCodeBench 코딩 점수는 29.1%에서 80.0%로 크게 향상되었습니다. 이는 단순한 개선이 아니라, 완전히 다른 수준의 모델을 의미합니다. Gemma 4는 네 가지 크기로 제공되며, 모두 Apache 2.0 라이선스 하에 배포됩니다. 텍스트, 이미지, 오디오를 지원하는 멀티모달(multimodal) 모델이며, 네 가지 모델 모두 로컬에서 미세 조정(fine-tune)할 수 있습니다. 제가 진행한 과정은 다음과 같습니다.

### 먼저 모델을 선택하세요

무엇보다 먼저, 사용 가능한 하드웨어를 파악해야 합니다.

E2B와 E4B는 휴대폰, 노트북, 라즈베리 파이(Raspberry Pi)와 같은 기기에서 엣지(edge) 배포를 위해 설계되었습니다. E2B는 학습을 위해 8GB의 VRAM을 필요로 하며, E4B는 10GB를 필요로 합니다. 일반적인 게이밍 GPU(gaming GPU)를 가지고 있다면 좋은 시작점입니다.

26B-A4B는 속도와 품질 사이에서 좋은 균형을 제공하는 전문가 혼합(Mixture-of-Experts, MoE) 모델입니다. A100 GPU가 필요합니다.

31B는 최상위 모델입니다. 메모리 제약이 문제가 되지 않고 최고 품질의 출력을 원할 때 선택하세요. 이 모델 또한 A100 GPU가 필요합니다.

E2B 및 E4B 모델용 무료 Google Colab 노트북이 제공되므로, 로컬 설정 없이도 완전히 학습을 진행할 수 있습니다.

### Unsloth를 선택한 이유

처음에는 표준 학습 방식을 사용했지만, 속도가 느리고 GPU 메모리가 자주 부족했습니다. Unsloth는 Gemma 4 모델을 1.5배 빠르게 학습시키면서 VRAM 사용량을 60% 절감합니다. 기업용 하드웨어가 없는 사용자에게 이 차이는 학습 가능 여부를 결정합니다.

여기서 숫자가 중요합니다:

*   E2B LoRA: 8–10GB VRAM으로 작동
*   E4B LoRA: 17GB 필요
*   31B QLoRA: 22GB로 작동

Unsloth Studio는 시각적인 인터페이스를 제공합니다. 파이썬(Python)을 사용하고 싶지 않다면 사용하지 않아도 됩니다.

### 1단계: Unsloth Studio 설치

macOS / Linux / WSL:

```bash
curl -fsSL https://unsloth.ai/install.sh | sh
```

Windows PowerShell:

```powershell
irm https://unsloth.ai/install.ps1 | iex
```

약 1~2분 소요됩니다.

### 2단계: 실행

```bash
unsloth studio -H 0.0.0.0 -p 8888
```

브라우저에서 `http://localhost:8888`을 엽니다. 초기 실행 시 비밀번호를 설정해야 합니다. 그 후 온보딩(onboarding) 마법사가 나타나는데, 이미 원하는 바를 잘 알고 있다면 건너뛸 수 있습니다.

### 3단계: 모델 및 데이터셋 선택

모델 바에서 "Gemma 4"를 검색합니다. 원하는 버전을 선택하세요.

학습 방식:

*   E4B: LoRA 16비트
*   제한된 VRAM에서 더 큰 모델: QLoRA

HuggingFace에서 데이터셋을 가져오거나 직접 업로드한 후, 컨텍스트 길이(context length), 학습률(learning rate), 하이퍼파라미터(hyperparameters)를 적절히 조정하세요.

### 4단계: 데이터셋을 올바르게 포맷하기

이 섹션은 대부분이 인식하는 것보다 더 중요합니다. Gemma 4는 네 가지 특정 포맷팅 규칙을 따릅니다.

1.  표준 채팅 역할(chat roles) 사용: `system`, `user`, `assistant`. 특이한 것은 사용하지 마세요.

2.  사고 모드(thinking mode)는 명시적입니다. 시스템 프롬프트(system prompt) 시작 부분에 `<|think|>`를 추가하여 활성화합니다. 사고 모드에는 `gemma-4-thinkingchat` 템플릿을, 표준 모드에는 `gemma-4`를 사용하세요. 동일한 데이터셋 내에서 두 가지를 혼합하지 마세요.

3.  다중 턴(multi-turn) 대화: 기록에는 최종 답변만 표시되도록 합니다. 이전 추론이나 응답을 후속 턴에 포함하지 마세요.

4.  멀티모달 프롬프트(multimodal prompts): 항상 이미지 또는 오디오를 텍스트 명령어 앞에 두세요.

다음은 비전(vision) 예시입니다:

```json
{
  "messages": [
    {
      "role": "user",
      "content": [
        {"type": "image", "image": "/path/to/image"},
        {"type": "text", "text": "Describe this image"}
      ]
    },
    {
      "role": "assistant",
      "content": [
        {"type": "text", "text": "This image shows..."}
      ]
    }
  ]
}
```

항상 이미지 먼저, 그 다음에 텍스트입니다.

### 5단계: 학습

"Start Training"을 클릭합니다.

손실(loss) 및 경사 노름(gradient norm) 업데이트를 실시간으로 모니터링하세요. 지속적으로 감소하는 손실은 건강한 학습 과정을 나타냅니다.

한 가지 주목할 점: E2B 및 E4B 멀티모달 모델은 13-15의 손실을 보이며, 이는 정상입니다. 당황할 필요가 없습니다. 반면, 26B 및 31B 파라미터를 가진 텍스트 전용 모델은 1-3으로 더 낮은 손실을 가집니다.

손실 값이 100 또는 300과 같이 나타나면 경사 축적(gradient accumulation) 문제가 있음을 나타냅니다. Unsloth는 이를 자동으로 관리하므로, 이러한 값을 본다면 구성 오류(misconfiguration)를 의미합니다.

학습이 완료되면 모델은 자동으로 저장됩니다.

### 6단계: 내보내기

세 가지 내보내기 형식:

*   **Full merged 16비트** — 추론(inference) 준비 완료, 바로 사용 가능
*   **LoRA 어댑터 파일(adapter files)만** — 더 작고, 별도로 로드
*   **GGUF** — llama.cpp, Ollama, LM Studio용

미세 조정된 모델을 기본 버전과 비교하려면 비교 모드(Compare Mode)를 사용하세요. 배포 전에 이 작업을 수행하는 것이 중요합니다.

### 실제로 도움이 되는 것들

직접적인 답변과 함께 추론 스타일 예시를 포함하여 추론 능력(reasoning skills)을 유지하세요. 데이터셋의 최소 75%가 추론 예시로 구성되도록 하세요. 26B 및 31B 모델의 경우 `gemma-4-thinking` 채팅 템플릿(chat template)을 사용하세요.

E4B는 일반적으로 E2B보다 성능이 뛰어납니다. E2B LoRA 대신 E4B QLoRA를 사용하세요. E4B가 더 크지만, 양자화 정확도(quantization accuracy)의 최소한의 차이로 인해 E4B가 시작하기에 더 나은 기준선(baseline)입니다.

내보낸 모델이 원본보다 성능이 좋지 않다면, 추론 중 잘못된 채팅 템플릿을 사용했기 때문일 가능성이 높습니다. 학습에 사용한 것과 동일한 템플릿을 사용하고 있는지 확인하세요. 이는 흔한 실수입니다.

### 이를 통해 실제로 할 수 있는 일

Gemma 4는 140개 언어를 지원합니다. 다국어 프로젝트의 경우, E2B 및 E4B 버전을 사용하면 한 번 튜닝하여 클라우드에 의존하지 않고 휴대폰과 노트북에 배포할 수 있습니다.

31B는 최고 품질의 출력이 필요하고 하드웨어가 이를 처리할 수 있을 때 이상적입니다. 법률, 의료 및 도메인별(domain-specific) 작업과 같은 전문화된 모델에서 탁월한 성능을 발휘합니다.

전체 노트북 컬렉션은 `unsloth.ai/docs/models/gemma-4/train`에서 확인할 수 있습니다. 위 내용은 해당 페이지에서 더 자세히 다루고 있습니다.
