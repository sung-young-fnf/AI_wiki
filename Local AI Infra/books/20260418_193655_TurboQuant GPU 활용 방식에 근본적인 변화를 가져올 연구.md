# TurboQuant: GPU 활용 방식에 근본적인 변화를 가져올 연구

Google의 최신 논문은 재훈련 없이, 거의 제로에 가까운 정확도 손실로 LLM (대규모 언어 모델) 메모리를 6배 압축하며, 이론적으로 거의 최적에 가깝습니다.

몇 달에 한 번씩, 머신러닝 (ML) 연구 분야에 문제에 대한 생각을 조용히 재정비하는 무언가가 등장합니다. TurboQuant는 바로 그런 논문 중 하나입니다. 2025년 4월 arXiv에 게재되고 ICLR 2026에 채택된 Google Research의 TurboQuant는 프로덕션 LLM 추론에서 가장 고통스러운 병목 현상 중 하나인 KV 캐시 (Key-Value cache)를 다룹니다.

### 아무도 충분히 이야기하지 않는 문제

트랜스포머 (Transformer)가 토큰을 생성할 때, 이전에 계산된 키-값 쌍을 KV 캐시라고 불리는 곳에 저장합니다. 이는 본질적으로 대화를 위한 모델의 작업 메모리입니다. 이는 모든 순방향 전달 (forward pass)에서 이전 토큰에 대한 어텐션 (attention)을 재계산하는 것을 피하기 위해 존재합니다.

문제는 무엇일까요? KV 캐시는 시퀀스 길이 (sequence length)에 따라 선형적으로 증가하며, 긴 컨텍스트 (context)에서는 모델 가중치 (model weights)가 아닌 주요 메모리 소비자가 됩니다. 128K 컨텍스트 윈도우를 가진 LLaMA-3.1-8B와 같은 모델의 경우, KV 캐시의 메모리 점유율이 모델 자체를 압도할 수 있습니다.

| 프롬프트 길이 | KV 캐시 크기 (16비트, 8B 모델) |
|---------------|----------------------------------|
| 4K 토큰       | ~0.5 GB                          |
| 32K 토큰      | ~4.0 GB                          |
| 128K 토큰     | ~16.0 GB ← 병목 지점             |

기존의 양자화 (quantization)는 도움이 되지만, 문제는 대부분의 양자화기가 값당 1~2비트의 추가 오버헤드를 발생시키는 블록별 정규화 상수 (스케일, 제로 포인트)를 저장해야 한다는 것입니다. 낮은 비트 폭에서는 이러한 오버헤드가 압축 이득의 상당 부분을 잠식합니다.

### TurboQuant의 실제 작동 방식

TurboQuant는 2단계 데이터 불인식 (data-oblivious) 벡터 양자화 (vector quantization) 알고리즘입니다. "데이터 불인식"이란 보정 데이터 (calibration data), 훈련 (training), 데이터셋별 튜닝 (tuning)이 필요 없음을 의미합니다. 온라인으로 작동하며, 벡터가 도착하는 즉시 양자화할 수 있습니다.

**아키텍처 개요**

```
입력 KV 벡터  x ∈ R^d
          │
          ▼
  ┌───────────────────────┐
  │  무작위 회전 (Π)      │  ← 무작위화 하마다르 변환 (Randomized Hadamard Transform)
  │  x̃ = Π · x            │     O(d log d), O(d²) 아님
  └───────────┬───────────┘
              │
              ▼  이제 좌표는 베타 분포를 따름
  ┌───────────────────────┐
  │  로이드-맥스 스칼라   │  ← 좌표별 최적의 스칼라 양자화기
  │  양자화 (b-1)         │    한 번 사전 계산
  │  좌표당 비트          │    블록별 스케일 저장 불필요
  └───────────┬───────────┘
              │
              ▼  잔여 오차 ε = x̃ - x̂
  ┌───────────────────────┐
  │  1비트 QJL 변환      │  ← 양자화 존슨-린덴스트라우스 (Quantized Johnson-Lindenstrauss)
  │  잔여 ε에 적용      │    내적 (inner product)의 편향 제거
  └───────────┬───────────┘
              │
              ▼
  양자화된 표현: (idx, qjl_bit)
  → 어텐션을 위한 비편향 내적 추정기
```

**1단계: 회전 + 양자화:** 무작위 직교 회전 (random orthogonal rotation)을 적용한 후, 각 좌표의 분포는 입력과 무관하게 예측 가능한 베타 분포 (Beta distribution)가 됩니다. 이 분포는 분석적으로 알려져 있으므로, 최적의 로이드-맥스 양자화기 (Lloyd-Max quantizer)를 한 번 사전 계산하고 영구적으로 재사용할 수 있습니다. 블록별 상수도, 보정 (calibration)도 필요 없습니다.

**2단계: QJL 잔여 보정:** 평균 제곱 오차 (MSE) 최적 양자화기는 내적 (어텐션이 계산하는 것)을 추정할 때 편향 (bias)을 도입합니다. TurboQuant는 잔여 (residual)에 1비트 양자화 존슨-린덴스트라우스 변환 (Quantized Johnson-Lindenstrauss transform)을 적용하여, 증명 가능한 보장을 가진 비편향 내적 추정기 (unbiased inner product estimator)를 생성합니다.

### 신뢰를 주는 수학적 근거

이것은 공학적 직관이 아니라 정보 이론 (information theory)에 기반을 두고 있습니다. 이 논문은 TurboQuant의 왜곡 (distortion)이 정보 이론적 하한 (information-theoretic lower bound)의 약 2.7배 이내임을 증명합니다. 데이터 불인식 알고리즘으로서는 본질적으로 최적에 가깝습니다.

> "우리는 모든 벡터 양자화기로 달성 가능한 최상의 왜곡률에 대한 정보 이론적 하한에 대한 공식적인 증명을 제공하며, TurboQuant가 이러한 하한과 밀접하게 일치함을 보여줍니다."
> Zandieh et al., ICLR 2026

### 주목할 만한 벤치마크 수치

| 정밀도 (Precision) | LongBench (Llama-3.1-8B) | FP16 대비 압축률 (Compression vs FP16) |
|--------------------|--------------------------|----------------------------------------|
| FP16 기준치        | 50.06                    | 1배                                    |
| 3.5비트 TurboQ     | 50.06 (동일)             | ~4.5배                                 |
| 2.5비트 TurboQ     | 49.44                    | ~6배                                   |

H100 GPU에서 4비트 TurboQuant는 32비트 비양자화 키 (unquantized keys) 대비 어텐션 로짓 계산 (attention-logit computation)에서 최대 8배의 속도 향상을 제공합니다.

### 최소한의 PyTorch 스케치

다음은 1단계 (MSE 변형)의 개념적 구조입니다.

```python
import torch
import torch.nn.functional as F

def hadamard_transform(x: torch.Tensor) -> torch.Tensor:
    """Fast Walsh-Hadamard Transform - O(d log d)"""
    d = x.shape[-1]
    assert (d & (d - 1)) == 0, "dimension must be power of 2"
    h = x.clone()
    step = 1
    while step < d:
        for i in range(0, d, step * 2):
            a = h[..., i:i+step]
            b = h[..., i+step:i+step*2]
            h[..., i:i+step] = a + b
            h[..., i+step:i+step*2] = a - b
        step *= 2
    return h / (d ** 0.5)

def turboquant_mse(x: torch.Tensor, bits: int, sign_pattern: torch.Tensor):
    """
    Stage 1: Rotate → scalar quantize per coordinate
    No per-block scale storage needed.
    sign_pattern: random ±1 signs (fixed, precomputed)
    """
    # Random sign flip + Hadamard = random orthogonal rotation
    x_signed = x * sign_pattern
    x_rot = hadamard_transform(x_signed)
    # Scalar quantize using precomputed Lloyd-Max centroids
    levels = 2 ** bits
    x_min, x_max = x_rot.min(dim=-1, keepdim=True).values, \
                   x_rot.max(dim=-1, keepdim=True).values
    scale = (x_max - x_min) / levels
    indices = ((x_rot - x_min) / scale).clamp(0, levels - 1).long()
    return indices, scale, x_min  # store once per vector, not per block
```

핵심 통찰: 스케일 (scale)과 x_min은 작은 블록 (small block)당 한 번이 아니라 벡터 (vector)당 한 번 저장되어, 대부분의 양자화기를 괴롭히는 오버헤드를 제거합니다.

### 개발자에게 이것이 의미하는 바

오늘날: 128K 컨텍스트를 가진 70B 모델을 단일 노드에서 실행하려면 H100 GPU로 가득 찬 서버가 필요하거나 고통스러운 정확도 절충 (accuracy trade-offs)이 불가피합니다. TurboQuant의 4.5~6배 압축은 이러한 계산을 바꿉니다.

실질적인 영향은 다음과 같습니다.

**TurboQuant 이전 (긴 컨텍스트, 대형 모델):**
```
  ┌─────────────────────────────────────┐
  │  가중치: 40 GB  │  KV 캐시: 16 GB │  → 80GB GPU에서 OOM (메모리 부족) 발생
  └─────────────────────────────────────┘
```

**TurboQuant 이후 (3.5비트 KV 캐시):**
```
  ┌─────────────────────────────────────┐
  │  가중치: 40 GB  │  KV 캐시:  3 GB │  → 여유롭게 적합
  └─────────────────────────────────────┘
```

커뮤니티 구현은 이미 활성화되어 있습니다. 한 개발자는 MLX를 통해 Apple Silicon에서 35B 모델로 이를 실행하여, 모든 양자화 수준에서 6/6의 '건초 더미 속 바늘 찾기 (needle-in-haystack)' 테스트를 통과했습니다. 다른 개발자는 Triton 커널 (kernel)을 구축하여 RTX 4090에서 2비트 정밀도로 FP16 기준치와 문자적으로 동일한 출력을 얻었습니다.

### 솔직한 유의사항

Google은 공식적인 프로덕션 코드를 출시하지 않았습니다. 6배 압축 수치는 FP16과 비교한 것이며, 대부분의 서빙 스택은 이미 가중치에 INT8 또는 INT4를 사용하므로, 배포된 시스템에서의 실제 이득은 헤드라인보다 작습니다. 또한, 8B 매개변수 (parameters)를 초과하는 대규모 동작은 아직 논문에서 특성화되지 않았습니다.

그럼에도 불구하고 이론적 기여만으로도 이 연구는 충분히 가치가 있습니다. TurboQuant는 단순히 메모리를 절약하는 것을 넘어, 한계점 (ceiling)이 어디인지 제시합니다. KV 캐시 압축은 정보 이론적 한계에 다가가고 있습니다. 다음 도약은 더 많은 비트를 압축하는 방식으로는 오지 않을 것입니다. 완전히 다른 접근 방식이 필요할 것입니다.
