MacBook 또는 게이밍 GPU에서 Gemma 4 모델 미세 조정하기: H100 없이 가능

Gemma 4는 Google의 가장 강력한 오픈 모델 제품군이며, 31B 변형 모델은 현재 Arena AI 텍스트 리더보드에서 4위를 차지하고 있습니다. 이 모델들은 크기에 비해 뛰어난 성능을 제공한다는 점에서 두각을 나타냅니다. 그 결과, 이번 주 HuggingFace Top 10에 모든 Gemma 4 모델이 이름을 올렸습니다. 특히 E2B 및 E4B 모델은 순수한 매개변수(parameter) 수보다는 멀티모달(multimodal) 기능, 저지연(low-latency) 처리, 그리고 원활한 생태계 통합에 중점을 둡니다.

H100을 대여할 필요 없이, 게이밍 GPU 또는 MacBook에서 텍스트, 이미지, 오디오 등 다양한 모달리티(modality)로 Gemma 모델을 미세 조정(fine-tune)할 수 있습니다. 세 가지 모달리티 모두 Apple Silicon에서 실행됩니다.

**Gemma Multimodal Fine-Tuner**는 macOS용 Gemma 모델을 위한 멀티모달 LoRA 미세 조정(fine-tuning) 워크벤치(workbench)로, 다음과 같은 기능을 제공합니다.

*   CSV 파일로부터 텍스트 전용 명령어(instruction) 또는 완성(completion) 튜닝
*   CSV 파일로부터 캡셔닝(captioning) 및 VQA(Visual Question Answering)를 위한 이미지+텍스트 튜닝
*   Apple Silicon에서 작동하도록 설계된 오디오+텍스트 튜닝 (ASR 스타일, CUDA 불필요)
*   매우 큰 데이터셋(예: 클라우드 스토리지)을 위한 선택적 스트리밍(streaming) 시나리오 지원 (테라바이트(terabyte) 규모의 데이터를 노트북으로 복사할 필요 없음)
*   대화형 마법사(wizard) 및 실시간 훈련 시각화를 제공하는 CLI(Command Line Interface) 기반 워크플로우(workflow)

다음은 일반적으로 사용되는 다른 미세 조정 툴킷(toolkit)과의 비교입니다.

이제 첫 모델을 어떻게 미세 조정할 수 있는지 살펴보겠습니다.

### Apple Silicon에서 설정하기

Gemma Fine-Tuner는 다음 환경을 지원합니다.

*   패키징 메타데이터(metadata)에 맞는 Python 3.10 이상
*   MPS(Metal Performance Shaders)를 위한 Apple Silicon 기반 macOS (로컬/MPS 경로에는 Rosetta Python 사용 지양)
*   PyTorch는 MPS, CUDA 또는 CPU 전용 환경에 따라 올바른 빌드(build)가 달라지므로 별도로 설치해야 합니다.

**가상 환경(virtual environment) 생성**

```bash
brew install python@3.12

python3.12 -m venv .venv
source .venv/bin/activate
```

**종속성(dependencies) 설치**

```bash
pip3 install torch torchaudio
git clone https://github.com/mattmireles/gemma-tuner-multimodal.git
cd gemma-tuner-multimodal
pip3 install -e .
```

**HuggingFace 인증**

```bash
# brew install hf
hf auth login
```

**Gemma 4 특정 요구 사항 설치**

```bash
pip3 install -r requirements/requirements-gemma4.txt
```

이제 마법사를 실행할 수 있습니다.

```bash
gemma-macos-tuner wizard
```

Gemma Fine-Tuner는 `data/datasets/sample-text/` 경로에 16개 행으로 구성된 명령어 튜닝 데이터셋(instruction-tuning dataset)을 제공합니다. 이 데이터셋은 번역, 요약, 퀴즈, 하이쿠(haiku), JSON 변환 등을 포함하며, 1분 이내에 완료할 수 있을 만큼 작으면서도 데이터 로딩, 토큰화(tokenization), LoRA, 체크포인트(checkpointing), 내보내기(export) 등 전체 파이프라인(pipeline)의 작동을 확인하기에 충분합니다.

Enter 키를 누르면 Gemma Fine-tuner가 합리적인 기본값(default)을 사용하여 훈련 설정 과정을 안내합니다. 설정을 확인하면 훈련이 시작됩니다. 훈련 진행 상황은 UI에서 확인할 수 있습니다. 끝입니다!

다음은 유용하게 사용할 수 있는 CLI(Command Line Interface) 치트 시트(cheat-sheet)입니다.

```bash
# 데이터셋 준비 (프로필 이름은 config/config.ini에서 가져옴)
gemma-macos-tuner prepare <dataset-profile>

# 훈련 (프로필의 모델은 Gemma ID 또는 문자열에 "gemma"가 포함된 로컬 경로여야 함)
gemma-macos-tuner finetune <profile> --json-logging

# 평가
gemma-macos-tuner evaluate <profile-or-run>

# 병합된 HF/SafeTensors 트리 내보내기 (adapter_config.json이 존재할 경우 LoRA 병합)
gemma-macos-tuner export <run-dir-or-profile>

# 오류로부터 블랙리스트(blacklist) 생성
gemma-macos-tuner blacklist <profile>

# 실행 인덱스
gemma-macos-tuner runs list

# 안내형 설정
gemma-macos-tuner wizard
```

### 스트리밍 및 대규모 데이터셋

Gemma Fine-tuner는 데이터셋 유틸리티 레이어(utility layer)에서 `streaming_enabled` 데이터셋 로드 경로를 지원하며, 패치(patch)/오버라이드(override) 시스템은 스트리밍(streaming) 반복(iteration)을 명시적으로 지원합니다 (데이터셋 반복 중 실시간으로 오버라이드/블랙리스트를 적용하며, `dataset.map`을 통한 방식이 아님).

명확히 이해해야 할 두 가지 경계선:

*   v1에서는 텍스트 및 이미지 미세 조정이 의도적으로 비스트리밍(non-streaming) 로컬 CSV로 제한되며, 데이터셋 로더는 해당 모달리티(modality)에 대한 Granary/BigQuery/GCS 스트리밍 어댑터(adapter)를 거부합니다.
*   오디오 워크플로우(workflow)에서는 스트리밍이 예상됩니다 (그리고 저장소의 릴리스 노트(release notes)에서 GCS/BigQuery 스트리밍을 차별점으로 강조합니다).

설정 및 코드 관점에서 "데이터셋 소스 어댑터" 로직은 단순하며 의도적으로 CSV 중심입니다.

*   로컬 CSV 비스트리밍 어댑터
*   `streaming_enabled`가 true이거나 `source_type`이 스트리밍/GCS를 나타낼 때의 스트리밍 CSV 어댑터
*   패치 디렉토리(patch directory)를 스트리밍된 데이터셋 ID와 정렬하기 위한 데이터셋 "소스" 이름 지정에 대한 특별 처리

이는 기본 오디오 자산이 원격에 있더라도 "무엇을 훈련하는지" 사람이 감사(audit)할 수 있도록(검사할 수 있는 CSV 행) 유지하려는 시도입니다.

### 실용적인 에이전트 루프 사고방식

이제 미세 조정된 Gemma 체크포인트(checkpoint)를 기반으로 에이전트(agentic) 시스템을 구축한다면, 상위 수준의 루프(loop)는 일반적으로 다음과 같습니다.

*   **관찰(Observation):** 입력 수집 (텍스트 + 선택적 오디오 + 선택적 이미지)
*   **계획(Plan):** 모델이 도구 호출(function calling)을 할지 또는 직접 응답할지 결정하게 함
*   **실행(Act):** 코드에서 도구 호출을 실행 (유효성 검사 및 접근 제어 포함)
*   **반성(Reflect):** 다음 단계 추론을 위해 도구 출력을 모델에 다시 전달
*   **확정(Commit):** 사용자에게 보이는 출력 생성 (필요시 스키마(schema) 검사 다시 수행)

Gemma 4 모델은 함수 호출(function calling)과 구조화된 JSON 출력을 에이전트 워크플로우(agentic workflow)를 가능하게 하는 핵심 요소로 명시적으로 내세웁니다. Gemma Fine-Tuner를 사용하면 에이전트(agent)를 '프롬프트 스파게티 몬스터(prompt spaghetti monster)'로 만들지 않고도 모델이 귀하의 도메인(domain) 관찰 및 행동에 능숙하도록 만들 수 있습니다.

지원되는 모달리티(modality) 및 데이터셋 형태와 밀접하게 관련된 몇 가지 예시:

*   **음성 중심 내부 도구 에이전트(agent):** 오디오 입력(통화, 회의), 텍스트 출력(구조화된 JSON 요약, 실행 항목, 도구 호출).
*   **화면 이해 지원 에이전트:** 스크린샷(screenshot) 이미지 입력, 계정 상태 검색을 위한 도구 호출, 티켓 생성을 위한 JSON 출력.
*   **온디바이스(on-device) 개인 정보 보호 보조 에이전트:** 사용자 데이터 제약 조건 하에 로컬에서 훈련 및 실행, 다운스트림(downstream) 배포 도구용으로 병합된 SafeTensors 아티팩트(artifact) 배포.
