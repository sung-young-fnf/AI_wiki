70B(700억) 모델을 다운로드했습니다. GPU의 VRAM(비디오 램)은 8GB입니다. Ollama가 레이어를 로드하기 시작하고, 시스템이 멈추더니 5분 후 메모리 부족(out-of-memory) 오류를 마주하게 됩니다. 익숙한 상황인가요?

로컬에서 LLM(대규모 언어 모델)을 실행하는 것은 Ollama, llama.cpp, MLX 등 덕분에 놀라울 정도로 쉬워졌습니다. 하지만 어떤 모델이 자신의 하드웨어에서 실제로 잘 작동하는지 알아내는 것은 여전히 대부분 시행착오를 거쳐야 합니다. 매개변수(parameter) 개수를 확인하고, 양자화(quantization) 테이블을 대충 훑어보며, VRAM(비디오 램) 요구 사항을 어림짐작으로 계산한 뒤 최선을 바라곤 합니다.

llmfit은 이러한 방식을 바꿔줍니다. 단 하나의 명령어로, 당신의 장치에서 무엇이 실행될 수 있는지 정확히 알려줍니다.

## llmfit이란 무엇인가요?

llmfit은 시스템의 RAM(램), CPU, GPU에 LLM(대규모 언어 모델)을 적절하게 조정해주는 오픈소스 Rust CLI(명령줄 인터페이스) 도구입니다. 이 도구는 하드웨어를 감지하고, 수백 개의 모델을 여러 기준에 따라 평가하여, 시간 낭비하며 다운로드하기 전에 어떤 모델이 실제로 잘 실행될지 알려줍니다.

llmfit은 기본적으로 대화형 TUI(텍스트 사용자 인터페이스), 기존 CLI(명령줄 인터페이스) 모드, 그리고 클러스터 스케줄링을 위한 REST API(애플리케이션 프로그래밍 인터페이스)까지 제공합니다. MIT 라이선스를 보유하고 있으며, 깃허브(GitHub)에서 13,000개 이상의 스타를 받았고, 몇 초 만에 설치됩니다:

```bash
brew install llmfit    # macOS / Linux
scoop install llmfit   # Windows
```

llmfit을 실행하면 시스템에 얼마나 잘 맞는지에 따라 순위가 매겨진, 스크롤 및 정렬이 가능한 모델 표를 얻을 수 있습니다. 하드웨어 사양은 상단에 표시됩니다. 각 행에는 종합 점수, 예상 초당 토큰 수(tokens per second), 설정에 가장 적합한 양자화(quantization), 실행 모드, 메모리 사용량, 그리고 사용 사례(use-case) 카테고리가 나타납니다.

## llmfit에 주목해야 하는 이유

다음 세 가지가 llmfit을 단순한 모델 목록 그 이상으로 만듭니다.

### 동적 양자화(Dynamic Quantization) 선택

고정된 양자화를 가정하는 대신, llmfit은 Q8_0(최고 품질)부터 Q2_K(최고 압축)까지의 계층을 탐색하여 사용 가능한 메모리에 맞는 최고 품질을 선택합니다. 전체 컨텍스트(full context)에서 맞는 것이 없으면, 절반 컨텍스트(half context)로 다시 시도합니다. 이를 통해 하드웨어가 지원할 수 있는 최상의 품질을 항상 얻을 수 있습니다.

### MoE(전문가 혼합) 인식 점수 계산

이것은 중요한 부분입니다. Mixtral 8x7B와 같은 모델은 총 467억 개의 매개변수(parameter)를 가지고 있지만, 토큰당 약 129억 개만 활성화됩니다. 대부분의 도구는 MoE(Mixture of Experts) 모델을 밀집(dense) 모델로 취급하여 메모리 요구 사항을 크게 과대평가합니다. llmfit은 이러한 차이를 이해하고 있으며, 활성 매개변수(active parameters)를 기반으로 VRAM(비디오 램)을 계산합니다. 이는 다른 방법으로는 실행 불가능해 보였던 MoE 모델들이 갑자기 실행 가능한 것으로 나타날 수 있음을 의미합니다.

### 실제 대역폭(Bandwidth) 기반 속도 예측

일반적인 벤치마크(benchmark)를 통해 초당 토큰 수를 추측하는 대신, llmfit은 NVIDIA, AMD, Apple Silicon의 약 80개 GPU에 대한 실제 GPU 메모리 대역폭 조회 테이블을 사용합니다. 공식은 간단합니다:

`(bandwidth_GB/s / model_size_GB) × efficiency_factor`

효율성 계수(efficiency factor)는 커널 오버헤드(kernel overhead), KV 캐시(KV-cache) 읽기, 메모리 컨트롤러(memory controller) 효과를 고려합니다. 완벽하지는 않지만, 단순히 '느낌'이 아니라 물리적 원리에 기반을 두고 있습니다.

## 점수 시스템

각 모델은 0~100점 척도로 다음 네 가지 기준에 따라 점수가 매겨집니다:

*   **품질(Quality)** — 매개변수 수, 모델 계열 평판, 양자화(quantization) 패널티, 작업 정렬(task alignment)
*   **속도(Speed)** — 백엔드(backend) 및 하드웨어 기반의 예상 초당 토큰 수
*   **적합성(Fit)** — 메모리 활용 효율성, 사용 가능한 메모리의 50~80%가 최적 지점
*   **컨텍스트(Context)** — 사용 사례(use case)에 실제로 필요한 컨텍스트 윈도우(context window) 기능

이러한 기준들은 가중치가 적용된 종합 점수로 합산됩니다. 가중치는 사용 사례에 따라 달라지는데, 예를 들어 채팅(Chat)은 속도에 더 높은 가중치를, 추론(Reasoning)은 품질에 더 높은 가중치를 부여하는 식입니다. 하드웨어에서 실행할 수 없는 모델은 항상 목록의 가장 아래로 배치됩니다.

## TUI(텍스트 사용자 인터페이스)를 넘어

llmfit은 단순히 보기 좋은 터미널 앱이 아닙니다. 다음 몇 가지 주목할 만한 기능이 있습니다:

*   **계획 모드(Plan Mode)** — 어떤 모델이든 `p`를 누르면 질문이 바뀝니다. "내 하드웨어에 무엇이 맞을까?" 대신 "이 모델을 위해 어떤 하드웨어가 필요할까?"를 추정합니다. 컨텍스트 길이, 양자화(quantization), 목표 초당 토큰 수를 조정할 수 있으며, 최소 및 권장 사양과 업그레이드 경로를 보여줍니다.

*   **런타임 통합(Runtime Integration)** — Ollama, llama.cpp, MLX를 자동으로 감지합니다. 이미 설치된 모델은 녹색 체크 표시가 나타납니다. `d`를 누르면 TUI(텍스트 사용자 인터페이스)에서 직접 다운로드할 수 있으며, 여러 제공자가 있을 경우 선택 도구가 열립니다.

*   **REST API(애플리케이션 프로그래밍 인터페이스)** — `llmfit serve`는 전체 점수 계산 및 필터링 엔진을 HTTP를 통해 노출합니다. 클러스터 스케줄러, 플릿 관리 또는 자체 도구 구축에 유용합니다:

    ```bash
    curl "http://localhost:8787/api/v1/models/top?limit=5&min_fit=good&use_case=coding"
    ```

*   **다중 GPU(Multi-GPU) 지원** — 감지된 모든 NVIDIA GPU의 VRAM(비디오 램)을 통합합니다. 또한 AMD(ROCm), Intel Arc, Apple Silicon 통합 메모리(unified memory), Ascend NPU(신경망 처리 장치)도 지원합니다.

## 누가 이 도구를 사용해야 할까요?

개발, 개인 정보 보호, 비용 절감 또는 단순히 호기심 때문에 로컬 모델을 실행하는 경우, llmfit은 추측을 제거해줍니다. 특히 다음과 같은 경우에 유용합니다:

*   GPU를 가지고 있고, 그것이 처리할 수 있는 최고의 모델을 알고 싶을 때
*   MoE(전문가 혼합) 모델로 작업하고 있으며, 과장된 메모리 추정에 지쳤을 때
*   여러 장치를 관리하고 배포를 계획해야 할 때
*   특정 모델을 위해 GPU 업그레이드가 가치 있는지 빠르게 평가하고 싶을 때

이 프로젝트는 활발히 유지보수되고 있으며(51개 릴리스, 가장 최근 릴리스는 불과 몇 시간 전), 모든 주요 계열(Llama, Qwen, Mistral, DeepSeek, Gemma, Phi 등)의 수백 가지 모델을 다루고, 데이터베이스는 쉽게 업데이트할 수 있습니다.

```bash
# 설치
brew install llmfit

# 대화형 TUI 실행
llmfit

# 또는 CLI 모드 사용
llmfit --cli

# 하드웨어 사양 확인
llmfit system

# JSON 형식의 상위 5개 추천
llmfit recommend --json --limit 5

# 이 모델에는 어떤 하드웨어가 필요할까?
llmfit plan "Qwen/Qwen3-4B-MLX-4bit" --context 8192
```

저장소(repo)는 [github.com/AlexsJones/llmfit](https://github.com/AlexsJones/llmfit)에 있습니다. MIT 라이선스이며 Rust로 작성되었고, 그저 작동합니다.
