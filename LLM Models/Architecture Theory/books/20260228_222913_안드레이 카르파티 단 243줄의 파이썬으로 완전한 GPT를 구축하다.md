## 파트 1: 토크나이저 (중요하다는 사실을 잊기 쉬운 부분)

```python
chars = ['<BOS>', '<EOS>'] + sorted(list(set(''.join(docs))))
vocab_size = len(chars)
stoi = { ch:i for i, ch in enumerate(chars) }
itos = { i:ch for i, ch in enumerate(chars) }
```

컴퓨터는 글자를 이해하지 못합니다. 숫자를 이해하죠.

따라서 가장 첫 번째 작업은 모든 문자를 숫자로 변환하는 것입니다. 예를 들어, 문자 'a'는 2가 되고, 'b'는 3이 되는 식이죠. 또한 두 개의 특별한 토큰(token)이 있습니다: `<BOS>` (Beginning Of Sequence, "모델아, 새로운 이름이 시작되고 있어!")와 `<EOS>` (End Of Sequence, "이 이름은 이제 끝났어!").

`stoi`는 문자열(string)을 정수(integer)로 변환하고, `itos`는 정수를 다시 문자열로 변환합니다. 이것이 여러분의 전체 토크나이저(tokenizer)입니다.

ChatGPT는 일반적인 문자 조합을 단일 토큰으로 묶는 BPE(Byte Pair Encoding)라는 훨씬 더 정교한 토크나이저를 사용합니다. 하지만 기본 아이디어는 동일합니다: 텍스트가 들어가면 숫자가 나옵니다.

## 파트 2: 오토그래드 엔진 (모든 것의 심장)

여기가 바로 아름다운 부분입니다.

```python
class Value:
    def __init__(self, data, _children=(), _op=''):
        self.data = data
        self.grad = 0
        self._backward = lambda: None
        self._prev = set(_children)
```

카르파티는 약 40줄의 코드로 파이토치(PyTorch)의 오토그래드 엔진(autograd engine) 미니 버전을 구축합니다.

왜 이것이 중요한지 설명해 드리겠습니다.

신경망(neural network)이 예측을 할 때, 처음에는 대개 틀립니다. 우리는 어떤 숫자(가중치, weights)를 얼마나 조정해야 덜 틀리게 만들 수 있을지 알아내야 합니다.

그것이 바로 역전파(backpropagation)가 하는 일입니다. 그리고 역전파는 기울기(gradients)를 필요로 합니다. 기본적으로, '이 가중치(weight)를 아주 조금 흔들면 최종 오차(error)가 얼마나 변할까?'를 알아야 하는 거죠.

`Value` 클래스는 네트워크 내의 모든 숫자를 감싸고 다음을 추적합니다:

*   `data`: 실제 숫자
*   `grad`: 이 숫자를 변경할 때 최종 손실(loss)이 얼마나 변하는지
*   `_backward`: 해당 기울기를 계산하기 위한 지침
*   `_prev`: 이 `Value`를 생성하는 데 사용된 다른 `Value` 객체들

덧셈, 곱셈, 거듭제곱 등 수학 연산을 할 때마다, `Value` 클래스는 보이지 않는 곳에서 조용히 연산 그래프(graph of operations)를 구축합니다. 하나의 예제에 대한 학습이 완료되면, `.backward()`를 호출하고 기울기(gradients)가 이 그래프를 통해 자동으로 역류합니다(flow back).

여기 덧셈 연산이 있습니다:

```python
def __add__(self, other):
    out = Value(self.data + other.data, (self, other), '+')
    def _backward():
        self.grad += out.grad
        other.grad += out.grad
    out._backward = _backward
    return out
```

만약 `c = a + b`이고, 누군가가 "c의 기울기가 5"라고 말한다면, `a`와 `b`의 기울기도 5입니다. 왜냐하면 `a`를 1만큼 증가시키면 `c`도 1만큼 증가하기 때문입니다. 이것이 바로 미적분학(calculus)의 연쇄 법칙(chain rule)이며, 4줄의 코드로 구현된 것입니다.

곱셈은 비슷하지만 '교환(swap)' 규칙을 사용합니다:

```python
def __mul__(self, other):
    out = Value(self.data * other.data, (self, other), '*')
    def _backward():
        self.grad += other.data * out.grad
        other.grad += self.data * out.grad
    out._backward = _backward
    return out
```

만약 `c = a * b`라면, `a`의 기울기는 `b * (c의 기울기)`이고, `b`의 기울기는 `a * (c의 기울기)`입니다. 이것이 전부입니다.

전체 역전파(backward pass)는 위상 정렬(topological sort)을 사용합니다. 이는 기울기(gradient)를 올바른 순서(출력이 먼저, 입력이 나중에)로 계산하도록 보장합니다.

```python
def backward(self):
    topo = []
    visited = set()
    def build_topo(v):
        if v not in visited:
            visited.add(v)
            for child in v._prev:
                build_topo(child)
            topo.append(v)
    build_topo(self)
    self.grad = 1
    for v in reversed(topo):
        v._backward()
```

이것은 `loss.backward()`를 호출할 때 파이토치(PyTorch) 내부에서 실행되는 것과 문자 그대로 동일한 알고리즘(algorithm)입니다. 카르파티는 파이썬 리스트와 재귀(recursion)를 사용하여 이를 처음부터 직접 작성했습니다.

## 파트 3: 모델 파라미터 (GPT의 "두뇌")

```python
matrix = lambda nout, nin, std=0.02: [[Value(random.gauss(0, std)) for _ in range(nin)] for _ in range(nout)]
state_dict = {'wte': matrix(vocab_size, n_embd), 'wpe': matrix(block_size, n_embd)}
```

모든 신경망(neural network)은 숫자(가중치 또는 파라미터, weights or parameters)의 집합일 뿐입니다. 훈련 전에는 무작위이지만, 훈련 후에는 학습된 패턴을 인코딩(encode)합니다.

각 가중치 행렬(weight matrix)이 하는 일은 다음과 같습니다:

*   `wte` (Word Token Embedding): 각 토큰 ID(token ID)를 16개의 숫자로 이루어진 벡터(vector)로 변환합니다. 이것을 각 문자에 "개성"을 부여하는 것으로 생각해보세요. 즉, 유사한 문자들은 서로 가까이 위치하게 되는 16차원 공간에서의 위치를 부여하는 것입니다.
*   `wpe` (Word Position Embedding): 각 문자가 시퀀스(sequence) 내 어디에 위치하는지 모델에게 알려줍니다. 1번째 위치의 "a"는 5번째 위치의 "a"와 다르게 처리되어야 합니다.

각 트랜스포머 레이어(transformer layer)에는 다음이 있습니다:

*   `attn_wq`, `attn_wk`, `attn_wv`: 어텐션(attention)을 위한 쿼리(Query), 키(Key), 값(Value) 행렬 (아래에서 더 자세히 설명합니다)
*   `attn_wo`: 어텐션의 출력 투영(output projection)
*   `mlp_fc1`, `mlp_fc2`: 어텐션된 정보를 처리하는 피드포워드 네트워크(feedforward network)

기본 구성: 16개의 임베딩 차원(embedding dimensions), 4개의 어텐션 헤드(attention heads), 1개의 레이어(layer), 시퀀스 길이 8. 이는 대략 4,000개의 파라미터(parameters)를 제공합니다. GPT-4는 1조 개가 넘습니다. 아키텍처(architecture)는 동일하지만, 규모(scale)는 엄청나게 다릅니다.

## 파트 4: GPT 아키텍처 (마법이 일어나는 곳)

이것이 핵심입니다. `gpt()` 함수를 하나씩 살펴보겠습니다.

### 단계 1: 임베딩 (Embedding)

```python
tok_emb = state_dict['wte'][token_id]
pos_emb = state_dict['wpe'][pos_id % block_size]
x = [t + p for t, p in zip(tok_emb, pos_emb)]
```

문자의 임베딩(embedding)을 가져옵니다. 위치의 임베딩을 가져옵니다. 이 둘을 더합니다. 이제 모델은 문자가 무엇인지, 그리고 어디에 위치하는지 모두 알게 됩니다.

### 단계 2: RMSNorm

```python
def rmsnorm(x):
    ms = sum(xi * xi for xi in x) / len(x)
    scale = (ms + 1e-5) ** -0.5
    return [xi * scale for xi in x]
```

각 주요 계산 전에, 값이 폭주하거나 0으로 수축하지 않도록 정규화(normalize)합니다. `RMSNorm`은 (원래 GPT-2가 사용하는) `LayerNorm`의 더 간단한 사촌 격입니다. 이는 모든 값의 제곱 평균 제곱근(root-mean-square)으로 각 값을 나눕니다.

이것을 그룹 토론 전에 모든 사람의 목소리 볼륨을 적절한 수준으로 유지하는 것에 비유해 보세요.

### 단계 3: 어텐션 (Attention) (쇼의 주인공)

이것이 모두가 이야기하는 부분입니다. 2017년의 유명한 논문인 "Attention is All You Need"입니다. 여기, 순수한 파이썬(Python)으로 구현된 어텐션(Attention)이 그대로 있습니다:

```python
q = linear(x, state_dict[f'layer{li}.attn_wq'])
k = linear(x, state_dict[f'layer{li}.attn_wk'])
val = linear(x, state_dict[f'layer{li}.attn_wv'])
```

현재 토큰(token)에 대해 세 가지를 계산합니다:

*   쿼리(Query, Q): "내가 무엇을 찾고 있지?"
*   키(Key, K): "내가 무엇을 담고 있지?"
*   값(Value, V): "내가 어떤 정보를 가지고 있지?"

그런 다음 각 어텐션 헤드(attention head)에 대해:

```python
attn_logits = [sum(q_h[j] * k_h[t][j] for j in range(head_dim)) / head_dim**0.5 
               for t in range(len(k_h))]
attn_weights = softmax(attn_logits)
head_out = [sum(attn_weights[t] * v_h[t][j] for t in range(len(v_h))) 
            for j in range(head_dim)]
```

쉽게 설명하자면 다음과 같습니다:

*   현재 토큰의 쿼리(Query)는 "어떤 이전 토큰들이 나에게 관련이 있을까?"라고 묻습니다.
*   이는 모든 이전 토큰의 키(Key)와 내적(dot product)을 계산합니다 (마치 호환성 점수처럼).
*   이 점수들은 소프트맥스(softmax)를 통과합니다 (이들을 확률처럼 합이 1이 되도록 만듭니다).
*   모든 토큰의 값(Value) 벡터(vector)는 이 점수들에 의해 가중치(weighted) 부여되어 혼합됩니다.

따라서 모델이 "Micha"라는 이름을 생성하고 다음 글자를 예측하려 할 때, 어텐션 메커니즘(attention mechanism)은 현재 위치가 'M, i, c, h, a'를 "돌아보고(look back)" 다음을 예측하는 데 어떤 것이 가장 중요한지 결정하게 합니다.

`head_dim**0.5`로 나누는 것을 스케일드 어텐션(scaled attention)이라고 합니다. 이는 내적(dot product)이 너무 커지는 것을 방지하여 소프트맥스(softmax)가 극단적인 확률(기본적으로 모두 0이고 하나만 1인)을 출력하는 것을 막습니다.

멀티 헤드 어텐션(Multi-head attention)은 이 과정을 병렬로 4번 실행한다는 의미입니다 (4개의 다른 쿼리(Q), 키(K), 값(V) 투영(projection)을 사용하며), 각각 16차원 임베딩(embedding)의 다른 4차원 슬라이스(slice)를 살펴봅니다. 한 헤드(head)는 모음(vowels)에 집중하는 것을 학습할 수 있고, 다른 헤드는 최근 문자에 집중할 수 있습니다. 이들은 모두 다른 관점을 제공합니다.

### 단계 4: 잔차 연결 (Residual Connection)

```python
x = [a + b for a, b in zip(x, x_residual)]
```

어텐션(attention) 후에 원본 입력(input)을 다시 더해줍니다. 이것을 잔차 연결(residual connection) 또는 스킵 연결(skip connection)이라고 합니다.

왜일까요? 학습 중 기울기(gradients)를 위한 "고속도로(highway)"를 만들기 때문입니다. 이것 없이는 깊은 네트워크(deep networks)는 학습시키기 거의 불가능해집니다. 이는 "어텐션에서 배운 것이 무엇이든, 이미 알고 있던 것에 그냥 추가해라"라고 말하는 것과 같습니다.

### 단계 5: 다층 퍼셉트론 (MLP, Feed-Forward Network)

```python
x = linear(x, state_dict[f'layer{li}.mlp_fc1'])
x = [xi.relu() ** 2 for xi in x]
x = linear(x, state_dict[f'layer{li}.mlp_fc2'])
```

어텐션(attention)이 어떤 토큰(token)에 집중할지 결정한 후, 다층 퍼셉트론(MLP, Feed-Forward Network)이 그 정보를 처리합니다.

다층 퍼셉트론(MLP)은 16차원 벡터(vector)를 64차원(4 * `n_embd`)으로 확장하고, 비선형성(non-linearity, 제곱 ReLU)을 적용한 후 다시 16차원으로 압축합니다.

왜 확장하고 압축할까요? 브레인스토밍(brainstorming)이라고 생각해 보세요. 먼저 많은 아이디어를 생성하고(64로 확장), 나쁜 아이디어를 걸러내고(ReLU는 음수를 제거하고, 제곱은 강한 신호를 증폭), 요약하는 것(16으로 압축)과 같습니다.

GELU 대신 제곱 ReLU(Squared ReLU)를 사용하는 것은 카르파티의 단순화 중 하나입니다. 일반적인 ReLU는 "음수이면 0으로 만들어라"고 말합니다. 제곱 ReLU는 "음수이면 0으로 만들고, 양수이면 제곱하라"고 말합니다. 이는 더 강력한 비선형성(non-linearity)을 제공합니다.

### 단계 6: 어휘로 투영 (Project to Vocabulary)

```python
logits = linear(x, state_dict['wte'])
```

마지막으로, 모델은 동일한 임베딩 행렬(`wte`, embedding matrix)을 사용하여 어휘 크기(vocabulary size)로 다시 투영(project)합니다. 이것을 가중치 공유(weight tying)라고 하는데, 토큰 ID(token ID)를 임베딩(embedding)으로 변환하는 데 사용된 동일한 가중치(weights)가 임베딩을 다시 예측으로 변환하는 데 재사용됩니다.

출력은 가능한 각 다음 문자에 대한 점수입니다. 점수가 높을수록 모델은 이 문자가 다음에 올 가능성이 더 높다고 생각합니다.

## 파트 5: 학습 루프 (모델이 학습하는 방법)

```python
for step in range(args.num_steps):
    doc = docs[step % len(docs)]
    tokens = [BOS] + [stoi[ch] for ch in doc] + [EOS]
    tokens = tokens[:block_size]
```

각 단계마다 데이터셋(dataset)에서 이름 하나를 가져옵니다. 이를 토큰 ID(token ID)로 변환하고, 시작 부분에 `<BOS>`를, 끝 부분에 `<EOS>`를 추가합니다.

```python
for pos_id in range(len(tokens) - 1):
    logits = gpt(tokens[pos_id], pos_id, keys, values)
    probs = softmax(logits)
    loss = -probs[tokens[pos_id + 1]].log()
```

각 위치에서 모델은 현재 토큰(token)을 보고 다음 토큰을 예측합니다. 손실(loss)은 음의 로그 우도(negative log likelihood)입니다: 모델이 올바른 다음 문자에 높은 확률(probability)을 할당하면 손실은 낮고, 낮은 확률을 할당하면 손실은 높아집니다.

그런 다음 `loss.backward()`는 모든 파라미터(parameter)에 대한 기울기(gradients)를 계산하고, 아담(Adam) 최적화기(optimizer)가 이를 업데이트합니다:

```python
m[i] = beta1 * m[i] + (1 - beta1) * p.grad
v[i] = beta2 * v[i] + (1 - beta2) * p.grad ** 2
m_hat = m[i] / (1 - beta1 ** (step + 1))
v_hat = v[i] / (1 - beta2 ** (step + 1))
p.data -= lr_t * m_hat / (v_hat ** 0.5 + eps_adam)
```

아담(Adam)은 메모리(memory)를 가진 경사 하강법(gradient descent)과 같습니다. 단순히 현재 기울기를 따르는 대신, 다음을 추적합니다:

*   `m`: 최근 기울기들의 평균 방향 (모멘텀(momentum): 언덕을 굴러 내려가는 공과 같음)
*   `v`: 최근 기울기들의 평균 크기 (정규화(normalize)에 도움: 큰 기울기가 지배하지 않도록 함)

이것은 GPT-4, DALL-E 및 오늘날의 모든 주요 모델을 훈련하는 데 사용되는 것과 동일한 최적화기(optimizer)입니다.

## 파트 6: 추론 (새로운 이름 생성)

```python
for sample_idx in range(5):
    token_id = BOS
    generated = []
    for pos_id in range(block_size):
        logits = gpt(token_id, pos_id, keys, values)
        probs = softmax(logits)
        token_id = random.choices(range(vocab_size), weights=[p.data for p in probs])[0]
        if token_id == EOS:
            break
        generated.append(itos[token_id])
```

`<BOS>`로 시작합니다. 모델에게 "다음에 무엇이 올까?"라고 묻습니다. 확률 분포(probability distribution)에서 샘플링(sample)합니다. 그 예측을 다시 입력으로 넣습니다. 모델이 `<EOS>`를 출력하거나 공간이 부족해질 때까지 반복합니다.

이것이 자기회귀 생성(autoregressive generation)입니다. 여러분이 ChatGPT에 메시지를 보낼 때마다 ChatGPT가 하는 것과 동일합니다. 한 번에 하나의 토큰(token)씩, 각각 이전의 모든 것에 조건화(conditioned)되어 생성됩니다.

## 큰 그림: 왜 이것이 중요한가

여기서 제가 놀랐던 점이 있습니다.

이 243줄을 넘어가는 모든 코드: 파이토치(PyTorch)의 수천 개 파일, 쿠다 커널(CUDA kernels), 분산 학습 프레임워크(distributed training frameworks), 정교한 토크나이저(tokenizer)들, 이 모든 것은 단지 최적화(optimization)일 뿐입니다. 더 빠르게 만들고, 더 크게 만들고, 수천 개의 그래픽 처리 장치(GPU)에서 실행되도록 하는 것이죠.

하지만 알고리즘(algorithm)은요? 실제 지능(intelligence)은요? 바로 여기에 있습니다.

카르파티가 말했듯이: "이것이 필요한 모든 알고리즘적 내용이다. 다른 모든 것은 효율성을 위한 것일 뿐이다. 나는 이것을 더 이상 단순화할 수 없다."

이것은 카르파티의 6년간의 압축 여정입니다:

*   2020: `micrograd` (오토그래드 엔진, autograd engine)
*   2020: `minGPT` (파이토치 GPT, PyTorch GPT)
*   2023: `nanoGPT` (실제 서비스 수준 훈련, production-grade training)
*   2024: `llm.c` (순수 C/쿠다, raw C/CUDA, 프레임워크 없음)
*   2026: `microGPT` (알고리즘 그 자체, the algorithm and nothing else)

각 단계는 추상화(abstraction)의 한 겹을 벗겨냈습니다. 이 최종 버전은 그 모든 것을 제거했습니다.

업계는 올해 인공지능(AI) 인프라(infrastructure)에 4천억 달러를 지출하고 있습니다. 하지만 그 모든 것을 구동하는 핵심 알고리즘(algorithm)은요? 대부분의 README 문서보다 작은 파일 하나에 들어갑니다.

대규모 언어 모델(LLM)이 어떻게 작동하는지 이해하려 한다면, 파이토치(PyTorch) 문서를 먼저 보지 마세요. 여기서부터 시작하세요.
