# IDE 속의 트로이 목마: 가짜 AI 코딩 어시스턴트가 30만 대 이상의 개발자 PC를 해킹한 방법

우리가 코드를 작성하기 위해 신뢰하고 사용하는 도구들이 국가 수준의 멀웨어(Malware)를 전달하는 완벽한 매개체가 되었을 때, 어떤 일이 벌어지는지 살펴봅니다.

2026년 1월 27일, 보안 기업 아이키도(Aikido)의 연구원들은 공식 비주얼 스튜디오 코드 마켓플레이스(Visual Studio Code Marketplace)에서 소름 끼치는 발견을 했습니다. 수많은 생산성 도구 사이에 "ClawdBot Agent — AI Coding Assistant"라는 이름의 '양의 탈을 쓴 늑대'가 숨어 있었던 것입니다. 겉보기에는 OpenAI, Anthropic, Google 등 10개 이상의 주요 서비스와 연동되는 정교한 AI 코딩 도구처럼 보였고, 실제로 자동 완성 기능과 지능형 제안 기능이 광고된 대로 작동했습니다.

하지만 이 매끄러운 겉모습 아래에는 훨씬 더 사악한 것이 숨겨져 있었습니다. 이것은 합법적인 도구가 아니라, 개발자 컴퓨터에 지속적인 원격 접속 권한을 부여하도록 설계된 완전한 기능을 갖춘 트로이 목마(Trojan)였습니다.

### 완벽한 사칭의 해부학

실제 'MoltBot'(구 ClawdBot)은 개발자들 사이에서 큰 인기를 끌고 있는 오픈소스 AI 어시스턴트입니다. 오스트리아의 개발자 피터 슈타인베르거(Peter Steinberger)가 만든 이 도구는 기록적인 속도로 GitHub 스타 8만 5천 개 이상을 획득했습니다. 공격자들은 이 도구가 엄청난 인기를 끌고 있음에도 불구하고 공식 VS Code 확장 프로그램이 없다는 틈새를 노려 이름을 먼저 선점했습니다.

이 가짜 확장 프로그램은 개발자의 심리와 보안 회피 기술을 깊이 이해하고 있었습니다. 공격자는 다음과 같은 요소를 포함했습니다.

*   전문적인 아이콘과 세련된 UI 요소
*   7개의 서로 다른 제공업체와 연동되는 실제 AI 기능
*   의심을 피하기 위한 실제 코드 완성 기능
*   신뢰할 수 있어 보이는 게시자 이름("clawdbot")

 이것은 단순한 피싱 시도가 아니라, 개발자들이 통합 개발 환경(IDE)에 부여하는 암묵적인 신뢰를 악용하도록 설계된 정교한 공급망 공격(Supply Chain Attack)이었습니다.

### 기술적 실행: 지속성 확보의 정수

이 공격이 특히 경악스러운 점은 기술적인 정교함에 있습니다. 확장 프로그램은 감지를 피하고 시스템에 상주하기 위해 VS Code 자체의 아키텍처를 활용했습니다.

**시작 시 자동 실행**
확장 프로그램의 `package.json` 파일에는 다음과 같은 강력한 한 줄이 포함되어 있었습니다.
`"activationEvents": ["onStartupFinished"]`
이를 통해 사용자의 상호작용이나 의심스러운 대화창 없이도 VS Code가 실행될 때마다 악성 페이로드(Payload)가 자동으로 활성화되었습니다.

**페이로드 전달 체계**
확장 프로그램의 `activate()` 함수는 정당한 AI 기능이 로드되기 전에 `initCore()`를 즉시 호출합니다. 이 함수는 특정 사이트에서 설정 파일을 가져와 "Code.exe"라는 이름의 바이너리를 다운로드하고 실행하도록 지시합니다.

이 실행 파일은 단순한 멀웨어가 아니라, 전 세계 IT 부서에서 사용하는 신뢰할 수 있는 원격 데스크톱 애플리케이션인 'ConnectWise ScreenConnect'의 정식 서명 복사본이었습니다. 합법적인 도구를 무기화함으로써, 공격자들은 대부분의 백신이나 보안 소프트웨어의 화이트리스트(Whitelist)를 통과할 수 있었습니다. 실행이 완료되면 공격자는 감염된 기기에 대해 전체 그래픽 제어 권한을 가진 지속적인 원격 접속 권한을 갖게 됩니다.

**대체 메커니즘 (Fallback Mechanisms)**
전문적인 공격답게 기본 인프라가 차단될 경우를 대비한 여러 층의 복구 수단을 마련해 두었습니다.
*   **DLL 사이드 로딩(DLL Side Loading):** 기본 다운로드가 실패하면 Rust로 작성된 DLL 파일을 통해 동일한 페이로드를 로드합니다.
*   **하드코딩된 URL:** 여러 보조 도메인에 동일한 페이로드를 호스팅했습니다.
*   **클라우드 저장소:** "Zoom 업데이트"로 위장한 Dropbox 링크를 통해 제3의 전달 경로를 확보했습니다.
*   **PowerShell 실행:** Node.js 방식이 실패할 경우를 대비해 배치 스크립트로 PowerShell 다운로드를 유도했습니다.

### 내부의 공급망 위기

이 사건은 실제 MoltBot 프로젝트의 보안 문제와 맞물려 혼란을 가중시켰습니다. 보안 연구원들에 따르면, 리버스 프록시(Reverse Proxy) 설정 오류로 인해 수백 개의 MoltBot 인스턴스가 인터넷에 무방비로 노출되었습니다. MoltBot은 기본적으로 '로컬' 연결을 자동 승인하는데, 프록시 환경에서는 외부 연결이 로컬로 오인되어 인증되지 않은 사용자가 다음과 같은 정보에 접근할 수 있었습니다.

*   설정 데이터 및 API 키
*   연결된 서비스의 OAuth 자격 증명
*   개인 채팅의 전체 대화 기록
*   통합 플랫폼에서의 도구 및 명령 실행 권한

더욱 심각한 것은 MoltBot이 자격 증명과 API 키 등을 로컬 디렉토리에 평문(Plaintext)으로 저장한다는 점입니다. 이미 RedLine, Lumma 같은 정보 탈취형 멀웨어(Infostealer)들은 이러한 특정 디렉토리 구조를 공격하도록 적응하고 있습니다.

### 'MaliciousCorgi' 캠페인: 또 다른 위협

가짜 MoltBot 외에도 "MaliciousCorgi"라고 명명된 또 다른 캠페인이 발견되었습니다. 이들은 총 150만 회 이상의 설치 횟수를 기록한 두 개의 확장 프로그램을 통해 대규모 데이터 유출을 시도했습니다.

*   ChatGPT — Chinese Edition (설치 134만 회)
*   ChatGPT — ChatMoss (설치 15만 회)

이 프로그램들은 원격 접속을 시도하는 대신, 편집기에서 열린 모든 파일을 읽어 Base64로 인코딩한 후 중국 서버로 전송했습니다. 소스 코드, 실시간 편집 내용, 클라우드 자격 증명이 포함된 설정 파일, 데이터베이스 비밀번호가 담긴 `.env` 파일 등이 모두 유출 대상이었습니다.

### 개발자가 특히 취약한 이유

이러한 공격의 성공은 현대 개발 워크플로우의 근본적인 취약성을 드러냅니다. VS Code 확장 프로그램은 IDE 자체와 동일한 권한으로 실행되며, 다음과 같은 권한을 가집니다.

*   프로젝트 디렉토리에 대한 전체 파일 시스템 접근
*   셸(Shell) 명령 실행 능력
*   데이터 유출을 위한 네트워크 접근
*   환경 변수 및 자격 증명 접근
*   클립보드 내용 읽기 권한

개발자들은 확장 프로그램을 설치할 때 마켓플레이스가 이를 검증했을 것이라고 암묵적으로 신뢰하지만, 현실은 다릅니다. 마이크로소프트의 VS Code 마켓플레이스에는 자동화된 보안 스캔 기능이 부족하며, 누구나 확장 프로그램을 게시할 수 있습니다. 2025년에만 110개의 악성 확장 프로그램이 제거되었지만, 공격 속도는 더욱 빨라지고 있습니다.

### 탐지 및 방어 대책

2026년 1월 27일경 게시된 가짜 MoltBot 확장 프로그램을 설치한 것으로 의심된다면 즉시 다음 조치를 취해야 합니다.

**즉각적인 대응**
1.  VS Code에서 해당 확장 프로그램을 즉시 삭제합니다.
2.  `C:\Program Files (x86)\ScreenConnect Client` 경로에 의심스러운 설치 사례가 있는지 확인하고 삭제합니다.
3.  임시 디렉토리(`%TEMP%\Lightshot`)를 삭제합니다.
4.  프로젝트 파일에 존재했던 모든 API 키, 데이터베이스 자격 증명, 클라우드 서비스 토큰을 재설정(Rotate)합니다.
5.  클라우드 제공업체의 액세스 로그에서 무단 활동을 검토합니다.

**일반 보안 수칙**
*   **확장 프로그램 감사:** 설치된 도구 목록을 검토하고 사용하지 않는 것은 제거하십시오. 게시자의 이력과 최근 리뷰를 확인하십시오.
*   **설치 격리:** 실험적인 AI 에이전트는 가상화 또는 샌드박스 환경에서 실행하십시오.
*   **네트워크 모니터링:** 알려진 악성 도메인으로의 아웃바운드 연결을 차단하십시오.
*   **최소 권한 원칙:** 확장 프로그램에 광범위한 파일 시스템 접근 권한을 부여하지 마십시오. 민감한 프로젝트에는 VS Code의 제한 모드(Restricted Mode)를 사용하십시오.
*   **신뢰 전 확인:** 공식 소스에서 제공하는 AI 도구만 설치하십시오. 인기 오픈소스 도구가 갑자기 마켓플레이스에 나타나면 원본 프로젝트 관리자에게 공식 여부를 확인하십시오.

### 소프트웨어 공급망 공격의 새로운 개척지

가짜 MoltBot 사례는 공격자들의 수법이 진화했음을 보여줍니다. 이들은 브랜드 인지도 악용, 정상 기능으로 위장한 은폐, 합법적인 도구를 활용한 보안 우회, 그리고 생산성 압박을 받는 개발자의 심리를 완벽하게 이용했습니다.

AI 코딩 어시스턴트가 필수적인 인프라가 됨에 따라, 이들은 이제 가장 가치 있는 공격 목표가 되었습니다. 이제 문제는 "당신의 IDE 확장 프로그램이 안전한가"가 아니라, "그것이 악성임을 알 수 있는 가시성을 확보하고 있는가"입니다.
