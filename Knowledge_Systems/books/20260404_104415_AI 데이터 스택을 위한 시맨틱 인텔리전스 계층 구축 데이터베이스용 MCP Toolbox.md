# AI 데이터 스택을 위한 시맨틱 인텔리전스 계층 구축: 데이터베이스용 MCP Toolbox

완벽하게 모델링되고 테스트된 데이터로 가득 찬 대시보드를 팀원들에게 보여주며 의사결정을 내렸지만, 몇 주 후 결과가 예상과 달랐던 경험이 있을 것입니다. 데이터 팀은 데이터를 수집, 정제, 시각화하는 데 어느 때보다 능숙해졌지만, 데이터의 시맨틱(Semantics, 의미론)에 내재된 가정, 한계 및 조건을 전달하는 데는 여전히 어려움을 겪고 있습니다.

일례로 간사이 공항(Kansai Airports)의 여러 부서에서는 '승객'에 대한 정의가 서로 달랐습니다. 어떤 팀은 유료 고객만을 승객으로 간주했고, 다른 팀은 승무원과 유아를 포함했습니다. 이 핵심 지표에 대해 표준화된 정의를 내리는 것은 데이터를 사람이 즉시 활용할 수 있도록(People-ready) 만드는 데 도움이 되었습니다. 하지만 AI가 활용 가능한 데이터(AI-ready data)로 도약하는 과정은 더욱 험난합니다. AI 에이전트는 서로 다른 두 가지 '승객' 수치를 보았을 때 멈춰서 고민하지 않기 때문입니다.

오픈메타데이터(OpenMetadata)가 공유된 정의, 관계 및 문맥을 저장하는 시맨틱 라이브러리 역할을 한다면, 데이터베이스용 MCP Toolbox는 AI 에이전트가 기업 데이터에 대한 깊은 이해를 바탕으로 애플리케이션 코드와 테스트를 생성하고 실행할 수 있도록 지원합니다. 이 두 가지 오픈 소스 도구는 시맨틱 인텔리전스(Semantic Intelligence)를 활용해 메타데이터를 의미 있는 정보로 변환하고, AI가 신뢰할 수 있고 설명 가능하며 안전한 조치를 취할 수 있도록 돕습니다.

## 단계별 가이드

오픈메타데이터(OpenMetadata)는 도커(Docker)를 사용하여 로컬 환경에 신속하게 구축할 수 있습니다. 각 배포판에는 오픈메타데이터 플랫폼에 통합된 임베디드 MCP 서버(MCP Server)가 포함되어 있습니다. 이 MCP 서버는 기존 오픈메타데이터 API에 설정된 것과 동일한 권한 부여 엔진을 활용합니다. 이를 통해 모든 MCP 클라이언트는 설정된 역할과 정책을 준수하면서 읽기 또는 쓰기 작업을 위해 기존 오픈메타데이터 통합 환경과 인터페이스할 수 있습니다. 결과적으로 에이전트에게 데이터 운영을 효과적으로 자동화하는 데 필요한 특정 데이터 및 기능적 로직에 대한 적절한 액세스 권한이 부여됩니다.

### 1단계: 오픈메타데이터 다운로드

도커를 설치한 후 다음 명령어를 실행하여 오픈메타데이터의 Docker Compose 파일을 다운로드합니다.

```bash
curl -sL -o docker-compose-postgres.yml https://github.com/open-metadata/OpenMetadata/releases/download/1.12.3-release/docker-compose-postgres.yml
```

다운로드가 완료되면 다음 명령어로 오픈메타데이터와 종속 항목을 실행합니다.

```bash
docker compose -f docker-compose-postgres.yml up --d
```

이를 통해 로컬 머신에 오픈메타데이터 독립 실행형 인스턴스가 설치됩니다. 임베디드 MCP 서버 아키텍처는 기본적으로 MCP 서버를 통합하고 있으므로, 별도의 설정 없이도 AI 에이전트가 저장된 자산에 접근하여 데이터 작업을 자동화할 수 있도록 지원합니다.

### 2단계: 루커(Looker) 자격 증명 생성

루커 API의 `client_id`와 `client_secret`이 필요합니다. 이 정보는 루커 API 인증 설정에서 찾을 수 있습니다. 만약 "Manage" 버튼이 비활성화되어 있다면 루커 관리자에게 도움을 요청하세요.

### 3단계: Gemini CLI에서 MCP Toolbox 및 오픈메타데이터 MCP 설정

이 섹션에서는 Gemini CLI와 함께 MCP Toolbox를 설치합니다.

먼저 Gemini CLI를 설치합니다.

```bash
npm install -g @google/gemini-cli
```

그 다음, MCP Toolbox 서버와 오픈메타데이터 MCP 서버를 추가합니다.

```bash
gemini mcp add -t stdio looker-toolbox \
  npx -y @toolbox-sdk/server --stdio --prebuilt looker \
  -e LOOKER_BASE_URL=https://looker.example.com \
  -e LOOKER_CLIENT_ID="<YOUR_LOOKER_CLIENT_ID>" \
  -e LOOKER_CLIENT_SECRET="<YOUR_LOOKER_CLIENT_SECRET>" \
  -e LOOKER_VERIFY_SSL=true

gemini mcp add -t stdio OpenMetadata \
  npx -y mcp-remote http://localhost:8585/mcp \
  --auth-server-url=http://localhost:8585/mcp --client-id=OpenMetadata \
  --verbose --clean "--header" "Authorization:${AUTH_HEADER}" \
  -e AUTH_HEADER="Bearer <YOUR_OPENMETADATA_JWT_TOKEN>"
```

`~/.gemini/settings.json` 파일을 편집하여 서버가 정상적으로 추가되었는지 확인합니다. 설정 파일에는 루커와 오픈메타데이터의 명령어(command), 인자(args), 환경 변수(env) 정보가 포함되어야 합니다.

설정을 적용하기 위해 Gemini CLI를 재시작해야 할 수도 있습니다. 재시작 후 `/mcp`를 입력하면 사용 가능한 도구 목록(예: `looker-toolbox__query`, `mcp_OpenMetadata_search_metadata` 등)을 확인할 수 있습니다.

### 4단계: 루커 메타데이터를 오픈메타데이터로 수집(Ingest)

오픈메타데이터는 120개 이상의 클라우드 데이터 서비스에서 메타데이터를 수집합니다. 루커를 추가하면 여러 데이터 소스와 부서에 걸친 루커 대시보드, 차트, LookML 모델의 소유권과 리니지(Lineage, 데이터 계보)를 추적하고 관리할 수 있습니다.

1. 오픈메타데이터 설정에서 "Services"를 클릭하고 "Dashboards" 서비스 유형에서 루커를 선택합니다.
2. 서비스 이름을 입력한 후, 루커 연결을 위한 다음 정보를 입력합니다.
   - **Client ID / Secret**: SDK 인증을 위한 자격 증명.
   - **Host and Port**: 루커 인스턴스 URL.
   - **GitHub 정보**: LookML 파일이 포함된 저장소 소유자 및 이름.
   - **API Token**: 프라이빗 저장소 접근을 위한 퍼스널 액세스 토큰(PAT).

설정이 완료되면 오픈메타데이터가 루커 메타데이터 수집을 시작합니다.

## 활용 사례(Use Cases)

오픈메타데이터, 루커, MCP Toolbox를 통합하면 AI 에이전트가 데이터 스택과 루커 환경 내의 시맨틱을 포괄적으로 이해할 수 있습니다.

### 1. 자동 태그 동기화 (Automated Tag Synchronization)

다양한 도구에서 새로운 데이터 자산이 생성됨에 따라, 오픈메타데이터의 MCP 서버와 MCP Toolbox는 태그, 설명 및 기타 메타데이터를 도구 간에 자동으로 동기화합니다. 이는 데이터 생태계가 확장되더라도 시맨틱 문맥이 모든 도구에서 일관되게 유지되도록 보장합니다.

**프롬프트 예시:**
> "오픈메타데이터를 사용하여 `ecommerce.customers` LookML 데이터 모델의 리니지를 추적하고, 업스트림 데이터 소스의 해당 필드에 있는 모든 태그와 필드 설명을 적용해줘."

### 2. 커스텀 메타데이터 인사이트 생성 (Generating Custom Metadata Insights)

개발자들은 오픈메타데이터 MCP 서버를 사용해 자산 및 사용자 수와 같은 통계를 추출하고, 데이터 서비스 도입 현황을 추적하는 루커 차트를 생성할 수 있습니다. 에이전트는 프롬프트를 통해 데이터 환경의 건전성에 대한 깊은 통찰력을 제공하는 결과물을 신속하게 만들어냅니다.

이 사례를 구현하려면 오픈메타데이터의 통계를 루커에서 소비하기 전에 데이터베이스(예: Postgres)에 기록해야 합니다. MCP Toolbox를 사용하여 데이터베이스에 연결하고 Gemini CLI를 통해 다음과 같이 요청할 수 있습니다.

**프롬프트 예시:**
> "오픈메타데이터의 자산 유형별 총 자산 수를 계산하고, 이 통계를 저장할 테이블을 Postgres에 생성한 뒤 데이터를 기록해줘."

이후 Gemini CLI가 Postgres와 루커를 연결하여 새로운 LookML 프로젝트와 모델을 생성하고, 이를 기반으로 대시보드를 구축하도록 지시할 수 있습니다.

## 결론

오픈메타데이터와 MCP Toolbox를 통합하면 루커를 포함한 전체 데이터 스택에 시맨틱 인텔리전스를 도입할 수 있습니다. 이는 AI 에이전트와 워크플로우가 데이터 자산, 관계 및 문맥을 깊이 있게 이해할 수 있는 강력한 기능을 제공합니다. 이러한 오픈 소스 도구들의 결합은 메타데이터를 실질적인 의미로 전환하여 더욱 신뢰할 수 있는 데이터 기반의 행동을 가능하게 합니다.
