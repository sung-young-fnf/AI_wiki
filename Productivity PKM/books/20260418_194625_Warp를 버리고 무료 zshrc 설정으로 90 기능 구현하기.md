# Warp를 버리고 무료 `zshrc` 설정으로 90% 기능 구현하기

이 설정으로 20분 만에 Warp 사용을 중단했습니다. 여기에 해당 파일이 있습니다.

터미널에 월 18달러를 지불한다고 생각해보세요. 정말 말이 되는 이야기일까요? 2020년 이전이었다면 엔지니어들은 그런 생각에 기겁했을 것입니다.

Warp의 기능은 훌륭합니다. 유용한 자동 완성(autocomplete), 내장된 AI 기능 등 30만 명의 개발자가 사용하고 있죠. 하지만 몇 달 전 Warp의 기능을 직접 구현해보려고 시도했고, 단 20분밖에 걸리지 않았습니다.

여러분은 직접 알아낼 필요 없이 제가 사용 중인 정확한 설정을 공유하고자 합니다.

## 터미널에 추가 판매가 있어서는 안 됩니다

Warp의 새로운 UI는 마치 모바일 게임 같고, 본연의 역할을 잊은 듯합니다. 업그레이드를 유도하는 패널, 페이월 뒤에 숨겨진 기능들, 이제 터미널에 자체 홈페이지까지 생겼습니다.

기업이 수익을 창출해야 한다는 점은 이해합니다. 하지만 소중한 개발 환경을 열었는데 무언가를 판매하려 한다는 것은 옳지 않다고 생각합니다. 모바일 게임도 마찬가지였습니다. 처음에는 좋았고, 그다음에는 무료였으며, 이제는 형편없어졌습니다.

제 책상에는 모니터 하나만 있습니다. 잡동사니가 없습니다.

"정돈된 공간은 맑은 정신을 만든다."

터미널도 마찬가지여야 한다고 생각합니다. 터미널은 실행되고, 기다렸다가, 방해가 되지 않아야 합니다. Warp는 더 이상 저에게 그런 역할을 해주지 못했습니다.

## 독자의 직접적인 질문

며칠 전 저는 `cmux` 사용에 관한 글을 게시했습니다. 한 독자가 이런 질문을 남겼습니다.

"`Warp.dev`를 사용해보셨나요? 당신의 생각이 궁금합니다."

댓글 감사합니다. 이 글은 바로 당신을 위한 것입니다!

저도 한때 Warp를 사용했고, 한동안은 좋아했습니다. 하지만 어느 순간부터 제 작업에 방해가 되기 시작했습니다.

"그냥 업그레이드하면 되잖아."

무엇을 위해서요?
*   자동 완성(autocomplete) (이것은 항상 무료여야 합니다).
*   AI 기반 수정(AI correction) (이 기능은 거의 사용하지 않았습니다).
*   이상한 코딩 대체 통합 개발 환경(IDE)요?

아니요, 사양하겠습니다. 대부분의 기능은 어차피 오픈 소스(open-source)입니다.

## Warp를 대체하는 6가지 도구

`zsh-autosuggestions`는 Warp의 자동 완성(autocomplete)을 대체합니다. 입력하는 대로 제안을 해주며, 무료이고 간단합니다.

```bash
brew install zsh-autosuggestions zsh-syntax-highlighting
```

`fzf`는 Warp의 명령어 검색을 대체합니다. `Ctrl+R`을 누르면 실행했던 모든 명령어의 퍼지 검색(fuzzy search) 가능한 기록을 얻을 수 있습니다.

```bash
brew install fzf
$(brew --prefix)/opt/fzf/install
```

`Starship`은 Warp의 프롬프트(prompt) UI를 대체합니다.

```bash
brew install starship
```

`zoxide`는 Warp의 최근 디렉토리 기능을 대체합니다. `cd ~/dev/clients/project` 대신 `z project`를 입력하면 됩니다. 어느 정도 학습하는 기능도 있습니다.

```bash
brew install zoxide
```

`eza`는 `ls`를 대체합니다. 아이콘, 색상, 그리고 인라인(inline) `git` 상태를 보여줍니다.

```bash
brew install eza
```

`bat`은 `cat`을 대체합니다. 구문 강조(syntax highlighting), 줄 번호, 자동 페이지 넘김 기능을 제공합니다.

```bash
brew install bat
```

## 제가 사용하는 정확한 파일. 여러분도 사용할 수 있습니다.

여기에 제가 실제로 사용하는 `./.zshrc` 파일이 있습니다. 꾸며낸 버전이 아닙니다.

```zsh
PROMPT='→ '

eval "$(fnm env --use-on-cd --shell zsh)"

# Plugins
# Must come in first
source $(brew --prefix)/share/zsh-autocomplete/zsh-autocomplete.plugin.zsh
source $(brew --prefix)/share/zsh-autosuggestions/zsh-autosuggestions.zsh
source $(brew --prefix)/share/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh

source <(fzf --zsh)

eval "$(starship init zsh)"
```

`zoxide`와 `bat`이 없는 것을 보실 수 있을 겁니다. 제가 사용해봤는데, 필요성을 느끼지 못했습니다.

핵심은 모든 것을 설치하는 것이 아니라, 정말 필요한 것만 설치하는 것입니다 (이것이 Warp가 저에게 과하다고 느껴졌던 또 다른 이유입니다).

자유롭게 복사해서 `source ~/.zshrc`를 실행하고, 새로운 터미널 창을 열어보세요. 20분 만에 전환이 완료될 것입니다.

## Warp의 장점

Warp는 특정 유형의 사용자에게 유용한 제품입니다.

터미널 사용이 처음이라면 Warp는 좋은 시작점이 될 수 있습니다. AI 오류 설명, 시각적 명령어 블록(command blocks), 검색 가능한 히스토리(history) 등 모든 것이 별도의 설정이나 플러그인(plugins) 없이 바로 작동합니다. 그저 실행만 하면 됩니다. 저는 이 점을 충분히 이해합니다.

하지만 터미널을 몇 년간 사용해왔다면, 당신에게 필요한 것은 도움보다는 속도와 명확성입니다. (개인적으로는 터미널의 기본을 먼저 배우는 것이 중요하다고 생각합니다).

Warp가 잘하는 또 다른 점은 팀 기능입니다. 프롬프트(prompts) 공유, 공유 워크플로우(workflows), 노트북(notebooks) 등이 있죠. 하지만 제가 느끼는 문제는, 실제 환경에서 전체 엔지니어링 팀에게 특정 터미널 사용을 설득하는 것이 거의 불가능하다는 것입니다 (경험에서 우러나온 이야기입니다).

## 시작하기

*   **5분 여유가 있다면:** 위에 있는 제 `.zshrc`를 복사하고 `source ~/.zshrc`를 실행하세요. 플러그인(plugins)도 설치해야 합니다 (AI의 도움을 받으세요).
*   **15분 여유가 있다면:** 6가지 `brew install` 명령어를 모두 실행해보고 어떤 것이 마음에 드는지 확인하세요.
*   **30분 여유가 있다면:** `Starship` 프롬프트(prompt)를 사용자 정의해보세요.

개인적으로 터미널은 빠르고, 조용하며, 무료여야 한다고 생각합니다.
