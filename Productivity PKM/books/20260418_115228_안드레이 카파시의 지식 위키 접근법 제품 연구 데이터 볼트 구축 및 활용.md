# 안드레이 카파시의 지식 위키 접근법: 제품 연구 데이터 볼트 구축 및 활용

이번 달 초, 안드레이 카파시(Andrej Karpathy)는 'LLM 지식 베이스(LLM Knowledge Bases)'라는 제목의 화제성 게시물을 공유했습니다. 이 게시물에서 그는 다양한 연구 관심 주제에 대한 지식 베이스를 구축하는 과정을 설명합니다.

이 접근 방식은 제품 지식 베이스 구축에 완벽하며, 본 글에서는 이 방식을 활용하여 제품 연구 데이터 볼트(data vault)를 구축하는 방법을 소개할 것입니다.

## 이 접근 방식을 사용해야 하는 이유

이 접근 방식을 시도해야 하는 주된 이유는, AI가 모든 것을 기억하고 데이터를 다룰 때 집중력을 유지하는 인간처럼 느껴지게 하기 때문입니다.

어떤 분들은 "잠깐, 그럼 노트북LM(NotebookLM)은 어떤가요?"라고 반문할 수도 있습니다.

노트북LM의 주요 문제는 대화가 일시적(ephemeral)이라는 점입니다. 노트북LM을 사용하면 대화 후에 지식이 사라집니다. (참고: 노트북LM에서 대화를 저장할 수 있다는 것을 알지만, 제 요점은 노트북LM이 데이터를 다루는 접근 방식에 관한 것입니다. 질문에 답하기 위해 데이터를 탐색하고 대화가 끝나면 즉시 해당 대화를 잊어버립니다.) 반면 카파시의 접근 방식은 지식이 은행의 이자처럼 복리(compound)로 쌓이게 합니다.

## 지식 베이스 생성 과정

지식 베이스(knowledge base)는 위키(Wiki)입니다. 이 위키의 개별 문서는 디렉터리 구조 내의 .md 파일들로 구성됩니다.

지식 베이스 생성 및 활용 과정은 다음 3단계로 이루어집니다.

1.  **데이터 수집(Data ingest):** 원본 소스 문서(아티클, 논문, 저장소, 데이터셋, 이미지 등)를 `raw/` 디렉터리에 추가한 다음, 클로드 코드(Claude Code)를 사용하여 위키를 "컴파일(compile)"합니다.
2.  **위키 미리보기용 IDE(통합 개발 환경):** 옵시디언(Obsidian)을 IDE "프런트엔드(frontend)"로 사용하여 원본 데이터, 컴파일된 위키 및 파생된 시각화를 볼 수 있습니다.
3.  **Q&A 상호작용:** 클로드 코드를 사용하여 지식 베이스에 질문할 수 있습니다. 연구 데이터 볼트의 맥락에서, 이는 연구 결과의 부족한 부분을 파악하여 추가 연구를 수행하고 그 간극을 메울 수 있도록 돕습니다.

생성될 위키 디렉터리 구조는 다음과 같습니다.

*   `raw/` 폴더: 프로젝트와 관련된 원본 데이터 파일(PDF, 문서 등)을 포함합니다. 이 폴더는 읽기 전용으로, 클로드 코드가 데이터를 읽기만 하고 이 폴더에 어떠한 변경도 가하지 않습니다.
*   `wiki/` 폴더: AI 처리된 데이터를 포함합니다. 클로드 코드는 `raw/` 폴더에서 데이터를 가져와 분석한 후 `wiki/` 폴더에 정리합니다.
*   `CLAUDE.md`: 클로드 코드가 이 데이터를 다룰 때 따라야 할 규칙들의 집합입니다. 클로드 코드에 더 나은 컨텍스트(context)를 제공하고 싶다면 `raw/` 폴더에 대한 데이터 스키마(schema)도 `CLAUDE.md`에 기술할 수 있습니다.

아래에서 실제 프로젝트에서 이 방식이 어떻게 작동하는지 보여드리겠습니다.

## 안드레이 카파시의 지식 위키 접근 방식 사용의 장점과 단점

이 접근 방식에는 세 가지 주요 이점이 있습니다.

1.  **낮은 진입 장벽:** 5분 만에 프로젝트용 지식 베이스를 설정할 수 있으며, 특별한 기술이나 도구가 필요하지 않습니다. 옵시디언(Obsidian)과 클로드 코드(Claude Code)만 설정하면 데이터를 바로 활용할 수 있습니다.
2.  **저렴한 비용:** 일반적으로 연구 데이터 볼트에 사용되며 비용이 많이 드는 시맨틱 검색 RAG(Semantic Search RAG)와 비교할 때, 안드레이 카파시의 접근 방식은 주로 토큰(token) 사용에 비용이 발생합니다. 지속적인 컴퓨팅 또는 스토리지 비용은 없습니다. 또한 이 접근 방식은 매우 토큰 효율적이어서 데이터 볼트를 다룰 때 많은 클로드 코드 토큰을 소비하지 않습니다.
3.  **LLM의 데이터 관계 및 패턴 이해:** AI는 제공된 원본 데이터를 스캔하여 다양한 데이터 포인트 간의 연결점을 생성하는 지식 그래프(knowledge graph)를 만듭니다.

하지만 이 접근 방식에는 한 가지 주요 단점이 있습니다: **확장성(scalability)**입니다. 중간 규모 프로젝트(파일 300개 또는 100만 단어 이하 프로젝트에서 매우 유용하다고 판단)에는 잘 작동하지만, 데이터 파일이 증가함에 따라 AI가 이를 처리하는 데 드는 비용이 점점 더 많이 들 것입니다.

## 특정 제품을 위한 지식 위키를 만드는 방법

새 지식 위키를 만드는 과정은 간단합니다. 라이브러리를 설치할 필요 없이 클로드 코드에 안드레이 카파시의 프롬프트(원본 프롬프트 링크)를 입력하면 AI가 나머지를 처리합니다.

예시로, 저는 Roxy라는 SaaS 프로젝트의 연구 데이터 볼트를 만들 것입니다. Roxy는 보안 분석 도구이며, 현재 목표는 이 서비스에 가장 적합한 가격 모델을 찾는 것입니다. 이 목표를 달성하기 위해 시장 조사를 수행해야 하며, 클로드 코드가 이를 도울 것입니다.

다음은 거쳐야 할 5단계입니다.

### 1단계: 옵시디언(Obsidian) 설치 및 새 프로젝트 생성

옵시디언은 데이터 볼트를 생성하는 데 사용되는 무료 도구입니다. 로컬 머신에 다운로드해야 합니다.

로컬 머신에 도구를 설치한 후, 새 볼트를 생성하고 로컬 드라이브에서 위치를 선택해야 합니다.

### 2단계: 클로드 코드(Claude Code)에서 옵시디언 프로젝트 열기 및 데이터 볼트 생성 프롬프트 입력

옵시디언에서 새 프로젝트를 생성한 후, 클로드 코드를 사용하는 IDE(저는 VSCode를 사용할 예정)에서 프로젝트 디렉터리를 열어야 합니다.

이제 전체 과정에서 가장 "복잡한" 부분입니다. 다음 프롬프트를 복사해야 합니다.

```
[Original PROMPT from Andrej Karpathy]
You are LLM Wiki agent. Implement the exact idea I've shared with you.
Do it step-by-step: create the CLAUDE.md schema file with rules, set up
index.md and log.md, define folder conventions, and provide the first
ingest example. Every interaction should follow the schema.
```

그리고 다음 지침과 함께 클로드 코드 프롬프트 창에 제출합니다.

클로드 코드는 이 작업을 수행하고 방금 제공한 지침에 따라 프로젝트 디렉터리를 파일로 업데이트할 것입니다. 이 과정이 끝나면 클로드 코드는 첫 번째 소스를 수집하도록 안내할 것입니다.

### 3단계: 첫 번째 데이터 소스를 클로드 코드에 수집(Ingest)

저는 2026년 SaaS 가격 책정 현황에 대한 PDF 보고서를 제공할 것입니다. 이 보고서는 매우 상세하고 심층적인 여러 페이지로 구성되어 있습니다.

저는 이 보고서 PDF를 다운로드하여 프로젝트의 `raw/` 폴더에 저장할 것입니다 (참고: `assets/` 하위 폴더가 아닌 `raw/` 폴더에 저장할 것입니다).

다음으로 클로드 코드에 이를 수집하도록 프롬프트를 입력할 것입니다.

```
I’ve added the report hy_SaaS-und-AI-Pricing-Report_2026_ENG.pdf into the raw.
Ingest it 
```

### 4단계: 데이터 수집 결과 미리보기

데이터 수집 과정에서 클로드 코드는 데이터의 성격에 맞춰 여러 개의 `.md` 파일을 생성할 수 있습니다. 이는 개념(concept)과 개체(entity)에 대한 `.md` 파일을 생성하여 특정 데이터 포인트 간의 관계를 추적하기 쉽게 만듭니다. 제 경우, 4개의 새로운 개념과 몇 개의 개체를 생성하고 `index.md`, `log.md`, `CLAUDE.md`를 새로운 세부 사항에 맞게 업데이트했습니다.

옵시디언으로 프로젝트를 열면 프로젝트 내 지식에 대한 그래프 시각화(graph visualization)가 생성된 것을 볼 수 있습니다. 따라서 2026년 SaaS 가격대, 인기 있는 가격 모델 또는 최적의 가격 전략과 같은 특정 주제로 쉽게 들어갈 수 있습니다. 또 다른 장점은 위키 페이지 콘텐츠에서 원본 소스로의 교차 링크(cross-links) 및 참조를 바로 확인할 수 있다는 것입니다.

### 5단계: 연구의 부족한 부분(gap) 찾기

AI 기반 연구 지식 볼트의 큰 장점은 AI에게 특정 세부 사항을 분석하도록 프롬프트를 입력할 수 있다는 것입니다. 예를 들어, 제가 추가한 보고서가 현재 당면한 문제, 즉 SaaS 서비스에 대한 완벽한 가격 모델을 만드는 데 도움이 되는지 알고 싶습니다.

다음은 클로드 코드에 사용할 프롬프트입니다.

```
I want to create the perfect pricing model for my SaaS service, Roxy. 
It’s an automatic security vulnerabilities analyser.
Please check if I have enough data in existing sources to make an
informed decision
```

그리고 도구가 생성해 준 응답입니다.

보시다시피, 제가 제공한 보고서는 필요한 내용의 30%만을 다루고 있으며, 이 거대한 부족한 부분을 채우기 위해 몇 가지를 더 수집해야 할 것입니다. 이 지식 볼트의 장점은 `raw/` 폴더에 더 많은 데이터 소스를 추가하고 클로드 코드에 이를 위키에 추가하도록 요청할 수 있다는 것입니다.
