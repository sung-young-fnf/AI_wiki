100% 활용 못 하는 macOS Tahoe의 9가지 엄청나게 유용한 설정

애플 자체 기능 페이지에서도 언급되지 않는 숨겨진 설정들입니다.

저는 macOS Tahoe가 출시된 날부터 계속 사용해왔고, 첫 주 안에 모든 기능을 탐색했다고 생각했습니다. 새로운 리퀴드 글라스(Liquid Glass) 디자인, 재설계된 제어 센터(Control Center), 그리고 모든 기술 블로그가 출시일에 게시하는 일반적인 기능 소개 글들을 읽었죠. 저는 그 글들을 읽고 고개를 끄덕이며 모든 것을 파악했다고 생각했습니다.

하지만 전혀 그렇지 않았습니다.

지난 몇 달 동안, 저는 너무 깊이 숨겨져 있고 문서화가 제대로 되어 있지 않은 설정들을 우연히 발견했습니다. 애플이 이러한 설정의 존재를 누구에게도 알리지 않을 것이었다면, 왜 굳이 만들었는지 이해할 수 없을 정도였습니다.

여기서는 애플이 도입했지만 토글 버튼을 제공하지 않아 시각적으로 거슬리는 부분을 해결하는 터미널(Terminal) 명령어, 출시 첫날부터 아무런 UI 표시 없이 존재했던 스포트라이트(Spotlight) 팁, 그리고 연간 40달러를 지불해야 하는 앱을 대체할 수 있는 자동화 기능들을 다룰 것입니다.

이것들은 흔한 macOS 기사에 나오는 "다크 모드(Dark Mode) 켜기" 같은 팁이 아닙니다. 만약 이 중 절반이라도 이미 알고 계셨다면, 정말 대단하다고 생각할 것입니다.

### #1 단일 터미널 명령으로 자동으로 추가된 메뉴 막대 아이콘 제거하기

Tahoe로 업데이트한 후 제가 가장 먼저 알아차린 것 중 하나는 앱 메뉴가 달라졌다는 점입니다.

잘라내기, 복사, 붙여넣기, 모두 선택 등 모든 표준 동작 옆에 작은 SF 심볼(SF Symbol) 아이콘이 갑자기 생겼습니다. 처음에는 큰 문제가 아니었지만, 며칠이 지나자 메뉴가 복잡하고 일관성 없어 보여 불편하기 시작했습니다(일부 항목에는 아이콘이 있고 다른 항목에는 없어 애플이 방향을 결정하지 못한 것처럼 보였습니다).

답답한 점은 시스템 설정(System Settings) 어디에도 이 기능에 대한 토글(toggle)이 없다는 것입니다. 애플은 macOS 26에서 이 아이콘들을 시스템 전체에 추가했고, 모든 사람이 괜찮을 것이라고 가정했습니다. 대부분의 사람들은 아마 괜찮을 테지만, 지난 20년 동안 그랬던 것처럼 깔끔한 텍스트 전용 메뉴를 선호하는 사용자라면 곤란할 수 있습니다. 적어도 그렇게 생각할 것입니다.

터미널(Terminal)을 열고 다음 명령어를 실행하세요:

```
defaults write NSGlobalDomain NSMenuEnableActionImages -bool false
```

로그아웃했다가 다시 로그인하면 자동으로 할당된 모든 아이콘이 사라집니다. 메뉴는 다시 깔끔한 텍스트로 돌아갑니다. 개발자가 앱에 특별히 추가한 아이콘은 그대로 유지되므로, 이 명령어는 애플 시스템이 자동으로 삽입하는 아이콘만 제거합니다.

먼저 특정 앱에만 적용하고 싶다면 다음과 같이 지정할 수 있습니다:

```
defaults write com.apple.finder NSMenuEnableActionImages -bool NO
```

나중에 마음이 바뀌면 다음 명령어로 되돌릴 수 있습니다:

```
defaults delete NSGlobalDomain NSMenuEnableActionImages
```

이 명령어는 제가 찾기 전까지 몇 주 동안 MacRumors 포럼 스레드에 떠돌고 있었습니다. 애플이 UI를 만들지 않은 채로 얼마나 많은 숨겨진 스위치를 남겨두었는지 궁금하게 만드는 `defaults write` 명령어 중 하나입니다.

### #2 애플이 보여주는 것 외에 모든 SF 심볼을 폴더에 적용하기

Tahoe에서 파인더(Finder) 폴더 아이콘을 SF 심볼(SF Symbol)로 사용자화할 수 있다는 것을 아마 보셨을 것입니다. 폴더를 마우스 오른쪽 버튼으로 클릭하고 '폴더 사용자화(Customize Folder)'를 선택한 다음, 격자에서 아이콘을 선택하면 됩니다.

아마 깨닫지 못했을 사실은, 애플이 GUI에 보여주는 격자가 실제로 사용 가능한 것의 아주 작은 부분이라는 것입니다.

시스템에는 수천 개의 SF 심볼이 있습니다. 애플은 그 선택기에서 약 백여 개 정도만 노출합니다. 나머지는 완벽하게 작동하지만 인터페이스에서 완전히 숨겨져 있습니다. 공식 SF 심볼 앱에 나타나지 않는 애플의 비공개 내부 심볼을 포함하여, 터미널(Terminal)을 통해 모두 접근할 수 있습니다.

방법은 다음과 같습니다:

```
xattr -w 'com.apple.icon.folder#S' '{"sym":"camera.viewfinder"}' /path/to/folder
```

`camera.viewfinder`를 유효한 SF 심볼 이름으로 바꾸세요. 애플의 SF 심볼 앱(developer.apple.com에서 무료 다운로드)에서 전체 라이브러리를 찾아보고 거기서 찾은 어떤 심볼 이름이든 사용할 수 있습니다.

더 좋은 점도 있습니다. 이모지(emoji)도 사용할 수 있습니다:

```
xattr -w 'com.apple.icon.folder#S' '{"emoji":"🔍"}' /path/to/folder
```

심지어 임의의 텍스트도 가능합니다:

```
xattr -w 'com.apple.icon.folder#S' '{"emoji":"WIP"}' /path/to/folder
```

명령어의 `#S`는 `XATTR_FLAG_SYNCABLE`을 의미하며, 이는 사용자 정의 아이콘이 아이클라우드 드라이브(iCloud Drive)를 통해 다른 Mac에도 동기화된다는 뜻입니다. 따라서 한 번 설정하면 모든 Mac에서 나타납니다.

사용자 정의 아이콘을 제거하려면:

```
xattr -d 'com.apple.icon.folder#S' /path/to/folder
```

저는 이 기능을 사용하여 GUI가 허용하는 것보다 훨씬 더 구체적인 폴더 시스템을 만들고 있습니다. 제 글쓰기 프로젝트 폴더는 연구 폴더와 다른 심볼을 사용하고, 스크린샷 폴더와도 다릅니다. 이 모든 심볼은 마우스 오른쪽 클릭 선택기에서는 사용할 수 없었습니다. 작은 일이지만, 사이드바에서 수십 개의 폴더를 훑어볼 때는 시각적 구분이 생각보다 중요합니다.

### #3 리퀴드 글라스 탭을 이전 모습으로 되돌리기

솔직히 말해서, 대부분의 경우 리퀴드 글라스(Liquid Glass) 디자인은 점점 마음에 들고 있습니다. 반투명한 윈도우 크롬(window chrome), 배경화면에 맞춰지는 방식, 그리고 전반적인 분위기가 좋습니다.

하지만 탭은 어떨까요? 탭은 좀 다릅니다. 파인더(Finder), 사파리(Safari), 터미널(Terminal) 같은 앱에서 리퀴드 글라스 탭 스타일은 어떤 탭이 활성 상태인지 구별하기 어렵게 만듭니다. 특히 창 뒤에 밝은 배경화면이 있을 때는 더욱 그렇습니다. 제목 표시줄에서는 아름답게 보이는 반투명 효과가 8개의 파인더 탭 사이를 구분하려고 할 때는 가독성 문제가 됩니다.

애플 개발자 스티브 트라우튼-스미스(Steve Troughton-Smith)는 문서화되지 않은 `defaults` 키를 발견했습니다. 이 키는 탭만(오직 탭만) Tahoe 이전 모습으로 되돌리면서 나머지 리퀴드 글라스는 그대로 유지합니다. 이는 제가 원하던 정확한 방식입니다.

```
defaults write -g NSSolariumWindowTabs -bool NO
```

이것으로 끝입니다. 대부분의 앱에서는 로그아웃이 필요 없지만(일부는 재시작해야 할 수 있습니다), 윈도우 탭은 macOS Sequoia의 단색으로 명확하게 구분되는 스타일로 돌아가고, 다른 모든 것은 리퀴드 글라스 상태를 유지합니다.

되돌리려면:

```
defaults delete -g NSSolariumWindowTabs
```

"Solarium"이라는 키 이름은 리퀴드 글라스 윈도우 탭 시스템에 대한 애플의 내부 코드명인데, 이는 애플이 이 기능을 의도적으로 만들었지만 노출하지 않기로 선택했다는 것을 확인시켜주는 흥미로운 작은 정보입니다.

### #4 스포트라이트가 당신이 했던 모든 검색을 기억합니다

이 기능은 시각적으로 존재한다는 어떤 표시도 없기 때문에 저를 진심으로 놀라게 했습니다.

`Cmd+Space`로 스포트라이트(Spotlight)를 여세요. 아무것도 입력하지 마세요. 이제 위쪽 화살표(Up Arrow) 키를 누르세요.

터미널(Terminal)의 명령어 기록처럼, 이전 검색 기록이 검색 필드를 통해 하나씩 스크롤되기 시작합니다. 스포트라이트에 입력했던 모든 쿼리(query)는 저장되어 있으며, 키 한 번만 누르면 접근할 수 있습니다.

언제 이 기능이 추가되었는지, 왜 애플이 한 번도 언급하지 않았는지 모르겠습니다. "최근 검색(recent searches)" 레이블도 없고, 드롭다운 메뉴도 없으며, 이 기능이 존재한다는 어떤 시각적 힌트도 없습니다. 당신이 알거나 모르거나 둘 중 하나인데, 사람들이 이 기능을 발견했을 때 온라인에서 본 반응들을 보면 거의 아무도 모르는 것 같습니다.

이것은 동일한 것을 반복적으로 검색하는 사람에게는 엄청나게 유용합니다. 저는 매일 같은 프로젝트 폴더, 같은 앱, 같은 시스템 설정(System Settings) 패널을 여러 번 엽니다. 매번 "에너지 절약"이나 "블루투스"를 다시 입력하는 대신, 스포트라이트를 열고 찾을 때까지 위쪽 화살표를 누르기만 하면 됩니다.

이것은 목록의 다음 설정과 특히 잘 어울립니다.

### #5 앱과 웹사이트를 열지 않고도 내부를 검색하기

이것은 아마도 애플이 지금까지 선보인 스포트라이트(Spotlight) 기능 중 가장 강력할 것입니다. 그런데도 이에 대해 이야기하는 사람을 거의 보지 못했습니다.

스포트라이트에 앱 이름이나 웹사이트를 입력한 다음, `Tab` 키를 누르세요. 스포트라이트가 특정 범위(scoped) 검색 모드로 전환되어, 다음 쿼리는 전체 Mac이 아닌 해당 앱이나 웹사이트 내부를 검색합니다.

예를 들어, "사진(Photos)"을 입력하고 `Tab`을 누른 다음 "해변 휴가(beach vacation)"를 입력하면, 스포트라이트가 사진 보관함에서 해변 휴가 사진을 검색합니다. "아마존(Amazon)"을 입력하고 `Tab`을 누른 다음 "USB-C 케이블"을 입력하면 아마존을 직접 검색합니다. "사파리(Safari)"를 입력하고 `Tab`을 누르면 열려 있는 탭, 책갈피, 검색 기록을 검색할 수 있습니다. "메시지(Messages)"를 입력하고 `Tab`을 누르면 대화를 검색합니다.

여기서 `Tab` 키가 모든 중요한 역할을 합니다. `Tab` 키 없이 검색하면 Mac 전체를 검색하는 것이고, `Tab` 키를 사용하면 특정 컨텍스트(context) 내부를 검색하는 것입니다. 그리고 이 모든 과정은 스포트라이트 창에서 이루어지므로, 앱을 열고, 로딩을 기다리고, 검색창을 찾아서 클릭한 다음 쿼리를 입력하는 번거로운 과정을 거칠 필요가 없습니다. 5단계 과정이 2단계로 줄어드는 셈입니다.

저는 이 기능을 사진(Photos) 보관함과 메모(Notes)를 검색할 때 계속 사용하고 있습니다. 두 앱 모두 시작하고 로딩하는 데 시간이 걸리고, 내부 검색창이 항상 빠르지는 않습니다. 스포트라이트의 범위 지정 검색(scoped search)은 앱을 여는 것보다 빠르며, 믿기지 않겠지만 항상 그렇습니다.

### #6 클립보드 기록은 기본적으로 비활성화되어 있으며, 보존 설정은 이상한 곳에 숨겨져 있습니다

macOS Tahoe에 클립보드 기록(Clipboard History) 기능이 추가되었다는 사실은 모두가 알고 있습니다. 하지만 대부분의 사람들이 놓친 것은, 개인 정보 보호 조치로 기본 설정이 꺼져 있다는 점입니다. 따라서 수동으로 이 기능을 활성화하지 않았다면 실제로 이 기능을 사용하고 있지 않은 것입니다.

활성화 방법: 시스템 설정(System Settings) > 스포트라이트(Spotlight) > "클립보드 결과(Results from Clipboard)"를 켜세요. 또는 스포트라이트를 열고 `Cmd+4`를 눌러 클립보드 섹션으로 바로 이동하면 거기서 활성화하라는 메시지가 나타날 것입니다.

하지만 거의 아무도 모르는 부분이 있습니다. 애플이 macOS 26.1 업데이트를 배포했을 때, 동일한 스포트라이트 설정 패널에 조용히 보존(retention) 제어 기능을 추가했습니다. 이제 클립보드 기록이 얼마나 오래 유지될지 선택할 수 있습니다: 30분, 8시간(기본값), 또는 7일. 수동으로 "클립보드 기록 지우기(Clear Clipboard History)" 버튼도 있습니다.

이것은 생각보다 중요합니다. 클립보드 기록은 비밀번호, 신용카드 번호, 인증 코드, 비공개 메시지 등 클립보드를 통과하는 모든 정보를 캡처합니다.

편의성이 괜찮다면, 8시간 또는 7일로 설정하면 여러 문서 간에 복사 및 붙여넣기를 하는 작업 세션에서 정말 유용합니다. 개인 정보 보호에 더 신경 쓰는 분이라면, 30분으로 설정하여 긴 흔적을 남기지 않고도 이점을 누릴 수 있습니다.

참고로, 페이스트(Paste)나 매시(Maccy) 같은 대부분의 전용 클립보드 관리 앱에는 비밀번호 관리자 항목 및 민감한 필드를 자동으로 제외하는 내장 규칙이 있습니다. 애플의 기본 버전은 아직 그 정도의 세밀한 제어 기능을 제공하지 않으므로, 무엇이 저장되는지 인지하고 있는 것이 좋습니다.

### #7 여러 개의 제어 센터를 만들고 각각 고유한 아이콘을 지정할 수 있습니다

Tahoe의 새로운 제어 센터(Control Center)는 이전 버전에 비해 이미 큰 개선을 이루었지만, 대부분의 사람들은 모든 것이 한 패널에 몰아넣어져 있는 형태로 사용하고 있습니다. 하지만 그렇게 할 필요는 없습니다.

편집 모드에 진입하면 "+" 버튼이 나타나는데, 이를 통해 완전히 별개의 제어 센터를 만들 수 있습니다. 각각의 제어 센터는 메뉴 막대(menu bar)에 자신만의 사용자 정의 아이콘을 가지며, 고유한 제어 기능 세트를 갖습니다.

설정 방법은 다음과 같습니다: 제어 센터를 열고 하단의 "제어 기능 편집(Edit Controls)"을 클릭합니다. 메뉴 막대 상단에 "+" 버튼이 보일 것입니다. 이를 클릭하면 새로운 제어 센터 아이콘이 나타납니다. 원하는 아이콘(음표, 집, 톱니바퀴 등 무엇이든)을 할당한 다음, 해당 카테고리에 속하는 제어 기능만 드래그하여 넣을 수 있습니다.

저는 현재 두 개의 제어 센터를 사용하고 있습니다. 하나는 오디오 제어, 에어플레이(AirPlay), 지금 재생(Now Playing)을 포함합니다. 다른 하나는 디스플레이 밝기, 나이트 시프트(Night Shift), 집중 모드(Focus modes), 방해금지 모드(Do Not Disturb)를 포함합니다. 필요한 것을 찾기 위해 긴 패널을 스크롤하는 대신, 관련 아이콘을 클릭하면 모든 것이 바로 거기에 있습니다.

이는 여러 개의 데스크톱이나 여러 개의 브라우저 프로필을 사용하는 것과 같은 아이디어입니다. 컨텍스트(context)별 분리는 모든 것을 더 빨리 찾을 수 있게 해주며, 한 번 설정하고 나면 모든 것이 한데 뒤섞여 있던 상태를 어떻게 참아왔는지 궁금해질 것입니다.

### #8 잠금 화면 시계 사용자화는 가능하지만, 배경화면 설정 아래에 숨겨져 있습니다

Mac 잠금 화면(Lock Screen)의 시계 글꼴을 사용자화하고 싶다면 어디를 찾아볼까요? 아마 잠금 화면 설정이겠죠. 어쩌면 디스플레이(Display) 설정일 수도 있고, 아니면 외형(Appearance) 설정일 수도 있습니다.

정답은 배경화면(Wallpaper) 설정 아래에 있습니다.

시스템 설정(System Settings) > 배경화면 > 시계 모양(Clock Appearance). 여기서 6가지 글꼴 스타일 옵션과 시계 텍스트의 두께를 조절하는 굵기 슬라이더(weight slider)를 사용할 수 있습니다. 또한 사용자 정의된 시계가 잠금 화면에만 표시될지, 아니면 잠금 화면과 화면 보호기(Screen Saver) 모두에 표시될지 선택할 수 있습니다.

애플이 왜 이 기능을 잠금 화면 설정이 아닌 배경화면 설정 아래에 숨겼는지 이해할 수 없습니다. 잠금 화면 설정 패널은 존재하며, 다른 잠금 화면 관련 옵션들도 있습니다. 그리고 시계는 문자 그대로 잠금 화면에서 가장 눈에 띄는 요소입니다. 하지만 그 사용자화 기능은 시스템 설정의 완전히 다른 섹션에 있어서 거의 아무도 찾지 못하고 있습니다.

작은 미용적인 부분이지만, Mac의 외형에 신경 쓰는 사람이라면(그리고 이처럼 숨겨진 macOS 설정에 대한 글을 읽고 있다면 아마도 그럴 것입니다), 각 글꼴 스타일을 시도해보고 배경화면에 가장 잘 어울리는 것을 선택하는 데 30초를 투자할 가치가 있습니다.

### #9 단축어는 이제 폴더를 감시하고 자동으로 실행될 수 있으며, 이는 사실상 Hazel을 대체합니다

이것은 제가 Mac을 사용하는 방식을 진정으로 바꾼 기능입니다.

Mac용 단축어(Shortcuts) 앱은 이제 자동화된 트리거(automated triggers)를 지원하며, 가장 강력한 기능 중 하나는 폴더 감시(folder watching)입니다. 특정 폴더에 파일이 추가되거나, 제거되거나, 변경될 때마다 자동으로 실행되는 단축어를 설정할 수 있습니다. 수동으로 트리거하거나 아무것도 클릭할 필요 없이 그냥 실행됩니다.

설정 방법: 단축어 앱을 열고, 자동화(Automations) 탭으로 이동한 다음, "+"를 클릭하고 트리거 유형을 선택하세요. 폴더 감시의 경우, 모니터링할 폴더를 선택한 다음 이벤트(파일 추가됨, 제거됨, 수정됨)를 선택합니다.

제가 실제로 사용하고 있는 몇 가지 예시입니다:

제 다운로드(Downloads) 폴더에는 파일 유형별로 파일을 정렬하는 자동화 기능이 있습니다. 새 파일이 나타나면 단축어가 확장자를 확인하고 올바른 하위 폴더로 이동시킵니다. PDF는 문서(Documents)로, 이미지는 스크린샷(Screenshots)으로 이동하고, DMG 파일은 자동으로 마운트(mount)되며, 설치가 끝나면 설치 프로그램은 삭제됩니다.

또한 스크린샷(Screenshots) 폴더에는 모든 WEBP 파일을 JPEG로 자동 변환하는 자동화 기능도 있습니다. 웹 스크린샷을 많이 찍거나 인터넷에서 이미지를 저장하는 경우, 이미지 파일의 절반이 모든 앱에서 지원되지 않는 형식일 때 얼마나 성가신 일인지 아실 것입니다. 이 자동화 기능은 파일이 도착하는 즉시 이를 해결해줍니다.

폴더 감시가 유일한 트리거는 아닙니다. 하루 중 특정 시간, 외부 디스플레이 연결, 배터리 잔량, Wi-Fi 네트워크 연결, 집중 모드(Focus mode) 활성화, 또는 캘린더 이벤트 수신을 기반으로 자동화를 설정할 수 있습니다. 외부 디스플레이 트리거는 특히 유용합니다. 저는 책상에 모니터를 연결할 때마다 작업 집중 모드를 활성화하고 다크 모드(Dark Mode)로 전환하는 자동화를 설정해두었습니다.

헤이즐(Hazel, 42달러)을 사용해본 적이 있다면, 이것은 사실상 macOS에 무료로 내장된 동일한 개념입니다. 규칙 복잡성 측면에서는 헤이즐만큼 강력하지는 않지만, 가장 일반적인 폴더 자동화 작업에 필요한 모든 것을 처리해줍니다.
