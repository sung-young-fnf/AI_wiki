## 컨텍스트 엔지니어링을 위한 에이전트 스킬

프로덕션 수준의 AI 에이전트 시스템 구축을 위한 컨텍스트 엔지니어링(context engineering) 원칙에 중점을 둔 에이전트 스킬(Agent Skills)의 포괄적인 오픈 컬렉션입니다. 이 스킬들은 모든 에이전트 플랫폼에서 에이전트의 효율성을 극대화하기 위해 컨텍스트(context)를 큐레이션하는 기술과 과학을 가르칩니다.

### 컨텍스트 엔지니어링(Context Engineering)이란 무엇인가요?

컨텍스트 엔지니어링은 언어 모델(Language Model)의 컨텍스트 창(context window)을 관리하는 분야입니다. 효과적인 지침을 만드는 데 초점을 맞추는 프롬프트 엔지니어링(prompt engineering)과 달리, 컨텍스트 엔지니어링은 시스템 프롬프트(system prompts), 도구 정의(tool definitions), 검색된 문서(retrieved documents), 메시지 기록(message history), 도구 출력(tool outputs) 등 모델의 제한된 어텐션 예산(attention budget)에 들어가는 모든 정보의 전반적인 큐레이션(holistic curation)을 다룹니다.

근본적인 과제는 컨텍스트 창이 원시 토큰 용량(raw token capacity)이 아니라 어텐션 메커니즘(attention mechanics)에 의해 제약을 받는다는 점입니다. 컨텍스트 길이가 증가함에 따라 모델은 "중간 실종(lost-in-the-middle)" 현상, U자형 어텐션 곡선(U-shaped attention curves), 어텐션 부족(attention scarcity)과 같은 예측 가능한 성능 저하 패턴을 보입니다. 효과적인 컨텍스트 엔지니어링은 원하는 결과가 나올 가능성을 극대화하는 최소한의 고품질 신호 토큰(high-signal tokens) 세트를 찾는 것을 의미합니다.

### 인정

이 저장소는 정적 스킬 아키텍처(static skill architecture)에 대한 기초 작업으로 학술 연구에 인용되었습니다.

"정적 스킬은 잘 알려져 있지만 [Anthropic, 2025b; Muratcan Koylan, 2025], MCE는 이를 동적으로 발전시킨 최초의 연구 중 하나이며, 수동 스킬 엔지니어링과 자율적 자기 개선을 연결합니다."
— Meta Context Engineering via Agentic Skill Evolution, 베이징 대학 일반 인공지능 국가 핵심 연구소 (2026)

### 스킬 개요

#### 기초 스킬 (Foundational Skills)

이 스킬들은 모든 후속 컨텍스트 엔지니어링 작업을 위해 필요한 기초적인 이해를 확립합니다.

| 스킬 (Skill) | 설명 (Description) |
|---|---|
| `context-fundamentals` | 컨텍스트가 무엇인지, 왜 중요한지, 에이전트 시스템에서 컨텍스트의 해부학적 구조 이해 |
| `context-degradation` | 컨텍스트 실패 패턴 인식: 중간 실종(lost-in-middle), 오염(poisoning), 방해(distraction), 충돌(clash) |
| `context-compression` | 장기 실행 세션(long-running sessions)을 위한 압축 전략 설계 및 평가 |

#### 아키텍처 스킬 (Architectural Skills)

이 스킬들은 효과적인 에이전트 시스템을 구축하기 위한 패턴과 구조를 다룹니다.

| 스킬 (Skill) | 설명 (Description) |
|---|---|
| `multi-agent-patterns` | 오케스트레이터(orchestrator), 피어 투 피어(peer-to-peer), 계층적 멀티 에이전트 아키텍처 마스터 |
| `memory-systems` | 단기, 장기, 그래프 기반 메모리 아키텍처 설계 |
| `tool-design` | 에이전트가 효과적으로 사용할 수 있는 도구(tools) 구축 |
| `filesystem-context` | 동적 컨텍스트 탐색, 도구 출력 오프로딩, 계획 영속화를 위한 파일 시스템 사용 |
| `hosted-agents` | **(NEW)** 샌드박스 VM(sandboxed VMs), 사전 구축된 이미지(pre-built images), 멀티플레이어 지원, 다중 클라이언트 인터페이스를 통해 백그라운드 코딩 에이전트 구축 |

#### 운영 스킬 (Operational Skills)

이 스킬들은 에이전트 시스템의 지속적인 운영 및 최적화를 다룹니다.

| 스킬 (Skill) | 설명 (Description) |
|---|---|
| `context-optimization` | 압축(compaction), 마스킹(masking), 캐싱(caching) 전략 적용 |
| `evaluation` | 에이전트 시스템을 위한 평가 프레임워크 구축 |
| `advanced-evaluation` | LLM-as-a-Judge 기법 마스터: 직접 점수 매기기(direct scoring), 쌍별 비교(pairwise comparison), 루브릭 생성(rubric generation), 편향 완화(bias mitigation) |

#### 개발 방법론 (Development Methodology)

이 스킬들은 LLM(Language Model) 기반 프로젝트 구축을 위한 메타 수준의 실천 방안을 다룹니다.

| 스킬 (Skill) | 설명 (Description) |
|---|---|
| `project-development` | 아이디어 구상부터 배포까지 LLM 프로젝트 설계 및 구축 (태스크-모델 적합성 분석(task-model fit analysis), 파이프라인 아키텍처, 구조화된 출력 설계 포함) |

#### 인지 아키텍처 스킬 (Cognitive Architecture Skills)

이 스킬들은 합리적인 에이전트 시스템을 위한 공식적인 인지 모델링을 다룹니다.

| 스킬 (Skill) | 설명 (Description) |
|---|---|
| `bdi-mental-states` | **(NEW)** 심의적 추론(deliberative reasoning) 및 설명 가능성(explainability)을 위해 공식 BDI 온톨로지 패턴(BDI ontology patterns)을 사용하여 외부 RDF 컨텍스트(external RDF context)를 에이전트의 정신 상태(신념, 욕망, 의도: beliefs, desires, intentions)로 변환 |

### 설계 철학 (Design Philosophy)

#### 점진적 공개 (Progressive Disclosure)

각 스킬은 효율적인 컨텍스트 사용을 위해 구조화되어 있습니다. 에이전트는 시작 시 스킬 이름과 설명만 로드하며, 관련 작업에 대해 스킬이 활성화될 때만 전체 콘텐츠를 로드합니다.

#### 플랫폼 불가지론 (Platform Agnosticism)

이 스킬들은 특정 공급업체 구현이 아닌, 전이 가능한 원칙에 중점을 둡니다. 이 패턴들은 Claude Code, Cursor 및 스킬을 지원하거나 사용자 정의 지침을 허용하는 모든 에이전트 플랫폼에서 작동합니다.

#### 실용적인 예시를 통한 개념적 기반 (Conceptual Foundation with Practical Examples)

스크립트와 예시는 특정 종속성 설치 없이 다양한 환경에서 작동하는 Python 의사 코드(pseudocode)를 사용하여 개념을 시연합니다.

#### 디지털 브레인 스킬 예시 (Digital Brain Skill Example)

`digital-brain-skill` 예시는 포괄적인 스킬 적용을 보여주는 완전한 개인 운영 체제입니다.

*   **점진적 공개 (Progressive Disclosure):** 3단계 로딩 (`SKILL.md` → `MODULE.md` → 데이터 파일)
*   **모듈 분리 (Module Isolation):** 6개의 독립적인 모듈 (신원(identity), 콘텐츠(content), 지식(knowledge), 네트워크(network), 운영(operations), 에이전트(agents))
*   **추가 전용 메모리 (Append-Only Memory):** 에이전트 친화적인 파싱을 위한 스키마 우선 라인(schema-first lines)이 있는 JSONL 파일
*   **자동화 스크립트 (Automation Scripts):** 4개의 통합 도구 (`weekly_review`, `content_ideas`, `stale_contacts`, `idea_to_draft`)

모든 아키텍처 결정을 특정 스킬 원칙에 매핑하는 `HOW-SKILLS-BUILT-THIS.md` 파일의 상세한 추적 가능성(traceability)을 포함합니다.

#### LLM-as-a-Judge 스킬 예시 (LLM-as-a-Judge Skills Example)

`llm-as-judge-skills` 예시는 다음을 보여주는 완전한 TypeScript 구현입니다.

*   **직접 점수 매기기 (Direct Scoring):** 루브릭(rubric) 지원과 함께 가중치 기준에 따라 응답 평가
*   **쌍별 비교 (Pairwise Comparison):** 위치 편향 완화(position bias mitigation)를 통해 응답 비교
*   **루브릭 생성 (Rubric Generation):** 도메인별 평가 표준 생성
*   **EvaluatorAgent:** 모든 평가 기능을 결합한 상위 수준 에이전트

#### Book SFT 파이프라인 예시 (Book SFT Pipeline Example)

`book-sft-pipeline` 예시는 어떤 작가의 스타일로든 글을 쓰도록 소형 모델(8B)을 훈련하는 것을 보여줍니다.

*   **지능형 분할 (Intelligent Segmentation):** 최대 훈련 예시를 위한 오버랩(overlap)이 있는 2단계 청킹(two-tier chunking)
*   **프롬프트 다양성 (Prompt Diversity):** 암기 방지 및 스타일 학습 강화를 위한 15개 이상의 템플릿
*   **Tinker 통합 (Tinker Integration):** 총 2달러 비용으로 완전한 LoRA(Low-Rank Adaptation) 훈련 워크플로우
*   **검증 방법론 (Validation Methodology):** 콘텐츠 암기가 아닌 스타일 전송을 증명하는 최신 시나리오 테스트

### 구조 (Structure)

각 스킬은 에이전트 스킬 사양(Agent Skills specification)을 따릅니다.

```
skill-name/
├── SKILL.md              # 필수: 지침 + 메타데이터
├── scripts/              # 선택 사항: 개념을 시연하는 실행 가능한 코드
└── references/           # 선택 사항: 추가 문서 및 자료
```

정식 스킬 구조는 `template` 폴더를 참조하십시오.
