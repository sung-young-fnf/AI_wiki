Agentic AI — 빠르고 실용적인 가이드
CrewAI를 활용하여 나만의 다중 에이전트 시스템을 구축하는 실습 튜토리얼과 Agentic AI의 실제 작동 방식 설명

에이전트형 AI (Agentic AI)는 2025년에 관심 주기(hype cycle)에 확고히 진입했습니다. 이전에는 기계가 포켓몬 게임을 하는 방식 정도로만 대중적 대화에 등장했지만, 이제 마이크로소프트와 같은 기술 제공 기업과 소매, 여행과 같은 기술 소비 산업 모두 AI 에이전트에 상당한 투자를 하고 있습니다.

AI 에이전트란 무엇일까요? 에이전트는 일반적으로 환경을 인식하고 목표 달성을 위해 행동하는 시스템입니다. 현대적 맥락에서 AI 에이전트는 두뇌(바람직한 결과를 얻기 위한 일련의 단계를 결정하는 대규모 언어 모델(Large Language Model, LLM))와 손(도구에 대한 접근 권한)을 가집니다. 이러한 도구를 통해 에이전트는 인터넷을 검색하거나 내부 데이터베이스를 쿼리하여 데이터를 수집할 수도 있고, 이메일을 보내거나 예약 시스템을 업데이트할 수도 있습니다.

이 글에서는 호텔 회사를 위한 에이전트형 시스템을 만들 것입니다. 수익 관리를 위해 대시보드와 임시 분석(ad hoc analytics)을 넘어, 우리의 에이전트는 예약이 예상보다 저조하게 들어오고 있음을 감지하고, 그 이유를 파악한 다음 전략적 권고 사항을 제공할 것입니다.

가상의 호텔 회사를 위해 하나의 집계 테이블(aggregate table)로 구성된 간단한 로컬 데이터베이스를 만들 것입니다. 이 테이블은 각 목적지와 월별로 현재까지의 실제 예약 건수와 예상 예약 건수, 최근 진행된 가격 인하 내역, 그리고 남아있는 미판매 객실의 평균 1박 가격을 추적합니다.

```python
import sqlite3

def setup_database():
    conn = sqlite3.connect('company_data.db')
    cursor = conn.cursor()

    # 1. Create the Aggregate Table
    cursor.execute('''
    DROP TABLE IF EXISTS destination_performance;
    ''')
    cursor.execute('''
    CREATE TABLE IF NOT EXISTS destination_performance (
        destination_id TEXT PRIMARY KEY,
        destination_name TEXT,
        month TEXT,
        confirmed_bookings INTEGER,
        predicted_bookings INTEGER,
        avg_price_drop_14d REAL,
        avg_remaining_nightly_price REAL,
        currency TEXT
    )
    ''')

    # 2. Insert the "Problem" Data
    # Scenario: We predicted 500 bookings. We have 320. 
    # We dropped prices by $45, but bookings aren't moving.
    data = [
        ('IT-TUS', 'Tuscany, Italy', 'March 2026', 320, 500, 45.00, 210, 'USD'),
        ('JP-KYO', 'Kyoto, Japan', 'March 2026', 480, 450, 0.00, 180, 'USD'),
        ('FR-PAR', 'Paris, France', 'March 2026', 600, 610, 10.00, 170, 'USD')
    ]

    cursor.executemany('INSERT OR REPLACE INTO destination_performance VALUES (?,?,?,?,?,?,?,?)', data)
    conn.commit()
    conn.close()
    print("Database 'company_data.db' initialized with aggregate performance data.")
```
이 코드는 `company_data.db` 데이터베이스를 집계된 성과 데이터로 초기화합니다.

### 우리의 '두뇌' 소개

우리의 에이전트형 시스템은 대규모 언어 모델(LLM) 형태의 두뇌를 필요로 할 것입니다. 이는 로컬에 저장된 LLM일 수도 있지만, 우리는 Gemini를 사용할 예정입니다. 여기에서 API 키를 생성(또는 기존 키 확인)하기만 하면 됩니다. 결제 설정을 하지 않으면 매우 엄격한 호출 제한(rate limit)이 적용되므로, '사용 및 결제' 탭으로 이동하여 프로젝트에 대한 결제 수단을 설정하는 것이 좋습니다(저는 'Default Gemini Project'를 사용하고 있습니다). 결제 수단을 제공하는 것이 요금이 청구된다는 의미는 아닙니다. 현재 Google은 첫 90일 동안 300달러의 크레딧을 제공하며, 다음과 같이 약속합니다:

"정식 종량제(pay-as-you-go) Google Cloud 계정을 활성화하거나 선결제를 선택해야만 요금이 청구되기 시작합니다."

이제 이 두뇌를 정의함으로써 메인 스크립트를 시작할 수 있습니다.

```python
import os

# =============================================================================
# 1. CONFIGURATION
# =============================================================================

# ENTER YOUR KEYS HERE
os.environ["GOOGLE_API_KEY"] = "PUT API KEY HERE"

# Initialize the Gemini Model
gemini_llm = LLM(
    model="gemini/gemini-flash-latest",
    api_key=os.environ["GOOGLE_API_KEY"],
    temperature=0.2
)
```

`gemini_llm.call("Test Prompt")`를 사용하여 모델에 테스트 프롬프트를 보내 모든 것이 올바르게 작동하는지 확인할 수 있습니다. 모든 것이 잘 작동한다면 Gemini가 대화하듯 응답하여 시스템 준비 상태를 알려줄 것입니다.

### 첫 번째 팀원 소개

우리는 CrewAI라는 프레임워크를 사용할 것입니다. 이는 AI 에이전트 팀을 구축하고 각 에이전트에게 역할과 사용할 수 있는 도구 세트를 부여할 수 있는 직관적인 접근 방식입니다.

우리의 첫 번째 팀원은 내부 성과 분석가(Internal Performance Analyst)입니다. 그의 역할은 데이터베이스를 검토하여 예약 진행 속도에 문제가 있는지 감지하는 것입니다. 우선, 그들이 사용할 도구를 만들어야 합니다.

```python
import sqlite3
from crewai.tools import tool

# =============================================================================
# 2. TOOL DEFINITIONS
# =============================================================================

class InternalDataTools:
    
    @tool("SQL Query Tool")
    def query_internal_database(query: str):
        """
        Queries the local 'company_data.db' for destination performance.
        Useful for finding confirmed vs predicted bookings and recent price drops.
        """
        try:
            # Connect to the local DB we just made
            conn = sqlite3.connect('company_data.db')
            cursor = conn.cursor()
            cursor.execute(query)
            result = cursor.fetchall()
            conn.close()
            return f"Database Result: {str(result)}"
        except Exception as e:
            return f"SQL Error: {str(e)}"
```

이것은 CrewAI에게 이 함수를 에이전트가 사용할 도구로 간주하도록 지시하는 `@tool` 데코레이터가 있는 사용자 지정 도구입니다. 이 도구는 직관적이며, 로컬 데이터베이스에 입력 쿼리를 실행하기만 합니다. 독스트링(docstring)은 가상 팀원이 이 도구가 무엇을 하는지 이해하는 데 사용될 것이므로 중요합니다.

이제 분석가를 온보딩할 시간입니다!

```python
from crewai import Agent, Task, Crew, Process, LLM

# =============================================================================
# 3. AGENT DEFINITIONS
# =============================================================================

# 1. The Analyst (Internal Data)
analyst = Agent(
    role='Internal Performance Analyst',
    goal='Identify underperforming destinations using internal database metrics.',
    backstory="""You are a senior data analyst. You look at the 'bookings' table.
    You pay special attention to the gap between 'Confirmed' and 'Predicted' bookings, 
    and identify what recent price drops we have already implemented.""",
    tools=[InternalDataTools.query_internal_database],
    llm=gemini_llm,  # Passing the fixed LLM object
    verbose=True,
    tracing=True
)
```

이제 배경 스토리를 가진 내부 성과 분석가(Internal Performance Analyst)가 생겼습니다. 가장 중요하게도, 우리는 그에게 사용할 수 있는 도구를 부여했으며, 앞서 정의한 Gemini LLM 형태의 두뇌를 주었습니다. 다음으로, 첫 출근일에 그가 수행해야 할 작업이 무엇인지 알려줘야 합니다.

```python
# Task 1: Check Internal Data
task_internal = Task(
    description="""
    Check the internal database for Tuscany. 
    Report on the variance between Predicted and Confirmed bookings, 
    and note if we have been discounting recently.
    """,
    expected_output="Internal Performance Report with booking numbers and price drop metrics.",
    agent=analyst
)
```

마지막으로, 분석가를 업무에 투입할 수 있습니다.

```python
revenue_management_squad = Crew(
    agents=[analyst],
    tasks=[task_internal],
    process=Process.sequential,
    verbose=True
)

print("### KICKING OFF REVENUE MANAGEMENT SQUAD ###")
final_result = revenue_management_squad.kickoff()

print("\n\n########################")
print("## ANALYST REPORT ##")
print("########################\n")
print(final_result)
```

여기서 우리는 내부 성과 분석가라는 단 한 명의 팀원으로 구성된 팀을 정의했으며, 이 1인 팀에게 내부 데이터베이스를 확인하고 보고하는 한 가지 작업을 요청했습니다.

이 코드를 실행하면 콘솔(또는 노트북 사용 시 셀)에 내부 성과 분석가가 생각하고 행동하는 과정을 보여주는 일련의 결과물이 나타날 것입니다. 제 경우, 그의 첫 번째 생각과 행동은 다음과 같았습니다.

좋습니다. 분석가는 유능한 인간 분석가처럼 데이터베이스의 테이블을 확인했습니다. 다음으로 그는 테이블에 어떤 열이 있는지, 그리고 어떤 열이 자신의 작업을 수행하는 데 도움이 될지 확인하고자 할 것입니다.

제 에이전트는 5번의 생각과 행동 반복을 거쳐 최종 답변을 도출했습니다.

만약 제가 더 많은 정보를 미리 제공했다면 이 과정은 훨씬 빨랐을 것입니다. 제 AI 분석가는 테이블 이름과 열 이름을 찾아야 했고, 실제 데이터가 'Tuscany, Italy'로 저장되어 있는데 'Tuscany'로 쿼리하여 어려움을 겪었습니다. 실제 상황에서는 도구를 더 구체적으로 만들 수 있었을 것입니다. 예를 들어, 목적지 이름만 입력으로 받는 하드코딩된 쿼리를 사용하거나, 데이터베이스의 각 테이블 스키마를 구체적으로 반환하는 두 번째 도구를 제공할 수도 있었습니다. 또는 도구의 독스트링(docstring)이나 작업 정의에 더 정확한 지침을 제공할 수도 있었습니다. 이 모든 것은 인간 분석가의 관리자로서 제가 할 수 있었던 일들입니다.

### 실제로는 어떤 일이 일어날까요?

이것은 정말 흥미롭지만, 그 이면의 현실은 겉보기만큼 마법 같지는 않습니다.

이제 우리는 Chat GPT와 같은 LLM에 모두 익숙합니다. LLM과 대화할 때, 그들은 답변에서 다음 토큰을 순차적으로 예측하여 응답합니다.

에이전트형 AI는 ReAct (추론 + 행동)라는 특정 프롬프트 엔지니어링 (prompt engineering) 기법에 의존합니다. 이것이 LLM을 추론하고, 행동하고, 관찰하고, 반복하는 에이전트처럼 작동하게 만드는 것입니다. 우리의 LLM이 이렇게 행동하는 데에는 두 가지 이유가 있습니다.

*   현대의 LLM은 에이전트형 행동 데이터셋으로 미세 조정되었습니다(fine-tuned). 이는 사용자가 문제에 대해 생각하고 도구를 선택하도록 요청했을 때, 기반 LLM에 올바른 응답이 어떤 모습인지에 대한 수백만 개의 완전한 예시가 주입되었음을 의미합니다.
*   CrewAI는 LLM에게 응답 구조를 명확히 지시하는 초기 프롬프트를 제공합니다.

다음 코드를 통해 CrewAI가 분석가의 LLM 두뇌로 무엇을 보내려 하는지 확인할 수 있습니다.

```python
from crewai.utilities.prompts import Prompts

# Create the prompt generator
prompt_generator = Prompts(
    agent=analyst,
    has_tools=len(analyst.tools) > 0,
    use_system_prompt=analyst.use_system_prompt
)

# Generate and inspect the actual prompt
generated_prompt = prompt_generator.task_execution()

# Print the complete system prompt that will be sent to the LLM
if "system" in generated_prompt:
    print("=== SYSTEM PROMPT ===")
    print(generated_prompt["system"])
    print("\n=== USER PROMPT ===")
    print(generated_prompt["user"])
else:
    print("=== COMPLETE PROMPT ===")
    print(generated_prompt["prompt"])

# You can also see how the task description gets formatted
print("\n=== TASK CONTEXT ===")
print(f"Task Description: {task_internal.description}")
print(f"Expected Output: {task_internal.expected_output}")
```

여기에는 `{tools}`와 같은 자리표시자(placeholder)가 있는데, 이는 우리가 작업을 시작할 때만 채워집니다. 우리의 경우, 실제 프롬프트에는 다음 내용이 포함됩니다.

시스템 프롬프트(system prompt)는 분석가의 두뇌가 매번 응답할 때 어떻게 행동해야 하는지에 대한 전반적인 지시입니다. 이러한 명시적인 지시와 함께, LLM이 유사한 지시가 주어졌을 때 올바른 응답이 어떤 모습인지에 대한 많은 예시를 학습했다는 사실은 LLM이 특정 추론을 수행하고 사용할 도구를 선택하도록 유도합니다. 이는 LLM이 자신의 추론 과정을 상세하게 서술하도록 강제함으로써 올바른 답변을 찾아낼 가능성을 높이는 Chain-of-Thought (사고의 연쇄) 프롬프팅의 진화된 형태입니다.

따라서 내부적으로 CrewAI는 본질적으로 while 루프 내에서 작동하는 매우 정교한 문자열 조작기에 불과합니다. 이는 사용 가능한 도구 정의를 포함한 시스템 프롬프트를 분석가의 LLM 두뇌로 보내 작업을 시작합니다. 이어서 작업 설명을 전달합니다. 또한 Gemini에게 `\nObservation:` 시퀀스를 작성하는 것을 감지하는 즉시 텍스트 생성을 중단하고 지금까지 생성된 텍스트를 반환해야 한다고 지시합니다(이는 노트북에서 `gemini_llm.stop_sequences`를 쿼리하여 확인할 수 있습니다).

그러면 우리는 루프를 통해 진행합니다. Gemini는 무엇을 할지 생각하고, 도구를 어떻게 사용할지 계획한 다음, 환각적인 관찰 결과를 생성하려 하는 즉시 중단됩니다. 그 후 CrewAI가 제어권을 넘겨받아 Gemini의 계획된 행동을 파싱하고, 관련 도구(단순한 Python 함수)를 관련 입력값(이 경우, SQL 쿼리를 문자열 입력으로 `query_internal_database` 함수를 호출)과 함께 호출하여 실제 관찰 결과를 생성합니다.

그다음 루프의 다음 반복에서 CrewAI는 이 도구 행동의 결과를 시스템 주입(System injection)으로 태그된 문자열 형태로 우리의 LLM 두뇌로 다시 전달합니다. 제어권이 Gemini로 다시 넘어가고, Gemini는 추론, 행동, 관찰의 이력을 확인하며 이 새로운 정보로 다시 추론하기 시작합니다. 이 과정은 최종 답변이 나올 때까지 계속됩니다.

### 팀 확장하기

CrewAI의 목적은 팀에 다양한 가상 인력을 쉽게 구성할 수 있도록 하는 것입니다. 이제 새로운 팀원을 고용해 봅시다.

*   **내부 성과 분석가 (Internal Performance Analyst)** — 우리가 이미 고용한 SQL 분석가로, 내부 데이터를 확인하여 문제를 발견합니다.
*   **가격 분석가 (Pricing Analyst)** — B2B API와 웹 검색을 사용하여 경쟁사 가격을 확인합니다.
*   **정성 분석가 (Qualitative Analyst)** — 웹을 검색하여 목적지의 수요에 영향을 미칠 수 있는 요인을 찾아냅니다.
*   **전략가 (Strategist)** — 세 분석가의 작업을 검토하고 전반적인 전략을 수립합니다.

우리는 웹 검색을 위해 Tavily (타빌리)를 사용할 것입니다. Tavily는 AI 에이전트를 위해 설계되었으며, 광고나 푸터 등의 불필요한 요소가 없는 깔끔하고 구조화된 데이터를 반환하여 에이전트를 돕습니다(기업 규모에서 LLM과의 대화 시 토큰당 비용을 지불하므로, 불필요한 요소를 제거하는 것은 비용 절감에도 도움이 됩니다). 여기에서 무료로 가입하고 API 키와 시크릿 키를 복사하기만 하면 됩니다.

가격 분석가(Pricing Analyst)의 경우, 아마데우스(Amadeus)와 같은 서비스를 통한 실제 프로세스 설정은 우리의 목적에 다소 복잡하므로, 정적인 가상 데이터를 반환하는 모의 도구(mock tool)를 사용할 것입니다. 또한 내부 성과 분석가(Internal Performance Analyst)에게 두 번째 도구를 제공하여 작업 속도를 높일 예정입니다. 종합적으로 우리의 설정 및 도구 정의는 다음과 같습니다.

```python
import os
import sqlite3
from crewai import Agent, Task, Crew, Process, LLM
from crewai.tools import tool
from crewai_tools import TavilySearchTool

# =============================================================================
# 1. CONFIGURATION
# =============================================================================

# ENTER YOUR KEYS HERE
os.environ["GOOGLE_API_KEY"] = "PUT API KEY HERE"
os.environ["TAVILY_API_KEY"] = "PUT API KEY HERE"

# Initialize the Gemini Model
gemini_llm = LLM(
    model="gemini/gemini-flash-latest",
    api_key=os.environ["GOOGLE_API_KEY"],
    temperature=0.2
)

# =============================================================================
# 2. TOOL DEFINITIONS
# =============================================================================

class InternalDataTools:

    @tool("SQL Schema Check")
    def check_schema(table_name: str):
        """
        Checks the column names and types for a given table.
        """
        try:
            # Connect to the local DB we just made
            conn = sqlite3.connect('company_data.db')
            cursor = conn.cursor()
            cursor.execute("PRAGMA table_info({})".format(table_name))
            result = cursor.fetchall()
            conn.close()
            return f"Schema Result: {str(result)}"
        except Exception as e:
            return f"SQL Error: {str(e)}"
    
    @tool("SQL Query Tool")
    def query_internal_database(query: str):
        """
        Queries the local 'company_data.db' for destination performance.
        Useful for finding confirmed vs predicted bookings and recent price drops.
        """
        try:
            # Connect to the local DB we just made
            conn = sqlite3.connect('company_data.db')
            cursor = conn.cursor()
            cursor.execute(query)
            result = cursor.fetchall()
            conn.close()
            return f"Database Result: {str(result)}"
        except Exception as e:
            return f"SQL Error: {str(e)}"

class ExternalDataTools:
    
    @tool("Check Competitor Hotel Rates")
    def check_partner_rates(destination: str, date_range: str):
        """
        Queries Amadeus to return hotel prices for a sample of
        competitor hotels in the given destination and date range.count
        Date range should be formatted as "Month Year", with a three-letter
        code for month and 4 digits for Year.
        """
        # Mocked competitor data - in the real world, we would use Amadeus or similar
        if ("tuscany" in destination.lower() or "florence" in destination.lower()) and date_range == "Mar 2026":
            return """
            COMPETITOR PRICE API RESULT:
            - Hotel: Villa Medicea (4 Star) -> $180/night
            - Hotel: Tuscan Nights (4 Star) -> $170/night
            - Hotel: Firenze Experience (4 Star) -> $175/night
            - Availability: High
            """
        else:
            return "No partner rates found for this location."

# Community Tools
# k=5 means "Read the top 5 websites and summarize them"
search_tool = TavilySearchTool(
    tavily_api_key=os.environ["TAVILY_API_KEY"],
    k=5, 
    search_depth="advanced" 
)
```

여기서 우리는 세 가지 사용자 지정 CrewAI 도구를 정의하고, Tavily를 위한 사전 구축된 도구 하나를 임포트했습니다. (참고: 실제로는 API 키를 이 스크립트에 직접 작성하지 않고, 다른 곳에서 환경 변수로 할당할 것입니다.)

이제 새로운 팀을 만나봅시다! 저는 내부 성과 분석가에게 남아있는 미판매 객실의 평균 가격을 확인하라는 추가적인 요청을 했습니다. 이는 팀의 나머지 구성원들이 경쟁사와 비교하여 우리의 가격이 저렴한지 비싼지 파악할 수 있도록 해줄 것입니다.

```python
# =============================================================================
# 3. DEFINE AGENTS
# =============================================================================

# 1. The Internal Performance Analyst (Internal Data)
sql_analyst = Agent(
    role='Internal Performance Analyst',
    goal='Identify underperforming destinations using internal database metrics.',
    backstory="""You are a senior data analyst. You look at the company_data database.
    You pay special attention to the gap between 'Confirmed' and 'Predicted' bookings, 
    and identify what recent price drops we have already implemented. You also note the
    current average price for the remaining unsold nights.""",
    tools=[InternalDataTools.query_internal_database, InternalDataTools.check_schema],
    llm=gemini_llm,  # Passing the fixed LLM object
    verbose=True)

# 2. Pricing Analyst
pricing_analyst = Agent(
    role='Pricing Analyst',
    goal='Compare market prices vs our partner rates.',
    backstory="""You check two things:
    1. General market prices (using Tavily Search) to see what consumers see.
    2. Our B2B tool to query rates at a sample of competitor hotels (using the Partner Tool).
    Make sure to search for the right month and year.""",
    tools=[search_tool, ExternalDataTools.check_partner_rates],
    llm=gemini_llm,
    verbose=True
)

# 3. The Scout (News & Sentiment)
qual_analyst = Agent(
    role='Qualitative Analyst',
    goal='Find external factors (news, weather, events) affecting demand for hotel stays.',
    backstory="""You search for qualitative reasons. 
    Use the search tool to find news about 'Strikes', 'Heatwaves', or 'Events' in the region.
    Don't just look at headlines; read the context.""",
    tools=[search_tool],
    llm=gemini_llm,
    verbose=True
)

# 4. The Strategist
strategist = Agent(
    role='Pricing Strategist',
    goal='Synthesize all reports into a unified recovery plan.',
    backstory="""You read the reports from the Internal Performance Analyst, Pricing Analyst, and Qualitative Analyst.
    You connect the dots and create a brief report which highlights any issues and theorises causes for them.""",
    llm=gemini_llm,
    verbose=True
)
```

이제 모든 팀원에게 업무를 할당해 봅시다. 작업 정의에 목적지와 날짜에 대한 자리표시자(placeholder)를 포함시켰다는 점에 유의하세요. 또한 각 작업이 최종 답변을 마크다운 파일에 작성하도록 요청했습니다.

```python
# =============================================================================
# 4. DEFINE TASKS
# =============================================================================

task_internal = Task(
    description="""
    Check the internal database for {destination} and {date_range}. 
    Report on the variance between Predicted and Confirmed bookings, 
    and note if we have been discounting recently.
    The relevant table is probably called destination_performance. This table
    stores dates as "Month Year", e.g. "December 2024".
    """,
    expected_output="Internal Performance Report with booking numbers and price drop metrics.",
    agent=sql_analyst,
    output_file='internal.md'
)

task_pricing = Task(
    description="""
    1. Check Competitor Hotel Rates for {destination} and {date_range}.
    2. Use Search to find 'Current 4-star hotel prices in {destination} for {date_range}'.
    3. Compare: Are we competitive?

    When searching the web, include which search terms you have used when thinking.
    """,
    expected_output="Price competitiveness analysis.",
    agent=pricing_analyst,
    output_file='pricing.md'
)

task_news = Task(
    description="""
    Search for travel news and problems related to the {destination} which could impact demand for hotel stays.
    When searching the web, include which search terms you have used when thinking.
    """,
    expected_output="Risk assessment (heatwaves, strikes, etc).",
    agent=qual_analyst,
    output_file='news.md'
)

task_strategy = Task(
    description="""
    Review the reports.
    Identify any issues and diagnose possible causes.
    Propose a 3-point strategy to fix it.
    """,
    expected_output="Final Strategic Memo.",
    agent=strategist,
    output_file='strategy.md'
)
```

마지막으로, 팀에게 작업을 시작하도록 요청합시다. 목적지와 날짜 범위를 입력값으로 넣었다는 점에 유의하세요. 이 값들은 작업 프롬프트의 자리표시자를 대체할 것입니다.

```python
# =============================================================================
# 5. RUN THE CREW
# =============================================================================

inputs = {
    'destination': 'Tuscany, Italy',
    'date_range': 'March 2026'
}

crew = Crew(
    agents=[sql_analyst, pricing_analyst, qual_analyst, strategist],
    tasks=[task_internal, task_pricing, task_news, task_strategy],
    process=Process.sequential,
    memory=False, 
    verbose=True
)

print("### KICKING OFF REVENUE MANAGEMENT SQUAD ###")
result = crew.kickoff(inputs=inputs)

print("\n\n########################")
print("## STRATEGIC DECISION ##")
print("########################\n")
print(result)
```

### 팀의 노력 검토

이번에는 내부 성과 분석가가 (작업 정의에서) 확인해야 할 테이블 이름을 알려줬기 때문에 최종 답변에 더 빠르게 도달합니다. 그는 또한 남아있는 미판매 객실의 평균 1박 가격도 추출합니다.

CrewAI는 그 후 내부 성과 분석가의 결과물을 문맥(context)으로 포함하여 가격 분석가(Pricing Analyst)에게 바통을 넘겨줍니다. 가격 분석가는 제 (가상의) 경쟁사 가격을 성공적으로 쿼리한 다음, 가격 추세에 대한 웹 검색을 수행합니다. 그의 최종 보고서는 다음과 같습니다.

현실적으로 웹 검색을 통해 얻은 경쟁사 가격이 특히 견고하다고 보기는 어렵습니다. 실제 상황에서는 아마데우스(Amadeus)와 같은 서비스나 데이터 제공업체에 의존할 것입니다.

다음은 정성 분석가(Qualitative Analyst)로, 두 번의 웹 브라우징을 수행한 후 보고했습니다.

그리고 전략가(Strategist)가 나서서 요약했습니다.

계획을 제시하기 전에:

각 팀원은 이전 작업의 결과를 알고 있다는 점에 유의하세요. 예를 들어, 정성 분석가(Qualitative Analyst)는 우리의 성과 문제와 경쟁사에 비해 높은 가격에 대한 인식을 언급합니다.

한 가지 중요한 경고는, 우리 에이전트들은 도구가 제대로 작동하지 않을 경우(예를 들어, Tavily API 키를 잘못 입력했을 경우) 이를 솔직하게 인정하지 않고 환각적인 관찰 결과를 생성할 때가 많다는 것입니다. 따라서 콘솔을 주시하여 어떤 일이 일어나고 있는지 확인해야 합니다. 웹 검색은 다음과 같아야 합니다.

최종 답변으로 곧바로 넘어가는 에이전트는 대부분 환각(hallucinating) 현상을 겪고 있을 가능성이 높습니다.

비교적 최소한의 코드로 우리는 복잡한 문제에 함께 접근하는 4명의 에이전트 팀을 만들었습니다. 그리고 상당히 성공적으로 문제를 해결했습니다! 그들은 예약률 문제를 정확히 파악하고, 시장에서 경쟁력이 없다는 것을 확인했으며, 수요에 영향을 미칠 수 있는 몇 가지 정성적 문제를 발견하고, 이 모든 것을 권고 사항과 함께 단일 보고서로 통합했습니다.

실제 세계에서 우리의 다음 단계는 무엇일까요? CrewAI는 LangChain 위에 구축된 고수준 추상화(high-level abstraction)입니다. 복잡한 시나리오에서는 CrewAI 대신 LangGraph를 사용해야 할 수도 있습니다. 이는 단순한 루프에서 다방향 논리 흐름도로 전환할 수 있도록 해줍니다. 예를 들어, 우리의 경우 시스템이 계속 진행하기 전에 인간 가격 분석가에게 결과를 확인하도록 전환해야 할지 평가하도록 구성할 수 있습니다.
