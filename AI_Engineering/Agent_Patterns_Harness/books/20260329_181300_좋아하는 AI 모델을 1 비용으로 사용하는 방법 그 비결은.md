# 좋아하는 AI 모델을 1% 비용으로 사용하는 방법. 그 비결은?

우리에게는 문제가 있습니다. 세상에서 가장 똑똑한 모델들은 사용하기에 너무 비싸다는 점입니다.

2025년 후반에 중요한 애플리케이션을 개발하고 있다면, 아마도 GPT-5 수준의 지능을 원할 것입니다. 이 모델은 복잡한 법률 계약을 추론하고, 완벽하고 안전한 SQL을 작성하며, 실제로 작동하는 계획을 수립할 수 있습니다.

하지만 GPT-5는 100만 토큰(token)당 30달러의 비용이 들고, 2초의 지연 시간(latency)을 가집니다. 이러한 모델은 실시간 루프(real-time loop)에 적용할 수 없습니다.

반대편에는 Llama-4-8B가 있습니다. 이 모델은 MacBook Air에서 실행할 수 있으며, 무료이고 빠릅니다(초당 100 토큰). 하지만 GPT-5에 비하면 "멍청"합니다. 환각 현상(hallucination)을 보이고, 지시사항을 잊어버리며, 안전하지 않은 코드를 작성합니다.

대부분의 개발자들은 선택의 기로에 놓여있다고 생각합니다. 똑똑하지만 느리거나, 멍청하지만 빠르거나 둘 중 하나를 선택해야 한다고 말이죠.

그들은 틀렸습니다. 둘 다 가질 수 있습니다.

GPT-5의 "두뇌"를 Llama-4의 "몸체"로 이전할 수 있습니다. 이 기술은 지식 증류(Knowledge Distillation), 특히 합성 추론 증류(Synthetic Reasoning Distillation)라고 불립니다.

이 기술을 통해 거대한 서버 팜(server farm)에서 추론하는 모델과 같은 성능을 로컬 노트북에서 실행할 수 있습니다.

### 오래된 방식: 모방은 학습이 아니다

2023년에는 사람들이 GPT-4의 답변을 사용하여 소규모 모델을 미세 조정(fine-tuning)함으로써 GPT-4를 "증류"하려고 시도했습니다.

그들은 "CSV를 파싱(parse)하는 파이썬(Python) 스크립트를 작성해줘"라는 질문과 GPT-4의 답변(import csv...)을 사용해 Llama-2를 훈련시켰습니다.

이는 실패했습니다. 소규모 모델은 답변을 흉내 내는 것만 배웠을 뿐, 코드를 작성하는 방법을 배우지는 못했습니다. 출력물을 암기했지만, 그 이면에 있는 알고리즘(algorithm)은 놓쳤습니다.

### 새로운 방식: 사고 과정을 복제하기

돌파구는 우리가 "답변"만을 학습시키는 것을 멈추고 "추론(reasoning)"을 학습시키기 시작했을 때 찾아왔습니다.

우리는 이것을 연쇄적 사고(Chain-of-Thought, CoT) 증류(Distillation)라고 부릅니다.

대규모 모델에게 단순히 코드만을 요청하는 대신, 우리는 "소리 내어 생각하도록(think out loud)" 요청합니다. 최종 답변 전에 모델의 내부 독백 단계를 출력하도록 강제하는 것입니다.

**교사 모델 (GPT-5) 출력:**

```
생각: 사용자는 CSV를 파싱하기를 원한다. 잠재적인 인코딩 오류를 처리해야 한다. 표준 라이브러리를 사용해야 한다. 충돌을 피하기 위해 파일이 먼저 존재하는지 확인해야 한다.

코드: import csv; import os...
```

우리는 이 전체 블록(생각과 코드)을 소규모 모델에 입력합니다.

우리는 Llama-4에게 코드를 작성하는 법을 가르치는 것이 아닙니다. 우리는 Llama-4에게 GPT-5의 추론 패턴(reasoning pattern)을 모방하도록 가르치고 있습니다. 소규모 모델이 잠시 멈춰서 계획을 세우고, 오류를 확인한 다음 실행하도록 강제하는 것입니다.

그 결과, "더 강력한" 성능을 내는 아주 작은 모델이 탄생합니다. 이 모델은 우수한 추론 경로를 따르기 때문에 자신보다 10배 큰 모델을 능가합니다.

### 파이프라인: 교사 모델에서 학생 모델로

이 작업을 수행하기 위해 박사 학위가 필요한 것은 아닙니다. 신용 카드와 스크립트(script)만 있으면 됩니다.

다음은 우리가 이러한 "전문가 모델(Specialist Models)"을 구축하는 데 사용하는 생산 워크플로우(production workflow)입니다.

**1단계: 골든 시드(Golden Seed)**
당신이 원하는 것의 완벽한 예시 50개를 작성합니다. "SQL 전문가"를 구축하는 경우, 50개의 어려운 자연어 질문과 그에 대한 올바른 SQL 답변을 작성합니다. 이는 품질 기준을 정의합니다.

**2단계: 확장 (합성 데이터, Synthetic Data)**
50개의 예시로는 충분하지 않습니다. 5,000개가 필요합니다. GPT-5에게 50개의 예시를 보고 유사하지만 구별되는 새로운 문제 5,000개를 생성하도록 요청합니다. 이를 데이터셋 확장(Dataset Expansion)이라고 합니다. 이제 우리는 방대한 어려운 질문 목록을 갖게 됩니다.

**3단계: 추론 추출(Reasoning Extraction)**
우리는 5,000개의 질문을 다시 GPT-5에 입력합니다. 하지만 이번에는 특별한 시스템 프롬프트(system prompt)를 사용합니다. "당신은 SQL 전문가입니다. 이 문제를 해결하세요. 답변하기 전에, 테이블 스키마(table schema)를 분석하고, 엣지 케이스(edge cases)를 고려하며, 쿼리(query)를 계획하는 `<thinking>` 블록을 출력해야 합니다."

우리는 이러한 "생각 + 답변" 쌍을 저장합니다. 이것이 우리의 훈련 세트(training set)입니다.

**4단계: 미세 조정(Fine-Tune)**
우리는 기본 모델(base model, Llama-4-8B 또는 Mistral-Small과 같은)을 가져와 이 데이터셋(dataset)으로 미세 조정합니다. 모델이 `<thinking>` 토큰(token)을 예측하도록 강제합니다.

### 데이터 생성기 구축하기

이 과정에서 가장 어려운 부분은 3단계입니다. 수천 개의 고품질 추론 흔적(reasoning traces)을 생성하기 위한 견고한 파이프라인(pipeline)이 필요합니다.

다음은 이 데이터셋을 생성하는 스크립트입니다. 우리는 데이터의 정합성을 보장하기 위해 엄격한 스키마(schema)와 함께 OpenAI SDK를 사용합니다.

```javascript
import OpenAI from "openai";
import { z } from "zod";
import { zodResponseFormat } from "openai/helpers/zod";
import fs from "fs/promises";
const openai = new OpenAI();
// 1. 훈련 데이터(Training Data)를 위한 스키마(Schema) 정의
// 우리는 단순히 답변만을 원하는 것이 아닙니다. 우리는 사고(THOUGHT) 과정을 원합니다.
// 이 구조는 모델이 자신의 논리(logic)에 대해 명확하게 표현하도록 강제합니다.
const TrainingDataSchema = z.object({
  topic: z.string().describe("문제의 하위 범주"),
  difficulty: z.enum(["easy", "medium", "hard"]),
  user_query: z.string().describe("학생 모델(student model)을 위한 입력 프롬프트(prompt)"),
  
  // '교사 모델의 두뇌' - 이것이 우리가 채굴하는 황금입니다.
  chain_of_thought: z.string().describe("단계별 추론 논리. <think> 태그를 사용하세요."),
  
  final_answer: z.string().describe("올바르고 검증 가능한 출력."),
});
type TrainingExample = z.infer<typeof TrainingDataSchema>;
// 2. 생성기 함수
async function generateReasoningData(seedTopic: string, count: number) {
  const dataset: TrainingExample[] = [];
  
  console.log(`Starting generation for topic: ${seedTopic}`);
  // 실제 운영 환경에서는 이러한 호출을 배치(batch) 처리합니다 (예: 한 번에 50개).
  for (let i = 0; i < count; i++) {
    try {
      const completion = await openai.beta.chat.completions.parse({
        model: "gpt-5-turbo", // 사용 가능한 가장 똑똑한 모델 사용
        messages: [
          {
            role: "system",
            content: `You are a synthetic data generator. 
                     Create a unique, complex problem about '${seedTopic}'.
                     You MUST include a detailed 'chain_of_thought' that explains 
                     HOW to solve it, catching potential edge cases.`
          },
          {
            role: "user",
            content: `Generate example #${i + 1}. Make it distinct from previous ones.`
          }
        ],
        response_format: zodResponseFormat(TrainingDataSchema, "training_example"),
      });
      const example = completion.choices[0].message.parsed;
      
      if (example) {
        dataset.push(example);
        console.log(`[${i + 1}/${count}] Generated: ${example.user_query.slice(0, 50)}...`);
      }
    } catch (err) {
      console.error(`Failed to generate example ${i}:`, err);
    }
  }
  // 3. JSONL로 저장 (미세 조정을 위한 표준 형식)
  // 훈련을 위해 표준 '메시지(messages)' 배열 형식으로 만듭니다.
  const jsonlData = dataset.map(item => JSON.stringify({
    messages: [
      { role: "user", content: item.user_query },
      { role: "assistant", content: `${item.chain_of_thought}\n\n${item.final_answer}` }
    ]
  })).join("\n");
  await fs.writeFile("reasoning_dataset.jsonl", jsonlData);
  console.log("Dataset saved. Ready for fine-tuning.");
}
// --- 파이프라인 실행 ---
// 작은 모델을 가르치기 위해 100개의 복잡한 SQL 문제를 생성합니다.
generateReasoningData("Advanced PostgreSQL Window Functions", 100);
```

### 비용 절감

계산을 살펴보겠습니다.

**시나리오 A: GPT-5 직접 사용**

한 달에 100만 건의 요청이 발생하는 앱이 있다고 가정해 봅시다.

*   **GPT-5 비용:** 100만 토큰(token)당 30달러.
*   **입력 + 출력 평균:** 1,000 토큰.
*   **총 비용:** 월 30,000달러.

**시나리오 B: 증류된 모델(Distilled Model)**

*   **일회성 비용:** GPT-5로 5,000개의 합성 예시(synthetic examples)를 생성하는 데 약 50달러.
*   **훈련 비용:** 클라우드 GPU(Cloud GPU, H100과 같은)에서 Llama-4-8B를 미세 조정(fine-tuning)하는 데 1시간 소요. 비용: 4달러.
*   **운영 비용:** Llama-4를 더 저렴한 인스턴스(instance)나 심지어 로컬 하드웨어(local hardware)에 호스팅합니다. 비용: 월 400달러.

당신은 한 번에 54달러를 지출하고, 매달 29,600달러를 절약합니다.

### "전문가 모델"의 이점

비용 외에 두 번째 이점이 있습니다. 바로 전문화(Specialization)입니다.

GPT-5는 일반주의자(generalist)입니다. 셰익스피어와 양자 물리학에 대해 알고 있습니다. 만약 "의료비 청구 코드(Medical Billing Codes)"를 위한 도구를 만들고 있다면, GPT-5 두뇌의 99%는 낭비되는 공간입니다.

증류할 때, 당신은 레이저처럼 집중하는 전문가(savant)를 만듭니다. 당신의 Llama-4 모델은 오직 의료비 청구에 대해서만 압니다. 누가 햄릿을 썼는지 모르고, 관심도 없습니다. 전문화되어 있기 때문에 특정 작업에서는 종종 교사 모델을 능가합니다. 결코 산만해지지 않으며, 안전 필터(safety filter) 오작동으로 인해 요청을 거부하는 일도 없습니다. 그저 자신의 일을 수행할 뿐입니다.

### 결론

우리는 "프롬프트 엔지니어링(Prompt Engineering)"의 시대를 넘어 "데이터 큐레이션(Data Curation)"의 시대로 진입하고 있습니다.

당신의 경쟁 우위는 프롬프트가 아닙니다. 모두가 동일한 프롬프트를 가지고 있습니다. 당신의 강점은 당신의 합성 데이터셋(Synthetic Dataset)입니다.

세상에서 가장 똑똑한 모델들의 추론 패턴을 추출하여 저렴하고 빠르며 로컬에서 실행되는 모델에 각인시키는 파이프라인(pipeline)을 구축할 수 있다면, 당신은 성공할 것입니다. 더 이상 지능을 임대하는 것이 아니라 소유하게 됩니다.
