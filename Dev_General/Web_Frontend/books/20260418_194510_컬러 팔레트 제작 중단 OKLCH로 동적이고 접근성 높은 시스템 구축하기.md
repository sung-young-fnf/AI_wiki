# 컬러 팔레트 제작 중단: OKLCH로 동적이고 접근성 높은 시스템 구축하기

제 경력 내내 수많은 컬러 팔레트를 만들었지만, 사실 만드는 것을 좋아하지 않습니다. 조화롭게 어울리고 브랜드 정체성을 유지하면서도 접근성 높은 UI를 구현하는 '올바른' 색상을 선택하는 데는 오랜 시간이 걸립니다.

제가 무슨 의미로 말씀드리는지 보여드리기 위해, 다음은 Vouchsafe를 위해 제가 만들었던 팔레트입니다:

정말 많은 색상이죠! 이는 UI 디자인에서 일반적인 관행이지만, 이 중 대부분은 저희 앱에서 전혀 사용되지 않았습니다. 더 나쁜 것은 무작위로 사용한다는 점입니다. Grey 400과 Grey 500 사이에는 큰 차이가 없는데도 어쩐지 둘 다 사용하고 있습니다.

작년 말 저희 리브랜딩을 시작했을 때, 저는 또 다른 비대한 팔레트를 만들고 싶지 않았습니다. 대신 스스로에게 과제를 주었죠.

단 하나의 컬러 토큰(colour token)만으로 필요한 모든 것을 얻을 수 있을까?

결론적으로 말하면, 가능했습니다.

단 하나의 색상으로 접근성 기준을 충족하고 시각적으로도 보기 좋은 전체 색상 시스템을 구축할 수 있습니다. 이 글에서는 저의 방법을 설명하고 직접 시도해볼 수 있는 시작점을 제시할 것입니다.

먼저, 이를 위해 완전히 새로운 색상 시스템인 OKLCH를 배워야 했던 이유를 설명해야 할 것입니다.

## OKLCH란 무엇인가?

OKLCH는 숫자 값으로 색상을 정의하는 방식인 색상 공간(colour space)입니다. 2022년부터 CSS에서 지원되었지만, 저는 작년 말에야 알게 되었습니다.

여러분은 아마 16진수 코드(hex codes)나 HSL로 작성된 색상에 더 익숙하실 것입니다. HSL에서는 색조(hue), 채도(saturation), 명도(lightness)를 사용하여 색상을 정의합니다. OKLCH는 매우 유사한 구조를 가집니다.

```
colour = oklch(L C H / A)

L = 인지 명도 (perceived lightness)    (0-1.0)
C = 크로마 (채도) (chroma, saturation)    (0-0.4)
H = 색조 (hue)                    (0-360°)
A = 알파 (alpha)                  (0-100%)
```

사소한 차이처럼 보이지만, 그 이면에는 많은 것이 숨어 있습니다. 자세히 설명하지는 않겠지만, 이 값들로부터 색상이 계산되는 방식은 OKLCH가 HSL과 다릅니다.

예를 들어 보겠습니다. 동일한 채도(saturation)와 명도(lightness) 값을 가지는 두 개의 버튼(파란색과 녹색)을 만들어 봅시다. 버튼 위의 흰색 텍스트가 읽기 쉬우려면 4.5:1의 대비율(contrast ratio)을 가져야 합니다.

파란색 버튼은 대비율을 통과하지만 녹색 버튼은 실패합니다.

이는 파란색과 녹색의 명도(lightness) 값이 둘 다 60임에도 불구하고, 우리 눈이 이 두 색상의 명도를 다르게 처리하기 때문입니다.

OKLCH는 인간이 색상을 보는 방식에 맞춰 색상을 조정합니다. 명도(lightness)와 채도(saturation) 값이 동일하게 유지될 때, 색상들이 더 비슷하게 보이게 됩니다. 색조(hue) 값만 조정해도 대비(contrast)가 일관되게 유지되는데, 이것이 바로 우리가 필요로 하는 기능입니다.

이 컬러 피커(colour picker)를 사용하여 실제 작동 방식을 확인해 볼 수 있습니다. OKLCH에 관심이 있다면 이 글을 읽어 볼 수도 있습니다. 하지만 이 기본적인 이해만으로도 새로운 팔레트를 만들기 시작할 수 있습니다!

## 팔레트 구축하기

저는 단일 색상에서 시작했습니다. 이는 16진수 코드(hex code) #7276cb를 가진 저희 브랜드 색상이지만, 여기서는 OKLCH 형식으로 변환했습니다.

이 하나의 색상으로 필요한 다른 모든 것을 생성할 것입니다.

주로 텍스트에 사용할 전경색(foreground colour)부터 시작합니다. 일반적으로는 주 색상과 '느낌상' 잘 어울리는 오프블랙(off-black)을 선택했을 것입니다. 명도(lightness)를 한참 낮추고 채도를 떨어뜨려 적당하다고 느껴질 때까지 조정하곤 했죠.

OKLCH를 사용하면 수정(modification)을 통해 저희 브랜드 보라색을 이상적인 조합으로 변경할 수 있습니다. 명도(lightness), 크로마(chroma), 색조(hue) 값을 개별적으로 조정하여 필요한 것을 얻을 것입니다. 제 코드에서는 다음 수식을 사용했습니다.

```
brand = oklch(0.6 0.128 280) /* 브랜드 색상 정의 */

fg = oklch(
        from var(brand)          /* 브랜드 색상 사용 */
        0.25                     /* 명도를 25%로 설정 */
        calc(c * 0.3)            /* 크로마 감소 */
        h                        /* 색조는 그대로 유지 */
      );
```

최종 전경색(foreground colour)은 다음과 같습니다.

바로 우리가 필요했던 것입니다! 검정색이 아니면서도 주 색상과 잘 어울리는 어두운 색상이죠. 같은 방식으로 배경색(background colour)도 만들 수 있습니다.

```
bg = oklch(
        from var(brand)          /* 브랜드 색상 사용 */
        0.97                     /* 명도를 97%로 설정 */
        calc(c * 0.2)            /* 크로마 감소 */
        h                        /* 색조는 그대로 유지 */
      );
```

이전에 만든 전경색(foreground colour)이 배경색과 강한 대비율(contrast ratio)을 이루는 것을 볼 수 있습니다. 명도(lightness) 값이 고정되어 있기 때문에 어떤 브랜드 색상을 선택하든 이 방식은 작동합니다. 몇 가지 예시는 다음과 같습니다.

전경색(foreground colour)과 배경색(background colour) 값에서 다른 색상을 파생시킬 수도 있습니다. 새로운 팔레트에서는 설명이나 부제목에 사용할 전경색의 차분한(muted) 버전을 만들 필요가 있습니다.

```
fg-muted = oklch(
        from var(fg)     /* 전경색(fg colour) 사용 */
        l                /* 다른 값들은 동일하게 유지 */
        c
        h
        / 55%            /* 알파(투명도)를 55%로 설정 */
      );
```

이처럼 계속해서 확장해나갈 수 있습니다. 시간이 지남에 따라 처음 정의한 색상을 기반으로 필요한 토큰(token)의 수를 늘려나갈 수 있습니다.

팔레트에 큰 변화를 가져올 한 가지 개념을 더 설명하고자 합니다: 클램핑(clamping).

## 클램핑(Clamping)

CSS에서 `clamp()`를 사용하여 계산된 값을 특정 숫자 범위 내로 제한할 수 있습니다. 이 기능은 명도(lightness)가 고정되지 않은 색상을 다루면서도 계산된 값에 최소 및 최대 값을 부여하고 싶을 때 유용합니다. 앞서 보았던 버튼을 예로 들어 설명하겠습니다.

저희의 원래 브랜드 색상은 명도(lightness)가 0.6으로, 버튼에 사용하기에는 좋지 않은 색상입니다. 흰색이든 검은색이든 텍스트를 선택하더라도 대비(contrast)가 낮아 읽기 어렵습니다.

OKLCH 값을 조정하여 약간 더 어두운 색상(명도 45% 설정)을 만들면 문제가 해결됩니다. 하지만 이는 저희 보라색에만 해당되는 이야기입니다. 더 밝은 브랜드 색상에는 이 방법이 통하지 않습니다.

값을 45%로 설정하면 어두운 버튼이 생성되는데, 이는 UI에서 페이지의 다른 요소들과 잘 어울리지 않을 수 있습니다. 명도 값에서 0.15를 빼면 대비(contrast)가 실패하는 결과를 초래할 것입니다.

`clamp()`를 사용하면 모든 버튼이 흰색 텍스트와 좋은 대비(contrast)를 가지도록 최대 명도(lightness) 수준을 설정할 수 있습니다.

```
cta-primary = oklch(
        from var(brand)
        clamp(0, calc(l - 0.15), 0.55)  /* 명도 -15%, 최대 55% */
        c
        h
      );
```

명도(lightness)가 높은 브랜드 색상은 클램핑(clamping)되어 버튼이 계속 읽기 쉽게 유지됩니다. 그리고 더 어두운 색상이 마음에 들지 않는다면, 좀 더 보기 좋은 색상을 얻을 때까지 색조(hue)를 변경할 수 있습니다.

하나의 브랜드 색상과 약간의 계산만 있으면 접근성 높고 조화로운 컬러 팔레트를 구축할 수 있습니다. 더 이상 추측이나 '느낌'에 의존할 필요가 없습니다.

## 고찰

제 경력의 첫 10년 동안 모든 디자인에서 HSL과 16진수 코드(hex codes)를 사용했습니다. OKLCH를 사용한 것은 이번이 첫 프로젝트였지만, 약간 다르게 생각할 수 있어 신선했습니다.

벌써 잠재력을 보고 있습니다! 다크 모드(dark mode)를 추가하는 것은 매우 쉬울 것입니다 (여기서 한 것을 반대로 적용하면 됩니다). 더 나은 접근성(accessibility)을 위해 고대비 모드(high contrast modes)를 만들거나, 특정 색맹(colour blindness)을 위한 브랜드 팔레트를 만들 수도 있습니다.

고객이 자신들의 브랜드 색상을 사용하도록 허용하는 아이디어도 그리 두렵지 않습니다. 클램프(clamp)가 적용되어 있기 때문에 어떤 색상을 선택하든 크게 잘못될 일은 없을 것입니다.

앞으로 어떤 일이 일어나든, 저는 다시는 맨 처음부터 UI 팔레트를 만들지 않을 것임을 알고 있습니다.
