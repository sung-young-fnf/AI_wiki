# 대부분의 튜토리얼이 건너뛰는 Python GUI 도구 (타당한 이유 없이)
자동화 스크립트를 실제 데스크톱 도구로 바꾸는 방법

4년 동안 저는 완벽하게 작동하는 자동화 시스템을 구축했습니다.

*   데이터를 스크랩했습니다.
*   문서를 요약했습니다.
*   임베딩(embeddings)을 생성했습니다.
*   PDF를 클러스터링했습니다.
*   RAG(Retrieval-Augmented Generation) 파이프라인을 구축했습니다.

하지만 거의 아무도 사용하지 않았습니다.

시스템이 나빴기 때문이 아닙니다.

터미널에서만 실행되었기 때문입니다.

만약 자동화 시스템을 사용하기 위해 터미널을 열고 파일 경로를 직접 수정해야 한다면, 그것은 도구가 아니라 숙제에 가깝습니다.

대부분의 Python GUI 튜토리얼은 초보자에게 Tkinter를 권장합니다. 일부는 웹 대시보드로 바로 넘어갑니다. PySide6에 대해 진지하게 다루는 경우는 극히 드뭅니다.

이것은 실수입니다.

왜냐하면 자동화 시스템이 소프트웨어로 발전해야 할 때, PySide6는 Python에서 가장 강력한 옵션 중 하나가 되기 때문입니다.

이것은 단순히 버튼을 그리는 것에 대한 이야기가 아닙니다.

이것은 자동화를 제품으로 바꾸는 것에 대한 이야기입니다.

## 진짜 문제: 접근성 없는 자동화

대부분의 개발자는 코드의 우아함에 최적화합니다.

전문가들은 채택률에 최적화합니다.

저는 AI 기반 PDF 정리 도구를 만들었습니다.

*   연구 논문에서 텍스트 추출
*   임베딩 생성
*   KMeans로 클러스터링
*   토픽 기반 폴더로 파일 이동

아름답게 작동했지만, 다음 사항이 필요했습니다.

*   스크립트 실행
*   매개변수 수동 편집
*   로그 스크롤 확인

이것은 마찰입니다.

그리고 마찰은 사용을 저해합니다.

**프로 팁:** 최고의 자동화는 사람들이 실제로 실행하는 자동화입니다.

## 1단계: 자동화와 인터페이스 분리

어떤 GUI를 구축하기 전에 백엔드를 분리하세요. UI 코드와 로직을 절대 섞지 마세요.

다음은 핵심 자동화 엔진입니다.

### PDF에서 텍스트 추출

```python
import fitz  # PyMuPDF

def extract_text(pdf_path):
    doc = fitz.open(pdf_path)
    text = ""

    for page in doc:
        text += page.get_text()

    return text[:3000]
```

이 함수의 역할:

*   PDF를 엽니다.
*   각 페이지를 반복합니다.
*   텍스트를 추출합니다.
*   성능을 위해 길이를 제한합니다.

이 함수는 단일 목적이며 깔끔합니다. 이는 UI와 연결할 때 중요합니다.

### 임베딩 생성

```python
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("all-MiniLM-L6-v2")

def generate_embeddings(text_list):
    return model.encode(text_list)
```

설명:

*   경량 임베딩 모델을 로드합니다.
*   텍스트를 밀집 벡터(dense vectors)로 변환합니다.
*   수치적 표현(numerical representations)을 반환합니다.

여기서 자동화는 지능적으로 변모합니다.

### 파일 클러스터링 및 정리

```python
from sklearn.cluster import KMeans
import os
import shutil

def cluster_files(file_paths, embeddings, base_dir, n_clusters):
    kmeans = KMeans(n_clusters=n_clusters, random_state=42)
    labels = kmeans.fit_predict(embeddings)

    for cluster_id in range(n_clusters):
        cluster_folder = os.path.join(base_dir, f"Cluster_{cluster_id}")
        os.makedirs(cluster_folder, exist_ok=True)

    for file_path, label in zip(file_paths, labels):
        destination = os.path.join(
            base_dir,
            f"Cluster_{label}",
            os.path.basename(file_path)
        )
        shutil.move(file_path, destination)
```

이 함수의 역할:

*   임베딩을 클러스터링합니다.
*   동적으로 폴더를 생성합니다.
*   파일을 자동으로 이동합니다.

백엔드 작업은 완료되었습니다. 이제 튜토리얼이 회피하는 부분이 나옵니다.

## 2단계: PySide6가 빠진 조각인 이유

대부분의 튜토리얼은 다음을 제안합니다.

*   Tkinter: 내장되어 있기 때문
*   Flask: 웹 앱이 현대적으로 느껴지기 때문
*   Electron: 멋있어 보이기 때문

하지만 다음을 원한다면:

*   네이티브 데스크톱 성능
*   깔끔한 아키텍처
*   확장 가능한 UI
*   전문적인 배포

PySide6는 Python 내에서 Qt 수준의 강력한 기능을 제공합니다.

PySide6는 전문 소프트웨어에 사용되는 것과 동일한 Qt 프레임워크입니다. 그리고 Python 자동화와 깔끔하게 통합됩니다.

## 3단계: 가벼운 GUI 레이어 구축

의존성 설치:

```bash
pip install PySide6
```

이제 최소한의 인터페이스를 만듭니다.

### 메인 윈도우 생성

```python
import sys
from PySide6.QtWidgets import (
    QApplication, QWidget, QPushButton,
    QVBoxLayout, QLabel, QFileDialog, QSpinBox
)

class OrganizerApp(QWidget):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("AI PDF Organizer")

        self.layout = QVBoxLayout()

        self.folder_label = QLabel("No folder selected")
        self.layout.addWidget(self.folder_label)

        self.select_button = QPushButton("Select Folder")
        self.select_button.clicked.connect(self.select_folder)
        self.layout.addWidget(self.select_button)

        self.cluster_input = QSpinBox()
        self.cluster_input.setMinimum(2)
        self.cluster_input.setMaximum(10)
        self.layout.addWidget(self.cluster_input)

        self.run_button = QPushButton("Run Automation")
        self.run_button.clicked.connect(self.run_automation)
        self.layout.addWidget(self.run_button)

        self.setLayout(self.layout)
        self.selected_folder = None
```

어떤 일이 일어나는가:

*   `QWidget`는 메인 윈도우를 생성합니다.
*   `QVBoxLayout`은 구성 요소를 수직으로 쌓습니다.
*   `QPushButton`은 액션을 트리거합니다.
*   `QSpinBox`는 사용자가 클러스터링을 제어할 수 있게 합니다.

중요한 점을 주목하세요.

여기에는 머신러닝 코드가 없습니다.

GUI는 계산을 수행하지 않습니다. 단지 오케스트레이션(orchestration)할 뿐입니다.

## 4단계: GUI를 백엔드 로직에 연결

이제 연결해 봅시다.

```python
    def select_folder(self):
        folder = QFileDialog.getExistingDirectory(self)
        if folder:
            self.selected_folder = folder
            self.folder_label.setText(folder)

    def run_automation(self):
        if not self.selected_folder:
            return

        import glob

        files = glob.glob(f"{self.selected_folder}/*.pdf")

        texts = [extract_text(file) for file in files]
        embeddings = generate_embeddings(texts)

        cluster_files(
            files,
            embeddings,
            self.selected_folder,
            n_clusters=self.cluster_input.value()
        )
```

설명:

*   네이티브 폴더 선택 대화상자
*   PDF 자동 수집
*   임베딩 생성
*   파일 클러스터링 및 이동

사용자는 버튼 하나만 클릭합니다.

모든 것이 실행됩니다.

이것이 이벤트 기반 자동화(event-driven automation)입니다.

## 6단계: 실제 배포를 위한 패키징

Python 스크립트는 개발자를 위한 것입니다. 실행 파일(executable)은 다른 모든 사람을 위한 것입니다.

PyInstaller 설치:

```bash
pip install pyinstaller
```

실행 파일 빌드:

```bash
pyinstaller --onefile main.py
```

이제 독립형 데스크톱 도구를 갖게 됩니다.

Python 설치가 필요 없습니다.

여기서 자동화는 진정한 영향력(leverage)을 갖게 됩니다.

## 대부분의 튜토리얼이 틀리는 점

튜토리얼은 다음을 가르칩니다.

*   버튼 그리기
*   폼 생성
*   색상 변경

하지만 다음은 거의 가르치지 않습니다.

*   실제 자동화를 감싸는 방법
*   아키텍처를 올바르게 분리하는 방법
*   사람들이 사용하는 도구를 배포하는 방법

GUI 개발은 미학에 관한 것이 아닙니다.

마찰을 제거하는 것에 관한 것입니다.

제품 엔지니어링에는 간단한 원칙이 있습니다: 활성화 에너지(activation energy)를 낮추는 것입니다.

버튼은 명령줄보다 활성화 에너지를 훨씬 더 많이 낮춥니다.

## 이것이 나의 관점을 바꾼 이유

수년간 저는 데스크톱 앱이 시대에 뒤떨어졌다고 생각했습니다.

그러다 PySide6로 진지한 자동화 도구를 하나 만들었고, 갑자기 다음과 같은 변화가 생겼습니다.

*   비기술적인 사용자들도 저의 ML 파이프라인을 실행할 수 있게 되었습니다.
*   고객들은 더 이상 사용 방법을 묻지 않았습니다.
*   저의 스크립트가 소프트웨어가 되었습니다.

백엔드는 변하지 않았습니다.

인터페이스가 변했습니다.

그리고 그것이 모든 것을 바꾸었습니다.
