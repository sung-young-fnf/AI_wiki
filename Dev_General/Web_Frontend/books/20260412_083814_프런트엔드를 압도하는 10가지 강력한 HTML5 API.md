프런트엔드를 압도하는 10가지 강력한 HTML5 API

저는 14개의 탭을 열어두고, 스택 오버플로우(Stack Overflow)에서 끈질기게 답을 찾고 있었고, 리액트(React) 앱은 전혀 협조적이지 않았습니다. 제가 원했던 것은 오직 하나였습니다. 파일을 로컬에 저장하고 자동으로 동기화하는 것이었죠.

하지만 결국 세 개의 라이브러리를 설치하고, 200줄의 글루 코드(glue code)를 작성해야 했으며, 그럼에도 불구하고 신뢰할 수 없었습니다.

그러다 순수한 좌절감에 몇 년 동안 무시했던 것을 시도해 보았습니다.

순수 HTML5 API(Vanilla HTML5 APIs).

프레임워크도, 의존성(dependency) 혼돈도 없었습니다. 오직 브라우저뿐이었습니다.

그다음 일어난 일은 정말 놀라웠습니다.

1.  **즉각적인 영속성을 위한 로컬 스토리지 API (Local Storage API)**

    예전에는 작은 사용자 데이터를 저장하기 위해 백엔드를 구축하곤 했습니다. 미친 짓이었죠.

    HTML5는 이미 이 문제를 해결했습니다.

    **실제 사용 사례:** 서버 없이 사용자 초안을 자동 저장합니다.

    ```javascript
    // 데이터 저장
    localStorage.setItem("draft", JSON.stringify({ text: "Hello world" }));

    // 데이터 검색
    const draft = JSON.parse(localStorage.getItem("draft"));
    console.log(draft.text);
    ```

    이 코드는 즉시 실행됩니다. 네트워크도, 지연(latency)도 없습니다.

    **전문가 팁:**
    앱이 느리게 느껴진다면, 새로운 API를 추가하기 전에 불필요한 API를 제거하세요.

2.  **상황 인식 앱을 위한 지오로케이션 API (Geolocation API)**

    저는 한때 API 키, 사용자 입력, 유효성 검사 등을 사용하여 날씨 앱을 만들었습니다... 불필요한 마찰이었죠.

    HTML5는 위치 정보를 무료로 제공합니다.

    ```javascript
    navigator.geolocation.getCurrentPosition((position) => {
      const { latitude, longitude } = position.coords;
      console.log(latitude, longitude);
    });
    ```

    이제 앱이 사용자에게 적응하며, 그 반대가 아닙니다.

    자동화가 항상 인공지능(AI)에 관한 것은 아닙니다. 때로는 이미 존재하는 것을 활용하는 것만으로도 충분합니다.

3.  **실질적인 사용자 경험(UX)을 위한 드래그 앤 드롭 API (Drag and Drop API)**

    UI(사용자 인터페이스)를 생동감 있게 만들기 위해 라이브러리가 필요하지 않습니다.

    ```html
    <div id="drop-zone">여기에 파일을 놓으세요</div>
    <script>
    const zone = document.getElementById("drop-zone");

    zone.addEventListener("dragover", (e) => e.preventDefault());

    zone.addEventListener("drop", (e) => {
      e.preventDefault();
      const files = e.dataTransfer.files;
      console.log(files);
    });
    </script>
    ```

    이것만으로도 제 대시보드 중 하나가 10배 더 전문적으로 느껴졌습니다.

4.  **백그라운드 처리를 위한 웹 워커 (Web Workers)**

    과도한 연산 때문에 UI가 멈춘 적이 있습니까?

    네, 저도 겪어봤습니다.

    웹 워커(Web Workers)가 이를 해결해 줍니다.

    ```javascript
    const worker = new Worker("worker.js");

    worker.postMessage("start");

    worker.onmessage = (e) => {
      console.log("결과:", e.data);
    };
    ```

    이제 앱이 전문가처럼 멀티태스킹합니다.

    그리고 더 이상 화면이 멈추는 일도 없습니다.

5.  **스마트 자동화를 위한 인터섹션 옵저버 (Intersection Observer)**

    스크롤 이벤트는 복잡하고, 자원 소모가 많으며, 버그가 많습니다.

    이 API가 모든 것을 바꿨습니다.

    ```javascript
    const observer = new IntersectionObserver((entries) => {
      entries.forEach(entry => {
        if (entry.isIntersecting) {
          console.log("요소가 화면에 보입니다.");
        }
      });
    });

    observer.observe(document.querySelector("#target"));
    ```

    다음 용도로 활용해 보세요:

    *   이미지 지연 로딩 (Lazy loading images)
    *   무한 스크롤 (Infinite scroll)
    *   자동 애니메이션 (Auto animations)

    깔끔하고 효율적이며, 꼼수가 필요 없습니다.

6.  **매끄러운 복사 기능을 위한 클립보드 API (Clipboard API)**

    저는 숨겨진 입력 필드와 이상한 꼼수에 의존하곤 했습니다.

    이제 그럴 필요가 없습니다.

    ```javascript
    navigator.clipboard.writeText("복사된 텍스트!");

    navigator.clipboard.readText().then(text => {
      console.log(text);
    });
    ```

    **간단한 UX 개선:**
    "복사" 버튼 하나만 추가해도 즉시 프리미엄처럼 느껴집니다.

7.  **몰입형 앱을 위한 풀스크린 API (Fullscreen API)**

    대시보드, 게임, 미디어 앱.

    풀스크린(Fullscreen)은 분위기를 바꿉니다.

    ```javascript
    const elem = document.documentElement;

    elem.requestFullscreen();

    document.exitFullscreen();
    ```

    라이브러리 없이, 단 한 번의 호출로.

8.  **싱글 페이지 애플리케이션(SPA) 내비게이션을 위한 히스토리 API (History API)**

    항상 리액트 라우터(React Router)가 필요한 것은 아닙니다.

    때로는 이것만으로도 충분합니다.

    ```javascript
    history.pushState({}, "", "/new-page");

    window.addEventListener("popstate", () => {
      console.log("사용자가 뒤로 이동했습니다.");
    });
    ```

    저는 이런 식으로 프레임워크 없이 가벼운 SPA를 만들었습니다.

    그리고 솔직히 말해서 더 깔끔하게 느껴졌습니다.

9.  **음성 자동화를 위한 웹 스피치 API (Web Speech API)**

    이것은 정말 저를 놀라게 했습니다.

    ```javascript
    const recognition = new webkitSpeechRecognition(); // 브라우저 접두사(prefix)에 유의하세요.

    recognition.onresult = (event) => {
      console.log(event.results[0][0].transcript);
    };

    recognition.start();
    ```

    음성 입력. 네이티브(Native) 기능.

    별도의 서드파티(third-party) 서비스가 필요하지 않습니다.

10. **실시간 참여를 위한 알림 API (Notification API)**

    앱이 실제 제품처럼 느껴지기를 원하시나요?

    알림(Notifications) 기능을 사용하세요.

    ```javascript
    Notification.requestPermission().then(permission => {
      if (permission === "granted") {
        new Notification("당신의 앱에서 온 알림!");
      }
    });
    ```

    이제 사용자들은 단순히 앱을 사용하는 것을 넘어, 앱을 '느끼게' 됩니다.

**저에게 변화가 찾아온 이유**

대부분의 개발자가 소리 내어 말하지 않는 진실이 있습니다.

우리는 플랫폼을 신뢰하지 않기 때문에 과도하게 설계(over-engineer)합니다.

저 역시 마찬가지였습니다.

저는 진정한 앱에는 다음이 필요하다고 생각했습니다.

*   프레임워크(frameworks)
*   복잡한 도구들 (heavy tooling)
*   복잡한 아키텍처(complex architecture)

하지만 우리가 의존성(dependencies)을 설치하느라 바쁜 동안 HTML5는 조용히 진화했습니다.

지금은 어떨까요?

*   더 빠르게 개발합니다.
*   디버깅(debugging)을 덜 합니다.
*   더 많은 것을 출시합니다.

그리고 때로는 프레임워크조차 열지 않습니다.

**마지막 생각**

브라우저는 더 이상 단순한 렌더러(renderer)가 아닙니다.

그것은 운영체제(operating system)입니다.

그리고 여전히 HTML5를 "단순한 마크업(markup)"으로 취급하고 있다면...

여러분은 엄청난 잠재력을 활용하지 못하고 있는 것입니다.

이 글이 프런트엔드에 대한 여러분의 생각을 바꿨다면, 여러분만 그런 것이 아닙니다.

누군가 이 사실을 몇 년 전에 저에게 알려줬더라면 좋았을 텐데 말이죠.😅
