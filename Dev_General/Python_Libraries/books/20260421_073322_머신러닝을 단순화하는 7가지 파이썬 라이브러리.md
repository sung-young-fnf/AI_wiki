머신러닝을 단순화하는 7가지 파이썬 라이브러리
복잡한 고민 없이 스마트한 시스템을 구축하세요.

머신러닝(Machine Learning)은 평판 문제가 있습니다. 사람들은 머신러닝이 트랜스포머(transformers), GPU 클러스터, 박사 학위 논문에서나 나올 법한 복잡한 수학으로만 이루어져 있다고 생각합니다. 하지만 현실은 어떨까요? 대부분의 ML 문제는 지독하게 실용적입니다. 지저분한 데이터, 변화하는 패턴, 제한된 컴퓨팅 자원, 그리고 학습 곡선 따위는 고려하지 않는 마감 기한 등과 씨름해야 합니다.

프로덕션 환경에서 파이썬 코드를 작성하고 실제로 유지되는 ML 시스템을 배포하며 4년 이상을 보낸 후, 저는 한 가지를 깨달았습니다.

올바른 라이브러리는 단순히 시간을 절약해 주는 것을 넘어, 문제를 바라보는 방식을 바꿔줍니다.

그래서 여기, 대부분의 개발자가 상상조차 하지 못했던 방식으로 머신러닝을 단순화하는, 잘 알려지지 않은 7가지 파이썬 라이브러리를 소개합니다.

군더더기 없이 핵심만 다루겠습니다. "텐서플로우를 import하고 기도하라"는 식의 이야기는 하지 않겠습니다.

### 1. River — 끊임없이 움직이는 데이터를 위한 머신러닝

대부분의 ML 라이브러리는 데이터가 정적으로 고정되어 있다고 가정합니다. 하지만 실제 데이터는 그렇지 않습니다.

River는 온라인 머신러닝(online machine learning)을 위해 설계되었습니다. 즉, 한 번에 하나의 데이터 포인트를 지속적으로 학습하는 모델을 위한 것입니다.

예를 들면 다음과 같습니다:
*   스트리밍 로그
*   IoT 센서
*   실시간 사용자 행동
*   금융 시세

**River가 다른 점**
*   처음부터 재학습할 필요 없음
*   일정한 메모리 사용량
*   개념 변화(concept drift, 시간이 지남에 따라 데이터가 변하는 현상)에 모델이 적응

**예시: 온라인 스팸 감지**
```python
from river import linear_model, preprocessing, metrics

model = preprocessing.StandardScaler() | linear_model.LogisticRegression()
metric = metrics.Accuracy()

for x, y in data_stream:  # 한 번에 하나의 샘플 처리
    y_pred = model.predict_one(x)
    metric.update(y, y_pred)
    model.learn_one(x, y)

print(metric)
```
이것은 "장난감 ML"이 아닙니다. 실시간 ML 시스템이 프로덕션 환경에서 살아남는 방식입니다.

배포 후에도 모델이 계속 학습해야 한다면, River는 엄청나게 저평가된 라이브러리입니다.

### 2. sktime — 복잡함 없이 시계열(Time Series) ML 다루기

시계열(time series) 문제는 뛰어난 엔지니어들이 고통받는 영역입니다.

sktime은 예측(forecasting), 분류(classification), 이상 감지(anomaly detection) 등 시계열 데이터에 구조, 조합성, 그리고 실제 ML 원칙을 적용합니다.

**sktime이 중요한 이유**
*   예측과 ML을 위한 통합 API
*   패널(panel) 및 계층적(hierarchical) 시계열에 대한 기본 지원
*   사이킷런(scikit-learn) 파이프라인과 호환 (정말로 가능합니다!)

**예시: 최소한의 노력으로 예측**
```python
from sktime.forecasting.arima import AutoARIMA
from sktime.forecasting.base import ForecastingHorizon

fh = ForecastingHorizon([1, 2, 3, 4, 5], is_relative=True)
model = AutoARIMA()

model.fit(y_train)
y_pred = model.predict(fh)
```
만약 새벽 2시에 ARIMA, Prophet, NumPy를 덕트 테이프로 이어 붙인 경험이 있다면… sktime은 우주로부터 온 사과 편지처럼 느껴질 겁니다.

### 3. skorch — 사이킷런(scikit-learn) 규율을 따르는 파이토치(PyTorch) 모델

파이토치(PyTorch)는 강력합니다.
사이킷런(scikit-learn)은 우아합니다.

skorch는 이 둘을 결합할 수 있게 해주며, 매번 커스텀 학습 루프(training loop)를 작성할 필요가 없습니다.

**skorch가 남몰래 탁월한 이유**
*   `.fit()` 및 `.predict()` 메서드를 가진 파이토치 모델
*   `GridSearchCV`와 완벽한 호환성
*   더 깔끔한 실험, 더 적은 버그

**예시: 사이킷런 모델처럼 신경망 사용하기**
```python
import torch.nn as nn
from skorch import NeuralNetClassifier

class Net(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc = nn.Linear(20, 2)

    def forward(self, X):
        return self.fc(X)

net = NeuralNetClassifier(Net, max_epochs=10, lr=0.01)
net.fit(X_train, y_train)
```
이것이 바로 딥러닝(deep learning)이 성장하여 소프트웨어 공학적 예절을 배운 결과입니다.

### 4. modAL — 레이블링 비용을 절감하는 능동 학습(Active Learning)

데이터 레이블링은 비용이 많이 듭니다. 잘못 선택된 데이터에 레이블을 붙이는 것은 더 나쁩니다.

modAL은 능동 학습(active learning)을 구현하여, 모델이 어떤 샘플에 다음으로 레이블을 붙일 가치가 있는지 스스로 결정하게 합니다.

**이것이 강력한 이유**
*   더 적은 레이블링된 샘플로 모델 학습
*   중요한 부분에 인력 집중
*   의료, 법률 또는 틈새(niche) 도메인에 이상적

**예시: 불확실성 샘플링(Uncertainty sampling)**
```python
from modAL.models import ActiveLearner
from sklearn.ensemble import RandomForestClassifier
learner = ActiveLearner(
    estimator=RandomForestClassifier(),
    X_training=X_initial,
    y_training=y_initial
)

query_idx, query_inst = learner.query(X_pool)
learner.teach(X_pool[query_idx], y_new_label)
```
능동 학습은 모두가 이야기하지만 실제로 구현하는 사람은 거의 없는 기술 중 하나입니다. modAL은 그 변명의 여지를 없애줍니다.

### 5. Alibi — 블랙박스 모델을 성숙하게 설명하기

정확도(accuracy)는 중요합니다. 하지만 규제 기관, 이해관계자, 사용자들은 설명(explanation)을 원합니다.

Alibi는 그럴듯하게 거짓말하는 특성 중요도(feature importance) 차트가 아니라, 모델 해석 가능성(model interpretability)에 중점을 둡니다.

**Alibi의 강점**
*   반사실적 설명(Counterfactual explanations)
*   앵커(Anchors, 사람이 읽기 쉬운 규칙)
*   딥러닝(deep learning) 및 고전적 모델 모두 작동

**예시: 반사실적 설명**
```python
from alibi.explainers import Counterfactual

cf = Counterfactual(predict_fn, shape=(1, 10))
explanation = cf.explain(X_instance)
```
누군가 "왜 모델이 이 사용자를 거부했나요?"라고 물을 때, Alibi는 당황하지 않도록 답을 제공합니다.

### 6. pytorch-tabnet — 테이블형 데이터(Tabular Data)를 위한 (실제로 작동하는) 딥러닝

딥러닝(deep learning)은 일반적으로 테이블형(tabular) 데이터에서 XGBoost에 밀리는 경향이 있습니다.

TabNet은 몇 안 되는 예외 중 하나이며, 이 구현체는 깔끔하고 빠르며 프로덕션 환경에서 사용하기에 적합합니다.

**TabNet이 돋보이는 이유**
*   특성에 대한 순차적 어텐션(sequential attention)
*   내장된 해석 가능성(interpretability)
*   최소한의 특성 공학(feature engineering)

**예시: TabNet 학습**
```python
from pytorch_tabnet.tab_model import TabNetClassifier

clf = TabNetClassifier()
clf.fit(X_train, y_train, max_epochs=100)
```
만약 데이터셋이 행과 열로 구성되어 있다면(대부분 그렇습니다), 이것은 시간을 투자할 가치가 있는 몇 안 되는 신경망 기반 접근법 중 하나입니다.

### 7. LightFM — 박사 학위 없이 추천 시스템 구축하기

대부분의 추천 시스템(recommender systems) 튜토리얼은 "Hello World"에서 "행렬 분해(matrix factorization) 유도"로 빠르게 넘어갑니다. LightFM은 실용성을 유지합니다.

**LightFM이 특별한 이유**
*   하이브리드 추천 시스템(Hybrid recommender systems, 콘텐츠 + 상호작용)
*   확장성이 좋음
*   실제 추천 파이프라인(pipelines)에서 사용됨

**예시: 협업 필터링(Collaborative filtering)**
```python
from lightfm import LightFM

model = LightFM(loss='warp')
model.fit(interactions, epochs=30, num_threads=4)
```
넷플릭스의 인프라를 처음부터 다시 만들지 않고 추천 시스템을 구축하고 싶었다면, 이것이 바로 시작점입니다.
