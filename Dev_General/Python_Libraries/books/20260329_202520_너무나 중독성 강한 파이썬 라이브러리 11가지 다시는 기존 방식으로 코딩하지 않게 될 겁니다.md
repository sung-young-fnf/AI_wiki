# 너무나 중독성 강한 파이썬 라이브러리 11가지: 다시는 기존 방식으로 코딩하지 않게 될 겁니다

이 라이브러리들을 한 번 사용해 보면, 기존의 코딩 습관이 우스꽝스럽게 느껴질 것입니다.

### 1. Pathlib + Pathvalidate — 파일 경로 문제로 골머리 앓지 않아도 되니까요

아직 `os.path` (os.path 모듈)로 씨름하고 있다면, 제가 시간을 절약해 드리겠습니다. `pathlib` (패스라이브)는 파일 관리를 시처럼 아름답게 만들어주고, `pathvalidate` (패스밸리데이트)는 OS 전반에 걸쳐 파일 이름을 깔끔하게 유지시켜 줍니다.

```python
from pathlib import Path
from pathvalidate import sanitize_filename

downloads = Path.home() / "Downloads"
for file in downloads.glob("*.pdf"):
    new_name = sanitize_filename(file.stem + "_cleaned") + ".pdf"
    file.rename(downloads / new_name)
```

**왜 중독성이 강할까요:** 선언적이고, 가독성이 뛰어나며, 깔끔합니다. 슬래시 문제와 유효하지 않은 이름에 대한 걱정을 멈추고 로직에 집중할 수 있게 됩니다.

### 2. Halo — 자동화 스크립트에도 멋이 필요하니까요

이 라이브러리는 긴 CLI (Command Line Interface, 명령줄 인터페이스) 작업을 실행하다가 발견했습니다. `Halo`는 거의 설정 없이 세련된 터미널 스피너를 제공합니다. 피드백이 필요한 자동화 스크립트에 아주 좋습니다.

```python
from halo import Halo
import time

spinner = Halo(text="Automating something cool...", spinner="dots")
spinner.start()
time.sleep(2)
spinner.succeed("Done!")
```

작지만 만족스럽습니다. 여러분의 스크립트가 갑자기 살아있는 것처럼 느껴질 겁니다.

**왜 저평가될까요:** 시각적 피드백은 개발자가 혼자 작업하더라도 신뢰도를 높여줍니다.

### 3. Tenacity — 논리적으로 합리적인 재시도 로직

때로는 네트워크 문제, 불안정한 API, 예상치 못한 오류(cosmic rays) 등 무작위적인 이유로 자동화 스크립트가 실패하기도 합니다. `Tenacity`는 스파게티 코드 없이도 우아하게 재시도를 처리하도록 돕습니다.

```python
from tenacity import retry, wait_fixed, stop_after_attempt
import requests

@retry(wait=wait_fixed(2), stop=stop_after_attempt(3))
def fetch_data(url):
    return requests.get(url).json()

data = fetch_data("https://jsonplaceholder.typicode.com/todos/1")
```

더 이상 보기 싫은 `while True` 루프는 없습니다. 깔끔하고 선언적인 복원력만 남습니다.

**전문가 팁:** `retry_if_exception_type`을 추가하여 세밀한 제어 기능을 활용해 보세요. 웹 스크레이핑(web scraping)이나 API 사용이 많은 자동화 작업에 완벽합니다.

### 4. Arrow — `datetime`이 여전히 불편하니까요

네, `datetime` (데이트타임) 모듈도 작동합니다. 하지만 시간대를 직접 변환해 본 적이 있다면, 그 고통을 아실 겁니다. `Arrow` (애로우)는 이 문제를 사람이 읽기 쉽고, 직관적이며, 시간대 안전(timezone-safe)하게 해결합니다.

```python
import arrow

utc = arrow.utcnow()
local = utc.to('Asia/Karachi')
print(local.humanize())  # “just now”
```

이것은 단순히 편의성만을 위한 것이 아닙니다. 깔끔한 시간 처리는 '다음 주 월요일'에만 나타나는 미묘한 자동화 버그를 방지합니다.

### 5. Tqdm — 필요한 줄 몰랐던 진행률 표시줄

저는 예전에 모든 곳에 `print()` 문을 추가하여 스크립트를 디버깅하곤 했습니다. 그러다가 `tqdm` (티큐디엠)을 만났습니다. 이 라이브러리는 어떤 루프에도 간편하게 진행률 표시줄을 추가하여, 자동화 스크립트를 즉시 전문적으로 보이게 만듭니다.

```python
from tqdm import tqdm
import time

for i in tqdm(range(100)):
    time.sleep(0.02)
```

이게 다입니다. 이제 여러분의 루프는 NASA에서 모니터링할 법한 모습으로 바뀝니다.

### 6. Autogluon — 10,000줄의 코드 없이 머신러닝을 자동화합니다

이 라이브러리는 ML (Machine Learning, 머신러닝) 워크플로를 테스트하다가 우연히 발견했습니다. `Autogluon` (오토글루온)을 사용하면 하이퍼파라미터(hyperparameter)의 복잡한 설정에 깊이 빠져들지 않고도 고품질 모델을 훈련할 수 있습니다.

```python
from autogluon.tabular import TabularPredictor

predictor = TabularPredictor(label="target").fit("data/train.csv")
preds = predictor.predict("data/test.csv")
```

ML이 너무 오래 걸린다고 주장하는 사람들을 위한 '변명의 여지가 없는' 라이브러리입니다.

**왜 중요할까요:** 자동화는 단순히 스크립트에 관한 것이 아니라, 의사결정 자동화에 관한 것이기도 합니다.

### 7. Ruff — 코드는 비판가보다 스타일리스트를 필요로 하니까요

저는 세상의 모든 린터(linter)를 시도해 봤습니다. `Ruff` (러프)는 마침내 제가 정착한 린터입니다. 빠르고(Rust로 작성됨), 구성 가능하며, 실제로 사용하기 즐겁습니다.

```bash
pip install ruff
ruff check .
```

`pre-commit` 훅(pre-commit hooks)에 추가하면, 감시받는 느낌 없이 코드베이스가 일관성 있게 깔끔하게 유지되는 것을 볼 수 있습니다.

### 8. Pyinput + Clipboard — 작은 해킹, 큰 이득

때로는 최고의 자동화는 로컬에서 이루어집니다. 저는 한때 키보드 입력을 감지하고 스니펫(snippets)을 자동으로 복사하는 작은 도구를 만들었습니다.

```python
import pyinput
import clipboard

@pyinput.listener
def on_key(key):
    if key == "F9":
        clipboard.copy("import numpy as np")
        print("Snippet copied!")
```

**왜 훌륭할까요:** 이것은 여러분의 워크플로에 맞춰 컴퓨터를 조정하기 시작하는 것과 같습니다. 거꾸로 컴퓨터의 워크플로에 맞춰가는 것이 아닙니다.

### 9. Prefect — 정말로 즐겁게 사용할 수 있는 오케스트레이션(orchestration) 도구

어느 시점이 되면 자동화 작업이 복잡해집니다. Cron 작업과 셸 스크립트로는 한계가 옵니다. `Prefect` (프리펙트)는 YAML 설정의 고통 없이 Python처럼 느껴지는 우아한 워크플로 관리자를 제공합니다.

```python
from prefect import flow, task

@task
def say_hello(name):
    print(f"Hello, {name}!")

@flow
def greet_flow():
    say_hello("Mahad")

greet_flow()
```

이것은 `Airflow` (에어플로우)의 젊고 멋진 사촌 같습니다.

**전문가 팁:** `Prefect Cloud` (프리펙트 클라우드)를 사용하여 플로우(flow)를 무료로 시각화해 보세요.

### 10. Rich-Argparse — 고통 없이 아름다운 CLI (명령줄 인터페이스)

기본 `argparse` (아그파스)는 1995년에 설계된 것처럼 보입니다(실제로 그렇습니다). `rich-argparse` (리치-아그파스)는 추가 보일러플레이트(boilerplate) 코드 없이 색상, 서식, 이모지를 추가합니다.

```python
from rich_argparse import RichHelpFormatter
import argparse

parser = argparse.ArgumentParser(
    description="Automate your day",
    formatter_class=RichHelpFormatter
)
parser.add_argument("--task", help="What should I automate?")
parser.parse_args()
```

이것은 여러분의 명령줄 도구를 즉시 프리미엄처럼 보이게 만듭니다.

### 11. HTTPX + Trio — 제대로 된 비동기(asynchronous) 자동화

네, 압니다. 비동기(async)는 어디에나 있습니다. 하지만 `HTTPX` (에이치티티피엑스)를 `Trio` (트리오)와 결합하면 동시 웹 요청을 깔끔하고, 빠르며, 파이썬답게 처리할 수 있습니다.

```python
import httpx
import trio

async def fetch(url):
    async with httpx.AsyncClient() as client:
        r = await client.get(url)
        print(url, r.status_code)

async def main():
    urls = ["https://example.com", "https://python.org"]
    async with trio.open_nursery() as nursery:
        for u in urls:
            nursery.start_soon(fetch, u)

trio.run(main)
```

**왜 배울 가치가 있을까요:** 진정한 자동화는 병렬로 이루어집니다.
