# 7 Python GUI Frameworks Entered. Only One Made Me Want to Keep Coding.

파이썬(Python) 자동화 도구를 구축하는 과정에서 사용자 인터페이스(UI)를 만드는 일은 생각보다 쉽지 않았습니다. 어떤 프레임워크는 2005년에 멈춰 있는 듯했고, 어떤 것은 단순한 기능을 구현하는 데도 너무 많은 보일러플레이트(boilerplate) 코드가 필요했습니다.

자동화 툴을 위한 최적의 GUI 도구를 찾기 위해 제가 세운 기준은 다음과 같습니다:

*   **구축 속도:** GUI 제작이 자동화 로직보다 오래 걸려서는 안 됨
*   **파이썬다운(Pythonic) 디자인:** 낯선 구문이 없을 것
*   **안정성 및 유지보수**
*   **현대적인 위젯 디자인**
*   **간편한 배포:** 앱 패키징 과정이 수월할 것
*   **자동화 워크플로우 지원**

테스트한 7가지 프레임워크의 상세 후기를 공유합니다.

---

### 1. Tkinter — 여전히 작동하는 '할아버지' 프레임워크

Tkinter는 가장 오래되었지만 여전히 현역입니다. 하지만 자동화 스크립트를 위한 깔끔한 대시보드를 만들려고 하면 레이아웃이 경직되어 있고 스타일링이 고통스럽다는 단점이 명확히 드러납니다.

*   **평가:** 아주 작은 도구에는 적합하지만, 현대적인 느낌을 주기에는 역부족입니다. 단순한 도구라고 해서 반드시 확장성이 좋은 것은 아닙니다.

### 2. PyQt — 강력하고 아름답지만, 너무 많은 공수가 필요함

PyQt는 완벽하고 아름다운 앱을 만들 수 있게 해줍니다. 하지만 자동화 제어판 하나를 만드는 데 실제 로직보다 시그널(signals), 슬롯(slots), 스타일링 작업에 더 많은 시간을 쏟게 됩니다.

*   **평가:** 대규모 기업용 앱에는 완벽하지만, 빠르고 가볍게 만들어야 하는 자동화 도구에는 과한(Overkill) 선택입니다.

### 3. PySide — PyQt의 조금 더 친절한 쌍둥이

라이선스 측면에서 PyQt보다 조금 더 관대하다는 점을 제외하면 기능이나 복잡성, 스타일링의 번거로움은 거의 동일합니다. 빠른 구축이 생명인 자동화 툴에서는 여전히 부담스러운 절차가 많습니다.

### 4. Kivy — 멋진 애니메이션, 하지만 혼란스러운 구조

Kivy는 터치 기반 앱이나 모바일 스타일의 UI, 화려한 제스처를 구현하는 데 뛰어납니다. 하지만 일반적인 파이썬 자동화 대시보드를 만들기에는 2+2를 계산하기 위해 고성능 GPU를 가져오는 것과 같은 과잉 성능 느낌을 줍니다. 생산성 대시보드에는 이상적이지 않습니다.

### 5. wxPython — 의외로 괜찮은 복병

네이티브(Native) 앱 같은 느낌을 주며 깔끔합니다. 하지만 여전히 '빠르게 만들어내는' 매력은 조금 부족합니다. 탄탄하고 정돈되어 있지만, 최고의 선택지는 아니었습니다.

### 6. DearPyGUI — 현대적이고 빠르지만, 틈새시장용

GPU 가속 위젯과 실시간 대시보드, 엄청난 속도의 플로팅(Plotting) 기능을 제공합니다. 데이터가 많고 시각화가 중요한 자동화 도구에는 훌륭하지만, 일반적인 목적의 GUI로 사용하기에는 다소 특화되어 있습니다.

### 7. CustomTkinter — 코딩을 계속하고 싶게 만든 단 하나의 선택 (우승)

CustomTkinter는 우리가 오래전부터 원했던 '현대적인 Tkinter'입니다. 깔끔한 디자인, 다크 모드 지원, 논리적인 API, 단순한 레이아웃을 제공하며 개발 주기가 매우 빠릅니다.

Qt나 Kivy처럼 복잡하지 않으면서도 파이썬답고 미니멀하며 현대적입니다. 테마 설정과 레이아웃 구성이 직관적이어서 자동화 도구 GUI를 20분 만에 뚝딱 완성할 수 있었습니다.

*   **우승 이유:** 빠른 구축 속도, 현대적인 UI, 과도한 고민이 필요 없는 간결함, 대부분의 자동화 작업에 충분한 기능성 사이의 완벽한 균형을 갖췄습니다.

---

### 결론: 7가지 프레임워크를 테스트하며 배운 점

대부분의 파이썬 GUI 프레임워크는 '애플리케이션'을 만들기 위해 설계되었지, '신속한 자동화 워크플로우'를 위해 설계되지 않았습니다.

개발자에게 필요한 것은 빠르고 기능적이며 가벼운 도구입니다. 그런 면에서 **CustomTkinter**는 자동화 개발자에게 가장 적합한 지점을 공략합니다.

**마지막 조언:** GUI 개발 시간은 주말 정도로 제한하세요. 여러분의 자동화 로직은 그 로직을 담는 창(Window)보다 훨씬 더 많은 가치를 지니고 있습니다.
