# 실제로 만들어보고 싶은 10가지 파이썬 프로젝트

단순히 기능을 구현하는 것을 넘어, 실력을 한 단계 끌어올려 줄 파이썬 프로젝트 10가지를 소개합니다. 흔한 '할 일 관리(To-do)' 앱이나 날씨 확인 프로그램은 문법(Syntax)을 가르쳐줄 순 있지만, 개발자에게 필요한 판단력을 길러주지는 못합니다.

### 1. Git 커밋 아키올로지스트 (Git Commit Archaeologist)
훌륭한 개발자는 코드를 읽는 것에 그치지 않고, 그 코드가 왜 존재하는지를 파악합니다. Git 저장소를 분석하여 다음과 같은 질문에 답하는 도구를 만들어 보세요.

*   어떤 파일이 가장 빈번하게 수정되는가?
*   어떤 커밋이 가장 많은 버그를 유발했는가?
*   실질적으로 코드베이스의 각 부분을 담당하는 사람은 누구인가?

```python
import subprocess
from collections import Counter

def git_log():
    result = subprocess.run(
        ["git", "log", "--name-only", "--pretty=format:"],
        capture_output=True,
        text=True
    )
    return result.stdout.splitlines()

files = [f for f in git_log() if f]
hotspots = Counter(files).most_common(10)

for file, count in hotspots:
    print(f"{file}: {count}번 변경됨")
```
**핵심 학습 포인트:** 시스템의 시간에 따른 변화를 추론하는 능력을 기를 수 있습니다. 이는 시니어 개발자가 갖춰야 할 핵심 역량 중 하나입니다.

### 2. 결정론적 작업 스케줄러 (Deterministic Task Scheduler)
cron을 사용하지 않고 직접 가벼운 스케줄러를 구축해 봅니다.

*   정해진 시간에 정확히 작업 실행
*   재시작 후에도 상태 유지(Persists state)
*   최대 한 번 실행(At-most-once execution) 보장

```python
import time
import json
from datetime import datetime

TASKS_FILE = "tasks.json"

def load_tasks():
    return json.load(open(TASKS_FILE))

def run():
    while True:
        now = datetime.utcnow().isoformat()
        tasks = load_tasks()
        for task in tasks:
            if task["run_at"] <= now and not task["done"]:
                exec(task["code"])
                task["done"] = True
        json.dump(tasks, open(TASKS_FILE, "w"))
        time.sleep(1)
```
**핵심 학습 포인트:** 멱등성(Idempotency), 영속성(Persistence), 실패 복구(Failure recovery) 등 분산 시스템의 사고방식을 익힐 수 있습니다.

### 3. 이진 파일 인스펙터 (Binary File Inspector)
대부분의 개발자는 텍스트 파일만 다룹니다. 이진(Binary) 파일을 직접 다루어 보는 것은 매우 값진 경험입니다. PNG 파일의 헤더를 파싱하여 메타데이터를 추출하고 손상 여부를 확인하는 도구를 만들어 보세요.

```python
import struct

def read_png_header(path):
    with open(path, "rb") as f:
        signature = f.read(8)
        if signature != b'\x89PNG\r\n\x1a\n':
            raise ValueError("PNG 파일이 아닙니다.")
        length, chunk_type = struct.unpack(">I4s", f.read(8))
        return length, chunk_type

print(read_png_header("image.png"))
```
**핵심 학습 포인트:** 바이트(Bytes), 엔디언(Endianness), 데이터 레이아웃 등 일반적인 파이썬 개발자가 놓치기 쉬운 저수준 프로그래밍 개념을 이해하게 됩니다.

### 4. 로그 상관관계 엔진 (Log Correlation Engine)
로그 자체가 우리를 실망시키는 것이 아니라, 로그를 읽는 능력이 부족한 것이 문제입니다. 여러 로그 파일을 통합하여 장애 타임라인을 재구성하는 도구를 만들어 보세요.

```python
from datetime import datetime

def parse(line):
    ts, msg = line.split(" ", 1)
    return datetime.fromisoformat(ts), msg

events = []
for file in ["api.log", "db.log"]:
    for line in open(file):
        events.append(parse(line))

for ts, msg in sorted(events):
    print(ts, msg)
```
**핵심 학습 포인트:** 가시성(Observability)과 근본 원인 분석(Root-cause analysis) 능력을 배양할 수 있습니다.

### 5. 제약 기반 설정 검증기 (Constraint-Based Configuration Validator)
설정값은 사람이 작성하기에 실수가 잦습니다. 여러 설정 파일 간의 규칙을 적용하고 모순을 찾아내는 검증기를 만들어 보세요.

```python
def validate(cfg):
    errors = []
    if cfg["retries"] > 5 and not cfg["timeout"]:
        errors.append("재시도 횟수가 5회 초과일 경우 타임아웃 설정이 필수입니다.")
    return errors
```
**핵심 학습 포인트:** 선언적 사고(Declarative thinking), 검증 레이어 구축, 방어적 프로그래밍(Defensive programming)을 익힙니다.

### 6. 데이터 드리프트 탐지기 (Data Drift Detector)
데이터는 조용히 변하고, 시스템은 요란하게 실패합니다. 오늘의 데이터 분포를 지난주와 비교하여 변화를 감지하는 스크립트를 작성해 보세요.

```python
import statistics

def drift(old, new):
    return abs(statistics.mean(old) - statistics.mean(new))

print(drift([1,2,3], [10,11,12]))
```
**핵심 학습 포인트:** 입력을 무조건 신뢰하지 않는 마음가짐을 배웁니다. 이는 프로덕션 시스템을 안정적으로 유지하는 비결입니다.

### 7. 미니멀 검색 엔진 (Minimal Search Engine)
라이브러리를 쓰지 않고 역색인(Inverted Index) 구조를 직접 구현해 봅니다.

```python
from collections import defaultdict

index = defaultdict(set)

docs = {
    1: "python makes you dangerous",
    2: "dangerous developers write tests",
}

for doc_id, text in docs.items():
    for word in text.split():
        index[word].add(doc_id)

print(index["dangerous"])
```
**핵심 학습 포인트:** 검색 엔진, 데이터베이스, 컴파일러의 핵심 동작 원리를 이해하게 됩니다.

### 8. 재생 가능한 버그 시뮬레이터 (Replayable Bug Simulator)
버그가 발생했을 때의 입력값과 상태를 기록하여, 나중에 동일한 상황을 결정론적으로 재현할 수 있는 시스템을 구축해 보세요.

```python
import json

def record(inputs, output):
    json.dump({"inputs": inputs, "output": output}, open("trace.json", "w"))
```
**핵심 학습 포인트:** 재현성(Reproducibility)은 막연한 추측보다 훨씬 강력한 디버깅 도구입니다.

### 9. 파일 시스템 스냅샷 도구 (File System Snapshot Tool)
파일의 해시값, 메타데이터 변경, 삭제 및 수정 사항을 추적하는 도구를 만들어 보세요.

```python
import os, hashlib

def snapshot(path):
    state = {}
    for root, _, files in os.walk(path):
        for f in files:
            p = os.path.join(root, f)
            state[p] = hashlib.md5(open(p,'rb').read()).hexdigest()
    return state
```
**핵심 학습 포인트:** 상태 비교, 불변성(Immutability), 변화 감지 로직을 학습할 수 있습니다.

### 10. "왜 고장 났을까?" 분석기 (“Why Did This Break?” Analyzer)
단순히 실패 로그를 남기는 것이 아니라, 프로그램의 '가정(Assumptions)'을 로그로 남기는 방식입니다. 기대하는 데이터의 범위나 불변성(Invariants)을 정의하고, 이를 위반할 때마다 기록합니다.

```python
def assert_between(x, low, high):
    if not (low <= x <= high):
        raise ValueError(f"{x}가 허용 범위 [{low}, {high}]를 벗어났습니다.")
```
**핵심 학습 포인트:** 버그를 바라보는 관점 자체를 바꾸어 줍니다. 방어적인 코드 작성이 습관화됩니다.
