## 1. `lazypredict` — '무엇이 효과적인지 보여주는' 라이브러리

새로운 머신러닝(ML) 프로젝트를 시작할 때, 저의 첫 질문은
"어떤 모델(model)을 튜닝(tune)해야 할까?"가 아닙니다.
오히려
"어떤 종류의 모델이 내 시간을 투자할 가치가 있을까?"입니다.

`lazypredict`는 이 질문에 잔인할 정도로 빠르게 답해줍니다.

```python
from lazypredict.Supervised import LazyClassifier
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split

X, y = load_breast_cancer(return_X_y=True)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)
clf = LazyClassifier(verbose=0)

models, predictions = clf.fit(X_train, X_test, y_train, y_test)

print(models.head())
```

한 번의 실행으로 수십 가지 모델에 대한 다음 정보를 얻을 수 있습니다:

*   정확도 (Accuracy)
*   ROC AUC (Receiver Operating Characteristic Area Under the Curve)
*   F1 점수 (F1 score)
*   학습 시간 (Training time)

**왜 이것이 중요할까요:**
모델에 대한 직관(intuition)을 길러줍니다. 추측을 멈추고 데이터셋(dataset) 전반의 패턴을 관찰하기 시작하게 됩니다.

숙련된 개발자들이 이것 하나만으로 며칠간의 시행착오를 절약하는 것을 보았습니다.

## 2. `skops` — 모델을 성숙하게 직렬화(Serialize)하는 방법

파이썬 피클(Pickle)은 작동합니다… 작동하지 않을 때까지는요.
보안 문제, 버전 불일치, 알 수 없는 오류(silent failures) 등이 발생할 수 있습니다.

`skops`는 머신러닝(ML) 모델, 특히 사이킷런(scikit-learn) 모델을 직렬화(serialize)하는 더 새롭고 안전한 방법입니다.

```python
from skops.io import dump, load
from sklearn.linear_model import LogisticRegression

model = LogisticRegression().fit(X_train, y_train)

dump(model, "model.skops")
loaded_model = load("model.skops")
```

**무엇이 특별할까요:**

*   명시적인 신뢰 정책 (Explicit trust policies)
*   검사 가능한 내용 (Inspectable contents)
*   설계상 피클(pickle)보다 안전함 (Safer than pickle by design)

모델을 배포하거나 팀 간에 공유한다면, 이 라이브러리는 더 일찍 도입했으면 좋았을 것이라고 생각하게 될 라이브러리 중 하나입니다.

## 3. `cleanlab` — 존재조차 몰랐던 레이블(Label) 오류를 찾아내다

여기 머신러닝(ML)의 숨겨진 비밀이 있습니다:

당신의 데이터셋(dataset)은 틀렸습니다.

'약간 노이즈(noisy)가 있다' 정도가 아닙니다.
적극적으로 잘못되어 있습니다.

`cleanlab`은 실제 정답(ground truth)이 없어도 잘못 레이블링(mislabeled)된 데이터를 찾아냅니다.

```python
from cleanlab.classification import CleanLearning
from sklearn.linear_model import LogisticRegression

cl = CleanLearning(LogisticRegression())
cl.fit(X_train, y_train)

label_issues = cl.get_label_issues()
print(label_issues[:10])
```

이 라이브러리는 동료 심사(peer-reviewed)를 거친 연구를 기반으로 하며, 실제 프로덕션(production) 파이프라인(pipeline)에서 사용됩니다.

**전문가들이 이 라이브러리를 좋아하는 이유:**
레이블을 수정하는 것이 모델을 바꾸는 것보다 성능을 향상시키는 경우가 많습니다.

저는 개인적으로 레이블을 정리하는 것만으로 정확도(accuracy)가 8~12% 향상되는 것을 보았습니다.

## 4. `river` — 끊임없이 들어오는 데이터를 위한 머신러닝

대부분의 튜토리얼(tutorial)은 정적 데이터셋(static dataset)을 가정합니다.
하지만 실제 상황은 그렇지 않습니다.

`river`는 한 번에 하나의 데이터 포인트(data point)씩 모델을 학습시킬 수 있도록 해줍니다.

```python
from river import linear_model, metrics

model = linear_model.LogisticRegression()
metric = metrics.Accuracy()

for x, y in stream_of_data:
    y_pred = model.predict_one(x)
    metric.update(y, y_pred)
    model.learn_one(x, y)

print(metric)
```

**다음에 적합합니다:**

*   스트림 (Streams)
*   로그 (Logs)
*   사물 인터넷 데이터 (IoT data)
*   실시간 사용자 행동 (Live user behavior)

**이 라이브러리가 가르치는 사고의 전환:**
머신러닝(ML)은 배치(batch) 기반일 필요가 없습니다. 모델은 진화할 수 있습니다.

이것을 이해하면 많은 '실시간 머신러닝(real-time ML)' 개념이 갑자기 명확해집니다.

## 5. `evidently` — 모델을 맹목적으로 신뢰하지 마세요

모델을 학습시키는 것은 쉽습니다.
하지만 언제 모델이 '거짓말을 하는지'(제대로 작동하지 않는지) 아는 것은 어렵습니다.

`evidently`는 다음을 감지하는 데 도움을 줍니다:

*   데이터 드리프트 (Data drift)
*   콘셉트 드리프트 (Concept drift)
*   성능 저하 (Performance decay)

```python
from evidently.report import Report
from evidently.metrics import DataDriftPreset

report = Report(metrics=[DataDriftPreset()])
report.run(reference_data=train_df, current_data=prod_df)

report.save_html("drift_report.html")
```

이것은 다음 두 가지 문장의 차이입니다:

"우리 모델은 테스트(testing)에서 훌륭했습니다."
그리고
"우리 모델은 오늘도 여전히 잘 작동하고 있습니다."

모니터링(monitoring) 없는 프로덕션(production) 머신러닝(ML)은 자신감을 가지고 도박을 하는 것과 같습니다.

## 6. `mlxtend` — 사람에게 설명하듯 머신러닝을 풀어내다

`mlxtend`는 터무니없을 정도로 저평가되어 있습니다.

이 라이브러리는 과대광고(hype)가 아닌 명확성에 초점을 맞춥니다.

```python
from mlxtend.feature_selection import SequentialFeatureSelector
from sklearn.linear_model import LogisticRegression

sfs = SequentialFeatureSelector(
    LogisticRegression(),
    k_features=5,
    forward=True,
    scoring='accuracy'
)

sfs.fit(X_train, y_train)
print(sfs.k_feature_names_)
```

**포함된 기능:**

*   특징 선택 (Feature selection)
*   편향-분산 도구 (Bias-variance tools)
*   시각화 보조 도구 (Visualization helpers)

단순히 머신러닝을 실행하는 것을 넘어 이해하고 싶다면, 이 라이브러리가 조용히 그 가치를 제공할 것입니다.

## 7. `interpret` — 블랙 박스(Black Box)를 열다

설명 가능성(explainability) 없는 정확도(accuracy)는 다음 분야에서 무용지물입니다:

*   금융 (Finance)
*   의료 (Healthcare)
*   채용 (Hiring)
*   결과에 영향을 미치는 모든 시스템 (Any system with consequences)

`interpret`는 강력하면서도 설명 가능한 글라스 박스(glass-box) 모델을 제공합니다.

```python
from interpret.glassbox import ExplainableBoostingClassifier

ebm = ExplainableBoostingClassifier()
ebm.fit(X_train, y_train)

ebm.explain_global()
```

특징 효과(feature effects)가 명확하게 시각화된 것을 처음 보게 되면, 모델에 대한 사고방식이 바뀔 것입니다.

맹목적인 최적화(optimization)를 멈추고 추론하기 시작할 것입니다.

---

## 8. `sklego` — 실제로 사용하는 실용적인 머신러닝(ML) 패턴

`sklego` (사이킷-레고, scikit-lego)는 사이킷런(scikit-learn)에 부족한 구성 요소(building blocks)들을 추가합니다.

한 가지 예: 불균형 데이터(imbalanced data)를 깔끔하게 처리하는 것.

```python
from sklego.meta import Thresholder
from sklearn.linear_model import LogisticRegression

model = Thresholder(
    LogisticRegression(),
    threshold=0.7
)

model.fit(X_train, y_train)
```

**포함된 기능:**

*   사용자 정의 손실 처리 (Custom loss handling)
*   비즈니스 규칙을 인지하는 모델 (Business-rule-aware models)
*   실용적인 파이프라인 (Practical pipelines)

이 라이브러리는 프로덕션(production) 환경에서 여러 번 곤욕을 치른 사람이 작성한 것 같은 느낌을 줍니다. 실제로도 그렇습니다.
