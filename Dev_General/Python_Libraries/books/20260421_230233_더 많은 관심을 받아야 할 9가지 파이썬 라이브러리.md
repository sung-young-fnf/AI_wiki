## 1. diskcache — 당신을 배신하지 않는 캐싱

대부분의 캐싱 솔루션은 다음과 같습니다:

*   너무 단순해서 (쓸모없거나)
*   너무 복잡해서 (중간에 포기하게 됩니다)

`diskcache`는 딱 좋은 지점을 찾아냅니다: 영구적이고, 빠르며, 스레드 안전(thread-safe)하고, 놀랍도록 사용하기 쉽습니다.

```python
from diskcache import Cache

cache = Cache('./cache_dir')

@cache.memoize()
def expensive_computation(x):
    print("Running computation...")
    return x * x

print(expensive_computation(10))
print(expensive_computation(10))  # Cached
```

왜 중요한가요:
인메모리 캐시(in-memory cache)와 달리, 이 캐시는 재시작 후에도 유지됩니다. 저는 이 라이브러리를 사용하여 한 프로젝트에서 API 비용을 약 40% 절감했습니다.

## 2. pyinstrument — 골치 아프지 않은 프로파일링

프로파일링(profiling) 도구는 대개... 고통을 안겨줍니다.

`pyinstrument`는 명확성을 제공합니다.

```python
from pyinstrument import Profiler

profiler = Profiler()
profiler.start()

# Your slow code
sum(i*i for i in range(10_000_000))

profiler.stop()
print(profiler.output_text(unicode=True, color=True))
```

얻게 될 것:
읽기 쉬운 호출 스택(call stack). 숫자의 벽이 아닙니다.

이것이 성능 문제를 짐작하는 것과 코드의 어느 부분에서 성능 저하가 발생하는지 정확히 아는 것의 차이입니다.

## 3. pendulum — 드디어 제대로 된 날짜/시간 처리

파이썬의 `datetime` 모듈은... 정중하게 말하자면... 좀 특이합니다.

`pendulum`이 이 문제를 해결합니다.

```python
import pendulum

now = pendulum.now("UTC")
print(now.add(days=3).to_datetime_string())

# Timezone sanity
dt = pendulum.parse("2024-05-01 12:00", tz="Asia/Karachi")
print(dt.in_timezone("Europe/Paris"))
```

왜 이 라이브러리가 존재할까요:
시간대(timezone)는 버그가 숨어 있기 좋은 곳입니다. `pendulum`은 그 버그들을 밝은 곳으로 끌어냅니다.

## 4. thefuzz — 스마트한 문자열 매칭

엉망인 사용자 입력을 매칭해 본 적이 있나요?

"john doe", "جون دو", "Jon Do"... 혼돈의 세계에 오신 것을 환영합니다.

```python
from thefuzz import fuzz

score = fuzz.ratio("john doe", "jon do")
print(score)
```

더 나은 것을 원하시나요?

```python
from thefuzz import process

choices = ["John Doe", "Jane Smith", "Johnny Do"]
match = process.extractOne("jon do", choices)

print(match)
```

실제 사용 사례:
검색 시스템, 중복 제거, 데이터셋 정리.

이것이 앱을 똑똑하게 만드는 방법입니다.

## 5. python-magic — (진짜로) 파일을 파악하기

파일 확장자는 거짓말을 합니다. 사용자도 거짓말을 합니다. 시스템도 거짓말을 합니다.

파일 시그니처(file signature)는 그렇지 않습니다.

```python
import magic

file_type = magic.from_file("unknown_file", mime=True)
print(file_type)
```

이 라이브러리가 왜 값진가요:
저는 한때 `.jpg` 파일이었지만 실제로는 악성 실행 파일이었던 것을 잡아낸 적이 있습니다. 확장자는 다른 것을 말했지만, 현실은 달랐습니다.

## 6. loguru — 정말 즐거운 로깅

내장 `logging` 모듈은 작동하지만... 설명서 없이 이케아 가구를 조립하는 기분입니다.

`loguru`는 플러그 앤 플레이(plug-and-play) 방식입니다.

```python
from loguru import logger

logger.add("app.log", rotation="1 MB")
logger.info("App started")

logger.error("Something broke")
```

추가 기능:
아름다운 로그. 최소한의 상용구 코드(boilerplate code).

로깅은 더 이상 귀찮은 일이 아니라 유용한 작업이 됩니다.

## 7. better-exceptions — 실제로 읽을 수 있는 오류 메시지

트레이스백(traceback)은 우리를 돕기 위해 존재하지만, 때로는 수수께끼처럼 느껴집니다.

이 라이브러리가 그 문제를 해결합니다.

```python
import better_exceptions
better_exceptions.hook()

def divide(a, b):
    return a / b

divide(10, 0)
```

보게 될 것:
트레이스백 내부에 있는 변수 값.

이것만으로도 디버깅 시간을 절반으로 줄일 수 있습니다.

## 8. h11 — 불필요한 기능 없는 HTTP

`fastapi`와 같은 프레임워크 아래에서 어떤 일이 벌어지는지 궁금해 본 적이 있다면...

`h11`은 원시적이고(raw), 최소한의(minimal) HTTP를 제공합니다.

```python
import h11

conn = h11.Connection(h11.CLIENT)

request = h11.Request(
    method="GET",
    target="/",
    headers=[("Host", "example.com")]
)

conn.send(request)
```

왜 중요한가요:
항상 전체 프레임워크가 필요한 것은 아닙니다. 때로는 그저 제어(control)가 필요할 뿐입니다.

이것은 도구들이 실제로 무엇을 하는지 배우는 방법입니다.

## 9. schedule — 고통 없는 크론 작업

크론 작업(cron job)을 설정하는 것은 고대 언어로 주문을 쓰는 것 같은 기분입니다.

이것은 더 깔끔합니다.

```python
import schedule
import time

def job():
    print("Running task...")

schedule.every(10).seconds.do(job)

while True:
    schedule.run_pending()
    time.sleep(1)
```

다음 용도로 완벽합니다:
빠른 자동화, 스크립트, 프로토타입.

설정 파일도, 골치 아픈 일도 없습니다.

읽어주셔서 감사합니다!
