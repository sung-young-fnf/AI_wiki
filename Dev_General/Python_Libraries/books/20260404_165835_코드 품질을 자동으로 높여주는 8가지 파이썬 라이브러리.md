# 코드 품질을 자동으로 높여주는 8가지 파이썬 라이브러리

파이썬을 수년간 작성해 왔다면 코드가 작동하고 테스트를 통과하며 CI(지속적 통합)가 정상적으로 작동할 것입니다. 하지만 단순히 '작동하는 코드'와 '깨끗한 코드(Clean Code)' 사이에는 분명한 차이가 있습니다. 

실제로 많은 개발자가 여전히 예외 상황(Edge cases)을 놓치거나, 타입 힌트(Type hint)를 누락하고, 문서화(Docstring)를 건너뛰곤 합니다. 전문적인 파이썬 개발자는 단순히 좋은 코드를 작성하는 데 그치지 않고, 좋은 코드를 작성할 수밖에 없는 시스템을 구축합니다. 코드 품질을 자동으로 개선하고 강제하는 8가지 라이브러리를 소개합니다.

### 1. Ruff — 린터, 포매터, 코드 비평가를 하나로

flake8, isort, pycodestyle, pylint를 별도로 실행하고 있다면 불필요한 작업을 하고 있는 것입니다. Ruff는 이 모든 도구를 대체하며, 러스트(Rust)로 작성되어 믿기지 않을 정도로 빠릅니다.

*   **기능:** 린팅(Linting), 자동 수정(Auto-fix), 임포트 정렬(Import sorting), 포매팅 적용, 미사용 변수 감지 등.
*   **핵심:** 엘리트 엔지니어는 사용하지 않는 임포트(Unused imports)를 커밋하지 않습니다. Ruff는 이를 즉각적으로 잡아냅니다.

### 2. MyPy — 파이썬을 엄격한 언어처럼 관리하기

타입 힌트를 반드시 사용할 필요는 없지만, 사용하지 않는다면 보이지 않는 위험을 안고 가는 것과 같습니다. MyPy는 런타임(Runtime) 전에 버그를 찾아냅니다.

*   **장점:** 함수 인자에 잘못된 타입이 들어오는 것을 사전에 방지합니다. 타입 체크를 철저히 하면 추측이 아닌 설계를 바탕으로 코딩할 수 있습니다.

### 3. Pydantic — 자동 데이터 검증

대부분의 버그는 잘못된 데이터(사용자 입력, API 페이로드, 환경 변수 등)에서 시작됩니다. Pydantic은 가공되지 않은 데이터를 검증된 객체로 변환합니다.

*   **효과:** 타입을 안전하게 자동 변환하며, 유효하지 않은 값이 들어올 경우 `ValidationError`를 발생시킵니다. 검증되지 않은 프로덕션 API는 시한폭탄과 같으며, Pydantic은 이를 해제하는 역할을 합니다.

### 4. Bandit — 내장 보안 스캐너

작성한 코드가 안전하다고 확신할 수 있습니까? Bandit은 코드에서 공통적인 보안 취약점을 점검합니다.

*   **기능:** 예를 들어 보안상 위험한 `subprocess.call(..., shell=True)`와 같은 구문을 찾아내 알림을 줍니다. 백엔드 코드를 작성한다면 보안 점검은 선택이 아닌 필수입니다.

### 5. Vulture — 미사용 코드(Dead Code) 즉시 탐색

잊어버린 함수, 사용되지 않는 변수, 나중에 삭제하려고 남겨둔 오래된 로직은 모두 기술 부채(Technical debt)입니다.

*   **효과:** Vulture는 프로젝트 내에서 호출되지 않는 코드를 찾아냅니다. 사용하지 않는 코드를 제거함으로써 인지적 과부하를 줄이고 군더더기 없는 코드를 유지할 수 있습니다.

### 6. Radon — 코드 복잡도 측정

함수 내부가 너무 복잡해 읽기 두렵다면, 그것은 순환 복잡도(Cyclomatic complexity)가 높기 때문입니다.

*   **기능:** Radon은 코드의 복잡도를 측정하여 점수를 매깁니다. 복잡도를 측정하기 시작하면 더 나은 함수를 설계하게 됩니다. 만약 함수 하나를 읽는 데 스크롤이 필요하다면, 그 함수는 분리해야 할 가능성이 높습니다.

### 7. Docformatter — 독스트링 자동 정리

독스트링(Docstring)은 코드의 가독성을 높이지만, 형식이 일정하지 않으면 지저분해 보입니다.

*   **효과:** Docformatter는 독스트링의 서식을 자동으로 교정합니다. 사소한 디테일처럼 보이지만, 이런 디테일이 모여 전문적인 코드를 만듭니다.

### 8. Pyupgrade — 최신 문법으로 자동 업데이트

여전히 파이썬 3.6 스타일의 코드를 작성하고 계신가요? Pyupgrade는 구형 구문을 최신 파이썬 문법으로 리팩터링(Refactoring)합니다.

*   **기능:** f-스트링(f-strings), 최신 타입 힌트, 더 이상 사용되지 않는(Deprecated) 구문 등을 현재 버전에 맞는 최적화된 코드로 업그레이드해 줍니다.

### 품질 파이프라인 구축 (자동화)

이러한 도구들을 수동으로 사용하는 것보다 더 중요한 것은 자동화입니다. `pre-commit` 설정을 통해 코드가 커밋되기 전에 자동으로 이러한 검사를 수행하도록 강제할 수 있습니다.

```yaml
# .pre-commit-config.yaml 예시
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.1.0
    hooks:
      - id: ruff
        args: [--fix]
  - repo: https://github.com/pre-commit/mirrors-mypy
    rev: v1.7.0
    hooks:
      - id: mypy
  - repo: https://github.com/PyCQA/bandit
    rev: 1.7.5
    hooks:
      - id: bandit
```

### 결론

훌륭한 개발자는 더 많은 코드를 쓰는 데 집중하는 것이 아니라, 나쁜 코드를 덜 쓰는 데 집중합니다. 동기부여나 생산성 팁에 의존하기보다 '안전 장치(Guardrails)'를 구축하십시오. 

이러한 라이브러리들을 워크플로우에 통합하면 코드 결함이 줄어들고, 코드 리뷰 시간은 단축되며, 개발자의 자신감은 상승할 것입니다. 이것이 바로 전문가가 일하는 방식입니다.
