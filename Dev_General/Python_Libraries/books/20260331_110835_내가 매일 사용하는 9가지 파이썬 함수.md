### 1. once() – 딱 한 번만 실행 보장 (임포트 시 포함)

파이썬 모듈은 기본적으로 한 번 실행되지만, 순환 참조(circular imports)나 재로드(reloads), 워커 프로세스 등 복잡한 상황에서는 의도치 않게 재실행될 수 있습니다. 이 함수는 호출 횟수와 상관없이 프로세스당 로직이 단 한 번만 실행되도록 보장합니다.

```python
from functools import wraps

def once(fn):
    called = False
    result = None

    @wraps(fn)
    def wrapper(*args, **kwargs):
        nonlocal called, result
        if not called:
            result = fn(*args, **kwargs)
            called = True
        return result
    return wrapper

# 사용 예시:
@once
def load_config():
    print("설정 로드 중...")
    return {"debug": False}

load_config()
load_config() # 다시 출력되지 않음
```

**사용 이유:** 불안정한 전역 플래그(global flags)를 대체하며 초기화 코드를 견고하게 만듭니다.

### 2. safe_open() – 파일 손상 없는 원자적 쓰기

파일을 쓰는 도중에 프로세스가 충돌하여 파일이 반만 작성된 채 손상된 경험이 있으신가요? 이 함수는 임시 파일에 먼저 내용을 쓴 뒤 원자적 이름 변경(atomic rename) 방식을 사용하여 파일 쓰기의 안전성을 보장합니다.

```python
import os
import tempfile
from contextlib import contextmanager

@contextmanager
def safe_open(path, mode='w', **kwargs):
    dir_name = os.path.dirname(path) or '.'
    with tempfile.NamedTemporaryFile(mode=mode, dir=dir_name, delete=False, **kwargs) as tmp:
        temp_name = tmp.name
        yield tmp
    os.replace(temp_name, path)

# 사용 예시:
with safe_open("data.json") as f:
    f.write("중요한 데이터")
```

**핵심:** POSIX 시스템에서 `os.replace`는 원자적(atomic)으로 작동하므로 작업 도중 충돌이 발생해도 기존 파일이 깨지지 않습니다.

### 3. chunks() – 메모리 안전한 반복 처리

데이터가 아무리 많아도 꼭 필요한 경우가 아니면 메모리에 전부 올리지 않는 것이 좋습니다. 이 함수는 리스트뿐만 아니라 모든 반복 가능한 객체(iterable)를 지정한 크기만큼 나누어 처리합니다.

```python
from itertools import islice

def chunks(iterable, size):
    it = iter(iterable)
    while True:
        chunk = list(islice(it, size))
        if not chunk:
            break
        yield chunk

# 사용 예시:
for batch in chunks(range(10_000_000), 1000):
    process(batch)
```

**사용 이유:** 개발 환경에서는 잘 작동하다가 실제 운영 데이터(production data)에서 터지는 코드를 방지해 줍니다.

### 4. timed() – 실제 실행 시간 측정

시스템 부하가 높은 상황에서 `time.time()`은 정확하지 않을 수 있습니다. 성능이 중요하다면 모노토닉 클락(monotonic clock)을 사용해야 합니다.

```python
import time
from contextlib import contextmanager

@contextmanager
def timed(label="block"):
    start = time.perf_counter()
    yield
    end = time.perf_counter()
    print(f"{label}: {end - start:.6f}s")

# 사용 예시:
with timed("DB 쿼리"):
    expensive_query()
```

**사용 이유:** 리팩토링 시 필수적인 도구입니다. 측정 없는 마이크로 최적화는 그저 느낌일 뿐입니다.

### 5. deep_get() – 복잡한 중첩 데이터의 안전한 접근

API 응답 데이터는 종종 러시아 인형처럼 복잡하게 중첩된 딕셔너리 형태로 들어옵니다. 이 함수는 `KeyError`나 `TypeError`로부터 개발자의 정신 건강을 지켜줍니다.

```python
def deep_get(data, path, default=None):
    for key in path:
        try:
            data = data[key]
        except (KeyError, TypeError, IndexError):
            return default
    return data

# 사용 예시:
email = deep_get(payload, ["user", "profile", "email"])
```

**사용 이유:** 연속적인 `.get()` 호출보다 가독성이 좋으며, 리스트와 딕셔너리가 혼합된 구조에서도 방어적으로 작동합니다.

### 6. trace_memory() – 숨은 메모리 누수 탐지

메모리 누수는 조용히 발생하여 프로세스를 서서히 죽게 만듭니다. 파이썬 표준 라이브러리인 `tracemalloc`을 사용해 메모리 사용량을 추적할 수 있습니다.

```python
import tracemalloc
from contextlib import contextmanager

@contextmanager
def trace_memory(label=""):
    tracemalloc.start()
    yield
    current, peak = tracemalloc.get_traced_memory()
    tracemalloc.stop()
    print(f"{label} | 현재: {current/1e6:.2f}MB, 최대: {peak/1e6:.2f}MB")

# 사용 예시:
with trace_memory("이미지 처리"):
    process_images()
```

**사용 이유:** "왜 이 서비스가 RAM을 2GB나 차지하지?"라는 의문을 해결하는 데 큰 도움을 줍니다.

### 7. fail_fast() – 명확한 맥락과 함께 종료

침묵하는 실패(Silent failure)는 크래시보다 더 위험합니다. 이 함수는 스레드 환경이나 특수한 상황에서도 예외가 발생했을 때 즉시 전체 트레이스백(traceback)을 출력하게 합니다.

```python
import faulthandler
import sys

def fail_fast():
    faulthandler.enable(file=sys.stderr, all_threads=True)

# 프로그램 시작 시 한 번 호출
```

**사용 이유:** 운영 환경에서 일부만 남은 로그는 무용지물입니다. 이 함수는 실패 원인을 솔직하고 명확하게 드러냅니다.

### 8. lazy_import() – 초기 구동 속도 개선

모든 모듈을 시작할 때 바로 임포트할 필요는 없습니다. 이 함수는 모듈을 실제로 처음 사용할 때까지 임포트를 미룹니다.

```python
import importlib

def lazy_import(module_name):
    module = None

    def load():
        nonlocal module
        if module is None:
            module = importlib.import_module(module_name)
        return module

    return load

# 사용 예시:
json = lazy_import("json")

def handler():
    data = json().loads(payload)
```

**사용 이유:** 초기 구동 시간(cold-start time)이 중요한 CLI 도구나 서버리스 함수(Serverless functions)에서 매우 유용합니다.

### 9. disk_cache() – 인프라 구축 없이 지속성 캐시 구현

캐싱은 필요하지만 Redis 같은 별도의 인프라를 구축하기 부담스러울 때가 있습니다. 파이썬의 `shelve`를 이용하면 파일 기반의 지속성 키-값 저장소를 만들 수 있습니다.

```python
import shelve
from functools import wraps

def disk_cache(path):
    def decorator(fn):
        @wraps(fn)
        def wrapper(*args):
            key = str(args)
            with shelve.open(path) as db:
                if key not in db:
                    db[key] = fn(*args)
                return db[key]
        return wrapper
    return decorator

# 사용 예시:
@disk_cache("prices.db")
def get_price(symbol):
    return expensive_api_call(symbol)
```

**참고:** `shelve`는 1999년부터 파이썬에 포함되어 있었지만 의외로 잘 쓰이지 않습니다. 하지만 활용 가치는 매우 높습니다.
