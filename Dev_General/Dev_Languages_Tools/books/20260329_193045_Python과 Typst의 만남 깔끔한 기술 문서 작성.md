하루 종일 Python 코드를 작성하다가 LaTeX 파일로 넘어가면 분위기가 달라집니다. 온통 백슬래시(`\`)로 가득하고, 긴 서문(`preamble`)과 다른 시대에서 온 듯한 구문(`syntax`)은 낯설게 느껴집니다.

Typst는 또 다른 해결책을 제시합니다. LaTeX의 정교함은 유지하면서도, 이해하기 쉬운 코드처럼 느껴지는 작고 표현력 있는 언어에 담아냈습니다.

Python 개발자의 관점에서 Typst는 문서 전용 도메인 특화 언어(DSL, Domain Specific Language)처럼 보입니다. 가벼운 마크업(`markup`)으로 텍스트를 작성하고, `#` 접두사로 표현식(`expression`)을 삽입하며, Python 개발자에게 익숙한 방식으로 함수(`function`), 변수(`variable`), 데이터 구조(`data structure`)를 정의할 수 있습니다. Typst는 조판(`typesetting`)을 마법처럼 다루는 대신, 페이지를 생성하는 프로그램으로 간주합니다.

위 링크를 클릭하거나 Typst Playground를 방문해 보세요.

## Python 사용자를 위한 Typst 한눈에 보기

Python 사용자가 Typst를 접할 때 중요한 세 가지 주요 계층이 있습니다.

일반 텍스트용 마크업(`markup`)이 있습니다. 이 부분은 Markdown과 유사하게 느껴집니다. 제목은 `=`으로 시작하고, 목록은 `-` 또는 `+`를 사용하며, 강조는 `*` 또는 `_`를 사용합니다. 수학 수식은 달러 기호(`$`) 사이에 위치합니다. 다음으로 `#`으로 작성된 코드 표현식(`code expression`)과 함수(`function`) 및 타입(`type`)의 표준 라이브러리(`standard library`)가 있습니다.

여기 기본적인 스타일을 보여주는 작은 문서가 있습니다.

```typst
#set page(paper: "a4")
#set text(lang: "en")

= Sample report

This report comes from a Typst template.

We can place inline math like $a^2 + b^2 = c^2$ in a sentence.

Now a short list:

- First item
- Second item
- Third item
```

전체 파일은 깔끔한 PDF로 컴파일(`compile`)됩니다. 추가 서문(`preamble`)이나 별도의 스타일 파일(`style file`)은 필요 없습니다. `#set` 호출은 스크립트 상단의 설정 코드처럼 작동합니다. 용지 크기, 언어, 텍스트 설정을 지정한 다음 내용을 작성할 수 있습니다.

`#` 표현식(`expression`)은 설정보다 훨씬 많은 것을 할 수 있습니다. Typst는 숫자, 문자열(`string`), 배열(`array`), 딕셔너리(`dictionary`), 함수(`function`), 조건문(`condition`), 반복문(`loop`)을 제공합니다. 이러한 도구들은 이 언어가 LaTeX 매크로(`macro`) 묶음이라기보다는 Python과 긴밀한 사촌 관계처럼 느껴지게 합니다.

## 작은 언어, 익숙한 태도

몇 개의 숫자와 표를 보여주는 간단한 보고서를 생각해 봅시다. Typst에서는 다음과 같이 작성할 수 있습니다.

```typst
#let scores = (alice: 92, bob: 81, chas: 95)

= Score report

We have #scores.len() students in this group.

#table(
  columns: 2,
  [*Name*], [*Score*],
  ..scores.keys().map(name => [#name]),
  ..scores.values().map(score => [#score]),
)
```

이 코드를 Python의 관점에서 살펴보세요.

*   `scores`는 문자열 키(`string key`)와 숫자 값(`numeric value`)을 가진 딕셔너리(`dictionary`)입니다.
*   `.len()`은 Python의 `len(scores)`와 유사합니다.
*   `.keys()`와 `.values()`는 반복 가능한 뷰(`view`)를 제공합니다.
*   `map`은 함수(`function`)를 받아 배열(`array`)의 각 요소에 적용합니다.
*   배열 앞에 붙는 `..`은 배열의 요소를 `table` 함수에 대한 인자 목록(`argument list`)으로 펼쳐줍니다.

그 결과는 이름과 점수를 위한 두 개의 열(`column`)로 구성된 표입니다. 외부 템플릿(`template`)도, 다른 언어의 반복문(`loop`)도 필요 없습니다. 데이터와 레이아웃(`layout`)이 나란히 존재합니다.

자신만의 도우미 함수(`helper function`)를 정의할 수도 있습니다.

```typst
#let grade(score) = if score >= 90 { "A" } else if score >= 80 { "B" } else { "C" }

= Graded report

#table(
  columns: 3,
  [*Name*], [*Score*], [*Grade*],
  ..scores.keys().zip(scores.values()).map(((name, score)) => (
    [#name], [#score], [#grade(score)]
  )),
)
```

여기서 `grade`는 작고 순수한 함수(`pure function`)처럼 작동합니다. 이를 표 안에 호출하면, 문서는 PDF를 생성하는 작고 읽기 쉬운 프로그램이 됩니다.

## 명령줄을 통해 Typst를 Python과 함께 사용하기

Python과 Typst를 연결하는 가장 간단한 방법은 Typst 명령줄 도구(`command line tool`)를 이용하는 것입니다. `.typ` 템플릿(`template`)을 작성한 다음, Python에서 `subprocess.run`을 사용하여 `typst` 바이너리(`binary`)를 호출할 수 있습니다.

다음 `report.typ` 파일을 상상해 보세요.

```typst
#import "sys.typ": sys

#let name = sys.inputs.name
#let score = int(sys.inputs.score)

= Report for #name

Your score is #score.

If this report looks correct, keep this PDF.
```

`sys.inputs` 객체는 외부에서 전달된 값들의 딕셔너리(`dictionary`)입니다. 실제 설정에서는 Typst에 문서화된 올바른 시스템 모듈(`system module`)을 임포트(`import`)한 다음, 안전한 접근 메서드(`access method`)로 값을 읽을 것입니다. 핵심 아이디어는 간단합니다. Python에서 키-값 쌍(`key-value pair`)을 전달하고, Typst 내부에서 읽는 것입니다.

Python 측에서는 다음과 같이 작성할 수 있습니다.

```python
import subprocess
from pathlib import Path

def render_report(template: Path, output: Path, name: str, score: int) -> None:
    subprocess.run(
        [
            "typst",
            "compile",
            "--input",
            f"name={name}",
            "--input",
            f"score={score}",
            str(template),
            str(output),
        ],
        check=True,
    )

if __name__ == "__main__":
    template = Path("report.typ")
    output = Path("report.pdf")
    render_report(template, output, name="Chas", score=95)
```

Python 코드는 데이터와 비즈니스 로직(`business logic`)을 담당합니다. Typst는 레이아웃(`layout`)과 문서 구조를 처리합니다. 두 부분의 경계는 `--input` 플래그(`flag`) 세트입니다. 값들은 문자열(`string`)로 경계를 넘어가고, Typst 템플릿(`template`)은 필요에 따라 이를 숫자나 다른 타입으로 변환합니다.

이 방식은 데이터를 수집하는 Python 프로젝트가 이미 있고 반복 가능한 보고서를 원할 때 유용합니다. Python에서 테스트(`test`)로 로직(`logic`)을 확인한 다음, 예약된 작업(`scheduled job`)이나 웹 워커(`web worker`)에서 동일한 보고서 생성 함수(`reporting function`)를 실행할 수 있습니다.

## Typst 내에서 데이터 가져오기

때로는 문서 자체에서 Python의 도움 없이 간단한 데이터 수집(`data ingestion`)을 처리할 수 있습니다. Typst는 CSV 및 기타 형식을 읽을 수 있습니다. 이 기능은 복잡한 작업이 업스트림(`upstream`)에서 발생하고 표시를 위한 가벼운 변환만 필요할 때 유용합니다.

Typst에서 CSV 데이터를 사용하는 한 가지 방법은 다음과 같습니다.

```typst
#let rows = csv("sales.csv", has-header: true)

= Sales report

#table(
  columns: 3,
  [*Product*], [*Units*], [*Revenue*],
  ..rows.map(row => (
    [#row.name],
    [#row.units],
    [$#row.revenue$],
  )),
)
```

데이터베이스 쿼리(`database query`)에서 `sales.csv` 파일을 작성하는 Python 스크립트를 상상할 수 있습니다. 스크립트가 완료되면 Typst 템플릿(`template`)이 파일을 읽고 표를 생성합니다. 파이프라인(`pipeline`)은 명확하게 유지됩니다.

*   Python은 데이터베이스(`database`)에 접근하여 `sales.csv`를 작성합니다.
*   Typst는 해당 파일을 읽고 보고서를 레이아웃(`layout`)하며 PDF로 컴파일(`compile`)합니다.
*   래퍼 스크립트(`wrapper script`)가 이러한 단계를 하나로 연결합니다.

더 복잡한 요약의 경우, 데이터베이스 계층(`database layer`)에서 뷰(`view`) 또는 구체화된 테이블(`materialized table`)을 작성하여 CSV 내보내기(`export`)를 간단하고 안정적으로 유지할 수 있습니다.

## Python에 로직을 유지해야 할 때

모든 유능한 문서 언어의 위험은 과용입니다. Typst가 반복(`loop`), 필터링(`filter`), 매핑(`map`)을 할 수 있다는 것을 알게 되면, `.typ` 파일 내부에 복잡한 비즈니스 로직(`business logic`)을 구현하려고 시도할 수도 있습니다. 이는 단기적으로는 작동할 수 있지만, 요구 사항이 변경됨에 따라 유지 관리하기 어려워질 수 있습니다.

Python과 Typst를 위한 좋은 경험 법칙(`rule of thumb`)은 다음과 같습니다.

*   데이터 접근은 Python에 유지하세요. 쿼리(`query`), API 호출(`API call`), 복잡한 변환(`transform`)은 해당 계층에 속합니다.
*   Typst는 레이아웃 로직(`layout logic`), 가벼운 서식(`formatting`), 간단한 계산 값에 사용하세요.
*   구조화된 데이터(`structured data`)는 Python에서 Typst로 입력(`input`) 또는 파일(`file`) 형태로 전달하세요.
*   코드보다 내용에 더 관심을 갖는 사람들이 읽기 쉽도록 Typst 템플릿(`template`)을 유지하세요.

이러한 분리는 Python 코드를 테스트 가능(`testable`)하게 유지합니다. 또한 템플릿(`template`)이 전체 프로그램이라기보다는 일부 도우미 함수(`helper function`)가 포함된 확장된 Markdown처럼 읽히기 때문에 관리가 용이합니다.

## 더 넓은 Python 툴체인에서의 Typst

Python에 오케스트레이션(`orchestration`)을 의존하게 되면, Typst는 파이프라인(`pipeline`)에 잘 맞는 또 다른 도구가 됩니다. 이를 웹 앱(`web app`), 배치 작업(`batch job`), 또는 과학적 워크플로우(`scientific workflow`)에 통합할 수 있습니다.

이러한 흐름의 예시는 다음과 같습니다.

*   FastAPI 백엔드(`backend`)는 인보이스(`invoice`)를 렌더링(`render`)합니다. Python은 품목(`line item`)과 세금 데이터를 수집한 다음 Typst를 호출하여 사용자가 다운로드할 수 있는 PDF를 생성합니다.
*   Jupyter Notebook의 연구 스크립트(`research script`)는 실험 측정 항목(`experiment metrics`)을 CSV로 작성한 다음, Typst 컴파일(`compile`)을 시작하여 논문 초안용으로 다듬어진 보고서를 생성합니다.
*   매일 실행되는 크론 작업(`cron job`)은 측정 항목을 수집하고, JSON 또는 CSV를 디렉터리에 덤프(`dump`)하며, 각 클라이언트(`client`)에 대해 Typst를 호출하고, PDF를 객체 스토리지(`object storage`)에 저장합니다.

각 경우에서 Python은 비즈니스 규칙(`business rule`)을 담당하며, Typst는 문서를 선명하고 일관성 있게 유지합니다.

## Python 개발자

Python 배경을 가진 사람에게 Typst는 LaTeX에 비해 몇 가지 분명한 장점이 있습니다.

*   구문(`syntax`)은 노이즈(`noise`)를 낮게 유지합니다. 일반 단락은 템플릿 언어(`template language`)가 아닌 일반적인 글처럼 보입니다. 코드 조각들은 명시적(`explicit`)입니다. `#` 접두사 때문에 표현식 모드(`expression mode`)에 진입했음을 알 수 있습니다.
*   언어 모델(`language model`)은 Python에서 온 여러분의 정신 모델(`mental model`)과 일치합니다. 값(`value`), 함수(`function`), 컨테이너(`container`), 예측 가능한 동작(`predictable behavior`)이 있습니다. Typst 함수를 읽고, 입력(`input`)과 출력(`output`)을 이해하며, 호출한 곳에서 실행될 것이라고 신뢰할 수 있습니다.
*   이 도구는 마찰 없이 스크립트(`script`)에 통합됩니다. Python에서 바이너리(`binary`)를 호출하고, 몇 가지 인자(`argument`)를 전달하면 PDF를 얻을 수 있습니다. 또는 템플릿(`template`)과 몇 가지 입력(`input`)을 받는 컴파일 함수(`compile function`)를 제공하는 래퍼 라이브러리(`wrapper library`)와 함께 작업할 수도 있습니다. 이러한 경험은 린터(`linter`)나 테스트 러너(`test runner`)를 호출하는 것과 유사하게 느껴집니다.

문서를 코드베이스(`codebase`)의 일부로 취급하고 싶다면, Typst는 LaTeX 구문(`syntax`)과 씨름할 필요 없이 그렇게 할 수 있는 방법을 제공합니다. Python 개발자에게 이는 형식(`format`)과 씨름하는 시간을 줄이고, 문서가 전달해야 할 스토리를 구성하는 데 더 많은 시간을 할애할 수 있다는 의미입니다.
