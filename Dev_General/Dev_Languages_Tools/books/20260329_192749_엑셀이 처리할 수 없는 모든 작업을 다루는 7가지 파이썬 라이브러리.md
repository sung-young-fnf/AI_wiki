Enter 키를 누르거나 클릭하여 이미지를 전체 크기로 보세요.

여러분은 Ctrl-C, Ctrl-V, "셀 서식"을 외울 때까지 `pandas`와 엑셀을 사용해왔을 것입니다. 하지만 데이터의 양이 늘어나거나, 로직이 복잡해지거나, 성능이 저하되면 스프레드시트는 한계를 드러냅니다.

이 7가지 라이브러리는 파이썬이 한 단계 도약하도록 돕습니다. 이들은 엑셀이 확장하기 어려운 분석, 차원 모델링 (dimension modeling), 사용자 정의 수식, 대규모 데이터셋, 실시간 UI (live UI) 등을 처리할 수 있습니다. 저는 "엑셀로는 더 이상 안 될 때" 이 라이브러리들을 사용합니다.

### 1. `vaex` — RAM 부족 없이 수십억 행의 데이터프레임 (DataFrame) 처리

**이유:** `pandas`가 "MemoryError"로 실패하는 데이터셋의 경우, `vaex`는 메모리 매핑 (memory mapping)을 사용하여 데이터를 느긋하게(lazily), 그리고 빠르게 필터링, 집계, 시각화할 수 있도록 합니다.

```python
import vaex

df = vaex.open('huge_dataset.hdf5')
df = df[df.x > 0]
agg = df.groupby('category', agg={'mean_y': vaex.agg.mean('y')})
print(agg)
```

저는 12억 행의 로그 데이터셋에 `vaex`를 실행해봤습니다. 메모리 부족 현상도, 충돌도 없었습니다. 오직 결과만 있었습니다.

### 2. `formulaic` — 파이썬 환경에서 엑셀 스타일 수식 활용

**이유:** `IF`, `SUMPRODUCT`와 같은 엑셀 수식의 표현력을 파이썬 환경에서, 데이터프레임 (DataFrame) 컨텍스트 및 타입 안정성 (type safety)과 함께 사용하고 싶을 때 유용합니다.

```python
from formulaic import Formula

formula = Formula("revenue ~ price * quantity + discount")
# You can bind it to a DataFrame-like object and evaluate.
```

이 라이브러리는 통계 모델링 (statistical modeling) 및 사용자 정의 DSL (Domain Specific Language) 엔진에 사용됩니다. 엑셀의 수식 엔진이 이제 여러분의 도구가 되는 셈입니다.

### 3. `PyDAX` — Power BI / DAX 로직을 파이썬으로 가져오기

**이유:** DAX (필터 컨텍스트 (filter context), `CALCULATE`, 시간 인텔리전스 (time intelligence))의 강력한 기능을 마이크로소프트 (Microsoft) 도구에 종속되지 않고 파이썬 내에서 활용해야 할 때 유용합니다.

```python
from pydax import evaluate, Context

ctx = Context(table={'sales': [{'date': '2023-10-01', 'amount': 100}, {'date': '2023-10-02', 'amount': 150}]})
expr = "SUMX(sales, sales[amount])"
res = evaluate(expr, ctx)
print(res)  # 250
```

DAX 스타일 로직을 스크립트 내에서 — 차원 높은 데이터 모델링 (data modeling)을 GUI (Graphical User Interface) 없이 구현할 수 있습니다.

### 4. `Panels` + 패널 데이터 (Panel Data) (`panel` + `hvPlot`을 통해) — 데이터프레임 (DataFrame)으로 인터랙티브 대시보드 만들기

**이유:** 엑셀의 피벗테이블 (PivotTable)과 슬라이서 (slicers)는 이제 구식 도구입니다. `panel`과 `hvplot`을 사용하면 코드가 노트북 (notebook)이나 웹에서 즉시 새로 고쳐지는 필터링 가능한 대시보드 (dashboard)로 변모합니다.

```python
import panel as pn
import hvplot.pandas
import pandas as pd

df = pd.DataFrame({'x': range(50), 'y': [i**2 for i in range(50)]})
plot = df.hvplot.line('x', 'y')
pn.Row(plot, df.hvplot.table()).servable()
```

HTML이나 자바스크립트 (JS)를 작성할 필요 없이 탐색을 위한 UI (User Interface)를 얻을 수 있습니다.

### 5. `PyJanitor` + `conditional_eval` — 스프레드시트 규칙처럼 변환을 표현하기

**이유:** 엑셀의 "만약 셀 > 임계값이라면 이 작업을 수행하라"는 식의 로직이 변환을 직접 연결하면 가독성 높은 코드로 바뀝니다.

```python
from janitor import clean_names, conditional_eval
import pandas as pd

df = pd.DataFrame({'Revenue': [100, 200, 0], 'Cost': [30, 80, 10]})
df = (
    df.clean_names()
      .conditional_eval(
          "profit_margin = (revenue - cost) / revenue if revenue > 0 else 0"
      )
)
print(df)
```

중첩된 람다 (lambda)나 행별 반복문 없이 — 파이프라인 (pipeline) 내에서 스프레드시트 로직을 구현할 수 있습니다.

### 6. `Pint` + `xarray` — 단위 인식 (Units-Aware), 물리적 의미를 가진 다차원 데이터 처리

**이유:** 스프레드시트는 단위를 처리하거나 다중 인덱스 공간/시간 그리드를 다룰 수 없습니다. `xarray` + `pint`는 단위와 축을 인식하는 배열을 제공합니다.

```python
import xarray as xr
import pint
from pint import UnitRegistry

ureg = UnitRegistry()
da = xr.DataArray([1, 2, 3], dims=['time'], attrs={'units': 'meter'})
da = da * 2 * ureg.meter
print(da)
```

과학 데이터 (scientific data), 기상 격자 (weather grids), 신호 처리 등에 사용하세요. 엑셀은 이 라이브러리에 비할 바가 못 됩니다.

### 7. `OpenRefine-Python` (`refineclient`) — 조정 API (Reconcile API) 패턴으로 지저분한 데이터 정리

**이유:** 엑셀을 이용한 데이터 정리는 수동적입니다. `refineclient`는 `OpenRefine`에 연결하여 파이썬에서 변환 (패싯 (facets), 클러스터링 (clustering), 대량 편집 (mass edits))을 스크립트로 작성할 수 있도록 합니다.

```python
from refineclient import refine
client = refine.RefineClient("http://localhost:3333")
project = client.new_project("dirty.csv")
project.apply_operations([{"op": "cluster", "on": "Name", "mode": "levenshtein"}])
project.export("cleaned.csv")
```

지저분한 항목은 클러스터를 통해 해결됩니다. GUI 클릭 작업을 직접 할 필요가 없습니다. 엑셀의 "데이터 → 정리" 기능 중 최고를 코드로 자동화한 것입니다.
