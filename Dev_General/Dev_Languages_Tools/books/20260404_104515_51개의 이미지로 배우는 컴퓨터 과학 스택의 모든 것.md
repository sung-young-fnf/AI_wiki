# 51개의 이미지로 배우는 컴퓨터 과학 스택의 모든 것

컴퓨터 과학은 종종 서로 연결되지 않은 파편화된 지식의 집합체처럼 교육되곤 하지만, 실제로는 차곡차곡 쌓아 올려진 아이디어의 스택(Stack)과 같습니다. 

이 글에서는 간단한 도식을 통해 컴퓨터 과학의 전체 스택을 시각적으로 살펴보고, 커리어 내내 도움이 될 정신적 모델(Mental models)을 구축하는 여정을 떠나보겠습니다.

---

### 1. 논리 게이트 (Logic Gates)

모든 컴퓨터의 가장 밑단에는 '켜짐(on)'과 '꺼짐(off)'이라는 두 가지 상태를 가진 매우 단순한 스위치가 존재합니다. 컴퓨터에서 **트랜지스터(Transistor)**는 전류의 흐름에 따라 초당 수십억 번 켜고 끌 수 있는 미세한 전자 스위치 역할을 합니다.

이 두 상태는 이진 정보를 나타내며, '켜짐'은 이진수 1, '꺼짐'은 이진수 0을 의미합니다. 여러 개의 트랜지스터가 결합하여 **논리 게이트(Logic gates)**를 형성합니다. 이는 이진 입력에 대해 기본적인 불 연산(Boolean operations)을 수행하여 하나의 이진 출력을 생성하는 회로입니다.

**7가지 기본 논리 게이트:**
*   **AND 게이트:** 모든 입력이 1일 때만 1을 출력
*   **OR 게이트:** 입력 중 하나라도 1이면 1을 출력
*   **NOT 게이트:** 입력 신호를 반전 (0은 1로, 1은 0으로)
*   **NAND (NOT-AND) 게이트:** 모든 입력이 1일 때만 0을 출력 (AND의 반대)
*   **NOR (NOT-OR) 게이트:** 모든 입력이 0일 때만 1을 출력 (OR의 반대)
*   **XOR (Exclusive OR) 게이트:** 입력값이 서로 다를 때 1을 출력
*   **XNOR (Exclusive NOR) 게이트:** 입력값이 같을 때 1을 출력

이와 함께 신호 증폭이나 격리를 위해 입력을 그대로 출력하는 **버퍼(Buffer) 게이트**가 사용됩니다. 논리 게이트들은 다시 결합하여 하나의 칩인 **집적 회로(Integrated Circuit, IC)**를 구성하며 더 복잡한 기능을 수행합니다.

특히 **범용 논리 게이트(Universal logic gate)**인 NAND와 NOR 게이트만 있으면 다른 어떤 논리 게이트도 구축할 수 있습니다. 즉, 아무리 복잡한 컴퓨터라도 이 게이트들만으로 만들 수 있다는 뜻입니다. (예: 아폴로 달 탐사선의 컴퓨터 AGC는 거의 대부분 3-입력 NOR 게이트 IC로 제작되었습니다.)

### 2. 중앙 처리 장치 (CPU)

컴퓨터의 **중앙 처리 장치(Central Processing Unit, CPU)**는 논리 게이트를 구현한 IC들을 결합하여 만들어집니다. CPU는 다음과 같이 구성됩니다.

*   **산술 논리 장치(Arithmetic Logic Unit, ALU):** 산술 연산(덧셈, 뺄셈 등)과 논리 비교(같음, 큼, 작음 등)를 수행합니다.
*   **레지스터(Registers):** CPU 내부에 직접 내장된 초고속 메모리 구성 요소로, 계산 중간 데이터와 다음에 실행할 명령어를 추적합니다.
*   **제어 장치(Control Unit, CU):** 오케스트레이터 역할을 하며 메모리에서 명령어를 가져오고(Fetch), 해석(Decode)하며, ALU, 레지스터, 외부 메모리 간의 데이터 흐름을 조정합니다.

CPU는 이러한 구성 요소를 사용하여 초당 수십억 개의 명령어를 실행하며 컴퓨터의 두뇌 역할을 수행합니다.

### 3. 튜링 머신 (Turing Machine)

하드웨어 위 단계로 넘어가기 전, 이론적 계산의 의미를 이해해야 합니다. 1936년 앨런 튜링은 다음과 같이 구성된 가상의 장치인 **튜링 머신(Turing Machine)**을 설명했습니다.

*   기호가 담긴 셀로 나뉜 무한한 테이프
*   테이프를 따라 이동하는 읽기/쓰기 헤드
*   현재 상태와 읽은 기호에 따라 기계의 동작을 결정하는 규칙 세트

튜링 머신 개념은 아무리 복잡한 문제라도 논리적 연산의 순서로 해결할 수 있음을 알려줍니다. **처치-튜링 명제(Church-Turing thesis)**에 따르면, 계산 가능한 모든 문제는 튜링 머신으로 계산할 수 있으며, 확장하면 충분한 시간과 메모리가 있는 모든 범용 컴퓨터로 해결할 수 있습니다. 현대의 모든 컴퓨터는 본질적으로 튜링 머신의 실용적이고 유한한 버전입니다.

### 4. 프로그래밍 언어, 컴파일러 및 인터프리터

이진 정보를 직접 다루는 것은 어렵기 때문에 프로그래밍 언어라는 추가적인 **추상화(Abstractions)** 층이 구축되었습니다. 이 언어들은 **컴파일러(Compiler)**와 **인터프리터(Interpreter)**라는 프로그램을 통해 CPU가 실행할 수 있는 이진 기계어 명령으로 번역됩니다.

*   **컴파일러:** 전체 소스 코드를 한꺼번에 이진 실행 파일로 번역합니다. (예: C, C++, Rust, Go) 실행 속도가 빠르지만, 플랫폼별로 다시 컴파일해야 하며 코드가 바뀔 때마다 재컴파일이 필요합니다.
*   **인터프리터:** 소스 코드를 읽고 즉석에서 해석하여 명령을 실행합니다. (예: Python, JavaScript, Ruby) 별도의 실행 파일을 만들지 않아 즉시 실행 가능하고 이식성이 좋지만, 실행 중에 번역이 일어나므로 컴파일 언어보다 느린 편입니다.

---

### 5. 자료구조 (Data Structures)

자료구조는 데이터를 효율적으로 사용할 수 있도록 조직화하는 방법입니다.

1.  **배열(Array):** 데이터를 연속된 메모리 공간에 저장합니다. 인덱스를 통한 접근은 빠르지만, 크기 변경이나 중간 삽입은 비용이 많이 듭니다.
2.  **연결 리스트(Linked List):** 포인터로 연결된 노드로 데이터를 저장합니다. 삽입과 삭제가 효율적이지만 데이터 접근 속도는 느립니다.
3.  **스택(Stack):** 후입선출(LIFO, Last-In First-Out) 원칙을 따릅니다.
4.  **큐(Queue):** 선입선출(FIFO, First-In First-Out) 원칙을 따릅니다.
5.  **해시 테이블(Hash Table):** 해시 함수를 사용하여 키-값(Key-Value) 쌍을 저장하며, 검색, 삽입, 삭제가 거의 상수 시간(Constant-time)에 이루어집니다.
6.  **그래프(Graph):** 노드와 간선으로 데이터 요소 간의 관계와 네트워크를 모델링합니다.
7.  **트리(Tree):** 하나의 루트 노드에서 부모-자식 관계를 가지는 계층적 구조의 그래프입니다.
8.  **힙(Heap):** 최솟값(Min Heap) 또는 최댓값(Max Heap)이 루트에 위치하도록 보장하는 완전 이진 트리입니다.

### 6. 알고리즘 (Algorithms)

알고리즘은 문제를 해결하기 위한 단계별 절차입니다. 좋은 알고리즘의 요건은 다음과 같습니다.
*   **정확성(Correctness):** 모든 유효한 입력에 대해 올바른 출력을 내야 함.
*   **효율성(Efficiency):** 시간과 메모리를 현명하게 사용해야 함 (**빅오 표기법**으로 측정).
*   **명확성(Clarity):** 이해, 디버깅, 유지보수가 쉬워야 함.
*   **종료성(Termination):** 반드시 유한한 단계 내에 끝나야 함.

주요 알고리즘으로는 **재귀(Recursion)**, **정렬 알고리즘(Sorting Algorithms)**, 그리고 정렬된 배열에서 탐색 범위를 반씩 줄여가는 **이진 탐색(Binary Search)** 등이 있습니다.

---

### 7. 빅오 표기법 (Big-O Notation)

알고리즘의 성능이 입력 크기에 따라 어떻게 확장되는지 설명하기 위해 사용됩니다.

*   **시간 복잡도(Time Complexity):** 입력 크기에 따른 실행 시간의 증가율.
*   **공간 복잡도(Space Complexity):** 입력 크기에 따른 메모리 사용량의 증가율.

**일반적인 시간 복잡도:**
*   O(1) (상수): 배열 인덱스 접근
*   O(log n) (로그): 이진 탐색
*   O(n) (선형): 선형 탐색
*   O(n log n) (선형 로그): 병합 정렬, 힙 정렬
*   O(n²) (이차): 버블 정렬
*   O(2ⁿ) (지수): 재귀를 이용한 피보나치 수열
*   O(n!) (팩토리얼): 외판원 문제(Traveling Salesman Problem)의 브루트 포스 해법

### 8. 프로그래밍 패러다임 (Programming Paradigms)

1.  **명령형 프로그래밍(Imperative):** 문제를 해결하는 **방법(How)**에 집중하며 단계별 지시사항을 명시합니다. (예: 절차적 프로그래밍, 객체 지향 프로그래밍)
    *   **객체 지향 프로그래밍(OOP)의 4대 요소:** 캡슐화(Encapsulation), 추상화(Abstraction), 상속(Inheritance), 다형성(Polymorphism).
2.  **선언형 프로그래밍(Declarative):** **무엇(What)**을 원하는지에 집중하며 시스템이 단계를 결정하게 합니다. (예: 함수형 프로그래밍)
    *   **함수형 프로그래밍의 원칙:** 순수 함수(Pure functions), 불변 데이터(Immutable data), 일급 함수(First-class functions).

---

### 9 & 10. 소프트웨어 디자인 패턴과 SOLID 원칙

확장 가능하고 유지보수가 쉬운 코드를 작성하기 위한 지침입니다.

*   **디자인 패턴:** 생성(Creational), 구조(Structural), 행위(Behavioral) 패턴으로 나뉘며 반복되는 문제에 대한 템플릿을 제공합니다.
*   **SOLID 원칙:**
    1.  **단일 책임 원칙 (SRP):** 클래스는 하나의 일만 해야 함.
    2.  **개방/폐쇄 원칙 (OCP):** 확장은 열려 있고 수정은 닫혀 있어야 함.
    3.  **리스코프 치환 원칙 (LSP):** 자식 클래스는 부모 클래스를 대체할 수 있어야 함.
    4.  **인터페이스 분리 원칙 (ISP):** 큰 인터페이스 하나보다 작은 인터페이스 여러 개가 나음.
    5.  **의존 역전 원칙 (DIP):** 구체적인 구현이 아닌 추상화에 의존해야 함.

---

### 11. 데이터베이스 (Databases)

*   **관계형/SQL 데이터베이스:** 미리 정의된 스키마를 가진 테이블 구조입니다. (예: PostgreSQL, MySQL). **ACID**(원자성, 일관성, 고립성, 지속성) 트랜잭션을 통해 데이터 무결성을 유지합니다.
*   **비관계형/NoSQL 데이터베이스:** 유연한 스키마를 사용합니다. (예: MongoDB, Redis). **BASE**(기본적 가용성, 소프트 상태, 결과적 일관성)를 통해 성능과 확장성을 확보합니다.

---

### 12. 네트워킹 (Networking)

컴퓨터 간의 효율적인 통신을 위한 표준 모델입니다.
*   **OSI 7계층:** 물리, 데이터 링크, 네트워크, 전송, 세션, 표현, 응용 계층.
*   **TCP/IP 4계층:** 네트워크 액세스, 인터넷, 전송, 응용 계층으로 OSI를 간소화한 실용적 모델입니다.

---

### 13. 멀티태스킹: 동시성과 병렬성

*   **동시성(Concurrency):** 여러 작업을 동시에 관리하는 것 (싱글 코어에서 문맥 전환(Context switching)을 통해 번갈아 실행).
*   **병렬성(Parallelism):** 여러 작업을 멀티 코어에서 실제로 동시에 실행하는 것.
*   **동기화 도구:** 공유 데이터에 대한 충돌을 방지하기 위해 **뮤텍스(Mutex, 상호 배제 잠금)**와 **세마포어(Semaphore)**를 사용합니다.

---

### 14. 사이버 보안 (Cybersecurity)

*   **인증(Authentication):** 신원 확인.
*   **인가(Authorization):** 권한 확인.
*   **암호화(Encryption):** 정보를 암호로 인코딩.
    *   **대칭키 암호화:** 암복호화에 같은 키 사용 (예: AES).
    *   **비대칭키(공개키) 암호화:** 공개키로 암호화하고 개인키로 복호화 (예: RSA).

---

### 15. 신기술 (Emerging Technologies)

1.  **인공지능(AI):** 명시적 프로그래밍 없이 데이터로부터 학습. 지도 학습, 비지도 학습, 강화 학습으로 분류됩니다.
2.  **블록체인(Blockchain):** 탈중앙화, 불변성, 투명성, 보안성을 갖춘 분산 장부 기술입니다.
3.  **양자 컴퓨팅(Quantum Computing):** 비트 대신 **큐비트(Qubits)**를 사용합니다. **중첩(Superposition)**과 **얽힘(Entanglement)**이라는 양자역학적 속성을 이용하여 특정 최적화 문제나 암호 해독 등을 기하급수적으로 빠르게 처리할 수 있습니다.

---

트랜지스터의 온/오프 상태에서 시작하여 양자 중첩에 이르기까지, 우리는 각 층이 아래 층을 기반으로 구축되는 컴퓨터 과학의 전체 스택을 살펴보았습니다. 이러한 기반을 이해하면 컴퓨터를 우아한 추상화의 계층 구조로 바라볼 수 있게 되며, 더 나은 소프트웨어 엔지니어로 거듭날 수 있을 것입니다.
