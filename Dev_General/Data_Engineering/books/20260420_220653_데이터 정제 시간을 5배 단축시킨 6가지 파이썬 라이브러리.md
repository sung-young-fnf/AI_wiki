# 데이터 정제 시간을 5배 단축시킨 6가지 파이썬 라이브러리

깨끗한 데이터는 깨끗한 인사이트로 이어집니다.

데이터셋을 보며 "이건 데이터가 아니야. CSV 옷을 입은 혼돈이야."라고 생각해 본 적이 있으신가요?

우리 모두 그런 경험이 있을 겁니다. 웹 스크래핑을 하든, 미심쩍은 API에서 데이터를 가져오든, 전임자의 스프레드시트 악몽을 물려받든, 지저분한 데이터는 항상 존재합니다.

하지만 수년간 데이터 전처리(data munging)에 깊이 파묻혀 지내면서, 저는 잘 알려지지 않았지만 놀랍도록 유용한 파이썬 라이브러리들을 발견했습니다. 이 라이브러리들은 저의 데이터 클리닝 워크플로(workflow)를 '삶의 Ctrl+Z'에서 '수월한 파이프라인(pipeline)의 행복'으로 바꿔주었습니다.

여기 저의 데이터 정제 시간을 최소 5배 단축시켜준 파이썬 라이브러리 6가지가 있습니다.
그리고 `pandas`는 포함하지 않습니다. `pandas`는 이미 다들 아시잖아요. 이제 다음 단계로 나아가 봅시다.

## 1. datacompy: 자동 데이터프레임(DataFrame) 비교 (더 이상의 수동 확인은 없다)

두 엑셀 파일을 행별로 일일이 비교해 본 적이 있으신가요?
네. 저도요. 다시는 안 할 겁니다.

```python
import datacompy
import pandas as pd

df1 = pd.read_csv('data_old.csv')
df2 = pd.read_csv('data_new.csv')

compare = datacompy.Compare(df1, df2, join_columns='id')
print(compare.report())
```

**왜 혁신적인가:** 불일치를 감지하기 위해 커스텀 스크립트를 작성하는 일은 잊어버리세요. `datacompy`는 어떤 행이 다르고, 어떤 필드가 변경되었는지 정확히 알려주며, 사람이 읽기 쉬운 보고서까지 출력해 줍니다. 데이터프레임(DataFrame)을 위한 `diff` 명령이라고 생각하면 되는데, 훨씬 강력합니다.

**전문가 팁:** 다중 열 조인(multi-column joins)과 퍼지 매칭(fuzzy matching)을 지원합니다. 이것을 버전 관리되는 데이터셋과 함께 사용하면 누구도 막을 수 없습니다.

## 2. pyjanitor: pandas에 MBA 학위를 더하다

만약 `pandas`가 전동 공구라면, `pyjanitor`는 개인 비서가 딸린 전동 공구입니다. 코드가 마치 이야기처럼 읽히도록 클리닝 단계를 연결할 수 있게 해줍니다.

```python
import pandas as pd
import janitor

df = (
    pd.read_csv('messy_data.csv')
    .clean_names()
    .remove_empty()
    .drop_duplicates(subset='email')
    .filter_on('age >= 18')
)
```

**왜 강력한가:** 보기 좋고, 연결 가능하며, 스파게티 코드처럼 보이지 않는 프로덕션 품질의 클리닝 코드를 얻을 수 있습니다. 게다가 `pandas`를 확장하여 도메인별 클리닝 도구 모음을 제공합니다.

**실제 사례:** 저는 `pyjanitor`를 사용하여 5백만 행의 고객 지원 데이터셋을 15줄 미만으로 정제했습니다. 일반 `pandas`로는 3배 이상의 시간이 걸렸을 겁니다.

## 3. dirty_cat: 모든 범주형 데이터가 똑같이 생성되는 것은 아니기에

제품 이름, 직책 또는 무작위 텍스트 항목과 같은 고유값 수가 많은 범주형 변수(high-cardinality categorical variables)를 다루고 있다면, 하던 일을 멈추고 이것을 설치하세요.

```bash
pip install dirty_cat
```
```python
from dirty_cat import SimilarityEncoder
import pandas as pd

X = pd.DataFrame({'job_title': ['data engineer', 'data scientist', 'ML engineer']})
encoder = SimilarityEncoder()
X_encoded = encoder.fit_transform(X[['job_title']])
```

**왜 천재적인가:** 전통적인 원-핫 인코딩(one-hot encoding)은 지저분한 문자열에는 약합니다. `dirty_cat`은 문자열 유사성을 사용하여 카테고리를 클러스터링하고 인코딩합니다. 이는 더 적은 차원(dimensions)과 머신러닝(ML) 모델을 위한 더 나은 신호를 의미합니다.

**재미있는 사실:** `scikit-learn` 팀이 개발했습니다. 정말 믿을 수 있습니다.

## 4. fuzzywuzzy: 사람처럼 오타를 정제하다

솔직히 말해서, 인간은 철자법에 약합니다. "NYC" ≠ "New York" ≠ "N.Y." 이지만 모두 같은 것을 의미합니다.

이런 상황에서 `fuzzywuzzy`가 해결책이 되어줍니다.

```python
from fuzzywuzzy import fuzz
from fuzzywuzzy import process

choices = ["New York", "Los Angeles", "Chicago"]
print(process.extract("N.Y.", choices, limit=2))
```

**왜 필수적인가:** 열을 병합하거나, 문자열 중복을 제거하거나, 일관성 없는 항목을 수정하거나, 유사성을 기반으로 열 값을 자동 수정하는 데 사용하십시오. 레벤슈타인 거리(Levenshtein distance)와 비슷하지만, 더 스마트한 점수 매기기를 제공합니다.

**주의 사항:** 대규모 데이터셋의 경우, 더 빠르고 드롭인(drop-in) 호환되는 `RapidFuzz`와 결합하여 사용하세요.

## 5. pandas_schema: 데이터가 당신의 삶을 망치기 전에 검증하라

"급여(salary)"로 모델을 훈련했는데, 값의 절반이 "N/A" 문자열이었다는 사실을 깨달은 적이 있으신가요? 😐 `pandas_schema`는 데이터프레임(DataFrame)에 스키마 규칙(schema rules)을 정의하고 적용할 수 있게 해줍니다.

```python
from pandas_schema import Column, Schema
from pandas_schema.validation import IsDtypeValidation, InRangeValidation

schema = Schema([
    Column('age', [IsDtypeValidation(int), InRangeValidation(0, 120)]),
    Column('email', [IsDtypeValidation(str)])
])

errors = schema.validate(df)
for error in errors:
    print(error)
```

**왜 중요한가:** 데이터를 위한 단위 테스트(unit tests)라고 생각하십시오. 쓰레기 데이터가 모델, 대시보드, 혹은 더 나쁘게는 상사에게 도달하기 전에 잡아낼 수 있습니다.

## 6. missingno: 결측값(Null)을 위한 X-레이

결측값(missing values)을 시각화하는 것은 `df.isnull().sum()`을 20분 동안 스크롤하는 것보다 2초 만에 더 많은 것을 알려줄 수 있습니다.

```python
import missingno as msno
msno.matrix(df)
```

또는 결측값(missing values) 간의 상관관계를 나타내는 히트맵(heatmap)을 얻을 수도 있습니다.

```python
msno.heatmap(df)
```

**왜 이것이 귀중한가:** 가격이 없는 모든 행에 할인 정보도 없는지, 아니면 결측값(null)이 특정 기간에 집중되어 있는지 즉시 파악할 수 있습니다. 이는 아무런 번거로움 없이 실행 가능한 인사이트(insight)를 제공합니다.
