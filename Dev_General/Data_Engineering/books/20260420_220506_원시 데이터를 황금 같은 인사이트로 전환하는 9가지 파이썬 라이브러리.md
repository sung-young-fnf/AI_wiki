# 원시 데이터를 황금 같은 인사이트로 전환하는 9가지 파이썬 라이브러리

나의 스프레드시트가 이야기로 변모했습니다.

이 라이브러리들은 지저분한 스프레드시트, 로그 또는 데이터베이스 덤프를 채용 담당자가 '상용 서비스 준비 완료(production-ready)'라고 부를 만한 결과물로 바꿔주는, 희귀하며 활용도 높은 도구들입니다. 아래 소개된 9가지 파이썬 라이브러리는 노이즈에서 의미 있는 신호(signal)를 추출해야 할 때 제가 실제로 사용하는 것들입니다. 일부는 특정 분야에 특화되어 있고, 일부는 저평가되어 있지만, 모두 제 역할을 톡톡히 해냅니다. 각 라이브러리 항목에는 간결한 설명, 바로 복사하여 붙여넣을 수 있는 작은 코드 조각(snippet), 그리고 즉시 활용할 수 있는 전문가 팁(pro tip)이 포함되어 있습니다.

군더더기 없이, 모든 문장이 핵심을 담고 있습니다. 시작하겠습니다.

### 1) duckdb — SQL 속도, 제로 운영(zero ops)

DuckDB는 인-프로세스(in-process) 분석 SQL 엔진입니다. CSV, Parquet 또는 pandas 데이터를 대상으로 아무것도 구축할 필요 없이 SQL을 실행할 수 있습니다. 즉각적인 분석(ad-hoc analysis)의 경우 대부분의 다른 옵션보다 빠르고 간단합니다.

```python
import duckdb
# query a CSV without loading it fully
res = duckdb.query("SELECT user_id, COUNT(*) AS c FROM 'events.csv' GROUP BY user_id ORDER BY c DESC LIMIT 5").to_df()
print(res)
```

**전문가 팁(Pro tip):** DuckDB를 사용하여 여러 Parquet 파일에 걸친 탐색적 조인(exploratory joins)을 수행하세요. ETL이 필요 없습니다.

### 2) vaex — 대용량 파일을 위한 메모리 맵(memory-mapped) DataFrame

Vaex는 DataFrame과 유사한 API를 제공하지만, 메모리 매핑(memory-mapping)과 지연 평가(lazy evaluation)를 사용하여 RAM에 맞지 않는 데이터셋으로 작업할 수 있습니다. 대규모 데이터에 대한 히스토그램, 그룹화(group-by) 및 벡터화된 변환에 탁월합니다.

```python
import vaex
df = vaex.open("big_dataset.hdf5")   # opens without loading
df = df[df.age > 18]                 # lazy filter
print(df.mean(df.salary).evaluate())
```

**전문가 팁(Pro tip):** 필요할 때까지 구체화(materializing)하는 것을 피하세요. Vaex는 변환을 지연 방식으로 연결할 때 진가를 발휘합니다.

### 3) polars — 표현력 풍부한 API를 갖춘 초고속 DataFrame

Polars는 Rust 기반의 DataFrame 라이브러리로, 표현력이 풍부한 지연 플래너(lazy planner)를 제공합니다. 멀티코어 워크로드에서 pandas보다 뛰어난 성능을 보이는 경우가 많으며, 집계 및 윈도우 함수(window functions)를 위한 훌륭한 API를 가지고 있습니다.

```python
import polars as pl
df = pl.read_parquet("data.parquet")
out = df.lazy().filter(pl.col("score") > 0).groupby("category").agg(pl.col("score").mean()).collect()
print(out)
```

**전문가 팁(Pro tip):** 복잡한 파이프라인에는 지연 API(lazy API)를 사용하세요. Polars가 쿼리 계획을 최적화할 것입니다.

### 4) datatable — 고속 데이터 수집(ingestion) 및 컬럼형 메모리 레이아웃(columnar memory layout) (H2O)

datatable은 C++ 코어를 기반으로 빠른 CSV/TSV 가져오기(import)와 효율적인 숫자 연산을 위해 구축되었습니다. 로드 속도에 병목 현상이 있거나 많은 CSV를 변환해야 한다면, 이 도구는 시간을 절약해 줄 수 있습니다.

```python
import datatable as dt
df = dt.fread("huge.csv")  # fast, multi-threaded
df[:, dt.sum(dt.f.salary), dt.by(dt.f.department)]
```

**전문가 팁(Pro tip):** 초기 데이터 수집(ingestion)에는 datatable을 사용하고, 이후 분석을 위해 주요 테이블을 Polars/PyArrow로 변환하세요.

### 5) featuretools — 자동화된 특징 엔지니어링(feature engineering) (심층)

직접 만든 휴리스틱(heuristics)이 아닌 특징(features)이 필요할 때, Featuretools는 관계형 데이터셋 전반에 걸쳐 집계(aggregations) 및 변환을 자동으로 구축합니다. 특히 테이블 형식 머신러닝(tabular ML)에 유용합니다.

```python
import featuretools as ft
es = ft.EntitySet()
es = es.add_dataframe("users", users_df, index="user_id")
es = es.add_dataframe("orders", orders_df, index="order_id", time_index="order_time")
es = es.add_relationship(ft.Relationship(es["users"]["user_id"], es["orders"]["user_id"]))
features, _ = ft.dfs(entityset=es, target_dataframe_name="users", max_depth=2)
print(features.head())
```

**전문가 팁(Pro tip):** 특징의 기하급수적 증가를 피하려면 `max_depth`를 제한하고, 이후 특징 중요도(feature importance)로 가지치기(prune)하세요.

### 6) woodwork — 테이블 데이터(tabular data)를 위한 의미론적 타이핑(semantic typing) 및 풍부한 dtype

Woodwork는 DataFrame에 의미론적 타이핑(예: 'email', 'zipcode', 'categorical')을 부여하여, 후속 변환, 유효성 검사 및 특징 엔지니어링(feature engineering)을 훨씬 더 의도 기반적이고 자체 문서화가 가능하도록 만듭니다.

```python
import woodwork as ww
import pandas as pd
df = pd.read_csv("raw.csv")
df.ww.init(logical_types={'id': 'Integer', 'signup_date': 'Datetime'})
print(df.ww.logical_types)
```

**전문가 팁(Pro tip):** 자동 특징 생성 시 의미론을 보존하기 위해 woodwork를 featuretools와 결합하여 사용하세요.

### 7) pandera — DataFrame을 위한 런타임 스키마(runtime schema) + 통계적 유효성 검사

Pandera를 사용하면 DataFrame 스키마(유형, 범위, 패턴)를 선언하고, ETL 출력이 기대치와 일치하는지 확인할 수 있습니다. 가볍고 읽기 쉬우며 파이프라인에서 매우 유용합니다.

```python
import pandera as pa
from pandera import Column, DataFrameSchema

schema = DataFrameSchema({
    "id": Column(int, pa.Check.gt(0)),
    "email": Column(str, pa.Check.str_matches(r".+@.+\..+")),
    "score": Column(float, pa.Check.in_range(0, 1))
})
validated = schema.validate(df)  # raises with precise errors
```

**전문가 팁(Pro tip):** 데이터 수집(ingest) 후, ML 전, 데이터 내보내기 전에 스키마 검사를 실행하세요.

### 8) bonobo — 작고 파이써닉한(Pythonic) ETL 파이프라인 라이브러리

Bonobo는 간단한 함수들을 파이프라인으로 구성하는 경량 ETL 프레임워크입니다. 일회성 또는 스트리밍 ETL 작업에 Airflow보다 부담이 적고 매우 읽기 쉽습니다.

```python
import bonobo

def extract():
    yield from open('data.csv')

def transform(line):
    return line.strip().split(',')

def load(row):
    print(row)

graph = bonobo.Graph(extract, transform, load)
bonobo.run(graph)
```

**전문가 팁(Pro tip):** 오케스트레이션(orchestration) 오버헤드 없이 읽기 쉽고 테스트 가능한 ETL이 필요할 때 Bonobo를 사용하세요.

### 9) pyjanitor — 클리닝 작업 연결(chaining cleaning ops), 매우 읽기 쉬움

PyJanitor (janitor라고도 함)는 컬럼 정리, 중복 처리, 재형성(reshaping) 헬퍼 등 상용구(boilerplate) 코드를 제거하는 작은 함수들로 pandas를 확장하여 유창한(fluent) API를 제공합니다.

```python
import janitor
import pandas as pd

df = pd.read_csv("messy.csv").clean_names().drop_duplicate_columns()
df = df.remove_empty()  # chainable helpers
```

**전문가 팁(Pro tip):** ETL 진입점(entry point) 전반에 걸쳐 사용하여 하위 코드(downstream code)가 정돈된 테이블을 볼 수 있도록 하세요.

### 한 가지 작은 팁 (새로운 라이브러리 없이)

알 수 없는 CSV를 수집(ingesting)할 때, `pyarrow`와 Parquet 변환을 표준적인 첫 단계로 사용하세요. 스키마 감지(schema detection)는 한 번만 수행하고, 그 이후에는 모든 곳에서 Parquet 파일을 읽는 방식입니다. 이는 재현 가능한 데이터 파이프라인을 위한 단연 최고의 엔지니어링 패턴입니다.
