## 클로드 플러그인이란 무엇이며 왜 중요한가?

클로드 플러그인을 "AI 두뇌의 팔다리"라고 생각하면 됩니다.

플러그인이 없으면 클로드는 텍스트 상자에 갇혀 있습니다. 생각은 할 수 있지만, 행동할 수는 없습니다.

플러그인을 사용하면 클로드는 다음을 수행할 수 있는 자율 에이전트(autonomous agent)가 됩니다.

✅ GitHub에서 코드를 가져와 PR(Pull Request) 생성
✅ PostgreSQL 데이터베이스(database) 질의
✅ 실시간 웹 검색
✅ 파일 읽기 및 쓰기
✅ 브라우저 제어
✅ Google Workspace 접근

## 기술적 돌파구: MCP(Model Context Protocol)

클로드 플러그인은 앤스로픽이 2025년 12월에 공개한 오픈 표준(open standard)인 MCP(Model Context Protocol)를 기반으로 구축되었습니다.

MCP는 AI 모델이 외부 데이터 및 도구에 연결하는 방식을 표준화하여, 새로운 서비스마다 맞춤형 통합(custom integrations)을 할 필요가 없게 합니다.

앤스로픽의 문서에 따르면:

"MCP는 AI 어시스턴트(AI assistants)가 각 리소스(resource)에 대한 맞춤형 통합 없이 로컬 및 원격 리소스를 검색하고 사용할 수 있도록 합니다."

해석: 하나의 프로토콜(protocol). 수천 가지 도구. 마찰 제로.

## 네 가지 플러그인 유형

1.  **MCP 서버(Servers)** — 클로드를 외부 서비스(GitHub, Slack, 데이터베이스)에 연결
2.  **스킬(Skills)** — 클로드가 필요할 때 자동으로 활성화하는 기능
3.  **명령어(Commands)** — 맞춤형 단축키(예: `/code-review`)
4.  **훅(Hooks)** — 이벤트(events) 발생 시 트리거되는 자동화(예: 코드 표준 강제)

이제 가장 큰 영향력을 가진 10가지 플러그인을 살펴보겠습니다.

## 2026년 클로드 플러그인 베스트 10

### 1. Feature-Dev (89,000회 이상 설치)

**무엇을 하는가:** 기능 요청을 7단계 워크플로(workflow)를 통해 프로덕션 코드(production code)로 전환합니다.

**왜 중요한가:**
대부분의 AI 도구는 코드만 생성합니다. Feature-Dev는 숙련된 엔지니어의 프로세스를 따릅니다.

*   요구사항 수집
*   코드베이스 탐색 (병렬 에이전트가 아키텍처(architecture) 연구)
*   아키텍처 설계
*   구현
*   테스트
*   코드 검토(Code Review)
*   문서화

`/feature-dev`로 호출하고 원하는 내용을 설명하면 클로드가 나머지를 처리합니다.

**가장 적합한 경우:** 코드 스니펫(code snippets)이 아닌 완전한 기능 구축

**설치:**
`npx skills add anthropic/feature-dev`

### 2. Frontend-Design (277,000회 이상 설치)

**무엇을 하는가:** CSS를 작성하기 전에 클로드에게 전문적인 디자인 직관을 부여합니다.

**왜 중요한가:**
이 플러그인이 없으면 클로드는 "AI 생성"으로 인식되는 시각적 특징, 즉 인상적이지 않은 디자인을 생성합니다. Frontend-Design을 사용하면 클로드는 다음을 고려합니다.

*   목적 및 톤(tone)
*   의도적인 제약 조건(constraints)
*   시각적 차별화
*   예상치 못한 타이포그래피(typography) 선택
*   비대칭 레이아웃(layouts)
*   스크롤 트리거 애니메이션(scroll-triggered animations)

미디엄(Medium)의 한 개발자는 다음과 같이 썼습니다.
"사람들이 제 디자이너가 누구냐고 물었습니다. 디자이너는 없습니다. Frontend-Design이 설치된 클로드입니다."

**가장 적합한 경우:** AI 미학에서 벗어나기

**설치:**
`npx skills add anthropic/frontend-design`

### 3. Context7 (Upstash)

**무엇을 하는가:** 클로드에게 실시간, 버전별 라이브러리(library) 문서 접근 권한을 부여합니다.

**해결하는 문제:**
클로드의 학습 데이터는 몇 달 전 정보일 수 있습니다. Next.js 15, React 19, 또는 Tailwind 4를 사용할 때 오래된 문서는 환각(hallucinated) API로 이어집니다. Context7은 사용 중인 정확한 버전의 실제 최신 문서를 질의합니다.

**작동 방식:**
두 가지 핵심 도구:

*   `resolve-library-id` — 라이브러리 조회
*   `query-docs` — 특정 문서 가져오기

클로드는 이제 학습 데이터에 기반한 추측 대신 "Next.js 15 문서에 따르면…"이라고 말할 수 있습니다.

**가장 적합한 경우:** 버전이 중요한 빠르게 변화하는 프레임워크(frameworks)

**설치:** Claude Desktop 설정(config)에서 MCP를 통해

### 4. GitHub MCP Server

**무엇을 하는가:** 클로드가 저장소(repos)를 검색하고, 코드를 읽고, 이슈(issues)를 열고, 풀 리퀘스트(pull requests)를 생성할 수 있도록 합니다.

**왜 중요한가:**
코드를 수동으로 복사하는 대신, 다음과 같이 질문할 수 있습니다.
"src/ 디렉토리를 분석하고 이슈 #42에 대한 해결책을 제안해 줘."

클로드는 다음을 수행합니다.

*   저장소 검색
*   관련 파일 읽기
*   해결책 제안
*   변경 사항이 포함된 PR 생성

**가장 적합한 경우:** GitHub로 작업하는 소프트웨어 개발자

**설치:** 앤스로픽 공식 마켓플레이스에서 이용 가능

### 5. PostgreSQL MCP

**무엇을 하는가:** PostgreSQL 데이터베이스(databases)에 대한 직접적인 읽기 전용 접근 권한을 제공합니다.

**왜 중요한가:**
다음과 같은 자연어 질문을 할 수 있습니다.
"지난달 지출 상위 5명 사용자를 보여줘."

클로드는 다음을 수행합니다.

*   데이터베이스 스키마(schema)를 자동으로 검사
*   SQL 쿼리(query) 생성
*   실행
*   실제 데이터 반환

테이블 이름을 복사할 필요도, SQL 구문 디버깅(debugging)도 필요 없습니다. 그저 결과만.

**가장 적합한 경우:** 데이터 분석가 및 엔지니어

**보안 참고:** 읽기 전용 접근은 우발적인 데이터 수정을 방지합니다.

### 6. Playwright MCP

**무엇을 하는가:** 클로드가 실시간 Chrome 브라우저를 제어하여 웹 앱(web apps)을 테스트합니다.

**왜 중요한가:**
플레이라이트(Playwright) 테스트 스크립트를 작성하는 대신, 다음과 같이 말할 수 있습니다.
"결제 흐름을 테스트해 줘: 제품 탐색, 장바구니 추가, 테스트 카드 정보 입력, 결제, 확인 메시지 검증."

클로드는 보이는 브라우저 창을 열고, 당신이 지켜보는 동안 테스트를 실행합니다.

**판도를 바꾸는 점:** 인증(authenticated) 기능의 경우, 수동으로 로그인한 다음 클로드가 그 시점부터 작업을 이어받습니다.

**가장 적합한 경우:** 웹 앱을 테스트하는 프런트엔드(Frontend) 개발자

**경고:** 리소스(resource) 집약적일 수 있음

### 7. Brave Search MCP

**무엇을 하는가:** 클로드를 실시간 인터넷에 연결합니다.

**왜 중요한가:**
클로드의 학습 데이터에는 마감 시점(2025년 1월)이 있습니다. Brave Search를 통해 클로드는 다음을 수행할 수 있습니다.

*   시사 연구
*   주식 가격 확인
*   최신 라이브러리(library) 문서 찾기
*   실시간으로 사실 확인

"이번 주 최신 AI 출시작은 무엇인가요?"라고 질문하면 클로드는 웹을 검색하고 최신 정보를 종합합니다.

**가장 적합한 경우:** 연구, 사실 확인, 시사

### 8. Google Workspace via gws CLI (3일 만에 4,900개 스타)

**무엇을 하는가:** 내장된 MCP를 갖춘 모든 Google Workspace API를 위한 통합 CLI(Command Line Interface)입니다.

**기술적 돌파구:**
`gws` 이전에는 클로드를 Google에 연결하려면 Drive, Gmail, Calendar, Sheets 각각에 대해 별도의 OAuth 설정이 필요했습니다. Google은 `gws`를 출시하여 모든 것을 바꿨습니다.

**설치:**
`npm install -g @googleworkspace/cli`
`gws mcp -s drive,gmail,calendar,sheets`

이제 클로드가 전체 Google Workspace에 직접 접근할 수 있습니다.

**예시 워크플로:**
"읽지 않은 이메일을 읽고, 긴급한 내용에 대한 답장을 초안으로 작성하고, 분기별 지표 스프레드시트를 업데이트하고, 다가오는 마감일에 대한 캘린더 이벤트를 생성하고, 요약 문서를 생성해 줘."

하나의 프롬프트(prompt). 복사-붙여넣기 제로.

**가장 적합한 경우:** Google Workspace를 사용하는 지식 노동자(knowledge workers)

### 9. Slack MCP Server

**무엇을 하는가:** 채널 기록 읽기, 스레드(threads) 요약, 메시지 게시를 수행합니다.

**왜 중요한가:**
2026년 1월 업데이트로 대화형 초안 작성(interactive drafting) 기능이 추가되어 이제 다음을 수행할 수 있습니다.

*   Slack 메시지 초안 작성
*   클로드에서 미리보기
*   검토 및 편집
*   게시 전 승인

**간과된 사용 사례:**
"답변하기 전에 지난 2일 동안의 #engineering 채널을 요약해 줘." 500개의 메시지를 읽지 않고도 완전한 맥락을 파악할 수 있습니다.

**가장 적합한 경우:** 여러 Slack 워크스페이스(workspaces)에서 작업하는 모든 사람

### 10. Memory Bank / Claude-Mem

**무엇을 하는가:** 클로드에게 세션(session) 간 영구적인 기억력을 부여합니다.

**해결하는 문제:**
LLM(Large Language Models)은 대화 사이에 컨텍스트를 잊어버립니다. Memory Bank는 다음을 저장하는 `.memory` 폴더를 생성합니다.

*   사용자 선호도
*   프로젝트 세부 정보
*   과거 결정
*   코딩 표준

**그 결과:**
클로드는 당신의 인증 패턴, 오류 처리 규칙, 테스트 요구 사항을 영구적으로 기억합니다.

한 번 설명하면 클로드는 영원히 기억합니다.

**가장 적합한 경우:** 컨텍스트(context)가 중요한 장기 프로젝트

## 플러그인 설치 방법 (3가지)

### 방법 1: Claude Desktop 설정(Config) (MCP 서버)

`claude_desktop_config.json` 편집:

```json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres", "postgresql://localhost/mydb"]
    }
  }
}
```

Claude Desktop을 다시 시작하세요.

### 방법 2: npm을 통한 스킬(Skills)

`npx skills add anthropic/frontend-design`
`npx skills add anthropic/feature-dev`

### 방법 3: Claude 코드 플러그인 시스템

`/plugin install feature-dev@claude-plugins-official`
