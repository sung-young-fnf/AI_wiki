# Claw Code: Claude Code 에이전트 하네스 클론이 급부상하는 이유 (별점 11만 개 돌파)

역사상 가장 빠르게 10만 개 이상의 별(Stars)을 획득한 저장소가 등장했습니다. 출시 후 단 몇 시간 만에 벌어진 일입니다. 'Claw Code'는 공개 2시간 만에 별 5만 개를 돌파하며 큰 화제를 모으고 있습니다.

이 현상의 배경에는 흥미로운 사건이 있습니다. 2026년 3월 31일, 앤스로픽(Anthropic)은 실수로 자사의 'Claude Code' npm 패키지에 `.map` 파일을 포함해 배포했습니다. 이 파일 하나로 인해 1,906개의 소스 파일에 걸쳐 있는 512,000줄의 타입스크립트(TypeScript) 코드가 노출되었으며, 이는 앤스로픽의 주력 AI 코딩 에이전트의 내부 아키텍처 전체를 드러내는 결과를 초래했습니다.

개발자 시그리드 진(Sigrid Jin)은 이 유출 직후, 단 한 줄의 독점 소스 코드도 복사하지 않고 핵심 아키텍처 패턴만을 파악하여 파이썬(Python)으로 새롭게 구현했습니다. 이것이 바로 'Claw Code' 프로젝트의 시작입니다.

### Claw Code란 무엇인가?

Claw Code는 오픈 소스 AI 코딩 에이전트 프레임워크입니다. 개발팀은 이를 Claude Code의 내부 아키텍처를 파이썬과 러스트(Rust)로 처음부터 다시 작성한 '클린룸 재작성(Clean-room rewrite)' 버전이라고 설명합니다.

**주요 특징:**
*   **법적 독립성:** 독점 코드를 복사하지 않았으며, 아키텍처를 새롭게 구현하여 법적으로 독립적입니다.
*   **공급자 불가지론(Provider-agnostic):** 클로드(Claude)뿐만 아니라 오픈AI(OpenAI) 또는 로컬 모델과도 연동됩니다.
*   **감사 및 확장 가능:** 전체 코드를 검토할 수 있으며 자유롭게 기능을 확장할 수 있습니다.
*   **무료 및 자체 호스팅:** 누구나 무료로 사용할 수 있으며 자체 인프라에 구축 가능합니다.

### 구축 과정 및 현재 상태

시그리드 진은 오픈AI의 코덱스(Codex)를 기반으로 한 워크플로우 레이어인 'oh-my-codex(OmX)'를 사용하여 이 프로젝트를 구축했습니다. 현재 Claw Code의 상태는 다음과 같습니다.

*   파이썬 기반의 토대가 마련되어 작동 중입니다.
*   현재 러스트(Rust)로의 포팅이 활발히 진행 중입니다 (현재 러스트 비중 약 89.6%).
*   조사, 매니페스트 확인, 검증 등을 위한 27개의 CLI 서브 명령어를 지원합니다.
*   아직 Claude Code를 완전히 대체할 수 있는 런타임 단계는 아니며, 에이전트 실행 기능은 개발 중에 있습니다.

### 아키텍처 및 핵심 구성 요소

Claw Code는 성능과 유연성을 모두 잡기 위해 이중 레이어 설계를 채택했습니다.
*   **러스트(Rust):** 성능이 중요한 런타임 경로 담당.
*   **파이썬(Python):** 에이전트 오케스트레이션(Orchestration) 및 LLM 통합 담당.

**주요 구성 요소:**
1.  **도구 시스템(Tool System):** 파일 I/O, 배시(bash) 실행, Git 작업 등 19개의 내장 도구를 제공하며, 각 도구는 설정 가능한 액세스 제어를 통해 샌드박스(Sandbox)화되어 있습니다.
2.  **쿼리 엔진(Query Engine):** 모든 LLM API 호출을 처리하며, 응답 스트리밍, 캐싱 및 다단계 오케스트레이션을 지원합니다. 특정 모델 공급자에 종속되지 않습니다.
3.  **멀티 에이전트 오케스트레이션:** '스웜(Swarms)'이라 불리는 하위 에이전트를 생성하여 복잡한 작업을 병렬로 처리합니다.
4.  **MCP 통합:** 모델 컨텍스트 프로토콜(Model Context Protocol)을 완벽하게 지원하여 다양한 전송 방식(Stdio, SSE, HTTP 등)을 통해 외부 도구 서버에 연결합니다.
5.  **세션 및 메모리:** 다층 메모리 시스템과 세션 지속성 기능을 갖추고 있으며, 컨텍스트 발견을 위해 `CLAUDE.md` 지침 파일 시스템을 사용합니다.

### 7단계 부트스트랩 시퀀스(Bootstrap Sequence)

Claw Code는 실행 시마다 다음과 같은 엄격한 단계를 거칩니다.
1.  **Prefetch:** 워크스페이스 메타데이터 및 설정 수집
2.  **Warning Handler:** 오류 처리기 설치
3.  **CLI Parser + Trust Gate:** 인자 분석 및 권한 검증
4.  **Setup + Parallel Load:** 워크스페이스 설정 및 명령어/에이전트 로드
5.  **Deferred Init:** 권한 검증 후 지연된 단계 실행
6.  **Mode Routing:** 런타임 모드(표준, 원격, SSH 등) 경로 설정
7.  **Query Engine Submit Loop:** 메인 대화 루프 진입 및 도구 호출 처리

### Claude Code vs Claw Code

**Claw Code의 장점:**
*   **투명성:** 모든 코드 라인을 읽고 검토할 수 있습니다.
*   **멀티 공급자 지원:** 특정 LLM 벤더에 갇히지 않습니다.
*   **커스터마이징:** 도구, 명령어, 오케스트레이션 방식을 직접 수정할 수 있습니다.

**Claude Code의 장점:**
*   생산 현장에서 검증된 안정적인 런타임입니다.
*   앤스로픽의 공식 지원 및 업데이트를 받습니다.
*   VS Code, JetBrains 등 IDE 통합이 잘 되어 있습니다.
*   KAIROS(선제적 배경 에이전트), ULTRAPLAN(고도화된 계획 수립) 등 향후 추가될 강력한 기능을 갖추고 있습니다.

### 결론

Claw Code가 DMCA 테이크다운(저작권 침해 삭제 요청)에서 살아남을 수 있는 이유는 유출된 소스를 그대로 복제한 것이 아니라 '클린룸 구현' 방식을 택했기 때문입니다. 다른 언어로 아키텍처 패턴을 재구현한 것은 새로운 창작물로 간주될 여지가 큽니다.

에이전트 하네스(Agent harness)가 어떻게 작동하는지 공부하고 싶거나, Claude Code의 엔진 구조를 이해하고 그 위에 자신만의 도구를 구축하고 싶은 개발자들에게 Claw Code는 훌륭한 시작점이 될 것입니다.
