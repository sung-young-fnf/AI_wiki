## 1. cmux란?

**`manaflow-ai/cmux`** — Ghostty 기반 **macOS 네이티브 터미널 앱**. AI 코딩 에이전트(Claude Code, Codex, Cursor CLI, Gemini CLI, Aider 등)를 **병렬로 오케스트레이션**하기 위해 설계됨.

- 개발사: **Manaflow** (Y Combinator 소속 2인 스타트업)
- 출시 2개월 만에 GitHub **10K⭐** 돌파
- 현재 지표: **14.7K⭐ / 1.1K forks / 90 contributors** (v0.63.2)

> ⚠️ **이름 혼동 주의** — "cmux"라는 이름의 다른 프로젝트도 존재:
> - `manaflow-ai/cmux-code` — 경량 터미널 코딩 에이전트 CLI (별개 프로젝트)
> - `coder/mux` — 병렬 에이전트 개발용 데스크톱 앱 (이름만 유사)
> - 전통적인 **"terminal multiplexer"(tmux 계열)와는 다름** — cmux는 GUI 앱

## 2. 핵심 기능

| 카테고리 | 기능 |
|---|---|
| **UI** | 수직 탭 사이드바(git branch, PR 상태, 작업 디렉토리, 포트, 알림 표시), 수평/수직 스플릿 페인 |
| **알림** | 에이전트가 응답/승인 요청 시 페인에 **파란 링(ring)** 표시, 전용 알림 패널 |
| **내장 브라우저** | 접근성 트리 스냅샷, 폼 자동입력, JS 실행 가능한 **스크립터블 웹뷰** — 문서/PR 리뷰 병행 |
| **브라우저 임포트** | Chrome / Firefox / Arc 등 20+ 브라우저 쿠키·세션 가져오기 |
| **SSH/리모트** | 원격 워크스페이스 생성, 브라우저 페인도 원격 네트워크 경유 |
| **자동화 API** | CLI + **Unix 도메인 소켓 API** — 에이전트가 페인 생성/진행상황 전송 가능 |
| **Claude Code Teams** | 팀메이트 모드 스폰 네이티브 지원 |
| **커스텀 커맨드** | `cmux.json`으로 프로젝트별 커맨드 정의 |

## 3. 아키텍처·스택

- **Swift / AppKit (78.2%)** + Python (15.4%) + TypeScript (2.5%) — **Non-Electron 네이티브**
- **libghostty** 기반 GPU 가속 렌더링 (Ghostty 포크 아님, 라이브러리로 임베드)
- macOS 전용 (Windows/Linux 미지원)

## 4. vs tmux 비교

| 항목 | cmux | tmux |
|---|---|---|
| **형태** | macOS 네이티브 GUI 앱 | CLI 멀티플렉서 (모든 터미널 내부 실행) |
| **조작** | 탭·클릭·단축키 (prefix 키 불필요) | `Ctrl-b` 등 prefix 필수, 설정파일 집약 |
| **AI 에이전트 지원** | 소켓 API·알림링 **네이티브** | 사용자가 수동 커스터마이징 |
| **브라우저** | 내장 스크립터블 웹뷰 | 없음 |
| **크로스플랫폼** | ❌ macOS only | ✅ Linux / macOS / Cygwin |
| **렌더링** | GPU 가속 | 텍스트 기반 |
| **학습 곡선** | 낮음 (제로 설정) | 높음 (설정 파일 중심) |

## 7. 타깃 사용자

- **1차**: 여러 AI 코딩 에이전트(Claude Code, Aider, Cline, Codex 등)를 **동시 구동**하며 멀티태스킹하는 개발자
- **2차**: tmux의 복잡성 없이 알림·조직화·브라우저 연동을 원하는 파워 유저

## 8. 인사이트

- 기존 터미널 + tmux 조합이 AI 에이전트 멀티 세션에 적합하지 않다는 시장 gap을 정확히 공략
- **"에이전트가 사람의 주의를 요구하는 시점"을 시각화**(파란 링)하는 UX가 핵심 차별점
- 소켓 API를 통해 에이전트가 UI를 역제어할 수 있는 구조는 향후 에이전트 오케스트레이션 표준이 될 가능성
- 네이티브 Swift 채택으로 Electron 기반 경쟁 제품(VS Code, Warp 등) 대비 성능·메모리 우위
