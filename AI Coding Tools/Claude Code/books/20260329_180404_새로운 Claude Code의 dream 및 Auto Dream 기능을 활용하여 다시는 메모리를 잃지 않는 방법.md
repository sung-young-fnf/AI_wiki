따라서 가장 먼저 이해해야 할 것은 Claude Code 메모리의 현재 상태와 비효율적인 이유입니다.

## Claude Code 메모리 문제

오토-메모리(Auto-memory)는 Claude Code의 가장 유용한 기능 중 하나입니다. 이 기능은 Claude가 여러 세션(session)에 걸쳐 프로젝트에 대한 메모를 작성할 수 있게 합니다.

제 경험과 테스트에 따르면, 처음 몇 세션 동안은 잘 작동하지만, 그 후에는 문제가 발생하기 시작합니다.

사용하면 할수록 메모리 폴더(folder)에 '노이즈(noise)'가 축적됩니다. 다음은 몇 가지 실제 예시입니다.

*   **모순 (Contradictions)** — 첫째 주에 Claude에게 항상 리액트(React)를 사용하라고 지시했습니다. 3주 후 뷰(Vue)로 전환했지만, 이제 두 지침이 모두 존재하며 Claude는 어떤 것을 따라야 할지 모릅니다.
*   **오래된 정보 (Stale information)** — 몇 달 전에 삭제한 파일을 참조하는 디버깅(debugging) 메모입니다. 번복되었지만 전혀 업데이트되지 않은 결정 사항들입니다.
*   **상대적인 날짜 (Relative dates)** — Claude가 '기능은 다음 주 금요일까지'라고 적힌 메모를 저장했습니다. 그 메모는 이제 6개월이 지나 구식이 되었습니다.
*   **과도하게 커진 인덱스 (Bloated index)** — `MEMORY.md` 인덱스(index) 파일은 모든 세션 시작 시 로드(load)됩니다. 파일이 커질수록 더 많은 컨텍스트(context)를 차지하게 됩니다.

Claude를 더 똑똑하게 만들기로 되어 있던 메모리가 이제는 Claude를 더 느리고 덜 정확하게 만들고 있습니다.

이것은 정리 프로세스(process) 없이 메모가 축적될 때 발생하는 현상입니다. 새로운 드림(dream) 기능은 이러한 메모리를 정리하도록 설계되었습니다.

그렇다면 `/dream`과 오토-드림(Auto-Dream) 중 어떤 것이 작동할까요?

## Claude Code의 /dream과 오토-드림(Auto-Dream)

이들은 동일한 기능은 아니지만, 같은 작업을 수행하도록 설계되었습니다.

### /dream — 수동 커맨드(Manual Command)

이것은 Claude Code 세션(session) 내에서 실행하는 슬래시 커맨드입니다.

이 커맨드를 실행하면, Claude는 메모리 파일(file)에 대한 통합 작업(consolidation pass)을 수행하여 모순을 정리하고, 중복을 병합하며, 오래된 날짜를 수정하고, 인덱스(index)를 강화합니다.

실행 시점을 사용자가 제어할 수 있습니다.

### 오토-드림(Auto-Dream) — 자동화된 버전(Automated Version)

오토-드림은 동일한 통합 작업을 수행하지만, 세션 간에 백그라운드(background)에서 작동합니다. 수동 트리거(manual trigger)를 필요로 하지 않으며, 다음 두 가지 조건이 모두 충족될 때 실행됩니다.

*   마지막 주기(cycle) 이후 최소 24시간이 경과했을 때
*   마지막 주기 이후 최소 5개의 세션이 완료되었을 때

지금 Claude Code를 열고 `/memory`를 실행하면, '오토-드림: 꺼짐 · 없음(Auto-dream: off · never)'으로 표시된 것을 볼 수 있을 것입니다.

*(이미지)*

이는 UI(사용자 인터페이스)에는 존재하지만, 현재 기능 플래그(feature flag) 뒤에 숨겨져 있어 대부분의 사용자에게는 아직 활성화되지 않았습니다.

반면에 `/dream` 커맨드는 Anthropic의 내장 릴리즈(release)를 통해서는 아니지만 사용할 수 있습니다.

이 커맨드의 프롬프트(prompt)는 깃허브(GitHub)에 공개되어 있어, 스킬(skill)로 구축하여 지금 바로 사용할 수 있습니다. 잠시 후에 더 자세히 설명하겠습니다.

이렇게 생각하면 됩니다. `/dream`은 수동 스위치(manual switch)이고, 오토-드림은 정해진 스케줄(schedule)에 따라 실행되도록 설정된 동일한 스위치입니다.

- 메모리 파일과 현재 코드베이스(codebase) 상태 간의 모순을 찾습니다.

### 3단계 — 통합 (Consolidate)
- 동일한 주제를 다루는 중복 메모리를 병합합니다.
- 상대 날짜를 절대 날짜로 변환합니다. (예: "지난 목요일" → "2026-03-20")

- 더 이상 존재하지 않는 오래된 파일 경로, 함수 이름 또는 플래그(flag)를 업데이트합니다.

## Claude Code 드림(Dream) 작동 방식

다음 4단계로 작동합니다.

### 1단계 — 방향 설정 (Orient)

드림(Dream)은 이미 존재하는 것을 조사하는 것으로 시작합니다. `MEMORY.md` 인덱스(index)를 읽고, 메모리 디렉토리(directory)를 나열하며, 기존 주제 파일(topic files)을 대략적으로 살펴봅니다.

이는 이미 존재하는 것의 중복 생성을 방지합니다.

### 2단계 — 최근 신호 수집 (Gather Recent Signal)

드림은 모든 세션 트랜스크립트(session transcript)를 읽지 않습니다. 그렇게 하면 느리고 비용이 많이 들기 때문입니다.

대신 먼저 일일 로그(daily logs)를 살펴본 다음, 현재 코드베이스(codebase)에 있는 내용과 달라진 기존 메모리(memory)의 사실이 있는지 확인합니다.

특정 컨텍스트(context)가 필요한 경우, 전체를 읽는 대신 JSONL 트랜스크립트 파일(JSONL transcript files)에 대해 특정 `grep` 검색을 사용합니다.

### 3단계 — 통합 (Consolidate)

여기서 정리 작업이 이루어집니다.

*   새로운 정보를 거의 중복된 형태로 생성하는 대신 기존 주제 파일(topic files)에 병합합니다.
*   "어제" 또는 "다음 주 금요일"과 같은 상대 날짜를 절대 타임스탬프(absolute timestamps)로 변환합니다.
*   모순된 사실을 삭제합니다.
*   프로젝트의 현재 상태에서 벗어난 모든 것을 업데이트합니다.

### 4단계 — 정리 및 인덱스화 (Prune and Index)

드림은 `MEMORY.md` 인덱스(index)를 한 가지 규칙에 따라 재구성합니다. 200줄 미만으로 유지하는 것입니다.

인덱스는 콘텐츠 덤프(content dump)가 아닌 가벼운 포인터(pointer) 파일이어야 합니다. 자세한 정보는 개별 주제 파일(topic files)에 있습니다.

마지막으로 드림은 변경, 병합 또는 제거된 내용에 대한 요약을 반환합니다. 만약 메모리가 이미 깨끗했다면, 그렇게 알리고 종료합니다.

## 최종 의견

오토-드림(Auto-Dream)은 아직 대부분의 사용자에게 활성화되지 않았지만, 드림(Dream) 프롬프트(prompt)가 깃허브(GitHub)에 공개되어 있기 때문에 지금 바로 사용을 시작할 수 있습니다.

위에서 시연한 것처럼 Claude Code 스킬(skill)로 구축하여 수동으로 실행할 수 있습니다.

### 드림 스킬(Dream Skill) 구축

빠르게 시작하려면, 아무 프로젝트에서나 Claude Code를 열고 다음을 실행하세요.

`Create a new skill called dream using the Dream memory consolidation prompt from https://github.com/Piebald-AI/claude-code-system-prompts`

Claude는 프롬프트를 가져와 스킬 파일을 생성할 것이며, `/dream`을 통해 스킬을 트리거(trigger)할 수 있습니다.

### 실행 방법

스킬이 준비되면, 통합하려는 대상에 따라 세 가지 옵션이 있습니다.

*   `/dream` — 현재 프로젝트의 메모리에만 통합을 실행합니다.
*   `/dream user` — 모든 프로젝트에 적용되는 사용자 수준 메모리(user-level memory)에 실행합니다.
*   `/dream all` — 두 가지를 동시에 실행합니다.

여러 이전 세션(session)이 있는 프로젝트에서 `/dream`으로 시작하세요. 이력이 많을수록 정리할 내용이 더 많을 것입니다. 합리적인 접근 방식은 길거나 복잡한 세션 후, 또는 Claude가 오래된 것을 참조하거나 스스로 모순되는 것을 발견할 때마다 실행하는 것입니다.

오토-드림(Auto-Dream)이 완전히 출시되면, 이 과정은 자동화될 것입니다.

Claude Code의 `/dream`을 사용해 보셨나요? 경험을 아래 댓글로 알려주세요.
