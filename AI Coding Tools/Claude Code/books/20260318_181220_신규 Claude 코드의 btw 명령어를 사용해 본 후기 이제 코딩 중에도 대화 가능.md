Claude Code의 `/btw`는 Claude가 작업을 수행하는 동안 보조 대화를 생성하는 새로운 기능입니다.

만약 Claude Code에서 복잡한 작업을 처리하는 것을 지켜보던 중 갑자기 궁금한 점이 생겼던 경험이 있다면, 아마도 작업을 기다리거나 중단해야 했을 것입니다. 이 두 가지 옵션 모두 만족스럽지 않습니다.

Anthropic은 이러한 문제를 해결하는 새로운 명령어인 `/btw`를 막 출시했습니다.

엔터 키를 누르거나 클릭하여 이미지를 전체 크기로 보세요.

이 명령어를 사용하면 Claude Code가 작업을 수행하는 동안 부가적인 질문을 할 수 있습니다. 답변은 오버레이(overlay) 형태로 나타나며, 주요 작업은 계속 실행되고 대화 기록은 깔끔하게 유지됩니다.

Anthropic 팀의 Erik Schluntz가 사이드 프로젝트로 개발한 이 기능은 일상적인 워크플로우에 큰 변화를 가져올 수 있는 작지만 중요한 기능 중 하나입니다.

이 글에서는 `/btw`가 어떻게 작동하는지, 설정 방법, 그리고 실제 예제를 통해 어떤 결과를 기대할 수 있는지 보여드리겠습니다.

### `/btw`란 무엇이며 어떻게 작동하는가

`/btw` 명령어는 현재 작업을 멈추지 않고 Claude에게 빠른 질문을 할 수 있게 해줍니다.

명령어는 다음과 같습니다:

`/btw <질문>`

엔터 키를 누르거나 클릭하여 이미지를 전체 크기로 보세요.

예시:

`/btw retry logic은 무엇을 하나요?`

엔터 키를 누르거나 클릭하여 이미지를 전체 크기로 보세요.

이 명령어를 실행하면 Claude Code는 답변이 포함된 오버레이를 띄웁니다. 주요 작업은 백그라운드에서 계속 실행됩니다.

엔터 키를 누르거나 클릭하여 이미지를 전체 크기로 보세요.

응답을 읽은 후, Space, Enter 또는 Escape 키를 눌러 오버레이를 닫을 수 있습니다.

주요 기능:

*   **병렬 실행** — 주요 작업이 일시 정지되거나 기다리지 않습니다.
*   **오버레이 응답** — 답변이 인라인(inline)이 아닌 팝업 형태로 나타납니다.
*   **기록 오염 없음** — 보조 질문이 주요 대화 컨텍스트(context)에 추가되지 않습니다.
*   **단일 턴(Single-turn)만 가능** — 하나의 응답만 받으며, 왕복 대화는 불가능합니다.
*   **도구 접근 불가** — 응답은 Claude의 지식과 현재 세션 컨텍스트(session context)에서 생성됩니다.

`/btw`는 도구 접근 권한이 없으므로, 파일을 읽거나, 명령어를 실행하거나, API 호출을 할 수 없습니다. 이는 빠른 지식 조회 및 설명을 위한 기능입니다.

이러한 설계는 기능을 가볍고 비용 효율적으로 유지합니다. 전체 에이전트(agent) 상호작용을 시작하는 것이 아니기 때문입니다.

이 명령어는 현재 세션 컨텍스트에서 정보를 가져오므로, 만약 파이썬 프로젝트를 작업 중이었다면 Claude는 이를 인지합니다. `/btw list와 tuple의 차이점은 무엇인가요?`라고 물으면 관련성 있고 상황에 맞는 응답을 얻을 수 있습니다.

### `/btw` 설정 및 사용 방법

`/btw` 명령어는 Claude Code 버전 2.1.72 이상에서 사용할 수 있습니다.

버전 확인은 다음 명령어를 실행합니다:

`claude --version`

엔터 키를 누르거나 클릭하여 이미지를 전체 크기로 보세요.

이전 버전을 사용 중이라면 다음 명령어로 업데이트하세요:

`claude update`

엔터 키를 누르거나 클릭하여 이미지를 전체 크기로 보세요.

올바른 버전으로 업데이트한 후, 평소처럼 Claude Code를 실행하세요:

`claude`

명령어 목록에 `/btw`가 표시될 것입니다.

엔터 키를 누르거나 클릭하여 이미지를 전체 크기로 보세요.

`/btw`를 입력하면 Claude Code는 "주요 대화를 방해하지 않고 빠른 보조 질문을 합니다."라는 설명을 보여줍니다.

`/btw <질문>`

`<질문>`을 원하는 내용으로 바꾸면 됩니다.

### 테스트 — 작동 중인 `/btw`

실제 예시를 통해 `/btw`가 어떻게 작동하는지 보여드리겠습니다.

FastAPI에 대해 묻는 간단한 테스트를 실행했습니다:

`/btw whats is fast api`

엔터 키를 누르거나 클릭하여 이미지를 전체 크기로 보세요.

응답은 즉시 오버레이 형태로 나타났고, 터미널을 가리며 FastAPI의 핵심 기능, 간단한 코드 예시, 개발자들이 이를 사용하는 이유, 그리고 Flask 및 Django와의 비교를 상세하게 설명했습니다.

엔터 키를 누르거나 클릭하여 이미지를 전체 크기로 보세요.

오버레이에는 빠른 조회에서 기대할 수 있는 모든 것이 포함되어 있습니다: 글머리 기호, 코드 스니펫(code snippets), 그리고 실용적인 컨텍스트.

Space, Enter 또는 Escape 키를 눌러 오버레이를 닫고 주요 세션으로 돌아갈 수 있습니다.

### 사용 사례

이 명령어는 특정 상황에 이상적입니다:

*   코드를 작성하는 동안 빠른 구문 조회(syntax lookups)
*   작업 흐름을 방해하지 않고 개념 명확화
*   작업 중 라이브러리 또는 함수가 무엇을 하는지 확인
*   즉석에서 정의나 설명 얻기

각 `/btw` 호출은 도구 접근이 없는 단일 턴 응답이므로, 전체 에이전트 상호작용에 토큰을 소모하지 않습니다.

긴 코딩 세션을 진행하는 동안 컨텍스트 비대화(context bloat)나 비용 급증(runaway costs) 걱정 없이 빠르게 질문을 던질 수 있습니다.

### 최종 의견

`/btw` 명령어는 작은 추가 기능이지만, Claude Code 워크플로우(workflow)에서 실제 불편함을 해결해줍니다.

이 명령어가 없었다면, 부가적인 질문을 하려면 작업이 끝날 때까지 기다리거나 작업을 중단해야 했습니다. 이제 작업 흐름을 유지하면서 답변을 얻고 계속 진행할 수 있습니다.

버전 2.1.72 이상을 사용 중이라면 한번 사용해보세요. 작업을 실행하고, `/btw`로 질문을 던져보고, 작업 흐름에 어떻게 맞는지 확인해보세요.
