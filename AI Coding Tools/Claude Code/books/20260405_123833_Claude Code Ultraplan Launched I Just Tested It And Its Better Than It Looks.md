# Claude Code Ultraplan Launched: I Just Tested It (And It’s Better Than It Looks)

앤스로픽(Anthropic)이 클로드 코드(Claude Code)에 '울트라플랜(Ultraplan)' 기능을 새롭게 추가했습니다. 이 기능은 특히 에이전트 기반의 계획 수립 워크플로우를 위해 설계되었습니다.

최근 앤스로픽은 프로(Pro) 및 맥스(Max) 구독자가 오픈클로(OpenClaw)와 같은 서드파티 도구를 통해 자동화 및 코딩 작업을 수행하는 것을 제한하기 시작했습니다. 이는 에이전트 루프가 시스템에 과도한 부하를 주기 때문이며, 대신 API 키 사용이나 울트라플랜 같은 자체 에이전트 도구 사용을 권장하려는 의도로 풀이됩니다.

### 울트라플랜(Ultraplan)이란?

울트라플랜은 CLI(명령줄 인터페이스)에서 계획 작업을 시작하고, 웹상의 클로드 코드에서 초안을 작성한 뒤, 이를 원격으로 실행하거나 다시 터미널로 가져와 실행할 수 있는 '핸드오프(Handoff)' 시스템입니다.

기존에는 클로드 코드에 복잡한 작업을 요청하면 터미널에서 계획이 생성될 때까지 기다려야 했지만, 울트라플랜은 이 과정을 클라우드로 옮겨 터미널을 자유롭게 사용할 수 있게 해줍니다. 계획이 준비되면 터미널이 아닌 브라우저에서 내용을 검토하고 수정 요청을 할 수 있어, 대규모 작업 시 훨씬 더 풍부한 리뷰 환경을 제공합니다.

현재 울트라플랜은 리서치 프리뷰(Research Preview) 단계에 있습니다.

### 울트라플랜 실행 방법

CLI에서 다음 세 가지 방법으로 실행할 수 있습니다.

1.  **명령어 사용**: `/ultraplan` 뒤에 프롬프트 입력
2.  **키워드 사용**: 일반 프롬프트 어디에나 `ultraplan` 단어 포함
3.  **로컬 계획에서 전환**: 로컬 계획이 완료된 후 승인 대화 상자에서 "No, refine with Ultraplan on Claude Code on the web" 선택

실행 후 CLI에 표시되는 상태 표시기를 통해 원격 세션의 진행 상황을 확인할 수 있으며, 그동안 터미널에서는 다른 작업을 계속할 수 있습니다.

### 요구 사항

울트라플랜을 사용하려면 두 가지 조건이 반드시 충족되어야 합니다.

*   웹용 클로드 코드 계정
*   **GitHub 리포지토리(필수)**: 클라우드 세션이 GitHub를 통해 코드에 접근하므로, 로컬 전용 프로젝트에서는 작동하지 않습니다.

### 브라우저에서의 계획 검토 및 실행

세션 링크를 클릭하면 웹용 클로드 코드에서 계획이 열립니다. 여기서는 다음과 같은 기능을 활용할 수 있습니다.

*   **인라인 댓글(Inline comments)**: 특정 섹션을 강조 표시하고 피드백 남기기
*   **이모지 반응**: 빠르게 승인하거나 우려 사항 표시
*   **개요 사이드바**: 계획의 각 섹션으로 빠르게 이동

검토가 끝나면 두 가지 방식으로 실행할 수 있습니다.

1.  **웹에서 실행**: 클라우드에서 구현을 완료하고 풀 리퀘스트(Pull Request)를 생성합니다.
2.  **터미널로 텔레포트(Teleport)**: 계획을 다시 CLI로 가져와 현재 세션에 주입하거나 새로운 세션에서 시작합니다.

### 요약 및 결론

울트라플랜은 터미널의 좁은 화면보다 더 넓은 환경에서 복잡한 작업을 관리해야 할 때 매우 유용합니다. 특히 다음과 같은 경우에 권장됩니다.

*   여러 파일에 걸친 복잡한 리팩토링 작업
*   이미 GitHub에 업로드된 프로젝트
*   실행 전 계획에 대한 반복적인 수정이 필요한 경우
*   계획 수립 중에도 터미널을 계속 사용해야 할 때

울트라플랜은 CLI와 클라우드 계획 수립, 브라우저 기반 검토를 결합하여 개발 생산성을 높이는 강력한 도구가 될 것입니다.
