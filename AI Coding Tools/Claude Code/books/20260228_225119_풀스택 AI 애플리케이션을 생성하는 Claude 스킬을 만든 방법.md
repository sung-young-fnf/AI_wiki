### 요약 (TL;DR)

이 글에서는 단일 프롬프트(prompt)로 완전한 풀스택(full-stack) AI 애플리케이션을 생성하도록 Claude를 학습시키는 방법을 배웁니다. 저는 다단계 개발 워크플로를 재사용 가능한 Claude 스킬(Skill)로 전환하는 전체 과정을 안내합니다. 이 스킬은 FastAPI 백엔드, 최신 React + Vite 프론트엔드, Tailwind + shadcn UI 컴포넌트, 그리고 내장 OpenAI 통합 기능을 자동으로 생성합니다.

생성형 AI(Generative AI) 분야의 빠른 발전 속에서, AI 코딩 어시스턴트(예: Claude Code 또는 OpenAI의 Codex)에 몇 줄의 프롬프트를 제공하여 최소한 기본적인 기능 프로토타입(prototype)을 구축하는 것이 점차 가능해지고 있습니다. AI 지원 코딩(AI-assisted coding)은 프로토타입을 빠르게 구축하고 아이디어를 신속하게 테스트하는 데 엄청난 지원을 제공하지만, 단순한 프롬프트만으로 풀스택 애플리케이션을 순수하게 구축하는 것은 여전히 매우 어려운 작업입니다. 기껏해야 Streamlit 또는 Gradio 기반의 간단한 사용자 인터페이스를 만들 수 있는데, 이는 테스트/개념 증명(proof-of-concept) 목적으로만 적합하며 프로덕션(production) 준비 시스템에는 적합하지 않습니다.

기존 애플리케이션을 코딩 어시스턴트로 풀스택 애플리케이션으로 변환하는 것은 상당히 어렵습니다. 코딩 어시스턴트가 적절한 도구와 방법을 사용하도록 안내하는 일련의 정확하고 구체적인 프롬프트를 따라야 하기 때문입니다. 따라서 풀스택 개발의 미묘한 차이를 이해하기 위한 기술적 전문 지식이 필요합니다.

이러한 격차는 현재의 코딩 어시스턴트들이 점점 강력해지고 있음에도 불구하고, 많은 파일에 걸친 컨텍스트 관리(context management), 장기적인 아키텍처(architecture) 추론, 적절한 프레임워크(framework) 선택, 그리고 코드베이스(codebase)가 커질 때 코드 품질(code quality)을 보장하는 데 여전히 어려움을 겪기 때문에 발생합니다. 결과적으로 프론트엔드(frontend)와 백엔드(backend) 연결, 환경 간 일관된 구성 유지, 오류 및 보안 문제 처리, 또는 시간이 지남에 따른 코드베이스 발전과 같은 작업은 까다로워질 수 있습니다.

그럼에도 불구하고 AI 지원 코딩의 가치는 여전히 엄청납니다. 단일 프롬프트로 완벽한 엔터프라이즈(enterprise)급 애플리케이션을 마법처럼 구축할 수는 없더라도, 프로젝트 구조 자동 생성, 백엔드 및 프론트엔드 스캐폴딩(scaffolding), 환경 변수 및 API 호출 연결, 작동하는 골격(skeleton) 제공을 통해 초기 마찰을 크게 줄일 수 있습니다. 일단 기반 작업이 준비되면, 다른 기능들은 단계별로 추가할 수 있습니다. 이는 무거운 초기 설정 부담을 훨씬 가벼운 작업으로 변화시켜, 개발자들이 상용구(boilerplate) 대신 핵심 애플리케이션 로직, 사용자 경험, 데이터 모델 또는 AI 특정 기능에 집중할 수 있도록 합니다.

이 글에서는 Python + FastAPI 백엔드와 React + Vite 프론트엔드를 결합한 재사용 가능한 풀스택 템플릿(template)을 생성하기 위해 Claude 스킬을 만든 방법을 설명합니다. 저는 Python으로 주로 작업하기 때문에 Python + FastAPI 백엔드를 선택했습니다. 그러나 개발된 Claude 스킬은 다른 백엔드/프론트엔드 프레임워크에도 수정하여 사용할 수 있습니다.

이전 글에서 Claude 스킬을 만드는 방법을 시연했으며, 당시 Claude 스킬을 사용하여 이력서 심사 에이전트(resume screening agent)를 만들었습니다.

### 스킬 사전 준비 (Pre-Skill Preparation)

Claude를 통한 자동화로 전환하기 전에, 저는 먼저 ChatGPT를 통해 GPT-5.1을 사용하여 백엔드 + 프론트엔드 애플리케이션 템플릿을 구축하기 위한 단계별 프롬프트 세트를 계획하고 만들었습니다. 저는 간단한 지침으로 시작했습니다.

```
Python 백엔드와 React 프론트엔드 (잠재적으로 Vite 포함)로 빈 프로젝트(템플릿)를 만들고 싶습니다. 
Python 백엔드는 테스트 코드와 OpenAI API 구성을 포함할 것입니다. 
프론트엔드는 테스트 페이지일 수 있습니다. 목적은 추가 특정 애플리케이션을 개발하는 데 
사용될 수 있는 완전한 템플릿을 만드는 것입니다. 
하지만 이를 Claude Code와 같은 코딩 어시스턴트를 사용하여 하고 싶습니다. 
어떻게 해야 할까요?
```

이에 대한 응답으로 ChatGPT (GPT-5.1)는 다음 일련의 프롬프트를 생성했습니다.

```bash
mkdir fullstack-template
cd fullstack-template

###

###

# Python 백엔드 초기화
cd backend
python -m venv .venv
# Linux / macOS
source .venv/bin/activate
# 또는 Windows
# .\.venv\Scripts\activate

pip install fastapi uvicorn python-dotenv openai
pip freeze > requirements.txt

###
# Codex가 상용구 백엔드 코드를 생성하도록 합니다.
"OpenAI의 ChatCompletions API를 호출하는 /test 엔드포인트 하나와 /chat 엔드포인트 하나를 포함하는 FastAPI 앱을 생성하세요. .env에서 API 키를 로드하세요. localhost:5173에 대한 CORS 설정을 포함하세요."
###

# 백엔드 실행:
uvicorn app:app --reload

# React + Vite 프론트엔드 초기화
cd ../frontend
npm create vite@latest
# 선택: React + JavaScript
npm install
npm install axios

###
# Codex가 프론트엔드 테스트 페이지를 생성하도록 합니다.
"백엔드(localhost:8000)의 /test 및 /chat 결과를 표시하는 입력 상자, 전송 버튼 및 표시 영역이 있는 간단한 React UI를 만드세요. axios를 사용하세요."
###

# 프론트엔드 시작 (터미널 2)
npm run dev

# 백엔드 시작 (터미널 1)
cd ../backend
uvicorn app:app --reload
```

저는 VS Code에서 Claude Code(Claude-sonnet-4.5)를 사용하여 이 단계/프롬프트를 하나씩 따랐습니다. 백엔드와 프론트엔드를 풀스택 애플리케이션 템플릿으로 성공적으로 연결할 수 있었지만, 마음에 들지 않는 기본적인 HTML+CSS UI를 가지고 있었습니다. UI 개선을 위한 프롬프트를 다시 GPT-5.1에 요청했습니다. GPT-5.1은 React UI 개선을 위해 Tailwind CSS + shadcn/ui 사용을 제안했습니다. 다음은 코딩 어시스턴트를 위해 제안한 프롬프트입니다.

```
"기존 React 프론트엔드 UI를 Tailwind CSS + shadcn/ui를 사용하여 현대적이고 전문적인 인터페이스로 리팩토링하세요.
요구 사항:
• 앱 제목이 있는 깔끔한 헤더를 추가하세요.
• 최대 너비로 트랜스크립션 카드(transcription card)를 중앙에 배치하세요.
• 파일 업로드 영역을 Tailwind로 스타일링된 드래그 앤 드롭(drag-and-drop) 드롭존으로 교체하세요.
• shadcn/ui의 Card, Button, Input, Textarea 컴포넌트를 사용하세요.
• Lucide 아이콘(Upload, FileAudio, Loader)을 추가하세요.
• UI를 반응형(데스크톱, 태블릿, 모바일)으로 만드세요.
• Framer Motion을 사용하여 부드러운 애니메이션을 추가하세요.
모든 트랜스크립션 로직은 동일하게 유지하세요. 스타일링과 컴포넌트 구조만 개선하세요."
```

저는 처음에 Claude Code에게 기존 React 프론트엔드 UI를 Tailwind CSS와 shadcn/ui를 사용하여 현대적이고 전문적인 인터페이스로 리팩토링하도록 지시했습니다. 그러나 여러 번의 시도에도 불구하고 Claude Code는 작업을 성공적으로 완료하지 못했습니다. 그 후 OpenAI의 Codex(gpt-5.1-codex 사용)를 시도했는데, 완벽하게 작동하여 인터페이스를 세련되고 시각적으로 매력적인 UI로 변환했습니다.

위에 설명된 단계와 프롬프트는 성공적인 반복만을 나타냅니다. 여러 번의 시도가 실패하여 다시 프롬프트하거나 조정해야 했습니다. 이는 매번 동일한 출력을 재현할 수 있는 반복 가능한 프로세스의 필요성을 강조했습니다. 따라서 전체 워크플로를 캡슐화하고 재현 가능한 결과를 생성하는 Claude 스킬을 만들기로 결정했습니다.

### 풀스택 개발을 위한 Claude 스킬 구축 (Building Claude Skill for Full-Stack Development)

스킬은 Claude가 전문화된 작업에서 성능을 향상시키기 위해 동적으로 로드하는 지침, 스크립트 및 리소스의 폴더입니다. 스킬은 Claude에게 회사 브랜드 가이드라인에 따라 문서를 만들거나, 조직의 특정 워크플로를 사용하여 데이터를 분석하거나, 개인 작업을 자동화하는 등 특정 작업을 반복 가능한 방식으로 완료하는 방법을 가르칩니다. Claude의 스킬에 대한 자세한 내용은 [이전 글](https://medium.com/data-science-collective/how-to-create-a-transparent-information-assessment-extraction-validation-and-generation-ai-agent-e63d3e813a0e)을 참조하세요.

AI 코딩 어시스턴트를 통해 풀스택 애플리케이션 템플릿을 개발하는 과정에서 제가 따랐던 단계와 얻은 경험을 Claude의 스킬에 포함시킬 수 있습니다. 이렇게 하면 Claude(또는 Claude 모델을 사용하는 코딩 어시스턴트)에게 요청할 때마다 동일한 경험과 단계를 사용하여 동일한 결과를 재현할 수 있습니다.

Claude 스킬은 여러 워크플로를 자동화하도록 생성될 수 있습니다. 스킬을 AI 에이전트(agent)와 통합하면 에이전트가 추가 기능, 도구, 지식 베이스, 코드 베이스 및 리소스를 갖추어 워크플로를 완전히 자동화할 수 있습니다.

GPT-5.1-codex가 React 프론트엔드 UI를 리팩토링하는 데 탁월한 성능을 보였기 때문에, 저는 VS Code용 OpenAI의 Codex 확장 내에서 GPT-5.1-codex를 사용하여 전용 "풀스택 개발" 스킬을 구축하기로 결정했습니다. 이를 준비하기 위해, 먼저 Codex에 Claude 스킬이 무엇이고 어떻게 구성되는지 설명하는 참조 링크를 제공했습니다. 그 후, 풀스택 애플리케이션 템플릿을 만들 때 제가 따랐던 모든 단계와 전체 프롬프트 기록을 제공했습니다. 이는 Codex가 상세 지침, 워크플로 로직, 코드 템플릿, 그리고 예제, 참조, 템플릿, 문서화를 포함한 모든 지원 자산(asset)이 포함된 포괄적인 스킬 패키지를 생성하기에 충분한 컨텍스트를 제공했습니다. 제공된 정보를 바탕으로 Codex는 `.claude/skills/fullstack-template-generator` 아래에 깔끔하고 모듈형 디렉터리 구조로 스킬을 구성했습니다. 많은 프론트엔드 코드 파일은 이전에 생성된 애플리케이션 템플릿에서 그대로 복사되었습니다.

# 풀스택 템플릿 생성기

## 개요

이 스킬은 다음과 같은 기능을 갖춘 프로덕션 준비 풀스택 애플리케이션 템플릿 생성을 자동화합니다:

### 백엔드 (Python + FastAPI)
- 비동기(async) 지원을 포함한 FastAPI 프레임워크
- OpenAI ChatGPT API 통합
- 프론트엔드 통신을 위해 구성된 CORS 미들웨어
- 포괄적인 오류 처리 및 유효성 검사
- 환경 기반 구성
- 자동 생성된 API 문서 (Swagger UI)
- 요청 유효성 검사를 위한 Pydantic 모델

### 프론트엔드 (React + Vite)
- 빠른 개발을 위한 React 19 및 Vite 7
- PostCSS + autoprefixer로 구성된 Tailwind CSS 3
- class-variance-authority, clsx, tailwind-merge로 구동되는 shadcn/ui 기본 요소 (Button, Card, Input, Textarea)
- 세련된 마이크로 인터랙션을 위한 Lucide 아이콘 및 Framer Motion
- API 통신을 위한 Axios
- Tailwind 유틸리티만으로 구축된 깔끔하고 반응형 채팅 UI
- 오류 처리 및 로딩 상태
- 핫 모듈 교체 (HMR)

## 템플릿 생성 방법

1.  **사용자에게 프로젝트 이름**과 대상 디렉터리 위치를 요청합니다.
2.  위에 표시된 대로 **디렉터리 구조를 생성**합니다.
3.  `templates/` 디렉터리에서 **템플릿 파일을 복사**합니다:
    - `templates/backend/`에서 백엔드 파일
    - `templates/frontend/`에서 프론트엔드 파일
    - `templates/README.md.template`에서 루트 README
4.  **`.template` 파일의 경우**: 복사할 때 `.template` 접미사를 제거합니다.
5.  **Tailwind/shadcn 자산이 포함되었는지 확인합니다**:
    - `tailwind.config.js`, `postcss.config.js`, `src/index.css`를 복사합니다.
    - `src/lib/utils.js`와 `src/components/ui` 디렉터리를 복사하여 shadcn 기본 요소가 즉시 사용 가능하도록 합니다.
6.  **필요에 따라 사용자 지정**: 요청 시 `package.json`의 프로젝트 이름을 업데이트합니다.
7.  **사용자에게 설정 지침을 제공**합니다:
    - 백엔드 설정 (`.env` 생성, 의존성 설치)
    - 프론트엔드 설정 (의존성 설치)
    - 두 서버 실행 방법
