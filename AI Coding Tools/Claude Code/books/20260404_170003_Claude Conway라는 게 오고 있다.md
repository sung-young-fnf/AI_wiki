# Claude Conway라는 게 오고 있다.

### 1. 원문 핵심 내용 분석 및 요약
원문은 Anthropic(앤스로픽)에서 개발 중인 것으로 알려진 **'Conway(컨웨이)'** 프로젝트를 통해 AI 생태계의 패러다임 변화를 경고하고 있습니다.

*   **Conway의 정체:** 단순한 대화형 AI(Claude)를 넘어, 시스템에 상주하며 스스로 판단하고 행동하는 **'상시 가동 에이전트(Always-on Agent)'**입니다.
*   **주요 기능:** 웹훅(Webhook)을 통한 외부 이벤트 수신, 브라우저 직접 조작, 전용 사이드바 UI, 독자적 확장 포맷(.cnw.zip) 등을 포함합니다.
*   **스타트업 위기:** 기존 AI 비서, 브라우저 자동화, 오케스트레이션 레이어 등을 서비스하던 스타트업들의 기능을 플랫폼(Anthropic)이 직접 흡수하면서 '셸프웨어(Shelfware)'가 될 위기에 처했습니다.
*   **변화된 속도전:** 과거와 달리 대기업(플랫폼사)이 AI를 활용해 직접 제품을 개발(Claude Code 등 활용)함에 따라, 스타트업의 유일한 무기였던 '속도'마저 위협받고 있습니다.

---

### 2. 관련 맥락 및 최신 동향 보충
원문에서 언급된 'Conway' 프로젝트와 관련된 업계의 흐름은 다음과 같습니다.

*   **에이전틱 워크플로우(Agentic Workflow):** 현재 AI 산업은 '더 똑똑한 모델'을 만드는 것에서 '일을 실제로 수행하는 에이전트'를 만드는 단계로 진화하고 있습니다. OpenAI의 '오퍼레이터(Operator)', 구글의 '자비스(Jarvis)' 프로젝트가 이와 궤를 같이합니다.
*   **Computer Use 기능의 확장:** Anthropic은 이미 '컴퓨터 사용(Computer Use)' 기능을 공개하여 AI가 마우스 클릭, 버튼 입력 등을 할 수 있음을 보여주었습니다. Conway는 이를 사용자 환경에 완전히 통합하여 백그라운드에서 실행되는 최종 형태일 가능성이 높습니다.
*   **수직적 통합:** 대규모 언어 모델(LLM) 제조사가 직접 애플리케이션 레이어(UI/UX)와 도구 연동 레이어까지 장악하려는 시도가 노골화되고 있습니다.

---

### 3. 원문에서 다루지 않은 비판적 관점 및 반론
원문은 플랫폼의 독주를 강조하고 있으나, 다음과 같은 반론과 고려 사항도 존재합니다.

*   **개인정보 및 보안 우려:** '상시 상주 에이전트'가 브라우저와 시스템 전체를 모니터링하는 것은 기업 입장에서는 심각한 보안 리스크입니다. 플랫폼사가 모든 데이터를 들여다보는 것에 대한 거부감이 스타트업의 '폐쇄형/온프레미스 에이전트' 수요를 창출할 수 있습니다.
*   **비용 및 효율성 문제:** 모든 사소한 이벤트(이메일 수신 등)에 고성능 LLM 에이전트가 깨어나 대응하는 것은 추론 비용 면에서 비효율적일 수 있습니다. 특정 목적에 특화된 가벼운 모델(SLM) 기반 스타트업이 가격 경쟁력을 가질 수 있습니다.
*   **플랫폼 종속성(Lock-in)에 대한 저항:** 기업들은 특정 플랫폼(Anthropic 혹은 OpenAI)에 지나치게 의존하는 것을 경계합니다. 멀티 모델을 지원하거나 중립적인 위치에서 에이전트를 관리하는 제3의 솔루션에 대한 니즈는 여전할 것입니다.

### 4. 핵심 인사이트
원문과 최신 동향을 종합한 핵심 인사이트는 다음과 같습니다.

1.  **'기능'이 아닌 '비즈니스'를 팔아야 한다:** 단순한 UI 래퍼(Wrapper)나 범용적인 자동화 도구는 플랫폼사에 의해 즉시 대체될 것입니다.
2.  **데이터의 해자(Moat) 구축:** 기술력은 상향 평준화되지만, 플랫폼사가 접근할 수 없는 **폐쇄적 도메인 데이터(규제 산업, 특정 기업 내부 프로세스)**를 확보한 곳만이 살아남습니다.
3.  **로컬 및 규제 대응:** 글로벌 플랫폼이 일일이 대응하기 힘든 국가별 특수 규제, 로컬 컴플라이언스(개인정보보호법 등)에 밀착 대응하는 것이 스타트업의 강력한 생존 전략입니다.
4.  **인프라 레이어의 기회:** 에이전트가 많아질수록 이들을 관리, 감시, 연결하는 '에이전트 인프라(Ops)' 시장이 커질 것이며, 이는 플랫폼과의 직접 경쟁을 피할 수 있는 좋은 대안입니다.

**결론:** AI 스타트업은 "우리가 만든 기능이 내일 아침 Claude나 ChatGPT의 기본 기능으로 탑재된다면 어떻게 할 것인가?"라는 질문에 즉각 답할 수 있어야 합니다.
