### Claude 코드의 새로운 모범 사례

저는 모든 모범 사례를 추출하고 개인적인 경험을 더하여 Claude 코드 모범 사례의 궁극적인 목록을 만들었습니다.

### 기초 팁

1.  **명확한 작업 프레이밍 (Clear Task Framing)** — Claude에게 무엇을 원하는지 가장 먼저 정확하게 명시하세요.
2.  **명령어 선행 배치 (Front Load Instructions)** — 항상 가장 중요한 명령어를 프롬프트 (prompt) 최상단에 두세요.
3.  **Claude가 작업을 확인할 방법을 제공하세요** — 테스트, 스크린샷 또는 예상 출력을 포함하여 Claude가 스스로 확인할 수 있도록 하세요. 이것이 가장 높은 효과를 낼 수 있는 단일 방법입니다.
4.  **프롬프트 구조 팁 (Prompt Structure Tip)** — 앞선 몇 가지 팁을 실용적으로 만들기 위해, 저는 다음 프롬프트 구조를 선호합니다: `[역할] + [작업] + [컨텍스트]`
5.  **크롬 확장 프로그램 팁 (Chrome Extension Tip)** — UI 변경 사항은 Claude 크롬 확장 프로그램을 사용하여 확인할 수 있습니다. 브라우저를 열고 UI를 테스트하며, 코드가 작동할 때까지 반복합니다.
6.  **먼저 탐색하고, 다음으로 계획하고, 그 다음 코딩하세요** — (다른 LLM을 포함할 수 있는) 조사를 한 다음, 계획 모드 (Plan Mode)에 진입하고, 다시 일반 모드로 전환하여 코드를 실행하세요.
7.  **프롬프트에 구체적인 컨텍스트 (context)를 제공하세요** — 명령이 정확할수록 더 좋습니다. Claude는 컨텍스트를 추론할 수만 있습니다.
8.  **컨텍스트가 없다고 가정 (Assume Zero Context)** — Claude가 프로젝트에 대해 아무것도 모른다고 가정하세요. 필요한 모든 것을 알려주세요.
9.  **풍부한 컨텍스트 (Rich Context)** — `@`를 사용하여 파일, 데이터, 이미지를 연결하세요.
10. **Claude.MD 팁** — `/init`을 실행하여 현재 프로젝트를 위한 기본 `CLAUDE.md` 파일을 생성하세요.

### 프로젝트 및 스킬 활용

11. **프로젝트 지침 (Project Instructions)** — 프롬프트를 반복하는 대신, 프로젝트 수준 지침을 사용하여 장기적인 행동을 정의하세요.
12. **프로젝트 메모리 (Project Memory)** — "메모리 (Memory)" 탭을 편집하여 Claude가 시간이 지남에 따라 무엇을 유지하거나 무시할지 정확하게 제어하세요 (이는 프로젝트에서도 작동합니다).
13. **Claude 스킬 (Claude Skills)** — 반복 가능한 워크플로우를 다시 프롬프트하는 대신 스킬로 전환하는 데 사용하세요.
14. **예시를 통한 스킬 생성 (Skill From Examples)** — 좋은 결과물을 붙여넣고 Claude에게 이를 재사용 가능한 스킬로 만들도록 요청하세요. 스크린샷을 업로드하고 Claude에게 복제하도록 요청하여 스킬로 전환할 수도 있습니다 (엘리트 스킬을 만드는 쉬운 방법).
15. **스킬 버전 관리 (Skill Versioning)** — 라이브 (live) 스킬을 편집하는 대신, 워크플로우를 개선하면서 스킬을 복제하고 버전을 만드세요.
16. **프로젝트 위생 (Project Hygiene)** — 드리프트 (drift)를 방지하기 위해 메모리, 파일, 지침을 정기적으로 정리하세요.
17. **프로젝트 컨텍스트 오염 (Project Context Bleed)** — 관련 없는 작업 스트림 (workstreams)은 컨텍스트 오염을 방지하기 위해 별도의 프로젝트로 분리하세요.
18. **Claude 스킬 레포 (Claude Skills Repo)** — 80,000개 이상의 Claude 스킬 라이브러리: https://skillsmp.com/
19. **Claude 스킬 라이브러리 (Claude Skills Library)** — 플러그 앤 플레이 스킬 등을 제공하는 멋진 웹사이트: https://mcpservers.org/claude-skills
20. **프로젝트 메모리 위치 (Project Memory Location)** — 프로젝트 메모리는 `./CLAUDE.md` 또는 `./.claude/CLAUDE.md`에 저장할 수 있습니다.

### 과소평가된 미니 팁 (대부분의 사람들이 모르는 내용)

21. **모델 스태킹 (Model Stacking)** — Claude 코드를 열기 전에 다른 LLM을 사용하여 프로젝트를 계획하고 고급 메가 프롬프트 (mega prompts)를 생성하세요. 이 전략은 계획 모드 (Plan Mode)의 토큰 (token) 사용량도 절약합니다.
22. **맞춤형 서브에이전트 (custom subagents) 생성** — `.claude/agents/`에 전문화된 비서를 정의하여 Claude가 개별 작업 (isolated tasks)을 위임할 수 있도록 하세요.
23. **출력 점수 매기기 (Output Scoring)** — Claude에게 미리 정의된 성공 기준에 따라 답변에 점수를 매기도록 요청하세요.
24. **플러그인 (Plug-ins) 설치** — `/plugin`을 실행하여 마켓플레이스를 탐색하세요. 플러그인은 구성 없이 스킬, 도구 및 통합 기능을 추가합니다.
25. **Claude 코드 내에서 배우는 Claude 코드 (Claude Code taught IN Claude Code)** — Claude 코드 내에서 직접 Claude 코드를 가르치는 강좌: https://ccforeveryone.com/
26. **Claude 인터뷰 (Claude Interviews)** — 더 큰 프로젝트의 경우, Claude가 먼저 여러분을 인터뷰하도록 하세요. 최소한의 프롬프트로 시작하여 Claude에게 `AskUserQuestion` 도구를 사용하여 여러분을 인터뷰하도록 요청하세요.
27. **자주 수정 (Correct Often)** — Claude를 자주 수정하세요. 잘못된 방향으로 가기 시작하면 즉시 중단하세요 (ESC 키로 Claude의 동작을 중단).
28. **지우기 (Clear)** — `/clear`를 실행하여 깨끗한 세션을 시작하세요.
29. **되감기 (Rewind)** — ESC 키를 두 번 누르거나 `/rewind`를 입력하여 체크포인트 메뉴를 여세요.
30. **여러 세션 실행 (Run Multiple Sessions)** — 병렬 세션을 실행하는 두 가지 주요 방법이 있습니다:
    *   **Claude 데스크톱:** 여러 로컬 세션을 시각적으로 관리합니다. 각 세션은 자체 격리된 워크트리 (worktree)를 가집니다.
    *   **Claude 웹:** 격리된 VM (Virtual Machine)에서 Anthropic의 보안 클라우드 인프라에서 실행됩니다.

### 디버깅, 오류 처리, 일반적인 실패 패턴

31. **단계 격리 (Step Isolation)** — 모든 것을 다시 생성하는 대신, 실패한 단계만 다시 실행하세요.
32. **오류 재현 (Error Reproduction)** — Claude에게 오류를 의도적으로 재현하여 이해하도록 요청하세요.
33. **프롬프트 롤백 (Rollback Prompts)** — 마지막으로 잘 작동했던 프롬프트로 되돌리고 변경 사항을 하나씩 다시 적용하세요.
34. **과도하게 명시된 CLAUDE.md (Over-Specified CLAUDE.md)** — `CLAUDE.md`가 너무 길면, Claude는 중요한 규칙이 소음에 묻혀 절반을 무시합니다.
    *   **해결책:** 가차 없이 정리하세요. Claude가 지시 없이도 이미 정확하게 무언가를 수행한다면, 해당 지시를 삭제하거나 훅 (hook)으로 변환하세요.
35. **이 실수를 저지르지 마세요** — 하나의 작업으로 시작했다가, Claude에게 관련 없는 질문을 한 다음, 다시 첫 번째 작업으로 돌아갑니다. 컨텍스트가 관련 없는 정보로 가득 차게 됩니다.
    *   **해결책:** 관련 없는 작업 사이에는 `/clear`를 사용하세요.
36. **과도한 수정 (Over-Correcting)** — Claude가 뭔가 잘못하면 수정하고, 여전히 잘못되면 다시 수정합니다. 컨텍스트가 실패한 접근 방식으로 오염됩니다.
    *   **해결책:** 두 번의 수정 시도가 실패한 후에는 `/clear`를 사용하고, 학습한 내용을 반영하여 더 나은 초기 프롬프트를 작성하세요.
37. **단계별 재생 (Step-by-Step Replay)** — Claude에게 답변을 어떻게 생성했는지 한 줄씩 설명하도록 하세요.
38. **무한 탐색 (The Infinite Exploration)** — 범위를 지정하지 않고 Claude에게 무언가를 '조사'하도록 요청합니다. Claude는 수백 개의 파일을 읽어 컨텍스트를 채웁니다.
    *   **해결책:** 조사의 범위를 좁히거나 서브에이전트 (subagents)를 사용하여 탐색이 메인 컨텍스트를 소모하지 않도록 하세요.
39. **디버깅 프로젝트 (Debugging Project)** — 코드 디버깅 전용 AI 프로젝트를 만드세요 (Grok 4 Heavy는 디버깅에 능숙합니다).
40. **컨텍스트 윈도우 관리 (Context Window Management)** — Claude의 컨텍스트 윈도우 (context window)는 빠르게 채워집니다. 이로 인해 Claude는 이전 지시 사항을 잊을 수 있습니다. 이 페이지가 그 문제를 해결하는 데 도움이 될 것입니다: https://code.claude.com/docs/en/costs#reduce-token-usage

### 최종 팁

41. **노션 데이터베이스 (Notion Database)** — 노션 데이터베이스를 Claude에 연결하여 가장 좋고 가장 자주 사용하는 프롬프트 (prompt)를 저장하세요.
42. **실전에서 Claude 코드 배우기 (Learn Claude Code in Action)** — Anthropic의 학습 자료: https://www.anthropic.com/learn
43. **Claude 강좌 (Claude Courses)** — Coursera의 강좌: https://www.anthropic.com/learn
44. **Boris의 설정 (Boris’ Set Up)** — Claude 코드의 개발자가 Claude 코드를 극대화하는 방법: Boris’ Claude Code Setup Cheatsheet
45. **Claude 코드 모범 사례 (DOC)** — 최신 문서 링크: https://code.claude.com/docs/en/best-practices
46. **안전 자율 모드 (Safe Autonomous Mode)** — `claude --dangerously-skip-permissions`를 사용하여 모든 권한 확인을 우회하고 Claude가 중단 없이 작동하도록 하세요. 이는 린트 오류 (lint errors) 수정 또는 상용구 코드 (boilerplate code) 생성과 같은 워크플로우에 잘 작동합니다.
47. **천천히 꾸준하게 (Slow & Steady)** — 시간을 가지세요. 특히 중요한 워크플로우를 구축할 때는 더욱 그렇습니다. 계획하고, 계획하고, 계획한 다음 실행하세요.
48. **Claude 슈퍼파워 (Claude Superpowers)** — Claude 코드 슈퍼파워 GitHub 리포지토리: https://github.com/obra/superpowers
49. **훅 (Hooks)** — 예외 없이 항상 발생해야 하는 동작에 가장 적합합니다.
50. **Claude 코드를 확장하는 방법 (How to Extend Claude Code)** — Anthropic 가이드: https://code.claude.com/docs/en/features-overview
