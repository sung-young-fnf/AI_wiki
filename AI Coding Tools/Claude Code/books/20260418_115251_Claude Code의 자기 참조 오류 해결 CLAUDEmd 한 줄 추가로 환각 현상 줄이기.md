# Claude Code의 자기 참조 오류 해결: CLAUDE.md 한 줄 추가로 '환각' 현상 줄이기

아마 많은 분이 비슷한 경험을 해보셨을 겁니다. Claude Code에 새로운 기능이 추가되어 사용해보고자 해당 기능에 대해 질문하면, Claude Code는 자신 있게 답변합니다.

하지만 그 답변은 틀린 경우가 많습니다. 실제 존재하는 기능이 없다고 하거나, 없는 옵션을 알려주고, 존재하지 않는 설정 방식을 안내하며, 심지어 그럴듯한 명령어를 지어내기도 합니다.

이러한 현상이 발생하는 이유는 Claude Code를 구동하는 모델의 학습 데이터에 마감 시점(cutoff date)이 있어, 그 이후에 추가된 기능에 대해서는 알지 못하기 때문입니다. 그 결과, 실제 존재하는 기능을 없다고 하거나 혼란스러운 '환각(hallucination)' 현상을 보입니다.

Claude Code의 새로운 기능은 빠르게 출시되지만, 이를 사용하는 방법을 Claude Code에게 직접 물어보면 잘못된 정보(hallucination)를 제공하는 경우가 많습니다. 이로 인해 새로운 기능을 설정하는 과정은 늘 '환각'과의 싸움이 되고, 결국 공식 문서를 찾아보는 번거로움으로 이어집니다.

---

그런데 이 문제는 놀라울 정도로 간단하게 해결할 수 있습니다.
Claude Code 내부에는 `claude-code-guide`라는 내장 서브 에이전트(sub-agent)가 존재합니다. 이는 레고 상자에 비유하자면, 일종의 부품 설명서와 같은 역할을 합니다.

해야 할 일은 단 하나입니다. CLAUDE.md 파일에 다음 규칙을 추가하는 것입니다.

```
"Claude Code 자체에 대한 질문에는 직접 답변하지 말고, 반드시 claude-code-guide 에이전트에게 위임하여 문서 기반으로 조사한 결과를 전달할 것."
```

이전에 겪었던 문제들로 인해 많은 스트레스를 받다가, 이 한 줄을 추가한 결과 Claude Code가 자신에 대해 잘못된 정보를 제공하는 현상이 현저히 줄어들었습니다.

이 글의 링크를 공유하거나 내용을 캡처하여 Claude Code에 '나도 이렇게 설정해 줘'라고 요청하면, 바로 반영될 것입니다.

PS. 만약 개인의 Claude.md 파일이 너무 커서 관련 내용이 묻힐 우려가 있다면, 해당 규칙의 위치를 파일 상단으로 옮기거나 문장을 강조하여 사용하면 강제성을 더 높일 수 있습니다. (다만, 후킹(hooking) 방식은 권장하지 않습니다.)
