# [신규] 클로드 코드(Claude Code) 텔레그램 연동 가이드: 어디서든 실행하는 방법

텔레그램(Telegram) 채널은 휴대전화에서 클로드 코드(Claude Code) 세션을 직접 실행하고 모니터링할 수 있는 새로운 네이티브 방식입니다. 이 가이드에서는 텔레그램을 클로드 코드에 연결하여 모바일로 코딩 작업을 전송하고 답변을 받는 과정을 단계별로 설명합니다.

### 준비 사항
설정을 시작하기 전에 다음 조건이 충족되었는지 확인하세요.
*   클로드 코드(Claude Code) v2.1.80 이상 버전
*   번(Bun) 설치 완료
*   텔레그램 계정

---

### 1단계: 텔레그램 봇 생성하기

텔레그램의 공식 봇 관리 도구인 **봇파더(@BotFather)**를 통해 봇을 생성합니다.

1.  **봇파더 시작:** 텔레그램에서 `@BotFather`를 검색해 채팅을 시작하고 `/newbot` 메시지를 보냅니다.
2.  **봇 이름 설정:** 채팅창 상단에 표시될 이름을 설정합니다 (예: Claude Code Agent).
3.  **사용자 이름(Username) 설정:** `bot`으로 끝나는 고유한 핸들을 설정합니다 (예: `my_claude_code_bot`). 생성 후 `t.me/봇이름` 형태의 공개 링크가 제공됩니다.
4.  **토큰(Token) 복사:** 봇파더가 전송한 API 토큰(`123456789:AAHfi...` 형태)을 안전한 곳에 복사해 둡니다.

---

### 2단계: 플러그인 설치 및 구성

이제 생성한 봇을 클로드 코드에 연결합니다.

1.  **플러그인 설치:** 프로젝트 폴더에서 클로드 코드 세션을 실행한 후 다음 명령어를 입력합니다.
    ```bash
    /plugin install telegram@claude-plugins-official
    ```
    설치가 완료되면 `/reload-plugins`를 실행하여 활성화합니다.

2.  **토큰 설정:** 클로드 코드 세션 내에서 다음 명령어로 토큰을 저장합니다.
    ```bash
    /telegram:configure <본인의-봇-토큰>
    ```
    이 설정은 프로젝트 디렉토리 내 `.claude/channels/telegram/.env` 파일에 저장됩니다.

3.  **채널 플래그와 함께 재시작:** 플러그인을 연결하려면 `--channels` 플래그를 사용해야 합니다. 기존 세션을 종료(`/exit`)하고 다음과 같이 재시작하세요.
    ```bash
    claude --channels plugin:telegram@claude-plugins-official
    ```

---

### 3단계: 페어링(Pairing) 및 보안 설정

봇이 활성화되면 텔레그램 계정을 클로드 코드와 페어링해야 합니다.

1.  **페어링 코드 생성:** 텔레그램에서 생성한 봇에게 "hello"와 같은 메시지를 보냅니다. 봇이 6자리의 페어링 코드를 답장으로 보낼 것입니다.
2.  **계정 연결:** 클로드 코드 터미널에서 다음 명령어를 실행합니다.
    ```bash
    /telegram:access pair <6자리-코드>
    ```
3.  **접근 권한 제한:** 보안을 위해 본인만 봇을 사용할 수 있도록 허용 목록(allowlist) 모드로 전환합니다.
    ```bash
    /telegram:access policy allowlist
    ```

---

### 4단계: 실전 테스트

이제 텔레그램에서 코딩 작업을 요청해 보세요. 예를 들어, "현재 디렉토리의 모든 파일을 나열하고 이 프로젝트가 무엇인지 설명해 줘"라고 메시지를 보내면 클로드가 요청을 처리하고 결과를 텔레그램으로 전송합니다.

더 복잡한 작업(예: 데이터베이스 연결 스크립트 작성 등)도 가능하며, 클로드가 파일을 생성하면 프로젝트 폴더에 즉시 반영됩니다.

---

### 주요 기능 및 도구

*   **사진 전송:** 이미지 파일을 봇에게 보내면 클로드가 이를 분석할 수 있습니다.
*   **입력 중 표시:** 클로드가 응답을 생성하는 동안 텔레그램에 "입력 중..." 상태가 표시됩니다.
*   **활용 도구(Tools):**
    *   `reply`: 메시지 및 파일 첨부(최대 50MB) 전송
    *   `react`: 메시지에 이모지 반응 추가
    *   `edit_message`: 이전에 보낸 메시지 수정 (진행 상황 업데이트 시 유용)

---

### 문제 해결(Troubleshooting)

1.  **봇이 응답하지 않음:** 클로드 코드가 `--channels` 플래그와 함께 실행 중인지, 토큰이 정확한지 확인하세요.
2.  **인증 충돌(Auth conflict):** API 키와 로그인 세션이 충돌할 경우 `/logout`을 실행하여 기존 로그인을 정리한 후 재시도하세요. 채널 기능은 클로드 API 키가 아닌 claude.ai 계정 로그인이 필요합니다.
3.  **권한 요청 대기:** 클로드가 작업 중 권한 승인을 요청하면 터미널에서 직접 승인할 때까지 세션이 일시 중지됩니다. 신뢰할 수 있는 환경이라면 `--dangerously-skip-permissions` 플래그를 사용할 수 있습니다.
4.  **토큰 유출 시:** 보안 사고가 의심된다면 봇파더에게 `/revoke` 명령어를 보내 새 토큰을 발급받으세요.
