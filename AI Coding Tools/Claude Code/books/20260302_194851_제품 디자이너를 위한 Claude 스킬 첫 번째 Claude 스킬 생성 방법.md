제품 디자인은 전적으로 프로세스에 관한 것입니다. 워크플로(workflow)가 잘 정리될수록 결과는 더욱 예측 가능해집니다. Claude Skills는 반복적인 사고와 분석을 재사용 가능하고 신뢰할 수 있는 워크플로로 전환하여, 단일 프롬프트(one-off prompts)를 넘어 그 영향력을 확장할 수 있도록 돕는 강력한 도구입니다.

이 글에서는 제품 디자이너가 Claude Skills를 활용하여 사용자 리서치(user research) 중 사용자 인터뷰 스크립트(transcripts) 분석과 같은 일반적인 워크플로를 자동화하고 표준화하는 방법을 보여드리겠습니다.

여기서는 Claude Code를 설정하고, 첫 번째 스킬(skill)을 생성하며, 결과물을 정교화하고, 실제 연구 데이터로 테스트하는 방법을 배울 것입니다. 엔지니어링 배경은 필요 없으며, 터미널(Terminal) 사용에 대한 기본적인 편의성과 원하는 결과에 대한 명확한 아이디어만 있으면 됩니다.

## Claude 환경 설정

클로드(Claude)에 접근하는 방법은 여러 가지가 있지만, 이 글에서는 Claude Code 환경을 사용하기로 했습니다. 실제 설정은 몇 분 안에 완료할 수 있는 두 단계 프로세스입니다.

### 단계 1: 로컬 머신에 Claude Code 설치

가장 먼저 할 일은 제 컴퓨터에 Claude Code 셸(shell)을 설치하는 것입니다. 저는 macOS를 사용하고 있으므로 터미널(Terminal)을 열고 다음 명령어를 입력하겠습니다:

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

설치 과정은 시간이 다소 걸리므로, 터미널(Terminal) 창에서 아무런 진행 상황이 보이지 않더라도 당황하지 마십시오.

[macOS 터미널 창 이미지]

설치 과정이 완료되면 다음 메시지가 표시될 것입니다. 이는 Claude Code 셸(shell)을 실행할 수 있다는 의미입니다.

[Claude Code 설치 성공 이미지]

**참고**: 보시다시피, 제 경우 설치 시 기본적으로 PATH에 포함되지 않았습니다. 이는 터미널(Terminal)에서 `claude --help`를 실행하면 "Claude not found."라는 오류 메시지가 표시될 것이라는 의미입니다. 따라서 PATH에 추가하기 위해 다음 명령어를 입력해야 했습니다:

```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc
```

## 첫 번째 스킬(skill) 생성 방법

이제 사용자 인터뷰 스크립트(transcripts)에서 가치 있는 통찰력을 추출하는 스킬(skill)을 만들어 봅시다. 간단히 말해, 모든 스킬은 클로드(Claude)가 특정 작업을 완료할 때 사용하는 마크다운(Markdown, .md) 파일에 작성된 명령어 목록입니다. 따라서 모든 스킬을 저장할 디렉터리(directory)를 생성해야 합니다.

### 단계 1: 스크립트를 위한 디렉터리(directory) 및 파일 생성

다음은 인터뷰 스크립트를 분석하기 위한 스크립트를 생성하기 위해 입력해야 하는 명령어입니다:

```bash
mkdir -p ~/.claude/skills/interview-insights
```

이 구조를 통해 모든 스킬을 `/skills/`라는 상위 디렉터리(parent directory)에 저장하게 됩니다.

### 단계 2: SKILL.md 작성

모든 스킬은 두 부분으로 구성된 `SKILL.md` 파일이 필요합니다:

*   YAML 형식의 프론트매터(Frontmatter, `---` 마커 사이). 이는 클로드(Claude)에게 언제 해당 스킬을 사용할지 알려줍니다.
*   스킬이 호출될 때 클로드(Claude)가 따를 지침이 포함된 마크다운(Markdown) 내용.

[interview-insights 스킬을 위한 디렉터리 생성 이미지]

이 명령어를 제출하면 (그리고 제가 명령어에서 파일 경로 앞에 `touch`를 사용하지 않았다는 점에 유의하세요), 클로드(Claude)는 .md 파일을 확인하고 존재하지 않는지 확인합니다. 그래서 새 .md 파일을 생성할 것인지 물을 것입니다. 사용자가 이를 확인하면 클로드(Claude)는 새로운 스킬을 생성해 줄 것입니다.

[SKILL.md 파일 생성 이미지]

보시다시피, 클로드(Claude)는 우리를 위해 전체 스킬을 생성했습니다. 특히 흥미로운 점은 스킬 이름(`interview-insights`)을 컨텍스트(context)로 사용하여 스킬 내용을 작성했다는 것입니다.

**참고**: `SKILL.md` 파일은 작업 공간(workspace) 내 `/.claude` 디렉터리(클로드(Claude)가 2단계에서 접근을 요청한 곳)에 생성되었지만, 실제 디렉터리는 시작 부분에 점(.) 기호가 있습니다. Mac에서는 점(.)으로 시작하는 모든 파일이나 디렉터리(폴더)는 시스템에 의해 숨김 항목으로 간주됩니다.

### 단계 3: SKILL.md 파일 검토 및 정교화

스킬 파일을 열면 다음 내용이 보일 것입니다:

[폴더 이름 'interview-insights'로 Claude가 생성한 SKILL.md 파일 이미지]

클로드(Claude)는 인터뷰 스크립트(transcripts)를 처리하고 구조화된 실행 가능한 통찰력을 추출하는 사용자 리서치(user research) 분석 스킬을 생성했습니다. 보시다시피, 파일은 명확한 계층적 구조(`#`, `##`)를 가지고 있어 쉽게 훑어볼 수 있습니다.

파일 내용은 매우 직관적이며, 이 스킬은 사용자 리서치(user research)에서 사용할 수 있는 실제 스킬의 훌륭한 초안입니다. 하지만 결과물에 대해 한 가지 주목할 만한 점을 강조하고 싶습니다. 이 스킬은 체계적으로 다음을 추출합니다:

*   테마 및 패턴(Themes and patterns)
*   정확한 인용문(Verbatim quotes)
*   문제점(Pain points)
*   사용자 요구 및 목표(User needs and goals)
*   예상치 못한 발견(Unexpected discoveries)

결과물을 조직의 요구에 맞게 조정하는 것은 항상 좋은 생각입니다. 예를 들어, 제 경우 저는 사람이 경험하는 감정 또한 강조하고 싶습니다. 이 지침을 스킬에 추가하려면, 원본 `SKILL.md` 파일을 변경하거나 (선호하는 편집기에서 간단히 편집) 클로드(Claude)에게 스킬 수정을 요청해야 합니다.

제 경우, 두 번째 방법을 따르고 Claude Code가 실행 중인 터미널(Terminal) 창에 다음을 입력하겠습니다:

```
modify the skill - make sure you highlight emotions that the interviewee is experiences
```

[Claude에게 SKILL.md의 로컬 경로를 제공하도록 요청하는 이미지]

프롬프트(prompt)를 제출하면, 클로드(Claude)는 원본 `SKILL.md` 파일에 대한 수정을 제안할 것입니다. 이는 무엇이 추가되거나 제거될지 확인할 수 있는 좋은 기회입니다.

[Claude에게 스킬 수정을 요청하는 이미지]

이 단계에서는 파일이 아직 수정되지 않았습니다. 하지만 변경 사항을 수락하면, 클로드(Claude)는 실제 파일에 편집 내용을 적용할 것입니다 (파일의 24번째 줄과 40번째 줄 참조).

[Claude에 의해 수정된 SKILL.md 파일 이미지]

### 단계 4: 스킬 테스트

마지막으로, 스킬을 테스트할 준비가 되었습니다. 이를 위해 `SKILL.md`에 인터뷰 스크립트(transcript)를 제공하여 마법을 부릴 수 있도록 해야 합니다.

**빠른 팁**: 만약 저처럼 긴 명령어를 작성하는 것을 좋아하지 않는다면, 클로드(Claude)에게 명령어를 대신 작성해 달라고 요청할 수 있습니다. 예를 들어, 제가 클로드(Claude)에게 요청한 내용은 다음과 같습니다:

```
how to launch this skill with my interview transcript
```

[Claude에게 스킬 실행 방법을 묻는 이미지]

클로드(Claude)는 전체 작업 폴더(work folder)에 접근할 수 있으므로, 스크립트(transcript)의 로컬 경로를 간단히 제공할 수 있습니다.

**참고**: 새로운 스킬을 처음 사용할 때 클로드(Claude)가 "Unknown skill."이라는 오류 메시지를 표시할 수 있습니다. 이 오류 메시지가 나타나면 스킬이 등록되었는지 확인하고 (`SKILL.md` 파일의 YAML 프론트매터(frontmatter)를 확인) 클로드(Claude)를 다시 시작하십시오.

[새 스킬 실행 시 흔히 발생하는 '스킬을 찾을 수 없습니다' 오류 이미지. YAML 프론트매터 유효성 확인 및 Claude 재시작 필요.]

제 경우, 터미널(Terminal) 창에 `exit`를 입력한 다음 `claude`를 입력하자 마침내 스킬을 실행할 수 있었습니다.

**재미있는 사실**: 제가 스크립트(transcript)의 파일 경로를 잘못 입력했지만 ("Downloads" 대신 "Dowloads"로 오타), 클로드(Claude)가 이를 감지하고 수정할 수 있었습니다. 그래서 결국, 제 스크립트의 멋진 요약을 제공해 주었습니다.

[Claude가 스킬을 사용하여 생성한 결과물 이미지]
