# Claude Code의 /btw, /fork, /rewind 마스터하기: 컨텍스트 위생 도구 모음

Claude Code를 사용하는 개발자라면 누구나 한 번쯤 겪는 문제가 있습니다. AI가 복잡한 모듈을 리팩토링하는 동안 궁금한 점이 생겨 질문을 던지면, Claude는 답변을 마친 후 다시 작업을 재개하려 하지만 이미 컨텍스트 창(Context window)은 작업과 무관한 수백 개의 문답 토큰으로 오염되어 버립니다.

이러한 **컨텍스트 오염(Context Pollution)**은 AI 보조 코딩의 생산성을 떨어뜨리는 보이지 않는 범인입니다. 2026년 3월 10일 출시된 Claude Code v2.1.72 버전은 이를 근본적으로 해결하기 위해 `/btw` 명령어를 도입했습니다. 기존의 `/fork`, `/rewind`와 함께 이 도구들은 AI 협업에서 가장 간과되기 쉬운 '컨텍스트 위생(Context hygiene)'을 완벽하게 관리할 수 있게 해줍니다.

---

### 컨텍스트 최적화가 중요한 이유

LLM의 컨텍스트 창은 회의실의 화이트보드와 같습니다. 주제와 무관한 내용이 화이트보드의 절반을 차지하면 실제 작업을 위한 공간이 줄어들 뿐만 아니라, 모델이 답변을 생성할 때 노이즈의 영향을 받아 출력 품질이 저하됩니다.

1. **토큰 예산 감소:** 불필요한 문답이 파일 내용이나 코드 분석을 담아야 할 공간을 소모합니다.
2. **신호 약화(Signal Degradation):** 모델이 노이즈를 실제 신호와 함께 처리하면서 작업의 정확도가 떨어집니다.

---

### 1. `/btw` 명령어: 일시적인 사이드 채널

`/btw`('by the way'의 약자)는 작업 도중 발생하는 가벼운 질문으로 인한 컨텍스트 오염을 방지합니다.

*   **작동 방식:** `/btw` 뒤에 질문을 입력하면 일시적인 에이전트가 생성되어 메인 세션의 프롬프트 캐시(Prompt cache)를 읽고 답변한 뒤 즉시 사라집니다. 이 문답은 메인 대화 기록에 남지 않습니다.
*   **활용 예시:** 특정 라이브러리를 왜 선택했는지, 혹은 특정 패턴이 어떻게 작동하는지 묻고 싶을 때 사용합니다. 메인 작업 흐름을 방해하지 않고 궁금증만 해결할 수 있습니다.
*   **장점:** 메인 세션의 캐시를 활용하므로 비용이 매우 저렴하며, 컨텍스트를 깨끗하게 유지합니다.
*   **제약:** 단발성 질문만 가능하며, 파일을 수정하거나 도구(Tool)를 사용할 수는 없습니다.

### 2. `/fork` 명령어: 깊이 있는 탐색을 위한 병렬 세션

질문 이상의 작업, 즉 추가 파일 읽기, 다이어그램 생성, 대안 프로토타입 제작 등이 필요할 때는 `/fork`를 사용합니다.

*   **작동 방식:** 현재까지의 대화 기록을 모두 상속받은 새로운 세션을 생성합니다. 원본 세션은 그대로 유지됩니다.
*   **활용 예시:** 현재 구현 중인 WebSocket 방식 대신 SSE(Server-Sent Events) 방식을 테스트해보고 싶을 때, 세션을 포크(Fork)하여 대안을 탐색한 뒤 결과만 비교할 수 있습니다.
*   **장점:** 처음부터 다시 설명할 필요 없이 기존의 모든 결정 사항과 맥락을 유지한 채로 새로운 실험을 할 수 있습니다.

### 3. `/rewind` 명령어: 정밀한 상태 복구

Claude가 잘못된 방향으로 작업을 진행했을 때 사용하는 도구입니다.

*   **작동 방식:** Claude가 변경을 수행할 때마다 생성되는 체크포인트로 되돌아갑니다.
    *   **Code only:** 코드는 되돌리고 대화는 유지
    *   **Conversation only:** 대화는 되돌리고 코드는 유지
    *   **Both:** 코드와 대화 모두 이전 상태로 복구
*   **활용 예시:** Claude가 요구사항을 잘못 이해했을 때, 말로 설명하며 수정하는 대신 `/rewind`로 에러가 발생하기 직전으로 돌아가 더 명확한 지시를 내리는 것이 컨텍스트 효율 면에서 훨씬 유리합니다.
*   **컨텍스트 요약(Summarization):** 세션이 너무 길어지면 `/rewind summarization`을 통해 지금까지의 주요 결정 사항만 요약하고 컨텍스트 창을 비워 신선한 상태로 작업을 이어갈 수 있습니다.

### 도구 선택 가이드라인

작업 중 어떤 도구를 사용할지 결정할 때 다음 흐름을 참고하세요.

| 상황 | 추천 도구 | 특징 |
| :--- | :--- | :--- |
| **가벼운 질문/확인** | `/btw` | 비용 매우 저렴, 컨텍스트 영향 제로 |
| **파일 읽기/도구 사용 필요** | `/fork` | 별도 세션에서 독립적 작업 수행 |
| **대안적 접근법 탐색** | `/fork` | 기존 맥락 유지하며 병렬 실험 |
| **잘못된 방향 수정** | `/rewind` | 오염된 문맥을 삭제하고 깨끗하게 복구 |
| **컨텍스트 창 포화** | `/rewind summarization` | 기록 압축 및 여유 공간 확보 |

---

### 고급 워크플로우 패턴

1. **모니터링-수정 루프 (Monitor-Correct Loop):** 복잡한 작업을 시킨 뒤 `/btw`로 진행 상황을 주기적으로 확인합니다. 만약 잘못된 방향으로 가고 있다면 즉시 `/rewind`로 되돌리고 지시를 수정합니다.
2. **탐색 브랜치 (Exploration Branch):** 중요한 아키텍처 결정을 내리기 전 `/fork`를 통해 여러 대안을 실제로 코딩해보고 가장 좋은 방안을 메인 세션에 적용합니다.
3. **단계별 압축 (Context Compression Pipeline):** 프로젝트의 한 단계(Phase)가 끝나면 요약 기능을 사용하여 완료된 작업의 세부 토큰을 비우고 다음 단계로 넘어갑니다.

### 요약

Claude Code를 가장 잘 활용하는 개발자는 단순히 프롬프트를 잘 쓰는 사람이 아니라, **컨텍스트 창을 희소한 자원으로 관리**하는 사람입니다. 

*   질문은 `/btw`로 처리하여 오염을 방지하세요.
*   탐색은 `/fork`로 격리하여 수행하세요.
*   실수는 `/rewind`로 깨끗하게 도려내세요.

컨텍스트 위생을 철저히 관리할수록 Claude의 답변 품질과 작업 속도는 비약적으로 향상될 것입니다.
