# 제가 (새로운) Claude Code 원격 제어 기능을 사용해 본 후기 (시간 낭비하기 전에 읽어보세요)

*Joe Njenga 작성*
*팔로잉 중*
*읽는 데 12분 소요*

새로운 Claude Code의 원격 제어(Remote Control) 기능은 OpenClaw와 동일한 편의성을 제공하면서도 모든 것을 사용자의 로컬 머신에서 실행합니다.

이는 로컬 개발 환경의 강력함을 희생하지 않고 어떤 기기에서든 원격 코딩 세션을 시작할 수 있도록 돕는 훌륭한 시도입니다.

하지만 우선, 왜 이 기능을 OpenClaw와 비교할까요?

OpenClaw는 사용자의 기기에서 실행되며 WhatsApp, Telegram, Slack, iMessage와 같은 채널을 통해 연결되는 개인 AI 어시스턴트입니다.

*Medium 유료 회원이 아니더라도 여기에서 이 기사를 무료로 읽을 수 있지만, 저의 작업을 지원하기 위해 Medium 가입을 고려해 주십시오. 감사합니다!*

OpenClaw는 Claude Code를 원격으로 트리거하여 "어디서든 코딩 시작" 기능을 제공할 수 있습니다.

원격 제어(Remote Control) 기능도 비슷한 일을 하지만, 핵심적인 차이점이 있습니다.

OpenClaw처럼 외부 채널을 통해 라우팅하는 대신, 원격 제어는 사용자의 브라우저나 휴대폰을 로컬 Claude Code 세션에 직접 연결합니다.

사용자의 노트북이 제어를 유지하고, MCP 서버가 활성 상태를 유지하며, 프로젝트 컨텍스트는 사용자의 머신을 떠나지 않습니다.

실제로 어떻게 작동하는지 보여드리겠습니다.

노트북에서 FastAPI 백엔드와 바닐라 JavaScript 프런트엔드를 구축하고 있다고 상상해 보세요. 인증 흐름(authentication flow)을 구현하던 중 잠시 자리를 비워야 합니다.

작업을 중단하거나 컨텍스트를 잃는 대신, 휴대폰을 들고 터미널에서 QR 코드를 스캔한 다음 중단했던 부분부터 정확히 이어서 작업할 수 있습니다.

세션은 노트북에서 계속 활성화되어 있고, MCP 서버는 계속 실행되며, 프로젝트 구성은 그대로 유지됩니다.

간단히 말해, Claude Code 원격 제어는 다른 기기에서 로컬 개발 환경을 제어할 수 있는 기능을 제공합니다.

하지만 이는 Anthropic의 클라우드 인프라(cloud infrastructure)에서 실행되는 웹상의 Claude Code와는 다릅니다. 원격 제어는 사용자의 머신에서 실행됩니다.

이 글에서는 원격 제어가 어떻게 작동하는지, 단계별 설정 방법, 그리고 사용 사례를 설명해 드리겠습니다.

하지만 기본적인 것부터 시작하겠습니다.

## Claude Code 원격 제어란 무엇인가요?

원격 제어(Remote Control)는 클라우드 서비스가 아니라, 사용자의 로컬 Claude Code 세션과 이를 제어하려는 모든 기기 간의 다리 역할을 합니다.

사용자의 노트북은 책상에서 Claude Code를 실행하고 있습니다. 원격 제어는 휴대폰, 태블릿 또는 다른 브라우저가 로컬 세션을 제어할 수 있도록 안전한 연결을 생성합니다.

하지만 모든 작업은 여전히 사용자의 머신에서 이루어집니다.

### 세 가지 연결 방법

원격 제어 세션을 시작하면 세 가지 연결 옵션을 얻게 됩니다.

*   **세션 URL(Session URL)**: 어떤 브라우저에서든 열 수 있는 직접 링크입니다. 복사하여 다른 기기에 붙여넣으면 로컬 세션에 연결됩니다.
*   **QR 코드(QR Code)**: 터미널에서 스페이스바를 누르면 QR 코드가 표시됩니다. 휴대폰의 Claude 앱으로 스캔하면 즉시 연결됩니다.
*   **세션 목록(Session List)**: claude.ai/code 또는 Claude 모바일 앱을 열면 원격 제어 세션이 컴퓨터 아이콘과 녹색 상태 표시등과 함께 목록에 나타납니다.

세 가지 방법 모두 사용자의 머신에서 실행되는 동일한 로컬 세션에 연결됩니다.

## 원격 제어 Claude Code 대 웹상의 Claude Code

웹상의 Claude Code는 Anthropic의 클라우드 인프라에서 실행됩니다. 로컬에 아무것도 설치할 필요가 없습니다.

빠른 작업이나 복제(clone)하지 않은 프로젝트에서 작업하기에 좋습니다.

원격 제어는 사용자의 머신에서 실행됩니다. 웹 또는 모바일 인터페이스는 로컬 세션으로 들어가는 창(window)일 뿐입니다.

*   사용자의 로컬 MCP 서버는 계속 액세스 가능합니다. Context7, GitHub CLI 또는 사용자 정의 데이터베이스 커넥터(custom database connectors)를 구성한 경우 원격으로 연결해도 동일하게 작동합니다.
*   사용자의 프로젝트 구성은 계속 활성 상태를 유지합니다. 모든 CLAUDE.md 파일, 사용자 정의 명령(custom commands), 워크스페이스 설정(workspace settings)이 유효합니다.
*   파일 시스템(filesystem)은 완전히 사용 가능합니다. Claude는 사용자가 책상에 앉아 있을 때처럼 로컬 프로젝트의 파일을 읽고, 편집하고, 생성할 수 있습니다.
*   사용자의 도구와 권한은 유지됩니다. 프로젝트에 구성한 모든 도구는 다른 기기에서 연결할 때도 구성된 상태를 유지합니다.

원격 제어는 작업을 클라우드로 옮기는 것이 아니라, 제어 인터페이스를 사용자가 로그인한 곳으로 옮깁니다.

## Claude Code 원격 제어 연결 작동 방식

Claude Code 원격 제어는 아웃바운드 HTTPS 연결(outbound HTTPS connections)만 사용합니다.

사용자의 머신은 인바운드 포트(inbound ports)를 열지 않으므로, 방화벽(firewall)과 NAT 뒤에서도 작동합니다.

1.  원격 제어를 시작하면 로컬 Claude Code 세션이 Anthropic API에 등록하고 작업을 폴링(polling)하기 시작합니다.
2.  휴대폰이나 브라우저에서 연결하면 서버는 스트리밍 연결(streaming connection)을 통해 원격 인터페이스와 로컬 세션 간에 메시지를 라우팅합니다.

모든 트래픽은 TLS를 통해 이동하며, 이는 모든 Claude Code 세션에 사용되는 것과 동일한 암호화 방식입니다.

연결은 여러 개의 단기 자격 증명(short-lived credentials)을 사용하며, 각 자격 증명은 단일 목적에 한정되며 독립적으로 만료됩니다.

### 세션 수명 주기(Session Lifecycle)

Claude Code 원격 제어 세션은 터미널 프로세스(terminal process)가 계속 실행되는 한 활성 상태를 유지합니다.

노트북이 절전 모드에 들어가거나 네트워크 연결이 끊겨도, 머신이 다시 온라인 상태가 되면 세션이 재연결됩니다.

대화는 연결된 모든 기기에서 동기화(sync)됩니다. 터미널에서 메시지를 보내고, 휴대폰에서 응답을 본 다음, 브라우저에서 실시간으로 대화를 계속할 수 있습니다.

머신이 약 10분 이상 오프라인 상태이면 세션 시간 초과(session timeout)가 발생하고 프로세스가 종료됩니다.

새 세션을 시작하려면 `claude remote-control` 명령을 다시 실행해야 합니다.

### 인스턴스당 하나의 세션

각 Claude Code 인스턴스는 한 번에 하나의 원격 세션만 지원합니다.

다른 프로젝트 디렉터리(project directory)에서 여러 Claude Code 인스턴스를 실행하는 경우, 각 인스턴스는 자체 환경과 구성을 가진 자체 원격 제어 세션을 갖게 됩니다.

여러 프로젝트에서 활성 원격 제어 세션을 가질 수 있지만, 각 프로젝트 디렉터리에서는 하나의 세션만 실행할 수 있습니다.

이제 원격 제어가 어떻게 작동하는지 이해했으니, 설정 방법과 첫 세션을 시작하는 방법을 보여드리겠습니다.

## 시작하기 — 첫 번째 원격 제어 세션

Claude Code가 설치되어 있는지 확인하세요. Windows를 사용하는 경우 다음 명령을 사용하세요.

```powershell
irm https://claude.ai/install.ps1 | iex
```

macOS, Linux, WSL에서는 다음 명령을 사용하세요.

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

Claude Code가 설치되면 원격 제어를 사용하려면 환경이 몇 가지 기본 요구 사항을 충족해야 합니다.

*   **구독 등급(Subscription Level)**: 원격 제어는 Pro 또는 Max 요금제(plan)가 필요합니다. 아직 Team 또는 Enterprise 요금제에서는 사용할 수 없으며, API 키는 작동하지 않습니다.
*   **인증(Authentication)**: claude.ai를 통해 로그인해야 합니다. 터미널에서 `claude`를 실행하고 아직 로그인하지 않았다면 `/login`을 사용하여 인증하세요.
*   **워크스페이스 신뢰(Workspace Trust)**: 프로젝트 디렉터리에서 Claude를 처음 실행할 때 워크스페이스 신뢰 대화 상자가 표시됩니다. 원격 제어가 작동하려면 이를 수락해야 합니다.

Claude Code 원격 제어를 사용할 때는 현재 API 인증을 사용할 수 없다는 점을 기억하세요. 예를 들어, 저는 API를 사용하여 Claude Code에 로그인했습니다.

`/logout` 명령을 사용하여 로그아웃한 다음 Claude Pro 계정을 사용하여 인증하겠습니다.

구독 계정으로 로그인했다면 Claude Code 원격 제어를 테스트할 차례입니다.

### 첫 번째 원격 제어 세션 시작하기

프로젝트 디렉터리로 이동하세요.

이 예시에서는 FastAPI 백엔드와 바닐라 JavaScript 프런트엔드가 있다고 가정하겠습니다.

```bash
cd remote-demo
```

이제 원격 제어 명령을 실행합니다.

```bash
claude remote-control
```

이 명령을 실행하면 다음과 같은 오류가 발생했습니다.

```
Error: Remote Control is not enabled for your account. 
Contact your administrator.
```

이는 Git에서 보고된 알려진 문제이며, 모든 방법을 시도해도 작동하지 않는다는 것을 알게 되었습니다.

제가 혼자가 아니라는 것이 밝혀졌습니다.

이 문제를 추적하는 활성 GitHub 이슈에는 macOS, Windows, Linux의 Pro 및 Max 요금제 사용자 수십 명이 동일한 오류를 보고하고 있습니다.

저도 Pro 계정을 가지고 있고 모든 설정 단계를 따랐음에도 불구하고 이런 일이 발생했습니다.

### 원격 제어가 작동하지 않는 이유 (아직)

GitHub 스레드와 사용자 보고서를 조사한 결과, 다음과 같은 상황이 벌어지고 있습니다.

원격 제어는 `tengu_ccr_bridge`라는 서버 측 기능 플래그(server-side feature flag)에 의해 제어되며, 이 플래그는 기본값이 `false`입니다.

이 플래그는 Anthropic 서버에서 평가되고 로컬에 캐싱(cached locally)됩니다. 사용자 스스로 활성화할 방법은 없습니다.

이 기능이 Claude Code 설치에 존재하더라도, 접근 권한은 점진적으로 배포되고 있습니다. 일부 Pro/Max 사용자는 이 기능을 사용할 수 있지만, 대부분은 사용할 수 없습니다.

"관리자에게 문의하세요(Contact your administrator)"라는 오류 메시지는 개인 계정의 경우 연락할 관리자가 없으므로 오해의 소지가 있습니다.

### 사람들이 시도한 방법

GitHub 스레드를 기반으로 사용자들이 시도한 해결책(workaround)은 다음과 같습니다.

**해결책 1: 재인증(Re-authenticate)**

일부 사용자는 로그아웃 후 다시 로그인하면 성공했다고 보고했습니다.

```bash
# In Claude Code
/logout
/login
```

저는 이것을 시도했지만 실패했습니다.

**해결책 2: 데이터 공유 활성화(Enable Data Sharing)**

몇몇 사용자는 원격 측정(telemetry)을 활성화하는 것이 도움이 된다는 것을 발견했습니다.

*   https://claude.ai/settings/data-privacy-controls 로 이동합니다.
*   "Claude 개선에 도움 주기(Help improve Claude)"를 활성화합니다.
*   Claude Code에서 로그아웃했다가 다시 로그인합니다.
*   `claude remote-control`을 다시 시도합니다.

저는 이것을 시도했지만 실패했습니다.

**해결책 3: 원격 측정 차단 제거(Remove Telemetry Blockers)**

`~/.claude/settings.json`에서 원격 측정 기능을 비활성화했다면 해당 설정을 제거해 보세요.

macOS/Linux:

```bash
jq 'del(.env.CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC)' ~/.claude/settings.json > /tmp/settings_tmp.json && mv /tmp/settings_tmp.json ~/.claude/settings.json
jq 'del(.env.DISABLE_TELEMETRY)' ~/.claude/settings.json > /tmp/settings_tmp.json && mv /tmp/settings_tmp.json ~/.claude/settings.json
```

Windows PowerShell:

```powershell
notepad $env:USERPROFILE\.claude\settings.json
# Remove lines with CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC and DISABLE_TELEMETRY
```

저는 이것을 시도했지만 실패했습니다.

이 중 어떤 것도 저에게는 작동하지 않았습니다.

그럼에도 불구하고, 이 초기 오류를 통과할 수 있다면 다음과 같이 작동해야 합니다.

## 원격 제어는 이렇게 작동해야 합니다

다음과 같은 세션 URL이 표시됩니다.

```
Remote Control session started
Session URL: https://claude.ai/code/session/abc123def456...
Press spacebar to show QR code
Waiting for remote connections...
```

프로세스는 터미널에서 계속 실행되며, 다른 기기에서 연결하기를 기다립니다.

### 1) 플래그(Flags) 이해하기

기본 `claude remote-control` 명령은 대부분의 경우에 작동하지만, 추가 옵션이 있습니다.

*   `--verbose`: 자세한 연결 로그와 세션 활동을 보여줍니다. 내부에서 어떤 일이 일어나는지 확인하거나 연결 문제를 디버그할 때 유용합니다.
*   `--sandbox` / `--no-sandbox`: 세션 중 파일 시스템(filesystem) 및 네트워크 격리(network isolation)를 제어합니다. 샌드박싱(Sandboxing)은 기본적으로 비활성화되어 있어 Claude가 프로젝트에 대한 완전한 액세스 권한을 가집니다.

예를 들어, 상세 로깅(`verbose logging`)으로 실행하기:

```bash
claude remote-control --verbose
```

연결 시도, 도구 실행(tool executions), 실시간 세션 활동을 보여주는 상세 출력(detailed output)이 표시됩니다.

### 2) 휴대폰에서 연결하기

원격 제어가 실행 중인 동안 터미널에서 스페이스바(spacebar)를 누릅니다. QR 코드(QR code)가 터미널 창에 바로 나타납니다.

휴대폰에서 Claude 모바일 앱(Claude mobile app)을 열고 QR 코드를 스캔합니다.

아직 앱이 없다면 Claude Code 내에서 `/mobile`을 실행하여 iOS 또는 Android용 다운로드 링크를 표시할 수 있습니다.

앱이 원격 제어 세션으로 직접 열립니다. 이제 휴대폰에서 로컬 Claude Code 인스턴스를 제어하고 있습니다.

휴대폰에서 하는 모든 작업은 노트북에서 발생합니다. Claude에게 파일을 생성하도록 요청하면, 머신의 프로젝트 디렉터리에 파일을 생성합니다.

### 3) 브라우저에서 연결하기

브라우저 액세스에는 두 가지 옵션이 있습니다.

**옵션 1: 직접 URL(Direct URL)**

터미널에서 세션 URL을 복사하여 아무 브라우저에나 붙여넣습니다. 원격 제어 세션이 활성화된 상태로 claude.ai/code로 바로 이동합니다.

**옵션 2: 세션 목록(Session List)**

브라우저에서 claude.ai/code를 엽니다. 원격 제어 세션은 온라인 상태일 때 컴퓨터 아이콘과 녹색 상태 표시등(green status dot)과 함께 세션 목록에 나타납니다.

세션을 클릭하여 엽니다.

세션은 마지막 메시지나 `/rename` 값에서 이름을 가져오거나, 대화 기록이 없는 경우 기본적으로 "Remote Control session"으로 설정됩니다.

### 4) 여러 기기에서 동시에 작업하기

터미널, 브라우저, 모바일 앱 모두를 동시에 동일한 세션에 연결할 수 있습니다. 대화는 모든 기기에서 완벽하게 동기화(perfectly in sync)됩니다.

휴대폰에서 메시지를 보내면 브라우저와 터미널에 즉시 나타납니다.

브라우저에서 대화를 계속합니다. 휴대폰은 실시간으로 업데이트됩니다.

이는 책상에서 작업을 시작하고, 회의에 가는 동안 휴대폰으로 검토하고, 다른 컴퓨터의 브라우저에서 작업을 마칠 수 있다는 의미입니다.

### 5) 연결 시 발생하는 일

원격 기기에서 연결하면 터미널에 연결 활동이 표시됩니다.

다음이 표시됩니다.

*   "연결 설정됨(Connection established)" 메시지
*   Claude가 작업하는 동안의 도구 사용 활동(tool usage activity)
*   프로젝트에서 발생하는 일의 실시간 로그

노트북은 항상 제어권을 유지합니다. 원격 기기는 로컬 Claude Code 세션에 명령을 보내는 것일 뿐입니다.

### 6) 모든 세션에 원격 제어 활성화하기

기본적으로 원격 제어는 `claude remote-control`을 실행할 때만 활성화됩니다.

모든 세션에 활성화하려면 Claude Code 내에서 `/config`를 실행하고 "모든 세션에 원격 제어 활성화(Enable Remote Control for all sessions)"를 `true`로 설정하세요.

이제 `claude`를 실행할 때마다 원격 제어가 자동으로 시작됩니다. 이 동작을 비활성화하려면 다시 `false`로 설정하세요.

### 7) 여러 프로젝트 관리하기

각 Claude Code 인스턴스는 한 번에 하나의 원격 제어 세션만 지원합니다. 하지만 다른 프로젝트 디렉터리에서 여러 인스턴스를 실행할 수 있습니다.

세 가지 프로젝트에서 작업하고 있다고 가정해 봅시다.

```bash
# Terminal 1 - 프런트엔드 프로젝트(Frontend project)
cd ~/projects/my-frontend
claude remote-control

# Terminal 2 - 백엔드 API(Backend API)
cd ~/projects/my-api
claude remote-control
# Terminal 3 - 모바일 앱(Mobile app)
cd ~/projects/my-mobile-app
claude remote-control
```

각 프로젝트는 자체 환경, 구성 및 세션 URL을 가진 자체 원격 제어 세션을 갖게 됩니다.

세션 목록에서 특정 세션을 선택하여 휴대폰이나 브라우저에서 이 세션 중 하나에 연결할 수 있습니다.

GitHub 이슈에는 이 기능을 사용하기 위해 Pro 또는 Max 요금제를 구매했지만 "활성화되지 않음(not enabled)" 오류에 부딪힌 사용자들의 50개 이상의 댓글이 있습니다.

이 테스트를 여기서 끝내야 한다는 것이 아쉽습니다. 저는 이 기능을 테스트하고 이야기를 공유하고 싶어 정말 흥분했었습니다. 따라서 관심이 있다면 며칠마다 다음을 확인해 주세요.


원격 제어가 널리 사용 가능해지고 전체 워크플로(workflow)를 테스트할 수 있게 되면 이 글을 업데이트하겠습니다.
