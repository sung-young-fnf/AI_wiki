# 5개월 만에 깃허브 스타 35.5만 개, 17% 방어율: OpenClaw 완전 분석 가이드

아키텍처, 생태계, 보안 및 모든 ML 엔지니어가 알아야 할 모든 것

저는 이 목적을 위해 구매한 Mac Mini M4에 OpenClaw를 설정하는 데 주말을 보냈습니다. 일요일 밤까지 OpenClaw는 자율적으로 일정 충돌을 재조정하고, 제가 읽지 않은 슬랙(Slack) 스레드 14개를 요약했으며, 심지어 제가 지시하지 않은 내용의 왓츠앱(WhatsApp) 메시지를 동료에게 보냈습니다. 저는 이 모든 일을 하라고 지시한 적이 없습니다. 단지 토요일 오후에 한 번만 설정했을 뿐입니다.

깃허브 스타 355,000개. 320만 명의 활성 사용자. 50만 개 이상의 실행 중인 인스턴스. 이 모든 것이 5개월도 채 안 되는 기간에 달성되었습니다. OpenClaw는 2026년 가장 중요한 오픈소스 프로젝트이거나 가장 위험한 프로젝트일 것입니다. 2주 동안 실행해본 결과, 저는 둘 다라고 생각합니다.

이 가이드는 OpenClaw가 실제로 무엇인지, 아키텍처가 어떻게 작동하는지, 2026년 4월 현재 생태계가 어떤 모습인지, 어떻게 배포하는지, 그리고 대부분의 '시작하기' 게시물이 편리하게 건너뛰는 보안 현실까지 모든 것을 다룹니다.

## OpenClaw가 실제로 하는 일

OpenClaw는 자체 호스팅되는 개인 AI 비서로, 모든 대규모 언어 모델(LLM)을 50개 이상의 메시징 플랫폼에 연결하여 실제 작업을 자율적으로 실행합니다. 셸(Shell) 명령어, 파일 관리, 브라우저 자동화, API 호출, 캘린더 일정 관리, 스마트 홈 제어 등이 포함됩니다.

LangGraph 또는 CrewAI로 에이전트를 구축해본 적이 있다면, '작업에 대해 추론할 수 있는 에이전트'와 '실제로 사용자 기기에서 작업을 수행하는 에이전트' 사이의 간극을 알고 계실 겁니다. 저는 "대부분의 AI 에이전트가 프로덕션에서 실패하는 이유"라는 글에서 이 점에 대해 다룬 바 있습니다. OpenClaw는 이 간극에서 '실제로 일을 하는' 쪽에 확실히 자리 잡고 있습니다.

OpenClaw는 사용자 하드웨어에서 실행됩니다. 사용자가 모델을 선택하고 데이터를 소유합니다. 세션 간 영구 메모리를 유지하고, 크론(cron) 작업을 통해 선제적인 백그라운드 작업을 실행하며, 모든 메시징 플랫폼에서 동시에 작동합니다.

이 프로젝트는 2026년 3월 3일 리액트(React)를 넘어 깃허브에서 가장 많은 스타를 받은 비집계 소프트웨어 프로젝트가 되었습니다. 리액트는 25만 개의 스타를 얻는 데 10년 이상 걸렸지만, OpenClaw는 60일 만에 이를 달성했습니다. 출시 행사, 프로덕트 헌트(Product Hunt) 캠페인 없이 순전히 입소문으로만 이뤄낸 성과입니다.

## 탄생 스토리

오스트리아 개발자이자 PSPDFKit의 창립자인 피터 슈타인버거(Peter Steinberger)는 2025년 11월 "클라우드봇(Clawdbot)"이라는 주말 해킹 프로젝트를 발표했습니다. 이는 앤트로픽(Anthropic)의 클로드(Claude)에 대한 재치 있는 오마주였습니다.

이 프로젝트는 두 달 동안 조용히 있었지만, 2026년 1월 말 AI 에이전트만을 위해 구축된 레딧(Reddit) 스타일의 소셜 네트워크인 몰트북(Moltbook)과 함께 빠르게 확산되었습니다. 앤트로픽의 법무팀이 1월 27일 상표권 침해 민원을 제기하여 "몰트봇(Moltbot)"으로 이름을 변경해야 했습니다. 사흘 뒤 슈타인버거는 몰트봇이라는 이름이 "입에 잘 붙지 않았다"는 이유로 "OpenClaw"를 최종 선택했습니다.

2월 14일, 그는 오픈AI(OpenAI)에 합류한다고 발표했습니다. 샘 올트먼(Sam Altman)은 그를 "매우 똑똑한 에이전트의 미래에 대한 놀라운 아이디어를 많이 가진 천재"라고 불렀습니다. 프로젝트를 관리하기 위해 501(c)(3) 재단이 설립되었습니다. 오픈AI가 후원하지만 코드를 소유하지는 않습니다. MIT 라이선스로 배포됩니다.

이 모든 과정은 약 90일 만에 일어났습니다. 주말 해킹 프로젝트에서 오픈AI와의 협력 단계까지 진행된 것입니다. 이러한 속도는 OpenClaw가 상징하는 것에 대한 수요가 얼마나 큰지 보여줍니다.

## 작동 방식: 아키텍처

저는 8년 동안 머신러닝(ML) 시스템을 구축해왔는데, OpenClaw에서 가장 놀랐던 점은 핵심 아키텍처가 얼마나 단순한지였습니다. 저는 오케스트레이션(orchestration) 계층, 에이전트 계층 구조, 플래너(planner) 트리를 예상했습니다. 하지만 비전(Vision) 문서에서는 이 모든 것을 명시적으로 거부합니다.

### 게이트웨이(The Gateway)

모든 것은 게이트웨이(Gateway)라고 불리는 단일 타입스크립트(TypeScript) 프로세스를 통해 실행됩니다. 이 프로세스는 포트 18789에서 웹소켓(WebSocket) 서버를 관리하며 모든 채널 연결, 세션 라우팅, 도구 실행을 처리합니다. 단일 프로세스, 단일 포트, 이것이 전부입니다.

처음 아키텍처 문서를 읽었을 때, 저는 '진짜' 오케스트레이션 계층을 계속 찾았습니다. 그런 것은 없습니다. 게이트웨이가 바로 이 제품의 핵심입니다. 연결된 모든 채널(왓츠앱, 텔레그램, 슬랙, 디스코드, 아이메시지, 시그널 등 50개 이상)에서 메시지를 수신하고, 선택한 LLM으로 라우팅하며, 도구를 통해 응답을 실행합니다.

```
게이트웨이 (포트 18789)
  ├── 채널 연결 (왓츠앱, 텔레그램, 슬랙, 디스코드, ...)
  ├── 세션 라우팅 (누가 대화하는지, 어떤 컨텍스트인지)
  ├── 도구 실행 (셸, 파일, 브라우저, API)
  └── 메모리 관리 (마크다운 파일 읽기/쓰기)
```

### 모델 중립성(Model Agnosticism)

기본적으로 25개 이상의 LLM 제공업체를 지원합니다. 클로드(Claude), GPT, 딥시크(DeepSeek), 제미니(Gemini) 및 올라마(Ollama)를 통한 로컬 모델을 포함합니다. 다른 것은 변경할 필요 없이 모델을 교체할 수 있습니다. 저는 첫 주에는 클로드 3.5 소네트(Sonnet)로 테스트했고, 그 다음에는 올라마를 통한 로컬 라마 3 8B(Llama 3 8B)로 기본적인 작업을 수행했습니다(최첨단 모델 추론이 필요하지 않지만 API 비용 없이 24시간 365일 가동이 필요한 종류의 작업).

### 3계층 확장 시스템

여기서 설계가 영리해집니다.

*   **도구(Tools)**는 특정 작업을 위한 타입이 지정된(typed) 타입스크립트(TypeScript) 함수입니다. 파일 읽기, 셸 실행, 웹 가져오기 등 저수준의 기본 기능(primitive)을 제공합니다.
*   **스킬(Skills)**은 `SKILL.md` 파일에 정의됩니다. 기능, 트리거, 동작을 설명하는 마크다운(Markdown) 파일입니다. 커스텀 명령어(custom instructions)가 있는 클로드 프로젝트(Claude Projects)를 사용해본 적이 있다면 익숙할 것입니다. 스킬은 다음과 같이 지정할 수 있습니다. "사용자가 캘린더에 대해 물어보면, 구글 캘린더(Google Calendar) API 도구를 사용하여 이벤트를 가져오고 충돌을 요약하라." LLM은 마크다운을 읽고 무엇을 해야 할지 압니다.
*   **플러그인(Plugins)**은 완전한 npm 패키지입니다. 복잡한 로직, 외부 종속성, 공유 상태가 필요할 때 선택할 수 있는 최후의 수단입니다.

다음은 기본적인 `SKILL.md` 파일의 모습입니다.

```markdown
---
name: morning-briefing
trigger: cron 0 7 * * *
---

# 아침 브리핑
매일 오전 7시:
1. 읽지 않은 이메일 확인 (지난 12시간)
2. 오늘 일정 충돌 요약
3. 검토가 필요한 오픈된 깃허브(GitHub) PR 목록 작성
4. 오늘 날씨 확인
5. 요약을 텔레그램(Telegram)으로 전송
200단어 이내로, 군더더기 없이. 긴급한 내용부터 시작하세요.
```

이것은 실제 스킬입니다. 파이썬(Python) 코드도, 프레임워크 코드도 없습니다. LLM에게 무엇을 언제 할지 알려주는 마크다운일 뿐입니다. (처음 이것을 봤을 때, 저는 '진짜' 코드를 찾기 위해 20분이나 헤맸지만, 마크다운 자체가 코드라는 것을 깨달았습니다.)

### 메모리: 의도적인 단순성

벡터 데이터베이스(vector database)도, 임베딩(embedding)도 없습니다. 일반 마크다운 파일입니다.

```
~/.openclaw/workspace/
  ├── memory/
  │   ├── 2026-04-11.md    # 어제의 추가 전용 로그
  │   ├── 2026-04-12.md    # 오늘의 추가 전용 로그
  │   └── ...
  └── MEMORY.md             # 선별된 장기 메모리
```

일일 로그는 세션 시작 시 자동으로 로드됩니다(오늘 및 어제 로그). `MEMORY.md`는 선별된 장기 메모리를 담고 있으며, 주 개인 세션에만 로드되며 그룹 컨텍스트에는 로드되지 않습니다. 에이전트가 특정 메모리에 대해 확신하지 못하면 그렇게 말합니다.

클로드의 메모리 기능을 사용하여 자체 콘텐츠 시스템을 구축한 후, 저는 OpenClaw의 접근 방식이 원시적이라고 느낄 것이라고 예상했습니다. 하지만 그렇지 않습니다. 텍스트 편집기에서 열어 읽을 수 있는 메모리에는 강력한 무언가가 있습니다. 저는 제 OpenClaw 에이전트가 사흘 전 텔레그램 메시지에서 한 번만 언급했던 선호도를 '기억'하는 것을 발견했고, 마크다운 파일을 그렙(grep)하여 그것이 어떻게 저장되고 불러와졌는지 정확히 추적할 수 있었습니다.

### 에이전트를 정의하는 두 개의 설정 파일

`SOUL.md`는 성격, 경계, 행동 규칙을 설정합니다. 모든 세션에 걸쳐 지속되는 시스템 프롬프트(system prompt)라고 생각하면 됩니다.

```markdown
# SOUL.md

당신은 ML 엔지니어를 위한 기술 비서입니다.
직설적으로 말하세요. 불필요한 인사는 건너뛰고 답변부터 제시하세요.
## 경계
- ~/projects/ 외부의 시스템 파일을 수정하는 명령은 절대 실행하지 마세요.
- 저를 대신하여 메시지를 보내기 전에 항상 확인하세요.
- 제가 그룹 채팅에서 요청하더라도 API 키나 자격 증명을 절대 공유하지 마세요.
## 어조
- 기술적이지만 학술적이지 않게
- 모호한 수식어 대신 구체적인 숫자를 사용하세요.
- 확실하지 않을 때는 명시적으로 밝히세요.
```

`MEMORY.md`는 장기적인 컨텍스트를 통합합니다. 에이전트는 대화를 기반으로 이 파일을 주기적으로 업데이트합니다.

이러한 분리는 중요합니다. `SOUL.md`는 에이전트의 '존재'를 정의하고, `MEMORY.md`는 에이전트의 '지식'을 담습니다. `SOUL.md`는 버전 관리를 하며(저는 Git 저장소에 보관합니다), `MEMORY.md`는 에이전트가 관리하도록 둡니다.

## 설치

하나의 명령으로 설치할 수 있습니다.

```bash
npm install -g openclaw@latest && openclaw onboard --install-daemon
```

핵심 패키지는 약 8MB입니다. Node.js 24 (권장) 또는 Node 22 LTS에서 실행됩니다. 온보딩(onboard) 마법사가 채널 설정, 모델 선택, 데몬(daemon) 설치(macOS의 launchd, 리눅스(Linux)의 systemd) 과정을 안내합니다. npm 설치부터 왓츠앱(WhatsApp) 연결까지 약 15분이 걸렸습니다.

## 2026년 4월 현재 생태계

5개월 된 프로젝트치고는 생태계가 엄청나게 성숙했습니다. 현재 존재하는 것들을 살펴보겠습니다.

### 공식 인프라

깃허브 조직에는 24개의 저장소가 있습니다. 메인 저장소는 pnpm으로 관리되는 타입스크립트(TypeScript) 모노레포(monorepo)입니다. 핵심 패키지는 core, gateway, agent, cli, sdk, ui입니다. 1,200명 이상의 기여자, 58,000개 이상의 포크(fork)를 가지고 있습니다. 88개의 npm 패키지가 OpenClaw에 직접적으로 의존합니다. 릴리스는 거의 이틀에 한 번 이루어집니다(이번 주에 버전 2026.4.2가 출시되었습니다).

### ClawHub: 스킬 마켓플레이스

2026년 4월 현재 44,000개 이상의 커뮤니티 제작 스킬이 있으며(2월 초 5,700개에서 증가), 65% 이상이 MCP 서버를 래핑합니다. 1,000개 이상의 커뮤니티 MCP 서버가 있습니다.

MCP 보안 문제에 대한 제 글을 읽어보셨다면, 여기서의 문제점을 즉시 파악할 수 있을 것입니다. 보안 섹션에서 자세히 다루겠습니다.

큐레이션된 대안으로는 깃허브의 awesome-openclaw-skills가 있으며, 5,400개 이상의 검증된 스킬로 필터링됩니다.

### 알아두면 좋은 도구들

*   **OpenClaw-RL** (프린스턴(Princeton) 연구진, 4,730개 이상의 스타): 실시간 대화에서 에이전트를 훈련하는 완전 비동기 강화 학습(reinforcement learning) 프레임워크입니다. 다음 상태 신호(next-state signals)를 학습 소스로 사용합니다. 주요 성과는 8~36번의 상호작용만으로 개인화 점수가 0.17에서 0.76으로 향상되었다는 것입니다. 관련 논문(arXiv 2603.10165)은 토큰(token) 수준 감독을 위한 후회 지향 온-정책 증류(Hindsight-Guided On-Policy Distillation)를 소개합니다. (이러한 종류의 연구는 일반적으로 논문에서 사용 가능한 프레임워크로 옮겨지는 데 몇 년이 걸리지만, 이들은 몇 주 만에 해냈습니다.)
*   **NemoClaw** (엔비디아(NVIDIA)): 관리형 추론(managed inference)을 통해 OpenClaw를 샌드박스(sandbox) 컨테이너에서 실행하는 강화된 래퍼(wrapper)입니다. 이는 "OpenClaw의 기능은 마음에 들지만 보안 팀이 반대할 것"이라는 기업의 우려에 대한 해답입니다.
*   **IronClaw** (NEAR AI): 프라이버시 우선 아키텍처를 갖춘 러스트(Rust) 재구현 버전입니다. 타입스크립트(TypeScript)가 자율 에이전트를 실행하는 데 적합하지 않다고 생각하는 사람들을 위한 것입니다.
*   **Lobster**: 공식 워크플로(workflow) 셸. 구성 가능한 에이전트 파이프라인(pipeline)을 위한 타입이 지정된(typed), 로컬-퍼스트(local-first) 매크로(macro) 엔진입니다. 에이전트 워크플로를 위해 특별히 제작된 스크립팅 언어라고 생각하면 됩니다.
*   **ClawRouter**: 제공업체 간 비용 인지 LLM 라우팅을 수행합니다.
*   **GitClaw**: 깃허브 액션(GitHub Actions)에서 OpenClaw를 전적으로 실행합니다.
*   **EverMemOS**: 24시간 365일 에이전트의 장기 메모리 관리를 지원합니다.

### 학술 연구

첫 3개월 만에 아카이브(arXiv)에 15개 이상의 논문이 발표되었습니다. 저는 8년 동안 ML 분야에 있었지만, 오픈소스 프로젝트에서 이렇게 높은 연구 밀도는 본 적이 없습니다.

연구 주제는 보안(47개 시나리오에 걸친 적대적 테스트, 190개 권고 사항을 분석한 취약점 분류), 사회적 행동(3주 만에 280만 AI 에이전트가 등록하여 인간의 감독 없이 '선별적 사회 규제'를 개발한 몰트북에서의 비상 거버넌스), 로봇 공학(OpenGo가 실시간 스킬 전환이 가능한 유니트리 Go2(Unitree Go2) 로봇 개에 OpenClaw를 배포), 포렌식 분석 등에 걸쳐 있습니다.

사회적 행동 연구는 특히 놀랍습니다. 몰트북의 댓글 지니 계수(Gini coefficient)는 0.889를 기록했습니다(arXiv 2602.18832). 이는 인간 사회 네트워크의 불평등 패턴을 반영합니다. AI 에이전트가 대규모로 상호작용하도록 방치되면, 인간과 동일한 참여 역학을 재현한다는 것입니다.

### 커뮤니티

디스코드(Discord) 회원은 169,545명입니다(서버 이름은 "Friends of the Crustacean 🦞🤝"). X/트위터(Twitter) 팔로워는 31,900명 이상입니다. 클로콘(ClawCon) SF 2026에는 34개국에서 1,200명의 참석자가 모였습니다. OpenClaw를 기반으로 구축된 180개 스타트업은 월 32만 달러 이상의 수익을 합산하여 창출하고 있습니다. 프로젝트 사이트에는 월 3,800만 명의 방문자가 방문합니다. 텐센트(Tencent) 및 바이두(Baidu) 통합 이후 중국에서의 성장률은 월별 1,436% 증가했습니다.

## 시작하기: 하드웨어 및 배포

OpenClaw는 사용자 하드웨어에서 실행됩니다. 커뮤니티는 모든 가격대별 배포 경로를 문서화했습니다.

*   **Mac Mini M4** (약 499~599달러): 커뮤니티에서 가장 선호하는 옵션입니다. 유휴 상태에서 4~8W를 소비하며, 연간 전기료는 약 15달러입니다. 올라마(Ollama)를 통해 로컬 7B-14B 파라미터 모델을 실행합니다. OpenClaw의 폭발적인 성장으로 실제로 맥 미니 품절 사태가 발생했습니다. (저는 품절 사태 직전에 제 것을 구입했습니다. 운이 좋았죠.)
*   **Raspberry Pi 5** (8GB, 약 80~120달러): 5W를 소비하는 예산 친화적인 옵션입니다. 로컬에서 모델을 실행하는 대신 클라우드(cloud) API 추론과 함께 사용하세요. 온디바이스(on-device) 추론이 필요 없는 개인 비서에게 충분합니다.
*   **Jetson Orin Nano "ClawBox"**: 15W에서 67 TOPS의 AI 연산 능력을 제공하며 OpenClaw가 사전 설치되어 있습니다. 열성적인 사용자들을 위한 하드웨어 옵션입니다.
*   **클라우드(Cloud)**: 디지털오션(DigitalOcean)은 원클릭 배포를 제공합니다. 헤츠너(Hetzner), 텐센트 클라우드(Tencent Cloud), 벌쳐(Vultr)가 인기 있는 대안입니다. OpenClawVPS는 월 19달러부터 자동 보안 업데이트가 포함된 관리형 호스팅을 제공합니다.

사람들이 실제로 배포하는 방식(2026년 4월 기준): 도커 컴포즈(Docker Compose)가 65%로 가장 많고, 네이티브(native) npm이 25%, 쿠버네티스(Kubernetes)가 10%를 차지합니다. 최소 요구 사항은 2GB RAM, 1 CPU 코어, 5GB 디스크로 적당합니다.

## 누구도 쓰고 싶어 하지 않는 보안 섹션

다른 섹션보다 이 섹션에 더 많은 시간을 할애할 것입니다. 대부분의 OpenClaw 가이드가 여기서 신뢰를 잃기 때문입니다. 그들은 이 부분을 아예 건너뛰거나 세 가지 요점만 언급합니다. 현실은 더 가혹합니다.

### 연구 결과는 좋지 않다고 말합니다

"클로가 당신의 손을 잡게 하지 마라(Don’t Let the Claw Grip Your Hand)" (arXiv 2603.10387) 논문은 6가지 공격 범주에 걸쳐 47개의 적대적 시나리오를 테스트했습니다. 평균 기본 방어율은 17%였습니다. 즉, 83%의 공격이 기본 설정 상태에서 성공했습니다.

배포된 OpenClaw 인스턴스에 대한 첫 번째 실제 안전성 평가인 "당신의 에이전트, 그들의 자산(Your Agent, Their Asset)" (arXiv 2604.04759) 논문은 단일 CIK 차원(역량(Capability), 신원(Identity), 지식(Knowledge))을 오염시키면 공격 성공률이 24.6%에서 64~74%로 증가한다는 것을 보여주었습니다.

체계적인 분류 논문(arXiv 2603.27517)은 190개의 보안 권고 사항을 분석하여 완전한 비인증 원격 코드 실행(unauthenticated remote code execution) 경로를 구성하는 세 가지 취약점을 발견했습니다.

### 취약점은 현실입니다

**클로잭드(ClawJacked)** (CVE-2026–25253, CVSS 8.8): 모든 웹사이트가 웹소켓(WebSocket)을 통해 개발자의 AI 에이전트를 조용히 완전히 제어할 수 있었습니다. v2026.2.25에서 패치되었지만, 공격 표면(attack surface)은 실제 존재했고 업데이트되지 않은 모든 인스턴스가 노출되었습니다.

시큐리티스코어카드(SecurityScorecard)는 40,214개의 노출된 인스턴스를 발견했으며, 그중 35.4%가 취약한 것으로 판명되었습니다. 135,000개 이상의 인스턴스가 로컬호스트(localhost) 대신 0.0.0.0에 바인딩(bind)되어 있었습니다.

### 스킬 마켓플레이스에 공급망 문제가 있습니다

시스코(Cisco) 보안 연구원들은 클로허브(ClawHub) 스킬을 테스트한 결과 17%가 잠재적으로 악성으로 분류되었습니다. 44,000개 이상의 스킬 중에서 말이죠. 바이러스토탈(VirusTotal)은 이제 제출된 스킬을 스캔합니다. 하지만 npm 악성 코드 사건을 지켜보셨다면, 스캔이 명백한 것들은 잡아내지만 창의적인 것들은 놓친다는 것을 알고 있을 것입니다.

이는 제가 MCP 보안 문제에 대한 글에서 설명했던 것과 동일한 패턴입니다. 에이전트에게 실제 작업을 실행할 수 있는 능력을 부여하고 광범위한 권한을 가진 커뮤니티 제작 확장 프로그램을 설치하도록 허용하면, 패키지 관리자가 10년 동안 싸워온 모든 공급망 위험을 그대로 물려받게 됩니다. 이제 이 '패키지'는 자율적으로 이메일을 보내고, 셸 명령어를 실행하며, 파일 시스템에 접근할 수 있다는 점만 다릅니다.

### 실제로 해야 할 일

스킬을 설치하기 전에 `openclaw security audit --deep`을 실행하세요. 전용 하드웨어(일상용 기기가 아닌)를 사용하세요. 로컬호스트(localhost)에만 바인딩하세요. 감당할 수 있는 가장 강력한 모델을 사용하세요(최첨단 모델은 작은 모델보다 프롬프트 인젝션(prompt injection)에 더 잘 저항합니다). 별도의 이메일 주소와 API 키를 사용하는 격리된 계정을 생성하세요. 명령 실행을 명시적인 허용 목록(allowlist)으로 제한하세요. 도커(Docker) 내부에서 실행하세요. API 지출 한도를 엄격하게 설정하세요.

프로젝트 유지 관리자 중 한 명인 섀도우(Shadow)는 이렇게 직설적으로 말했습니다. "명령줄을 실행하는 방법을 이해하지 못한다면, 이 프로젝트는 안전하게 사용하기에 너무 위험합니다."

공식 입장은 솔직합니다. "완벽하게 안전한 설정은 없습니다."

저는 그 솔직함을 존중합니다. 이 정도 규모의 대부분 프로젝트는 변경 로그 각주에 보안 관련 내용을 숨겼을 것입니다.

## 누가 사용해야 하는가 (그리고 누가 사용하지 말아야 하는가)

터미널(terminal) 사용에 능숙한 개발자이고, 직접 제어하는 하드웨어에서 24시간 365일 실행되는 AI 비서를 원하며, 기꺼이 유지보수 및 업데이트를 할 의향이 있다면, OpenClaw는 현재 사용 가능한 가장 유능한 도구입니다. 스킬 생태계만으로도 시도해볼 가치가 있습니다. 새벽 2시에 왓츠앱(WhatsApp)으로 에이전트에게 메시지를 보내고 기기에서 실제로 작업을 수행하게 하는 것은 채팅창에 입력하는 것과는 다른 경험입니다.

오늘 당장 엔터프라이즈(enterprise) 보안 보장이 필요하다면, 이 프로젝트는 아직 준비되지 않았습니다. 프레임워크는 5개월밖에 안 되었고 보안 연구 결과가 이를 명확히 보여줍니다. 단순히 챗봇이 필요하다면, 클로드(Claude)나 ChatGPT의 웹 인터페이스가 더 간단합니다. 그리고 AI 에이전트에게 기기에서 실제 권한(셸 접근, 파일 시스템 접근, 자신을 대신한 메시징)을 부여하는 것이 불편하다면, 샌드박싱(sandboxing) 기술이 성숙해질 때까지 기다려야 합니다. 엔비디아(NVIDIA)의 NemoClaw와 NEAR AI의 IronClaw는 이 문제를 해결하기 위해 노력하고 있습니다.

솔직한 요약: OpenClaw는 제가 올해 사용해본 AI 프로젝트 중 가장 인상 깊고 동시에 가장 두려운 프로젝트입니다. 아키텍처는 우아하고, 생태계는 오픈소스에서 제가 본 어떤 것보다 빠르게 발전하고 있습니다. 하지만 보안 공격 표면(security surface area)은 저를 밤잠 설치게 합니다.

빠르게 출시하고, 끊임없이 감사를 수행하는 것. 이것이 이 프로젝트를 운영하는 유일한 책임감 있는 방법입니다.
