# 신규 클로드 코드 채널 테스트 후기 (진정한 OpenClaw 킬러)

_작성자: Joe Njenga | 9분 소요 | 1일 전_
_이 글은 작가가 유료로 설정했으며, 독자가 열람 시 미디엄(Medium) 파트너 프로그램(Partner Program)을 통해 수익을 얻는 회원 전용 콘텐츠입니다._

클로드 코드 채널(Claude Code Channels)은 클로드 코드 워크플로우(Claude Code workflows)에서 OpenClaw를 대체하도록 설계된 최신 기능으로, 이제 즐겨 사용하는 앱에서 자유롭게 클로드 코드를 실행할 수 있게 해줍니다.

만약 텔레그램(Telegram)이나 디스코드(Discord)를 통해 AI 에이전트(AI agent)에 메시지를 보내기 위해 OpenClaw를 사용해왔다면, 복잡한 설정, 보안 문제, 그리고 최근 Anthropic의 구독 토큰(subscription tokens) 차단으로 인해 좌절감을 느꼈을 것입니다.

Anthropic은 이와 유사하지만 네이티브(native) 방식으로 작동하는 솔루션을 막 출시했습니다.

_미디엄(Medium) 유료 회원이 아니시라면 이 글을 여기서 무료로 읽으실 수 있습니다. 하지만 제 작업을 지원하기 위해 미디엄 가입을 고려해 주시면 감사하겠습니다!_

---

제 클로드 코드 세션(Claude Code session)을 위한 채팅 앱에서 보낸 첫 번째 테스트 메시지와 그에 대한 응답입니다.

이는 제 메시징 앱과 실행 중인 클로드 코드 세션(Claude Code session) 간의 양방향 대화입니다.

텔레그램(Telegram)으로 이를 설정했을 때, 제가 작업을 보내면 클로드(Claude)가 이를 받아 작업을 수행하고 텔레그램으로 직접 답변을 보냈습니다.

이것이 바로 클로드 코드 채널(Claude Code Channels)이 작동하는 방식입니다!

그렇다면,

## 클로드 코드 채널(Claude Code Channel)이란 무엇일까요?

채널은 실행 중인 클로드 코드 세션(Claude Code session)으로 이벤트를 푸시(push)하는 MCP 서버(MCP server)입니다.

텔레그램(Telegram)이나 디스코드(Discord)에서 메시지를 보낼 수 있으며, 사용자가 자리를 비웠을 때도 클로드(Claude)가 반응합니다. 이는 휴대폰에서 바로 클로드 코드(Claude Code)를 사용하기 위해 OpenClaw를 설정하는 것과 동일한 과정입니다.

작동 방식은 다음과 같습니다:

1.  텔레그램(Telegram) 또는 디스코드(Discord)에서 봇(bot)에게 메시지를 보냅니다.
2.  채널 플러그인(channel plugin)이 해당 메시지를 클로드 코드 세션(Claude Code session)으로 전달합니다.
3.  클로드(Claude)는 메시지를 읽고 작업을 수행한 후 동일한 채널을 통해 답변을 보냅니다.
4.  답변이 휴대폰에 표시됩니다.

현재 Anthropic은 연구 프리뷰(research preview) 단계에서 텔레그램(Telegram)과 디스코드(Discord) 두 가지 플랫폼을 지원하고 있습니다.

또한 실제 플랫폼을 연결하기 전에 테스트를 위한 Fakechat이라는 로컬호스트(localhost) 데모도 있습니다.

설정 및 테스트로 넘어가기 전에, 이 기능이 OpenClaw 문제를 어떻게 해결하는지 살펴봐야 합니다.

## OpenClaw 문제점

OpenClaw가 가장 빠르게 성장하는 오픈소스(open-source) AI 프로젝트가 된 데에는 합당한 이유가 있었습니다. 개발자들은 휴대폰에서 AI 에이전트(AI agent)와 대화하기를 원하며, 이는 실제적인 요구사항입니다.

이상적으로는 출퇴근 중에 텔레그램(Telegram)으로 작업을 보내고, 자리를 비웠을 때 디스코드(Discord)에서 빌드 상태를 확인하는 과정입니다.

하지만 OpenClaw에는 다음과 같은 문제점들이 있었습니다:

*   **보안 위험(Security risks)** — 연구원들은 공개 인터넷에서 135,000개 이상의 노출된 인스턴스(instance)를 발견했으며, 그중 50,000개 이상이 원격 코드 실행(remote code execution)에 취약했습니다. ClawHub 감사 결과, 커뮤니티 스킬(community skills) 중 약 12%가 악성 코드인 것으로 드러났습니다.
*   **복잡한 설정(Complex setup)** — 전용 하드웨어(dedicated hardware)(많은 개발자들이 이를 위해 맥 미니(Mac Mini)를 구매했습니다), 도커(Docker) 구성, 그리고 수 시간의 설정 작업이 요구되었습니다. 저는 이전 튜토리얼에서 맥 미니(Mac Mini)와 윈도우(Windows)에 설정하는 데모를 진행했습니다.
*   **구독 차단(Subscription block)** — Anthropic은 서드파티 도구(third-party tools)가 클로드 프로/맥스(Claude Pro/Max) 구독 토큰(subscription tokens)을 사용하는 것을 차단했습니다. 이로 인해 클로드(Claude)에 접근하는 OpenClaw의 가장 저렴한 경로는 더 이상 작동하지 않게 되었습니다.
*   **설립자 이탈(Founder departure)** — Peter Steinberger가 OpenAI에 합류하고 프로젝트가 재단으로 이관되면서, 프로젝트의 향후 방향이 불확실한 상태입니다.

클로드 코드 채널(Claude Code Channels)은 이 모든 문제를 네이티브(native) 방식으로 해결합니다. 더 이상 서드파티 에이전트(third-party agents), 노출된 인스턴스(exposed instances) 또는 구독 해킹(subscription hacks)이 필요하지 않습니다.

또한 Anthropic의 보안 모델(security model)이 내장되어 있어, 발신자 허용 목록(sender allowlists), `--channels` 플래그(flag)를 통한 세션별 옵트인(per-session opt-in), 그리고 팀을 위한 엔터프라이즈 제어(enterprise controls) 기능을 제공합니다.

이 주제는 복잡하므로, 시리즈로 나누어 설명하겠습니다. 이 시리즈에서는 플랫폼에 연결하기 전에 흐름을 이해하기 위한 간단한 Fakechat 데모부터 시작하여 텔레그램(Telegram)과 디스코드(Discord)의 전체 설정을 안내해 드릴 것입니다.

이 첫 번째 글에서는 다음 내용을 다룰 것입니다:

*   사전 요구 사항(Prerequisites) 및 모든 종속성(dependencies) 설정
*   Fakechat 데모 설정 (먼저 흐름 이해하기)

다음 튜토리얼에서는 다음 내용에 대해 자세히 다룰 것입니다:

*   텔레그램(Telegram) 봇(bot)을 클로드 코드(Claude Code)에 연결하기 (단계별 설명)
*   디스코드(Discord) 봇(bot)을 클로드 코드(Claude Code)에 연결하기 (단계별 설명)
*   보안 및 접근 제어(access control) (보안 강화 방법)
*   채널(Channels)과 OpenClaw 비교 (솔직한 비교)

Fakechat 데모부터 시작하겠습니다.

## 클로드 코드 채널(Claude Code Channels) 시작하기

첫 번째 단계는 클로드 코드(Claude Code)가 최신 상태인지 확인하고 다른 모든 종속성(dependencies)이 제대로 설정되어 있는지 확인하는 것입니다.

### 사전 요구 사항(Prerequisites)

어떤 채널을 설정하기 전에 다음 두 가지가 준비되어 있어야 합니다:

*   클로드 코드(Claude Code) v2.1.80 이상 (이전 버전에서는 채널이 작동하지 않습니다)
*   Bun 런타임(Bun Runtime) (모든 채널 플러그인(channel plugins)은 Bun에서 실행됩니다)

### 1단계: 클로드 코드(Claude Code) 버전 확인

터미널(terminal)에서 다음 명령을 실행합니다:

```bash
claude --version
```

예상 출력: 버전 2.1.80 이상

버전이 이전이라면 먼저 업데이트합니다:

```bash
claude update
```

그 다음 동일한 명령을 사용하여 다시 확인합니다.

### 2단계: Bun 설치 확인

실행합니다:

```bash
bun --version
```

버전 번호가 표시되면 정상적으로 설치된 것이므로 3단계로 넘어갑니다.

제 경우처럼 실패한다면, Bun을 설치합니다:

**Mac/Linux**

```bash
curl -fsSL https://bun.sh/install | bash
```

**Windows (PowerShell):**

```powershell
irm bun.sh/install.ps1 | iex
```

설치 후 확인합니다:

```bash
bun --version
```

---

한 가지 더, 채널(channels)은 claude.ai 로그인(login)을 필요로 합니다. 콘솔(Console) 및 API 키(API key) 인증은 여기서는 작동하지 않습니다. 팀(Team) 또는 엔터프라이즈(Enterprise) 요금제를 사용 중인 경우, 관리자(admin)가 관리 설정(managed settings)에서 채널을 활성화해야 사용할 수 있습니다.

### 3단계: 클로드 코드(Claude Code) 시작

프로젝트 폴더(제 경우 `channels` 폴더) 안에 있는 상태에서 클로드 코드 세션(Claude Code session)을 시작합니다:

```bash
claude
```

클로드 코드(Claude Code)가 로드될 때까지 기다립니다.

인증 오류(auth error)가 발생하면 claude.ai 계정으로 로그인했는지 확인하세요 (API 키(API key) 또는 콘솔(Console)이 아님). 필요한 경우 `/login`을 실행합니다.

## Fakechat — 먼저 흐름 테스트하기

텔레그램(Telegram) 또는 디스코드(Discord)에 연결하기 전에 Fakechat부터 시작하는 것을 권장합니다.

이는 브라우저(browser)에서 채팅 UI(Chat UI)를 실행하는 로컬호스트(localhost) 데모로, 별도의 인증이나 외부 서비스 구성 없이 사용할 수 있습니다.

봇 토큰(bot tokens) 및 페어링 코드(pairing codes)를 다루기 전에 채널 메시지 흐름이 어떻게 작동하는지 보여줍니다.

### 1단계: Fakechat 플러그인(Plugin) 설치

클로드 코드 세션(Claude Code session)을 시작하고 다음을 실행합니다:

```bash
/plugin install fakechat@claude-plugins-official
```

플러그인(plugin) 설치 옵션이 제시될 것입니다. 제 경우, 사용자 범위(User Scope)에 설치하겠습니다:

성공하면, 활성화를 위해 다시 로드하라는 메시지가 표시될 것입니다:

```
✓ Installed fakechat. Run /reload-plugins to activate.
```

FakeChat 플러그인(plugin)이 활성화되었는지 확인하려면 `/plugin` 명령을 다시 실행하고 'Installed' 탭으로 이동하여 확인하십시오.

---

### 설치 오류 (Installation Error)

마켓플레이스(marketplace)를 찾을 수 없다는 오류가 발생하면, 먼저 다음 명령을 실행합니다:

```bash
/plugin marketplace add anthropics/claude-plugins-official
```

---

### 2단계: 채널(Channels) 활성화하여 다시 시작

클로드 코드(Claude Code)를 종료한 다음, `--channels` 플래그(flag)와 함께 다시 시작합니다:

```bash
claude --channels plugin:fakechat@claude-plugins-official
```

위에서 볼 수 있듯이 Fakechat 채널이 활성화되었음을 확인하는 메시지를 확인하십시오. Fakechat 서버(server)가 시작되어 수신 대기(listening) 중입니다.

### 3단계: 브라우저(Browser)에서 Fakechat 열기

터미널(terminal)을 계속 실행 상태로 유지하십시오. 절대 닫지 마십시오.

브라우저(browser)를 열고 다음 주소로 이동합니다:

```
http://localhost:8787
```

위와 같은 간단한 채팅 인터페이스(chat interface)가 보일 것입니다. 이 단계에 도달하면 설정이 완료되어 테스트를 진행할 준비가 된 것입니다.

메시지를 입력합니다:

```
hey, what's in my working directory?
```

전송 버튼을 누른 후, 클로드 코드 터미널(Claude Code terminal)을 주시하십시오.

메시지는 `<channel source="fakechat">` 이벤트(event)로 도착합니다. 클로드(Claude)는 이를 읽고 작업을 수행한 후 Fakechat의 답변 도구(reply tool)를 호출하며, 그 답변이 화면에 나타납니다.

## 권한(Permissions)에 대한 간략한 참고 사항

클로드(Claude)가 채널 메시지를 처리하는 동안 권한 프롬프트(permission prompt)에 도달했는데 사용자가 터미널(terminal)에 없다면, 세션(session)은 일시 중지됩니다.

계속 진행하기 전에 로컬 승인(local approval)을 기다립니다.

무인 사용(unattended use)을 위해서는 `--dangerously-skip-permissions` 옵션이 있지만, 이는 전적으로 신뢰하는 환경에서만 사용해야 합니다. 이름 자체가 모든 것을 말해줍니다.

## 마무리 생각

Fakechat을 테스트한 결과, 채널 흐름은 예상대로 작동했습니다.

메시지는 브라우저(browser) 데모에서 클로드 코드 세션(Claude Code session)으로 이동하고, 클로드(Claude)가 이를 처리한 후 응답이 돌아오는 방식입니다.

Fakechat 데모는 로컬호스트(localhost)에서 개념을 증명하지만, 진정한 가치는 텔레그램(Telegram) 또는 디스코드(Discord)에 연결하여 휴대폰에서 실행 중인 클로드 코드 세션(Claude Code session)으로 작업을 보내기 시작할 때 드러납니다.

왓츠앱(WhatsApp), 시그널(Signal) 등과 같은 다른 플랫폼 지원도 추가되기를 바랍니다.

클로드 코드 채널(Claude Code Channels)은 아직 연구 프리뷰(research preview) 단계라는 점을 명심하십시오. `--channels` 플래그(flag) 구문(syntax) 및 프로토콜(protocol)은 피드백(feedback)에 따라 변경될 수 있습니다.

프리뷰(preview) 기간 동안에는 Anthropic이 관리하는 허용 목록(allowlist)의 플러그인(plugins)만 허용됩니다.

하지만 이 초기 단계에서도 이 기능이 매우 유용하다는 것은 분명합니다.

따라서 다음 시리즈에서는 다음 내용을 다룰 것입니다:

1.  **텔레그램(Telegram)을 클로드 코드(Claude Code)에 연결하기**
    봇파더(BotFather)로 봇(bot)을 생성하는 것부터 계정을 페어링(pairing)하고 휴대폰에서 실제 코딩 작업(coding tasks)을 보내는 것까지 텔레그램(Telegram)의 전체 설정을 자세히 안내해 드릴 것입니다.
2.  **디스코드(Discord)를 클로드 코드(Claude Code)에 연결하기**
    봇 생성, 권한(permissions) 설정, 그리고 텔레그램(Telegram)보다 디스코드(Discord)가 제공하는 추가 도구(메시지 기록 및 첨부 파일 다운로드)를 포함한 디스코드(Discord) 설정을 다룰 것입니다.

또한 두 채널을 동시에 실행하는 방법, 보안 강화(security lockdown), 엔터프라이즈 제어(enterprise controls), 그리고 채널(Channels)과 OpenClaw의 솔직한 비교도 다룰 것입니다.
