# 바이브 코딩(Vibe Coding)을 위한 Sonnet 4.6 vs Opus 4.6 테스트 — 승자는?

모델 출시 속도가 너무 빨라서 실제 진전과 단순한 점진적 개선을 구분하기 어렵습니다.

이번 주 Sonnet 4.6이 출시되면서 Opus 4.6과 직접 비교 테스트를 진행하기로 결정했습니다.

AI 연구소에서 발표하는 벤치마크(benchmark)는 점점 신뢰도가 떨어지고 있어, 저는 Converge에서 몇 가지 간단한 테스트 프롬프트(prompt)를 실행했습니다.

## 테스트

첫 번째 프롬프트는 타워 디펜스(Tower Defense) 게임을 만드는 것이었습니다. 이 작업은 주로 모델이 상태(state), 사용자 인터페이스(UI), 렌더링(rendering), 게임 로직(game logic)을 동시에 처리해야 하는 프런트엔드(frontend) 유형의 작업입니다.

사용한 프롬프트는 다음과 같습니다.

> 적들이 웨이브(wave)로 생성되고, 처치 시 돈을 벌며, 적들이 끝에 도달하면 라이프(life)를 잃는 고정 경로의 완전한 타워 디펜스 게임을 만드세요. 업그레이드가 가능한 최소 3가지 타워 유형(다른 사거리/피해량/공격 속도)을 포함하고, 타워 배치/판매/업그레이드 및 다음 웨이브 시작을 위한 간단한 UI를 추가하세요. 코드는 깔끔하고 모듈식(modular)으로 유지하고, 플레이 가능하며 균형 잡힌 최소 실행 가능 제품(MVP)을 제공하세요. 일시 정지/재시작 + 화면 통계(웨이브, 라이프, 돈)와 같은 기본적인 완성도(polish)를 포함하세요.

## 평가 기준 (The rubric)

단순히 "느낌(vibes)"만으로 평가하지 않도록 프롬프트를 체크리스트로 나누었습니다.

*   즉시 실행됨 (누락된 부분 없이 한 번에 실행)
*   고정 경로 + 적들이 웨이브로 생성
*   처치 시 돈 획득 + 누출 시 라이프 감소
*   3가지 타워 유형 (사거리/피해량/공격 속도)
*   업그레이드 작동
*   UI: 타워 배치/판매/업그레이드
*   다음 웨이브 시작 제어
*   기본적인 완성도: 일시 정지/재시작 + 화면 통계(웨이브/라이프/돈)
*   출시 가능한 MVP처럼 느껴짐 (망가진 데모 아님)

## 추가 테스트: ChatGPT 클론(clone) 구축

Sonnet 4.6이 타워 디펜스 게임을 상당히 쉽게 처리했기 때문에, 저는 이를 더 나아가 ChatGPT를 재현하도록 요청했습니다. 이 역시 Converge에서 수행되었습니다.

프롬프트는 다음과 같습니다.

> 다음과 같은 고급 기능을 포함하여 ChatGPT를 복제한 완전한 기능의 AI 채팅 애플리케이션을 만드세요.
>
> **핵심 기능 (Core Features):**
>
> *   문맥 인식(context awareness) 및 다중 턴(multi-turn) 대화를 통한 자연어 대화
> *   다양한 서식(formatting) (굵게, 기울임꼴, 코드 블록(code blocks))을 지원하는 텍스트 입력 및 출력
> *   실시간 입력 표시기(typing indicators) 및 메시지 전달 상태
> *   사용자 인증(authentication) 및 프로필 관리
> *   검색 및 내보내기(export) 옵션을 포함한 대화 기록
> *   사용자 설정(테마, 글꼴 크기, 알림 환경 설정) 사용자 지정 가능
>
> **고급 기능 (Advanced Functionalities):**
>
> *   멀티미디어(multimedia) 입력(이미지, 오디오) 처리 및 설명적인 답변 생성 기능
> *   과거 대화를 참조할 수 있는 문맥 메모리(contextual memory)
> *   사용자 상호 작용(interaction)을 기반으로 응답을 개인화하는 적응형 학습(adaptive learning)
>
> **사용자 인터페이스 디자인 (User Interface Design):**
>
> *   차분한 색상 팔레트(예: 짙은 남색 #1A1F36, 부드러운 청록색 #4FB6AC, 밝은 회색 #F5F7FA, 흰색)를 사용한 깔끔하고 현대적이며 미니멀리스트(minimalistic) 레이아웃(layout)
> *   명확한 계층 구조(hierarchy)와 충분한 여백(white space)을 가진 읽기 쉬운 산세리프(sans-serif) 타이포그래피(typography)
> *   데스크톱, 태블릿, 모바일 장치에 최적화된 반응형 디자인(responsive design)
> *   메시지 전환 및 상호작용 요소(interactive elements)를 위한 부드러운 애니메이션
> *   키보드 탐색(keyboard navigation), 화면 판독기(screen reader) 지원 및 충분한 대비를 갖춘 접근성 디자인(accessible design)
>
> **상호작용 및 피드백 (Interaction and Feedback):**
>
> *   사용자 행동(보내기, 받기, 오류)에 대한 명확한 시각적 피드백(visual feedback)
> *   향상된 커뮤니케이션 흐름을 위한 입력 표시기 및 읽음 확인(read receipts)
> *   빠른 상호작용을 위한 빠른 답장 제안 및 자동 완성(auto-complete)
>
> 이 애플리케이션이 장치 전반에 걸쳐 확장 가능하고 다양한 사용자 요구에 적응하며 직관적이고 신뢰할 수 있으며 매력적인 대화형 AI 경험을 제공하도록 보장하세요.

첫 번째 생성에서 많은 부분이 작동했지만, 몇 가지 기능이 누락되었습니다.

(생성된 채팅 애플리케이션 UI 이미지)

매우 인상 깊었던 점 중 하나는 스레드 간 검색(cross thread search)을 완벽하게 해냈다는 것입니다. 아래 이미지에서 제가 레이커스 팬(Lakers fan)이라고 언급한 것을 볼 수 있습니다.

(채팅 내용 이미지)

그리고 나서 새 채팅을 시작하고 제가 좋아하는 팀이 무엇인지 물었더니, 기억하고 있었습니다! 이 모든 것은 Converge의 에이전트(agent) 구성 요소(component)를 통해 내부적으로 구동됩니다.

(채팅 내용 이미지)

프롬프트를 체크리스트로 변환:

*   다중 턴 메모리(Multi-turn memory) (스레드별 + 스레드 간 문맥) — ✅
*   인증(Authentication) + 사용자 계정 (OAuth / SSO 지원) — ✅
*   영구적인 대화 기록(Persistent conversation history) (검색 + 내보내기) — ✅
*   전달 상태를 포함한 스트리밍 응답(Streaming responses) — ✅
*   스레드 간 검색(Cross thread search) — ✅
*   리치 텍스트(Rich text) + 코드 렌더링(code rendering) — ❌ (다소)
*   멀티모달(Multimodal) 입력 (이미지 + 오디오 업로드) — ✅
*   이미지 이해(Image understanding) — ✅
*   개인화 레이어(Personalization layer) (적응형 메모리) — ❌
*   파일 업로드 처리 — ✅
*   반응형(Responsive), 접근성 UI (데스크톱 → 모바일) — ✅
*   사용자 설정(User settings) (테마, 알림, 환경 설정) — ❌

## 결론 (Verdict)

Sonnet 4.6은 정말 멋진 모델입니다. 일주일 내내 테스트해 본 결과, Opus 4.6만큼 좋거나 그 이상이며, 가격도 더 저렴합니다.

AI 세계의 변화 속도는 멈출 줄 모릅니다!
