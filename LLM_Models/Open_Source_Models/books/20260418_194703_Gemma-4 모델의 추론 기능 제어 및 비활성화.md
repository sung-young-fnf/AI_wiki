Gemma-4 모델의 추론 기능 제어 및 비활성화

Gemma-4-25b 모델은 난베이지(Nanbeige) 모델의 사고 방식과 유사한 것으로 밝혀졌습니다. 이 Gemma-4 모델 변형은 매우 핵심적인 사고 능력을 지니고 있습니다. 그 사고는 핵심을 찌르며, 다른 일부 모델처럼 결과를 내기 전에 너무 길게 늘어지는 불필요한 생각들을 뱉어내는 경우는 보지 못했습니다. 하지만 아직 어려운 문제를 제시하지는 않았습니다.

Gemma-4에 시스템 프롬프트(system prompt)를 활성화하기 위해 chatybot을 업데이트하는 과정에서 추론(reasoning) 기능을 끄는 것이 상당히 어렵다는 것을 발견했습니다.

*   시스템 프롬프트가 없으면 추론 기능이 활성화됩니다.
*   시스템 프롬프트가 추론하지 말라고 지시해도 추론 기능은 여전히 활성화됩니다.
*   `<thinking>` 태그 대신 `<thought>` 태그를 사용합니다.

추론 기능을 확실히 비활성화하려면 시스템 프롬프트와 사용자 프롬프트(user prompt) 모두에 추론을 원하지 않는다고 명시해야 합니다. 가끔 어려운 질문을 던지면 추론 모드로 전환될 수 있습니다. 이 모델은 내부적으로 훈련되고 제어되기 때문에 제가 시도한 방법이 대부분의 경우 작동하는 것 같습니다.

또한 `<no thought>`와 같은 태그 유형은 모델에 어느 정도 영향을 미치는 것으로 보입니다.

```
chat --> /system "you are a helpful assitant.
chat --> <thought off> list 5 cities in Canada
Payload:
-----------------------------
{
  "model": "gemma-4-26b-a4b-it",
  "messages": [
    {
      "role": "system",
      "content": "you are a helpful assitant."
    },
    {
      "role": "user",
      "content": "<thought off> list 5 cities in Canada"
    }
  ],
  "temperature": 0.2
}
---- end of payload ---
<thought>*   Target: 5 cities.
    *   Location: Canada.
    *   Constraint: List format.

    *   Toronto (Ontario)
    *   Vancouver (British Columbia)
    *   Montreal (Quebec)
    *   Calgary (Alberta)
    *   Ottawa (Ontario)

    *   Toronto
    *   Vancouver
    *   Montreal
    *   Calgary
    *   Ottawa

    1. Toronto
    2. Vancouver
    3. Montreal
    4. Calgary
    5. Ottawa</thought>Here are 5 cities in Canada:

1. Toronto
2. Vancouver
3. Montreal
4. Calgary
5. Ottawa
```
위 예시에서 `system` 프롬프트에 아무런 지시가 없고 `user` 프롬프트에 `<thought off>` 태그가 있음에도 불구하고, 모델은 응답 전에 내부적인 사고 과정(`* Target: 5 cities.` 등으로 시작하는 `<thought>` 블록)을 거쳤습니다.

```
chat --> /system "you are a helpful assitant. no thought tokens"
chat --> <thought off> list 5 cities in Mexico
Payload:
-----------------------------
{
  "model": "gemma-4-26b-a4b-it",
  "messages": [
    {
      "role": "system",
      "content": "you are a helpful assitant. no thought tokens"
    },
    {
      "role": "user",
      "content": "<thought off> list 5 cities in Mexico"
    }
  ],
  "temperature": 0.2
}
---- end of payload ---
Here are 5 cities in Mexico:

1. Mexico City
2. Guadalajara
3. Monterrey
4. Cancún
5. Puebla
```
이 예시에서는 `system` 프롬프트에 `no thought tokens`라는 지시를 추가한 후, 모델이 `<thought>` 블록 없이 직접 답변을 생성했습니다.

신뢰할 수 있게 추론 기능을 끄려면 최소 3번 ( 'and'를 포함하면 4번) 지시해야 합니다. 즉, "Disable reasoning and thought"라고 말한 다음 `<no thought>` 태그를 사용하는 것입니다.
