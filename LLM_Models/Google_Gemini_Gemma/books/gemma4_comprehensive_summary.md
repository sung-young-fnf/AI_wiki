# Gemma 4 종합 분석 리포트

> 작성일: 2026-04-05 | Google 발표일: 2026-04-02

---

## 1. Gemma 4 개요

Google DeepMind가 Gemini 3와 동일한 연구/기술 기반으로 구축한 차세대 오픈 모델 패밀리. **Apache 2.0 라이선스**로 전환하여 상업적 이용에 제한이 없으며, 엣지 디바이스부터 대규모 서버까지 아우르는 4가지 모델 라인업을 제공한다.

---

## 2. 모델 라인업

| 모델 | 파라미터 | 컨텍스트 | 입력 모달리티 | 용도 |
|------|----------|----------|--------------|------|
| **E2B** | 2.3B (Dense) | 128K | 텍스트 + 이미지 + 오디오 + 비디오 | 모바일/IoT 엣지 디바이스 |
| **E4B** | 4.5B (Dense) | 128K | 텍스트 + 이미지 + 오디오 + 비디오 | 노트북/엣지 디바이스 |
| **26B A4B** | 총 26B / 활성 4B (MoE) | 256K | 텍스트 + 이미지 | 효율적 추론 (속도+성능 균형) |
| **31B** | 31B (Dense) | 256K | 텍스트 + 이미지 | 최고 품질 추론 |

### 핵심 포인트
- **E2B/E4B**: USM 스타일 Conformer 오디오 인코더 내장. 별도 STT 없이 네이티브 오디오 처리 가능
- **26B A4B (MoE)**: 128개 전문가 중 8개(+공유 1개)만 활성화하여 4B급 속도로 26B급 성능 달성
- **31B Dense**: Arena AI 오픈모델 리더보드 3위 (Elo 1452), 자기 크기 20배 모델을 능가

## 3. 아키텍처 혁신

### Per-Layer Embeddings (PLE)
입구에서 한 번만 수행하던 임베딩 조회를 모든 디코더 레이어에 잔차 신호로 제공. 플래시 메모리에 저장하고 필요 시에만 조회하여 VRAM이 부족한 모바일 기기에서도 고성능 모델 구동 가능.

### 공유 KV 캐시 (Shared KV Cache)
마지막 몇 레이어에서 이전 레이어의 Key-Value 텐서를 재사용. 256K 컨텍스트 처리 시 메모리 부담을 크게 절감. K=V 기법(Key와 Value 동일 설정) 및 GQA도 적용.

### 이중 RoPE (Dual RoPE) + 인터리빙 어텐션
로컬 슬라이딩 윈도우 어텐션(512~1024 토큰)과 글로벌 풀 컨텍스트 어텐션을 교차 배치. 마지막 레이어는 항상 글로벌 어텐션으로 고정하여 최종 출력 품질 보장. 연산량 대비 성능을 극대화하는 설계.

### 가변 종횡비 비전 인코더
2D RoPE와 소프트 토큰 버짓으로 이미지 원본 비율을 보존하며, 이미지당 토큰 할당량을 유연하게 조절 가능.

## 5. Apache 2.0 라이선스 전환의 의미

이전 Gemma 모델의 독자적 커스텀 라이선스에서 Apache 2.0으로 전환한 것은 벤치마크 성능보다 더 큰 전략적 의미를 가진다:
- 상업적 이용, 파인튜닝, 재배포에 **제한 없음**
- MAU(월간 활성 사용자) 제한 없음
- 법적 준수를 중시하는 기업 시장에서 Qwen/Mistral 대비 강력한 채택 근거

## 6. 오픈소스 커뮤니티 반응 및 생태계

출시 수 시간 만에 폭발적 대응:
- **MLX**: Apple Silicon 최적화 96개 양자화 변형 공개
- **Unsloth**: 미세 조정용 19개 패키지 즉시 출시 (표준 QLoRA 대비 메모리 70% 절감, 속도 2.5배)
- **ONNX**: WebGPU를 통한 브라우저 내 실행 체크포인트 공개
- **llama.cpp**: GGUF 지원
- **Ollama**: v0.20.0에서 Day 0 지원

## 7. 현재 알려진 문제점 (출시 72시간 기준)

### llama.cpp 충돌
이미지 입력 시 `GGML_ASSERT` 오류로 47건의 충돌 보고. 비전 인코더가 모든 이미지 토큰을 한 번에 로드해야 하지만 기본값 `ubatch=512`로는 처리 불가.

### Transformers 호환성
안정화 버전(5.4.0)이 Gemma 4를 인식하지 못함. 개발 브랜치(5.5.0.dev0)에서 직접 설치 필요:
```bash
pip install git+https://github.com/huggingface/transformers.git
```

### PEFT/LoRA 문제
표준 PEFT가 `Gemma4ClippableLinear` 레이어를 인식하지 못함. **Unsloth** 사용 권장.

### 도구 호출 성능
소형 모델의 경우 Qwen 3.5 4B가 도구 호출에서 Gemma 4보다 정확하다는 초기 보고 존재.

> 미세 조정 툴체인 완전 안정화까지 약 2~4주 예상

## 8. MacBook Pro M1 32GB에서 사용 가능한 Gemma 4 모델

Apple Silicon의 **통합 메모리(Unified Memory)** 아키텍처 덕분에 CPU/GPU가 메모리를 공유하여 LLM 추론에 유리하다. 32GB M1에서 실행 가능한 모델과 권장 설정은 다음과 같다.

### 모델별 메모리 요구량 및 실행 가능 여부

| 모델 | Q4 (4bit) 크기 | Q8 (8bit) 크기 | 32GB M1 실행 | 비고 |
|------|---------------|---------------|-------------|------|
| **E2B** (2.3B) | ~1.5GB | ~2.5GB | **매우 여유** | 어떤 양자화든 문제 없음 |
| **E4B** (4.5B) | ~3GB | ~5GB | **매우 여유** | Q8로도 쾌적, 오디오 입력 지원 |
| **26B A4B MoE** | **~18GB** | ~26GB | **Q4 추천** | 활성 파라미터 4B라 실제 메모리 압력 낮음 |
| **31B Dense** | **~20GB** | ~32GB | **Q4로 가능** | 컨텍스트 길이 제한적 사용 권장 |

### 추천 구성

#### 1순위: 26B A4B MoE (Q4_K_M) - **가장 추천**
```bash
ollama pull gemma4:26b-a4b-it-q4_K_M
ollama run gemma4:26b-a4b-it-q4_K_M
```
- 메모리: ~18GB → 32GB에서 14GB 여유
- 실제 추론 시 4B 파라미터만 활성화되어 빠른 속도
- 31B Dense와 성능 차이 근소 (LMArena 기준 11점 차)
- **성능 대비 효율이 가장 뛰어남**

#### 2순위: E4B (Q8_0) - 빠른 응답 + 오디오 지원
```bash
ollama pull gemma4:e4b
ollama run gemma4:e4b
```
- 메모리: ~5GB → 여유 넘침
- Q8(8bit)으로도 충분히 가벼움
- 네이티브 오디오 입력 지원 (음성 비서, 통번역 등)
- 간단한 작업에 빠른 응답

#### 3순위: 31B Dense (Q4_K_M) - 최고 품질, 타이트한 운영
```bash
ollama pull gemma4:31b
ollama run gemma4:31b
```
- 메모리: ~20GB → 12GB 여유 (OS + 앱 고려 시 빠듯)
- 컨텍스트 윈도우를 8192 이하로 제한 권장
- 최고 품질이 필요한 작업에 한정 사용

### M1 최적화 팁
```bash
# GPU 레이어 최대 활용 (통합 메모리이므로 항상 최대)
export OLLAMA_NUM_GPU=99

# MLX 프레임워크 — Ollama v0.19+에서 자동 적용
# Apple Silicon에서 별도 설정 없이 Metal 가속 사용
```

### MLX 직접 사용 (Ollama 외)
MLX 커뮤니티에서 96개의 양자화 변형이 공개되어 있어, Hugging Face에서 직접 다운로드하여 `mlx-lm` 으로 실행할 수도 있다:
```bash
pip install mlx-lm
mlx_lm.generate --model mlx-community/gemma-4-26b-a4b-it-4bit
```

## 9. 핵심 결론

1. **Apache 2.0 전환이 가장 큰 의미** — 벤치마크 1~2% 차이보다 라이선스 자유가 기업 채택에 결정적
2. **MoE 26B A4B가 가성비 킹** — 4B 활성 파라미터로 31B급 성능, M1 32GB에서 쾌적 구동
3. **엣지 AI에서 경쟁자 없음** — E2B/E4B의 네이티브 멀티모달(오디오 포함)은 현재 유일
4. **생태계가 빠르게 성숙 중** — 출시 72시간 만에 MLX, Ollama, Unsloth, ONNX 등 즉시 지원
5. **툴체인은 아직 불안정** — llama.cpp 충돌, Transformers 비호환 등 2~4주 안정화 예상
6. **M1 32GB 최적 선택**: 범용은 **26B A4B MoE (Q4)**, 빠른 응답은 **E4B (Q8)**, 최고 품질은 **31B Dense (Q4, 컨텍스트 제한)**
