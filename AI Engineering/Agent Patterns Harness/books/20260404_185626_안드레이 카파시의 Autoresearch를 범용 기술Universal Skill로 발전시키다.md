# 안드레이 카파시의 Autoresearch를 범용 기술(Universal Skill)로 발전시키다

저는 문서 저장소, 마크다운(Markdown) 파일, API 레퍼런스, 스타일 가이드, SEO 감사 등을 다루는 테크니컬 라이터(Technical Writer)입니다. 언어 모델을 학습시키거나 CUDA 커널을 작성하지는 않지만, 안드레이 카파시(Andrej Karpathy)가 `autoresearch`를 공개했을 때 그 아이디어에 완전히 매료되었습니다.

### 핵심 아이디어: 자율적인 실험과 개선
카파시의 프로젝트는 매우 단순하면서도 명쾌했습니다. AI 에이전트가 스스로 실험을 실행하고 결과를 측정하며, 효과가 있는 것은 유지하고 그렇지 않은 것은 버리는 과정을 반복하며 성능을 개선하는 것입니다. 카파시는 이를 소형 GPT 모델 학습에 사용했습니다. 에이전트가 학습 코드를 수정하고, 5분간 실행한 뒤 모델이 개선되었는지 확인하여 코드를 유지하거나 되돌리는(Revert) 과정을 자율적으로 수행하게 한 것입니다.

이후 닉 사라예브(Nick Saraev)라는 유튜버가 이 패턴을 텍스트-이미지 프롬프트 최적화에 적용하는 것을 보고, 저는 이것이 문서화나 프롬프트 최적화 등 측정 가능한 모든 작업에 적용될 수 있다는 것을 깨달았습니다.

| 구성 요소 | 카파시의 Autoresearch | 프롬프트 최적화 (Prompt Optimization) |
| :--- | :--- | :--- |
| **수정 대상** | train.py (코드) | prompt.txt (프롬프트) |
| **객관적 지표** | val_bpb (검증 손실값) | 40점 만점의 평가 점수 |
| **자동 측정** | evaluate_bpb() (테스트) | LLM을 활용한 기준별 Yes/No 평가 |
| **루프 로직** | Git 유지/되돌리기 | 최적 프롬프트 유지/되돌리기 |

### 범용 기술(Universal Skill) 구축 과정
저는 모든 저장소와 기술 스택에 적응할 수 있는 '범용 autoresearch 기술'을 구축하기로 했습니다. Cursor와 같은 AI 기반 코드 에디터를 활용하여 이 기술을 구체화했습니다.

#### 버전 1.1의 5단계 프로세스
1.  **저장소 탐색(Repo Discovery):** 코드베이스를 스캔하여 언어, 프레임워크, 목적 파악
2.  **대상 제안(Target Suggestions):** 테스트 품질, 문서 완결성, 접근성 등 최적화 대상 제안
3.  **지표 정의(Metric Definition):** 업계 표준에 기반한 4~6개의 이진(Yes/No) 평가 기준 자동 생성
4.  **기준선 설정(Baseline):** 초기 점수 확립을 위해 프롬프트 1회 실행
5.  **자율 루프(Autoresearch Loop):** 생성, 평가, 점수 기록, 변이(Mutation) 과정을 자율적으로 반복

#### 주요 개선 사항
AI를 통한 자기 비판을 거쳐 v1의 단점들을 보완했습니다.
*   **변이 전략(Mutation Strategy) 구체화:** 제약 조건 추가, 부정적 예시 추가, 언어 정제 등 6가지 명시적 변이 연산자를 정의했습니다.
*   **평가 격리(Eval Isolation):** 평가 시 프롬프트 문맥을 제외하고 결과물만 보고 판단하게 하여 객관성을 높였습니다.
*   **컨텍스트 윈도우 관리(Context Window Management):** 대화가 길어져 정보가 유실되는 것을 방지하기 위해 매 사이클마다 디스크에서 상태를 다시 읽어오도록 했습니다.
*   **검증 데이터셋(Validation Set):** 고정된 샘플과 순환 샘플을 조합하여 평가의 일관성을 유지했습니다.

### 실제 적용 사례: 기술 문서 SEO 최적화
Docusaurus로 구축된 문서 사이트의 SEO 성능을 높이기 위해 이 기술을 적용했습니다.

1.  **에이전트 스캔:** 저장소의 스택(Node.js, Markdown 등) 파악
2.  **지표 정의:** 제목 길이(60자 미만), 메타 설명(120-160자), 단일 H1 태그 사용, 이미지 대체 텍스트(Alt text) 유무 등 5개 기준 설정
3.  **자율 반복:**
    *   **Baseline:** 24/40점에서 시작
    *   **Run 4 (변이):** 부정적 예시(이미지, 스크린샷 등의 일반적 단어 사용 금지) 추가 → 31/40점
    *   **Run 9 (변이):** 대조 사례(Good/Bad) 추가 → 38/40점
    *   **Run 12:** 40/40점 달성 및 루프 종료

최종 결과물로 `best_prompt.txt`에 실전 테스트를 거친 최적화된 프롬프트가 저장되었습니다.

### 버전 2.0: 초기 집중도 향상
v1을 실제 프로젝트에 사용하며 발견한 문제는 초기 프롬프트가 너무 광범위하다는 점이었습니다. 이를 해결하기 위해 v2에서는 입력을 더 체계화했습니다.

*   **세 가지 필드 최적화 템플릿:** 대상(Target), 범위(Scope), 컨텍스트(Context)를 명확히 입력받아 에이전트가 엉뚱한 방향으로 나아가지 않게 했습니다.
*   **보편적 품질 차원(Universal Quality Dimensions):** 정확성, 성능, 보안, 유지보수성 등 9가지 차원을 바탕으로 저장소에 맞는 최적화 제안을 생성합니다.
*   **고객 중심 흐름:** 에이전트의 제안을 기다리기보다 사용자가 먼저 아이디어를 낼 수 있는 구조로 변경했습니다.

### 결론: AI가 AI를 최적화하는 시대
이 과정에서 가장 놀라운 점은 제가 기술 제작 과정에서 단 한 줄의 코드도 직접 작성하지 않았다는 것입니다. 자연어로 원하는 기능을 설명하고, AI의 비판을 받아 수정을 거듭했을 뿐입니다.

이제 테크니컬 라이터가 세계적인 머신러닝 연구자의 개념을 가져와 실제 업무에 바로 사용할 수 있는 수준의 도구를 만들 수 있는 시대가 되었습니다. 파이썬 전문가가 아니어도, 머신러닝 배경지식이 없어도 괜찮습니다. 아이디어를 가지고 무엇을 원하는지 정확히 설명할 수만 있다면 충분합니다. AI가 생성물을 만들고, 스스로 평가하며, 최고의 결과물을 찾아내는 이 자율적인 루프는 모든 분야에 적용될 수 있는 보편적인 패턴입니다.
