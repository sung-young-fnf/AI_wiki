_이미지를 전체 크기로 보려면 Enter 키를 누르거나 클릭하세요._

빠르게 발전하는 생성형 AI(Generative AI) 세상에서 웹 브라우징 기능은 다음 개척 과제(next frontier)가 되고 있습니다. 거대 언어 모델(Large Language Models, LLM)은 요약, 추론, 생성 작업을 수행할 수 있지만, 실시간 정보 처리에는 종종 어려움을 겪습니다.

모델이 실시간 웹(live web)을 대규모로 탐색하고, 검색하며, 해석할 수 있도록 설계된 AI 웹 브라우저 엔진인 BLAST를 살펴보겠습니다.

_게시물 전체를 볼 수 없고 읽기를 원하시면 다음을 이용하세요: "전체 게시물 읽기"_

스탠포드 MAST 팀이 개발한 BLAST는 AI 애플리케이션을 위한 고성능 서빙 엔진(serving engine) 역할을 하며, 상용(production-ready) 및 OpenAI 호환 인터페이스(OpenAI-compatible interface)를 통해 웹 접근성을 제공합니다.

## BLAST의 차별점

### 데모용이 아닌 프로덕션용으로 구축

대부분의 AI 브라우저는 실험용으로는 적합합니다. 하지만 BLAST는 실제 환경의 동시성(concurrency), 캐싱(caching), 비용 관리(cost management)를 위해 설계되어 개발자들이 웹 브라우징 기능을 프로덕션 시스템(production systems)에 안심하고 통합할 수 있게 합니다.

### OpenAI 호환 인터페이스

OpenAI 호환 API(OpenAI-compatible API)를 통해 개발자들은 기존 SDK나 에이전트(agents)와 BLAST를 코드 재작성 없이 연결할 수 있습니다.
`base_url`만 변경하면 LLM이 이제 실시간 웹을 탐색합니다.

### 자동 병렬 처리(Automatic Parallelisation)

BLAST는 동시 브라우징(concurrent browsing)을 위해 작업을 자동으로 분할합니다.

지원 기능:

*   태스크 병렬 처리(Task parallelism) (여러 독립적인 하위 작업)
*   데이터 병렬 처리(Data parallelism) (청크(chunk) 기반 크롤링)
*   First-of-N (하나가 완료되면 느린 병렬 작업을 취소)

이들을 결합하면 검색 및 요약 작업에서 최대 5배의 속도 향상을 가져옵니다.

### 지능형 캐싱(Intelligent Caching)

캐싱은 더 이상 단순히 결과에만 국한되지 않습니다. BLAST는 계획(plans)도 캐시합니다.
이는 LLM이 특정 사이트를 탐색하는 방법을 알면 다음 번에 해당 전략을 재사용하여 시간과 컴퓨팅 자원을 모두 절약할 수 있다는 의미입니다.

### 실시간 스트리밍(Real-Time Streaming)

BLAST는 생각(thoughts)과 스크린샷(screenshots)을 토큰과 유사한 델타(delta) 형태로 스트리밍합니다.
사용자 인터페이스(UI)는 모델이 추론하는 동안 모델이 보는 것을 표시하여 투명하고 상호작용적인 경험을 제공할 수 있습니다.

## 지금 BLAST가 중요한 이유

### LLM을 위한 프로덕션급 웹 브라우징

많은 "웹을 탐색하는 AI 에이전트" 데모는 확장성(scale) 문제로 실패합니다. BLAST는 서빙(serving)에 중점을 두어 동시성, 리소스 제어, 스트리밍 UX, 비용 제한 등을 통해 최종 사용자에게 안전하게 브라우징 기능을 제공할 수 있습니다.

### 개발자 친화적: OpenAI 호환성

기존 OpenAI SDK가 BLAST의 `base_url`을 가리키도록 설정하기만 하면 됩니다. 이러한 방식(OpenAI 호환 표면)은 여러 제공업체와 도구에서 빠르게 표준이 되어 통합 마찰을 줄이고 있습니다.

### 내장된 성능 레버

자동 병렬 처리(Automatic parallelism) (태스크/데이터/First-of-N), 브라우저 작업용 접두사 인식 캐싱(prefix-aware caching), 스트리밍 가능한 실행(streamable execution)은 다중 페이지, 다중 탭 크롤링(crawls)에 대화형 수준의 지연 시간(latencies)을 제공합니다.

## BLAST의 실제 모습

OpenAI 호환 API, 스트리밍, 동시성, 자동 병렬 처리를 갖춘 브라우저 증강 LLM(browser-augmented LLMs)을 위한 고성능 서빙 엔진입니다.

### 주요 기능

*   **OpenAI 호환 엔드포인트(OpenAI-compatible endpoints)** (`/v1/chat/completions` 및 `/v1/responses`)
    *   표준 OpenAI 클라이언트와 함께 작동하며, `base_url`만 설정하면 됩니다.
*   **실시간 스트리밍(Real-time streaming)**
    *   생각/액션을 텍스트로, 스크린샷을 토큰과 유사한 "델타(delta)" 이벤트(공백 없음)로 스트리밍하여 UI가 실시간 브라우징을 렌더링할 수 있게 합니다.
*   **자동 병렬 처리(Automatic parallelism)**
    *   태스크 병렬 처리(Task parallelism) (하위 작업을 동시에 생성)
    *   데이터 병렬 처리(Data parallelism) (페이지 청크(chunk)화; 더 작은 LLM으로 요약)
    *   First-of-N (먼저 완료된 사본을 가져오고 나머지는 취소)
    *   깊이 제한을 통한 중첩
    *   BLAST 문서는 데이터 병렬 처리로 약 5배, 청크 요약에 더 작은 LLM을 사용하여 약 2배의 비공식적인(unscientific) 속도 향상을 보고합니다.
*   **지능형 캐싱(Intelligent caching)**
    *   최종 결과뿐만 아니라 계획(plans) (LLM이 결과를 얻기 위해 생성한 단계)도 캐시하며, `no-cache`, `no-store`, `no-cache-plan`, `no-store-plan`과 같은 캐시 제어 플래그를 지원합니다. 프로그래밍 방식으로 지울 수 있는 영구 온디스크 캐시(persistent on-disk caches)도 지원합니다.
*   **동시성(Concurrency) 및 리소스 제어(resource control)**
    *   다중 테넌트 스케줄러(Multi-tenant scheduler), 비용 상한(cost caps), 메모리 제한(memory limits), 최대 동시 브라우저 수, 헤드리스(headless) 요구 사항, 브라우저 재사용 등을 제공합니다.

## 아키텍처(Architecture)

_이미지를 전체 크기로 보려면 Enter 키를 누르거나 클릭하세요._

**주요 구성 요소:**

*   **클라이언트 레이어(Client Layer):** OpenAI 호환 API를 통해 연결됩니다.
*   **스케줄러(Scheduler):** 동시 브라우징 작업을 관리합니다.
*   **캐시 관리자(Cache Manager):** 결과 및 계획 캐싱을 처리합니다.
*   **리소스 관리자(Resource Manager):** 메모리, 비용, 브라우저 사용을 제어합니다.
*   **플래너 + 실행기(Planner + Executors):** LLM을 사용하여 작업 결정 및 수행합니다.
*   **병렬 워커(Parallel Workers):** 분산된 브라우저 작업을 처리합니다.

이러한 구성 요소들이 함께 작동하여 속도와 비용에 최적화된 확장 가능한 '계획-실행-캐시-응답' 루프를 생성합니다.

### OpenAI 클라이언트를 BLAST로 지정

```python
from openai import OpenAI
client = OpenAI(api_key="not-needed", base_url="http://127.0.0.1:8000")
```

### 실시간 브라우징 스트리밍

```python
stream = client.responses.create(
    model="not-needed",
    input="Compare fried chicken reviews for top 10 fast food restaurants",
    stream=True
)
for event in stream:
    if event.type == "response.output_text.delta":
        print(event.delta if " " in event.delta else "<screenshot>", end="", flush=True)
```

더 자세한 내용은 [공식 GitHub 페이지](https://github.com/stanford-mast/blast)를 참조하세요.

## OpenAI 호환 인터페이스 사용

### 채팅 완료(Chat Completions)

```python
resp = client.chat.completions.create(
  model="not-needed",
  messages=[{"role":"user","content":"Find the 10 heaviest gorillas"}],
  stream=True
)
for chunk in resp:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")
```

"델타(delta)"를 스트리밍하는 것은 OpenAI의 의미론을 따르며, BLAST는 공백이 없는 델타를 스크린샷으로 처리합니다.

### 응답 API(Responses API)

```python
stream = client.responses.create(
  model="not-needed",
  input="Go to python.org and click Documentation",
  stream=True
)
for event in stream:
    # response.created, response.in_progress, response.output_text.delta, response.completed
    ...
```

`previous_response_id`를 사용하여 다중 턴 브라우징 세션(multi-turn browsing sessions)을 연결할 수 있습니다.

## 성능 플레이북(Performance playbook)

### 병렬 처리 활성화

`config.yaml`에서 태스크/데이터 병렬 처리와 (선택적으로) First-of-N을 활성화합니다.

```yaml
constraints:
  allow_parallelism:
    task: true
    data: true
    first_of_n: false
  max_parallelism_nesting_depth: 1
```

### LLM 크기 최적화

계획에는 더 큰 모델을 사용하고, 청크 요약에는 작고 빠른 모델을 사용하세요.

```yaml
constraints:
  llm_model: "openai:gpt-5"
  llm_model_mini: "openai:gpt-5-mini"
```

**팁:** 추론에는 `gpt-5`를, 요약에는 `gpt-5-mini`를 사용하여 품질 저하 없이 비용을 절감하세요.

### 계획 + 결과 캐싱 활용

캐시는 기본적으로 활성화되어 있으며, 요청별로 재정의할 수 있습니다.

```python
client.responses.create(
  model="not-needed",
  input="Search Python docs",
  cache_control="no-cache-plan"  # 여전히 결과 캐시를 사용
)
```

웜 스타트(warm starts)를 원한다면 실행 간에 캐시를 영구적으로 유지하세요.

```yaml
settings:
  persist_cache: true
```

API를 통해 캐시를 지우거나 캐시 디렉터리를 삭제하여 캐시를 지울 수 있습니다.

### 스트리밍으로 상호작용 유지

더 긴 탐색 중에도 사용자 참여를 유지하기 위해 생각/액션과 스크린샷을 스트리밍하세요.

## 프로덕션 환경에서의 운영

### 동시성 및 리소스 제한

동시 브라우저 수, 메모리, 비용을 제한합니다. BLAST는 `max_concurrent_browsers`, `max_memory`와 같은 제약 조건과 `max_cost_per_minute`와 같은 예산 보호 장치(budget guards)를 제공합니다.

### 관측 가능성(Observability) 및 로그

로그를 폴더로 보내고, 엔진 대 브라우저 작업을 다른 수준으로 로깅하며, 실행 중인 태스크, 브라우저, 메모리에 대한 지표(metrics)를 폴링(engine.get_metrics())할 수 있습니다.

### 보안 비밀(Secrets) 및 인증된 사이트

브라우징 대상에 인증이 필요한 경우, 사이트 자격 증명(예: JIRA)을 `settings.secrets_file_path`에 의해 참조되는 보안 비밀 파일에 넣으세요.

### 헤드리스(Headless) 및 브라우저 재사용

오버헤드(overhead)를 줄이기 위해 헤드리스 모드(headless mode)를 강제하고 여러 컨텍스트(contexts)에 걸쳐 브라우저 프로세스를 공유하세요.

## BLAST 비교

### 대 "브라우저 사용" 라이브러리

Browser Use와 같은 도구는 브라우저 제어를 위한 에이전트 로직/API에 중점을 둡니다. BLAST는 그 아래에서 속도, 캐싱, 스트리밍, 다중 테넌트 동시성(multi-tenant concurrency)에 최적화된 서빙 엔진으로, 사용자 대면 프로덕션 앱(user-facing production apps)을 위해 만들어졌습니다.

### 대 Playwright + 서버 직접 구축

직접 구축하는 경우 스케줄링, 캐시 의미론(계획 재사용 포함), 예산 보호 장치, 스트리밍 계약, 병렬 오케스트레이션(parallel orchestration)을 다시 구현해야 합니다. BLAST는 이러한 기본 요소들을 즉시 제공합니다.

### OpenAI 호환성 트렌드와의 정렬

많은 제공업체가 이제 OpenAI 호환 API(예: Together, Google Gemini)를 노출하고 있으며, 이는 BLAST의 인터페이스(surface area)를 기존 스택(stacks)에 쉽게 통합할 수 있도록 합니다.

## 예시: 제한된, 병렬 처리된, 캐시된 "비교 및 순위 지정" 작업

```yaml
# config.yaml
settings:
  persist_cache: true
  logs_dir: "blast-logs/"
  blastai_log_level: "info"

constraints:
  max_concurrent_browsers: 4
  max_memory: "6GB"
  max_cost_per_minute: 0.25
  allow_parallelism:
    task: true
    data: true
```

```python
from openai import OpenAI
client = OpenAI(api_key="not-needed", base_url="http://127.0.0.1:8000")

q = ("Compare laptop reviews for 5 models under ₹90k, "
     "score by display, battery and performance, then recommend top 2.")
stream = client.responses.create(model="not-needed", input=q, stream=True)
for ev in stream:
    if ev.type == "response.output_text.delta":
        if " " in ev.delta:  # thought/action
            print(ev.delta, end="", flush=True)
        else:                 # screenshot token
            print("<screenshot>", end="")
```

이 설정은 BLAST가 페이지 읽기를 병렬 처리하고, 유사한 프롬프트에서 캐시된 계획을 재사용하며, 비용 예산 내에서 유지되고, 에이전트가 작업하는 동안 사용자 참여를 유지하게 합니다.

## 제한 사항, 로드맵 및 향후 방향

BLAST는 안정성, 비용 최적화, 성능, 인간-팀 UX(human-teaming UX), 신뢰성 벤치마킹을 적극적으로 우선순위로 둡니다. 새로운 "브라우저 사용(browser-use)" 스택이나 비전 모델(vision model)을 발명하려고 의도적으로 노력하지 않고, 제약 조건 하에서 이들을 서빙하고 확장하는 데 중점을 둡니다.

연구 에이전트, 경쟁 정보 요약, 데이터 추출 파이프라인 또는 대규모 브라우징이 필요한 어시스턴트 UI를 구축한다면, BLAST는 그렇지 않으면 직접 구축해야 할 서빙 인체공학적 요소(serving ergonomics)를 제공합니다: 산업 표준 API 표면, 실시간 스트림, 성능 레버(병렬 처리 + 캐싱), 엄격한 리소스/비용 제어.

이는 데모용이 아닌, "진정으로 웹을 탐색하는 AI"를 프로덕션 환경에 배포하는 실용적인 방법입니다.
