다트(Dart) 및 플러터(Flutter) 커뮤니티는 항상 화면을 넘나드는 가능성의 한계를 확장해왔습니다. 여러분은 고품질의 멀티플랫폼(multi-platform) 애플리케이션(application)을 구축하는 데 개발자 경험(developer experience)을 타협할 필요가 없음을 보여주었습니다. 이제 우리는 동일한 "한 번 작성하여 어디서든 실행(write once, run anywhere)"이라는 철학을 AI 기반 기능 및 애플리케이션에 적용합니다.

젠킷 다트(Genkit Dart) 출시 발표 (프리뷰)

## 젠킷 다트를 선택하는 이유

젠킷 다트(Genkit Dart)는 다음과 같은 기능을 제공합니다.

*   **모델 불가지론적(Model-agnostic) API**: 구글(Google), 앤트로픽(Anthropic), 오픈AI(OpenAI) 및 오픈AI API 호환 모델을 기본적으로(out-of-the-box) 지원합니다. 단일 제공업체에 종속되지 않습니다.
*   **타입 안정성(Type safety)**: 다트(Dart)의 강력한 타입 시스템(type system)을 `schemantic` 패키지와 함께 사용하여 강력한 타입의 데이터(strongly typed data)를 생성하고 타입 안정성(type-safe) AI 플로우(flow)를 만듭니다.
*   **어디서든 코드 실행**: AI 로직(logic)을 한 번 작성하여 백엔드 서비스(backend service)로 실행하거나 플러터(Flutter) 앱 내에서 직접 실행할 수 있습니다.
*   **개발자 UI (Developer UI)**: 프롬프트(prompt)를 테스트하고, 트레이스(trace)를 확인하며, 플로우(flow)를 디버그(debug)할 수 있는 로컬호스트(localhost) 웹 UI(Web UI)를 포함합니다.
*   **완전한 AI 툴킷(toolkit)**: 구조화된 출력(structured output), 도구(tool), 다단계 플로우(multi-step flow), 관측 가능성(observability) 등 고품질 AI 기능을 구축하는 데 필요한 모든 것을 제공합니다.

## 모델 불가지론적 API

젠킷(Genkit)은 모든 LLM (Large Language Model) 제공업체를 지원하도록 설계되었으며, 이번 릴리스(release)에서는 구글(Google), 앤트로픽(Anthropic), 오픈AI(OpenAI) 및 오픈AI API 호환 모델을 기본적으로(out-of-the-box) 지원합니다. 이를 통해 최소한의 코드 변경으로 제공업체 간 전환이 가능합니다.

```dart
import 'package:genkit/genkit.dart';
import 'package:genkit_google_genai/genkit_google_genai.dart';
import 'package:genkit_anthropic/genkit_anthropic.dart';

void main() async {
  // 젠킷을 플러그인(plugin)으로 초기화
  final ai = Genkit(plugins: [
    googleAI(),
    anthropic(),
  ]);

  // 구글 제미나이(Gemini) 호출
  final geminiResponse = await ai.generate(
    model: googleAI.gemini('gemini-3.1-pro-preview'),
    prompt: 'Hello from Gemini',
  );

  // 앤트로픽 클로드(Claude) 호출
  final claudeResponse = await ai.generate(
    model: anthropic.model('claude-opus-4.6'),
    prompt: 'Hello from Claude',
  );
}
```

## 타입 안전성(Type-safe) AI 플로우(flow)

젠킷(Genkit)을 사용하면 AI 로직(logic)을 테스트 가능하고(testable), 관측 가능하며(observable), 배포 가능한(deployable) '플로우(flow)'라고 불리는 함수로 묶을 수 있습니다.

다음은 도구 호출(tool calling) 기능을 사용하여 강력한 타입의 입력 및 출력 스키마(schema)를 활용하는 여행 플래너(Travel Planner) 플로우(flow)의 예시입니다.

```dart
import 'package:genkit/genkit.dart';
import 'package:genkit_google_genai/genkit_google_genai.dart';
import 'package:schemantic/schemantic.dart';

part 'travel_flow.g.dart';

// schemantic으로 플로우 입력 스키마 정의
@Schema()
abstract class $TripRequest {
  String get destination;
  int get days;
}

// 도구 입력 스키마 정의
@Schema()
abstract class $WeatherRequest {
  @Field(description: 'The city name')
  String get city;
}

void main() async {
  // 젠킷 초기화 및 구글 AI 플러그인 등록
  final ai = Genkit(plugins: [googleAI()]);

  // 모델이 라이브(live) 데이터를 가져오기 위해 호출할 수 있는 도구 정의
  ai.defineTool(
    name: 'fetchWeather',
    description: 'Retrieves the current weather forecast for a given city',
    inputSchema: WeatherRequest.$schema,
    fn: (request, _) async => request.city.toLowerCase() == 'seattle' ? 'Rainy' : 'Sunny', 
  );

  // 강력한 타입의 관측 가능한 플로우 구성
  final tripPlannerFlow = ai.defineFlow(
    name: 'planTrip',
    inputSchema: TripRequest.$schema,
    outputSchema: .string(),
    fn: (request, _) async {
      // 모델과 도구를 사용하여 콘텐츠 생성
      final response = await ai.generate(
        model: googleAI.gemini('gemini-3.1-pro-preview'),
        prompt: 'Build a ${request.days}-day travel itinerary for ${request.destination}. '
                'Make sure to check the weather forecast first to suggest appropriate activities.',
        toolNames: ['fetchWeather'],
      );
      
      return response.text;
    },
  );

  // 플로우 실행
  final itinerary = await tripPlannerFlow(
    TripRequest(destination: 'Seattle', days: 3)
  );
  print(itinerary);
}
```

준비가 되면 `genkit_shelf` 패키지를 사용하여 플로우(flow)를 API로 쉽게 노출하고, 다트(Dart)를 지원하는 모든 플랫폼에 배포할 수 있습니다.

```dart
import 'package:genkit_shelf/genkit_shelf.dart';
import 'package:shelf_router/shelf_router.dart';
import 'package:shelf/shelf_io.dart' as io;

void main() async {
  // ... 젠킷을 초기화하고 tripPlannerFlow를 정의 ...

  final router = Router()
    ..post(
      '/api/planTrip',
      shelfHandler(tripPlannerFlow),
    );

  await io.serve(router.call, 'localhost', 8080);
}
```

## 다트가 실행되는 모든 곳에서 실행

대부분의 복잡한 AI 로직(logic)은 서버에서 실행됩니다. 하지만 다트(Dart)는 프런트엔드(frontend)와 백엔드(backend) 모두에서 작동하므로, 젠킷(Genkit)을 사용하면 AI 코드를 서버와 플러터(Flutter) 앱 간에 쉽게 이동할 수 있습니다.

젠킷 다트(Genkit Dart)로 구축할 수 있는 몇 가지 방법은 다음과 같습니다.

### 1. 프로토타이핑(prototyping)을 위한 플러터(Flutter) 내 전체 구현

모델 호출(model call)을 포함한 모든 젠킷(Genkit) 로직(logic)을 플러터(Flutter) 앱 내에서 직접 작성할 수 있습니다. 이는 사용자가 자체 API 키(API key)를 제공하고 프롬프트(prompt)가 비공개가 아닌 프로토타입(prototype) 또는 앱에 적합합니다.

**경고**: API 키(API key)가 소스 코드(source code)에 내장된 앱은 추출되어 타인에 의해 사용될 수 있으므로 절대 게시해서는 안 됩니다.

### 2. 플러터(Flutter)에서 백엔드(backend) 플로우(flow) 호출

프롬프트(prompt)가 민감하거나 AI 로직(logic)이 복잡한 경우, 전체 플로우(flow)를 백엔드(backend)로 옮길 수 있습니다. 플러터(Flutter) 앱은 이를 "원격 액션(remote action)"으로 정의하여 이 플로우를 호출할 수 있습니다. 백엔드와 프런트엔드(frontend)가 모두 다트(Dart)로 되어 있기 때문에, 종단 간(end-to-end) 타입 안전성(type safety)을 위해 동일한 스키마(schema)를 공유할 수 있습니다.

다음은 앞에서 정의한 여행 플래너 백엔드 플로우를 플러터 앱에서 호출하는 방법을 보여주는 예시입니다.

```dart
import 'package:genkit/client.dart';
import 'package:my_shared_models/models.dart'; // 공유 스키마

final tripPlannerFlow = defineRemoteAction(
  url: 'https://your-server.com/api/planTrip',
  inputSchema: TripRequest.$schema,
  outputSchema: .string(),
);

final itinerary = await tripPlannerFlow(
  input: TripRequest(destination: 'Tokyo', days: 5),
);
```

### 3. 원격 모델(remote model)을 사용한 플러터(Flutter) 앱

핵심 AI 로직(logic)을 플러터(Flutter) 앱에 유지하면서 API 키(API key)를 안전하게 보호하려면, 맞춤형 인증 로직(authorization logic)을 사용하여 모델 제공업체에 요청을 프록시(proxy)하는 작은 젠킷(Genkit) 백엔드(backend)를 만들 수 있습니다. 이 백엔드를 통해 노출되는 모델은 원격 모델(remote model)입니다.

```dart
import 'package:genkit_google_genai/genkit_google_genai.dart';
import 'package:genkit_shelf/genkit_shelf.dart';
import 'package:shelf_router/shelf_router.dart';
import 'package:shelf/shelf_io.dart' as io;

// 백엔드는 모델에 대한 요청을 안전하게 프록시합니다.
void main() async {
  final geminiApi = googleAI();
  final targetModel = geminiApi.model('gemini-3.1-flash-lite-preview');
  final router = Router()
    ..post(
      '/api/gemini-model',
      shelfHandler(
        targetModel,
        // 여기에 맞춤형 인증 로직 삽입
        contextProvider: (req) async => {'customAuth': true}, 
      ),
    );
  await io.serve(router.call, 'localhost', 8080);
}
```

플러터(Flutter) 앱에서는 직접 모델 플러그인(plugin) 대신 원격 모델(remote model)을 사용하고, 서버에 필요한 모든 헤더(header)를 전달합니다. 이렇게 하면 API 키(API key) 노출을 방지하고 요청 인증(authorization)에 대한 더 많은 제어를 할 수 있습니다.

```dart
import 'package:genkit/genkit.dart';

// 플러터 앱이 프록시 서버와 통신합니다.
final ai = Genkit();
final secureModel = ai.defineRemoteModel(
  name: 'secureModel',
  url: 'https://api.yourdomain.com/api/gemini-model',
  headers: (context) => {'Authorization': 'Bearer ${fetchSessionToken()}'}, 
);
final response = await ai.generate(model: secureModel, prompt: 'Write me a poem.');
```

## AI 개발을 위한 강력한 도구

고품질 AI 애플리케이션(application)을 구축하려면 신뢰할 수 있는 결과를 얻기 위해 철저한 테스트(testing)와 지속적인 반복(iteration)이 필요합니다. 이를 돕기 위해 젠킷(Genkit)은 강력한 로컬 개발자 UI (Developer UI)를 제공합니다.

젠킷 CLI (Command Line Interface)를 사용하여 앱을 실행함으로써 코드와 함께 개발자 UI를 시작할 수 있습니다.

```bash
genkit start -- dart run bin/server.dart
```

다음은 개발자 UI에서 여행 플래너 플로우(flow)의 고급 버전을 테스트하는 모습입니다.
