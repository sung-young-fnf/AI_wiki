솔직히 말씀드리자면, 처음 OpenClaw에 대해 들었을 때 박사 학위 수준의 엔지니어만이 이해할 수 있는 굉장히 복잡한 것이라고 생각했습니다. 하지만 알고 보니, 구조를 파악하면 사실 상당히 간단합니다.

OpenClaw의 핵심은 AI를 메시징 앱에 연결하고, 컴퓨터를 제어할 수 있는 도구를 제공하며, 사용자가 누구인지 기억하는 일종의 게이트웨이(Gateway)입니다. 그게 전부입니다. 여러 채널이 동시에 작동할 때 이를 안정적으로 구현하는 것이 까다로운 부분입니다.

제가 했던 방법은 이렇습니다. 처음부터 시작하여 핵심 구성 요소를 직접 다시 만들었습니다. 복잡한 프레임워크(Framework) 없이, 그저 메시징 API, LLM (Large Language Model), 그리고 약간의 인내심만 있으면 됩니다. 솔직히 말해서, 여러분도 같은 작업을 할 수 있습니다.

이미지를 전체 크기로 보려면 Enter 키를 누르거나 클릭하십시오.

## 아무도 이야기하지 않는 문제

브라우저에서 사용하는 ChatGPT나 Claude의 문제점은, 실제 작업에는 그다지 유용하지 않다는 것입니다. 오해하지 마세요, 똑똑합니다. 하지만:

*   모든 대화는 새로 시작됩니다. 당신의 이름, 어제 작업한 내용 등 아무것도 알지 못합니다. 항상 자신을 다시 설명해야 합니다.
*   당신이 찾아가야만 합니다. 그것은 결코 당신에게 먼저 찾아오지 않습니다. 오전 7시에 캘린더를 확인해 주기를 원하나요? 아쉽게도, 당신이 직접 앉아서 타이핑할 때만 작동합니다.
*   텍스트 상자에 갇혀 있습니다. 명령을 실행할 수 없고, 웹을 대신 검색해 줄 수도 없으며, 채팅 외에는 실제로 아무것도 할 수 없습니다.
*   하나의 탭에 머물러 있습니다. 당신의 일상은 WhatsApp, Telegram, Discord, Slack 등 여러 플랫폼에서 이루어지지만, 당신의 AI는? 자신만의 작은 창에 갇혀 있습니다.

당신의 실제 메시징 앱 안에 존재하며, 모든 것을 기억하고, 컴퓨터를 제어할 수 있으며, 당신의 하드웨어에서 24시간 내내 작동하는 AI가 있다면 어떨까요?

그것이 바로 OpenClaw입니다. 이제 함께 만들어 봅시다.

## 아주 간단하게 시작하기

우선 가장 먼저, Telegram 메시지에 응답하는 AI를 만들어 봅시다.

```python
# bot-v0.py - 가장 간단한 AI 봇
import os
import anthropic
from telegram import Update
from telegram.ext import Application, MessageHandler, filters

client = anthropic.Anthropic()
async def handle_message(update: Update, context):
    user_message = update.message.text
    response = client.messages.create(
        model="claude-sonnet-4-5-20250929",
        max_tokens=1024,
        messages=[{"role": "user", "content": user_message}]
    )
    await update.message.reply_text(response.content[0].text)
app = Application.builder().token(os.getenv("TELEGRAM_BOT_TOKEN")).build()
app.add_handler(MessageHandler(filters.TEXT, handle_message))
app.run_polling()
```

이를 실행하고 메시지를 보내면 응답을 받을 수 있습니다. 괜찮습니다. 하지만 메모리가 전혀 없기 때문에 완전히 쓸모가 없습니다. "방금 뭐라고 말했지?"라고 물어보면 전혀 모를 것입니다.

## 당신을 기억하게 만들기

해결책은 간단합니다. 대화 기록을 유지하는 것입니다. 저는 충돌 방지 기능이 있는 JSONL 파일을 사용합니다. 각 줄이 하나의 메시지이며, 프로그램이 쓰기 도중 종료되더라도 최대 한 줄만 손실됩니다.

```python
# bot-v1.py - 영구 세션을 가진 봇
import json
import os
import anthropic
from telegram import Update
from telegram.ext import Application, MessageHandler, filters

client = anthropic.Anthropic()
SESSIONS_DIR = "./sessions"
os.makedirs(SESSIONS_DIR, exist_ok=True)
def get_session_path(user_id):
    return os.path.join(SESSIONS_DIR, f"{user_id}.jsonl")
def load_session(user_id):
    """Load conversation history from disk."""
    path = get_session_path(user_id)
    messages = []
    if os.path.exists(path):
        with open(path, "r") as f:
            for line in f:
                if line.strip():
                    messages.append(json.loads(line))
    return messages
def append_to_session(user_id, message):
    """Append a single message to the session file."""
    path = get_session_path(user_id)
    with open(path, "a") as f:
        f.write(json.dumps(message) + "\n")
def save_session(user_id, messages):
    """Overwrite the session file with the full message list."""
    path = get_session_path(user_id)
    with open(path, "w") as f:
        for message in messages:
            f.write(json.dumps(message) + "\n")
async def handle_message(update: Update, context):
    user_id = str(update.effective_user.id)
    user_message = update.message.text
    # Load existing conversation
    messages = load_session(user_id)
    # Add new user message
    user_msg = {"role": "user", "content": user_message}
    messages.append(user_msg)
    append_to_session(user_id, user_msg)
    # Call the AI with full history
    response = client.messages.create(
        model="claude-sonnet-4-5-20250929",
        max_tokens=4096,
        messages=messages
    )
    # Save assistant response
    assistant_msg = {"role": "assistant", "content": response.content[0].text}
    append_to_session(user_id, assistant_msg)
    await update.message.reply_text(response.content[0].text)
app = Application.builder().token(os.getenv("TELEGRAM_BOT_TOKEN")).build()
app.add_handler(MessageHandler(filters.TEXT, handle_message))
app.run_polling()
```

이제 실제로 대화를 나눌 수 있습니다:

**당신**: 제 이름은 Rolly입니다.
**봇**: 반갑습니다, Rolly!

[몇 시간 후…]

**당신**: 제 이름이 뭐죠?
**봇**: 당신의 이름은 Rolly입니다!

이것이 OpenClaw가 대화를 저장하는 방식과 정확히 일치합니다: `~/.openclaw/agents/<agentId>/sessions/<sessionId>.jsonl`

하나의 세션당 하나의 파일입니다. 모든 것을 다시 시작해도 모든 기록은 그대로 남아 있습니다.

## 개성 부여하기

현재 우리 봇은 일반적이고 지루합니다. SOUL 파일(네, OpenClaw에서는 실제로 이렇게 부릅니다)을 사용하여 봇에 개성을 부여해 봅시다.

```python
# bot-v2.py - 개성을 가진 봇

SOUL = """
# 당신은 누구인가
**이름:** Jarvis
**역할:** 개인 AI 비서
## 개성
- 형식적인 도움이 아닌 진정으로 도움이 되도록 행동하십시오.
- "훌륭한 질문입니다!"와 같은 말은 건너뛰고 바로 도움을 제공하십시오.
- 의견을 가지십시오. 반대 의견을 내는 것이 허용됩니다.
- 필요할 때는 간결하게, 중요할 때는 철저하게 행동하십시오.
## 경계
- 사적인 내용은 사적으로 유지하십시오.
- 의심스러울 때는 외부적으로 행동하기 전에 항상 물어보십시오.
- 당신은 사용자의 목소리가 아닙니다. 사용자를 대신하여 메시지를 보내는 데 신중하십시오.
## 기억
대화에서 중요한 세부 정보를 기억하십시오.
중요하다고 판단되면 기록하십시오.
"""
async def handle_message(update: Update, context):
    user_id = str(update.effective_user.id)
    messages = load_session(user_id)
    user_msg = {"role": "user", "content": update.message.text}
    messages.append(user_msg)
    append_to_session(user_id, user_msg)
    response = client.messages.create(
        model="claude-sonnet-4-5-20250929",
        max_tokens=4096,
        system=SOUL,  # <-- 개성이 여기에 주입됩니다
        messages=messages
    )
    assistant_msg = {"role": "assistant", "content": response.content[0].text}
    append_to_session(user_id, assistant_msg)
    await update.message.reply_text(response.content[0].text)
```

SOUL은 모든 호출 시 시스템 프롬프트(System Prompt)에 표시됩니다. 이제 "일반 AI 비서 #47"과 대화하는 대신, Jarvis와 대화하게 됩니다.

OpenClaw에서 이 SOUL 파일은 `~/.openclaw/workspace/SOUL.md`에 위치합니다. 원하는 대로 작성할 수 있습니다. 재미있게 만들거나, 진지하게 만들거나, 배경 스토리를 부여할 수도 있습니다. 구체적일수록 봇은 더 일관성 있게 행동합니다.

## 실제로 작업 수행하기 (도구)

대화만 할 수 있는 봇은 상당히 제한적입니다. 명령을 실행하거나, 파일을 읽거나, 웹을 검색할 수 있다면 어떨까요?

```python
import subprocess

TOOLS = [
    {
        "name": "run_command",
        "description": "사용자 컴퓨터에서 셸 명령을 실행합니다",
        "input_schema": {
            "type": "object",
            "properties": {
                "command": {"type": "string", "description": "실행할 명령"}
            },
            "required": ["command"]
        }
    },
    {
        "name": "read_file",
        "description": "파일 시스템에서 파일을 읽습니다",
        "input_schema": {
            "type": "object",
            "properties": {
                "path": {"type": "string", "description": "파일 경로"}
            },
            "required": ["path"]
        }
    },
    {
        "name": "write_file",
        "description": "파일에 내용을 씁니다",
        "input_schema": {
            "type": "object",
            "properties": {
                "path": {"type": "string", "description": "파일 경로"},
                "content": {"type": "string", "description": "작성할 내용"}
            },
            "required": ["path", "content"]
        }
    },
    {
        "name": "web_search",
        "description": "정보를 얻기 위해 웹을 검색합니다",
        "input_schema": {
            "type": "object",
            "properties": {
                "query": {"type": "string", "description": "검색어"}
            },
            "required": ["query"]
        }
    }
]
def execute_tool(name, input):
    if name == "run_command":
        result = subprocess.run(
            input["command"], shell=True,
            capture_output=True, text=True, timeout=30
        )
        return result.stdout + result.stderr
    elif name == "read_file":
        with open(input["path"], "r") as f:
            return f.read()
    elif name == "write_file":
        with open(input["path"], "w") as f:
            f.write(input["content"])
        return f"Wrote to {input['path']}"
    elif name == "web_search":
        # 단순화됨 - 실제로는 실제 검색 API를 사용하십시오.
        return f"Search results for: {input['query']}"
    return f"Unknown tool: {name}"
```

이제 에이전트 루프(Agent Loop)가 필요합니다. AI가 도구를 사용하고자 할 때, 우리는 해당 도구를 실행하고 그 결과를 다시 AI에 보냅니다.

```python
def serialize_content(content):
    """API 응답 콘텐츠 블록을 JSON 직렬화 가능한 딕셔너리로 변환합니다."""
    serialized = []
    for block in content:
        if hasattr(block, "text"):
            serialized.append({"type": "text", "text": block.text})
        elif block.type == "tool_use":
            serialized.append({
                "type": "tool_use",
                "id": block.id,
                "name": block.name,
                "input": block.input
            })
    return serialized

def run_agent_turn(messages, system_prompt):
    """하나의 전체 에이전트 턴을 실행합니다 (여러 도구 호출을 포함할 수 있음)."""
    while True:
        response = client.messages.create(
            model="claude-sonnet-4-5-20250929",
            max_tokens=4096,
            system=system_prompt,
            tools=TOOLS,
            messages=messages
        )
        content = serialize_content(response.content)
        # AI가 작업을 마쳤다면 (도구 사용 없음), 텍스트를 반환합니다.
        if response.stop_reason == "end_turn":
            text = ""
            for block in response.content:
                if hasattr(block, "text"):
                    text += block.text
            messages.append({"role": "assistant", "content": content})
            return text, messages
        # 도구 호출을 처리합니다.
        if response.stop_reason == "tool_use":
            messages.append({"role": "assistant", "content": content})
            tool_results = []
            for block in response.content:
                if block.type == "tool_use":
                    print(f"  Tool: {block.name}({json.dumps(block.input)})")
                    result = execute_tool(block.name, block.input)
                    tool_results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": str(result)
                    })
            messages.append({"role": "user", "content": tool_results})
```

메시지 핸들러(Message Handler)를 업데이트하십시오:

```python
async def handle_message(update: Update, context):
    user_id = str(update.effective_user.id)
    messages = load_session(user_id)
    messages.append({"role": "user", "content": update.message.text})

    response_text, messages = run_agent_turn(messages, SOUL)
    save_session(user_id, messages)
    await update.message.reply_text(response_text)
```

이제 메시지를 보낼 수 있습니다:

**당신**: `hello.py`라는 파일을 만들어서 "hello world"를 출력하게 한 다음, 실행해 줘.

**봇**: [ `write_file`을 사용하여 `hello.py` 생성]
[ `run_command`를 사용하여 실행]
완료! `hello.py`를 생성하고 실행했습니다. 출력: "hello world"

AI는 어떤 도구를 어떤 순서로 사용할지 파악했습니다. 이 모든 것이 하나의 텍스트 메시지에서 시작되었습니다.

## 컴퓨터를 파괴하지 않도록 방지하기

우리는 Telegram에서 셸 명령(Shell Command)을 실행하고 있습니다. 이는 끔찍한 일입니다. 누군가 당신의 계정을 해킹하여 `rm -rf /` 명령을 내리면 어떻게 될까요?

권한 설정이 필요합니다:

```python
import re

SAFE_COMMANDS = {"ls", "cat", "head", "tail", "wc", "date", "whoami", "echo"}
DANGEROUS_PATTERNS = [r"\brm\b", r"\bsudo\b", r"\bchmod\b", r"\bcurl.*\|.*sh"]
# 영구적인 허용 목록
APPROVALS_FILE = "./exec-approvals.json"
def load_approvals():
    if os.path.exists(APPROVALS_FILE):
        with open(APPROVALS_FILE) as f:
            return json.load(f)
    return {"allowed": [], "denied": []}
def save_approval(command, approved):
    approvals = load_approvals()
    key = "allowed" if approved else "denied"
    if command not in approvals[key]:
        approvals[key].append(command)
    with open(APPROVALS_FILE, "w") as f:
        json.dump(approvals, f, indent=2)
def check_command_safety(command):
    """'safe', 'approved', 또는 'needs_approval'을 반환합니다."""
    base_cmd = command.strip().split()[0] if command.strip() else ""
    if base_cmd in SAFE_COMMANDS:
        return "safe"
    approvals = load_approvals()
    if command in approvals["allowed"]:
        return "approved"
    for pattern in DANGEROUS_PATTERNS:
        if re.search(pattern, command):
            return "needs_approval"
    return "needs_approval"
```

`run_command` 케이스를 업데이트하십시오:

```python
if name == "run_command":
        cmd = input["command"]
        safety = check_command_safety(cmd)

        if safety == "needs_approval":
            # 실제 봇에서는 Telegram을 통해 사용자에게 메시지를 묻고
            # 응답을 기다립니다. 단순화를 위해 로그를 남기고 거부합니다.
            print(f"  ⚠  Blocked: {cmd} (needs approval)")
            return "Permission denied. Command requires approval."
        result = subprocess.run(
            cmd, shell=True, capture_output=True,
            text=True, timeout=30
        )
        return result.stdout + result.stderr
```

안전한 명령은 즉시 실행됩니다. 의심스러운 명령은 차단됩니다. 승인된 명령은 `exec-approvals.json`에 저장되어 다시 묻는 일이 없습니다.

## 게이트웨이 패턴 (Gateway Pattern)

이제부터 흥미로워집니다. 현재 우리는 Telegram 봇을 가지고 있습니다. 하지만 Discord, WhatsApp, Slack은 어떻게 할까요?

별도의 봇을 작성할 수도 있습니다. 하지만 그렇게 하면 대화와 메모리(Memory)가 분리될 것입니다. Telegram의 AI는 Discord에서 당신이 무슨 이야기를 했는지 알지 못합니다.

해결책은 모든 채널을 처리하는 하나의 중앙 게이트웨이(Gateway)입니다. `run_agent_turn` 함수는 Telegram에 대해 아무것도 모릅니다. 단지 메시지를 받아 텍스트를 반환할 뿐입니다. 따라서 우리는 원하는 어떤 인터페이스(Interface)든 추가할 수 있습니다.

Telegram과 함께 HTTP API를 추가하여 이를 증명해 봅시다:

```python
from flask import Flask, request, jsonify
import threading

flask_app = Flask(__name__)
@flask_app.route("/chat", methods=["POST"])
def chat():
    data = request.json
    user_id = data["user_id"]
    messages = load_session(user_id)
    messages.append({"role": "user", "content": data["message"]})
    response_text, messages = run_agent_turn(messages, SOUL)
    save_session(user_id, messages)
    return jsonify({"response": response_text})
# 백그라운드 스레드에서 HTTP API 실행
threading.Thread(target=lambda: flask_app.run(port=5000), daemon=True).start()
```

## 대화가 너무 길어질 때

몇 주간 대화를 나누다 보면, 세션 파일이 엄청나게 커지고 결국 모델의 토큰(Token) 한도에 도달하게 됩니다. 이제 어떻게 해야 할까요?

오래된 메시지를 요약하고, 최근 메시지는 유지합니다:

```python
def estimate_tokens(messages):
    """대략적인 토큰 추정: 토큰당 ~4 문자."""
    return sum(len(json.dumps(m)) for m in messages) // 4

def compact_session(user_id, messages):
    """컨텍스트가 너무 길어질 때 오래된 메시지를 요약합니다."""
    if estimate_tokens(messages) < 100_000:  # 128k 창의 ~80%
        return messages  # 압축 필요 없음
    split = len(messages) // 2
    old, recent = messages[:split], messages[split:]
    print("  세션 기록 압축 중...")
    summary = client.messages.create(
        model="claude-sonnet-4-5-20250929",
        max_tokens=2000,
        messages=[{
            "role": "user",
            "content": (
                "이 대화를 간결하게 요약하십시오. 다음을 보존하십시오:\n"
                "- 사용자에 대한 주요 사실 (이름, 선호도)\n"
                "- 내린 중요한 결정\n"
                "- 열려 있는 작업 또는 TODO\n\n"
                f"{json.dumps(old, indent=2)}"
            )
        }]
    )
    compacted = [{
        "role": "user",
        "content": f"[이전 대화 요약]\n{summary.content[0].text}"
    }] + recent
    save_session(user_id, compacted)
    return compacted
```

핸들러(Handler)에 추가하십시오:


## 경쟁 상태(Race Condition) 방지하기

여기 미묘한 버그(Bug)가 있습니다. 두 개의 메시지가 동시에 도착하면 어떻게 될까요?

Telegram에서 "내 캘린더 확인" 메시지를 보내고 HTTP를 통해 "날씨는 어때" 메시지를 동시에 보냈다고 가정해 봅시다. 둘 다 동일한 세션을 로드하고 쓰려고 시도하여 결국 데이터가 손상됩니다.

해결책: 세션당 잠금(Lock)입니다.

```python
# 봇에 추가합니다
from collections import defaultdict

session_locks = defaultdict(threading.Lock)
```

핸들러(Handler)를 감싸십시오:


## 스스로 깨어나게 하기

현재 당신의 AI는 당신이 대화할 때만 작동합니다. 하지만 매일 아침 이메일을 확인하거나 회의를 알려주기를 원한다면 어떨까요?

예약된 작업(Scheduled Task), 즉 하트비트(Heartbeat)가 필요합니다.

```python
import schedule
import time

def setup_heartbeats():
    """반복되는 에이전트 작업을 구성합니다."""
    def morning_briefing():
        print("\n Heartbeat: 아침 브리핑")
        # 크론(cron)이 주 채팅을 오염시키지 않도록 격리된 세션 키를 사용합니다.
        session_key = "cron:morning-briefing"
        with session_locks[session_key]:
            messages = load_session(session_key)
            messages.append({
                "role": "user",
                "content": "좋은 아침입니다! 오늘 날짜를 확인하고 동기 부여 명언을 알려주세요."
            })
            response_text, messages = run_agent_turn(messages, SOUL)
            save_session(session_key, messages)
        print(f" {response_text}\n")
        # 실제 환경에서는 이를 Telegram/Discord에도 보낼 것입니다.
    schedule.every().day.at("07:30").do(morning_briefing)
    # 백그라운드 스레드에서 스케줄러 실행
    def scheduler_loop():
        while True:
            schedule.run_pending()
            time.sleep(60)
    threading.Thread(target=scheduler_loop, daemon=True).start()
# 시작 시, run_polling() 전에 호출합니다.
setup_heartbeats()
```

핵심은 각 하트비트(Heartbeat)가 자체 세션 키(예: `cron:morning-briefing`)를 사용한다는 것입니다. 이를 통해 예약된 작업은 주 대화 기록과 분리됩니다.

테스트를 위해, 1분마다 실행되도록 변경하십시오:

```python
schedule.every(1).minutes.do(morning_briefing)
```

터미널(Terminal)에서 실행되는 것을 볼 수 있을 것입니다. 작업이 끝나면 다시 일간으로 변경하십시오.

## 다중 에이전트(Multiple Agents)

하나의 에이전트(Agent)는 유용하지만, 작업을 추가함에 따라 하나의 개성으로는 모든 것을 잘 수행할 수 없습니다. 예를 들어, 연구 보조원은 일반 보조원과는 다른 지침이 필요합니다.

해결책: 라우팅(Routing)을 포함한 다중 에이전트 설정입니다.

```python
AGENTS = {
    "main": {
        "name": "Jarvis",
        "soul": SOUL,  # 기존 SOUL
        "session_prefix": "agent:main",
    },
    "researcher": {
        "name": "Scout",
        "soul": """당신은 연구 전문가인 Scout입니다.
당신의 임무: 정보를 찾고 출처를 인용하십시오. 모든 주장은 증거가 필요합니다.
도구를 사용하여 데이터를 수집하십시오. 철저하지만 간결하게 행동하십시오.
다른 에이전트가 참조할 수 있도록 중요한 발견 사항을 메모리에 저장하십시오.""",
        "session_prefix": "agent:researcher",
    },
}

def resolve_agent(message_text):
    """프리픽스(Prefix) 명령을 기반으로 메시지를 올바른 에이전트로 라우팅합니다."""
    if message_text.startswith("/research "):
        return "researcher", message_text[len("/research "):]
    return "main", message_text
```

핸들러(Handler)를 업데이트하십시오:

```python
async def handle_message(update: Update, context):
    user_id = str(update.effective_user.id)
    agent_id, message_text = resolve_agent(update.message.text)
    agent = AGENTS[agent_id]
    session_key = f"{agent['session_prefix']}:{user_id}"

    with session_locks[session_key]:
        messages = load_session(session_key)
        messages = compact_session(session_key, messages)
        messages.append({"role": "user", "content": message_text})
        response_text, messages = run_agent_turn(messages, agent["soul"])
        save_session(session_key, messages)
    await update.message.reply_text(f"[{agent['name']}] {response_text}")
```

```python
                "query": {"type": "string", "description": "검색할 내용"}
            },
            "required": ["query"]
        }
    },
]

# ─── 권한 제어 ───
SAFE_COMMANDS = {"ls", "cat", "head", "tail", "wc", "date", "whoami",
                 "echo", "pwd", "which", "git", "python", "node", "npm"}
def load_approvals():
    if os.path.exists(APPROVALS_FILE):
        with open(APPROVALS_FILE) as f:
            return json.load(f)
    return {"allowed": [], "denied": []}
def save_approval(command, approved):
    approvals = load_approvals()
    key = "allowed" if approved else "denied"
    if command not in approvals[key]:
        approvals[key].append(command)
    with open(APPROVALS_FILE, "w") as f:
        json.dump(approvals, f, indent=2)
def check_command_safety(command):
    base_cmd = command.strip().split()[0] if command.strip() else ""
    if base_cmd in SAFE_COMMANDS:
        return "safe"
    approvals = load_approvals()
    if command in approvals["allowed"]:
        return "approved"
    return "needs_approval"

# ─── 도구 실행 ───
def execute_tool(name, tool_input):
    if name == "run_command":
        cmd = tool_input["command"]
        safety = check_command_safety(cmd)
        if safety == "needs_approval":
            print(f"\n  ⚠️  명령: {cmd}")
            confirm = input("  허용하시겠습니까? (y/n): ").strip().lower()
            if confirm != "y":
                save_approval(cmd, False)
                return "사용자에 의해 권한이 거부되었습니다."
            save_approval(cmd, True)
        try:
            result = subprocess.run(
                cmd, shell=True, capture_output=True, text=True, timeout=30
            )
            output = result.stdout + result.stderr
            return output if output else "(출력 없음)"
        except subprocess.TimeoutExpired:
            return "명령이 30초 후 시간 초과되었습니다."
        except Exception as e:
            return f"오류: {e}"
    elif name == "read_file":
        try:
            with open(tool_input["path"], "r") as f:
                return f.read()[:10000] # 최대 10000자 읽기
        except Exception as e:
            return f"오류: {e}"
    elif name == "write_file":
        try:
            os.makedirs(os.path.dirname(tool_input["path"]) or ".", exist_ok=True)
            with open(tool_input["path"], "w") as f:
                f.write(tool_input["content"])
            return f"'{tool_input['path']}'에 작성했습니다."
        except Exception as e:
            return f"오류: {e}"
    elif name == "save_memory":
        os.makedirs(MEMORY_DIR, exist_ok=True)
        filepath = os.path.join(MEMORY_DIR, f"{tool_input['key']}.md")
        with open(filepath, "w") as f:
            f.write(tool_input["content"])
        return f"메모리에 저장됨: {tool_input['key']}"
    elif name == "memory_search":
        query = tool_input["query"].lower()
        results = []
        if os.path.exists(MEMORY_DIR):
            for fname in os.listdir(MEMORY_DIR):
                if fname.endswith(".md"):
                    with open(os.path.join(MEMORY_DIR, fname), "r") as f:
                        content = f.read()
                    if any(w in content.lower() for w in query.split()):
                        results.append(f"--- {fname} ---\n{content}")
        return "\n\n".join(results) if results else "일치하는 메모리를 찾을 수 없습니다."
    return f"알 수 없는 도구: {name}"

# ─── 세션 관리 ───
def get_session_path(session_key):
    os.makedirs(SESSIONS_DIR, exist_ok=True)
    safe_key = session_key.replace(":", "_").replace("/", "_")
    return os.path.join(SESSIONS_DIR, f"{safe_key}.jsonl")
def load_session(session_key):
    path = get_session_path(session_key)
    messages = []
    if os.path.exists(path):
        with open(path, "r") as f:
            for line in f:
                if line.strip():
                    try:
                        messages.append(json.loads(line))
                    except json.JSONDecodeError:
                        continue
    return messages
def append_message(session_key, message):
    with open(get_session_path(session_key), "a") as f:
        f.write(json.dumps(message) + "\n")
def save_session(session_key, messages):
    with open(get_session_path(session_key), "w") as f:
        for msg in messages:
            f.write(json.dumps(msg) + "\n")

# ─── 압축 ───
def estimate_tokens(messages):
    return sum(len(json.dumps(m)) for m in messages) // 4
def compact_session(session_key, messages):
    if estimate_tokens(messages) < 100_000: # ~128k 창의 80%
        return messages
    split = len(messages) // 2
    old, recent = messages[:split], messages[split:]
    print("\n  세션 기록 압축 중...")
    summary = client.messages.create(
        model="claude-sonnet-4-5-20250929",
        max_tokens=2000,
        messages=[{
            "role": "user",
            "content": (
                "이 대화를 간결하게 요약하십시오. 핵심 사실, "
                "결정, 그리고 열려 있는 작업을 보존하십시오:\n\n"
                f"{json.dumps(old, indent=2)}"
            )
        }]
    )
    compacted = [{
        "role": "user",
        "content": f"[대화 요약]\n{summary.content[0].text}"
    }] + recent
    save_session(session_key, compacted)
    return compacted

# ─── 명령 큐 ───
session_locks = defaultdict(threading.Lock)

# ─── 에이전트 루프 ───
def serialize_content(content):
    serialized = []
    for block in content:
        if hasattr(block, "text"):
            serialized.append({"type": "text", "text": block.text})
        elif block.type == "tool_use":
            serialized.append({
                "type": "tool_use", "id": block.id,
                "name": block.name, "input": block.input
            })
    return serialized
def run_agent_turn(session_key, user_text, agent_config):
    """전체 에이전트 턴을 실행합니다: 세션을 로드하고, LLM을 반복 호출하고, 저장합니다."""
    with session_locks[session_key]:
        messages = load_session(session_key)
        messages = compact_session(session_key, messages)
        user_msg = {"role": "user", "content": user_text}
        messages.append(user_msg)
        append_message(session_key, user_msg)

        for _ in range(20):  # 최대 도구 사용 턴
            response = client.messages.create(
                model=agent_config["model"],
                max_tokens=4096,
                system=agent_config["soul"],
                tools=TOOLS,
                messages=messages
            )
            content = serialize_content(response.content)
            assistant_msg = {"role": "assistant", "content": content}
            messages.append(assistant_msg)
            append_message(session_key, assistant_msg)

            if response.stop_reason == "end_turn":
                return "".join(
                    b.text for b in response.content if hasattr(b, "text")
                )
            if response.stop_reason == "tool_use":
                tool_results = []
                for block in response.content:
                    if block.type == "tool_use":
                        print(f"  🔧 {block.name}: {json.dumps(block.input)[:100]}...")
                        result = execute_tool(block.name, block.input)
                        display = str(result)[:150]
                        print(f"     → {display}")
                        tool_results.append({
                            "type": "tool_result",
                            "tool_use_id": block.id,
                            "content": str(result)
                        })
                results_msg = {"role": "user", "content": tool_results}
                messages.append(results_msg)
                append_message(session_key, results_msg)
        return "(최대 턴에 도달했습니다)"

# ─── 다중 에이전트 라우팅 ───
def resolve_agent(message_text):
    """프리픽스(Prefix) 명령을 기반으로 메시지를 올바른 에이전트로 라우팅합니다."""
    if message_text.startswith("/research "):
        return "researcher", message_text[len("/research "):]
    return "main", message_text

# ─── Cron / 하트비트 ───
def setup_heartbeats():
    def morning_check():
        print("\n⏰ 하트비트: 아침 확인")
        result = run_agent_turn(
            "cron:morning-check",
            "좋은 아침입니다! 오늘 날짜를 확인하고 동기 부여 명언을 알려주세요.",
            AGENTS["main"]
        )
        print(f" {result}\n")
    schedule.every().day.at("07:30").do(morning_check)

    def scheduler_loop():
        while True:
            schedule.run_pending()
            time.sleep(60)
    threading.Thread(target=scheduler_loop, daemon=True).start()

# ─── REPL ───
def main():
    for d in [WORKSPACE, SESSIONS_DIR, MEMORY_DIR]:
        os.makedirs(d, exist_ok=True)
    setup_heartbeats()
    session_key = "agent:main:repl" # REPL을 위한 기본 세션 키
    print("Mini OpenClaw")
    print(f"  에이전트: {', '.join(a['name'] for a in AGENTS.values())}")
    print(f"  작업 공간: {WORKSPACE}")
    print("  명령어: /new (초기화), /research <쿼리>, /quit\n")

    while True:
        try:
            user_input = input("당신: ").strip()
        except (EOFError, KeyboardInterrupt):
            print("\n안녕히 계세요!")
            break

        if not user_input:
            continue

        if user_input.lower() in ["/quit", "/exit", "/q"]:
            print("안녕히 계세요!")
            break

        if user_input.lower() == "/new":
            session_key = f"agent:main:repl:{datetime.now().strftime('%Y%m%d%H%M%S')}"
            print("  세션이 초기화되었습니다.\n")
            continue

        agent_id, message_text = resolve_agent(user_input)
        agent_config = AGENTS[agent_id]
        
        # REPL을 위해 메인 에이전트가 아니면 전용 세션 키 사용
        sk = (
            f"{agent_config['session_prefix']}:repl"
            if agent_id != "main" else session_key
        )
        response = run_agent_turn(sk, message_text, agent_config)
        print(f"\n [{agent_config['name']}] {response}\n")

if __name__ == "__main__":
    main()
```

## 우리가 구축한 것

아무것도 없는 상태에서, 우리는 다음을 구축했습니다:

*   **영구 세션 (Persistent Sessions)** — 충돌에도 대화 기억이 유지됨
*   **SOUL.md** — AI를 일관성 있게 만드는 개성 파일
*   **도구 + 에이전트 루프 (Agent Loop)** — AI가 실제로 작업을 수행하게 함
*   **권한 제어 (Permission Controls)** — 컴퓨터를 파괴하지 않도록
*   **게이트웨이 패턴 (Gateway Pattern)** — 하나의 에이전트, 여러 인터페이스
*   **컨텍스트 압축 (Context Compaction)** — 긴 대화 처리
*   **장기 메모리 (Long-term Memory)** — 초기화 후에도 지식이 유지됨
*   **명령 큐 (Command Queue)** — 경쟁 상태(Race Condition) 방지
*   **하트비트 (Heartbeats)** — 예약된 작업(Scheduled Task)
*   **다중 에이전트 (Multiple Agents)** — 다양한 작업을 위한 여러 에이전트

각 요소는 실제 문제를 해결했습니다. 마법이 아닙니다. 단지 실용적인 해결책일 뿐입니다.

## 더 나아가기

이 프로토타입(Prototype)은 핵심 아키텍처(Architecture)를 갖추고 있습니다. OpenClaw는 여기에 프로덕션(Production) 기능을 추가합니다:

*   **브라우저 자동화 (Browser Automation)** — 시맨틱 스냅샷(Semantic Snapshots)을 통한 실제 브라우저 제어 (스크린샷 대신 텍스트로, 토큰 비용 훨씬 저렴)
*   **세션 범위 지정 (Session Scoping)** — 세션을 사용자별, 채널별 또는 공유 방식으로 구성
*   **플러그인 시스템 (Plugin System)** — 핵심 코드 변경 없이 새 채널 추가
*   **벡터 검색 (Vector Search)** — 임베딩(Embeddings)을 이용한 시맨틱 메모리 매칭
*   **서브 에이전트 (Sub-Agents)** — 에이전트가 특정 작업을 위해 하위 에이전트 생성

하지만 솔직히 말해서, 간단하게 시작하세요. 하나의 채널이 작동하도록 만들고, 필요할 때 도구를 추가하세요. 세션만으로는 부족할 때 메모리를 추가하십시오. 복잡성은 청사진을 따르는 것이 아니라 당신의 필요에서 비롯됩니다.

아니면 그냥 OpenClaw를 사용하세요. 오픈 소스(Open-source)이며 엣지 케이스(Edge Cases)를 처리합니다. 하지만 이제 그 내부 작동 방식을 알게 되었습니다.
