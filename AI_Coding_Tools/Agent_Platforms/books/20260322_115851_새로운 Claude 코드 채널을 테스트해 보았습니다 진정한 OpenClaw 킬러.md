# 새로운 Claude 코드 채널을 테스트해 보았습니다 (진정한 OpenClaw 킬러)

조 녠가 (Joe Njenga)
팔로우 중 (Following)

---

Claude 코드 채널(Claude Code channels)은 Claude 코드 워크플로(workflows)에서 OpenClaw를 대체하도록 설계된 최신 기능으로, 이제 즐겨 사용하는 앱에서 자유롭게 실행할 수 있습니다.

Telegram 또는 Discord에서 AI 에이전트(AI agent)에게 메시지를 보내기 위해 OpenClaw를 사용해왔다면, 복잡한 설정, 보안 문제, 그리고 최근 Anthropic의 구독 토큰(subscription tokens) 차단으로 인해 답답한 경험을 하셨을 것입니다.

Anthropic은 이와 유사하지만 네이티브(native) 솔루션을 막 출시했습니다.

제가 Telegram에 이 기능을 설정했을 때, 작업을 보내면 Claude가 이를 받아 처리하고 Telegram으로 직접 응답합니다.

## Claude 코드 채널이란 무엇인가요?

---

채널은 실행 중인 Claude 코드 세션으로 이벤트를 푸시(push)하는 MCP 서버입니다.

Telegram이나 Discord에서 메시지를 보내면 Claude가 사용자가 자리를 비웠을 때도 반응합니다. 이는 휴대폰에서 Claude 코드를 사용하기 위해 OpenClaw를 설정하는 것과 동일한 과정입니다.

작동 흐름은 다음과 같습니다:

*   Telegram 또는 Discord에서 봇(bot)에게 메시지를 보냅니다.
*   채널 플러그인(plugin)이 해당 메시지를 Claude 코드 세션으로 전달합니다.
*   Claude는 이를 읽고 작업을 수행한 후 동일한 채널을 통해 응답합니다.
*   응답이 휴대폰에 표시됩니다.

---

현재 Anthropic은 연구 프리뷰(research preview) 단계에서 두 가지 플랫폼(Telegram 및 Discord)을 지원합니다.

실제 플랫폼을 연결하기 전에 테스트할 수 있는 Fakechat이라는 로컬호스트(localhost) 데모도 있습니다.

설정 및 테스트로 넘어가기 전에, 이것이 OpenClaw 문제를 어떻게 해결하는지 살펴봐야 합니다.

## OpenClaw 문제

---

OpenClaw는 합당한 이유로 가장 빠르게 성장하는 오픈 소스(open-source) AI 프로젝트가 되었습니다. 개발자들은 휴대폰으로 AI 에이전트와 대화하고 싶어 하며, 이는 실제적인 필요입니다.

이상적으로는 통근 중에 Telegram에서 작업을 보내고, 책상에서 떨어져 있는 동안 Discord에서 빌드(build) 상태를 확인하는 과정입니다.

하지만 OpenClaw는 다음과 같은 문제점을 안고 있었습니다:

*   **보안 위험**: 연구자들은 공용 인터넷에서 135,000개 이상의 노출된 인스턴스(instances)를 발견했으며, 50,000개 이상이 원격 코드 실행(remote code execution)에 취약했습니다. ClawHub 감사를 통해 커뮤니티 스킬(community skills)의 약 12%가 악성 코드(malicious)인 것으로 밝혀졌습니다.
*   **복잡한 설정**: 전용 하드웨어(많은 개발자들이 이를 위해 Mac Mini를 구매), Docker 구성, 그리고 많은 시간 투자 및 조정이 필요했습니다. 저는 이전 튜토리얼에서 Mac Mini와 Windows에 설정하는 데모를 진행했습니다.
*   **구독 차단**: Anthropic은 Claude Pro/Max 구독 토큰을 활용하는 타사 도구들을 차단했습니다. OpenClaw를 통해 Claude에 접근하는 가장 저렴한 방법이 더 이상 작동하지 않게 된 것입니다.
*   **창립자 이탈**: Peter Steinberger가 OpenAI에 합류하면서 프로젝트가 재단으로 이전되었습니다. 프로젝트의 향후 방향은 불확실합니다.

Claude 코드 채널은 이 모든 것을 네이티브(native) 방식으로 해결합니다. 타사 에이전트, 노출된 인스턴스, 또는 구독 해킹이 필요 없습니다.

또한 Anthropic의 보안 모델이 내장되어 있습니다. 발신자 허용 목록(sender allowlists), 세션별 옵트인(per-session opt-in) `--channels` 플래그 사용, 팀을 위한 엔터프라이즈(enterprise) 제어 기능 등입니다.

이것은 복잡한 주제이므로, 여러 시리즈로 나누어 설명하겠습니다. 이 시리즈에서는 Fakechat 데모를 통해 흐름을 먼저 이해한 다음, Telegram과 Discord의 완벽한 설정 과정을 안내해 드릴 것입니다.

이 첫 번째 기사에서는 다음 내용을 다룰 것입니다:

*   선행 조건 및 모든 종속성(dependencies) 설정
*   Fakechat 데모 설정 (흐름 먼저 이해하기)

다음 튜토리얼에서는 다음 내용들을 심층적으로 다룰 것입니다:

*   Telegram 봇(bot)을 Claude 코드에 연결하기 (단계별)
*   Discord 봇을 Claude 코드에 연결하기 (단계별)
*   보안 및 접근 제어 (잠금 설정 방법)
*   채널과 OpenClaw 비교 (솔직한 비교)

Fakechat 데모부터 시작해 봅시다.

## Claude 코드 채널 시작하기

가장 먼저 Claude 코드가 업데이트되었고 다른 모든 종속성이 제대로 설정되었는지 확인해야 합니다.

### 선행 조건 (Prerequisites)

어떤 채널이든 설정하기 전에 두 가지가 준비되어 있어야 합니다.

*   Claude Code v2.1.80 이상 — 이전 버전에서는 채널이 작동하지 않습니다.
*   Bun 런타임(Runtime) — 모든 채널 플러그인(plugin)은 Bun에서 실행됩니다.

### 1단계: Claude 코드 버전 확인

터미널에서 다음을 실행합니다:

```bash
claude --version
```

---

예상 출력: 버전 2.1.80 이상

버전이 더 오래된 경우 먼저 업데이트합니다:

```bash
claude update
```

---

그런 다음 동일한 명령을 사용하여 다시 확인합니다.

### 2단계: Bun 설치 여부 확인

다음을 실행합니다:

```bash
bun --version
```

---

버전 번호가 보이면 좋습니다. 3단계로 이동하세요.

위의 제 경우처럼 실패하면 Bun을 설치합니다:

**Mac/Linux**

```bash
curl -fsSL https://bun.sh/install | bash
```

**Windows (PowerShell):**

```powershell
irm bun.sh/install.ps1 | iex
```

---

설치 후 확인합니다:

```bash
bun --version
```

---

한 가지 더, 채널을 사용하려면 claude.ai 로그인(login)이 필요합니다. 콘솔(Console) 및 API 키(API key) 인증은 여기에서 작동하지 않습니다. 팀 또는 엔터프라이즈(Enterprise) 요금제를 사용 중인 경우, 관리자가 관리 설정(managed settings)에서 채널을 활성화해야 사용할 수 있습니다.

### 3단계: Claude 코드 시작

프로젝트 폴더 안에 있어야 합니다. 제 경우에는 `channels` 폴더입니다. Claude 코드 세션을 시작합니다:

---

Claude 코드가 로드될 때까지 기다립니다.

인증 오류가 발생하는 경우 – claude.ai 계정으로 로그인(API 키 또는 콘솔 아님)했는지 확인하세요. 필요한 경우 `/login`을 실행합니다.

## Fakechat — 흐름 먼저 테스트하기

Telegram이나 Discord에 연결하기 전에 Fakechat으로 시작하는 것을 권장합니다.

이는 브라우저에서 채팅 UI를 실행하는 로컬호스트(localhost) 데모로, 인증할 것도 없고 외부 서비스를 구성할 필요도 없습니다.

봇 토큰(bot tokens)과 페어링 코드(pairing codes)를 다루기 전에 채널 메시지 흐름이 어떻게 작동하는지 보여줍니다.

### 1단계: Fakechat 플러그인(Plugin) 설치

Claude 코드 세션을 시작하고 다음을 실행합니다:

```bash
/plugin install fakechat@claude-plugins-official
```

---

플러그인 설치 옵션이 표시될 것입니다. 제 경우, 사용자 범위(User Scope)에 설치하겠습니다:

---

성공하면 활성화를 위해 다시 로드하라는 메시지가 표시됩니다:

```
✓ Installed fakechat. Run /reload-plugins to activate.
```

---

Fakechat 플러그인이 활성화되었는지 확인하려면, `/plugin` 명령을 다시 실행하고 `installed` 탭으로 이동하여 확인합니다.

---

**설치 오류**

마켓플레이스(marketplace)를 찾을 수 없다는 오류가 발생하면 먼저 다음을 실행합니다:

```bash
/plugin marketplace add anthropics/claude-plugins-official
```

### 2단계: 채널 활성화하여 다시 시작

Claude 코드를 종료한 다음, `--channels` 플래그(flag)와 함께 다시 시작합니다:

```bash
claude --channels plugin:fakechat@claude-plugins-official
```

---

위에서 보듯이 Fakechat 채널이 활성화되었음을 확인하는 메시지를 찾으세요. Fakechat 서버가 시작되어 수신 대기 중입니다.

### 3단계: 브라우저에서 Fakechat 열기

터미널을 계속 실행 상태로 유지하세요. **닫지 마세요.**

브라우저를 열고 다음으로 이동합니다:

```
http://localhost:8787
```

---

위와 같이 간단한 채팅 인터페이스가 보일 것입니다. 이 단계에 도달하면 설정이 완료되어 테스트 준비가 된 것입니다.

메시지를 입력합니다:

```
hey, what's in my working directory?
```

---

보내기 버튼을 누릅니다. 이제 Claude 코드 터미널을 주시하세요.

메시지는 `<channel source="fakechat">` 이벤트로 도착합니다. Claude는 이를 읽고 작업을 수행한 후 Fakechat의 응답 도구(reply tool)를 호출하고 답변이 표시됩니다.

---

### 권한에 대한 간단한 참고 사항

Claude가 채널 메시지를 처리하는 동안 권한 프롬프트(permission prompt)가 발생하고 터미널에 없는 경우, 세션이 일시 중지됩니다.

계속 진행하기 전에 로컬 승인을 기다립니다.

무인 사용을 위해서는 `--dangerously-skip-permissions` 플래그가 있지만, 완전히 신뢰하는 환경에서만 사용해야 합니다. 이름 자체가 모든 것을 말해줍니다.

## 결론

Fakechat을 테스트한 결과, 채널 흐름은 예상대로 작동합니다.

메시지가 브라우저 데모에서 Claude 코드 세션으로 이동하고, Claude가 이를 처리하며, 응답이 돌아옵니다.

Fakechat 데모는 로컬호스트(localhost)에서 개념을 증명하지만, 실제 가치는 Telegram이나 Discord에 연결하여 휴대폰에서 실행 중인 Claude 코드 세션으로 작업을 보내기 시작할 때 나타납니다.

또한 WhatsApp, Signal 등과 같은 플랫폼 지원을 추가할 수 있기를 바랍니다.

Claude 코드 채널은 아직 연구 프리뷰(research preview) 단계임을 명심하세요. `--channels` 플래그 구문과 프로토콜(protocol)은 피드백에 따라 변경될 수 있습니다.

프리뷰 기간 동안에는 Anthropic이 관리하는 허용 목록(allowlist)에 있는 플러그인만 허용됩니다.

하지만 이 초기 단계에서도 이것이 매우 유용하다는 것은 분명합니다.

따라서 다음 시리즈에서는 다음 내용을 다룰 것입니다.

**1: Telegram을 Claude 코드에 연결하기**

봇파더(BotFather)로 봇을 생성하는 것부터 계정 페어링(pairing) 및 휴대폰에서 실제 코딩 작업 전송에 이르기까지 전체 Telegram 설정을 안내해 드릴 것입니다.

**2: Discord를 Claude 코드에 연결하기**

봇 생성, 권한 설정, 그리고 Telegram보다 Discord가 제공하는 추가 도구(메시지 기록 및 첨부 파일 다운로드)를 포함한 Discord 설정을 다룹니다.

또한 두 채널을 동시에 실행하는 방법, 보안 잠금, 엔터프라이즈(enterprise) 제어, 그리고 채널과 OpenClaw의 솔직한 비교도 다룰 것입니다.

다음 편을 놓치지 않으려면 팔로우해주세요.

이 시리즈에서 보고 싶은 다른 내용이 있다면 아래에 댓글로 남겨주세요.

## Claude 코드 마스터클래스 강좌

저는 매일 여러 에이전트(agents)를 조정하여 복잡한 개발 작업을 수행하는 워크플로를 만드는 방법을 시연하는 궁극의 Claude 코드 강좌를 구축하기 위해 노력하고 있습니다. 곧 출시될 예정입니다.

이 강좌는 이 기사에서 배운 내용을 다음 단계의 완전한 자동화로 끌어올릴 것입니다.

Claude 코드에는 매일 새로운 기능이 추가되고 있으며, 이를 따라잡는 것은 어렵습니다.

이 강좌는 많은 개발자들이 알지 못할 에이전트(Agents), 훅(Hooks), 고급 워크플로, 생산성 기술을 탐구합니다.

가입하시면 새로운 기능이 출시될 때마다 모든 업데이트를 받으실 수 있습니다.

이 강좌는 다음 내용을 다룰 것입니다:

*   고급 서브에이전트(subagent) 패턴 및 워크플로
*   제품 수준의 훅(hook) 구성
*   외부 도구를 위한 MCP 서버 통합
*   팀 협업 전략
*   엔터프라이즈(Enterprise) 배포 패턴
*   저의 컨설팅 작업에서 얻은 실제 사례 연구

Claude 코드 강좌가 출시될 때 알림을 받고 싶으시다면 여기를 클릭하여 얼리 액세스(early access) 목록에 참여하세요 →

(현재 26,000명 이상의 개발자가 이미 가입했습니다)

저는 목록에 있는 분들과 독점적인 미리 보기, 얼리 액세스 가격, 보너스 자료를 공유할 것입니다.
