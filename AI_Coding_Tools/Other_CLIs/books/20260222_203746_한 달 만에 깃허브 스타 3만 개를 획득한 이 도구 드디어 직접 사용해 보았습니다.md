# 한 달 만에 깃허브 스타 3만 개를 획득한 이 도구 — 드디어 직접 사용해 보았습니다

계속 미뤄왔습니다. 커서(Cursor)나 클로드 코드(Claude Code)를 사용하는 것이 편했으니까요. 하지만 한 달도 채 되지 않아 깃허브(GitHub) 스타가 39,800개에서 71,900개로 급증하는 것을 지켜보며, 제가 무언가 놓치고 있다는 사실을 깨달았습니다.

직접 이 도구로 개발해 보았고, 그 결과를 공유합니다.

### 오픈코드(OpenCode)란 무엇인가?

오픈코드는 터미널용으로 구축된 오픈 소스 AI 코딩 에이전트(AI coding agent)입니다. 뛰어난 실력을 갖춘 SST(Serverless) 팀이 개발했습니다.

이 팀은 터미널을 속속들이 잘 아는 개발자들로 구성되어 있습니다. 오픈코드가 처음부터 터미널 네이티브(Terminal native) 앱으로 설계되었다는 점에서 이 점은 매우 중요합니다.

오픈코드의 핵심적인 차별점은 오픈 소스이기 때문에 특정 AI 공급자(AI provider)에 종속되지 않는다는 것입니다.

### 놀라운 사실들

본격적으로 들어가기 전에 알아두어야 할 사항은 다음과 같습니다.

*   오픈코드는 100% 무료이며 오픈 소스입니다.
*   사용자가 선택한 AI 모델 비용만 지불하면 됩니다.
*   70개 이상의 공급자를 지원합니다.
*   커서(Cursor)와 마찬가지로 기획 모드(Plan mode)와 빌드 모드(Build mode)를 지원합니다.
*   터미널 인터페이스가 훌륭합니다.
*   매달 65만 명 이상의 개발자가 사용하고 있습니다.

### 왜 클로드 코드(Claude Code)에서 갈아타나요?

첫째, **공급자의 유연성(Provider flexibility)** 때문입니다.

클로드 코드는 사용자를 앤스로픽(Anthropic)에만 머물게 합니다. 지난주까지는 대부분의 개발자에게 문제가 없었습니다.

하지만 지난 1월 9일, 앤스로픽은 예고 없이 클로드 구독 정책을 변경하여 제3자 앱 사용을 불허했습니다. 그 악명 높은 오류 메시지는 다음과 같습니다:
*"이 자격 증명은 클로드 코드(Claude Code)에서만 사용할 수 있도록 승인되었습니다."*

앤스로픽을 변호하자면, 엄밀히 말해 이는 항상 서비스 약관에 포함되어 있던 내용이었습니다. 하지만 이는 단일 공급자에 의존할 때(벤더 종속성, Vendor lock-in) 어떤 일이 벌어지는지를 상기시켜 주는 계기가 되었습니다.

반면, 오픈코드는 설정을 바꾸지 않고도 공급자를 자유롭게 교체할 수 있습니다. 오픈코드 팀은 이를 다음과 같이 설명합니다.
> "모델이 발전함에 따라 모델 간의 격차는 줄어들고 가격은 낮아질 것입니다. 따라서 공급자 독립적(Provider-agnostic)인 상태를 유지하는 것이 중요합니다."

단일 공급자를 중심으로 시스템을 구축하는 것이 위험할 수 있다는 점에 저도 동의합니다.

### 워크플로우(Workflow)는 어떤 방식인가요?

두 가지 내장 에이전트가 제공되며, 탭(Tab) 키를 사용해 전환할 수 있습니다(커서와 동일).

1.  **빌드 모드(Build mode)** — 코드 작성, 리팩터링(Refactoring) 및 테스트용.
2.  **기획 모드(Plan mode)** — 접근 방식을 결정하고 아키텍처를 설계하기 위한 코드 탐색용.

새 프로젝트를 시작할 때 `/init`을 입력하면, 모든 파일을 확인한 후 `AGENTS.md` 파일을 생성합니다. 매우 간단하고 직관적입니다.

### 터미널 인터페이스 비교

이 부분에서 오픈코드가 우위를 점합니다.

클로드 코드가 터미널에서 '괜찮은' 수준이라면, 오픈코드는 '놀라운' 수준입니다. 오픈코드는 버퍼 시스템(Buffer system)을 갖추고 있어 창 크기를 조절해도 올바르게 작동합니다. 터미널에서 이상한 스크롤 현상이 발생하지 않습니다. 이는 빔(Vim)이나 이맥스(Emacs)와 상당히 유사한 경험을 제공합니다.

이 기능을 사용하다가 다시 클로드 코드로 돌아가면 매우 생소하게 느껴질 정도입니다.

### 비용은 어떠한가요?

여기서 상황이 크게 달라집니다.

오픈코드라는 도구 자체는 무료입니다. 선택한 공급자의 사용량에 대해서만 비용을 지불하면 됩니다. 하지만 대부분의 사람들은 현존 최고 모델인 클로드 오퍼스 4.5(Claude Opus 4.5)와 유사한 대안을 원할 것입니다.

이때 **GLM-4.7**이 등장합니다. 모델 성능 면에서 가장 가까운 비교 대상입니다. 요금제를 비교해 보겠습니다.

**클로드 코드(Claude Code) 비용:**
*   Pro: 월 $17
*   Max: 월 $100

**제가 사용하는 공급자 기준 GLM 비용:**
*   Lite: 월 $6
*   Pro: 월 $30
*   Max: 월 $60

저는 이전에 클로드 코드 Max 플랜의 제한 수치에 도달한 적이 몇 번 있었습니다. 하지만 GLM Max 플랜을 사용하면서는 아직 한 번도 제한에 걸린 적이 없습니다.

### 클로드 코드보다 나은가요?

솔직히 말씀드리겠습니다.

클로드 코드는 매우 영리합니다. 오퍼스(Opus)를 사용하는 모든 엔지니어는 이것이 단연 최고의 모델이라고 말합니다. 비용 걱정이 없고 오로지 결과물의 퀄리티가 최우선이라면(회사가 Max 플랜을 결제해 준다면), 클로드 코드가 승자입니다.

하지만 **개발자 경험(DX), 비용, 또는 벤더 종속성**이 걱정된다면? 그렇다면 오픈코드가 확실한 승자입니다.

### 시작하는 방법

다음 배시(Bash) 명령어를 사용하여 설치하세요:
```bash
curl -fsSL https://opencode.ai/install | bash
```

또는 패키지 관리자를 사용할 수 있습니다:
```bash
npm install -g opencode-ai
```

개인적으로 저는 브루(Brew)를 사용합니다:
```bash
brew install anomalyco/tap/opencode
```

공식 문서에 다른 설치 방법들도 자세히 안내되어 있습니다.

### 결론

이미 터미널을 사랑하는 개발자에게 오픈코드는 벤더 종속성 없는 당연한 선택(No brainer)입니다.

낮은 노력으로 사용하기를 원하고 가격에 큰 불만이 없는 개발자에게는 클로드 코드가 여전히 훌륭한 선택지입니다.

터미널을 좋아하지 않는 분들이라면 커서(Cursor) 역시 훌륭한 대안입니다.

아직 클로드 코드나 오픈코드를 사용해 보지 않으셨다면, 가능한 한 빨리 시도해 보시길 권장합니다! 추천 사항이나 의견이 있다면 댓글로 알려주세요.
