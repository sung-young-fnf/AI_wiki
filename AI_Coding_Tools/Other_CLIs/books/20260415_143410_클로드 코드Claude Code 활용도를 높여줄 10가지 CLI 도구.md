# 클로드 코드(Claude Code) 활용도를 높여줄 10가지 CLI 도구

클로드 코드(Claude Code)를 단순한 코드 비서에서 실제로 결과물을 배포할 수 있는 도구로 변화시키는 것들입니다.

클로드 코드(Claude Code)를 깊이 있게 사용하면서 가장 큰 발전은 모델 자체가 아니라, 그 주변 도구들이었습니다.

대부분의 사람들은 클로드 코드를 설정하고 프로젝트에 연결한 다음 사용을 마칩니다. 물론 그것도 좋습니다. 하지만 올바른 CLI(Command Line Interface) 도구들과 연결하면, 클로드 코드는 똑똑한 자동 완성 기능을 넘어 코드를 배포하고, 인프라를 관리하며, 결제를 처리하고, 비디오를 편집하는 등 실제 작업을 수행할 수 있는 도구처럼 느껴지기 시작합니다.

제가 계속 사용하게 되는 10가지 도구들입니다.

### 1. CLI Anything

이 도구는 메타(meta)적인 성격을 띠고 있습니다. CLI Anything은 다른 CLI 도구들을 활용하여 새로운 CLI 도구를 생성합니다. 특정 오픈 소스 프로젝트를 지정하면 클로드 코드가 해당 인터페이스를 파악하여 터미널에서 실제로 실행 가능한 도구를 구축해 줍니다.

CLI 래퍼(wrapper)가 없는 프로젝트에 빠르게 CLI 기능을 추가하고 싶을 때 유용합니다.

### 2. Playwright CLI

클로드 코드가 브라우저를 제어하여 클릭, 양식 작성, 페이지 스크래핑, 종단간(end-to-end) 테스트 실행 등을 수행하게 하고 싶다면 이 도구가 필요합니다. Playwright CLI는 클로드 코드에게 실제 브라우저 환경 내에서 직접 조작할 수 있는 능력을 부여합니다.

사용자 인터페이스(UI) 뒤에 숨겨져 API가 없는 작업을 자동화해야 할 때 자주 사용하지만, 이런 경우가 생각보다 많습니다.

### 3. LLM Fit

로컬에서 모델을 실행하는 것은 멋지게 들리지만, Hugging Face 다운로드 목록을 보며 노트북이 과열되지 않고 제대로 작동할 모델이 무엇일지 고민하는 순간부터는 그렇지 않습니다. LLM Fit은 이 문제를 해결해 줍니다. 기기 사양을 알려주면 어떤 모델이 적합한지 알려주어 더 이상 시행착오를 겪을 필요가 없습니다.

### 4. Google Workspace CLI

이 도구는 클로드 코드가 전체 구글 워크스페이스(Google Workspace) 제품군을 제어할 수 있게 합니다. 문서(Docs), 스프레드시트(Sheets), 드라이브(Drive), 캘린더(Calendar) 등 모든 것을 터미널에서 스크립팅할 수 있습니다.

클라이언트의 구글 설정과 연동되는 무언가를 구축하거나, 단순히 자신의 워크플로우를 자동화하고 싶다면 알아둘 가치가 있습니다.

### 5. GitHub CLI

이 도구의 중요성은 이미 잘 알고 계실 테니 간략하게 설명하겠습니다. 코드를 푸시(push)한다면 이 도구를 사용해야 합니다. 브랜치(branch) 생성, 풀 리퀘스트(PRs), 이슈 트래킹(issue tracking) 등 모든 것을 브라우저를 켜지 않고 터미널에서 처리할 수 있습니다.

클로드 코드와 GitHub CLI를 함께 사용하면 터미널 세션을 벗어나지 않고 아이디어 구상부터 풀 리퀘스트(pull request) 생성까지 진행할 수 있습니다.

### 6. Vercel CLI

Vercel의 무료(free) 티어는 정말 관대하며, 매번 대시보드를 클릭하는 것보다 터미널에서 배포하는 것이 훨씬 빠릅니다. Vercel CLI를 클로드 코드와 연결하면 워크플로우에서 직접 배포할 수 있습니다.

이는 문맥 전환(context-switching)을 줄이고, 열려 있는 탭을 하나 줄여줍니다.

### 7. Supabase CLI

데이터베이스나 사용자 인증이 필요한 무언가를 구축하고 있나요? Supabase는 제가 가장 선호하는 도구이며, CLI를 통해 전체 설정을 스크립트화할 수 있습니다.

인증(Auth), PostgreSQL(Postgres), 스토리지(storage) 등 모든 것을 터미널에서 관리할 수 있습니다. 이 도구를 연결하면 클로드 코드가 직접 구성하고 쿼리하는 데 도움을 줄 수 있습니다.

### 8. FFmpeg

명령줄(command line)에서 오디오 및 비디오를 조작합니다. 클립 자르기, 형식 변환, 오디오 추출, 파일 압축 등 FFmpeg은 거의 모든 미디어 작업을 처리할 수 있습니다.

이 도구는 악명 높을 정도로 API가 복잡합니다. 클로드 코드의 도움을 받아 올바른 FFmpeg 명령어를 구성하는 것은 AI 기반 개발(AI-assisted development)의 가장 좋은 활용 사례 중 하나입니다.

### 9. notebooklm-py

이 목록에서 제가 개인적으로 가장 좋아하는 도구입니다.

NotebookLM은 이미 강력한 연구 도구입니다. 이 파이썬(Python) 라이브러리는 NotebookLM을 클로드 코드와 연결하여 문서를 프로그래밍 방식으로 제공하고 결과를 워크플로우에 다시 통합할 수 있게 해줍니다.

문서, 사내 위키(internal wikis), PDF 등을 기반으로 하는 연구 중심 개발을 수행한다면, 이 조합을 설정할 가치가 충분합니다.

### 10. Stripe CLI

웹 애플리케이션에 결제 기능을 추가하는 것은 간단하게 들리지만 실제로는 그렇지 않습니다. 웹훅(Webhooks), 테스트 이벤트, 환경 설정 등 고려할 것이 많습니다.

Stripe CLI는 로컬 개발 경험을 훨씬 덜 번거롭게 만듭니다. 아무것도 배포하지 않고도 결제 이벤트를 시뮬레이션하고, 웹훅을 수신하며, 통합을 테스트할 수 있습니다. 클로드 코드의 도움을 받아 통합 기능을 구축하고 CLI를 사용하여 실시간으로 테스트할 수 있습니다.

---

이 도구들은 설치가 전혀 복잡하지 않습니다. 대부분 `npm install` 또는 `brew install` 명령 한 번으로 설치할 수 있습니다. 그러나 이들을 함께 사용하면 클로드 코드를 단순한 코드 작성 비서가 아닌, 브라우저를 제어하고, 클라우드 인프라를 관리하며, 결제를 처리하고, 미디어를 조작하는 도구로 변화시킬 수 있습니다. 이 모든 것이 터미널에서 가능합니다.

현재 구축하고 있는 프로젝트에 맞는 도구부터 시작하세요. 나머지는 필요할 때 자연스럽게 이해될 것입니다.
