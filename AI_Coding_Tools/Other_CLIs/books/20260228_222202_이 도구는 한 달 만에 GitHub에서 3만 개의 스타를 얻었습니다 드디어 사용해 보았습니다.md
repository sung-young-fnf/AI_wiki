# 이 도구는 한 달 만에 GitHub에서 3만 개의 스타를 얻었습니다 — 드디어 사용해 보았습니다

“OpenCode를 사용해 보셨나요?”라는 질문을 끊임없이 받았습니다.

미루고 있었습니다. Cursor나 Claude Code 사용에 익숙했기 때문입니다. 하지만 OpenCode의 GitHub 스타가 (한 달도 안 되어) 39,800개에서 71,900개로 늘어나는 것을 보았습니다. 이런 성장은 우연이 아닙니다. 제가 무언가를 놓치고 있다는 것을 깨달았습니다.

진지하게 OpenCode를 사용하여 개발해 보았고, 그 결과는 다음과 같습니다.

## OpenCode란 무엇인가요?

OpenCode는 터미널(terminal)용으로 구축된 오픈 소스(open-source) AI 코딩 에이전트(AI coding agent)입니다. 놀라운 SST (Serverless) 팀이 개발했습니다.

이 팀은 터미널에 대해 손바닥 보듯 잘 아는 개발자들로 구성되어 있습니다. OpenCode가 처음부터 터미널 네이티브 앱(terminal native app)이라는 점에서 이는 중요합니다.

OpenCode의 핵심 차이점은 특정 AI 공급업체(provider)에 종속되지 않는다는 것입니다. 이는 오픈 소스이기 때문입니다.

## 놀라운 사실들

자세히 알아보기 전에 다음 사항을 알아두세요:

*   OpenCode는 100% 무료이며 오픈 소스(open source)입니다.
*   사용하는 AI 모델에 대해서만 비용을 지불합니다.
*   70개 이상의 공급업체를 지원합니다.
*   계획(plan) 모드와 빌드(build) 모드를 제공합니다 (Cursor와 유사하게).
*   터미널 인터페이스(Terminal interface)가 놀랍습니다.
*   매월 650,000명 이상의 개발자가 사용하고 있습니다.

## Claude Code에서 전환해야 하는 이유?

첫째, 공급업체 유연성(provider flexibility)입니다.

Claude Code는 Anthropic에 종속됩니다. 대부분의 개발자에게는 (지난주까지는) 전혀 문제가 없었습니다.

1월 9일, Anthropic은 예기치 않게 Claude 구독 정책을 변경하여 타사 앱(3rd-party apps) 사용을 불허했습니다. 악명 높은 오류 메시지는 다음과 같았습니다:

“이 자격 증명은 Claude Code에서만 사용할 수 있도록 승인되었습니다.”

Anthropic의 입장에서는, 이것은 기술적으로 항상 그들의 서비스 약관(Terms of Service)에 명시되어 있었습니다. 이는 단일 공급업체에 의존하는 것이 어떤 의미인지 상기시켜주는 계기가 되었습니다 (많은 이들이 공급업체 종속(vendor lock-in)을 싫어합니다).

반면에 OpenCode는 아무것도 변경하지 않고도 공급업체를 전환할 수 있도록 해줍니다.

OpenCode 팀은 이렇게 말합니다:

“모델이 발전함에 따라 모델 간의 격차는 줄어들고 가격은 하락할 것이므로, 공급업체에 구애받지 않는 것(provider-agnostic)이 중요합니다.”

동의합니다. 단일 공급업체를 중심으로 개발하는 것은 불안할 수 있습니다.

## 워크플로는 어떻게 작동하나요?

두 가지 내장 에이전트(agent)가 제공되며, Tab 키를 사용하여 전환할 수 있습니다 (Cursor와 유사하게).

*   **빌드(Build) 모드** — 코드 작성, 리팩터링(refactoring), 테스트용.
*   **계획(Plan) 모드** — 접근 방식 결정 및 아키텍처(architecture) 계획을 위한 코드 탐색용.

새 프로젝트를 시작할 때는 `/init`을 입력합니다. 그러면 모든 파일을 확인하고 `AGENTS.md` 파일을 생성합니다.

매우 간단하고 직관적입니다.

## 터미널 인터페이스 비교

이것이 OpenCode가 우위를 점하는 부분입니다.

Claude Code는 터미널에서 괜찮은 느낌을 줍니다. OpenCode는 놀랍습니다. 버퍼 시스템(buffer system)을 가지고 있어 창 크기 조정이 올바르게 작동합니다. 터미널에서 이상한 스크롤이 발생하지 않습니다.

이는 Vim이나 Emacs와 상당히 유사합니다.

이것을 사용하다가 다시 Claude Code로 돌아가면 매우 이상하게 느껴집니다.

## 비용은 어떤가요?

이 부분에서 상황이 매우 달라집니다.

OpenCode 도구 자체는 무료입니다. 사용자는 특정 공급업체(provider)를 통해 사용한 만큼만 비용을 지불합니다.

하지만 대부분의 사람들은 Claude Opus 4.5와 유사한 대안을 사용하고 싶어할 것입니다. 가장 좋기 때문입니다.

여기서 GLM-4.7이 등장합니다.

이제 가장 유사한 모델들을 비교할 수 있습니다. 요금제를 비교해 봅시다.

Claude Code 비용은 다음과 같습니다:

*   Pro — 월 $17.
*   Max — 월 $100.

제가 사용하는 공급업체 기준으로 GLM 비용은 다음과 같습니다:

*   Lite — 월 $6.
*   Pro — 월 $30.
*   Max — 월 $60.

저는 이전에 Claude Code Max 요금제 한도에 몇 번 도달한 적이 있습니다. GLM Max 요금제로는 아직 한도에 도달한 적이 없습니다.

## Claude Code보다 더 나은가요?

솔직하게 말씀드리겠습니다.

Claude Code는 매우 영리합니다. 제가 아는 Opus 사용 엔지니어들은 모두 똑같이 말했습니다. 단연코 최고의 모델이라고요. 만약 비용에 대한 걱정이 없고 오직 결과물(output)만이 유일한 관심사라면 (회사가 Max 요금제를 지불하는 경우), Claude Code가 승자입니다.

하지만 개발자 경험(developer experience), 비용 또는 공급업체 종속(vendor lock-in)이 문제라면 어떨까요?

그렇다면 OpenCode가 확실한 승자입니다.

## 시작하는 방법

다음 Bash 명령어(Bash command)를 사용하여 설치하세요:

```bash
curl -fsSL https://opencode.ai/install | bash
```

또는 패키지 관리자(package manager)를 사용할 수 있습니다:

```bash
npm install -g opencode-ai
```

개인적으로는 Brew를 사용합니다:

```bash
brew install anomalyco/tap/opencode
```

공식 문서(docs)에는 다른 여러 설치 방법이 나와 있습니다.

## 결론적으로

이미 터미널을 선호하는 개발자들에게 OpenCode는 공급업체 종속(vendor lock-in) 없이 탁월한 선택입니다.

적은 노력으로 사용하고 싶고 가격에 문제가 없는 개발자들에게는 Claude Code가 여전히 훌륭한 선택입니다.

그리고 터미널을 좋아하지 않는 사람들도 있는데, 이 경우에는 Cursor도 훌륭합니다.

Claude Code나 OpenCode 중 어느 하나라도 사용해 보지 않았다면, 가능한 한 빨리 시도해 보시길 권합니다!
