유료 Medium(미디엄) 회원이 아니라면 여기에서 전체 스토리를 무료로 읽을 수 있지만, 제 작업을 지원하기 위해 Medium 가입을 고려해 주시면 감사하겠습니다!

최근 Ollama(올라마)를 사용하여 Claude Code를 실행하는 방법에 대한 자세한 가이드를 공유했지만, 혹시 놓치셨다면 현재 단계까지 안내해 줄 게시물들이 여기 있습니다.

*   제가 (새로운) Ollama Claude Code, Codex, OpenCode 실행을 테스트했습니다 (더 이상 설정 불필요)
*   새로운 Claude Code Ollama 워크플로를 시도해봤습니다 (놀랍도록 자유롭습니다)
*   Claude Code + MiniMax M2.5를 테스트했습니다 (단 한 번의 CLI 도구 빌드로 저를 깜짝 놀라게 했습니다)

이 가이드들이 이 설정이 작동하는 방식을 빠르게 이해하는 데 최고이지만, 이제 새로운 변형을 통해 한 단계 더 나아가고자 합니다.

Claude Code를 Ollama에서 실행하는 것은 비용을 절약하거나 Anthropic(앤트로픽) 모델에 장애가 발생한 날에도 (이 글을 쓰는 시점처럼) 작업을 계속할 수 없다면 그리 흥미롭지 않을 것입니다.

(이미지를 전체 크기로 보려면 Enter 키를 누르거나 클릭하세요)

그러한 점이 흥미롭다면, 프로토타입(prototype)에 아주 유용한 서브 에이전트(subagent)와 웹 검색 에이전트를 실행하여 더 나은 경험을 할 수 있습니다.

덜 민감한 작업을 다른 모델로 구축하여 토큰 비용을 절약하는 것이 핵심입니다.

현재로서는 Ollama를 사용하여 원하는 모델을 실행하는 것보다 더 나은 방법이 없으며, 이제 서브 에이전트와 웹 검색 기능을 실행하는 것이 마법처럼 작동합니다.

간단히 설명하자면, 프로젝트를 구축하고, 프런트엔드(frontend)와 백엔드(backend)를 생성하며, API(애플리케이션 프로그래밍 인터페이스) 문서를 작성하는 여러 에이전트를 비용 없이 즉석에서 가동한다고 상상해 보십시오:

## 서브 에이전트

서브 에이전트는 파일 검색, 코드 탐색, 연구와 같은 작업을 각자의 컨텍스트(context) 내에서 병렬로 실행할 수 있습니다.

더 긴 코딩 세션이 더욱 생산적으로 변합니다.

일부 모델은 필요할 때 서브 에이전트를 트리거하지만(minimax-m2.5, glm-5, kimi-k2.5), 모델에게 “use/spawn/create subagents(서브 에이전트 사용/생성/만들기)”라고 지시하여 서브 에이전트를 강제로 트리거할 수도 있습니다.

다음은 실행할 수 있는 프롬프트(prompt) 예시입니다:

```
spawn subagents to explore the auth flow, payment integration,
 and notification system
```

(인증 흐름, 결제 통합, 알림 시스템을 탐색하기 위한 서브 에이전트 생성)

```
audit security issues, find performance bottlenecks, 
and check accessibility in parallel with subagents
```

(서브 에이전트를 이용해 보안 문제 감사, 성능 병목 현상 파악, 접근성 검사 병렬 실행)

```
create subagents to map the database queries, trace the API routes, 
and catalog error handling patterns
```

(데이터베이스 쿼리 매핑, API 경로 추적, 오류 처리 패턴 분류를 위한 서브 에이전트 생성)

처음부터 시작하여 어떻게 작동하는지 살펴보겠습니다.

### 서브 에이전트를 이용한 API 테스트

이제 앱이 실행 중이니, 다음 서브 에이전트들을 가동하여 API를 테스트할 수 있습니다.

```
spawn subagents to test the FastAPI backend. One subagent should 
check if the API routes are correct, another should test the database models,
 and another should create a simple test script to verify the endpoints work.
```

(FastAPI 백엔드를 테스트하기 위한 서브 에이전트 생성. 한 서브 에이전트는 API 경로가 올바른지 확인하고, 다른 서브 에이전트는 데이터베이스 모델을 테스트하며, 또 다른 서브 에이전트는 엔드포인트(endpoint) 작동 여부를 확인하는 간단한 테스트 스크립트(script)를 생성해야 합니다.)

(이미지를 전체 크기로 보려면 Enter 키를 누르거나 클릭하세요)

세 에이전트가 3분 이내에 작업을 마쳤습니다:

*   API 경로 검사기 — 도구 사용 4회, 10.5k 토큰
*   데이터베이스 모델 테스터 — 도구 사용 6회, 10.5k 토큰
*   테스트 스크립트 생성기 — 도구 사용 8회, 13.6k 토큰

이보다 더 좋을 수는 없습니다:

(이미지를 전체 크기로 보려면 Enter 키를 누르거나 클릭하세요)

테스트 결과는 인상적이었고, 서브 에이전트들은 다음과 같은 문제들을 발견했습니다:

**API 경로 문제:**

*   경로 순서 버그 (심각) — `/expenses/summary`는 `/expenses/{id}` 앞에 위치해야 함
*   CORS(교차 출처 리소스 공유) 보안 취약점 (심각) — 와일드카드 오리진(wildcard origin)과 함께 `allow_credentials=True` 사용
*   `DELETE` 메서드에 `response_model` 누락 (경미)

**데이터베이스 모델 문제:**

*   사용되지 않는 `datetime.utcnow` (심각) — `datetime.now(timezone.utc)`를 사용해야 함
*   금액 값에 Float(부동 소수점) 사용 (심각) — `Numeric(10,2)`를 사용해야 함
*   문자열 길이 제한 없음 (경고)
*   입력 유효성 검사 없음 (경고)

세 번째 에이전트는 모든 CRUD(생성, 읽기, 업데이트, 삭제) 작업을 다루는 완전한 테스트 스크립트를 `backend/test_endpoints.py`에 생성했습니다.

(이미지를 전체 크기로 보려면 Enter 키를 누르거나 클릭하세요)

이제 이 서브 에이전트들이 작동하니, 웹 검색 기능을 추가하여 다음 단계로 나아가 볼까요?

## 웹 검색

Ollama의 웹 검색 기능은 Anthropic(앤트로픽) 호환성 계층(compatibility layer)에 내장되어 있습니다.

모델이 최신 정보를 필요로 할 때, Ollama는 추가 구성 없이 검색을 처리하고 결과를 반환합니다.

서브 에이전트는 웹 검색을 활용하여 주제를 병렬로 연구하고 실행 가능한 결과를 도출할 수 있습니다.

프롬프트 예시:

```
research the postgres 18 release notes, audit our queries for 
deprecated patterns, and create migration tasks
```

(PostgreSQL(포스트그레스SQL) 18 릴리스 노트를 연구하고, 사용되지 않는 패턴에 대해 쿼리를 감사하며, 마이그레이션(migration) 작업을 생성)

```
create 3 research agents to research how our top 3 competitors price their API
 tiers, compare against our current pricing, and draft recommendations
```

(상위 3개 경쟁업체가 API 티어(tier)를 어떻게 책정하는지 조사하고, 현재 가격과 비교하며, 권장 사항을 초안하는 3개의 연구 에이전트 생성)

```
study how top open source projects handle their release process, 
review our CI/CD pipeline, and draft improvements
```

(최고의 오픈 소스 프로젝트가 릴리스(release) 프로세스를 어떻게 처리하는지 연구하고, CI/CD(지속적 통합/지속적 배포) 파이프라인(pipeline)을 검토하며, 개선 사항 초안 작성)

### 웹 검색 테스트: API 문제 연구

서브 에이전트가 이전에 발견했던 문제들을 기억하십니까?

사용되지 않는 `datetime.utcnow`, CORS 취약점, 그리고 금액 값에 사용된 Float(부동 소수점) 말입니다.

웹 검색이 최신 모범 사례를 통해 이러한 문제들을 해결하는 데 도움이 될 수 있는지 확인하고 싶었습니다.

```
spawn research agents to find the latest FastAPI 0.115 release features, 
research best practices for expense tracker API design, and summarize 
Python REST API security guidelines
```

(최신 FastAPI 0.115 릴리스 기능을 찾고, 지출 추적기 API 설계의 모범 사례를 연구하며, Python(파이썬) REST API 보안 가이드라인을 요약하기 위한 연구 에이전트 생성)

(이미지를 전체 크기로 보려면 Enter 키를 누르거나 클릭하세요)

세 개의 연구 에이전트가 가동되었습니다:

*   FastAPI 0.115 기능 — 최신 릴리스 연구
*   지출 추적기 API 모범 사례 — 설계 패턴
*   Python REST API 보안 — 보안 가이드라인

결과는 구조화된 표와 실행 가능한 권장 사항으로 돌아왔습니다.

**FastAPI 0.115 기능:**

연구 에이전트는 FastAPI 0.115.x 버전이 요청 매개변수(parameter)에 대한 Pydantic(파이단틱) 모델 지원을 도입했음을 발견했습니다. 이제 쿼리(query) 매개변수, 헤더(header), 쿠키(cookie)를 유효성 검사된 Pydantic 모델로 그룹화할 수 있습니다.

(이미지를 전체 크기로 보려면 Enter 키를 누르거나 클릭하세요)

```python
class FilterParams(BaseModel):
    limit: int = Field(100, gt=0, le=100)
    offset: int = Field(0, ge=0)

@app.get("/items/")
async def read_items(filter_query: Annotated[FilterParams, Query()]):
    return filter_query
```

**지출 추적기 API 모범 사례:**

*   복수 명사 및 케밥 케이스(kebab-case) 사용: `/api/v1/expenses`
*   `/api/v1/` 접두사(prefix)로 API 버전 관리
*   금액에는 Float(부동 소수점) 대신 Decimal(십진수) 사용
*   대규모 데이터셋(dataset)을 위한 커서 기반 페이지네이션(pagination)
*   오류 형식에 RFC 7807 문제 상세(Problem Details) 사용

**Python REST API 보안 가이드라인:**

*   짧은 만료 기간(15~30분)의 JWT(JSON Web Token)
*   `allow_credentials=True`와 함께 `allow_origins=["*"]`를 절대 사용하지 말 것
*   속도 제한을 위해 `slowapi` 사용
*   HSTS 헤더와 함께 TLS 1.2 이상 적용
*   매월 `pip-audit` 실행

연구 에이전트들은 코드 검토 서브 에이전트들이 발견한 내용, 즉 CORS 설정, 금액에 대한 Float 사용, 그리고 사용되지 않는 datetime을 확인해 주었습니다. 이제 최신 모범 사례로 뒷받침되는 해결책을 얻게 된 것입니다.

하지만 모든 Ollama 클라우드 모델이 이렇게 빠르게 작동하는 것은 아닙니다. 그렇다면 어떤 Ollama 클라우드 코드가 Claude Code 서브 에이전트와 잘 작동할까요?

## 권장 클라우드 모델

모든 모델이 Ollama에서 서브 에이전트와 웹 검색을 지원하는 것은 아닙니다. 다음 세 가지 모델이 작동합니다:

*   `minimax-m2.5:cloud`
*   `glm-5:cloud`
*   `kimi-k2.5:cloud`

저는 이 글 전반에 걸쳐 `minimax-m2.5:cloud`로 테스트했습니다. 서브 에이전트 생성, 병렬 작업 실행, 웹 검색을 문제없이 처리했습니다.

모델을 전환하려면 Claude Code에서 `/model`을 실행하고 원하는 옵션을 선택하십시오.

제 지출 추적기 구축에는 세 개의 서브 에이전트에서 3만 개 이상의 토큰이 사용되었습니다. 유료 API였다면 상당한 비용이 들었을 것입니다.

## 마지막 생각

Ollama에서 서브 에이전트와 웹 검색을 통해 Claude Code를 실행하는 것은 API 요금에 신경 쓰지 않고 실험할 수 있는 방식을 변화시킵니다.

이 테스트에서 저는 세 개의 병렬 서브 에이전트를 사용하여 바닐라 JS(Vanilla JS) 프런트엔드와 FastAPI(패스트API) 백엔드를 갖춘 완전한 지출 추적기를 구축했는데, 이는 인상적인 결과입니다.

(이미지를 전체 크기로 보려면 Enter 키를 누르거나 클릭하세요)

그 다음 코드 감사를 위해 세 개의 서브 에이전트를 추가로 실행했습니다. 이들은 CORS 취약점 및 사용되지 않는 datetime 패턴과 같은 실제 문제들을 발견했습니다.

하지만 중요한 점은 이 설정이 프로토타입(prototype) 제작 및 학습에 가장 적합하다는 것입니다. 프로덕션(production) 작업이나 민감한 코드베이스(codebase)의 경우 유료 API를 고수해야 합니다.

이 시리즈의 이전 튜토리얼을 놓치셨다면:

*   제가 (새로운) Ollama Claude Code, Codex, OpenCode 실행을 테스트했습니다 (더 이상 설정 불필요)
*   새로운 Claude Code Ollama 워크플로를 시도해봤습니다 (놀랍도록 자유롭습니다)
*   Claude Code + MiniMax M2.5를 테스트했습니다 (단 한 번의 CLI 도구 빌드로 저를 깜짝 놀라게 했습니다)

다음에 Anthropic에 장애가 발생하거나 토큰 소모 없이 무언가를 테스트하고 싶을 때, 무엇을 해야 할지 아실 겁니다.
