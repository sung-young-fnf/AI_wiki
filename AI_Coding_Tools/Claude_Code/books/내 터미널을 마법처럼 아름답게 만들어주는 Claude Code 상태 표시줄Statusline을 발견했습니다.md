# 내 터미널을 마법처럼 아름답게 만들어주는 Claude Code 상태 표시줄(Statusline)을 발견했습니다

기본으로 제공되는 클로드 코드(Claude Code) 상태 표시줄은 잊으셔도 좋습니다. 아름답고 생산적인 작업 환경을 중시하는 분들이라면 **CCstatusline**이야말로 진정한 해답입니다.

클로드 코드에서 많은 시간을 보낸다면, 터미널이 곧 여러분의 거주지나 다름없다는 것을 잘 아실 겁니다. 기본 상태 표시줄(statusline)도 제 역할을 다하긴 하지만, 훨씬 더 멋지고 효율적으로 바꿀 수 있습니다.

최근 CCstatusline을 발견해 직접 테스트해 보았는데, 이는 클로드 코드 터미널의 시각적 느낌과 사용 경험을 완전히 바꿔 놓았습니다. 이 도구는 실시간 모델 정보, Git 브랜치(branch), 토큰 사용량(token usage), 세션 비용(session cost), 블록 타이머(block timer) 및 전체 파워라인(Powerline) 지원 기능을 추가해 줍니다. 또한 설정 파일을 직접 수정할 필요 없이 상태 표시줄에서 바로 완벽하게 커스터마이징이 가능합니다.

이 글에서는 CCstatusline이 무엇인지, 어떻게 설치하고 내장된 TUI(터미널 사용자 인터페이스)를 통해 설정하는지, 그리고 실제 클로드 코드 세션에서 어떻게 보이는지 소개하겠습니다.

---

### CCstatusline이란 무엇인가요?

CCstatusline은 클로드 코드를 위해 구축된 오픈 소스 npm 패키지로, 실시간 지표를 포함한 완벽하게 커스터마이징 가능한 상태 표시줄을 제공합니다.

*   **파워라인(Powerline) 지원**
*   **다양한 테마**
*   **설정을 위한 내장 TUI(Terminal User Interface)**

실행하는 즉시 다음과 같은 실시간 미리보기를 확인할 수 있습니다:
`Ctx: 18.6k | ⎇ main | (+42,-10)`

이것은 기본 설정일 뿐이며, 설정을 마치고 나면 완전히 새로운 모습으로 탈바꿈합니다.

---

### 표시할 수 있는 정보 (위젯)

CCstatusline은 위젯 기반으로 작동합니다. 상태 표시줄에 표시할 항목을 선택하고 원하는 대로 배치할 수 있습니다.

*   **모델명(Model Name)**: 현재 세션에서 활성화된 클로드 모델
*   **컨텍스트 길이(Context Length)**: 현재 토큰 사용량을 한눈에 확인
*   **Git 브랜치(Git Branch)**: 현재 작업 중인 브랜치
*   **Git 변경 사항(Git Changes)**: 추가 및 삭제된 코드 라인 수
*   **세션 지속 시간(Session Duration)**: 현재 세션이 실행된 시간
*   **블록 타이머(Block Timer)**: 클로드 코드의 5시간 단위 블록 진행 상황 추적
*   **세션 비용(Session Cost)**: 실시간 USD 비용 추적
*   **현재 작업 디렉토리(Current Working Directory)**: 스마트한 경로 단축 기능 포함
*   **커스텀 텍스트(Custom Text)**: 환경 표시기 같은 정적 레이블
*   **커스텀 명령(Custom Commands)**: 임의의 셸 명령을 실행하고 그 결과를 실시간으로 표시

무제한으로 상태 표시줄 라인을 구성할 수 있으며, 각 라인마다 고유한 위젯 세트를 배치할 수 있습니다.

### 설정 파일이 필요 없는 TUI 환경

이 점이 CCstatusline의 가장 큰 장점입니다. JSON이나 YAML 파일을 수동으로 편집하는 대신, 대화형 터미널 UI를 통해 모든 것을 설정할 수 있습니다.

`npx ccstatusline@latest`를 실행하면 7가지 옵션이 있는 메인 메뉴가 나타납니다.

1.  **📝 Edit Lines**: 위젯 추가, 삭제, 순서 변경
2.  **🎨 Edit Colors**: 위젯별 전경색, 배경색, 굵게 표시 설정
3.  **⚡ Powerline Setup**: 파워라인 폰트 및 화살표 구분 기호 활성화
4.  **💻 Terminal Options**: 색상 레벨 및 터미널 너비 동작 설정
5.  **🌐 Global Overrides**: 모든 위젯에 패딩, 구분 기호, 굵게 적용
6.  **📦 Install to Claude Code**: 클로드 코드 설정에 활성화
7.  **💾 Save & Exit**

모든 변경 사항은 화면 상단에 실시간 미리보기로 표시되므로, 적용하기 전에 상태 표시줄의 모습을 정확히 확인할 수 있습니다.

---

### 파워라인(Powerline) 지원

시각적으로 가장 큰 차이를 만드는 기능입니다. 파워라인 모드는 평범한 `|` 구분 기호를 위젯 사이를 흐르는 화살표 모양의 글리프(glyph)로 대체하여, Oh My Zsh나 Starship 같은 인기 있는 설정처럼 세련되고 전문적인 느낌을 줍니다.

이를 사용하려면 터미널에 **너드 폰트(Nerd Font)**가 설치되어 있어야 합니다. CCstatusline은 'Powerline Setup' 메뉴에서 이를 처리합니다. `i`를 눌러 폰트를 설치하고 `t`를 눌러 파워라인 모드를 켤 수 있습니다.

### CCstatusline 설치 및 설정 방법

설치는 매우 빠르며, 명령어 하나와 대화형 설정으로 끝납니다.

#### 1. 빠른 시작
터미널에서 바로 다음 명령을 실행하세요:

```bash
# npm 사용 시
npx ccstatusline@latest

# 또는 Bun 사용 시 (더 빠름)
bunx ccstatusline@latest
```

실행 즉시 설정 TUI가 시작되며 상단에서 실시간 미리보기를 볼 수 있습니다.

#### 2. 파워라인 설정
메인 메뉴에서 `⚡ Powerline Setup`으로 이동합니다.
*   `i`를 눌러 파워라인 폰트를 자동 설치합니다.
*   `t`를 눌러 파워라인 모드를 활성화합니다.

Windows 터미널 사용자의 경우, 기호가 깨져 보인다면 winget을 통해 너드 폰트를 설치하세요:
`winget install DEVCOM.JetBrainsMonoNerdFont`

#### 3. 위젯 구성
`📝 Edit Lines` → `Line 1`에서 구성을 시작합니다.
*   `↑↓`: 위젯 이동
*   `a`: 새 위젯 추가
*   `d`: 위젯 삭제
*   `←→`: 위젯 유형 변경
*   `r`: 원시 값(raw value) 모드 (레이블 제거)

추천 위젯: **Model, Block Timer, Session Cost, Git Branch, Context Length.**

#### 4. 색상 커스터마이징
`🎨 Edit Colors`에서 위젯별 색상을 설정합니다. 16색 기본 모드부터 256색, 트루컬러(Truecolor)까지 지원합니다.

#### 5. 클로드 코드에 설치
설정이 마음에 든다면 메인 메뉴의 `📦 Install to Claude Code`를 선택하세요. 이 작업은 CCstatusline 훅(hook)을 클로드 코드 설정에 직접 기록합니다. 이제 모든 클로드 코드 세션에서 커스텀 상태 표시줄이 나타납니다.

마지막으로 `💾 Save & Exit`를 눌러 저장합니다.

### 유용한 위젯 상세 소개

#### 블록 타이머(Block Timer)
제가 가장 좋아하는 위젯입니다. 클로드 코드는 5시간 단위 대화 블록으로 작동합니다. 블록이 끝나면 컨텍스트가 초기화되는데, 이 타이머는 현재 어느 지점에 있는지 실시간으로 추적해 줍니다. 진행률 표시줄(progress bar) 모드로도 사용 가능합니다.

#### 세션 비용(Session Cost)
사용량 기반 요금제를 사용 중이라면 매우 유용합니다. 실시간으로 누적 비용을 USD로 보여줍니다. (클로드 코드 1.0.85 버전 이상 필요)

#### 커스텀 명령 위젯(Custom Command Widget)
가장 강력하지만 많은 분이 간과하는 위젯입니다. 임의의 셸 명령을 실행하고 그 출력을 상태 표시줄에 표시할 수 있습니다.
*   현재 시간 표시: `date +%H:%M`
*   현재 커밋 해시: `git rev-parse --short HEAD`
*   클로드 사용량 지표: `npx -y ccusage@latest statusline`

### 마치며

CCstatusline은 설치하기 전까지는 필요성을 잘 모르다가, 한 번 써보면 없이는 못 사는 도구 중 하나입니다. 특히 **블록 타이머** 하나만으로도 설치할 가치가 충분합니다. 5시간 블록 중 어디쯤 와 있는지 알면 작업의 속도를 조절하고 계획하는 방식이 완전히 달라지기 때문입니다.

클로드 코드에서 진지하게 작업을 수행하는 개발자라면, 이 도구는 작업 환경을 업그레이드할 수 있는 가장 간단하고 확실한 방법입니다. 지금 바로 시도해 보세요!
