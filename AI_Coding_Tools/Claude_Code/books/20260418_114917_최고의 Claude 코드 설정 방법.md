# 최고의 Claude 코드 설정 방법

이 글에서는 에이전트 기반(agentic) 개발 생산성을 10배 향상시킨 저의 Claude 코드 설정 방법을 소개합니다.

지난 몇 년간 저는 코딩 에이전트(coding agent)가 핵심 작업을 수행할 때 소프트웨어가 어떻게 구축되는지에 대해 실험해 왔습니다.

개발자 출신으로서, 소프트웨어 제품을 생계로 만드는 대부분의 사람들과 마찬가지로, 저는 도구들이 빠르게 변화하는 것을 지켜봤습니다. 자동 완성 및 코드 제안(code suggestion)으로 시작된 것이 이제는 훨씬 더 강력한 능력을 갖춘 존재가 되었습니다. 바로 작업을 추론하고, 기능을 스캐폴딩(scaffolding)하며, 코드베이스의 상당 부분을 리팩토링(refactoring)하고, 문제를 디버그(debug)하며, 놀라운 속도로 풀스택(full-stack) 앱을 출시하는 데 도움을 줄 수 있는 에이전트들입니다.

사람들이 현재 '바이브 코딩(vibe coding)'이라고 부르는 지난 3년간, 저는 다양한 도구와 워크플로우를 시도해 봤습니다. 대부분 다음 세 가지 범주로 나뉩니다.

*   통합 개발 환경(IDE)
*   CLI(Command Line Interface) 내부의 코딩 에이전트
*   데스크톱 앱

물론 이 모든 도구에는 장단점이 있습니다.

## IDE vs CLI vs 데스크톱 앱

Antigravity, Cursor, Kiro와 같은 IDE는 모두 VS Code의 포크(fork) 버전입니다. 이들은 개발자에게는 익숙한 환경을, 초보자에게는 사용하기 쉬운 UI를 제공합니다.

하지만 단점은 속도입니다.

모든 UI 구성 요소가 한 번에 로드되기 때문에 지연 시간(latency)이 눈에 띄게 발생할 수 있습니다. 저에게 속도는 매우 중요하기 때문에, IDE가 느려지거나 충돌하면 매우 답답함을 느낍니다.

또 다른 문제는 프롬프트 입력 필드와 터미널(terminal) 사이를 전환해야 한다는 점입니다. 이 부분이 다소 번거롭고, 둘 사이에서 혼란을 겪을 때가 많습니다.

다음은 터미널입니다.

하드코어 개발자들은 항상 터미널 사용을 선호할 것입니다. 명령을 빠르게 실행할 수 있고, 지연 시간으로 인한 불편함도 적습니다. 빠르고 유연하며, 익숙해지면 자연스럽게 느껴집니다.

Warp를 사용하기 전에는 Mac의 일반 CLI에서 Claude Code를 실행했습니다. 개발자로서 빠르고 유연하며 유용했습니다.

하지만 몇 가지 문제점이 있었습니다.

*   소스 코드(source code)를 확인하려면 변경 사항을 보기 위해 별도의 IDE를 열어야 했습니다.
*   여러 개의 열린 터미널에서 여러 코딩 에이전트 인스턴스(instance)를 정리하는 것은 탭(tab) 없이는 복잡했습니다.
*   에이전트가 생각 중이거나, 토큰(token)이 부족하거나, 작업이 완료되었을 때 알림이 없었습니다.

프로젝트가 커질수록 코드 검토, 작업 모니터링, 여러 에이전트 관리 등이 어려워졌습니다.

데스크톱 앱은 제가 가장 선호하지 않는 옵션입니다. 현재 OpenAI의 Codex와 Anthropic의 Claude 데스크톱 앱이 가장 인기가 많습니다.

저의 작업 방식에는 너무 단순하고 느립니다. 초보자에게는 명령(command)과 같은 기술적인 문제에 신경 쓸 필요가 없으므로 괜찮을 수 있습니다. 필요한 것은 좋은 프롬프트(prompt)를 작성하고 AI가 대부분의 작업을 처리하도록 두는 것뿐입니다.

하지만 중요한 점은 이러한 환경 중에서 하나를 선택할 필요가 없다는 것입니다.

CLI의 속도와 가벼움, 그리고 더 시각적인 도구의 사용 편의성을 모두 제공하는 Warp라는 또 다른 에이전트 기반 코딩 도구가 있습니다.

이제 Warp를 사용하여 Claude Code를 어떻게 활용하는지 설명해 드리겠습니다.

## 제가 Warp로 전환한 이유

Warp는 터미널에서 시작된 에이전트 기반 개발 환경입니다. 약 1년 전에 사용해 보고 즉시 마음에 들었습니다.

즉시 매료된 점은 이것이 무겁고 복잡한 IDE처럼 느껴지지 않으면서도, 모든 것을 스스로 해결해야 하는 밋밋한 터미널 같지도 않다는 것이었습니다. 중간 지점에 위치하며, 이것이 바로 제가 코딩 에이전트와 작업할 때 원하는 지점입니다.

가볍고 빠르며, 여러 모델을 지원합니다. 에이전트 기반처럼 느껴지며, 터미널과 완전한 IDE 사이의 무언가를 제공하면서도 어느 한쪽에도 구속되지 않게 해줍니다.

터미널에서 Claude Code를 사용하는 데 좋았던 부분들을 대체하지 않았습니다. 그저 주변 경험을 더 좋게 만들었습니다.

## Warp에서 Claude Code를 사용하는 방법

프롬프트 필드에서 Warp는 터미널 명령을 받아들입니다. 따라서 '/model'을 입력하면 지원되는 모든 모델이 표시됩니다. 'Full Terminal Use' 탭을 열면 사용 가능한 모든 Claude 모델을 볼 수 있습니다.

다른 모델을 선택하거나 Warp의 Oz 오케스트레이터(orchestrator)를 실행하여 에이전트 스캐폴딩을 처리할 수도 있습니다.

이제 Warp의 내장 컴포저(composer) 대신 Claude Code 내에서 작업하고 싶을 때면 언제든지 'claude'를 실행하고 CLI 내에서 사용하듯이 계속 사용할 수 있습니다.

정말 멋지지 않나요? 컴포저로 다시 전환하려면 'Compose' 버튼을 클릭하기만 하면 됩니다.

## CLI 에이전트 툴바

제가 기본 CLI 터미널보다 Warp를 선호하는 또 다른 이유는 툴바(toolbar) 섹션의 사용자 정의 가능한 컨트롤(control) 때문입니다.

고급 사용자(power user)는 자신의 에이전트 워크플로우를 맞춤 설정하기를 원합니다. 툴바는 터미널을 떠나지 않고도 필요한 컨트롤을 즉시 사용할 수 있게 해줍니다.

툴바에 접근하려면 툴바 섹션을 마우스 오른쪽 버튼으로 클릭하고 'CLI agent toolbelt 편집'을 선택합니다.

이렇게 하면 툴벨트(toolbelt)의 왼쪽과 오른쪽에 표시될 도구들을 사용자 정의할 수 있는 모달(modal) 창이 나타납니다.

저의 기본 설정은 다음과 같습니다: 프롬프트 입력을 선호하므로 음성 입력은 사용하지 않고, 솔로 개발자이므로 세션 공유는 하지 않으며, 항상 어떤 프로젝트 폴더에서 어떤 브랜치(branch)가 활성화되어 있는지 확인하고 싶습니다.

팀으로 작업하는 경우, 현재 로그인한 사용자 표시 및 다른 팀원과 세션을 공유하는 단축키를 활성화하는 것이 좋습니다.

저는 에이전트와 작업할 때 실제로 필요한 것들에 빠르게 접근할 수 있는 단일 환경에 머무릅니다. 더 나은 컨텍스트(context)를 제공하거나 변경 사항을 검토하기 위해 여러 앱 사이를 오갈 필요가 없습니다.

이것이 Warp가 일반 터미널보다 더 완전하게 느껴지는 큰 이유 중 하나입니다.

## 통합 알림

에이전트는 백그라운드에서 실행되므로 항상 즉각적인 주의가 필요한 것은 아닙니다.

*   때로는 생각 중이거나,
*   때로는 응답을 기다리고 있거나,
*   때로는 토큰이 부족하거나,
*   때로는 작업이 완료되어 다음 지시를 기다리고 있습니다.

일반 터미널에서는 이 모든 것을 직접 확인해야 합니다.

Warp는 탭(tab) 전반에 걸쳐 모든 에이전트와 동기화 상태를 유지하는 통합 알림 UI로 이 문제를 해결합니다. 데스크톱 알림도 지원하여 멀티태스킹(multitasking) 시 특히 유용합니다.

'Claude Code 알림 활성화'를 클릭하면 Claude Code 플러그인(plugin)이 설치됩니다.

오래 걸리는 작업이 있는 대규모 프로젝트에서는 알림에 계속 의존합니다. 예전에는 에이전트가 멈췄는지 또는 저를 기다리고 있는지 확인하기 위해 터미널 창 사이를 Alt-Tab으로 전환하곤 했습니다. 그런 방식은 금방 지겨워졌지만, 이제는 그럴 필요가 없습니다.

## 코드 검토 댓글

에이전트는 코드를 빠르게 작성할 수 있지만, 코드가 실제로 좋은지 확인하는 것은 여전히 인간의 책임입니다.

그래서 저는 Warp의 코드 검토 댓글(comment) 기능이 마음에 듭니다.

코드 검토를 에이전트 워크플로우와 완전히 분리된 것으로 취급하는 대신, Warp는 이 과정을 긴밀하게 유지합니다. 저는 에이전트가 생성한 코드를 검토하고, 인라인(inline) 댓글을 남긴 다음, 해당 피드백을 실행 중인 타사 에이전트 세션에 직접 보낼 수 있습니다.

다음은 예시입니다.

이는 에이전트가 제가 다른 창에서 모든 것을 수동으로 다시 설명할 필요 없이 매우 구체적인 피드백을 기반으로 반복 작업(iterate)을 수행할 수 있음을 의미합니다.

이는 정말 스마트한 워크플로우 개선입니다.

현재 에이전트 기반 개발의 가장 큰 약점 중 하나는 생성은 쉽지만, 반복 작업은 여전히 번거롭다는 것입니다. Warp는 이 과정을 훨씬 더 매끄럽게 만듭니다. 한곳에서 생성, 검토, 댓글 작성, 수정 작업을 모두 수행할 수 있습니다.

## 이러한 설정이 중요한 이유

코딩 에이전트를 이미 사용하고 있거나 더 자주 사용하려는 경우, 이러한 설정은 몇 가지 중요한 이유 때문에 중요합니다.

첫 번째 이유는 속도입니다.

모델 속도만을 의미하는 것이 아닙니다. 개발자로서의 실제 작업 속도를 의미합니다. 적절한 설정을 얼마나 빠르게 시작하고, 컨텍스트를 전달하고, 변경 사항을 검토하고, 진행 상황을 모니터링하며, 집중력을 잃지 않고 작업을 전환할 수 있을까요? 바로 이 지점에서 에이전트를 둘러싼 환경이 차이를 만듭니다. 일반 터미널은 크게 도움이 되지 않으며, IDE는 도움이 될 수 있지만, 그 대가로 무거움과 지연 시간을 지불해야 합니다.

두 번째 이유는 가시성(visibility)입니다.

에이전트를 더 많이 사용하기 시작하면, 각 세션이 무엇을 하고 있는지 더 잘 파악해야 합니다. 에이전트가 주의를 필요로 할 때, 작업이 완료되었을 때, 또는 토큰 제한에 도달할 때를 쉽게 알 수 있는 방법이 필요합니다. 이는 여러 창을 덕트 테이프(duct-tape)로 엮어 사용하는 것보다, 환경 자체가 이러한 목적을 위해 설계되었을 때 훨씬 쉽습니다.

세 번째 이유는 유연성(flexibility)입니다.

단일 모델에 국한되지 않습니다. 필요한 것에 따라 모델을 전환할 수 있습니다. 때로는 더 빠른 것을 원하고, 때로는 더 강력한 추론 능력을 원하며, 때로는 비용을 절약하고 싶을 수도 있습니다. 올바른 작업에 적합한 모델을 선택할 수 있는 것은 더 효율적으로 작업하고 비용을 통제하는 가장 쉬운 방법 중 하나입니다.

네 번째 이유는 Warp가 개발자들의 의견에 귀 기울이는 것 같다는 점입니다.

이는 에이전트 기반 개발처럼 빠르게 변화하는 분야에서 매우 중요합니다. 이 분야에서 성공하는 도구는 빠르게 움직이고, 개발자들이 실제로 어떻게 작업하는지에 주의를 기울이며, 유용한 기능들을 계속 출시하는 도구입니다. 제가 보기에 Warp는 대부분의 경쟁사보다 이 점에서 더 일관적입니다.

그들이 얼마나 귀 기울이고, 모든 주요 우려 사항을 얼마나 신속하게 해결하는지 확인하려면 그들의 X(구 트위터) 계정을 팔로우해 보시길 권합니다.

개인적인 경험으로는 유용했습니다. 저는 Warp로 앱을 개발해 왔으며, 이 덕분에 저의 Claude Code 워크플로우가 눈에 띄게 향상되었습니다. 이는 Claude 자체의 기능이 변해서가 아니라, Claude를 둘러싼 모든 것을 더 원활하게 만들기 때문입니다.

## 최종 의견

모든 것을 요약하자면, 저의 궁극적인 Claude Code 설정은 특정 환경 하나에만 전적으로 의존하지 않고, 여러 환경의 장점을 결합하는 것입니다.

저는 IDE, CLI, 데스크톱 앱 간의 장단점에 대해 이야기했습니다. IDE는 익숙하고 접근하기 쉽지만 무겁게 느껴질 수 있습니다. CLI는 빠르고 유연하지만, 프로젝트와 에이전트 세션이 늘어남에 따라 복잡해집니다. 데스크톱 앱은 단순하지만, 저에게는 진지한 작업에는 너무 제한적으로 느껴질 때가 많습니다.

이것이 Warp가 이러한 설정에서 잘 작동하는 이유입니다.

Warp는 터미널에서 얻고자 하는 속도와 가벼움을 제공하는 동시에, 코드 검토, 작업 모니터링, 여러 에이전트 세션, 프롬프트 구성 등에서 더 나은 경험을 선사합니다. 수직 탭(Vertical Tabs), 알림, CLI 에이전트 툴바, 코드 검토 댓글, 모델 유연성과 같은 기능들은 실제 에이전트 기반 개발에 훨씬 더 실용적인 워크플로우를 구축합니다.

개인적으로 Warp와 함께 Claude Code를 사용하면서 저는 더 생산적이고, 체계적이며, 대규모 프로젝트를 훨씬 더 편안하게 작업할 수 있게 되었습니다. 이는 제가 몰입 상태를 유지하고, 더 많은 에이전트를 동시에 관리하며, 워크플로우의 번거로운 부분에 시간을 덜 소비하도록 돕습니다.

물론, 이 모든 것은 저의 개인적인 선호도에 기반한 것입니다. 사람마다 작업 방식이 다르고, 그것은 전적으로 괜찮습니다. 좋은 점은 Warp가 앱을 탐색하고 원하는 대로 설정을 사용자 정의할 수 있는 충분한 유연성을 제공한다는 것입니다.

만약 Claude Code를 이미 사용하고 있다면, Warp를 시도해 볼 가치가 있다고 생각합니다.
