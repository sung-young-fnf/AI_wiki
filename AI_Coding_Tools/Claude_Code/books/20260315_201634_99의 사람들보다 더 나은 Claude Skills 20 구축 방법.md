“매번 AI에게 똑같은 지시를 내리는 것이 번거롭다…”  
“AI가 회사 규정과 형식을 전혀 기억하지 못한다…”

Anthropic은 이러한 문제들을 해결할 수 있는 놀라운 기능을 막 업그레이드했습니다. 바로 “Claude Skills (클로드 스킬)”입니다. 이것은 단순한 AI 에이전트 (AI agent) 업데이트가 아닙니다. AI에게 비즈니스 프로세스와 전문 지식을 가르쳐 회사 요구에 맞춰 궁극적인 전문가로 성장시킬 수 있는 차세대 기능입니다.

Claude Skills는 Claude에게 특정 작업과 워크플로 (workflow)를 가르치는 가장 강력한 기능입니다. 이 기능을 사용하면서 제가 가장 흥미롭게 느낀 점은 매번 대화할 때마다 선호도와 프로세스를 설명할 필요가 없다는 것입니다.

스킬 (Skill)은 간단한 폴더에 패키징된 일련의 지침으로, 한 번 설정하면 매번 이점을 얻을 수 있습니다. 사양 (specs)에서 프런트엔드 (front-end) 디자인을 생성하거나 팀의 스타일 가이드 (style guide)에 맞춰 문서를 작성하는 것과 같이 일관된 워크플로가 있을 때 특히 빛을 발합니다.

제 경험상, 스킬은 단순히 “매크로 (macros)”나 “템플릿 (templates)”이 아니라 Claude의 의사 결정 능력을 향상시키는 “지식 기반 (knowledge base)” 역할을 합니다. 코드 실행 (code execution) 및 문서 작성과 같은 내장 기능 (built-in functions)과 특히 잘 작동하여, 복잡한 작업도 원활하게 처리할 수 있게 해줍니다.

## 스킬 (Skills)이란 무엇인가요?

스킬은 Claude Code가 특정 작업을 수행하기 위해 참조할 수 있는 재사용 가능한 지식과 절차 조각입니다. 각 스킬은 주로 마크다운 (Markdown) 파일 (SKILL.md)로 정의되며, 필요에 따라 관련 스크립트 (scripts)와 리소스 (resources)를 포함할 수 있습니다.

Claude Code는 사용자 요청에 응답하여 적절한 스킬을 로드 (load)하고 지침에 따라 작업을 실행합니다. 이를 통해 복잡한 워크플로와 일상적인 작업을 일관되게 자동화할 수 있습니다.

Claude Code 자체는 참조 지식 기반으로 작동하여 스크립트를 직접 실행하고 워크플로를 관리할 수 있게 해줍니다. 따라서 규칙 기반 시스템 (rule-based system)을 사용하여 어떤 작업을 언제 수행해야 할지 정의하고 실행할 수 있습니다.

## 스킬 크리에이터 (Skill-creator)란 무엇인가요?

스킬 크리에이터는 스킬을 한 번에 생성, 테스트 및 개선할 수 있게 해주는 “메타 스킬 (meta skill)”입니다.

대략적으로 스킬 크리에이터가 하는 일은 다음 다섯 가지입니다.

1.  “어떤 종류의 스킬을 개발하고 싶으신가요?”라고 묻습니다.
2.  SKILL.md 초안을 자동으로 생성합니다.
3.  테스트 프롬프트 (test prompt)로 실제로 실행하여 테스트합니다.
4.  결과를 평가하고 개선 사항을 제안합니다.
5.  만족할 때까지 2단계에서 4단계를 반복합니다.

핵심 아이디어는 Claude Code 자체적으로 SKILL.md를 수동으로 작성하고, 시도하며, 수정하는 과정을 거친다는 것입니다.

## SKILL.md 작성 방법

**파일 구조**

기본적으로 `.claude/skills` 폴더 아래에 스킬을 위한 SKILL.md 파일을 생성하기만 하면 스킬을 만들 수 있습니다.

SKILL.md의 내용은 YAML과 마크다운 (Markdown)으로 다음과 같이 작성됩니다.

```markdown
---
name: Your Skill Name
description: Brief description of what this Skill does and when to use it
---
# Your Skill Name
## Instructions
Provide clear, step-by-step guidance for Claude.
## Examples
Show concrete examples of using this Skill.
```

먼저 YAML 부분에 대해 설명하겠습니다.

```yaml
---
name: Your Skill Name
description: Brief description of what this Skill does and when to use it
---
```

이것은 메타데이터 (metadata)라고 불리며, 스킬을 생성하는 데 매우 중요한 부분입니다.

Claude는 시작 시 메타데이터를 읽고 각 스킬이 언제 존재하고 언제 사용 가능한지 파악하여 이를 시스템 프롬프트 (system prompt)에 통합합니다. 이 접근 방식은 컨텍스트 (context)를 불필요하게 부풀리지 않고도 많은 스킬을 가질 수 있게 해줍니다.

프롬프트 (prompt) 또는 요청이 스킬의 메타데이터와 일치하면 Claude는 파일 시스템 (file system)에서 SKILL.md를 읽어옵니다.

어떤 작업이 실제로 실행될지 여부의 정확성은 메타데이터의 내용에 크게 좌우되므로, 이는 매우 중요한 요소입니다.

다음으로 내용 섹션에 대해 설명하겠습니다.

```markdown
# Your Skill Name
## Instructions
Provide clear, step-by-step guidance for Claude.
## Examples
Show concrete examples of using this Skill.
```

메타데이터는 Claude가 시작될 때 항상 로드되지만, 내용 부분은 런타임 (runtime)에 로드됩니다. 그런 다음 에이전트 스킬 (agent skill)이 실행될 때, Claude는 내용 부분의 내용을 처리합니다.

공식 모범 사례 (best practices)는 SKILL.md를 500줄 미만으로 유지할 것을 권장합니다. 만약 이를 초과하면, 상세 참조 자료를 별도의 파일로 분리하십시오:

```
.claude/skills/
  my-skill/
    SKILL.md          ← 메인 지침 (500줄 이내)
    templates/        ← 템플릿 파일
    reference.md      ← 상세 참조 자료
```

SKILL.md의 지침을 사용하여 Claude가 필요할 때만 추가 파일을 로드하도록 안내하고, 모든 것을 한 번에 로드하는 대신 필요에 따라 압축을 해제하도록 하세요. 이것이 바로 점진적 공개 (Progressive Disclosure)입니다. 즉, 핵심 지침을 먼저 제공하고 필요에 따라 세부 정보를 펼쳐 보이는 방식입니다.

여기서 핵심은 에이전트 스킬이 효율적인 로딩 작업을 수행하더라도 내용 부분 또한 간결하게 유지하는 것이 좋다는 것입니다. Claude가 내용 부분을 로드할 때, 이는 대화 기록 및 다른 컨텍스트와 경쟁하게 됩니다.

그러므로 내용 섹션에서 시스템 프롬프트, 프로그래밍 언어 (programming languages), 라이브러리 (libraries) 등에 대한 일반적인 참조는 CLAUDE.md에서 생략하십시오. 고도로 정확한 스킬을 생성하는 비결 중 하나는 어떤 부분을 생략하고 내용 섹션의 어디부터 시작할지 결정하는 것입니다.

## 왜 이 스킬이 필요한가요?

PR (Pull Request) 검토 댓글에 응답할 때 다음과 같은 어려움에 직면했습니다:

*   모든 검토 댓글을 확인하는 데 시간이 많이 걸립니다.
*   처리되지 않은 댓글이 무엇인지 알기 어렵습니다.
*   검토 댓글의 내용을 Claude Code에 전달하는 것이 번거롭습니다.

이 스킬을 사용하면 Claude Code가 처리되지 않은 댓글을 자동으로 검색하고 수정 사항을 제안합니다.

## 스킬 작동 방식

**스킬은 단순히 명령어를 담고 있는 폴더입니다.**

스킬의 핵심은 SKILL.md 파일을 포함하는 폴더입니다. 이 마크다운 (Markdown) 파일은 YAML 프론트 매터 (YAML Front Matter)를 사용하여 메타데이터 (이름, 설명 등)를 정의하며, 본문에는 명확하고 단계별 작업 지침 및 예시가 포함됩니다.

```
my-Skill.zip
  my-Skill/
  Skill.md
  resources/
```

**Claude는 관련 스킬을 자동으로 검색하고 로드합니다.**

수동 스킬 트리거 (skill triggering)는 필요하지 않습니다. 세션 (session) 시작 시 Claude는 설치된 모든 스킬의 메타데이터 (이름 및 설명)를 스캔하고 이 간략한 정보를 시스템 프롬프트에 로드합니다.

요청이 스킬 설명과 일치하면 Claude는 해당 스킬의 전체 지침을 자동으로 읽어 로드합니다.

**“점진적 공개 (progressive disclosure)” 메커니즘은 스킬을 매우 효율적으로 만듭니다.**

스킬은 3단계 구조 (YAML 서문, 본문, 파일 참조)를 사용하여 정보를 모델 컨텍스트 (model context)에 점진적이고 온디맨드 (on-demand) 방식으로 공급함으로써, 한 번에 과부하되는 것을 방지하고 효율성 및 토큰 경제성 (token economy)을 향상시킵니다.

전체 크기로 이미지를 보려면 Enter 키를 누르거나 클릭하세요.

스킬은 토큰 효율성 (token efficiency)을 염두에 두고 설계되었습니다. 초기 로딩 시, 각 스킬은 메타데이터를 저장하는 데 불과 수십 개의 토큰만 사용합니다. 스킬에 대한 상세 지침은 스킬이 트리거 (trigger)될 때만 컨텍스트 창 (context window)에 표시됩니다.

이 온디맨드 로딩 (on-demand loading) 메커니즘은 컨텍스트 창이 가득 차서 모델 성능에 영향을 주지 않고도 많은 수의 스킬을 설치할 수 있음을 의미합니다.

더 복잡한 스킬의 경우, 여러 파일에 걸쳐 다른 지침을 분리할 수 있으며, Claude는 현재 작업에 필요한 부분만 읽어 토큰 (tokens)을 더욱 절약합니다.

## MCP 대 스킬 (MCP Vs Skills)

스킬은 이미 MCP (Model Context Protocol, 모델 컨텍스트 프로토콜)를 사용하는 사람들을 위한 또 다른 강력한 레이어 (layer)입니다. 저는 이 관계를 부엌과 요리 레시피에 비유하는 것이 가장 이해하기 쉽다고 생각합니다.

MCP는 전문가용 주방을 제공하여 도구, 재료, 장비에 접근할 수 있게 해줍니다. 반면 스킬은 가치 있는 것을 만들기 위한 단계별 지침을 제공하는 레시피입니다.

이 둘을 결합하면 사용자는 모든 단계를 직접 파악할 필요 없이 복잡한 작업을 수행할 수 있습니다. 제가 처음 MCP 서버 (server)를 구축했을 때, 도구에 대한 접근 권한만 제공하면 충분하다고 생각했지만, 실제로는 도구 사용 방법에 대한 워크플로 지침이 부족하여 사용자들을 혼란스럽게 했습니다.

스킬을 도입한 후 역할 분담이 명확해졌습니다. MCP는 '무엇을 할 수 있는지'를 정의하고, 스킬은 '어떻게 하는지'를 가르치면서 사용자 경험 (user experience)이 극적으로 향상되었습니다.

## 이 스킬 설치하기

Claude Code를 최고의 형태와 모든 최신 기능으로 설치하고 싶다면 잘 찾아오셨습니다. Visual Studio Code (VS Code)가 있는지 확인하세요. 없다면 설치하십시오. 이 글에서는 설치 방법을 다루지 않습니다.

VS Code를 열고 새 프로젝트를 시작하세요. 여기에서 VS Code 내부에 Claude Code를 설치해 봅시다. 확장 프로그램 (extensions)을 열고 검색창에 'Claude code'를 입력하세요. 확인 기호 (verification symbol)가 표시되는지 확인하고 확장 프로그램을 설치하십시오.

Claude Code를 설치한 후, 맨 위를 보면 작은 로고가 보일 것입니다. 그것을 클릭하세요.

스킬은 사실 플러그인 (plugin)의 한 형태입니다. 우리는 anthropic/skills 마켓플레이스 (marketplace)를 통해 플러그인 형태로 스킬을 설치하며, Claude는 필요할 때 자동으로 로드합니다.

**스킬 플러그인 마켓플레이스 추가**

또한 `/plugin`을 입력하여 플러그인 마켓플레이스를 추가할 수 있습니다. 그런 다음 공식 GitHub Skills 주소를 입력하세요:

`https://github.com/anthropics/skills`

**스킬 플러그인 설치**

마켓을 추가하면 스킬 플러그인을 설치하라는 메시지가 표시됩니다:

다음 명령어를 사용하여 스킬을 빠르게 설치할 수도 있습니다:

```
/plugin install document-skills@anthropic-agent-skills
/plugin install example-skills@anthropic-agent-skills
```

두 스킬 플러그인의 공식 용도는 다음과 같습니다:

*   **document-skills**: Excel, Word, PPT, PDF 등 문서를 처리할 수 있는 문서 스킬 패키지 (package).
*   **example-skills**: 스킬 생성, MCP 구축, 비주얼 디자인 (visual design), 알고리즘 아트 (algorithmic art), 웹 테스트 (web testing), Slack GIF 생성, 테마 스타일링 (theme styling) 등을 처리할 수 있는 샘플 스킬 세트 (set).

설치 성공. `/plugin` 명령 프롬프트 (command prompt)에 입력하고 '마켓플레이스 선택'을 통해 추가된 스킬 플러그인과 마켓플레이스를 확인할 수 있습니다.

`/plugin` 명령줄 (command line)의 '플러그인 관리'를 통해 스킬 플러그인에 접근하여 업데이트 및 삭제와 같은 작업을 수행할 수 있습니다:

설치 후, `/skill-creator`가 사용 가능한지 확인하기 위해 Claude Code에게 질문하겠습니다.

```
do u have the skill creator skill, and what does it do ?
```

여기서 보시다시피 우리는 스킬 크리에이터를 가지고 있습니다. 이제 플랜 모드 (plan mode)로 전환하여 새로운 스킬을 구축하도록 요청하겠습니다.

```
i want you to create a skill that helps me plan a complete one-month app launch
.I need it to break down the launch into manageable weekly chunks - first
two weeks for getting everything ready (finishing features, creating app
store materials, setting up marketing), third week for the actual launch
(testing with a small group first, reaching out to press, going live),
and the final week for monitoring how it’s doing and making quick fixes.
Include some templates I can actually use like launch checklists and social
media posts. The skill should activate whenever I mention things like "app launch plan" or
"launch my app in 30 days." It should work whether I'm launching an iPhone app,
```

주요 텍스트는 Claude가 모르는 내용만 포함해야 합니다.

스킬 크리에이터 가이드는 다음과 같이 설명합니다:

```
Default assumption: Claude is already very smart. Only
add context Claude doesn’t already have.
```

기본 전제는 Claude가 처음부터 똑똑하다는 것이므로, SKILL.md에 일반 지식이나 프로그래밍 기본 사항을 작성하는 것은 토큰만 낭비할 뿐입니다.

여러분은 Claude가 알지 못할 정보 (회사별 규칙, 특정 라이브러리 (libraries)의 특이점, 도메인별 워크플로 등)에 집중해야 합니다.

장황한 설명을 피하고 명령적이고 간결한 문체 (writing style)를 사용하는 것이 좋습니다.

**지침의 “자유도 (degrees of freedom)”를 작업에 맞추세요**

모든 것을 세부적으로 지정할 필요는 없습니다. 핵심은 작업에 맞게 지침의 세분성 (granularity)을 조정하는 것입니다.

*   **높은 자유도 (텍스트 기반 지침)**… 글쓰기와 같이 여러 접근 방식이 효과적일 때
*   **적당한 유연성 (가상 코드 (pseudocode) 또는 매개변수 (parameters)가 있는 스크립트)** — 권장되는 패턴이 있지만, 약간의 변형은 괜찮을 때
*   **낮은 유연성 (특정 스크립트, 적은 매개변수)** … 절차의 일관성이 중요하고 실수가 치명적일 때

## Claude Skills에는 어떤 유형이 있으며 어디서 찾을 수 있나요?

사용 측면에서는 두 가지 유형이 있습니다. Claude는 현재 공식 내장 스킬 (built-in Skill)과 로컬로 업로드된 스킬 (locally uploaded Skills) 사용을 지원합니다.

스킬의 출처를 기준으로 세 가지 유형으로 나눌 수 있습니다:

1.  **Anthropic 및 그 파트너가 제공하는 공식 스킬 (Official Skills).**  
    `https://github.com/anthropics/skills`  
    전체 크기로 이미지를 보려면 Enter 키를 누르거나 클릭하세요.  
    예를 들어, Claude.ai 웹 버전에서 사용하는 "웹 애플리케이션 (web application) 개발", "이 PDF 문서 분석", "Snake 게임 작성 및 미리보기"와 같은 매끄러운 기능 뒤에 있는 로직 코드 (logic code)는 모두 이 저장소 (repository)에 있습니다!

2.  **맞춤형 스킬 (Custom Skills).** 개인화된 사용자 지정 (customization)이 필요한 사용자에게 적합합니다. 스킬 크리에이터를 사용하여 스킬 파일을 생성하고 업로드합니다.

3.  **커뮤니티 스킬 (Community skills).** 다른 사용자가 공유한 스킬로, 쉽게 구할 수 있으며 처음부터 다시 만드는 것보다 훨씬 빠릅니다. 스킬 선택 및 수정에 이상적입니다. 다운로드하여 업로드하기만 하면 되지만, 사용 전 보안 위험 (security risks)을 인지하십시오.  
    `https://skillsmp.com/`  
    전체 크기로 이미지를 보려면 Enter 키를 누르거나 클릭하세요.  
    `https://www.aitmpl.com/skills`  
    전체 크기로 이미지를 보려면 Enter 키를 누르거나 클릭하세요.

## 어떤 작업이 스킬로 만들기에 적합한지 어떻게 판단하나요?

Claude에게 동일한 유형의 작업을 자주 요청하거나, 반복적으로 사용해야 하는 템플릿 (templates)이나 에셋 (assets)이 있는 경우 (예:):

*   “회사 템플릿을 사용하여 주간 보고서 작성을 도와주세요”: 매주 팀 주간 보고서를 작성해야 하며, 매번 Claude에게 “이번 주 성과, 직면한 어려움, 다음 단계”의 세 부분으로 내용을 정리하라고 지시해야 합니다. 이때 “팀 주간 보고서 생성기” 스킬을 만들 수 있습니다.
*   “우리 회사 스타일에 맞춰 프레젠테이션 (presentation)을 만들어주세요”: 로고 (logo) 사용, 브랜드 색상 (brand colors), 회사 이름, 회사 사업 내용, 전문성 기대치 등 브랜드 가이드라인 (brand guidelines)을 엄격하게 준수하는 것이 종종 필수적입니다. 이러한 가이드라인을 “브랜드 프레젠테이션 스타일” 스킬로 패키징할 수 있습니다.
*   “특정 형식에 맞춰 시장 분석 보고서 구성/경쟁사 조사 수행”: 예를 들어, 시장 분석 보고서를 작성하려면 세 가지 경쟁사 데이터 세트 (data set), 한 가지 내부 판매 데이터, 그리고 고정된 분석 프레임워크 (analytical framework)를 결합해야 할 수 있습니다. 이 전체 복잡한 프로세스를 “시장 분석 보고서” 스킬로 캡슐화 (encapsulate)할 수 있습니다.

반대로, 가끔 한 번 요청하는 단발성 요청이라면 채팅에 간단히 언급하면 되며, 스킬을 만들 필요는 없습니다.

## 결론

Claude Skills는 반복적이고 일상적인 작업을 자주 수행하는 사람들에게는 절대적으로 필수적인 기능입니다. 이는 여러분의 "불분명한 업무 경험"을 AI가 이해할 수 있는 "명시적인 규칙 (explicit rules)"으로 변환하여, Anthropic의 도구를 여러분의 필요에 완벽하게 맞출 수 있도록 합니다.

제품 책임자 (product owner), 프로젝트 관리자 (project manager), 카피라이터 (copywriter) 등 직장에서 Claude를 사용하는 누구든지 스킬을 통해 반복 작업을 줄이고 일관된 결과물을 보장할 수 있습니다. 이것이 바로 스킬의 핵심 가치입니다.
