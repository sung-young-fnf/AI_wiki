# 이 앱으로 Claude Code 에이전트 10개를 손쉽게 실행하는 방법

아직 잘 알려지지 않은 이 앱에 대해 이야기해보고자 합니다.

지난달 세션이 끝날 무렵, 저는 모든 것을 근본적으로 바꿀 만한 것을 발견했습니다. 바로 10개의 Claude Code 에이전트를 동시에 손쉽게 실행할 수 있게 된 것입니다.

제가 원하던 조직 방식으로 에이전트들을 관리했으며, 도움이 필요할 때마다 정확하게 알림을 받을 수 있었습니다. 그중 하나는 분할 창(split pane)에서 브라우저를 열고 탐색하며 테스트를 진행하기도 했습니다.

저는 무료 앱을 사용하고 있었는데, 놀랍게도 많은 개발자가 이 앱에 대해 알지 못하고 있습니다. 그 이름은 cmux이며, 현재 가장 저평가된 AI 개발 도구 중 하나라고 생각합니다.

## 모두가 느끼는 문제점

직장에서 새 노트북을 지급받아 깨끗한 환경에서 설정을 시작했습니다. 이제 가장 큰 문제 중 하나에 집중할 시간입니다.

모든 개발자는 Claude Code나 AI 코딩 환경을 최적으로 구축하여 구독료를 최대한 활용하고, 업무나 사이드 프로젝트에서 최고의 성과를 내고 싶어 합니다. 이를 위해서는 여러 인스턴스를 열고 AI 도구들이 각자의 작업을 수행하도록 해야 합니다.

그런데 30분쯤 지나면, 그중 하나가 다음 작업을 위해 도움을 요청하며 가만히 있었다는 것을 깨닫습니다. 다른 일에 몰두하느라 놓쳐버린 것이죠.

이는 마치 인턴을 교육할 때와 비슷합니다. 인턴이 막히면 화를 낼 수는 없습니다. 그저 "도움이 필요하면 언제든지 물어봐"라고 말하면, 그들은 "바쁘신 것 같아서 덜 중요한 일로 방해하고 싶지 않았어요"라고 답합니다.

AI 코딩의 속도는 놀랍지만, 그 관리는 형편없습니다. AI가 제 역할을 못해서가 아니라, 어떤 일이 일어나고 있는지 전혀 알 수 없기 때문에 시간을 낭비하게 됩니다.

cmux는 이 문제를 완전히 해결해 줍니다.

## cmux의 실제 모습

cmux는 Ghostty 기반으로 구축된 무료 Mac 네이티브 터미널(terminal) 앱으로, 여러 에이전트를 체계적으로 실행할 수 있도록 지원합니다. 탭, 분할 창(split panes), 워크스페이스(workspaces), 브라우저 지원 기능이 내장되어 있습니다.

진정한 마법은 단순한 레이아웃(그것은 tmux와 같을 뿐입니다)이 아닙니다. 바로 알림 시스템이 얼마나 훌륭하게 작동하는지에 있습니다.

에이전트가 여러분의 입력이 필요할 때, 단순히 데스크톱 알림만 오는 것이 아니라 사라지지 않는 파란색 표시등이 번쩍입니다. 어떤 세션이 여러분의 주의와 도움이 필요한지 항상 알 수 있으며, 어떤 세션이 계속 실행 중인지도 파악할 수 있습니다. 더 이상 여러 터미널 세션을 맹목적으로 확인할 필요가 없습니다.

작은 기능처럼 들릴 수 있지만, 결코 그렇지 않습니다. 모든 것을 더 나은 방향으로 변화시킵니다.

## 제가 가장 좋아하는 세 가지 기능

### 1: 알림 시스템

저는 많은 Claude 에이전트를 병렬로 실행해 왔습니다. 문제는 항상 똑같았습니다. 피드백이 불충분하거나 놓치는 일이 많았죠.

cmux는 쉬운 알림 시스템으로 이 문제를 해결합니다. 파란색 아이콘이 깜빡이고 일반적인 브라우저 알림도 제공됩니다.

처음으로 여러 에이전트를 병렬로 실행하면서 매우 차분함을 느꼈습니다.

### 2: 내장 브라우저

cmux에는 Claude 에이전트가 터미널(terminal) 내에서 직접 브라우저를 열고, 탐색하고, 제어할 수 있는 매우 훌륭한 내장 브라우저가 있습니다.

저는 Claude에게 웹사이트를 열고, 양식을 작성하고, 검색을 수행하도록 요청했는데 정말 놀라웠습니다. 심지어 개발자 도구(developer tools)도 내장되어 있습니다.

### 3: 원하는 방식으로 에이전트 실행

저는 Claude에게 자신을 두 개의 분할 창(split panes)으로 생성하도록 요청했습니다. 하나는 프로젝트를 이해하는 데 사용하고, 다른 하나는 코드를 분석하는 데 사용하도록 했습니다. 둘 다 작업이 완료되면 보고하고 스스로 종료하도록 설정했습니다.

## 몇 분이면 완료되는 설정

cmux는 무료입니다. Mac에서만 지원됩니다 (Windows 사용자분들께는 죄송하지만, 여러분은 터미널 문제로 더 많은 고생을 하셨으니 잠시 쉬실 자격이 있습니다).

설치 후:

*   프로젝트별로 워크스페이스(workspace)를 생성하세요 (색상 변경 가능).
*   폴더를 열고 워크스페이스 이름을 변경하세요.
*   Claude Code를 시작하세요.
*   `claude --dangerously-skip-permissions` 명령어를 실행하세요 (가상 머신에서도 강력히 권장됩니다).

이것으로 끝입니다. 엄청난 이점을 누리세요.

여러분에게 필요한 피드백을 놓치지 않고 받을 수 있습니다. 저는 cmux 없이는 살 수 없을 것 같습니다.

## 큰 그림

이것이 제가 계속 생각하게 만드는 부분입니다.

AI 개발의 병목 현상은 더 이상 AI 자체가 아니라, 우리의 인간적인 집중력과 관리 능력에 있습니다.

Claude는 저보다 10배 빠르게 작업할 수 있지만, 한 번에 하나의 터미널(terminal)만 주시한다면 (항상 그렇습니다. 가끔이 아니라), 여러분이 병목 현상이라는 느낌을 받기 시작할 것입니다.

cmux는 제가 사용해 본 도구 중 이 문제를 진정으로 해결하는 첫 번째 도구입니다. 압도적인 혼란을 우리가 실제로 관리할 수 있는 무언가로 정리해 줍니다.

## 시작하기

**5분 정도 시간이 있다면:** cmux를 다운로드하고 그 안에서 단일 Claude 세션을 시작해 보세요. 인터페이스만 테스트해 보세요.

**15분 정도 시간이 있다면:** 두 가지 다른 작업을 위한 두 개의 공간을 설정하세요. 병렬로 실행하고 알림을 기다려 보세요.

**30분 정도 시간이 있다면:** cmux 개발자의 저장소에서 cmux 브라우저 기술을 사용하고 Claude에게 브라우저를 사용하여 자신의 작업을 검증하도록 명확한 지침을 주세요. 어떤 일이 일어나는지 지켜보세요.

알림 시스템만으로도 충분한 가치가 있습니다. 다른 모든 것은 추가적인 이점입니다.

저는 cmux와 아무런 관련이 없으며, 모든 생각은 저의 개인적인 의견입니다.
