# Opencode를 연구 보조 도구로 활용하기

Opencode는 Claude 코드 및 다른 코딩 에이전트 (agent) 솔루션의 훌륭한 오픈소스 대안입니다. 하지만 Opencode가 유일하게 사용하는 LLM (Large Language Model) 애플리케이션이라면, 연구 및 정보 추적과 같이 AI 솔루션에서 기대할 수 있는 모든 것을 포괄하지는 못합니다. 그러나 간단한 추가 에이전트 (agent)를 통해 이 모든 기능을 터미널 (terminal)에서 바로 사용할 수 있습니다.

이 글은 ChatGPT, Claude 또는 유사 서비스 구독자에게는 크게 흥미롭지 않을 수 있습니다. 하지만 Opencode 내에서 LLM을 99% 코딩 도구로 사용하는 사람들에게는 이 아이디어가 유용할 것입니다.

## 문제는 무엇인가요?

Opencode는 훌륭합니다. 내장된 도구와 에이전트 (agent)는 코딩 에이전트로서의 기능을 탁월하게 만듭니다. 하지만 아이디어를 연구하고 싶다면 몇 가지 어려움이 있습니다. 내장된 두 에이전트 모두 개방형 연구 질문에는 적합하지 않습니다. 빌드 에이전트 (build agent)는 빌드에 너무 집중할 가능성이 높고, 플랜 에이전트 (plan agent)는 주로 문제에 대한 구현 계획을 세우려고 할 것입니다.

이는 각 에이전트의 주요 목적이므로 타당합니다. 하지만 어려운 문제에 대한 올바른 해결책을 찾는 데 도움을 줄 도구를 원한다면, 다른 유형의 대화 방식이 필요합니다.

두 번째 문제는 이전에 나눴던 대화로 다시 돌아갈 수 없다는 것입니다. 물론 이전 세션 (session)을 찾으려고 시도할 수는 있지만, Opencode는 기본적으로 이전 논의 내용을 기억하지 않습니다 (이는 주요 목적에는 좋은 점입니다).

## 간단한 해결책

Opencode는 훌륭한 연구 보조 도구가 필요로 하는 모든 도구를 이미 내장하고 있습니다. 웹 검색 (websearch) 및 다른 도구들은 이러한 관점에서 필요한 모든 것을 제공합니다. 기억력과 도구가 올바르게 대화하는 방식을 추가하려면 다음 설정을 통해 가능합니다.

1.  앞으로 연구 주제에 Opencode를 사용할 새 폴더를 생성합니다.
2.  이 폴더 내의 `.opencode/agents` 폴더에 유용한 연구 보조 도구 역할을 수행하도록 지시하는 에이전트 (agent)를 추가합니다. 이 에이전트는 논의하는 모든 주제에 대한 요약을 `.md` 파일에 저장하고, 이를 `Agent.md` 파일 내에서 추적합니다.
3.  메인 폴더 안에 `knowledge.md` 파일들에 대한 링크 모음 역할을 하는 `Agent.md` 파일을 추가합니다.

이 간단한 설정으로 Opencode를 연구 보조 도구로 더욱 효과적으로 사용하는 데 필요한 모든 것을 갖추게 됩니다.

## 참고 자료: `research_assistant.md` 및 `Agent.md` 파일

다음은 제가 사용하는 `research_assistant.md` 에이전트 파일과 `Agent.md` 파일입니다.

### `research_assistant.md`

```markdown
---
description: 대화형 지식 기반 에이전트 (agent)로, 노트, 결정 및 맥락이 포함된 주제 파일을 생성하고 유지 관리합니다.
mode: primary
tools:
  websearch: true
---

당신은 대화형 작업 공간 (workspace) 에이전트입니다. 이 작업 공간은 전통적인 코드베이스 (codebase)가 아니라, 대화를 통해 구축된 영구적인 지식 기반 (knowledge base) 역할을 합니다.

## 역할

- 특정 주제, 해결책 또는 프로젝트를 논의할 때, 작업 공간 (workspace) 루트에 상세한 노트, 결정 및 맥락을 담은 전용 `.md` 파일을 생성합니다.
- 각 주제 파일을 명확한 제목, 요약 및 실행 가능한 세부 정보로 잘 구성합니다.
- 대화가 진행됨에 따라 기존 주제 파일을 업데이트하여, 기록을 보존하고 새로운 통찰력을 추가합니다.
- `Agent.md` 파일을 중앙 인덱스 (index)로 유지합니다. 주제 파일을 생성하거나 중요하게 업데이트할 때마다 `Agent.md` 파일의 프로젝트/주제 테이블 (table)에 링크와 간략한 요약을 함께 업데이트합니다.

## 웹 연구

- 주제가 최신 정보, 시사 문제, 기술 연구 또는 훈련 데이터 (training data)를 넘어서는 외부 맥락을 필요로 할 때, `websearch` 도구를 사용하여 관련 정보를 찾습니다.
- 웹 연구 결과를 요약하고 관련 주제 파일에 통합하며, 적절한 경우 출처를 인용합니다.
- 검색을 통해 발견되거나 사용자가 제공한 특정 URL에서 전체 콘텐츠를 가져오려면 `webfetch`를 사용합니다.

## 지침

- 명확하고 간결한 언어로 작성합니다.
- 여러 세션에 걸쳐 향후 참고 자료로 유용하도록 노트를 구성합니다.
- 한 주제가 여러 대화에 걸쳐 있다면, 중복 파일을 생성하는 대신 기존 파일에 추가합니다.
- 주제 파일이 이미 존재한다면, 변경하기 전에 먼저 읽어서 연속성을 유지합니다.
- 논의된 내용에 대한 단일 진실 공급원 (single source of truth)으로 `Agent.md`의 프로젝트/주제 테이블을 사용합니다.
```

### `Agent.md`

```markdown
# 에이전트 작업 공간 (Agent Workspace)

이 디렉터리 (directory)는 대화형 에이전트 (conversational agent)로서 Opencode와 상호 작용하기 위한 작업 공간 (workspace)입니다. 전통적인 코드베이스 (codebase)가 아니라, 대화를 통해 구축된 영구적인 지식 기반 (knowledge base) 역할을 합니다.

## 작동 방식

- 특정 주제, 해결책 또는 프로젝트를 논의할 때, 저는 상세한 노트, 결정 및 맥락을 담은 전용 `.md` 파일을 생성합니다.
- 각 파일은 아래 표에 간략한 요약과 함께 링크됩니다.
- 파일은 대화가 진행됨에 따라 생성 및 업데이트되어, 여러 세션에 걸쳐 맥락을 계속 이용할 수 있도록 합니다.

## 프로젝트 / 주제

| 파일 | 요약 |
|------|---------|
| [shadcn-ui-evaluation.md](shadcn-ui-evaluation.md) | shadcn/ui 평가: Tailwind 프로젝트와의 적합성, 복사-붙여넣기 모델의 장단점, 라이선스 (licensing) 세부 정보 (출처 포함). |
| [ai-agent-platform-research.md](ai-agent-platform-research.md) | Claude Cowork 대안 구축을 위한 연구: 이름 충돌 ("OpenWork"는 이미 사용 중), 기술 스택 (tech stack) 옵션 (Python ✅), 에이전트 (agent) 프레임워크 (LangGraph vs CrewAI), 권장 아키텍처 (architecture). |
```

이 설정이 완벽할까요? 물론 아닐 것입니다. 하지만 Opencode가 제공하는 기회와 그 역량을 보여주는 훌륭한 예시라고 생각합니다.
