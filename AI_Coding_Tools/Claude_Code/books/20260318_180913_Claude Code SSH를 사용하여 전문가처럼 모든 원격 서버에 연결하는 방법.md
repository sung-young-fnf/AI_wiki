# Claude Code SSH를 사용하여 전문가처럼 모든 원격 서버에 연결하는 방법

최근 Claude Code 데스크톱 앱에 SSH 지원이 추가되었지만, 많은 독자분들이 이것이 무엇을 의미하는지, 어떤 이점을 제공하는지 알지 못한다는 것을 깨달았습니다.

만약 여러분이 진지한 Claude Code 사용자인데 Claude Code SSH의 작동 방식을 이해하지 못한다면, 많은 기회를 놓치고 있는 것입니다. 이 글에서 그 이유를 보여드리겠습니다.

대부분의 개발자는 로컬 머신이 아닌 다른 곳, 즉 클라우드 서버(cloud server), 원격 개발 박스(remote dev box) 또는 스테이징 환경(staging environment)에 코드를 저장합니다.

SSH 지원이 있기 전에는 모든 것이 올바르게 설치되고 구성된 로컬 머신에서 작업해야 했습니다.

Claude Code SSH는 이제 Claude Code를 모든 원격 서버에 직접 연결하여, 코드가 위치한 실제 환경에서 작업할 수 있도록 해줍니다.

복잡하게 들릴 수 있지만, 이 글에서 단계별로 시연할 것이므로 그렇지 않습니다.

Claude Code SSH가 작동하는 두 가지 방식을 보여드리고, 설정 과정을 안내하며, 실제 원격 서버에서 실시간으로 시연하여 실제로 어떻게 보이는지 확인하실 수 있도록 하겠습니다.

이 글이 처음이시라면, 여기에서 제 다른 Claude Code 튜토리얼을 확인하고, 향후 튜토리얼을 놓치지 않도록 Medium에서 저를 팔로우하는 것을 잊지 마세요.

이제 Claude Code SSH 설정을 시작하겠습니다.

## Claude Code SSH

SSH를 생각하면 복잡하게 느끼기 쉽지만, 가장 좋은 이해 방법은 Claude Code 구성을 건드리기 전에 테스트 서버나 로컬 머신에 먼저 설정하여 직접 실험해보는 것입니다.

시작하기 좋은 지점은 기본 사항입니다: SSH가 무엇을 하는지, 그리고 왜 필요한지.

### Claude Code SSH란 무엇인가요?

**SSH (Secure Shell)**는 로컬 터미널에서 원격 머신에 안전하게 연결하고 명령을 실행할 수 있도록 하는 프로토콜입니다.

개발자들은 매일 이를 사용하여 클라우드 서버, VPS (Virtual Private Server) 인스턴스, 원격 개발 환경에 접속합니다.

Claude Code SSH는 이러한 개념을 한 단계 더 발전시킵니다.

사용자가 서버에 SSH로 접속하여 수동으로 명령을 실행하는 대신, Claude Code가 원격 머신에 직접 연결하여 파일 읽기, 코드 실행, 프로젝트가 있는 실제 환경에서의 변경 작업을 수행합니다.

### Claude Code SSH가 해결하는 문제

대부분의 개발자에게 익숙한 시나리오가 있습니다.

운영 중인 앱이 Linux 서버에서 실행됩니다. 여러분의 로컬 머신은 Windows 또는 Mac입니다. 올바른 Node 버전, 환경 변수, 그리고 필요한 의존성(dependencies)을 서버에 설정하는 데 몇 시간을 보냈습니다.

그 후 로컬에서 Claude Code를 실행하면, 로컬 환경에서 작동하기 때문에 그러한 컨텍스트(context)에 대해 전혀 알지 못합니다.

이것이 Claude Code SSH가 해결하는 문제입니다.

Claude Code가 SSH를 통해 원격 서버에 연결되면, 해당 환경 내에서 작동합니다. 올바른 파일, 의존성(dependencies) 및 구성을 볼 수 있습니다.

### Claude Code SSH의 두 가지 작동 방식

Claude Code는 워크플로우에 따라 두 가지 고유한 접근 방식을 제공합니다:

1.  **데스크톱 앱 SSH 연결 (Desktop App SSH Connection):** 이는 GUI (그래픽 사용자 인터페이스) 방식입니다. Claude Code 데스크톱 앱 내에서 원격 서버를 SSH 연결로 추가합니다. 연결되면, 시각적 차이 비교(visual diffs), 권한 관리(permission management), 파일 탐색(file browsing) 등 완전한 데스크톱 경험을 얻을 수 있지만, 모든 작업은 원격 머신에서 실행됩니다.

2.  **SSH를 통한 CLI (CLI Over SSH):** 이는 터미널 방식입니다. 원격 서버에 SSH로 접속한 다음, 해당 SSH 세션 내에서 `claude` 명령을 실행합니다. Claude Code는 사용자의 터미널을 인터페이스로 사용하여 원격 머신에서 실행됩니다.

두 가지 접근 방식 모두 동일한 핵심 이점을 제공합니다. 즉, Claude Code가 원격 환경 내에서 작동한다는 점입니다. 차이점은 단지 접근 방식에 있습니다.

이제 Claude Code를 도입하기 전에 어떤 일이 일어나는지 이해하실 수 있도록 깨끗한 서버부터 시작하여 SSH 설정 과정을 단계별로 안내해 드리겠습니다.

## SSH 접근 설정

Claude Code를 원격 서버에 연결하기 전에, SSH가 어떻게 작동하는지 그리고 어떻게 설정하는지 이해해야 합니다.

이 데모에서는 WSL (Windows Subsystem for Linux)을 통해 Windows 머신에 설치된 Ubuntu 서버를 사용합니다.

이는 AWS, DigitalOcean 또는 다른 클라우드 제공업체에서 실행하는 모든 원격 서버와 동작이 동일하며, SSH 설정 단계도 유사합니다.

유일한 차이점은 제 "서버"가 클라우드가 아닌 로컬 머신에 있다는 것입니다. 여기에서 보시는 모든 내용은 실제 원격 서버에도 적용됩니다.

### 1단계 — Ubuntu 서버 실행

WSL에서 Windows 검색을 통해 Ubuntu 앱을 열면, `root`로 로그인된 터미널로 들어갑니다:

```bash
root@DESKTOP-K96H7KV:~#
```

실제 클라우드 서버에서는 이는 제공업체가 새 인스턴스를 시작한 후 터미널 접근 권한을 주는 것과 동일합니다. 일반적으로 제공된 키를 사용하여 SSH로 접속합니다.

### 2단계 — OpenSSH 서버 설치

어떤 서버에서든 가장 먼저 필요한 것은 OpenSSH 서버 패키지입니다. 이것이 다른 머신이 SSH를 통해 서버에 연결할 수 있도록 해줍니다.

실행:

```bash
sudo apt update
sudo apt install openssh-server -y
```

Ubuntu가 패키지를 가져와 설치하는 것을 볼 수 있습니다. 완료되면 SSH 서비스를 시작합니다:

```bash
sudo service ssh start
```

**AWS 또는 DigitalOcean:** 대부분의 클라우드 제공업체는 Ubuntu 이미지에 OpenSSH를 사전 설치하므로, 이 단계를 건너뛸 수도 있습니다. 하지만 확인하는 것이 좋습니다.

### 3단계 — 서버 IP 주소 가져오기

모든 서버는 IP 주소를 가지고 있습니다. 이것이 다른 머신이 서버를 찾아 연결하는 방법입니다.

실행:

```bash
ip addr show eth0 | grep "inet "
```

다음과 같은 출력을 볼 수 있습니다:

```
inet 172.29.5.190/20 brd 172.29.15.255 scope global eth0
```

제 서버 IP는 `172.29.5.190`입니다. 여러분의 IP 주소를 적어두세요. 모든 SSH 연결에 필요할 것입니다.

**AWS:** 여러분의 IP는 EC2 대시보드에서 찾을 수 있는 `54.x.x.x`와 같은 공용 IP가 될 것입니다. **DigitalOcean:** 인스턴스가 생성되는 즉시 Droplet 대시보드에 표시됩니다.

### 4단계 — 암호 로그인 허용을 위한 SSH 구성

기본적으로 Ubuntu는 보안상의 이유로 SSH를 통한 `root` 로그인(root login)을 비활성화합니다. 이 데모에서는 암호로 연결할 수 있도록 이를 활성화할 것입니다.

SSH 구성을 업데이트하려면 다음 두 명령을 실행하십시오:

```bash
sed -i 's/#PermitRootLogin prohibit-password/PermitRootLogin yes/' /etc/ssh/sshd_config
sed -i 's/#PasswordAuthentication yes/PasswordAuthentication yes/' /etc/ssh/sshd_config
```

그 다음 `root` 암호를 설정하십시오:

```bash
passwd
```

새 암호를 입력하고 확인하십시오.

변경 사항을 적용하려면 SSH를 다시 시작하십시오:

```bash
service ssh restart
```

**운영 환경 참고:** 실제 서버에서는 암호 대신 SSH 키 인증(SSH key authentication)을 사용하는 것이 더 안전합니다. 하지만 이 데모에서는 암호 로그인(password login)을 사용하여 과정을 간단하고 쉽게 따라 할 수 있도록 했습니다.

### 5단계 — Windows에서 서버에 SSH 접속

이제 별도의 Windows 터미널 또는 PowerShell 창을 열고 다음을 실행하십시오:

```bash
ssh root@172.29.5.190
```

IP 주소를 여러분의 것으로 교체하십시오. 프롬프트가 나타나면 `yes`를 입력하여 연결을 확인한 다음 암호를 입력하십시오.

성공적인 연결은 다음과 같습니다:

```
Welcome to Ubuntu 24.04.1 LTS (GNU/Linux 6.6.87.2-microsoft-standard-WSL2 x86_64)
root@DESKTOP-K96H7KV:~#
```

Windows 터미널에 이 프롬프트가 나타났다는 것은 이제 원격 머신 내부에 있다는 것을 의미합니다. 여기서 실행하는 모든 명령은 로컬 Windows 머신이 아닌 서버에서 실행됩니다.

이는 AWS EC2 인스턴스 또는 DigitalOcean Droplet에 SSH로 접속할 때 발생하는 상황과 유사합니다.

이제 SSH가 설정되고 작동합니다. 이 서버에 Claude Code를 연결해 보겠습니다. 먼저 데스크톱 앱을 통해, 그 다음 CLI를 통해 연결하겠습니다.

## Claude Code SSH 테스트

먼저 원격 서버에 Claude Code를 설치해야 합니다.

이 경우, 우리는 원격 서버 역할을 하는 WSL에 Claude Code를 설치할 것입니다.

WSL Ubuntu 터미널에서 다음을 실행하십시오:

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

이것은 WSL "서버"에 Claude Code CLI (명령줄 인터페이스)를 설치합니다. 완료되면 작동하는지 확인하십시오:

```bash
claude --version
```

이제 원격 머신 내부에 있습니다. 여기서 실행하는 모든 명령은 서버에서 실행됩니다.

### Claude Code 실행

#### 1단계 — 서버에 SSH 접속

Windows 터미널 또는 PowerShell에서 서버에 연결하십시오:

```bash
ssh root@172.29.5.190 // 여러분의 서버 IP로 교체하세요
```

프롬프트가 나타나면 암호를 입력하십시오. 다음과 같은 서버 프롬프트가 나타날 것입니다:

```bash
root@DESKTOP-K96H7KV:~#
```

이제 원격 머신 내부에 있습니다. 여기서 실행하는 모든 명령은 로컬 Windows 머신이 아닌 서버에서 실행됩니다.

#### 2단계 — Claude Code 실행

이제 간단히 다음을 실행하십시오:

```bash
claude
```

Claude Code가 SSH 세션 내에서 실행됩니다.

이는 전적으로 원격 서버에서 실행되며, 서버의 파일을 읽고, 서버의 환경을 사용하며, 거기서 명령을 실행합니다.

#### 3단계 — 작업 부여

올바른 환경에서 작동하는지 확인하기 위해 간단한 작업을 시도해 보세요:


`/root` 디렉터리와 모든 설정 파일은 Ubuntu 서버에 속하며, Claude Code가 이를 읽고 있습니다.

목록에서 보이는 `.claude/` 디렉터리는 서버에 있는 Claude Code의 구성입니다. 이는 Claude Code가 원격 머신에 설치되어 작동 중임을 확인시켜 줍니다.

Claude Code가 원격 서버에 파일을 생성할 수 있는지 테스트해 봅시다.

Claude Code에게 서버에 파일을 생성하도록 요청했고, 원격 서버의 `/root`에 `hello.txt` 파일을 생성했습니다.

그 다음 디렉터리를 나열하고 파일을 다시 읽어보라고 요청했고, 그 결과 "Claude Code SSH is working!"이 나타났습니다.

Claude Code는 해당 파일을 서버에 생성했고, 그 파일은 서버에 존재합니다.

만약 별도의 터미널에서 동일한 서버에 SSH로 접속하면, 그 파일이 거기 그대로 있습니다.

이것이 Claude Code를 로컬에서 실행하는 것과 SSH를 통해 실행하는 것의 차이점입니다. 이제 원격 서버 내에서 직접 작업할 수 있습니다.

### `tmux`로 세션 유지하기

SSH를 통한 CLI (명령줄 인터페이스) 사용 시 한 가지 실질적인 문제는, 연결이 끊어지면 Claude 세션도 함께 종료된다는 점입니다.

해결책은 `tmux`입니다. `tmux`는 연결이 끊어져도 세션을 계속 실행되게 하는 터미널 멀티플렉서(terminal multiplexer)입니다.

서버에 설치하십시오:

```bash
apt install tmux -y
```

Claude Code를 실행하기 전에 이름이 지정된 `tmux` 세션을 시작하십시오:

```bash
tmux new -s claude-session
```

이제 그 안에서 Claude Code를 실행하십시오:

```bash
claude
```

연결을 끊어야 할 때, 세션을 종료하지 않고 `tmux`에서 분리하십시오:

```
Ctrl+B then D
```

Claude 세션은 서버에서 계속 실행됩니다. 나중에 다시 연결할 때, 다음 명령으로 재접속하십시오:

```bash
tmux attach -t claude-session
```

모든 것이 여러분이 떠난 그 자리 그대로 있습니다.

이것은 Claude Code를 장시간 작업에 사용할 때 특히 유용합니다. 터미널을 닫고 다른 일을 하다가 다시 돌아와도 실행 중인 세션을 이어서 작업할 수 있습니다.

## 데스크톱 앱 SSH 연결

데스크톱 앱 방식은 Claude Code가 원격 서버에서 작동하는 동안 완전한 GUI (그래픽 사용자 인터페이스) 경험을 제공합니다.

이는 로컬에서 작업하는 것과 같은 느낌을 주기 때문에 대부분의 사용자가 선호할 접근 방식입니다.

### 사전 준비 사항

연결하기 전에 원격 서버에 Claude Code CLI (명령줄 인터페이스)가 설치되어 있는지 확인하십시오.

우리는 이미 이 작업을 수행했습니다. 만약 새로운 클라우드 서버에 설정하는 경우, 먼저 다음을 실행하십시오:

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

### 1단계 — 환경 드롭다운 열기

Claude Code 데스크톱 앱에서 인터페이스 상단의 환경 선택기(environment selector)를 찾으세요. 클릭하여 드롭다운 메뉴를 엽니다.

### 2단계 — SSH 연결 추가

드롭다운 메뉴에서 `+ Add SSH Connection`을 클릭하십시오.

### 3단계 — 연결 세부 정보 입력

다음 정보를 요청하는 양식이 나타날 것입니다:

*   **이름 (Name)** — `WSL Ubuntu Demo`와 같이 친숙한 이름을 지정하십시오.
*   **SSH 호스트 (SSH Host)** — `root@172.29.5.190` (여러분의 사용자 이름@서버-IP)를 입력하십시오.
*   **포트 (Port)** — 22 (기본값)로 두십시오.
*   **ID 파일 (Identity File)** — 키 인증을 사용하는 경우 개인 SSH 키(private SSH key)의 경로를 입력하고, 암호 인증의 경우 비워두십시오.

`Connect`를 클릭하십시오.

### 4단계 — 인증 및 연결

프롬프트가 나타나면 서버 암호를 입력하십시오.

데스크톱 앱은 SSH 연결을 설정하고 환경을 원격 서버로 전환할 것입니다.

연결되면 환경 표시기(environment indicator)가 원격 서버 이름으로 변경되는 것을 확인하십시오.

이제 시각적 차이 비교(visual diffs), 권한 프롬프트(permission prompts), 파일 탐색(file browsing) 등 완전한 Claude Code 데스크톱 인터페이스를 사용하고 있지만, 모든 작업은 원격 머신에서 실행됩니다.

## 최종 생각

Claude Code SSH는 사용해보기 전까지는 매우 기술적으로 들리는 기능 중 하나입니다.

Claude Code를 실제 서버에 연결하여 코드가 있는 실제 환경에서 파일을 읽고, 명령을 실행하고, 변경 사항을 적용할 수 있는 능력은 훌륭한 워크플로우 업그레이드입니다.

이 글에서 테스트한 모든 내용을 바탕으로, 여러분이 얻어가야 할 내용은 다음과 같습니다:

*   **SSH를 통한 CLI 방식 (CLI over SSH method)은 완벽하게 작동합니다.** 서버에 SSH로 접속하고 `claude`를 실행하면 모든 준비가 완료됩니다. 이는 운영 체제(operating system)에 관계없이 가장 신뢰할 수 있는 접근 방식입니다.
*   **`tmux`는 장시간 실행되는 세션에 가장 좋은 선택입니다.** 원격 서버에서 Claude Code를 시작하기 전에 항상 `tmux` 세션을 시작하여 연결이 끊어져도 작업이 유지되도록 하십시오.
*   **데스크톱 앱 SSH 연결은 Windows에서 버그가 있습니다.** Windows 환경에 없는 `/usr/bin/ssh`를 찾습니다. Mac 및 Linux 사용자는 문제없이 사용할 수 있습니다. Windows 사용자라면 현재로서는 CLI 방식이 최선의 방법입니다.

클라우드 서버, VPS 인스턴스 또는 모든 원격 환경에서 작업하는 개발자에게는 이 설정이 필요합니다. 서버를 실행하고, Claude Code CLI를 설치하고, SSH로 접속하여 Claude Code가 코드가 있는 곳에서 작업하도록 하세요.
