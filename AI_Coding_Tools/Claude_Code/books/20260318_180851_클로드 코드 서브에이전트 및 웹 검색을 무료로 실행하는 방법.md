# 클로드 코드 서브에이전트 및 웹 검색을 무료로 실행하는 방법

최근에는 비용(At No Cost)을 들이지 않고 Claude Code(클로드 코드)를 스마트하게 사용하는 다양한 방법이 있습니다.

네, 토큰 소모를 멈출 수 있지만, 먼저 이 방법이 자신에게 맞는지 선택해야 합니다. 이 부분은 나중에 이야기하겠습니다.

유료 Medium(미디엄) 회원이 아니라면, 여기에서 전체 글을 무료로 읽을 수 있습니다. 하지만 제 작업을 지원하기 위해 Medium(미디엄) 가입을 고려해주시면 감사하겠습니다!

최근 저는 Ollama(올라마)를 사용하여 Claude Code(클로드 코드)를 실행하는 방법에 대한 자세한 가이드를 공유했습니다. 혹시 놓치셨다면, 현재까지의 과정을 처음부터 안내해 줄 게시물들은 다음과 같습니다.

*   (신규) Ollama(올라마)의 Claude Code(클로드 코드), Codex(코덱스), OpenCode(오픈코드) 실행 기능을 테스트하다 (더 이상 설정 불필요)
*   새로운 Claude Code(클로드 코드) Ollama(올라마) 워크플로우를 사용해보다 (놀랍고 자유로움)
*   Claude Code(클로드 코드) + MiniMax M2.5(미니맥스 M2.5)를 테스트하다 (원샷 CLI 도구 빌드로 정말 놀랐습니다)

이 튜토리얼들이 이 설정이 어떻게 작동하는지 빠르게 이해하는 데 가장 좋은 자료이지만, 이제 우리는 새로운 방식으로 한 단계 더 나아가고자 합니다.

Claude Code(클로드 코드)를 Ollama(올라마)에서 실행하는 것은 돈을 절약하거나, 적어도 Anthropic(앤트로픽) 모델에 장애가 발생한 날에도 계속 작업할 수 없다면 (이 글을 쓰는 시점처럼) 그다지 흥미롭지 않을 것입니다.

만약 이 점이 여러분을 흥분시킨다면, 프로토타입(prototype) 작업에 매우 유용한 서브에이전트(subagent)와 웹 검색(web search) 에이전트를 실행하여 더 나은 경험을 제공할 수 있습니다.

핵심 아이디어는 덜 민감한 작업을 다른 모델로 구축하고 토큰을 절약하는 것입니다.

현재로서는 Ollama(올라마)를 사용하여 원하는 모델을 실행하는 것보다 더 좋은 방법은 없습니다. 이제 이 방법은 마법처럼 작동하며 서브에이전트(subagent)와 웹 검색(web search)을 실행합니다.

간략히 설명하자면, 비용 없이 즉석에서 여러 에이전트(agent)를 구동하여 프로젝트를 구축하고, 프런트엔드(frontend)와 백엔드(backend)를 만들고, 심지어 API(응용 프로그래밍 인터페이스) 문서까지 작성하는 것을 상상해 보세요.

기억하신다면, 이전 테스트는 실망스러웠습니다. 이제 다시 실행하여 무엇이 가능한지 봅시다.

## 시작하기

한 줄 명령어로 실행:

`ollama launch claude --model minimax-m2.5:cloud`

Ollama(올라마) 클라우드의 모든 모델에서 작동합니다.

이 실행이 얼마나 빠른지 보세요:

저는 최신 Claude Code(클로드 코드) 버전을 사용하고 있으며, `minimax-m2.5:cloud` 모델을 실행 중인지 확인하기 위해 모델을 점검할 수 있습니다.

## 서브에이전트 (Subagents)

서브에이전트(subagent)는 파일 검색, 코드 탐색, 연구 등 여러 작업을 각자의 컨텍스트(context)에서 병렬로 실행할 수 있습니다.

긴 코딩 세션이 더욱 생산적으로 변합니다.

일부 모델(minimax-m2.5, glm-5, kimi-k2.5)은 필요할 때 서브에이전트를 트리거(trigger)하지만, 모델에게 “use/spawn/create subagents(서브에이전트 사용/생성/생성)”라고 지시하여 서브에이전트 트리거를 강제할 수도 있습니다.

다음은 실행할 수 있는 프롬프트(prompt) 예시입니다:

```
인증 흐름(auth flow), 결제 통합(payment integration),
 그리고 알림 시스템을 탐색하도록 서브에이전트를 생성하세요.
보안 문제 감사, 성능 병목 현상(performance bottlenecks) 파악,
그리고 접근성(accessibility)을 병렬로 서브에이전트(subagent)로 확인하세요.
데이터베이스 쿼리(database query) 매핑, API(응용 프로그래밍 인터페이스) 경로 추적,
그리고 오류 처리 패턴(error handling patterns)을 카탈로그화하도록 서브에이전트를 생성하세요.
```

처음부터 시작하여 어떻게 작동하는지 봅시다.

### 서브에이전트 테스트: 지출 추적기 구축

실제 구축 작업에 얼마나 잘 작동하는지 보고 싶었습니다.

Claude Code(클로드 코드)가 실행되었습니다. 세 개의 서브에이전트(subagent)가 즉시 실행되었습니다:

*   프런트엔드 UI 전문가 (frontend-ui-expert) — HTML, CSS, 바닐라 JS(vanilla JS) 처리 (11.4k 토큰)
*   백엔드 에이전트 (Backend agent) — FastAPI(패스트API) 경로 및 모델 설정 (11.0k 토큰)
*   프로젝트 구조 에이전트 (Project structure agent) — 파일 및 README(읽기 파일) 정리 (10.8k 토큰)

이제 진지한 프로젝트를 구축하는 것이 아니라 단순히 실험만 한다고 해도 얼마나 많은 토큰을 소모할 수 있는지 상상해 보세요.

저희 에이전트들이 작업 중이며, 프로젝트가 빠르게 형태를 갖춰가고 있습니다:

그리고 매우 빠르게, 이 테스트 프로젝트가 완료되었습니다.

각 에이전트는 독립적으로 작업했습니다.

*   프런트엔드 에이전트는 전체 양식, 카테고리 필터, 지출 목록 UI를 포함하는 `index.html`을 생성하기 시작했습니다.
*   다른 두 에이전트는 각자의 작업을 처리했습니다.

이것이 Ollama(올라마)에서 서브에이전트(subagent)를 실행하는 힘입니다. Claude API(클로드 API) 토큰 비용을 지불하지 않고도 병렬 작업 실행을 할 수 있습니다.

### 서브에이전트(Subagents)로 API 테스트

이제 앱이 실행 중이므로, 다음 서브에이전트(subagent)를 구동하여 API(응용 프로그래밍 인터페이스)를 테스트할 수 있습니다.

```
FastAPI(패스트API) 백엔드(backend)를 테스트하도록 서브에이전트를 생성하세요. 한 서브에이전트는
API(응용 프로그래밍 인터페이스) 경로가 올바른지 확인하고, 다른 서브에이전트는 데이터베이스 모델을 테스트하며,
또 다른 서브에이전트는 엔드포인트(endpoint)가 작동하는지 확인하는 간단한 테스트 스크립트를 생성해야 합니다.
```

세 에이전트가 3분 이내에 작업을 완료했습니다:

*   API 경로 검사기 (API routes checker) — 4회 도구 사용, 10.5k 토큰
*   데이터베이스 모델 테스터 (Database models tester) — 6회 도구 사용, 10.5k 토큰
*   테스트 스크립트 생성기 (Test script creator) — 8회 도구 사용, 13.6k 토큰

이보다 더 좋을 수는 없습니다:

테스트 결과는 인상적이었고, 서브에이전트(subagent)들이 문제점을 발견했습니다:

**API 경로 문제:**

*   경로 순서 버그 (CRITICAL) — `/expenses/summary`는 `/expenses/{id}`보다 먼저 와야 합니다.
*   CORS(크로스 오리진 리소스 공유) 보안 취약점 (CRITICAL) — `allow_credentials=True`와 와일드카드(wildcard) 오리진(origin)의 조합.
*   `DELETE`에 `response_model` 누락 (Minor)

**데이터베이스 모델 문제:**

*   사용 중단된 `datetime.utcnow` (CRITICAL) — `datetime.now(timezone.utc)`를 사용해야 합니다.
*   금액에 `Float` 사용 (CRITICAL) — `Numeric(10,2)`를 사용해야 합니다.
*   문자열 길이 제한 없음 (Warning)
*   입력 유효성 검사 없음 (Warning)

세 번째 에이전트는 모든 CRUD(생성, 읽기, 업데이트, 삭제) 작업을 다루는 완전한 테스트 스크립트를 `backend/test_endpoints.py`에 생성했습니다.

이제 이 서브에이전트(subagent)들이 작동하므로, 다음 단계로 나아가기 위해 웹 검색(web search)을 추가하는 것은 어떨까요?

## 웹 검색 (Web Search)

Ollama(올라마)의 웹 검색(web search)은 Anthropic(앤트로픽) 호환성 레이어(compatibility layer)에 내장되어 있습니다.

모델이 최신 정보를 필요로 할 때, Ollama(올라마)는 추가 설정 없이 검색을 처리하고 결과를 반환합니다.

서브에이전트(subagent)는 웹 검색(web search)을 활용하여 주제를 병렬로 연구하고 실행 가능한 결과(actionable results)를 도출할 수 있습니다.

프롬프트(prompt) 예시:

```
Postgres(포스트그레스) 18 릴리스 노트(release notes)를 조사하고, 사용 중단된 패턴에 대해 쿼리(query)를 감사하며, 마이그레이션(migration) 작업을 생성하세요.
상위 3개 경쟁사가 API(응용 프로그래밍 인터페이스) 티어(tier) 가격을 책정하는 방식을 조사하고, 현재 가격과 비교하며, 권장 사항을 초안으로 작성하도록 3개의 연구 에이전트(research agent)를 생성하세요.
최고의 오픈소스(open source) 프로젝트들이 릴리스 프로세스(release process)를 처리하는 방법을 연구하고, CI/CD 파이프라인(pipeline)을 검토하며, 개선 사항을 초안으로 작성하세요.
```

### 웹 검색 테스트: API 문제 연구

서브에이전트(subagent)가 이전에 발견했던 문제들을 기억하시나요?

사용 중단된 `datetime.utcnow`, CORS(크로스 오리진 리소스 공유) 취약점, 그리고 금액에 사용된 `Float`입니다.

웹 검색(web search)이 현재의 모범 사례(best practices)를 통해 이 문제들을 해결하는 데 도움이 될 수 있는지 확인하고 싶었습니다.

```
최신 FastAPI(패스트API) 0.115 릴리스 기능(release features)을 찾고,
지출 추적기 API(응용 프로그래밍 인터페이스) 설계의 모범 사례(best practices)를 연구하며,
Python(파이썬) REST API(레스트 API) 보안 가이드라인(security guidelines)을 요약하도록 연구 에이전트(research agent)를 생성하세요.
```

세 개의 연구 에이전트(research agent)가 실행되었습니다:

*   FastAPI(패스트API) 0.115 기능 — 최신 릴리스(release) 연구
*   지출 추적기 API 모범 사례(best practices) — 디자인 패턴(design patterns)
*   Python(파이썬) REST API(레스트 API) 보안 — 보안 가이드라인(security guidelines)

결과는 구조화된 표와 실행 가능한 권장 사항(actionable recommendations)과 함께 돌아왔습니다.

**FastAPI(패스트API) 0.115 기능:**

연구 에이전트(research agent)는 FastAPI(패스트API) 0.115.x 버전이 요청 파라미터(request parameters)에 대한 Pydantic(파이단틱) 모델 지원을 도입했다는 것을 발견했습니다. 이제 쿼리 파라미터(query parameters), 헤더(headers), 쿠키(cookies)를 유효성 검사된 Pydantic(파이단틱) 모델로 그룹화할 수 있습니다.

```python
class FilterParams(BaseModel):
    limit: int = Field(100, gt=0, le=100)
    offset: int = Field(0, ge=0)

@app.get("/items/")
async def read_items(filter_query: Annotated[FilterParams, Query()]):
    return filter_query
```

**지출 추적기 API 모범 사례(Best Practices):**

*   복수 명사와 케밥 케이스(kebab-case) 사용: `/api/v1/expenses`
*   `/api/v1/` 접두사(prefix)로 API 버전 관리
*   금액에는 `Float` 대신 `Decimal` 사용
*   대규모 데이터셋(dataset)에는 커서 기반 페이지네이션(cursor-based pagination) 사용
*   오류 형식에는 RFC 7807 Problem Details 사용

**Python(파이썬) REST API(레스트 API) 보안 가이드라인(Security Guidelines):**

*   짧은 만료 기간(15-30분)을 가진 JWT(JSON Web Token) 사용
*   `allow_credentials=True`와 함께 `allow_origins=["*"]`를 절대 사용하지 마세요.
*   속도 제한(rate limiting)을 위해 `slowapi` 사용
*   HSTS(HTTP Strict Transport Security) 헤더(headers)로 TLS(Transport Layer Security) 1.2+ 강제 적용
*   `pip-audit`를 매월 실행

연구 에이전트(research agent)들은 코드 검토 서브에이전트(code review subagent)가 발견했던 CORS(크로스 오리진 리소스 공유) 설정, 금액에 사용된 `Float`, 그리고 사용 중단된 `datetime` 문제를 확인했습니다. 이제 저는 현재의 모범 사례(best practices)에 기반한 수정 사항을 갖게 되었습니다.

하지만 모든 Ollama(올라마) 클라우드 모델이 이렇게 빠르게 작동하는 것은 아닙니다. 그렇다면 어떤 Ollama(올라마) 클라우드 코드가 Claude Code(클로드 코드) 서브에이전트(subagent)와 잘 작동할까요?

## 권장 클라우드 모델 (Recommended Cloud Models)

모든 모델이 Ollama(올라마)에서 서브에이전트(subagent)와 웹 검색(web search)을 지원하는 것은 아닙니다. 다음 세 가지 모델이 작동합니다:

*   minimax-m2.5:cloud
*   glm-5:cloud
*   kimi-k2.5:cloud

저는 이 글 전체에서 minimax-m2.5:cloud(미니맥스 M2.5:클라우드)로 테스트했습니다. 이 모델은 서브에이전트(subagent) 생성, 병렬 작업 실행, 웹 검색(web search)을 문제없이 처리했습니다.

모델을 전환하려면 Claude Code(클로드 코드)에서 `/model`을 실행하고 원하는 옵션을 선택하세요.

제 지출 추적기 빌드는 세 개의 서브에이전트(subagent)에 걸쳐 30k 이상의 토큰을 사용했습니다. 유료 API(응용 프로그래밍 인터페이스)에서는 상당한 금액입니다.

## 마지막 생각 (Final Thoughts)

Ollama(올라마)에서 서브에이전트(subagent)와 웹 검색(web search)을 사용하여 Claude Code(클로드 코드)를 실행하는 것은 API(응용 프로그래밍 인터페이스) 요금 청구서에 신경 쓰지 않고 실험할 수 있는 방식을 변화시킵니다.

이 테스트에서 저는 바닐라 JS(vanilla JS) 프런트엔드(frontend)와 FastAPI(패스트API) 백엔드(backend)를 갖춘 완전한 지출 추적기를 세 개의 병렬 서브에이전트(subagent)를 사용하여 구축했는데, 이는 인상적입니다.

그 다음, 저는 코드 감사를 위해 또 다른 세 개의 서브에이전트(subagent)를 실행했습니다. 그들은 CORS(크로스 오리진 리소스 공유) 취약점 및 사용 중단된 `datetime` 패턴과 같은 실제 문제들을 발견했습니다.

하지만 이 설정은 프로토타입(prototype) 및 학습 목적에 가장 적합하다는 단점이 있습니다. 프로덕션(production) 작업이나 민감한 코드베이스(codebases)의 경우, 유료 API(응용 프로그래밍 인터페이스)를 사용하는 것이 좋습니다.

이 시리즈의 이전 튜토리얼들을 놓치셨다면:

*   (신규) Ollama(올라마)의 Claude Code(클로드 코드), Codex(코덱스), OpenCode(오픈코드) 실행 기능을 테스트하다 (더 이상 설정 불필요)
*   새로운 Claude Code(클로드 코드) Ollama(올라마) 워크플로우를 사용해보다 (놀랍고 자유로움)
*   Claude Code(클로드 코드) + MiniMax M2.5(미니맥스 M2.5)를 테스트하다 (원샷 CLI 도구 빌드로 정말 놀랐습니다)

다음에 Anthropic(앤트로픽)에 장애가 발생하거나 토큰을 소모하지 않고 무언가를 테스트하고 싶다면, 무엇을 해야 할지 아실 것입니다.
