클로드 코드(Claude Code)는 50개 이상의 명령어를 가지고 있습니다.

대부분의 개발자는 그 중 3~5개만 사용합니다.

나머지 명령어들은 `/help`에 문서화되지 않은 채 잠들어 있으며, 그 존재를 안다면 당신의 생산성을 10배 높여줄 것입니다.

2026년 3월 데이터에 따르면, 15개 이상의 클로드 코드 명령어를 마스터한 개발자는 클로드 코드를 "터미널 (terminal) 안의 ChatGPT"처럼 사용하는 개발자보다 3~4배 더 빠르게 기능을 출시합니다.

이것은 아무도 쓰지 않았던 완벽한 참조 가이드입니다.

모든 슬래시 명령어, 모든 CLI 플래그, 모든 키보드 단축키, 그리고 개발팀이 "별다른 발표 없이 무심코 추가한" 모든 숨겨진 기능까지 담았습니다.

이 글을 마칠 때쯤이면, 대부분의 클로드 코드 사용자들이 존재조차 모르는 명령어들을 알게 될 것입니다.

이제 자세히 알아보겠습니다.

비회원 링크!

엔터 (Enter) 키를 누르거나 클릭하여 이미지를 전체 크기로 보세요

### 세 가지 명령어 유형 이해하기

모든 명령어를 살펴보기 전에, 클로드 코드 명령어가 어떻게 작동하는지 이해해야 합니다.

**유형 1: CLI 명령어 (터미널 실행)**

이 명령어들은 터미널에서 클로드 코드를 시작할 때 실행됩니다.

`claude` `# 현재 디렉터리에서 시작`
`claude -c` `# 가장 최근 세션 이어가기`
`claude --print "질문"` `# 단일 질의 실행 후 종료`

**유형 2: 슬래시 명령어 (대화형 세션)**

이 명령어들은 클로드 코드 세션 내에서 `/`를 입력하여 실행됩니다.

`/init` `# CLAUDE.md 초기화`
`/compact` `# 컨텍스트 압축`
`/model` `# 모델 전환`

`/`를 입력하여 사용 가능한 모든 명령어를 확인하고, 입력하면서 필터링할 수 있습니다.

**유형 3: 키보드 단축키 (효율성 부스터)**

이 단축키들은 대화형 세션 중에 작동합니다.

`Ctrl+C` `# 현재 생성 중인 작업 취소`
`Ctrl+R` `# 명령어 기록 검색`
`Shift+Tab` `# 모드 전환 (일반 → 자동 승인 → 계획)`

이제 각 명령어를 하나씩 자세히 살펴보겠습니다.

### 1부: 필수 일상 명령어 (핵심 10가지)

이 명령어들은 매일 사용하게 될 것들입니다. 이들을 먼저 숙달하세요.

#### 1. /init — 프로젝트 설정

**기능:**
프로젝트 루트에 `CLAUDE.md` 파일을 생성합니다. 이 파일은 클로드(Claude)가 매 세션마다 읽는 영구적인 메모리 파일입니다.

**사용 방법:**

`/init`

클로드가 다음과 같은 내용으로 `CLAUDE.md` 시작 파일을 생성합니다.

*   프로젝트 설명
*   기술 스택
*   코드 스타일 선호 사항
*   일반적인 패턴

**중요한 이유:**

개발자 워크플로우 (workflow)에 따르면, 모든 프로젝트를 `/init`으로 시작하면 반복적인 컨텍스트 설정의 80%를 줄일 수 있습니다.

매 세션마다 "프로미스 (promises) 대신 비동기/대기 (async/await) 사용"이라고 설명할 필요 없이, `CLAUDE.md`에 영구적으로 기록됩니다.

**전문가 팁:**
`/init` 후 즉시 자신만의 규칙을 추가하세요.

```markdown
# CLAUDE.md

## 인증 (Authentication)
- 세션(sessions) 대신 JWT 토큰(tokens) 사용
- httpOnly 쿠키에 저장

## 테스트 (Testing)
- 모든 API 엔드포인트(endpoints)에 대한 테스트 작성
- Mocha 대신 Jest 사용

## 오류 처리 (Error Handling)
- 구조화된 오류 반환: { error: string, code: number }
```

#### 2. /compact — 컨텍스트 압축

**기능:**
컨텍스트 창이 가득 찼을 때 대화 기록을 압축하여 요약하고 공간을 확보합니다.

**사용 방법:**

```bash
# 기본 압축
/compact

# 특정 컨텍스트 유지 (대상 압축)
/compact 오류 처리 패턴과 인증 모듈 변경 사항을 유지
```

**언제 사용해야 하는가:**

*   세션에서 30분 이상 작업했을 때
*   "컨텍스트가 너무 커지고 있습니다" 경고가 표시될 때
*   클로드가 이전에 결정했던 사항들을 잊어버리기 시작할 때

**기술 세부 사항:**

2026년 2월 릴리스 노트 (release notes)에 따르면, `/compact`는 이제 즉시 완료됩니다 (이전에는 3~5초 소요).

압축 시 보존되는 항목: ✅ CLAUDE.md 내용, ✅ 작업 목록 항목, ✅ 주요 결정 및 패턴

압축 시 삭제되는 항목: ❌ 장황한 대화 내용, ❌ 더 이상 사용되지 않는 코드 반복

**피해야 할 실수:**

컨텍스트가 가득 찰 때까지 기다리지 마세요. 70~80% 사용량일 때 선제적으로 실행하세요.

`/context`로 컨텍스트를 확인하세요.

#### 3. /clear — 하드 리셋 (Hard Reset)

**기능:**
대화 기록을 완전히 지웁니다. 새로운 시작.

**사용 방법:**

`/clear`

**언제 사용해야 하는가:**

*   완전히 다른 작업으로 전환할 때
*   기능 개발을 완료한 후
*   다음 작업과 컨텍스트가 관련 없을 때

**/compact 대 /clear:**

| 명령어 (Command) | 효과 (Effect)           | 사용 시점 (Use When)                   |
| :--------------- | :---------------------- | :------------------------------------- |
| `/compact`       | 요약 및 컨텍스트 유지   | 동일 작업 계속, 컨텍스트 과부하        |
| `/clear`         | 하드 리셋, 새 시작      | 다른 작업으로 전환                     |

**경고:**

`/clear`는 해당 디렉터리의 명령어 기록도 지웁니다. 이전 프롬프트 (prompt)에 대한 접근을 유지하려면 `/compact`를 사용하세요.

#### 4. /model — 모델 전환

**기능:**
세션 도중 클로드 소네트 (Claude Sonnet), 오푸스 (Opus), 하이쿠 (Haiku) 모델 간을 전환합니다.

**사용 방법:**

`/model` `# 대화형 선택기`
`/model sonnet` `# Sonnet 4.6으로 전환`
`/model opus` `# Opus 4.6으로 전환`
`/model haiku` `# Haiku 4.5로 전환`

**각 모델을 언제 사용해야 하는가:**

**소네트 4.6 (Pro/Max5의 기본값):**

*   일상적인 코딩
*   리팩토링 (refactoring)
*   버그 수정 (bug fixes)
*   대부분의 간단한 작업에 대해 강력한 결과

**오푸스 4.6 (최고 모델, Max20과 함께 사용):**

*   복잡한 다단계 계획
*   아키텍처 (architecture) 결정
*   중요 프로덕션 코드
*   품질이 비용보다 중요할 때

**하이쿠 4.5 (가장 빠르고 저렴):**

*   간단한 편집
*   보일러플레이트 (boilerplate) 생성
*   빠른 질문
*   속도가 정교함보다 중요할 때

**전문가 워크플로우:**

소네트로 시작하고, 막힐 때 오푸스로 전환하며, 사소한 작업에는 하이쿠를 사용하세요.

#### 5. /cost — 토큰 사용량

**기능:**
현재 세션의 토큰 소비량과 비용을 보여줍니다.

**사용 방법:**

`/cost`

**출력:**

```
세션 비용: $2.47
입력 토큰: 48,392
출력 토큰: 12,847
모델: claude-sonnet-4-20250514
```

**중요한 이유:**

활발한 개발 시, 세션 비용은 모델과 사용 시간에 따라 $5~$50에 달할 수 있습니다.

**비용 최적화 팁:**

*   간단한 작업에는 하이쿠 사용
*   `/compact` 실행으로 컨텍스트 비용 절감
*   가능하면 오푸스에서 소네트로 전환
*   주요 상호작용 후 `/cost`로 모니터링

#### 6. /context — 컨텍스트 창 사용량

**기능:**
실시간 컨텍스트 사용량을 창 대비 백분율로 표시합니다.

**사용 방법:**

`/context`

**출력:**

`컨텍스트 사용량: 67% (134,400 / 200,000 토큰)`

**언제 압축해야 하는가:**

*   70–80% → `/compact`를 선제적으로 실행
*   90% 이상 → 클로드가 이전 컨텍스트를 잊어버리기 시작할 수 있음

**전문가 팁:**

너무 많은 컨텍스트는 LLM (거대 언어 모델)의 성능을 저하시킵니다. `/context`를 사용하여 상황을 파악하고, 한계에 도달하기 전에 압축하세요.

#### 7. /diff — 최근 변경 사항 보기

**기능:**
클로드가 세션에서 변경한 내용의 깃 디프 (Git diff)를 보여줍니다.

**사용 방법:**

`/diff` `# 모든 변경 사항 표시`
`/diff src/auth.ts` `# 특정 파일의 변경 사항 표시`

**언제 사용해야 하는가:**

*   코드 커밋 (commit) 전
*   클로드가 실제로 무엇을 변경했는지 검토할 때
*   의도치 않은 수정 사항을 발견할 때

**전문가 워크플로우:**

모든 기능 개발을 `/diff` → 검토 → 커밋으로 마무리하세요. 이것은 사전 커밋 코드 검토 (pre-commit code review)입니다.

#### 8. /help — 명령어 목록

**기능:**
사용 가능한 모든 슬래시 명령어를 표시합니다.

**사용 방법:**

`/help`

**중요한 이유:**

클로드 코드 업데이트에 따라 명령어 가용성이 변경됩니다. `/help`는 항상 현재 버전의 신뢰할 수 있는 정보원입니다.

**현재 버전 확인:**

`claude --version`

#### 9. /memory — CLAUDE.md 편집

**기능:**
세션을 떠나지 않고 `CLAUDE.md`를 편집할 수 있도록 엽니다.

**사용 방법:**

`/memory`

**언제 사용해야 하는가:**

*   세션 도중 새로운 코딩 표준을 추가할 때
*   프로젝트 컨텍스트를 업데이트할 때
*   클로드가 따라야 할 새로운 패턴을 문서화할 때

**빠른 메모리 구문:**

```markdown
# 에디터(editor)를 열지 않고 메모리에 추가
# 모든 데이터베이스 쿼리(queries)에 async/await 사용
```

`#` 접두사는 `CLAUDE.md`에 추가되는 빠른 메모리 노트를 추가합니다.

#### 10. /resume — 이전 세션 이어가기

**기능:**
이전 대화를 로드하고 이어갑니다.

**사용 방법:**

```bash
# 최신 세션 이어가기
claude --resume

# 이름으로 특정 세션 이어가기
claude --resume auth-refactor

# 세션 목록에서 선택
/resume
```

**숨겨진 팁:**

세션 데이터는 `~/.claude/projects/`에 저장됩니다. 클로드에게 "2024년 12월 세션을 찾아줘"라고 물으면 클로드가 검색해 줄 것입니다.

### 2부: 고급 생산성 명령어

이 명령어들은 숙련된 사용자 (power users)와 초보자를 구분합니다.

#### 11. /btw — 컨텍스트를 벗어나지 않고 질문하기

**기능:**
클로드의 현재 작업을 방해하거나 컨텍스트를 오염시키지 않고 보조 질문을 합니다.

**사용 방법:**

```bash
# 클로드가 큰 모듈을 리팩토링 중
# 갑자기 무언가를 확인해야 할 때

/btw useEffect와 useLayoutEffect의 차이점은 무엇인가요?
# 클로드가 답변한 후 리팩토링을 계속합니다.
```

**이것이 혁신적인 이유:**

`/btw` 이전에는 다음을 해야 했습니다.

*   현재 작업 취소
*   질문하기
*   원래 작업 재요청 (re-prompt)

이제 `/btw`만 사용하고 계속 진행할 수 있습니다.

출시일: 2026년 3월 11일 (바이럴 트윗 (viral tweet) 조회수 220만 회 돌파)

#### 12. /fast — 속도 모드 (Speed Mode)

**기능:**
속도 최적화된 API 설정을 사용하여 고속 모드 (Fast Mode)를 활성화합니다.

**사용 방법:**

`/fast` `# 켜기/끄기 전환`

**기술 세부 사항:**

고속 모드는 다른 모델이 아닙니다. 동일한 오푸스 4.6 (Opus 4.6) 모델을 최적화된 API 설정으로 실행합니다.

**사용 시점:**

*   대화형 고속 반복 (rapid iteration)
*   실시간 디버깅 (debugging)
*   빠른 실험

**끌 시점:**

*   비용 효율성
*   프로덕션 코드

**중요:**
세션 도중 고속 모드가 활성화되면, 이전의 전체 컨텍스트 (context)가 고속 모드 요율로 다시 청구됩니다.

#### 13. /plan — 계획 모드 (읽기 전용)

**기능:**
코드 변경 사항이 실행 전에 승인이 필요한 계획으로 제안됩니다.

**사용 방법:**

`# 계획 모드 전환`
`Shift+Tab` `# 모드 순환`

`# 또는 명시적으로`
`/plan`

**세 가지 모드:**

*   **일반 (Normal):** 각 도구 실행 전에 확인을 요청합니다.
*   **자동 승인 (Auto-Accept):** 확인 없이 모든 도구를 실행합니다.
*   **계획 모드 (Plan Mode):** 변경 사항을 제안하고, 실행 전에 승인을 기다립니다.

**실용적인 워크플로우:**

*   **자동 승인:** 테스트 작성, 보일러플레이트 생성
*   **계획 모드:** 프로덕션에 중요한 파일 (설정, 마이그레이션 (migrations), `package.json` 등)을 다룰 때

"계획 보기 → 승인 → 실행" 흐름은 사고를 예방합니다.

### 3부: CLI 플래그 (CLI Flags) 및 실행 옵션

이 플래그들은 터미널에서 클로드 코드를 시작할 때의 동작을 제어합니다.

#### 21. claude --print — 단일 질의 실행

**기능:**
단일 질의를 실행하고 종료합니다 (대화형 세션 없음).

**사용 방법:**

```bash
# 질문하고, 답변 얻고, 종료
claude --print "async/await와 promise의 차이점을 설명해줘"

# 스크립팅(scripting)에 완벽
result=$(claude --print "무작위 UUID를 생성해줘")
echo $result
```

**사용 사례:**

*   셸 스크립트 (Shell scripts)
*   CI/CD 파이프라인
*   전체 세션을 실행하지 않고 빠른 질문

#### 22. claude -c 또는 --continue — 최신 세션 이어가기

**기능:**
현재 디렉터리에서 가장 최근 세션을 이어갑니다.

**사용 방법:**

```bash
cd ~/projects/my-app
claude -c   # 이 디렉터리의 최신 세션을 계속합니다.
```

**다른 디렉터리에서 이어가기:**

`claude --resume session-id`

**풀 리퀘스트 (Pull Request)에서 이어가기:**

`claude --from-pr 123`
PR #123과 연결된 세션을 직접 이어갑니다.

#### 23. --append-system-prompt 대 --system-prompt

**기능:**
시스템 지침을 사용자 정의합니다.

**사용 방법:**

```bash
# 기본 지침에 추가 (안전)
claude --append-system-prompt "항상 TypeScript strict 모드를 사용하세요"

# 모든 기본 지침 교체 (위험)
claude --system-prompt "당신은 파이썬(Python) 전문가입니다"
```

**각 옵션 사용 시점:**

`--append-system-prompt` (권장):

*   기본 클로드 코드 기능을 유지하면서 사용자 정의 규칙을 추가합니다.
*   대부분의 사용 사례에 안전합니다.

`--system-prompt` (고급):

*   기본 지침을 완전히 교체합니다.
*   전체 제어가 필요할 때만 사용하세요.
*   모든 기본 클로드 코드 동작을 제거합니다.

#### 24. --dangerously-skip-permissions

**기능:**
묻지 않고 모든 작업을 자동 승인합니다.

**사용 방법:**

`# ⚠️ 경고: 신뢰할 수 있는 컨테이너(containers)에서만 사용하세요`
`claude --dangerously-skip-permissions`

**사용 시점:**

ONLY 신뢰할 수 있는 컨테이너 환경 (도커 (Docker), CI/CD)에서만 사용하세요.

절대로 프로덕션 (운영) 데이터가 있는 로컬 머신에서 사용하지 마세요.

이 플래그를 사용하기 전에 "자동 승인 안전 사용 가이드"를 참조하세요.

### 4부: 키보드 단축키 (효율성 증대 도구)

이 단축키들은 워크플로우 (workflow) 속도를 크게 향상시킵니다.

**필수 단축키**

*   `Ctrl+C` 현재 생성 중인 작업 취소
*   `Ctrl+R` 명령어 기록 검색
*   `Tab` 생각 중 표시 전환
*   `Shift+Tab` 모드 순환 (일반 → 자동 승인 → 계획)
*   `Esc Esc` 되감기 메뉴

**탐색 및 편집**

*   `Ctrl+T` 작업 목록 전환
*   `Alt+M` 모드 전환 (Shift+Tab과 동일)
*   `Alt+P` 이전 메시지
*   `Alt+N` 다음 메시지
*   `Alt+B` 대화에서 뒤로
*   `Alt+F` 대화에서 앞으로

**macOS 사용자:**

Alt 키를 사용하는 단축키는 터미널에서 옵션 (Option) 키를 메타 (Meta) 키로 구성해야 합니다.

iTerm2: 설정 → 프로필 → 키 → 왼쪽/오른쪽 옵션을 "Esc+"로 설정

**고급 단축키**

*   `Shift+Enter` 전송 없이 여러 줄 입력
*   `Ctrl+L` 화면 지우기 (미관상, 대화 내용은 지우지 않음)
*   `Ctrl+D` 클로드 코드 종료

**WSL 사용자:**

윈도우 터미널 (Windows Terminal)에서 Shift+Enter가 바로 작동하지 않을 수 있습니다.

`/terminal-setup`을 실행하여 키 바인딩 (keybinding)을 설치하세요.

**파일 및 명령어 단축키**

*   `@ + 경로` 파일 자동 완성
*   `! + 명령어` 직접 배시 (Bash) 실행
*   `# + 텍스트` 빠른 메모리 추가

**예시:**

```bash
# 파일 자동 완성
@src/auth.ts   # 전체 경로로 자동 완성

# 직접 배시 실행
! git status   # 즉시 실행되며, 출력은 컨텍스트로 전달

# 빠른 메모리
# 인증에 JWT 토큰 사용
```

### 6부: 구성 파일 및 사용자 정의

클로드 코드가 데이터를 저장하는 위치와 사용자 정의 방법을 이해합니다.

**파일 위치**

*   `~/.claude/` `# 주요 구성 디렉터리`
*   `~/.claude/projects/` `# 세션 기록`
*   `~/.claude/commands/` `# 사용자 정의 슬래시 명령어 (레거시)`
*   `~/.claude/skills/` `# 에이전트 스킬 (2026 표준)`
*   `~/.claude/output-modes/` `# 사용자 정의 출력 스타일`
*   `~/.claude/keybindings.json` `# 키보드 단축키`

**CLAUDE.md — 프로젝트 메모리**

위치: 프로젝트 루트

**저장하는 내용:**

*   프로젝트 설명
*   기술 스택
*   코드 스타일 규칙
*   일반적인 패턴
*   테스트 요구 사항

**예시:**

```markdown
# 프로젝트: 전자상거래 API (E-Commerce API)

## 기술 스택 (Tech Stack)
- Node.js + Express
- Prisma를 통한 PostgreSQL
- JWT 인증

## 규칙 (Rules)
- 콜백(callbacks) 대신 async/await 사용
- 모든 엔드포인트에 대한 테스트 작성
- 구조화된 오류 반환: { error, code }

## 패턴 (Patterns)
- 모든 데이터베이스 쿼리는 /services에
- 라우트 핸들러(Route handlers)는 /routes에
- 미들웨어(Middleware)는 /middleware에
```

**사용자 정의 스킬 (명령어와 통합됨)**

위치: `~/.claude/skills/`

**2026년에 변경된 사항:**

기존의 사용자 정의 명령어 (`.claude/commands/`)와 스킬 (`.claude/skills/`)이 통합되었습니다.

기존의 `.claude/commands/` 파일은 여전히 작동하지만, 새로운 표준은 스킬입니다.

**스킬의 추가 기능:**

*   프런트매터 (Frontmatter) 기반의 자동 호출 제어
*   파일 관리 기능
*   에이전트 스킬 오픈 표준 (Agent Skills Open Standard) 준수

**스킬 예시:**

```yaml
---
name: deploy-staging
description: 현재 브랜치(branch)를 스테이징(staging) 환경에 배포
auto_invoke: false
---

# 스테이징 환경에 배포
1. 테스트 실행
2. 프로덕션 번들(bundle) 빌드
3. 스테이징 서버에 푸시(push)
4. 스모크 테스트(smoke tests) 실행
```

**키 바인딩 (Keybindings) 사용자 정의**

위치: `~/.claude/keybindings.json`

**편집 방법:**

`/keybindings`

**예시:**

```json
{
  "toggle_thinking": "Tab",
  "cancel": "Ctrl+C",
  "search_history": "Ctrl+R",
  "toggle_task_list": "Ctrl+T"
}
```

클로드 코드를 다시 시작할 필요 없이 변경 사항이 즉시 적용됩니다.

### 핵심 요약

**1. 핵심 10가지부터 마스터하세요.**
`/init, /compact, /clear, /model, /cost, /context, /diff, /help, /memory, /resume`
이들만으로도 생산성을 3~4배 높일 수 있습니다.

**2. /btw는 혁신적입니다.**
컨텍스트를 벗어나지 않고 질문하세요. 자유롭게 사용하세요.

**3. 수동적이 아닌 선제적으로 압축하세요.**
컨텍스트가 95%가 아닌 70-80%일 때 `/compact`를 실행하세요.

**4. CLAUDE.md는 생산성 증대 도구입니다.**
한 번 설정하면 매 세션마다 10~15분을 절약할 수 있습니다.

**5. 모드 전환은 사고를 방지합니다.**

*   **자동 승인:** 보일러플레이트 및 테스트
*   **계획 모드:** 프로덕션에 중요한 파일
*   **일반:** 그 외 모든 것

**6. 중요한 세션을 내보내세요.**
`/export`는 해결책을 검색 가능한 문서로 바꿔줍니다.

**7. 오푸스 (Opus) 사용 시 비용을 모니터링하세요.**
주요 상호작용 후 `/cost`를 사용하세요. 오푸스 세션은 $50에 달할 수 있습니다.

**8. 키보드 단축키는 속도를 10배 높여줍니다.**
`Shift+Tab`, `Ctrl+T`, `Esc Esc`, `Ctrl+R` — 이들을 익히세요.

**9. 서브 에이전트는 컨텍스트를 깔끔하게 유지합니다.**
특정 작업을 위임하세요. 메인 대화는 집중된 상태를 유지합니다.

**10. 숨겨진 기능이 존재합니다.**
`/vim`, `/remote-control`, `/usage-report` — `/help`를 정기적으로 탐색하세요.
