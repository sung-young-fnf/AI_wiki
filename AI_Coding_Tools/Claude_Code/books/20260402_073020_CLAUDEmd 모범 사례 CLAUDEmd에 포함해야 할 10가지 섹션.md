# CLAUDE.md 모범 사례: CLAUDE.md에 포함해야 할 10가지 섹션

`CLAUDE.md`는 클로드 코드(Claude Code)에 프로젝트별 컨텍스트(Context)를 제공하는 핵심 파일입니다. 이 파일은 AI가 프로젝트 요구 사항과 코드베이스를 파악할 수 있도록 돕는 온보딩(Onboarding) 가이드 역할을 합니다. 클로드 AI 모델이 프로젝트에서 코드를 새로 작성하거나 수정할 때 가장 먼저 확인하는 것이 바로 이 `CLAUDE.md` 파일입니다.

### 파일 위치 및 형식
`CLAUDE.md` 파일은 일반적으로 프로젝트 저장소의 루트(Root) 디렉토리에 위치합니다. 파일 형식은 마크다운(.md)을 사용합니다. 앤스로픽(Anthropic)에 따르면, 클로드 코드는 대화를 시작할 때마다 이 파일을 프로젝트 메모리의 일부로 로드하며, 이를 강제적인 규칙이라기보다는 작업 시 참고할 컨텍스트로 취급합니다.

---

### CLAUDE.md에 포함해야 할 10가지 핵심 섹션

#### 1. 프로젝트 개요 (Project Overview)
클로드에게 프로젝트의 배경을 설명하는 가장 가치 있는 섹션입니다. 제품의 정의, 대상 사용자, 최적화 목표, 주요 비즈니스 및 UX 제약 사항을 평이한 언어로 설명합니다. 장황한 브랜드 스토리보다는 명확한 멘탈 모델(Mental Model)을 제공하는 것이 좋습니다.

#### 2. 기술 스택 (Tech Stack)
클로드가 프로젝트와 맞지 않는 엉뚱한 라이브러리를 도입하는 것을 방지합니다. 사용 중인 프레임워크, 프로그래밍 언어, 스타일링 시스템, 상태 관리, 테스트 프레임워크 등을 구체적으로 명시하세요. 또한 사용하지 말아야 할 기술(예: "Material UI 사용 금지")을 명시하는 것도 효과적입니다.

#### 3. 아키텍처 (Architecture)
저장소가 어떻게 조직되어 있는지 설명합니다. 주요 디렉토리 구조, 각 영역의 역할, 데이터 흐름, 관심사 분리(Separation of Concerns) 원칙을 포함하세요. 
* **보안 주의사항:** API 키와 같은 민감한 정보는 `.env` 파일에 보관하고 있음을 클로드에게 명확히 지시하여, 키가 엉뚱한 곳에 저장되거나 노출되는 위험을 최소화해야 합니다.

#### 4. 코딩 규칙 (Coding Conventions)
생성되는 코드의 품질과 가독성에 직접적인 영향을 미치는 섹션입니다. 명명 규칙(Naming Conventions), 컴포넌트 패턴, 타입 표준(Typing Standards), 파일 크기 제한, 임포트(Import) 규칙 등을 정의합니다. "깨끗한 코드를 작성하라"는 식의 모호한 표현보다는 "라우트 파일을 제외하고는 이름 있는 내보내기(Named Exports)를 사용하라"는 식의 구체적인 규칙이 좋습니다.

#### 5. UI 및 디자인 시스템 규칙 (UI and Design System Rules)
프런트엔드 프로젝트에서 특히 중요합니다. 시각적 스타일, 간격(Spacing) 철학, 타이포그래피, 반응형 대응 방식, 접근성(Accessibility) 기대치 등을 정의합니다. 단순히 "현대적인 디자인"이라고 하기보다는 구체적인 구현 지침(예: "8px 그리드 시스템 사용")을 제공하세요.

#### 6. 콘텐츠 및 카피 가이드라인 (Content and Copy Guidance)
제품의 어조와 매너(Tone and Voice)를 설정합니다. 문구가 간결해야 하는지 상세해야 하는지, 기술적인 용어를 선호하는지 평이한 언어를 선호하는지 등을 명시합니다.

#### 7. 테스트 및 품질 기준 (Testing and Quality Bar)
클로드가 작업 완료 후 검증하는 방법을 정의합니다. 어떤 테스트를 추가해야 하는지, 린트(Lint)나 타입 체크(Typecheck) 요구 사항은 무엇인지, 작업이 "완료(Done)"되었다고 판단하는 기준이 무엇인지 설명합니다.

#### 8. 파일 및 컴포넌트 배치 규칙 (File and Component Placement Rules)
저장소의 일관성을 유지하기 위한 규칙입니다. 새 파일을 생성할 위치, 기존 파일을 수정해야 할 시점, 추상화가 필요한 경우 등을 정의하여 코드베이스가 난잡해지는 것을 방지합니다.

#### 9. 안전 변경 규칙 (Safe-change Rules)
실제 프로젝트 운영 시 매우 중요합니다. 클로드가 함부로 변경해서는 안 될 영역(예: 공용 API 라우트, 데이터베이스 스키마, 인증 흐름 등)을 명시하여 운영상의 리스크를 줄입니다.

#### 10. 특정 명령어 (Specific Commands)
클로드가 프로젝트를 운영할 때 직접 실행할 수 있는 명령어들을 제공합니다. 의존성 설치, 개발 서버 실행, 빌드, 린트, 테스트 등 실제 유효한 명령어들을 목록화합니다.

### CLAUDE.md 예시 구성 (React 프로젝트 기준)

```markdown
## Project Overview
이 프로젝트는 디자이너가 AI로 랜딩 페이지를 제작하는 웹 앱입니다. 빠른 반복 작업과 시각적 완성도에 최적화되어 있습니다.

## Tech Stack
- Next.js 15 (App Router)
- TypeScript
- Tailwind CSS
- shadcn/ui

## Coding Conventions
- 모든 컴포넌트는 함수형으로 작성
- 'any' 사용 금지, 엄격한 타입 준수
- 비동기 패턴에는 async/await 선호

## Commands
- 설치: `pnpm install`
- 개발: `pnpm dev`
- 빌드: `pnpm build`
```
