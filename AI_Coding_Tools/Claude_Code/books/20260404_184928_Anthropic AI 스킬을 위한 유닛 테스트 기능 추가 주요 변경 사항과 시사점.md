# Anthropic, AI 스킬을 위한 유닛 테스트 기능 추가: 주요 변경 사항과 시사점

Anthropic은 지난 3월 3일, `skill-creator`에 평가(evals), 벤치마크(benchmarks), 그리고 블라인드 A/B 테스트 기능을 업데이트했습니다. 이제 클로드(Claude)를 위한 커스텀 스킬을 개발할 때, 막연한 추측 대신 성능을 직접 증명할 수 있게 되었습니다. 이번 업데이트의 구체적인 내용과 해결된 문제, 그리고 여전한 한계점을 정리합니다.

### '감'에 의존하던 기존 워크플로우의 한계
기존에 클로드 코드(Claude Code)나 Claude.ai용 스킬을 만들 때는 `SKILL.md` 파일을 작성하고 한두 번의 프롬프트로 테스트한 뒤, 결과가 괜찮아 보이면 바로 배포하는 방식이 일반적이었습니다. Anthropic은 대다수의 스킬 제작자가 엔지니어가 아닌 해당 분야의 전문가라는 점을 인정하며, 모델이 업데이트되었을 때 기존 스킬이 여전히 잘 작동하는지 확인할 도구가 부족했음을 시인했습니다.

실제로 모델 업데이트 이후, 잘 작동하던 보고서 생성 스킬이 아무런 오류 메시지 없이 데이터를 잘못된 열에 입력하는 '소리 없는 실패(silent failure)' 사례가 발생하기도 했습니다. 이번 업데이트는 단순히 스킬 제작을 돕는 것을 넘어, "이 스킬이 여전히 제대로 작동하는가"라는 신뢰성의 문제를 해결하는 데 초점을 맞추고 있습니다.

### 새롭게 추가된 네 가지 모드

새로운 `skill-creator`는 자연어 기반으로 네 가지 모드를 제공하며, 별도의 코딩이 필요하지 않습니다.

1.  **생성(Create):** 워크플로우를 설명하면 `SKILL.md` 파일을 생성합니다. (기존과 동일)
2.  **평가(Eval):** 이번 업데이트의 핵심입니다. 테스트 프롬프트를 정의하고 예상되는 올바른 출력을 기술하면, `skill-creator`가 스킬을 실행하여 합격률(pass rate), 소요 시간, 토큰 사용량을 보고합니다.
    ```json
    {
      "skill_name": "client-report-generator",
      "evals": [
        {
          "id": 1,
          "prompt": "최신 CRM 데이터를 사용하여 Acme Corp의 주간 고객 보고서 생성",
          "expected_output": "수익이 정확한 열에 위치하고, 지출이 분리되어 있으며, 합계가 원본 데이터와 일치함",
          "files": ["acme-crm-export.csv"]
        }
      ]
    }
    ```
3.  **개선(Improve):** 스킬의 지침뿐만 아니라 클로드가 스킬을 활성화할지 여부를 결정하는 '프론트매터(frontmatter)' 설명을 최적화합니다. 테스트 쿼리를 학습용과 검증용으로 나누어 반복 실행함으로써 스킬 호출(trigger) 정확도를 높입니다.
4.  **벤치마크(Benchmark):** 평가를 표준화된 테스트로 실행합니다. 모델 업데이트 후나 스킬 수정 후에 실행하여 버전별 성능 변화를 추적할 수 있습니다.

### 병렬 실행과 블라인드 비교를 통한 객관성 확보
`skill-creator`는 테스트 케이스별로 스킬이 있는 상태와 없는 상태의 독립적인 에이전트 두 개를 동시에 실행합니다. 이러한 **병렬 실행(Parallel execution)**은 이전 테스트의 결과가 다음 테스트에 영향을 주는 컨텍스트 전이(context bleed)를 방지합니다.

또한, A/B 테스트를 위해 **블라인드 비교 에이전트(blind comparator agent)**를 사용합니다. 이 에이전트는 어떤 출력이 수정된 버전인지 모르는 상태에서 결과를 평가하므로, 평가의 편향을 제거하고 객관성을 높입니다.

### 스킬의 유형에 따른 관리 전략
Anthropic은 테스트 시 고려해야 할 두 가지 스킬 유형을 제시합니다.

*   **역량 강화 스킬(Capability uplift skills):** 모델의 부족한 기능을 보완하는 스킬입니다. 시간이 지나 모델 자체가 개선되면 스킬이 불필요해질 수 있습니다. 평가는 이러한 스킬의 '은퇴 시기'를 결정하는 지표가 됩니다.
*   **부호화된 선호도 스킬(Encoded preference skills):** 특정 조직의 템플릿이나 규칙을 따르는 스킬입니다. 모델의 변화보다는 조직의 프로세스 변화로 인해 작동이 멈추는 경우가 많으므로, 워크플로우 준수 여부를 체크하는 평가가 중요합니다.

### 한계점과 과제
이번 업데이트가 획기적이지만, 몇 가지 한계는 여전히 존재합니다.
*   **CI/CD 통합 부재:** 평가 결과를 CI 시스템에 자동으로 연결하는 기본 기능이나 GitHub Action이 아직 없습니다.
*   **교차 모델 테스트 미지원:** Sonnet과 Opus 등 여러 모델에서의 작동 여부를 확인하려면 수동으로 각각 벤치마크를 실행해야 합니다.
*   **자동 회귀 알림 부족:** 성능 저하 시 사용자에게 즉시 알림을 보내는 기능은 포함되지 않았습니다.

### 결론: 명세서로서의 평가(Evals)
이번 업데이트는 단순히 테스트 인프라를 추가한 것 이상의 의미를 갖습니다. Anthropic은 "평가가 곧 스킬 자체가 될 수도 있다"고 시사합니다. 현재 `SKILL.md`가 단계별 절차를 설명한다면, 미래에는 평가를 통해 '무엇(what)'이 결과물로 나와야 하는지만 명확히 정의함으로써 모델이 스스로 '어떻게(how)' 할지 찾아내게 될 것입니다.

단기적으로 이 기능은 사용자에게 "내 AI 스킬이 제대로 작동할 것"이라는 확신을 주는 강력한 도구가 될 것입니다.
