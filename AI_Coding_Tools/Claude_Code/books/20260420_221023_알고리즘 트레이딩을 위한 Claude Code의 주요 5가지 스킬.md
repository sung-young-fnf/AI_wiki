# 알고리즘 트레이딩을 위한 Claude Code의 주요 5가지 스킬

대부분의 트레이더는 병목 현상이 전략에 있다고 생각합니다. 하지만 그렇지 않습니다. 진정한 병목은 '아이디어를 떠올렸다'와 '이것이 실제 시스템에서 작동한다' 사이의 시간입니다. 상용구(boilerplate) 작성, API 연결, 새벽 2시에 `vectorbt` 오류를 디버깅하는 과정에서 대부분의 전략이 좌절됩니다.

만약 당신이:

*   Python으로 시스템 트레이딩 시스템(systematic trading systems)을 구축하고 있거나,
*   실제 자본을 투입하기 전에 기술 지표(technical indicators)를 테스트하거나,
*   수 주가 걸리는 퀀트 작업을 며칠로 단축하려 한다면,

이 내용은 중요합니다.

Claude Code는 단순히 챗봇이 아닙니다. 이는 터미널에서 실행되고 파일을 읽고 머신에서 코드를 실행하는 에이전트 기반 환경(agentic environment)입니다. 알고리즘 트레이딩에서는 이것이 가능성의 영역을 변화시킵니다.

진정한 힘은 스킬(skills)에서 나옵니다. SKILL.md 파일은 특정 작업을 Claude가 어떻게 처리해야 하는지 정확히 알려주는 명령어 세트(instruction sets) 역할을 합니다. 이를 레시피(recipes)로 생각해보세요. 작업이 일치하면 Claude가 자동으로 따르는 단계별 워크플로우(workflow)입니다. 하나를 설치하면 Claude는 해당 도메인의 전문가로 변모합니다.

다음은 공개 리포지토리(public repositories)에서 가져온 5가지 스킬로, EODHD를 시장 데이터(market data)로 사용하여 Claude Code를 트레이딩에 정말 유용하게 만드는 방법과 설치 명령, 내부 레시피, 작동하는 Python 코드가 포함되어 있습니다.

### 1. 백테스팅 전문가 — 체계적인 전략 테스트

출처: `tradermonty/claude-trading-skills`

```bash
git clone https://github.com/tradermonty/claude-trading-skills.git
cp -r claude-trading-skills/skills/backtest-expert ~/.claude/skills/
```

이 스킬은 Claude Code를 전담 백테스팅 분석가로 변모시킵니다. SKILL.md 레시피는 전략을 설명할 때마다 Claude가 구조화된 워크플로우를 따르도록 지시합니다:

**스킬 레시피 (backtest-expert):**
1.  전략 규칙 및 파라미터(parameters) 확인
2.  지정된 심볼(symbol) 및 기간에 대한 과거 OHLCV 데이터 가져오기
3.  지표(indicator)를 처음부터 계산 — 라이브러리 블랙박스(black boxes) 사용 금지
4.  명확한 조건을 가진 진입/청산 신호(entry/exit signals) 생성
5.  벡터화된 백테스트 실행: 수익률, 샤프 지수(Sharpe), 최대 낙폭(max drawdown), 승률(win rate)
6.  자산 곡선(equity curve) + 요약 테이블 출력
7.  표본 내(in-sample) 기간이 2년 미만일 경우 과적합(overfitting) 위험 플래그 지정

실제로 당신은 아이디어를 평이한 영어로 설명하고, Claude는 레시피를 따릅니다.

```python
import requests
import pandas as pd
import numpy as np
API_KEY = "YOUR_EODHD_KEY"
# Fetch EOD historical prices via EODHD REST API
url = f"https://eodhd.com/api/eod/AAPL.US"
params = {
    "api_token": API_KEY,
    "from": "2022-01-01",
    "to": "2024-12-31",
    "period": "d",
    "fmt": "json"
}
data = requests.get(url, params=params).json()
df = pd.DataFrame(data)[["date", "adjusted_close", "volume"]]
df["date"] = pd.to_datetime(df["date"])
df.set_index("date", inplace=True)
df.columns = ["close", "volume"]
# RSI from scratch
delta = df["close"].diff()
gain  = delta.clip(lower=0).rolling(14).mean()
loss  = -delta.clip(upper=0).rolling(14).mean()
df["rsi"] = 100 - (100 / (1 + gain / loss))
# Signals
df["signal"] = 0
df.loc[df["rsi"] < 30, "signal"] =  1  # buy
df.loc[df["rsi"] > 70, "signal"] = -1  # sell
# Vectorized strategy returns
df["returns"]  = df["close"].pct_change()
df["strategy"] = df["signal"].shift(1) * df["returns"]
sharpe = df["strategy"].mean() / df["strategy"].std() * np.sqrt(252)
total  = (1 + df["strategy"]).prod() - 1
print(f"Sharpe: {sharpe:.2f} | Total return: {total:.2%}")
```

여기서부터 워크-포워드 검증(walk-forward validation) 루프, 다중 티커 스크리너(multi-ticker screeners), 파라미터 민감도 그리드(parameter sensitivity grids) 등을 구축할 수 있습니다.

**장점**
*   구조화된 워크플로우로 과적합(overfitting) 검사를 생략하는 것을 방지합니다.
*   명시적인 신호 로직(signal logic)을 강제하여 암묵적인 라이브러리 가정을 배제합니다.

**단점**
*   사전에 명확한 전략 설명이 필요합니다. 모호한 입력은 일반적인 백테스트를 생성합니다.

**가장 적합한 대상:** 매주 5~10가지 전략 변형을 테스트하고 모든 결과에 대해 재현 가능하고 비교 가능한 결과를 필요로 하는 퀀트(quant) 투자자.

### 2. 시장 데이터 파이프라인 — EODHD 통합

출처: `JoelLewis/finance_skills · trading-operations plugin`

```bash
npx skills add JoelLewis/finance_skills --plugin trading-operations
```

이 스킬은 전체 데이터 수집 계층을 처리합니다. 이 레시피는 Claude가 단일 공급자로부터 종가(EOD prices), 장중(intraday) 바, 기본 재무 데이터(fundamentals), 실시간 시세(real-time quotes)와 같은 시장 데이터를 가져오고 구조화하는 방식을 표준화합니다.

**스킬 레시피 (market-data-pipeline):**
1.  필요한 데이터 유형 식별: 종가(EOD) / 장중(intraday) / 기본 재무 데이터(fundamentals) / 실시간(real-time)
2.  해당 데이터 유형에 맞는 EODHD 엔드포인트(endpoint) 선택
3.  적절한 파라미터(parameters)로 요청 빌드 (심볼 형식, 날짜 범위, 간격)
4.  응답을 표준 열 이름(standard column names)을 가진 DataFrame으로 정규화
5.  과거 가격 분석을 위해 기업 활동 조정(corporate-action adjustments) 적용
6.  동일 세션 내에서 중복 API 호출을 피하기 위해 결과를 캐싱(cache)
7.  지표 계산 준비가 된 DataFrame 반환

부정확한 데이터는 백테스트의 조용한 살인자입니다. 생존자 편향(survivorship bias), 조정되지 않은 가격, 누락된 기업 활동(corporate actions) 등은 오류를 발생시키지 않습니다. 그저 당신의 전략이 실제보다 더 좋아 보이게 만들 뿐입니다.

```python
import requests
import pandas as pd
API_KEY = "YOUR_EODHD_KEY"
def fetch_eod(symbol: str, start: str, end: str) -> pd.DataFrame:
    r = requests.get(
        f"https://eodhd.com/api/eod/{symbol}",
        params={"api_token": API_KEY, "from": start, "to": end, "period": "d", "fmt": "json"}
    )
    df = pd.DataFrame(r.json())
    df["date"] = pd.to_datetime(df["date"])
    return df.set_index("date")[["open", "high", "low", "close", "adjusted_close", "volume"]]
def fetch_intraday(symbol: str, interval: str = "1m") -> pd.DataFrame:
    r = requests.get(
        f"https://eodhd.com/api/intraday/{symbol}",
        params={"api_token": API_KEY, "interval": interval, "fmt": "json"}
    )
    df = pd.DataFrame(r.json())
    df["datetime"] = pd.to_datetime(df["datetime"])
    return df.set_index("datetime")[["open", "high", "low", "close", "volume"]]
def fetch_fundamentals(symbol: str) -> dict:
    r = requests.get(
        f"https://eodhd.com/api/fundamentals/{symbol}",
        params={"api_token": API_KEY, "fmt": "json"}
    )
    return r.json()
# Usage
aapl     = fetch_eod("AAPL.US", "2023-01-01", "2024-12-31")
fund     = fetch_fundamentals("AAPL.US")
eps      = fund["Highlights"]["EPS"]
print(f"EPS: {eps} | Latest adjusted close: ${aapl['adjusted_close'].iloc[-1]:.2f}")
```

EODHD는 전 세계 70개 이상의 거래소를 지원하며 — 미국 주식(US equities), ETF, 외환(forex), 암호화폐(crypto) — 기업 활동이 조정된 가격을 제공하며, 프로토타이핑을 위한 무료 티어도 있습니다.

**장점**
*   단일 API로 조정된 종가(EOD) + 장중(intraday, 1분) + 기본 재무 데이터(fundamentals) + 실시간 데이터 제공
*   무료 티어 이용 가능; 유료 플랜은 월 €19.99부터 시작

**단점**
*   지연 없는 실시간 시세(real-time non-delayed quotes)는 유료 플랜이 필요합니다.

**가장 적합한 대상:** 인디 개발자 가격으로 기관급 품질의 데이터가 필요한 진지한 백테스팅 또는 라이브 시스템을 구축하는 모든 사람.

### 3. 신호 생성기 — 전략 로직을 실행 가능한 코드로

출처: `ScientiaCapital/skills · active/signal-generation`

```bash
git clone https://github.com/scientiacapital/skills.git
cp -r skills/active/signal-generation ~/.claude/skills/
```

당신은 전략을 가지고 있습니다. 그 전략은 당신의 머릿속에 있거나, 연구 노트에 있거나, TradingView Pine Script에 있을 수 있습니다. 당신은 그것을 깔끔하고 벡터화된(vectorized) Python 코드로 필요로 합니다. 이 스킬이 이러한 변환을 처리합니다.

**스킬 레시피 (signal-generation):**
1.  자연어(natural language)로 된 전략 규칙을 명시적인 조건으로 파싱(parse)
2.  각 조건을 pandas/numpy 연산에 매핑
3.  벡터화된 연산(vectorized operations)을 사용하여 지표(indicators) 계산 (행 단위 루프 없음)
4.  진입(entry) 및 청산(exit) 시리즈를 별도로 구축
5.  지정된 경우 시간대(time-of-day) 또는 세션 필터(session filters) 적용
6.  DataFrame 출력: `[close, *indicators, signal, position]`
7.  검증: 미래 정보 편향(lookahead bias) 없음 (모든 지표는 신호 비교 전에 시프트됨)

```python
import requests
import pandas as pd
import numpy as np
API_KEY = "YOUR_EODHD_KEY"
r = requests.get(
    "https://eodhd.com/api/eod/TSLA.US",
    params={"api_token": API_KEY, "from": "2023-01-01", "to": "2024-12-31", "period": "d", "fmt": "json"}
)
df = pd.DataFrame(r.json())
df["date"] = pd.to_datetime(df["date"])
df = df.set_index("date")[["high", "low", "adjusted_close"]].rename(columns={"adjusted_close": "close"})
# EMA crossover + ADX filter - no TA-Lib
def ema(s, n):
    return s.ewm(span=n, adjust=False).mean()
def compute_adx(df, n=14):
    tr = pd.concat([
        df["high"] - df["low"],
        (df["high"] - df["close"].shift()).abs(),
        (df["low"]  - df["close"].shift()).abs()
    ], axis=1).max(axis=1)
    dm_pos = (df["high"] - df["high"].shift()).clip(lower=0)
    dm_neg = (df["low"].shift() - df["low"]).clip(lower=0)
    atr    = tr.ewm(span=n, adjust=False).mean()
    di_pos = 100 * dm_pos.ewm(span=n, adjust=False).mean() / atr
    di_neg = 100 * dm_neg.ewm(span=n, adjust=False).mean() / atr
    dx     = (100 * (di_pos - di_neg).abs() / (di_pos + di_neg))
    return dx.ewm(span=n, adjust=False).mean()
df["ema20"] = ema(df["close"], 20)
df["ema50"] = ema(df["close"], 50)
df["adx"]   = compute_adx(df)
cross_up     = (df["ema20"] > df["ema50"]) & (df["ema20"].shift() <= df["ema50"].shift())
adx_confirmed = df["adx"] > 25
df["signal"] = (cross_up & adx_confirmed).astype(int)
print(df[df["signal"] == 1][["close", "ema20", "ema50", "adx"]].tail())
```

**장점**
*   미래 정보 편향(lookahead-bias) 검사가 레시피에 포함되어 있어 코드를 전달하기 전에 Claude가 이를 플래그합니다.
*   벡터화된 pandas 연산을 생성하여 프로덕션(production) 준비가 된 코드를 제공합니다.

**단점**
*   실제 자본과 연결하기 전에 항상 신호 로직(signal logic)을 수동으로 검증해야 합니다.

**가장 적합한 대상:** 전략 규칙으로 생각하고 Claude가 Python 번역을 처리해주기를 원하는 트레이더.

### 4. 리스크 관리자 — 포지션 규모 및 노출 제어

출처: `JoelLewis/finance_skills · wealth-management plugin (risk-measurement skill)`

```bash
npx skills add JoelLewis/finance_skills --plugin wealth-management
```

리스크 관리(risk management) 없는 전략은 추가 단계를 거치는 투기에 불과합니다. 이 스킬은 모든 규모 및 노출 계산에 대해 Claude에게 구조화된 의사결정 프레임워크를 제공합니다.

**스킬 레시피 (risk-management):**
1.  현재 설정에 대한 ATR 기반 손절매(stop) 거리 계산
2.  고정 비율 규모(fixed-fractional sizing) 적용 (기본값: 거래당 계좌 리스크 1%)
3.  포트폴리오 과열(portfolio heat) 확인: 총 노출이 자산의 5%를 초과하는 경우 플래그
4.  현재 포트폴리오에 대한 과거 VaR(Historical VaR, 95% 신뢰 수준) 계산
5.  출력: 진입가, 손절매가, 주식 수, 달러 리스크, R-배수 목표(R-multiple target)
6.  킬 스위치(kill-switch) 임계값 정의: 낙폭(drawdown)이 X%를 초과하면 새로운 진입 중단

```python
import requests
import pandas as pd
import numpy as np
API_KEY = "YOUR_EODHD_KEY"
r = requests.get(
    "https://eodhd.com/api/eod/AAPL.US",
    params={"api_token": API_KEY, "from": "2024-10-01", "to": "2024-12-31", "period": "d", "fmt": "json"}
)
df = pd.DataFrame(r.json())
df["date"] = pd.to_datetime(df["date"])
df = df.set_index("date")[["high", "low", "adjusted_close"]].rename(columns={"adjusted_close": "close"})
# ATR (14-period)
tr  = pd.concat([
    df["high"] - df["low"],
    (df["high"] - df["close"].shift()).abs(),
    (df["low"]  - df["close"].shift()).abs()
], axis=1).max(axis=1)
atr = tr.ewm(span=14, adjust=False).mean().iloc[-1]
def size_position(equity, risk_pct, entry, stop):
    risk_per_share = abs(entry - stop)
    if risk_per_share == 0:
        return 0
    return int((equity * risk_pct / 100) / risk_per_share)
equity      = 10_000
entry_price = df["close"].iloc[-1]
stop_price  = entry_price - (2 * atr)
shares      = size_position(equity, 1.0, entry_price, stop_price)
print(f"Entry:         ${entry_price:.2f}")
print(f"Stop (2×ATR):  ${stop_price:.2f}  |  ATR: ${atr:.2f}")
print(f"Position size: {shares} shares")
print(f"Dollar risk:   ${shares * abs(entry_price - stop_price):.2f}")
```

**장점**
*   재사용 가능하고 파라미터화된 모듈을 생성합니다. 거래당 하드코딩된 값은 없습니다.
*   레시피에 거래당 규모 조정과 함께 포트폴리오 과열(portfolio heat) 확인이 포함됩니다.

**단점**
*   규모 조정 모델은 인간의 검토가 필요합니다. Claude는 개인적인 낙폭 허용치를 알지 못합니다.

**가장 적합한 대상:** 일관되고 감사 가능한 리스크 프레임워크가 필요한 여러 전략을 동시에 실행하는 시스템 트레이더.

### 5. 라이브 신호 모니터 — 실시간 알림

출처: `roman-rr/trading-skills · trading-signals skill`


마지막 스킬은 연구에서 라이브 모니터링까지의 순환을 완성합니다. 백테스트를 실행하는 대신, 실시간으로 신호를 감시하고 조건이 충족되면 알림을 트리거합니다.

**스킬 레시피 (live-signal-monitor):**
1.  EODHD REST API를 통해 실시간 시세(real-time quote) 및 최근 종가(EOD bars) 가져오기
2.  메모리에 이동 창(rolling window) 바 유지
3.  새로운 바가 들어올 때마다 지표(indicators) 재계산
4.  신호 조건 평가 — 진입, 청산 또는 보유
5.  신호 트리거 시: 타임스탬프, 심볼, 가격, 지표 값 기록
6.  알림 전송 (인쇄 / 텔레그램 / 이메일)
7.  주문을 직접 실행하지 않음 — 신호만 출력

```python
import requests
import pandas as pd
import numpy as np
import time
from datetime import datetime
API_KEY = "YOUR_EODHD_KEY"
def fetch_live_quote(symbol: str) -> float:
    r = requests.get(
        f"https://eodhd.com/api/real-time/{symbol}",
        params={"api_token": API_KEY, "fmt": "json"}
    )
    return r.json().get("close", None)
def fetch_recent_eod(symbol: str, n: int = 30) -> pd.Series:
    r = requests.get(
        f"https://eodhd.com/api/eod/{symbol}",
        params={"api_token": API_KEY, "period": "d", "fmt": "json"}
    )
    df = pd.DataFrame(r.json()).tail(n)
    df["date"] = pd.to_datetime(df["date"])
    return df.set_index("date")["adjusted_close"].rename("close")
def compute_rsi(series, n=14):
    delta = series.diff()
    gain  = delta.clip(lower=0).rolling(n).mean()
    loss  = -delta.clip(upper=0).rolling(n).mean()
    return 100 - (100 / (1 + gain / loss))
symbol = "AAPL.US"
print(f"Monitoring {symbol} | {datetime.now():%Y-%m-%d %H:%M}")
while True:
    close_series = fetch_recent_eod(symbol, n=30)
    live_price   = fetch_live_quote(symbol) or close_series.iloc[-1]
    # Append live price as today's bar
    today = pd.Timestamp.now().normalize()
    close_series[today] = live_price
    rsi    = compute_rsi(close_series).iloc[-1]
    status = "OVERSOLD → BUY WATCH" if rsi < 30 else "OVERBOUGHT → SELL WATCH" if rsi > 70 else "NEUTRAL"
    print(f"[{datetime.now():%H:%M:%S}] {symbol} | ${live_price:.2f} | RSI: {rsi:.1f} | {status}")
    if rsi < 30:
        print(f"  🔔 ALERT: RSI oversold at {rsi:.1f} - review entry")
    time.sleep(60)
```

이를 텔레그램 봇과 결합하면 수동으로 차트를 감시할 필요 없이 24시간 내내 자동화된 신호 감지를 실행할 수 있습니다.

**장점**
*   폴링(polling) 방식은 무료 EODHD 티어에서도 작동하며, WebSocket 구독이 필요하지 않습니다.
*   스킬은 신호/실행 분리를 강제하여 의도치 않은 라이브 주문을 방지합니다.

**단점**
*   60초 폴링은 1분 미만 전략에는 적합하지 않습니다 (이러한 전략에는 EODHD의 WebSocket 피드를 사용하세요).

**가장 적합한 대상:** 전체 실행 시스템을 구축하지 않고 자동화된 신호 감지를 원하는 스윙(swing) 및 장중(intraday) 트레이더.

### 변화

Claude Code 스킬은 마법이 아닙니다. 이들은 모든 새로운 프로젝트에서 '맨땅에서부터 알아내는' 수고를 없애주는 구조화된 레시피입니다.

다섯 가지 스킬. 하나의 신뢰할 수 있는 데이터 계층. 병목 현상은 구현에서 본래의 자리로 이동합니다: 어떤 전략을 테스트할 가치가 있는지 결정하는 것.

대부분의 퀀트들은 아이디어가 부족한 것이 아닙니다. 실제로 작동하는 아이디어를 찾기 위해 충분히 빠르게 구현할 수 있는 여력이 부족할 뿐입니다.

그것이 바로 변화입니다.
