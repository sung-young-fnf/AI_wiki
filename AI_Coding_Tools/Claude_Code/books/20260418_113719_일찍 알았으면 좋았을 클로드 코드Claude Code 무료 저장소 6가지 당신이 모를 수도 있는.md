## 일찍 알았으면 좋았을 클로드 코드(Claude Code) 무료 저장소 6가지 (당신이 모를 수도 있는)

첫 번째 저장소는 클로드 코드(Claude Code)에 무의식을 부여합니다. 여러분의 세션 밑에서 실행되며, 관찰하고 기억을 구축한 다음 다음 프롬프트(prompt)에 지침을 속삭여주는 두 번째 에이전트(agent)입니다.

이들은 X(구 트위터)에서 흔히 공유되는 유행하는 저장소가 아니라, 매우 특정한 문제를 해결하기 때문에 독특하고 유용합니다. 대부분 클로드 코드의 한계에 좌절한 개발자들이 직접 개선하기 위해 만들었습니다.

어떤 저장소는 클로드가 작업을 마치거나 막혔을 때 사용자에게 전화할 수 있게 합니다. 다른 하나는 클로드가 프롬프트를 보기 전에 모호한 프롬프트를 수정합니다. 또 다른 저장소는 여러 클로드 인스턴스가 사용자의 개입 없이 에이전트 팀처럼 실시간으로 서로 메시지를 주고받을 수 있도록 합니다.

지난주에 시간을 내어 이들을 테스트했습니다. 다음은 클로드 코드를 독특한 방식으로 확장하는 6가지 저장소(repository)입니다.

### 1. Claude Subconscious

클로드 코드(Claude Code)는 세션(session) 간에 모든 것을 잊어버립니다. Claude Subconscious는 두 번째 에이전트를 실행하여 이 문제를 해결합니다. 이 에이전트는 세션을 관찰하고 파일을 읽으며 시간이 지남에 따라 기억을 구축합니다.

새로운 세션을 시작하면, 무의식 에이전트가 이전 세션에서 학습한 내용을 바탕으로 지침을 주입합니다. 이 에이전트는 도구 접근 권한을 가지며, 파일을 읽고, 코드베이스를 검색하고, 웹을 탐색할 수 있습니다.

하나의 에이전트 두뇌는 모든 프로젝트에서 공유됩니다. 세 개의 다른 저장소에서 작업하더라도, 모든 정보가 같은 무의식 에이전트에 통합됩니다.

#### 주요 기능 (Key Features)
*   세션 간에 지속적인 기억을 가진 백그라운드 에이전트 (agent)
*   실제 도구 접근 (파일 읽기, 웹 검색, 코드베이스 탐색)
*   여러 클로드 코드 세션 동시 지원
*   버전 관리를 위한 Git 기반 메모리 파일 시스템
*   제로-설정 (Zero-config) – API 키(API key)만 설정하면 됨

Letta가 자사의 코드 SDK(Software Development Kit) 데모로 구축했습니다.

### 2. CallMe

CallMe는 클로드 코드(Claude Code)가 사용자에게 전화할 수 있도록 하는 최소한의 플러그인(plugin)입니다.

이것은 긴 작업을 실행하고 터미널(terminal)을 계속 지켜보고 싶지 않을 때 유용합니다. 세션을 시작하고 자리를 비우면, 클로드가 주의가 필요할 때 사용자에게 전화합니다. 사용자는 결정 사항을 논의하고 전화를 끊으면 클로드는 계속 작업합니다.

이 플러그인은 다중 턴 대화(multi-turn conversations)를 지원합니다. 클로드가 질문을 하고, 사용자 답변을 기다린 다음, 추가 질문을 할 수 있습니다. 통화 중에도 웹 검색을 수행할 수 있습니다.

#### 주요 기능 (Key Features)
*   실제 전화 통화 (스마트폰, 스마트워치, 유선 전화에서 작동)
*   클로드와의 다중 턴 음성 대화 (multi-turn voice conversations)
*   도구 사용 결합 가능 – 클로드가 통화 중 웹 검색 가능
*   Telnyx(저렴) 또는 Twilio와 연동
*   Kokoro Docker 컨테이너를 통한 로컬 TTS(Text-to-Speech) 옵션

설정하려면 전화 서비스 제공업체 계정과 음성-텍스트 변환(speech-to-text)을 위한 OpenAI API(Application Programming Interface)가 필요합니다.

### 3. Claude Code Prompt Improver

Prompt Improver는 클로드(Claude)가 프롬프트(prompt)를 실행하기 전에 모호한 프롬프트를 가로채고 먼저 구체적인 질문을 합니다.

이것은 사용자가 제출하는 모든 프롬프트를 평가하는 훅(hook)입니다. 명확한 프롬프트는 그대로 통과되지만, 모호한 프롬프트는 클로드가 코드베이스를 스캔하고 문서를 확인하며 1~6개의 근거 있는 질문을 생성하는 '조사 단계'를 트리거합니다.

프롬프트에 문맥이 부족할 때만 거의 개입합니다.

#### 주요 기능 (Key Features)
*   프롬프트 실행 전 가로채서 평가
*   명확한 프롬프트는 오버헤드 없이 통과
*   모호한 프롬프트는 조사 + 1~6개의 명확화 질문 트리거
*   사용자 코드베이스를 사용하여 동적인 조사 계획 생성
*   기술 기반 아키텍처로 31% 토큰 감소 (token reduction)

이는 "테스트 추가"와 같은 모호한 요청을 입력할 때 유용합니다. 그러면 "어떤 모듈에 대한 테스트를 추가해야 할까요? 데이터베이스를 모의(mock)해야 할까요?"와 같은 명확화 질문이 트리거됩니다.

### 4. Claude Canvas

Claude Canvas는 tmux를 통해 대화형 터미널 창(interactive terminal panes)을 생성하여 클로드(Claude)에게 자체적인 디스플레이를 제공합니다.

클로드에게 이메일 초안 작성을 요청하면, 적절한 헤더와 함께 별도의 창에 렌더링됩니다. 사용자는 전체 구조를 확인한 다음 요청을 할 수 있습니다. 예를 들어, "제목을 더 짧게 만들어줘"라고 말하면 캔버스(canvas)가 실시간으로 업데이트됩니다.

캘린더 사용 사례의 경우, 회의 일정을 잡을 때 클로드가 양쪽 캘린더를 나란히 표시하고 상호 가능한 시간을 강조할 수 있습니다.

#### 주요 기능 (Key Features)
*   tmux를 통해 대화형 터미널 창 생성
*   양방향 상호작용 (클로드 업데이트, 사용자 클릭, 클로드 응답)
*   시각적 미리보기가 있는 이메일 초안 작성
*   가능한 시간이 강조된 캘린더 비교
*   클로드 코드(Claude Code)가 실행되는 모든 환경에서 작동

tmux가 필요하지만, 터미널에서 클로드 코드를 실행하고 있다면 아마 이미 설치되어 있을 것입니다.

### 5. Claude Peers MCP

여러 클로드 코드(Claude Code) 세션이 다른 프로젝트에서 실행될 때, 각 인스턴스는 격리되어 서로 대화할 수 없습니다.

Claude Peers MCP는 클로드 인스턴스들이 실시간으로 서로 메시지를 주고받을 수 있게 합니다. 한 세션에 "다른 클로드들은 무엇을 작업하고 있어?"라고 물으면 목록이 반환됩니다. 각 인스턴스는 작업 디렉토리, Git 저장소(repo), 현재 작업 요약을 보여줍니다. 그중 아무에게나 메시지를 보내면 즉시 응답합니다.

이 아키텍처는 로컬호스트(localhost)에서 SQLite 데이터베이스(SQLite database)와 함께 실행되는 브로커 데몬(broker daemon)입니다. 각 클로드 코드 세션은 MCP 서버(MCP server)를 생성하여 브로커에 등록하고 메시지를 폴링(polling)합니다.

#### 주요 기능 (Key Features)
*   클로드 코드 인스턴스 간 실시간 메시징
*   피어(peer) 탐색 (다른 인스턴스들이 무엇을 작업하는지 확인)
*   각 인스턴스의 현재 작업에 대한 자동 생성 요약
*   머신, 디렉토리 또는 Git 저장소별 범위 지정
*   첫 세션에서 브로커 자동 실행, 종료된 피어 정리

설정하려면 개발 채널을 로드하기 위해 특별한 플래그(flag)로 클로드를 실행해야 합니다.

### 6. Career-Ops

Career-Ops는 클로드 코드(Claude Code)를 기반으로 구축된 완전한 구직 시스템입니다. 제안 평가, 맞춤형 이력서 생성, 채용 포털 스캔, 지원서 추적을 위한 14가지 운영 모드(operational modes)를 제공합니다.

제작자는 이 시스템을 사용하여 740개 이상의 채용 제안을 평가하고 '인공지능 적용 책임자(Head of Applied AI)' 직책을 얻었습니다. 그는 스코어링 로직, 협상 스크립트, 증명 포인트 구조를 포함하여 자신이 사용했던 정확한 시스템을 오픈 소스로 공개했습니다.

직무 기술서(Job Description, JD)를 붙여넣으면 Career-Ops는 전체 파이프라인(pipeline)을 실행합니다. 요구 사항을 추출하고, 10가지 차원(A-F 등급)에 걸쳐 적합도를 평가하며, JD 키워드가 주입된 ATS(Applicant Tracking System) 최적화 PDF를 생성하고, 이를 추적기에 추가합니다.

#### 주요 기능 (Key Features)
*   14가지 운영 모드 (스캔, PDF, 배치, 지원, 심층 연구 등)
*   제안별 10가지 차원 스코어링 (A-F 등급)
*   각 JD에 맞춤화된 ATS 최적화 PDF 생성
*   45개 이상의 기업 사전 구성 (Anthropic, OpenAI, Retool 등)
*   병렬 클로드 워커(worker)를 사용한 배치(batch) 처리
*   파이프라인 관리를 위한 Go 기반 터미널 대시보드

클로드(Claude)는 평가하고 추천하며, 사용자는 결정하고 행동합니다. 즉, 설계상 휴먼 인 더 루프(human-in-the-loop) 방식입니다.

### 최종 결론

이 저장소들은 공식 릴리스에서 다루지 않은 방향으로 클로드 코드(Claude Code)를 확장합니다.

무의식 에이전트(agent)와 피어(peer) 메시징 기능은 클로드 코드 세션이 지속적으로 기억하고 조정될 수 있다는 점에서 놀라웠는데, 비록 이들이 실험적인 플러그인(plugin)이지만 말입니다.

하지만 몇 달 안에 이들 중 일부 기능이 핵심 기능으로 출시될 가능성이 높습니다.

CallMe과 Canvas는 워크플로 문제를 해결하며, Prompt Improver는 기본적으로 제공되어야 할 기능이고, Career-Ops는 예외적인 경우입니다.

저는 앞으로도 이와 같은 저장소들을 계속 테스트하고 여러분의 시간을 투자할 가치가 있는 것들을 공유할 것입니다.

만약 여러분이 주목할 만한 다른 클로드 코드 저장소를 발견했다면 댓글로 공유해 주세요.
