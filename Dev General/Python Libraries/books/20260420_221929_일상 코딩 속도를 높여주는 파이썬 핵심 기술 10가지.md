### 1. 값비싼 함수 호출 캐싱하기 (로직 변경 없이)

여러분은 동일한 함수를 동일한 입력값으로 여러 번 호출하고 있을 수 있습니다. 다만 그것을 깨닫지 못할 뿐입니다.

불필요한 연산 주기를 낭비하지 마세요.

```python
from functools import lru_cache

@lru_cache(maxsize=None)
def expensive_function(x):
    print("Computing...")
    return x * x

print(expensive_function(10))
print(expensive_function(10))  # Instant
```

첫 번째 호출은 느립니다.
두 번째 호출은 거의 무료에 가깝습니다.

사실: 실제 API와 데이터 파이프라인에서 캐싱(caching)만으로도 런타임(runtime)을 70% 이상 줄일 수 있습니다.

### 2. 어떤 스크립트든 30초 만에 CLI 도구로 만들기

값을 하드코딩(hardcoding)하는 것을 멈추세요. 스크립트를 즉시 재사용 가능하게 만드세요.

```python
import argparse

parser = argparse.ArgumentParser()
parser.add_argument("--name", required=True)
args = parser.parse_args()

print(f"Hello, {args.name}")
```

다음과 같이 실행하세요:

`python script.py --name Ali`

이제 여러분의 '간단한 스크립트'는 팀 전체가 사용할 수 있는 도구가 되었습니다.

### 3. 프로파일링(Profiling) 먼저, 최적화는 나중에 (올바른 방법)

대부분의 개발자들은 무엇이 느린지 추측합니다. 그리고 대개는 틀립니다.

```python
import cProfile

def slow_function():
    total = 0
    for i in range(10**6):
        total += i
    return total

cProfile.run("slow_function()")
```

이것은 시간이 정확히 어디에 소요되는지 보여줍니다.

규칙: 프로파일링(profiling)하기 전에는 절대 최적화하지 마세요.
사람들이 0.2초 걸리는 것을 최적화하는 데 몇 시간을 낭비하는 이유가 바로 이것입니다.

### 4. `__slots__`를 사용하여 메모리 사용량 절반으로 줄이기

수천 개의 객체(object)를 생성하는 경우, 이 기술은 중요합니다.

```python
class Normal:
    def __init__(self, x, y):
        self.x = x
        self.y = y

class Optimized:
    __slots__ = ('x', 'y')
    def __init__(self, x, y):
        self.x = x
        self.y = y
```

`__slots__`는 동적인 속성 저장 공간을 제거합니다.

더 적은 메모리 사용. 더 빠른 접근.

저는 데이터 집약적인 애플리케이션에서 이 방법으로 메모리 사용량을 40~60% 줄이는 것을 보았습니다.

### 5. 코드를 동적으로 실행하기 (하지만 안전하게)

때로는 유연성이 필요합니다. 또 다른 `if-else`문이 난무하는 복잡한 코드가 아니라요.

```python
def run_operation(op, a, b):
    operations = {
        'add': lambda x, y: x + y,
        'mul': lambda x, y: x * y
    }
    return operations[op](a, b)

print(run_operation('mul', 3, 5))
```

조건문보다 깔끔하고, 확장이 더 용이합니다.

### 6. RAM을 과부하 시키지 않고 대용량 파일 읽기

만약 여전히 대용량 파일에 `read()` 메서드를 사용하고 있다면... 이야기를 좀 해야겠습니다.

```python
def process_large_file(path):
    with open(path, 'r') as f:
        for line in f:
            yield line.strip()

for line in process_large_file('big_file.txt'):
    pass
```

이 방식은 파일을 한 줄씩 스트리밍(streaming)합니다.

사실: 이 접근 방식은 시스템 메모리보다 큰 파일도 처리할 수 있습니다.

### 7. 전문가처럼 코드 실행 시간 측정하기

더 이상 `print("start")`를 사용하지 마세요.

```python
import time

start = time.perf_counter()

# 여기에 코드 작성
sum(range(10**6))

end = time.perf_counter()

print(f"Execution time: {end - start:.6f} seconds")
```

`perf_counter()`는 마이크로 최적화(micro-optimization)에 충분히 정밀합니다.

### 8. 모든 것에 컨텍스트 관리자(Context Manager) 사용하기

만약 리소스(resource)를 수동으로 열고 닫고 있다면, 버그를 초래할 수 있습니다.

직접 만들어 보세요:

```python
class Timer:
    def __enter__(self):
        import time
        self.start = time.time()
        return self

    def __exit__(self, *args):
        import time
        print(f"Took {time.time() - self.start:.4f} seconds")
with Timer():
    sum(range(10**6))
```

더 깔끔한 코드. 자동 정리. 더 적은 실수.

### 9. CPU 작업을 효율적으로 병렬 처리하기

스레드(thread)는 CPU 바운드(CPU-bound) 작업에 도움이 되지 않습니다. 프로세스(process)를 사용하세요.

```python
from multiprocessing import Pool

def square(x):
    return x * x

with Pool(4) as p:
    results = p.map(square, range(10))
    print(results)
```

이제 여러분의 CPU가 실제로 여러분을 위해 일할 것입니다.

### 10. `inspect` 모듈로 더 스마트하게 디버깅하기

함수가 런타임(runtime)에 어떻게 작동하는지 궁금했던 적이 있나요?

```python
import inspect

def debug():
    frame = inspect.currentframe()
    print(inspect.getframeinfo(frame))

debug()
```

이것은 파일 이름, 줄 번호, 그리고 컨텍스트(context)를 제공합니다.

마치 코드에 X-레이 투시경을 켜는 것과 같습니다.
