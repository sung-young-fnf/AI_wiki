# 어떤 프로젝트든 시작하기 전에 설치하는 7가지 파이썬 라이브러리

파이썬 개발 경험이 어느 정도 쌓인 분들이라면 문법에 익숙하고, 코드를 배포해 본 적도 있으며, 새벽까지 디버깅을 하느라 고생해 본 적도 있을 것입니다. 하지만 새로운 프로젝트를 시작할 때마다 빈 폴더와 가상 환경을 마주하며 "분명 더 스마트한 방법이 있을 텐데"라는 생각을 하곤 합니다.

4년 이상 전문적으로 파이썬 개발을 하며 얻은 교훈이 하나 있습니다. **프로젝트 시작 후 처음 10분 동안 어떤 라이브러리를 설치하느냐가 향후 3개월의 고통 수치를 결정한다**는 점입니다. 

비즈니스 로직을 단 한 줄도 쓰기 전에 제가 반드시 설치하는 7가지 라이브러리를 소개합니다. 이들은 유행을 타거나 과대평가된 것이 아닌, 압도적인 효율을 제공하는 도구들입니다.

---

### 1. rich — 출력문(Print)의 품격

여전히 디버깅을 위해 단순한 `print()` 함수만 사용하고 있다면, 이는 구석기 시대의 방식과 다름없습니다. `rich`는 여러분의 터미널을 개발자 전용 대시보드로 탈바꿈시켜 줍니다.

*   **주요 장점:** 가독성 높은 로그로 디버깅 시간을 대폭 단축하며, 단순한 텍스트 나열이 아닌 JSON, 트레이스백(tracebacks), 진행 표시줄(progress bars), 트리 구조 등을 시각화할 수 있습니다. 
*   개발자의 업무 시간 중 약 35~40%가 디버깅에 소요됩니다. `rich`는 그 시간을 훨씬 효율적으로 만들어 줍니다.

### 2. pydantic — 데이터 유효성 검증의 자동화

거의 모든 버그는 잘못된 입력 데이터에서 시작됩니다. `pydantic`은 데이터가 시스템에 유입되어 혼란을 일으키기 전에 구조를 강제합니다.

*   **주요 장점:** 자동 타입 검증(Type validation), 자기 문서화 모델(Self-documenting models), 그리고 문제가 발생했을 때 즉시 명확하게 오류를 알리는 기능을 제공합니다.
*   FastAPI가 강력한 이유이기도 하며, 스크립트나 데이터 파이프라인, CLI 도구 제작 시 API나 설정값, 사용자 입력을 다룬다면 반드시 사용해야 합니다.

### 3. python-dotenv — 코드에서 보안 정보 분리

초보자는 API 키를 코드에 직접 적어 넣고(hardcoding), 전문가는 이를 후회합니다. 보안 정보는 절대 코드와 함께 저장되어서는 안 됩니다.

*   **주요 장점:** 보안 정보가 Git에 유출되는 것을 방지하고, 로컬·도커(Docker)·CI 등 환경에 관계없이 일관된 배포를 가능하게 합니다.
*   실제로 자격 증명 유출은 운영 환경 사고의 가장 큰 원인 중 하나입니다. 이 라이브러리는 거의 아무런 노력 없이 이를 방지해 줍니다.

### 4. typer — 고성능 CLI 도구 제작

제대로 된 프로젝트라면 결국 명령줄 인터페이스(CLI)가 필요하기 마련입니다. 기존의 `argparse`가 기능에 충실하다면, `typer`는 사용법이 비현실적으로 쉽습니다.

*   **주요 장점:** 도움말 문서 자동 생성, 타입 힌트(Type hints)를 통한 CLI 유효성 검증 기능을 제공합니다.
*   내부용 도구를 만들 때도 유용하며, 나중에 기능을 확장할 때 과거의 자신에게 고마워하게 될 것입니다.

### 5. loguru — 스트레스 없는 로깅

파이썬의 기본 로깅 모듈(logging)은 설정이 꽤 까다롭습니다. 반면 `loguru`는 로깅 시스템이 처음부터 갖췄어야 할 이상적인 모습을 보여줍니다.

*   **주요 장점:** 복잡한 설정이 전혀 필요 없으며, 기본적으로 정형화된 로그를 제공하고 파일 순환(rotation) 기능을 간단하게 구현할 수 있습니다.
*   좋은 로그는 시스템이 잘 돌아갈 때가 아니라, 모든 것이 잘못되고 있는 절체절명의 순간에 빛을 발합니다.

### 6. tenacity — 재시도(Retry) 로직의 정석

네트워크는 불안정하고, 데이터베이스는 가끔 응답하지 않으며, API는 실패할 수 있습니다. `tenacity`는 이러한 상황에서 품격 있는 재시도 메커니즘을 제공합니다.

*   **주요 장점:** 일시적인 오류로 인한 오작동을 제거하고, 지수 백오프(exponential wait) 등을 통해 실패 상황을 예측 가능하게 관리합니다.
*   네트워크를 통해 무언가를 주고받는 프로젝트라면 필수적인 도구입니다.

### 7. watchdog — 파일 시스템 변화 감지

의외로 과소평가받는 라이브러리인 `watchdog`은 파일 시스템의 이벤트를 실시간으로 감시하고 반응하게 해줍니다.

*   **주요 장점:** 설정 파일의 자동 재로드, 파일 생성 시 파이프라인 트리거, 폴링(polling) 방식이 아닌 이벤트 기반의 개발 도구 구축 등에 활용됩니다.
*   한 번 반응형 시스템을 구축해 보면, 주기적으로 상태를 확인하는 기존 방식이 얼마나 원시적인지 깨닫게 될 것입니다.

---

### 결론

많은 개발자가 프레임워크에 집착할 때, 전문가는 기초(foundations)에 집중합니다. 위에서 소개한 라이브러리들은 단순히 시간을 절약해 주는 것에 그치지 않고, 시스템이 압박을 받는 상황에서도 코드가 견고하게 동작하도록 체질을 바꿔줍니다.

이 일곱 가지를 먼저 설치하고 시작한다면, 여러분의 프로젝트는 개발자의 인내심보다 훨씬 더 오랫동안 건강하게 유지될 것입니다.
