3년 전, 저는 오후 3시 17분에 프로덕션 스크립트(production script)를 뚫어져라 보고 있었습니다.

1,200줄짜리 코드. 신(God) 함수. 여기저기 복사-붙여넣기된 로직. 거짓말하는 로깅. 조용히 실패하는 재시도 로직.

가장 최악은요?
제가 짠 코드였다는 겁니다.

그날 저는 스크립트를 다시 쓰지 않았습니다. 파이썬(Python)을 작성하는 방식을 완전히 바꿨죠.

이어지는 내용은 제 작업 흐름을 조용히 바꾼 바로 그 라이브러리들입니다.
초보자용 장난감이 아닙니다. 트위터(Twitter)에서 유행하는 과대광고도 아닙니다.
코드가 당신을 몇 번 괴롭힌 후에야 비로소 발견하게 될 도구들이죠.

이제 자세히 살펴보겠습니다.

## 1. attrs — 클래스가 더 이상 상투적인(boilerplate) 공장이 아니게 되었을 때

저는 예전에는 자동 조종 로봇처럼 데이터 중심의 클래스를 작성하곤 했습니다.
`__init__`, 유효성 검사(validation), `repr`, 비교(comparisons). 계속해서 반복적으로요.

`attrs`는 그 대부분이 불필요한 의식(ceremony)이었음을 깨닫게 해주었습니다.

**왜 모든 것을 바꿨는가**
*   선언적이고 읽기 쉬운 모델
*   내장된 유효성 검사
*   마법 같은 요소(magic) 없고, 런타임(runtime)에 놀랄 일 없음

**실제 사용 예시**
```python
import attr

@attr.define
class JobConfig:
    retries: int = attr.field(default=3)
    timeout: float = attr.field(default=5.0)
    critical: bool = False
```
그게 다입니다.
사용자 정의 `init`도 없고, 방어적인 코드도 없습니다. 오직 의도(intent)만 존재합니다.

## 2. tenacity — 당신에게 거짓말하지 않는 재시도 로직

`tenacity` 이전에는 제 재시도 로직은 `try/except` 루프와 희망이 뒤섞인 난장판이었습니다.

희망은 전략이 아닙니다.

**왜 중요한가**
*   지수 백오프(exponential backoff)
*   세밀한 제어
*   읽기 쉬운 실패 동작

**실제 사용 예시**
```python
from tenacity import retry, stop_after_attempt, wait_exponential

@retry(stop=stop_after_attempt(5), wait=wait_exponential())
def fetch_remote_config():
    return call_flaky_service()
```
이것 하나만으로도 저는 프로덕션(production) 환경에서의 특정 종류의 버그를 완전히 제거할 수 있었습니다.

## 3. diskcache — RAM은 비싸고 API는 느리기 때문에

캐싱(Caching)은 선택 사항이 아닙니다.
하지만 레디스(Redis)는 종종 과도합니다(overkill).

`diskcache`가 딱 알맞은 지점(sweet spot)을 찾았습니다.

**왜 신뢰하는가**
*   스레드 안전(Thread-safe)
*   디스크 기반(Disk-backed)
*   로컬 자동화에 말도 안 되게 빠름

**실제 사용 예시**
```python
from diskcache import Cache

cache = Cache("./cache")

@cache.memoize(expire=3600)
def expensive_call(x):
    return compute(x)
```
하나의 데코레이터(decorator).
제 스크립트는 아마추어처럼 API를 계속 호출하는 것을 멈췄습니다.

## 4. anyio — 특정 방식에 얽매이지 않는 비동기 처리

비동기 파이썬(Async Python)은 강력합니다.
비동기 파이썬은 또한 혼란스럽습니다.

`anyio`는 제가 한쪽을 선택해야 하는 상황에서 벗어나게 해주었습니다.

**왜 저평가되는가**
*   `asyncio` 및 `trio`와 함께 작동
*   구조화된 동시성(Structured concurrency)
*   예측 가능한 취소

**실제 사용 예시**
```python
import anyio

async def worker():
    await anyio.sleep(1)

anyio.run(worker)
```
깔끔하고. 이식성이 좋고(Portable). 미래에도 유효합니다(Future-proof).

## 5. watchdog-events — 폴링(polling) 지옥 없는 파일 자동화

파일 기반 워크플로(workflow)는 어디에나 있습니다.
로그(log). 드롭(drop). 파이프라인(pipeline).

수동으로 폴링(polling)하는 것은 당황스러운 일입니다.

**왜 이 라이브러리를 계속 설치해두는가**
*   이벤트 기반(Event-driven)
*   낮은 CPU 사용량
*   크로스 플랫폼(cross-platform)에서 작동

**실제 사용 예시**
```python
from watchdog.events import FileSystemEventHandler

class Handler(FileSystemEventHandler):
    def on_created(self, event):
        process_file(event.src_path)
```
이벤트 기반으로 한 번 전환하면, 다시는 이전으로 돌아가지 못할 겁니다.

## 6. structlog — 진실을 말해주는 로깅

기존 로깅(traditional logging)은 누락(omission)으로 인해 거짓말을 합니다.
컨텍스트(context)를 숨기고, 구조(structure)를 숨깁니다.

`structlog`가 이 문제를 해결했습니다.

**왜 뛰어난가**
*   JSON 우선
*   컨텍스트 인지(Context-aware)
*   관측 가능성 도구(observability tools)와 완벽하게 연동

**실제 사용 예시**
```python
import structlog

log = structlog.get_logger()

log.info("job_started", job_id=42, priority="high")
```
로그(log)는 더 이상 노이즈(noise)가 아니었습니다.
데이터(data)가 되었죠.

## 7. schedule — 정신적 부담 없는 Cron

크론(Cron)은 강력합니다.
크론은 또한 다루기 어렵습니다(hostile).

`schedule`은 제게 명확성을 주었습니다.

**왜 여전히 사용하는가**
*   읽기 쉬움
*   테스트 가능
*   내부 자동화에 완벽함

**실제 사용 예시**
```python
import schedule
import time

schedule.every(10).minutes.do(run_job)

while True:
    schedule.run_pending()
    time.sleep(1)
```
YAML도 없고. 시스템 수준의 부작용도 없습니다. 오직 파이썬(Python)뿐입니다.

## 8. msgspec — 초고속 직렬화(serialization)

JSON은 느립니다.
Pickle은 안전하지 않습니다.
`msgspec`은 성숙한(grown-up) 선택지입니다.

**왜 성능 엔지니어(performance engineer)들이 좋아하는가**
*   극도로 빠름
*   타입 기반(Type-driven)
*   예측 가능한 메모리 사용량

**실제 사용 예시**
```python
import msgspec

class Event(msgspec.Struct):
    id: int
    payload: str

encoded = msgspec.json.encode(Event(1, "ok"))
```
이것은 제게 전체 직렬화 계층(serialization layers)을 대체했습니다.

## 9. pluggy — 확장 가능한 시스템(extensible systems)의 비밀

`pytest`가 무한히 확장 가능하면서도 어떻게 깔끔함을 유지하는지 궁금해 본 적이 있습니까?

`pluggy` 덕분입니다.

**왜 강력한가**
*   명시적인 플러그인 계약(plugin contracts)
*   런타임(runtime) 해킹(hacks) 없음
*   깔끔하게 확장됨

**실제 사용 예시**
```python
import pluggy

hookspec = pluggy.HookspecMarker("app")
hookimpl = pluggy.HookimplMarker("app")
```
이것이 붕괴하지 않고 발전하는 시스템을 구축하는 방법입니다.

### 하나의 값진 진실

대부분의 사람들이 말해주지 않는 것이 있습니다:

깔끔한 코드(clean code)는 구문(syntax)에 대한 것이 거의 아닙니다.
그것은 초기에 올바른 추상화(abstractions)를 선택하는 것에 대한 것입니다.

라이브러리가 당신을 더 똑똑하게 만들지는 않습니다.
당신의 실수를 더 작게 만들 뿐입니다.

그리고 그것이 진정한 초능력(superpower)입니다.
