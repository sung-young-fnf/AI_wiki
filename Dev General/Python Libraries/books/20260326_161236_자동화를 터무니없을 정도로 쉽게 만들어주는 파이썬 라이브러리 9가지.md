# 자동화를 터무니없을 정도로 쉽게 만들어주는 파이썬 라이브러리 9가지

저는 파이썬(Python) 코드를 충분히 오랫동안 작성해 오면서 같은 작업을 두 번 반복하는 것에 가벼운 알레르기 반응을 일으키게 되었습니다. 어떤 작업이 반복된다면, 그것은 자동화될 것입니다. 반복될 가능성이 있다면, 그것은 자동화될 것입니다. 한 달에 한 번 발생하지만 30초 동안 저를 괴롭히는 일이라면… 맞습니다, 자동화될 것입니다.

재미있는 점은 무엇일까요? 대부분의 사람들은 모두가 이야기하는 똑같은 5가지 라이브러리로 이 작업을 시도합니다. 그것들은 작동합니다. 괜찮습니다. 하지만 진정한 '레버리지(leverage)'가 있는 곳은 아닙니다.

아래에는 '아, 이건 나중에 해야지'라는 생각을 '커피가 식기 전에 끝냈다'로 조용히 바꿔주는, 덜 알려진 파이썬 라이브러리 9가지가 있습니다. 저는 여기에 소개된 모든 패턴을 실제 프로젝트에서 사용해 보았습니다. 이 중 일부는 몇 시간을 절약해 주었고, 그 중 하나는 새벽 2시에 발생할 수 있었던 프로덕션 장애(production incident)로부터 저를 구했습니다. (어떻게 아는지 궁금하시면 물어보세요.)

군더더기 없이, 날카로운 도구들만을 소개합니다.

## 1. pexpect — 마법사처럼 상호작용 프로그램 자동화하기

터미널(terminal)에서 계속 질문을 던지는 것을 자동화하려고 해본 적이 있으신가요?
SSH, FTP, 설치 프로그램, 제대로 스크립트화하기를 거부하는 이상한 레거시 CLI(Command Line Interface) 말입니다.

바로 `pexpect`가 활약하는 영역입니다.

```python
import pexpect

child = pexpect.spawn('ssh user@server.com')
child.expect('password:')
child.sendline('my_secret_password')
child.expect('$')
child.sendline('uptime')
child.interact()
```

**이것이 왜 금과 같은가:** 단순히 명령이 아니라 프로그램과의 '대화'를 스크립트화할 수 있습니다. 저는 이 라이브러리를 사용하여 세 가지 상호작용 프롬프트(interactive prompt)가 있고 문서화가 전혀 없는 배포 프로세스(deployment process)를 자동화한 적이 있습니다. 마치 속임수를 쓰는 것 같았습니다.

**전문가 팁:** 이 라이브러리를 `cron`과 결합하면, 갑자기 '수동 서버 의식'이 '지루한 스케줄링된 작업(scheduled job)'으로 변모합니다.

## 2. watchfiles — 파일 변경 사항에 실시간으로 반응하기 (가장 빠른 방법)

대부분의 사람들은 `watchdog`을 사용합니다. `watchdog`도 작동합니다. 하지만 `watchfiles`는 더 빠르고, 더 간단하며, 내부적으로 러스트(Rust) 기반으로 구축되었습니다.

```python
from watchfiles import watch

for changes in watch('./incoming'):
    print("Something changed:", changes)
    # Trigger your automation here
```

**사용 사례:**
*   폴더에 추가된 파일을 자동 처리
*   변경 사항 발생 시 에셋(asset) 재구축
*   보고서가 나타날 때 스크립트 실행

저는 이 라이브러리를 사용하여 데이터 덤프(data dump)가 도착하는 즉시 자동으로 유효성 검사를 수행합니다. 폴링(polling)도, 기다림도 없습니다. 오직 반응뿐입니다.

## 3. plumbum — 쉘 스크립팅(Shell Scripting)을 즐겁게

쉘 명령(shell command)을 주로 호출하는 파이썬 코드를 작성해 본 적이 있다면… 당신은 충분히 고통받았을 것입니다.

`plumbum`은 쉘 명령을 파이썬 객체(object)처럼 느끼게 만듭니다.

```python
from plumbum import local

ls = local["ls"]
grep = local["grep"]

print((ls["-a"] | grep[".py"])())
```

**무엇이 다른가:** 파이프(pipe), 리디렉션(redirection), 조합(composition)을 코드를 문자열 연결(string-concatenation)의 악몽으로 만들지 않고도 사용할 수 있습니다.

저는 이 라이브러리를 사용하여 200줄짜리 Bash 스크립트를 약 40줄의 가독성 좋은 파이썬 코드로 대체했습니다. 미래의 제가 고맙다는 인사를 보냈을 정도입니다.

## 4. python-fire — 어떤 스크립트든 10초 만에 CLI로 만들기

당신이 '플래그(flag) 하나만 더 필요해'라는 이유로 계속해서 수정하는 스크립트가 있나요?
멈추세요. `fire`를 사용하세요.

```python
import fire

def resize(path, width=800, height=600):
    print(f"Resizing {path} to {width}x{height}")

if __name__ == "__main__":
    fire.Fire(resize)
```

이제 다음을 얻게 됩니다:

```bash
python script.py ./images --width=1024 --height=768
```

**이것이 왜 위험한가:** 모든 것을 CLI(Command Line Interface)로 바꾸기 시작할 것입니다. 솔직히 말해서, 그것은 좋은 문제입니다.

## 5. tenacity — 제정신을 잃지 않고 재시도 로직(Retry Logic) 구현하기

네트워크 호출(network call)은 실패합니다. API(Application Programming Interface)는 타임아웃됩니다. 데이터베이스는 깜빡거립니다.
취약한 스크립트와 프로덕션(production) 환경에 배포할 수 있는 스크립트의 차이는 바로 재시도(retry) 여부입니다.

```python
from tenacity import retry, stop_after_attempt, wait_fixed

@retry(stop=stop_after_attempt(5), wait=wait_fixed(2))
def fetch_data():
    print("Trying...")
    raise Exception("Temporary failure")
```

**왜 이것이 중요한가:** 실제 자동화는 실제 세상에서 살아갑니다. 그리고 실제 세상은 불안정합니다(flaky).

저는 이 단 하나의 라이브러리가 '한 번 실패하면 복구되지 않는' 버그(bug) 유형을 통째로 제거하는 것을 보았습니다.

## 6. streamz — 이벤트에 반응하는 데이터 파이프라인(Data Pipeline) 구축하기

`streamz`를 '스크립트가 아닌 흐름(flow)을 위한 자동화'라고 생각해보세요.

```python
from streamz import Stream

source = Stream()

source.map(lambda x: x * 2).sink(print)

source.emit(10)
source.emit(20)
```

이제 상상해 보세요:
*   파일이 도착하고
*   메시지가 들어오고
*   센서가 업데이트되고
*   로그가 스트리밍되는 상황을 말입니다.

당신은 파이프라인을 실행하는 것이 아닙니다. 당신은 파이프라인에 데이터를 공급하고, 파이프라인은 반응합니다.

**이것이 빛을 발하는 곳:** 실시간 자동화(real-time automation), 모니터링(monitoring), 이벤트 기반 시스템(event-driven system)입니다. 보통 지저분한 `while-true` 루프(loop)로 귀결되는 작업들 말입니다.

## 7. diskcache — 스크립트가 정보를 (제대로) 기억하게 하기

모든 것을 잊어버리는 자동화는 느린 자동화입니다.

`diskcache`는 거의 노력 없이 빠르고 영구적인 캐시(cache)를 제공합니다.

```python
from diskcache import Cache

cache = Cache('./mycache')

@cache.memoize()
def expensive_call(x):
    print("Computing...")
    return x * x

print(expensive_call(10))
print(expensive_call(10))  # This one is instant
```

**이것이 왜 과소평가되는가:** 캐싱(caching)은 '10분 걸리던 작업'을 '10초 만에 실행되는 작업'으로 바꿉니다. 대규모 환경(at scale)에서는 이것이 '괜찮은 스크립트'와 '핵심 인프라스트럭처(core infrastructure)'의 차이입니다.

## 8. pywinauto — 실제 Windows 애플리케이션 자동화하기

모든 것에 API(Application Programming Interface)가 있는 것은 아닙니다. 어떤 것들은 오직… 버튼(button)만 가지고 있습니다.

`pywinauto`는 인내심 있는 로봇처럼 Windows GUI(Graphical User Interface)를 스크립트화할 수 있도록 해줍니다.

```python
from pywinauto import Application

app = Application().start("notepad.exe")
app.Notepad.menu_select("Help->About Notepad")
app.Notepad.AboutNotepad.OK.click()
```

네, 정말입니다.
저는 이 라이브러리를 사용하여 REST, JSON은 물론, 아마도 전기가 발명되기 이전부터 존재했을 레거시 회계 도구(legacy accounting tool)를 자동화한 적이 있습니다.

창(window)이 있는 것이라면, 자동화할 수 있습니다.

## 9. boltons — 진지한 스크립트를 위한 사라진 도구 상자(Toolbox)

`boltons`는 캐싱(caching), 이터유틸(iterutils), 딕트유틸(dictutils), 타임유틸(timeutils) 등 놀랍도록 실용적인 유틸리티(utility) 모음입니다.

```python
from boltons.iterutils import chunked

data = list(range(10))
print(list(chunked(data, 3)))
```

**이것이 제가 작성하는 모든 코드에 슬며시 들어가는 이유:** 이것은 1년 동안 조용히 몇 시간을 낭비하게 만드는 작고 지루한 문제들을 제거합니다. 자동화는 단지 큰 성공만을 의미하는 것이 아닙니다. 그것은 수많은 작은 불편함("death by a thousand paper cuts")을 피하는 것입니다.
