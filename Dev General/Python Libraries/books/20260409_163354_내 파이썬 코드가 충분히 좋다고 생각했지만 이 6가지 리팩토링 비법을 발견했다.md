### 내 코드가 나를 배신한 날

지난겨울 새벽 2시쯤, 나는 몇 달 전 직접 작성하고 'final_final_version2.py'라고 자랑스럽게 이름 붙였던 파이썬 스크립트(Python script)를 멍하니 쳐다보고 있었다. 작은 클라이언트를 위한 송장(invoice) 생성을 자동화하는 간단한 작업이었다. 하지만 아주 작은 기능 하나를 업데이트하려던 순간, 모든 것이 잠 못 드는 비둘기가 쌓은 젠가(Jenga) 탑처럼 무너져 내렸다.

함수들은 다섯 가지 다른 일을 처리하고 있었고, `x1`, `tmp`, `final_result_really` 같은 변수들이 나를 괴롭혔다. 나는 속삭였다:

"누가 이 쓰레기 코드를 썼지?"

그리고 `git blame`을 확인했다.

나였다.

그날 밤, 나는 결심했다. 무언가를 자동화하려면 내 코드가 미래의 나에게서 살아남아야 한다고. 그때 나는 내 파이썬 코드를 "일단 작동은 하지만 (건드리지 마)"에서 "젠장, 정말 깔끔하군"으로 바꿔놓은 여섯 가지 리팩토링(refactoring) 비법을 발견했다.

### 1. 단일 책임 스크립트 (Single Responsibility Scripts)

대부분의 자동화 스크립트(automation script)는 작게 시작하지만, 어느새 그렇지 않게 된다. 나는 한 파일 안에서 데이터를 스크래핑(scraping)하고, 정리하고, CSV를 생성하고, 이메일로 보내는 400줄짜리 괴물 스크립트를 작성하곤 했다.

지금은? 파일당 하나의 작업. 협상 불가.

```python
# bad: does everything
def process_data():
    data = fetch_data()  # scrape
    clean = clean_data(data)  # clean
    export_csv(clean)  # save
    send_email(clean)  # notify

# good: split tasks
def fetch_data(): ...
def clean_data(data): ...
def save_file(data): ...
def notify_user(file_path): ...
```

이것이 중요한 이유: 갑자기 테스트(testing)가 쉬워졌다. 오류가 명확해졌고, 코드를 다시 작성할 필요 없이 코드 조각들을 재사용할 수 있게 되었다.

### 2. 딕셔너리를 여기저기 넘기는 것을 멈춰라

나는 예전에 거대한 딕셔너리(dictionary)를 뜨거운 감자처럼 함수들 사이로 넘기곤 했다. 그러다 어느 날, 딕셔너리 중 하나에서 키(key)를 잊어버렸고, 존재하지 않는 'email' 키를 디버깅(debugging)하는 데 3시간을 허비했다.

이제 나는 작고 명시적인 데이터 구조를 사용한다.

```python
from dataclasses import dataclass

@dataclass
class Invoice:
    client: str
    amount: float
    due_date: str

def send_invoice(invoice: Invoice):
    print(f"Sending invoice to {invoice.client} for {invoice.amount}")
```

과하다고 느껴지는가? 그렇지 않다. 이것이 당신의 코드가 취약함을 멈추고 신뢰할 수 있게 되는 방법이다.

### 3. 반복문(Loop)을 제너레이터(Generator)로 대체하기 (메모리 지옥과 작별)

내 자동화 스크립트 중 하나는 2GB에 달하는 거대한 로그 파일(log file)을 처리할 때 항상 충돌하곤 했다. 왜 그랬을까? 모든 것을 리스트(list)에 로드(load)했기 때문이다.

```python
# before: kills RAM
logs = [line.strip() for line in open("system.log")]

# after: generator magic
def read_logs(path):
    with open(path) as f:
        for line in f:
            yield line.strip()
```

전문가 팁: "효율성은 일을 올바르게 하는 것이고, 효과성은 올바른 일을 하는 것이다." - 피터 드러커(Peter Drucker)
제너레이터(Generator)는 이 둘 모두를 만족시킨다.

### 4. 전략 패턴(Strategy Pattern)으로 반복적인 코드 제거하기 (스크립트에서도 가능)

나는 JSON, CSV, YAML 세 가지 종류의 보고서 형식(report format)을 처리하는 스크립트를 가지고 있었다. 내 코드는 `if-else` 스파게티 접시 같았다.

```python
def export_report(data, format):
    if format == 'csv':
        ...
    elif format == 'json':
        ...
    elif format == 'yaml':
        ...
```

지금은:

```python
class ExportStrategy:
    def export(self, data): pass

class CSVExport(ExportStrategy):
    def export(self, data): ...

class JSONExport(ExportStrategy):
    def export(self, data): ...

def export_report(data, strategy: ExportStrategy):
    strategy.export(data)
```

갑자기 형식(format)을 추가하는 것이 새벽 3시에 함수 전체를 리팩토링(refactor)하는 일이 아니라 5줄짜리 작업이 되었다.

### 5. 원시 데이터(Raw Data)를 반환하지 마라. 액션(Action)을 반환하라.

이전에는 모든 함수가 딕셔너리(dictionary)나 리스트(list)를 반환했고, 다른 함수가 그것들을 가지고 무엇을 할지 결정했다. 작동은 했지만… 요구사항이 변경될 때까지였다.

그래서 나는 함수를 반환하기 시작했다.

```python
def create_cleanup_task(path):
    def task():
        import os
        if os.path.exists(path):
            os.remove(path)
    return task

# usage
task = create_cleanup_task("temp.log")
task()  # executes later
```

이것은 스케줄링(scheduling), 재시도(retries), 로깅(logging) 등 모든 것을 바꿨다. 갑자기 자동화(automation)가 모듈화(modular)되었다.

### 6. 로깅(Logging)은 프린팅(Printing)이 아니다

만약 당신의 자동화 스크립트(automation script)가 아직도 `print()`를 사용한다면, 이 글을 두 번 읽어주길 바란다.

```python
# old me
print("Downloading file...")
print("Done!")

# new me (using built-in logging)
import logging

logging.basicConfig(filename='task.log', level=logging.INFO)
logging.info("Downloading file...")
logging.info("Done.")
```

이것이 중요한 이유:

*   타임스탬프(time-stamp)가 기록된다.
*   파일로 저장된다.
*   크론 잡(cron job)에서 작동한다.
*   미래의 당신이 새벽 2시에 울지 않고 디버그(debug)할 수 있도록 돕는다.

### 마지막 생각

나의 가장 큰 실수는 지저분한 코드가 아니었다. "작동하기만 하면 충분하다"고 생각했던 것이었다. 자동화는 파이썬 스크립트(Python script)를 작성하는 것이 아니라, 폭발하지 않고 진화할 수 있는 스크립트를 작성하는 것이다.

이 리팩토링(refactoring) 비법들은 단지 내 코드를 깔끔하게 만든 것이 아니었다. 그것들은 코드를 확장 가능하고(scalable), 재사용 가능하며(reusable), 놀랍게도… 다시 돌아와 작업하는 것을 즐겁게 만들었다.
