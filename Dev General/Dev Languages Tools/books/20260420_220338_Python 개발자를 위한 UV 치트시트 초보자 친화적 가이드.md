# Python 개발자를 위한 UV 치트시트: 초보자 친화적 가이드

Python 개발자이면서 `pip install`에 걸리는 시간에 좌절했거나, 파이썬 의존성 관리(Python dependency management)가 Node만큼 빠르고 원활하기를 바랐다면, `UV`는 바로 당신이 기다려온 도구입니다.

`UV`는 Astral 팀이 개발한 초고속 파이썬 패키지 관리자(Python package manager)입니다. `pip`과 `virtualenv`를 대체하는 간단한 도구로, 주로 속도, 보안 및 최신 개발자 워크플로우를 위해 설계되었습니다.

이 치트시트는 `UV`의 기본 사항, 설정 방법, 그리고 파이썬 개발자로서 필요한 가장 필수적인 명령어들을 안내합니다.

# 🛠️ UV란 무엇인가?

`UV`는 다음과 같습니다:

*   `pip`과 같은 파이썬 패키지 관리자이며 `pip`보다 10~100배 빠릅니다.
*   가상 환경 관리자 (`virtualenv` 또는 `venv`와 유사)
*   Rust로 작성되어 매우 빠릅니다.
*   `pyproject.toml`과 완벽하게 호환됩니다.
*   `pip`, `pip-tools`, `pipx`, `poetry`, `pyenv`, `twine`, `virtualenv` 등을 대체하는 단일 도구입니다.

# ⚙️ 설치

스탠드얼론 설치 프로그램 사용:

`curl`을 사용하여 스크립트를 다운로드하고 `sh`로 실행합니다:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

`uv`는 Homebrew의 핵심 패키지로 제공됩니다.

```bash
brew install uv
```

`irm`을 사용하여 스크립트를 다운로드하고 `iex`로 실행합니다:

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

PyPI 사용:

PyPI에서 설치

```bash
pipx install uv
```

하지만 `pip`도 사용할 수 있습니다:

```bash
pip install uv
```

설치 후 확인:

```bash
uv --version
```

## `uv` 업그레이드

`uv`가 스탠드얼론 설치 프로그램으로 설치된 경우, 필요에 따라 자체 업데이트할 수 있습니다:

```bash
uv self update
```

다른 설치 방법을 사용한 경우, 해당 패키지 관리자의 업그레이드 방법을 사용하세요.

```bash
pip install --upgrade uv
```

## `uv` 제거

저장된 데이터 정리:

```bash
uv cache clean
rm -r "$(uv python dir)"
rm -r "$(uv tool dir)"
```

`uv` 및 `uvx` 바이너리 제거:

macOS/Linux:

```bash
rm ~/.local/bin/uv ~/.local/bin/uvx
```

Windows:

```bash
rm $HOME\.local\bin\uv.exe
rm $HOME\.local\bin\uvx.exe
```

# 파이썬 설치

최신 파이썬 버전 설치:

```bash
uv python install
```

특정 파이썬 버전 설치:

```bash
uv python install 3.12
```

여러 파이썬 버전 설치:

```bash
uv python install 3.11 3.12
```

지원되는 최신 패치 릴리스(patch release)로 파이썬 버전 업그레이드:

```bash
uv python upgrade 3.12
```

`uv`가 관리하는 모든 파이썬 버전 업그레이드:

```bash
uv python upgrade
```

## 파이썬 재설치

`uv`가 관리하는 파이썬 버전을 재설치하려면 `--reinstall` 플래그를 사용합니다 (예시):

```bash
uv python install --reinstall
```

## 파이썬 설치 확인

사용 가능한 및 설치된 파이썬 버전 확인:

```bash
uv python list
```

## 파이썬 제거

파이썬 버전 제거:

```bash
uv python uninstall
```

# 🧪 UV로 프로젝트 생성

`UV`는 `pyproject.toml` 기반 워크플로우(workflow)와 잘 작동합니다.

1.  새 파이썬 프로젝트 디렉터리 생성

    ```bash
    mkdir my-python-app
    cd my-python-app
    ```

2.  📄 프로젝트 초기화

    ```bash
    uv init
    ```

    이 명령은 다음 파일들을 생성합니다:

    *   `pyproject.toml`
    *   `.python-version`
    *   `README.md`
    *   `main.py`

    이제 구조화된 파이썬 프로젝트를 바로 시작할 준비가 되었습니다!

# 📦 의존성 설치

`UV`는 익숙한 `pip` 스타일의 설치 명령을 사용하지만 훨씬 빠릅니다.

```bash
uv pip install requests
```

`uv` 명령에서 `pip`을 사용하려면 환경을 생성하거나, 가상 환경이 아닌 곳에 설치하기 위해 `--system` 플래그를 전달해야 합니다.

환경 생성:

```bash
uv venv
```

그러면 위에서 설명한 `pip` 스타일의 설치 명령을 실행할 수 있습니다.

또는 `uv`를 직접 사용하여 패키지를 추가할 때 `uv`가 환경을 생성하도록 할 수 있습니다.

```bash
uv add requests
```

이것은 패키지를 설치하고 `pyproject.toml` 및 `uv.lock` 파일을 업데이트합니다.

`requirements.txt` 파일에서 마이그레이션하는 경우, `-r` 플래그와 함께 `uv add`를 사용하여 파일의 모든 의존성(dependencies)을 추가할 수 있습니다:

```bash
# `requirements.txt`의 모든 의존성을 추가합니다.
uv add -r requirements.txt -c constraints.txt
```

## 개발 의존성 추가

```bash
uv add requests --dev
# 또는
uv add --dev requests
```

`pip`을 사용하여 개발 의존성 추가

```bash
uv pip install --dev black
```

기본적으로 개발 의존성(dev dependencies)은 `lockfile`의 별도 섹션에 기록됩니다.

## 의존성 제거

의존성을 제거하려면 `uv remove`를 사용할 수 있습니다:

```bash
uv remove requests
```

개발 의존성을 제거하려면 다음을 사용할 수 있습니다:

```bash
uv remove requests --dev
```

# 🔄 의존성 업데이트

패키지를 업그레이드하려면 `--upgrade-package` 플래그와 함께 `uv lock`을 실행합니다:

```bash
uv lock --upgrade-package requests
```

## 단일 패키지 업그레이드

```bash
uv lock --upgrade-package <package_name>
# 또는
uv sync --upgrade-package <package_name>
```

`uv lock --upgrade-package <package_name>` 명령은 `lockfile`에서 지정된 패키지를 업데이트합니다. 이를 통해 다음번에 `uv sync`를 실행할 때 업그레이드된 버전의 패키지를 사용하도록 보장합니다.

`uv sync --upgrade-package <package_name>` 명령은 `lockfile`을 업데이트할 뿐만 아니라 가상 환경(virtual environment)을 새 패키지 버전과 동기화합니다.

`pip` 사용 시

```bash
uv pip install --upgrade requests
```

## 모든 의존성 업그레이드

프로젝트의 모든 패키지를 업그레이드하려면 `uv lock --upgrade` 또는 `uv sync --upgrade`를 사용할 수 있습니다.

`pip` 사용 시

```bash
uv pip upgrade
```

이는 모든 새 버전을 해결하고 `lockfile`을 업데이트합니다.

# 📋 의존성 확인

`pip list`와 같이 설치된 모든 패키지를 나열합니다.

```bash
uv pip list
```

설치된 모든 패키지를 트리 형태로 나열합니다:

```bash
uv tree
```

패키지 정보 표시:

```bash
uv pip show requests
```

# 🔐 가상 환경 관리

`UV`는 프로젝트 폴더에 `.venv`를 자동으로 생성하고 활성화합니다.

수동으로 활성화하려면:

```bash
source .venv/bin/activate
```

하지만 일반적으로 수동으로 활성화할 필요는 없습니다. `UV`는 프로젝트 디렉터리에서 실행될 때 올바른 환경을 자동으로 감지하고 사용합니다.

# 🧰 기타 유용한 명령

환경을 `lockfile`과 동기화

```bash
uv sync
```

`pip` 사용 시

```bash
uv pip sync
```

요구 사항(requirements)을 `lockfile`로 컴파일

```bash
uv pip compile
```

프로젝트를 패키지 인덱스(package index)에 게시

```bash
uv publish
```

프로젝트 의존성(dependencies)을 위한 `lockfile` 생성

```bash
uv lock
```

# 🔚 결론

`UV`는 파이썬 생태계(ecosystem)에 강력한 추가 요소입니다. 더 원활한 워크플로우(workflow)를 찾는 초보 개발자든, 빠르고 신뢰할 수 있는 의존성 관리(dependency management)를 원하는 숙련된 개발자든, `UV`는 시도해 볼 가치가 있습니다.
