# 수 시간을 아껴준 10가지 터미널 명령어 (10 Terminal Commands That Saved Me Hours of Clicking)

저는 터미널을 싫어했던 것이 아니라 두려워했습니다.

깜빡이는 커서가 있는 어두운 창은 차갑고 냉정하게 느껴졌습니다. 오타 하나만으로도 컴퓨터 전체가 삭제될 것 같은 공포를 느꼈죠. 반면 마우스는 친숙하고 안전했습니다. 모든 것을 눈으로 볼 수 있었으니까요.

그래서 저는 계속 클릭했습니다. 폴더 안의 폴더를 뒤지고, 우클릭 메뉴를 열고, 끝없이 스크롤했습니다. 그러다 어느 순간, 실제로 코드를 작성하는 시간보다 파일을 찾는 데 더 많은 시간을 쓰고 있다는 사실을 깨달았습니다.

결국 마지못해 터미널을 열었고, 그 이후 다시는 이전으로 돌아가지 않게 되었습니다. 두려움이 사라지자 한 가지 사실이 명확해졌습니다. 터미널은 무서운 것이 아니라, 압도적으로 빠르다는 점입니다.

마우스만 사용하던 초보자에서 컴퓨터를 진정으로 제어한다는 느낌을 갖게 해준 10가지 간단한 명령어를 소개합니다. 모든 것을 외울 필요는 없습니다. 그저 이 명령어들부터 시작해 보세요.

### 1. 어디든 즉시 이동하기: cd (Tab 키 활용)
**기존 방식:** 파일 탐색기를 열고 폴더를 대여섯 번 클릭하다가 잘못 클릭하면 한숨을 쉬며 뒤로 돌아갑니다.  
**터미널 방식:** 
```bash
cd ~/Doc[TAB]/Cod[TAB]/Proj[TAB]/web[TAB]
```
`cd`는 디렉터리 변경(change directory)을 의미합니다. Tab 키를 누르면 터미널이 폴더 이름을 자동 완성(auto-complete)해 주므로 이름을 정확히 기억하거나 타이핑할 필요가 없습니다. 걷는 대신 텔레포트를 하는 기분을 느낄 수 있습니다.

### 2. 한 번에 폴더 구조 만들기: mkdir -p
**마우스 방식:** 폴더 생성, 진입, 하위 폴더 생성 과정을 반복합니다.  
**터미널 방식:**
```bash
mkdir -p my-app/{src/components,assets,tests}
```
하나의 명령어로 중첩된 여러 폴더를 즉시 생성할 수 있습니다. 새 프로젝트를 시작할 때 클릭이나 기다림 없이 구조를 잡기에 완벽합니다.

### 3. 즉각적인 백업 생성: cp -r
위험한 설정을 변경하기 전, "전부 망가지면 어떡하지?"라는 걱정이 들 때 유용합니다.
```bash
cp -r my-project my-project-backup
```
`-r`은 폴더 내부의 모든 내용을 복사한다는 의미입니다. 빠르고 신뢰할 수 있으며, 실험적인 코드를 작성하기 전 마음의 평안을 줍니다.

### 4. 어지러운 폴더 빠르게 정리하기: mv + *
파일을 일일이 선택하는 대신 와일드카드(wildcard, `*`)를 사용합니다.
```bash
mv ~/Downloads/*.png ~/Pictures/
mv ~/Downloads/*.jpg ~/Pictures/
```
`*` 기호는 "모든 것"을 매칭한다는 뜻입니다. 파일 정리가 놀라울 정도로 쾌적해집니다.

### 5. 파일 내 모든 내용 찾기: grep -r
수십 개의 파일을 수동으로 뒤지는 대신 다음 명령어를 사용하세요.
```bash
grep -r "api_key_v2" .
```
이 명령어는 모든 폴더와 파일을 즉시 검색합니다. 전체 프로젝트를 대상으로 하는 `Ctrl+F`와 같습니다.

### 6. 폴더 내용 확인하기: ls
폴더를 무작정 열어보기 전에 터미널에 입력합니다.
```bash
ls
# 또는 더 자세한 정보를 위해
ls -la
```
숨겨진 파일을 포함하여 모든 파일을 즉시 확인할 수 있습니다. 폴더를 열고 닫는 번거로움이 사라집니다.

### 7. 에디터 없이 파일 생성하기: touch
새 파일이 빠르게 필요할 때 사용합니다.
```bash
touch index.html
touch app.js
```
빈 파일을 즉시 만들어주므로 빠른 설정에 매우 유용합니다.

### 8. 파일 삭제하기 (신중하게): rm
파일 삭제:
```bash
rm old.txt
```
폴더 삭제:
```bash
rm -r old-folder
```
터미널에서는 '실행 취소' 버튼이 없으므로 엔터를 누르기 전에 항상 다시 한번 확인하세요.

### 9. 열지 않고 파일 내용 읽기: cat / less
파일 내용을 빠르게 훑어볼 때 사용합니다.
```bash
cat README.md
# 긴 파일의 경우
less server.log
```
별도의 앱을 실행하거나 에디터를 열지 않고도 내용을 확인할 수 있습니다.

### 10. 현재 위치 확인하기: pwd
폴더 구조 안에서 길을 잃었을 때 사용합니다.
```bash
pwd
```
현재 작업 중인 위치의 경로를 출력해 줍니다. 단순하지만 의외로 매우 유용합니다.

### GUI를 완전히 버릴 필요는 없습니다
저도 여전히 VS Code를 쓰고, 브라우저를 사용하며, 마우스를 클릭합니다. 하지만 터미널은 저에게 **자신감**이라는 다른 가치를 주었습니다. 단순히 컴퓨터를 탐색하는 것이 아니라, 컴퓨터에 직접 명령을 내리고 있다는 확신을 갖게 되었습니다.

딱 일주일 동안만 다음 작업에 터미널을 사용해 보세요.
- 파일 이동
- 폴더 생성
- 코드 검색

그것만으로도 마우스를 쥔 손은 휴식을 얻고, 업무 흐름(workflow)은 빨라질 것입니다. 그리고 그 검은 화면이 어느덧 집처럼 편안하게 느껴지기 시작할 것입니다.
