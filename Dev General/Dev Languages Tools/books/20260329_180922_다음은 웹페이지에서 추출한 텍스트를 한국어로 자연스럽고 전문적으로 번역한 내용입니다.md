오래된 유닉스 도구들을 당신의 시간을 아껴주는 현대적인 대안으로 바꾸세요.

저는 업무 시간의 약 70%를 터미널에서 보냅니다. 이제는 셸(shell)이 제 뇌의 확장판이나 다름없습니다.

수년 동안 저는 모두가 사용하는 표준 유닉스(Unix) 도구인 `grep`, `find`, `cat`, `ps`, `top` 등을 사용해 왔습니다. 이 도구들은 작동하며 신뢰할 수 있습니다. 하지만 1970년대에 만들어져 그 연식을 보여줍니다.

하지만 지난 몇 년간, 새로운 세대의 CLI (Command Line Interface) 도구들이 등장했습니다. 이 도구들은 단순히 이전 도구들을 복제하는 것을 넘어, 훨씬 더 개선되었습니다. 더 나은 기본 설정, 더 나은 출력, 더 나은 성능, 모든 면에서 더 우수합니다.

여기 제가 매일 사용하며 제 작업 방식을 근본적으로 바꾼 8가지 CLI 도구들이 있습니다. 일부는 오래된 유닉스(Unix) 유틸리티를 대체했고, 다른 일부는 제가 존재조차 몰랐던 문제들을 해결해 줍니다.

## 1. fzf — 모든 것을 퍼지(Fuzzy) 방식으로 찾기

이 목록에서 단 하나의 도구만 남길 수 있다면, 저는 `fzf`를 선택할 것입니다.

**수행 기능:** 말 그대로 모든 것을 퍼지(Fuzzy) 방식으로 찾습니다. 파일, 명령어 기록, 프로세스, Git 브랜치 등 목록 형태의 모든 것을 `fzf`로 검색할 수 있습니다.

**놀라운 이유:**

```bash
# 모든 파일 찾아서 열기
vim $(fzf)

# 명령어 기록 검색 (저는 Ctrl-R로 설정했습니다)
history | fzf
# Git 브랜치 대화형(interactively)으로 체크아웃
git checkout $(git branch | fzf)
# 이름으로 프로세스 종료
kill -9 $(ps aux | fzf | awk '{print $2}')
```

하지만 진정한 위력은 통합 기능에서 나옵니다. 다음 내용을 `.bashrc` 또는 `.zshrc`에 추가하세요:

```bash
# Ctrl-T: 선택한 파일 경로 붙여넣기
# Ctrl-R: 명령어 기록 검색
# Alt-C: 선택한 디렉토리로 cd 이동
[ -f ~/.fzf.bash ] && source ~/.fzf.bash
```

이제 터미널에 모든 것에 퍼지 검색 기능이 내장됩니다.

저는 또한 사용자 정의 함수에도 사용합니다:

```bash
# Git에 추적되는 모든 파일을 vim에서 열기
function vf() {
  local file
  file=$(git ls-files | fzf --preview 'bat --color=always {}' --preview-window=right:60%:wrap)
  [ -n "$file" ] && vim "$file"
}
```

검색하는 동안 미리보기 창에 파일 내용이 표시됩니다. 정말 마법 같아요.

**설치:**

```bash
# macOS
brew install fzf

# Ubuntu/Debian
sudo apt install fzf
# 설치 후, 셸 통합 설정을 위해 실행
$(brew --prefix)/opt/fzf/install
```

**내 설정:**

```bash
export FZF_DEFAULT_OPTS="
  --height 40%
  --layout=reverse
  --border
  --inline-info
  --preview 'bat --style=numbers --color=always --line-range :500 {}'
"

# find 대신 fd 사용
export FZF_DEFAULT_COMMAND='fd --type f --hidden --follow --exclude .git'
```

저는 `fzf`를 하루에 약 50번 정도 사용합니다. 이제는 근육 기억(muscle memory)처럼 사용하게 되었습니다.

## 2. ripgrep (rg) — 진정으로 훌륭한 grep

`grep`은 괜찮습니다. `ripgrep`은 놀랍습니다.

**수행 기능:** 파일에서 패턴을 검색하는데, `grep`보다 훨씬 빠르고 스마트합니다.

**속도 비교 (2GB 코드베이스 검색):**

```bash
grep -r "function authenticate" .  # 12.4초
rg "function authenticate"         # 0.8초
```

15배 더 빠릅니다. 게다가 기능에 대해서는 말할 것도 없습니다.

**스마트한 기본 설정 (Smart defaults):**

```bash
# ripgrep은 자동으로:
# - .gitignore 파일을 건너뜁니다
# - 바이너리 파일을 건너뜁니다
# - 재귀적으로 검색합니다
# - 색상을 사용합니다
# - 줄 번호를 표시합니다

rg "TODO"  # 그냥 작동합니다
```

`grep`과 비교:

```bash
grep -r -n --color=always --exclude-dir={.git,node_modules} "TODO" .
```

**유용한 패턴:**

```bash
# 특정 파일 형식 검색
rg "useState" -t typescript

# 대소문자 무시
rg -i "error"
# 컨텍스트 표시 (앞뒤 2줄)
rg -C 2 "function"
# 일치하는 파일 목록 표시 (grep -l과 유사)
rg --files-with-matches "import.*React"
# 검색 및 교체 미리보기
rg "oldName" --replace "newName" --passthru
# 파일 이름만 검색
rg --files | rg "test"
```

**가장 많이 사용하는 명령어:**

```bash
# 함수 정의 검색
rg "def \w+|function \w+|const \w+ ="
```

**설치:**

```bash
# macOS
brew install ripgrep

# Ubuntu/Debian
sudo apt install ripgrep
```

저는 `grep`을 `rg`로 별칭(alias) 설정하고 다시는 `grep`을 사용하지 않았습니다:

```bash
alias grep='rg'
```

## 3. bat — 구문 강조 기능이 있는 cat

`cat`은 파일 내용을 보여줍니다. `bat`은 파일 내용을 아름답게 보여줍니다.

**비교:**

```bash
# 기존 방식
cat src/app.js

# 현대적인 방식
bat src/app.js
```

`bat`이 제공하는 기능:

*   200개 이상의 언어에 대한 구문 강조 (Syntax Highlighting)
*   Git 통합 (수정 사항 표시)
*   줄 번호
*   긴 파일에 대한 페이지 넘기기 (Paging)
*   출력 불가능 문자 시각화 (Non-printable character visualization)

**유용한 옵션:**

```bash
# 모두 표시 (cat -A와 유사)
bat --show-all file.txt

# 일반 스타일 (장식 없음)
bat --style=plain file.txt
# 특정 언어
bat --language=json config.txt
# 고유한 헤더를 가진 여러 파일
bat app.js utils.js config.js
```

**다른 도구와의 통합:**

```bash
# fzf에서 파일 미리보기
fzf --preview 'bat --color=always {}'

# 색상이 적용된 man 페이지
export MANPAGER="sh -c 'col -bx | bat -l man -p'"
```

**설치:**

```bash
# macOS
brew install bat

# Ubuntu/Debian
sudo apt install bat
# 참고: Debian/Ubuntu에서는 'batcat'으로 설치됩니다.
```

**내 별칭 (aliases):**

```bash
alias cat='bat --paging=never'
alias less='bat'
```

이제 파일을 볼 때마다 자동으로 구문 강조가 적용됩니다.

## 4. fd — 현대적인 find

`find`는 강력합니다. 하지만 사용법을 기억하기가 불가능합니다.

`fd`는 직관적이고 빠르며, 대부분의 경우 당신이 실제로 원하는 것을 수행합니다.

**비교:**

```bash
# JavaScript 파일 찾기 (기존 방식)
find . -type f -name "*.js" -not -path "*/node_modules/*"

# JavaScript 파일 찾기 (현대적인 방식)
fd "\.js$"
# 또는 더 간단하게
fd js
```

**스마트한 기본 설정 (Smart defaults):**

```bash
# fd는 자동으로:
# - .gitignore 패턴을 무시합니다
# - 숨김 파일을 무시합니다 (포함하려면 -H 사용)
# - 색상 출력을 사용합니다
# - 기본적으로 대소문자를 구분하지 않습니다 (대소문자 구분을 위해 -s 사용)

fd config  # config.js, Config.py, my-config.yaml 등을 찾습니다.
```

**일반적인 패턴:**

```bash
# 확장자로 파일 찾기
fd -e py

# 디렉토리만 찾기
fd -t d node_modules
# 각 파일에 대해 명령 실행
fd -e js -x prettier --write {}
# 전체 경로 표시
fd -a README
# 특정 디렉토리에서 검색
fd test src/
# 패턴 제외
fd -E "*.test.js" component
```

**가장 좋아하는 사용 사례:**

```bash
# 모든 node_modules 디렉토리 찾아서 삭제
fd -t d node_modules -x rm -rf {}

# 큰 파일 찾기
fd -t f -x ls -lh {} | sort -k5 -h
```

**설치:**

```bash
# macOS
brew install fd

# Ubuntu/Debian
sudo apt install fd-find
```

저는 파일을 편집하거나, 빌드를 정리하거나, 다른 명령으로 파이프(pipe)하기 위해 매일 `fd`를 사용합니다.

## 5. exa — 슈퍼파워가 있는 ls

`ls`는 파일 목록을 보여줍니다. `exa`는 파일 목록을 아름답고 유익하게 만듭니다.

**기본 비교:**

```bash
# 기존 방식
ls -lah

# 새로운 방식
exa -lah
```

출력은 비슷하지만, `exa`는 다음을 추가합니다:

*   각 파일의 Git 상태
*   파일 아이콘 (적절한 글꼴 사용 시)
*   내장된 트리 뷰 (Tree view)
*   더 나은 색상
*   파일 유형 분류

**가장 많이 사용하는 명령어:**

```bash
# Git 상태와 함께 목록 표시
exa -lah --git

# 트리 뷰 (tree 명령어보다 우수)
exa --tree --level=2
# 크기 순으로 정렬 (가장 큰 것부터)
exa -lah --sort=size --reverse
# 수정 시간 순으로 정렬
exa -lah --sort=modified
# 아이콘이 있는 그리드 뷰
exa --icons --grid
```

**핵심 기능 — Git 상태가 있는 트리 뷰:**

```bash
exa --tree --level=3 --git-ignore --icons
```

이것은 디렉토리 구조를 보여주고, `.gitignore`를 존중하며, Git 상태를 표시하고, 아이콘을 표시합니다. 하나의 명령으로 완전한 프로젝트 개요를 볼 수 있습니다.

**설치:**

```bash
# macOS
brew install exa

# Ubuntu/Debian
sudo apt install exa
```

**내 별칭 (aliases):**

```bash
alias ls='exa --icons'
alias ll='exa -lah --icons --git'
alias lt='exa --tree --level=2 --icons'
alias la='exa -a --icons'
```

이제 `ls`가 실제로 유용합니다.

## 6. zoxide — 학습하는 cd

`cd`는 전체 경로를 입력하게 합니다. `zoxide`는 당신의 마음을 읽습니다.

**수행 기능:** 방문한 디렉토리를 추적하고, 부분 일치로 해당 디렉토리로 이동할 수 있게 합니다.

```bash
# 처음에는 전체 경로 사용
cd ~/projects/work/backend/api

# 나중에는 그냥 입력
z api
# 또는 이렇게도 가능
z be ap
# "proj"와 가장 자주 사용되고 최근에 사용된(frecency) 일치 항목으로 이동
z proj
```

`zoxide`는 "프리퀀시(frecency)"—빈도(frequency)와 최근 사용(recency)의 조합—를 사용합니다. 디렉토리를 더 많이, 그리고 더 최근에 방문할수록 순위가 높아집니다.

**추가 기능:**

```bash
# 모호할 때 대화형 선택
zi api

# 파일 관리자에서 열기
zo api
# 모든 일치 항목 나열
zoxide query --list api
# 경로 제거
zoxide remove ~/old/path
```

**실제 예시:**

```bash
# 기존 워크플로
cd ~/projects/client-work/acme-corp/frontend/src/components

# zoxide 사용 시 (한 번 방문 후)
z comp
```

**설치:**

```bash
# macOS
brew install zoxide

# Ubuntu/Debian
curl -sS https://raw.githubusercontent.com/ajeetdsouza/zoxide/main/install.sh | bash
# 셸 설정에 추가
echo 'eval "$(zoxide init bash)"' >> ~/.bashrc
# 또는 zsh의 경우
echo 'eval "$(zoxide init zsh)"' >> ~/.zshrc
```

일주일 동안 사용하고 나면, `zoxide` 없이 어떻게 살았는지 궁금해할 것입니다. 저는 몇 달 동안 전체 디렉토리 경로를 입력한 적이 없습니다.

## 7. httpie — 인간을 위한 curl

`curl`은 강력합니다. 하지만 기본적인 요청 이상에서는 복잡하고 알 수 없는 명령어 덩어리입니다.

`httpie`는 HTTP 요청을 읽기 쉽고 직관적으로 만듭니다.

**비교:**

```bash
# curl
curl -X POST https://api.example.com/users \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer token123" \
  -d '{"name":"John","email":"john@example.com"}'

# httpie
http POST https://api.example.com/users \
  Authorization:"Bearer token123" \
  name=John \
  email=john@example.com
```

**주요 기능:**

```bash
# GET 요청 (기본값)
http example.com/api/users

# JSON은 자동 처리
http POST api.example.com/posts title="Hello" body="World"
# 쿼리 매개변수
http GET api.example.com/search q==python limit==10
# 헤더
http example.com Authorization:"Bearer token" User-Agent:"MyApp/1.0"
# 파일 다운로드
http --download example.com/file.zip
# 세션 (헤더/인증 정보 기억)
http --session=./session.json POST api.example.com/login username=admin password=secret
http --session=./session.json GET api.example.com/profile
```

**아름다운 출력:**

응답은 구문 강조와 함께 자동으로 포맷됩니다:

```bash
http https://api.github.com/users/octocat
```

```
HTTP/1.1 200 OK
Content-Type: application/json
{
    "login": "octocat",
    "id": 1,
    "name": "The Octocat",
    ...
}
```

**내 일반적인 패턴:**

```bash
# 로컬 API 테스트
http :3000/api/users

# 파일과 함께 POST
http POST api.example.com/upload < data.json
# 폼 데이터
http --form POST example.com/form name="John" photo@~/picture.jpg
# 응답 본문만 표시
http --body example.com/api
```

**설치:**

```bash
# macOS
brew install httpie

# Ubuntu/Debian
sudo apt install httpie
# 또는 pip을 통해
pip install httpie
```

API 개발 및 테스트에 `httpie`는 매우 유용합니다. 읽기 쉬운 문법 덕분에 문서를 계속 확인하지 않고도 명령어를 실제로 기억할 수 있습니다.

## 8. tldr — 엉망진창이 아닌 man 페이지

`man` 페이지는 포괄적입니다. 하지만 또한 너무 방대해서 압도적입니다.

`tldr`은 대신 실용적인 예시를 제공합니다.

**비교:**

```bash
man tar  # 1,000줄 이상의 문서

tldr tar # 20가지의 실용적인 예시
```

**예시 출력:**

```bash
$ tldr tar

  tar
  Archiving utility.
  Often combined with a compression method, such as gzip or bzip2.
  - Create an archive from files:
    tar cf target.tar file1 file2 file3
  - Create a gzipped archive:
    tar czf target.tar.gz file1 file2 file3
  - Extract a (compressed) archive into the current directory:
    tar xf source.tar[.gz|.bz2|.xz]
  - Extract an archive into a target directory:
    tar xf source.tar -C directory
  - List the contents of a tar file:
    tar tvf source.tar
```

핵심만 간단하게. 복사하고, 붙여넣고, 수정하면 끝입니다.

**다음과 같은 경우에 유용합니다:**

```bash
# 모든 명령어에 대한 빠른 참조
tldr git
tldr docker
tldr ffmpeg
tldr ssh

# 캐시를 주기적으로 업데이트
tldr --update
```

**설치:**

```bash
# macOS
brew install tldr

# Ubuntu/Debian
sudo apt install tldr
# 또는 npm을 통해
npm install -g tldr
# 데이터베이스 업데이트
tldr --update
```

저는 자주 사용하지 않는 명령어에 대해서는 `tldr`을 먼저 사용합니다. 더 자세한 정보가 필요하면 그 다음에 `man` 페이지를 확인합니다. 하지만 80%의 경우 `tldr`이 제가 필요한 것을 정확히 제공합니다.

## 설치 스크립트

모든 것을 한 번에 설치하고 싶으신가요? 다음 스크립트를 참고하세요:

```bash
#!/bin/bash

# macOS용 (Homebrew 이용)
if [[ "$OSTYPE" == "darwin"* ]]; then
  brew install fzf ripgrep bat fd exa zoxide httpie tldr
  
  # fzf 키 바인딩 설정
  $(brew --prefix)/opt/fzf/install --all
# Ubuntu/Debian용
elif [[ "$OSTYPE" == "linux-gnu"* ]]; then
  sudo apt update
  sudo apt install -y fzf ripgrep bat fd-find exa httpie tldr
  
  # zoxide는 별도로 설치
  curl -sS https://raw.githubusercontent.com/ajeetdsouza/zoxide/main/install.sh | bash
fi
# tldr 데이터베이스 업데이트
tldr --update
echo "완료되었습니다! 셸을 재시작하거나 설정 파일을 소싱(source)하세요."
```

## 학습 곡선

이 모든 것을 한 번에 적용하려고 하지 마세요. 한두 가지부터 시작하세요:

*   **`fzf`부터 시작하세요** — 기존 워크플로에 즉시 통합됩니다.
*   **`ripgrep`을 추가하세요** — 코드를 검색할 때 지속적으로 사용하게 될 것입니다.
*   **`bat`과 `exa`를 시도해 보세요** — 기존 도구를 대체하는 방식으로 쉽게 적용할 수 있습니다.
*   **`zoxide`로 넘어가세요** — 기록이 쌓이는 데 일주일이 걸리지만, 그 후에는 필수 불가결해집니다.

매일 일주일 동안 사용해 보세요. 일단 근육 기억(muscle memory)처럼 익숙해지면, 더 많은 도구를 추가하세요.

## 이것이 중요한 이유

이것들은 단순히 빛나는 새 장난감이 아닙니다. 실제 문제들을 해결해 줍니다:

*   **더 빠른 워크플로** — `ripgrep`은 `grep`보다 15배 빠릅니다.
*   **더 나은 기본 설정 (Better defaults)** — 도구들이 `.gitignore`를 존중하고, 색상을 사용하며, 유용한 정보를 표시합니다.
*   **인지 부하 (Cognitive load) 감소** — `httpie`는 읽기 쉽고, `tldr`은 간결하며, `fzf`는 생각을 덜어줍니다.
*   **시간 절약** — `zoxide`는 매일 수십 번의 키 입력을 줄여줍니다.

이 도구들이 매일 30~60분, 주당 2~4시간을 절약해 준다고 추정합니다. 1년으로 따지면 100시간 이상입니다.

하지만 더 중요한 것은, 터미널 작업이 지루하지 않고 즐거워진다는 점입니다.
