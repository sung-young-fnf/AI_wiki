## 집 컴퓨터를 어디서든 접속 가능하게 만드는 방법 (공용 IP, 라우터 설정 없이)

집 컴퓨터를 어디서든 이용할 수 있다고 상상해 보세요.

상상해 보세요: 카페에서 노트북을 열고 주소 하나만 입력하면 집에 있는 컴퓨터에 연결되어 파일, 도구, 서버 등 모든 것을 이용할 수 있습니다. 심지어 노트북 배터리를 절약하면서도 고사양 작업을 처리할 수 있습니다!

이것이 바로 꿈입니다. 하지만 이를 실현하려고 시도하는 순간, 오래된 규칙을 깨닫게 됩니다: 외부에서 컴퓨터에 접속하려면, 컴퓨터가 외부에서 접근 가능해야 한다는 것이죠.

전통적으로 이는 인터넷 서비스 제공업체(ISP)에 공용 IP (Public IP)를 요청하거나, “공용 IPv4 (Public IPv4) 및 CGNAT 제거” 또는 “고정 IP (Static IP) 추가 기능”을 신청한 다음, 라우터에서 포트 포워딩 (Port Forwarding)을 구성하는 것을 의미합니다. 이 방법은 작동하지만, 설정하는 데 많은 시간이 소요됩니다. 게다가 추가 비용까지 발생합니다.

이제 상상해 보세요: 고정 IP (Static IP)를 구매할 필요 없이, 기다릴 필요 없이, 라우터를 건드릴 필요 없이 동일한 결과를 얻을 수 있습니다. — 즉시! 설치 및 설정 후 10분 이내에 (모든 것이 순조롭게 진행된다면 말이죠 — 하지만 저는 여러분이 시행착오 시간을 절약할 수 있도록 제가 직접 알아낸 정보를 알려드리고자 합니다).

Tailscale (테일스케일)을 소개합니다. (그리고 나서 NoMachine (노머신)을 추가할 예정인데, 아마 SSH (Secure Shell)만으로는 만족하지 않고 실제 데스크톱 환경을 원하실 테니까요.)

이 방법을 훨씬 더 일찍 알았더라면 좋았을 텐데! 여러분도 그렇게 느끼실 겁니다!

### 필요한 것

**Tailscale (테일스케일):** 여러분의 노트북과 집 컴퓨터가 마치 동일한 사설 LAN (Private LAN)에 연결된 것처럼 작동하게 해줍니다 (인터넷을 통해서든, 주기적으로 IP 주소가 변경되는 CGNAT을 통해서든, 심지어 다른 장치들 간에도 각 장치에 Tailscale이 설치 및 실행되어 있는 한 말이죠).
**NoMachine (노머신):** 안정적인 원격 데스크톱 (Remote Desktop) 세션을 제공합니다 (타이핑 감각이 일반적이고, 클립보드가 작동하며, OAuth (Open Authorization)/브라우저 관련 기능도 정상적으로 작동하고, 설정 없이 바로 복사/붙여넣기가 가능합니다).

그게 전부입니다. 정말 쉽죠!

### 0단계 — '홈 (Home)'과 '여행 (Travel)' 컴퓨터 결정

*   **홈 컴퓨터 (Home machine):** 접속하려는 컴퓨터 (Windows / Linux / Mac).
*   **여행용 컴퓨터 (Travel machine):** 여러분의 노트북 (제 경우에는 MacBook).

두 컴퓨터 모두에 Tailscale을 설치할 것입니다.

NoMachine도 두 컴퓨터 모두에 설치합니다! 홈 컴퓨터(서버)와 노트북(클라이언트)에요.

### 1단계 — macOS (여행용 노트북)에 Tailscale 설치

Homebrew (홈브루)를 선호한다면:

```bash
brew install --cask tailscale-app
open -a Tailscale
```

Tailscale 앱에 한 번 로그인합니다. 끝.

(여행용 컴퓨터가 Linux (리눅스) 또는 Windows (윈도우)인 경우, 2단계의 지침을 따르세요.)

### 2단계 — 홈 컴퓨터에 Tailscale 설치

#### Linux (리눅스) (홈)

```bash
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up
```

#### Windows 10 (윈도우 10) (홈) — Scoop (스쿱)을 이용한 깔끔한 방법

PowerShell (파워셸)을 열고 다음을 실행하세요:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
iwr -useb get.scoop.sh | iex
scoop bucket add extras
scoop install tailscale
```

그런 다음 Tailscale을 열고 로그인합니다.

### 3단계 — 홈 컴퓨터의 Tailscale IP 주소 확인

홈 컴퓨터에서 (또는 Tailscale CLI (명령줄 인터페이스)가 설치된 어느 곳에서든):

```bash
tailscale ip -4
```

다음과 같은 주소를 얻게 될 것입니다: `100.xxx.xx.xxx`

이것이 바로 여러분의 "인터넷을 통한 사설 LAN" 주소입니다.

선택적으로 다음 명령으로 확인할 수도 있습니다:

```bash
tailscale status
```

두 컴퓨터 모두 목록에 표시되어야 합니다.

클라이언트 컴퓨터 (노트북)에서 홈 컴퓨터의 IP 주소에 접근 가능한지 테스트합니다:

```bash
ping 100.xxx.xx.xxx
```

긍정적인 결과가 나와야 합니다.

### 4단계 — NoMachine 설치

#### Linux (리눅스) (홈 컴퓨터)

여기로 이동하여 Linux 패키지를 다운로드하세요:

https://www.nomachine.com/everybody/nomachine-for-your-os

.deb 패키지 설치 (Ubuntu/Debian (우분투/데비안) 예시):

```bash
sudo dpkg -i nomachine_*_amd64.deb
sudo apt -f install
```

#### Windows 10 (윈도우 10) (홈 컴퓨터)

동일한 다운로드 페이지에서 Windows 설치 프로그램을 선택하고 일반적인 방법으로 설치합니다.

#### macOS (여행용 노트북)

```bash
brew install --cask nomachine
```

macOS에서 처음 실행 시: 요청하는 권한 (화면 기록 등)을 부여하세요. 원격 데스크톱 앱에는 이러한 권한이 필요합니다.

### 5단계 — 간편한 '경로가 열려 있는지' 테스트 (선택 사항이지만 만족스러움)

MacBook에서 홈 컴퓨터의 Tailscale IP에 NoMachine의 기본 포트를 테스트합니다:

```bash
nc -vz 100.xxx.xx.xxx 4000
```

연결되면 성공입니다.

(포트 4000은 NoMachine NX (엔엑스)의 기본 포트입니다.)

### 6단계 — Tailscale을 통해 NoMachine으로 연결

MacBook에서:

1.  NoMachine을 엽니다.
2.  새 연결을 생성합니다.
3.  **호스트 (Host):** 홈 컴퓨터의 Tailscale IP (`100.xxx.xx.xxx`)
4.  **포트 (Port):** `4000`
5.  일반 OS 사용자 계정으로 연결하고 로그인합니다.

이제 라우터가 전혀 알지 못하는 상태에서, 어디서든 집 데스크톱에 접속했습니다.

### 이것이 핵심 요령입니다

여러분은 홈 컴퓨터를 공용 인터넷 (Public Internet)에 "노출"시키지 않았습니다.

고정 IP (Static IP)를 구매하지 않았습니다.

라우터 UI (사용자 인터페이스) 설정과 씨름할 필요도 없었습니다.

그저 두 장치에 사설 네트워크 경로 (Tailscale)를 제공하고, 이 경로를 통해 제대로 된 데스크톱 세션 (NoMachine)을 이용한 것입니다.

### 간단 체크리스트

**두 컴퓨터 모두에**
**홈 컴퓨터에서**
`tailscale ip -4` # 또는 `tailscale status`를 사용하여 Tailscale IP 주소 (`100.xx.xxx.xxx`)를 확인합니다.

**노트북에서**
마찬가지로 NoMachine에서 홈 컴퓨터로 새 연결을 설정합니다.

이를 위해서는 홈 컴퓨터의 사용자 이름과 비밀번호, 그리고 홈 컴퓨터의 Tailscale IP 주소가 필요합니다.

**정상 작동 테스트 (Sanity Test)**
`nc -vz <tailscale-ip> 4000`

이제 마우스와 키보드로 해당 컴퓨터를 원격으로 제어할 수 있습니다!

제 경우, 대상 컴퓨터는 Linux이고 클라이언트 컴퓨터는 macOS입니다. 이로 인해 물론 키보드 사용에 약간의 복잡성이 따릅니다. 저는 Mac에서 Cmd (커맨드) 버튼을 누르는 데 익숙하지만, Mac 화면에서 NoMachine을 통해 Linux 컴퓨터를 제어할 때는 Ctrl (컨트롤) 버튼을 사용해야 합니다. 하지만 곧 익숙해지실 겁니다.
