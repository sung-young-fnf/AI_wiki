# 사내 도구(Internal Tools) 구축에 완벽한 7가지 파이썬 라이브러리

파이썬(Python)으로 개발을 해왔다면, 관리자로부터 "이 작업을 위한 간단한 사내 대시보드 하나 만들어 줄 수 있어?"라는 요청을 받아본 적이 있을 것입니다. 대시보드, 승인 워크플로우, KPI 트래커, 자동화 봇과 같은 사내 도구는 기업 운영의 핵심이며, 파이썬은 이 모든 것을 구축하기에 가장 적합한 도구입니다.

지난 4년간 다양한 팀을 위해 사내 시스템을 구축하며 발견한, 개발자의 위상을 높여줄 7가지 파이썬 라이브러리를 소개합니다.

---

### 1. Reflex (프론트엔드 지식 없이 구축하는 웹 앱)

대부분의 사내 도구 프로젝트는 "누가 리액트(React)로 프론트엔드를 작성할 것인가?"라는 질문 앞에서 좌절됩니다. **Reflex**(구 Pynecone)는 이 고민을 해결해 줍니다. 개발자는 오직 파이썬만 작성하면 되며, Reflex가 내부적으로 리액트 프론트엔드를 생성합니다.

*   **장점:** 자바스크립트(JavaScript) 불필요, 빠른 클라우드 배포, 실시간 상태 동기화, 세련된 기본 UI.

**예시: 20줄로 만드는 실시간 KPI 대시보드**
```python
import reflex as rx

class State(rx.State):
    sales = 128
    bugs = 3

def dashboard():
    return rx.vstack(
        rx.heading("사내 대시보드"),
        rx.text(f"일일 판매량: {State.sales}"),
        rx.text(f"미해결 버그: {State.bugs}")
    )

app = rx.App()
app.add_page(dashboard)
app.compile()
```

### 2. FastAPI (사내 도구의 핵심 API)

**FastAPI**는 이미 유명하지만, 사내 도구로서의 가치는 여전히 과소평가되어 있습니다. 엑셀을 사용하는 팀을 위한 마이크로 API, 내부 인증 프록시, 승인 워크플로우 엔드포인트 등을 구축할 때 탁월합니다. 특히 자동 문서화 기능 덕분에 개발자가 아닌 인원도 API 구조를 쉽게 이해할 수 있습니다.

**예시: 간단한 승인 엔드포인트**
```python
from fastapi import FastAPI

app = FastAPI()
approvals = {"pending": [], "approved": []}

@app.post("/approve/{item}")
def approve(item: str):
    approvals["approved"].append(item)
    return {"status": "approved", "item": item}
```

### 3. NiceGUI (커스텀 요소가 강점인 대시보드)

Streamlit이 지나치게 데이터 과학에 치중되어 있다고 느껴진다면 **NiceGUI**가 대안이 될 수 있습니다. 내부적으로 Vue.js를 사용하지만 개발자는 파이썬만 다루면 됩니다. 테이블, 차트, 대화 상자 등 풍부한 웹 컴포넌트를 제공하며 모바일 환경에서도 잘 작동합니다.

**예시: 실시간 작업 트래커**
```python
from nicegui import ui

tasks = []
def add_task(task):
    tasks.append(task)
    table.update_rows(tasks)

with ui.row():
    input_box = ui.input("작업 내용")
    ui.button("추가", on_click=lambda: add_task(input_box.value))

table = ui.table(columns=["Task"], rows=tasks)
ui.run()
```

### 4. Textual (현대적인 터미널 UI)

터미널 환경에서 주로 작업하는 개발자나 데브옵스(DevOps) 엔지니어를 위한 라이브러리입니다. **Textual**을 사용하면 터미널 내부에서 작동하는 현대적인 그래픽 인터페이스(TUI)를 구축할 수 있습니다. SSH를 통해 원격으로 실행할 수 있다는 점이 큰 장점입니다.

**예시: 터미널 기반 서버 모니터**
```python
from textual.app import App
from textual.widgets import Header, Footer, Static
import psutil

class Monitor(App):
    async def on_mount(self):
        self.cpu = Static()
        await self.view.dock(Header(), Footer(), self.cpu)

    async def on_interval(self, *_):
        usage = psutil.cpu_percent()
        self.cpu.update(f"CPU 사용률: {usage}%")

Monitor().run()
```

### 5. RQ + Redis (가볍고 강력한 작업 큐)

이메일 발송, 대량 데이터 처리, 보고서 생성 등 백그라운드 작업이 필요한 경우 Celery를 떠올리곤 합니다. 하지만 사내 도구 수준에서는 Celery는 너무 과할 수 있습니다. **RQ**(Redis Queue)는 훨씬 단순하고 신뢰할 수 있는 대안입니다.

**예시: 무거운 작업 비동기 처리**
```python
# worker.py (작업 정의)
import time
def heavy_task():
    time.sleep(5)
    return "완료!"

# queue.py (작업 등록)
from redis import Redis
from rq import Queue
from worker import heavy_task

q = Queue(connection=Redis())
job = q.enqueue(heavy_task)
print(job.get_id())
```

### 6. Pandera (데이터 파이프라인 유효성 검증)

사내 도구는 종종 다른 팀에서 전달받은 '형식이 자주 바뀌는' CSV 파일 때문에 문제를 일으킵니다. **Pandera**는 데이터프레임(DataFrame)을 위한 유효성 검증 라이브러리로, 데이터 오염을 사전에 방지합니다.

**예시: CSV 스키마 검증**
```python
import pandera as pa
from pandera import Column, DataFrameSchema
import pandas as pd

schema = DataFrameSchema({
    "user_id": Column(int),
    "hours": Column(float, checks=pa.Check.ge(0)),
})

df = pd.read_csv("timesheet.csv")
validated = schema.validate(df)
```

### 7. Prefect (워크플로우 오케스트레이션)

Airflow의 복잡한 설정과 버그에 지쳤다면 **Prefect**가 정답입니다. 사내 데이터 요약, 정기 보고서 발송, 알림 자동화 등을 관리하기에 최적화되어 있습니다. 작업의 재시도(Retry), 로깅, 상태 추적을 직관적으로 수행할 수 있습니다.

**예시: 매시간 실행되는 판매량 워크플로우**
```python
from prefect import flow, task

@task
def fetch_sales():
    return 1200  # API 호출 시뮬레이션

@task
def log_sales(sales):
    print(f"현재 판매량: {sales}")

@flow
def sales_flow():
    sales = fetch_sales()
    log_sales(sales)

sales_flow()
```
