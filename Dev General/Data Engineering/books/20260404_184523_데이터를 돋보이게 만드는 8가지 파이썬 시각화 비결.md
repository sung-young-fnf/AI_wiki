# 데이터를 돋보이게 만드는 8가지 파이썬 시각화 비결

기술적으로는 정확하지만 시각적으로는 밋밋한 차트에 지치셨나요? 파워포인트 없이도 상사를 감동시킬 수 있는 방법이 있습니다. 대규모 데이터를 처리하거나 다른 도구가 제대로 작동하지 않는 환경에서도 효과적으로 소통할 수 있는, 활용도 높은 8가지 파이썬 시각화 기법과 코드 레시피를 소개합니다.

### 1) Datashader + HoloViews를 이용한 수십억 개의 데이터 포인트 렌더링

**문제:** 수백만 개의 점이 포함된 산점도(scatterplot)는 데이터가 뭉쳐 보여 식별이 어렵습니다.  
**해결책:** GPU/CPU에서 데이터를 집계(aggregation)하여 밀도를 적절하게 렌더링합니다.

```python
import holoviews as hv
import datashader as ds
from holoviews.operation.datashader import rasterize

hv.extension('bokeh')

# df에 'x', 'y' 컬럼이 있다고 가정
points = hv.Points(df, ['x','y'])
agg = rasterize(points)   # 내부적으로 datashader를 사용
hv.output(agg)
```

*   **특징:** 대부분의 경우 데이터를 다운샘플링하여 신호를 손실하지만, Datashader는 올바른 집계 방식을 통해 조밀한 클러스터나 미세한 구조를 그대로 유지합니다. 지리 공간 텔레메트리, 로그 데이터 또는 포인트 클라우드(point-cloud) 미리보기에 적합합니다.

### 2) Perspective를 활용한 엑셀 방식의 피벗 UI 제공

**문제:** 이해관계자들이 데이터를 실시간으로 피벗하고 슬라이싱하기를 원하지만, 이를 직접 구축하는 것은 매우 번거롭습니다.  
**해결책:** 수백만 행을 처리할 수 있는 라이브 피벗/웹 위젯인 Perspective를 사용합니다.

```python
from perspective import Table, PerspectiveWidget
import pandas as pd
t = Table(pd.DataFrame({"time": times, "value": vals}))
widget = PerspectiveWidget(t)
display(widget)   # 주피터랩 또는 주피터 노트북에서 실행
```

*   **특징:** Perspective는 WASM(WebAssembly) 기반의 피벗 그리드로, 브라우저에서 스트리밍 및 피벗 작업을 수행합니다. 정적인 데이터프레임보다 훨씬 뛰어난 상호작용성을 제공하며 탐색적 분석에 최적입니다.

### 3) plotly-resampler를 이용한 실시간 수준의 인터랙티브 다운샘플링

**문제:** 방대한 시계열 데이터를 브라우저에서 시각화하면 속도가 매우 느려집니다.  
**해결책:** 피크(peak)와 추세를 유지하면서 서버 또는 클라이언트 측에서 데이터를 지능적으로 재샘플링(resampling)합니다.

```python
import pandas as pd
import plotly.graph_objects as go
from plotly_resampler import FigureResampler

df = pd.read_csv("ticks.csv", parse_dates=["ts"])
fig = FigureResampler(go.Figure())
fig.add_trace(df["value"], hf_x=df["ts"], hf_y=df["value"], name="price")
fig.show()
```

*   **특징:** 단순히 모든 데이터를 Plotly로 보내는 대신, 시각적 정확도를 유지하면서도 브라우저에 전달되는 데이터양을 자동으로 줄여 확대/축소 및 이동 시 쾌적한 환경을 제공합니다.

### 4) PyDeck(deck.gl)을 이용한 GPU 가속 지리 공간 헥스빈 및 3D 시각화

**문제:** 수만 개의 위치 기반 이벤트를 매핑하면 속도가 느려지거나 디자인이 조잡해집니다.  
**해결책:** WebGL 기반의 deck.gl을 사용하여 빠르고 아름다운 시각화를 구현합니다.

```python
import pydeck as pdk
import pandas as pd

df = pd.read_csv("gps.csv")
layer = pdk.Layer(
    "HexagonLayer",
    df,
    get_position=["lon", "lat"],
    radius=200,
    elevation_scale=50,
    pickable=True,
)
r = pdk.Deck(layers=[layer], initial_view_state=pdk.ViewState(latitude=..., longitude=..., zoom=10))
r.to_html("map.html")
```

*   **특징:** 정적인 지도 대신 GPU 가속을 통한 헥스 집계(hex-aggregation)와 3D 높이 표현이 가능합니다. 모빌리티 데이터나 히트맵 분석에 이상적입니다.

### 5) Rich + Textual을 이용한 터미널 대시보드 구축

**문제:** GUI가 필요 없거나 SSH 세션, tmux 환경에서 즉각적인 모니터링 대시보드가 필요할 때가 있습니다.  
**해결책:** 터미널 사용자 인터페이스(TUI) 라이브러리를 사용하여 터미널 내에서 실행되는 대시보드를 만듭니다.

```python
from rich.console import Console
from rich.table import Table

console = Console()
table = Table()
table.add_column("metric")
table.add_column("value", justify="right")
table.add_row("requests/s", "235")
table.add_row("errors", "0.3%")
console.print(table)
```

*   **특징:** 시각화가 반드시 브라우저에서 이루어질 필요는 없습니다. 터미널 대시보드는 반복 작업 속도가 빠르고 SSH를 통해 공유하기 쉬우며, 라이브 시스템 데모 시 매우 효과적입니다.

### 6) VisPy를 이용한 임베딩 데이터의 GPU 가속 시각화

**문제:** 수십만 개의 데이터 포인트가 포함된 2D/3D 임베딩(embeddings)을 부드럽게 탐색하고 싶을 때 성능 한계에 부딪힙니다.  
**해결책:** OpenGL을 직접 사용하는 VisPy를 통해 네이티브 GPU 성능을 활용합니다.

```python
import vispy.scene
from vispy import app
import numpy as np

canvas = vispy.scene.SceneCanvas(keys='interactive', show=True)
view = canvas.central_widget.add_view()
pts = vispy.scene.visuals.Markers()
pts.set_data(np.random.randn(200000, 2), face_color=(1,1,1,0.5), size=2)
view.add(pts)
app.run()
```

*   **특징:** Matplotlib이나 Plotly가 처리하기 힘든 대규모 산점도 탐색 작업에서 뛰어난 반응성을 보여줍니다.

### 7) mplcursors를 이용한 Matplotlib 호버 주석 기능 추가

**문제:** 정적인 이미지 형태의 차트에서는 특정 데이터 포인트가 무엇인지 즉시 파악하기 어렵습니다.  
**해결책:** 무거운 자바스크립트(JS) 도구 없이도 Matplotlib에 가벼운 상호작용 기능을 추가합니다.

```python
import matplotlib.pyplot as plt
import mplcursors

fig, ax = plt.subplots()
sc = ax.scatter(x, y)
mplcursors.cursor(sc, hover=True).connect(
    "add", lambda sel: sel.annotation.set_text(f"x={x[sel.index]:.2f}\ny={y[sel.index]:.2f}")
)
plt.show()
```

*   **특징:** 복잡한 인터랙티브 스택으로 넘어가지 않고도 노트북 환경에서 간단하게 탐색적 분석을 수행할 수 있게 해주는 가벼운 도구입니다.

### 8) Vaex를 이용한 메모리 초과 대용량 테이블 시각화

**문제:** 100GB가 넘는 CSV 파일을 클러스터 없이 분석하고 시각화해야 합니다.  
**해결책:** 메모리 매핑(memory mapping)과 지연 평가(lazy evaluation)를 사용하는 Vaex를 활용합니다.

```python
import vaex
ds = vaex.open("huge_dataset.hdf5")   # 메모리 매핑 방식으로 처리
ds.plot1d(ds.x, binby=ds.y, limits='minmax')  # 빠른 히스토그램
ds.plot(ds.x, ds.y, f='log', shape=512)       # 빠른 2D 밀도 그래프
```

*   **특징:** Pandas와 달리 메모리에 데이터를 모두 올리지 않고도 대규모 데이터셋에 대한 통계 수치와 시각화를 즉각적으로 생성합니다. 로그 데이터 분석이나 대규모 텔레메트리 데이터 처리에 매우 효율적입니다.
