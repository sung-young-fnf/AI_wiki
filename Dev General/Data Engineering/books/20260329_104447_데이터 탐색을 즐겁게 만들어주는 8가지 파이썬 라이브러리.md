`Sora로 생성된 이미지`

가공되지 않은 데이터(raw data)는 솔직합니다.
하지만 무례하고, 일관성이 없으며, 마치 스키마(schema)를 한 번도 경험해 본 적 없는 사람이 내보낸 CSV 파일 같은 냄새를 풍기기도 합니다.

로그, 스크랩된 데이터, 불완전한 분석 내보내기 파일 등 실제 데이터셋(dataset)을 4년간 다루면서 저는 불편한 진실 하나를 깨달았습니다.

데이터 탐색(data exploration)의 대부분 어려움은 판다스(pandas)가 제 역할을 하기도 전에 발생합니다.

여기에 그 어려운 부분을 조용히 해결해 주는 8가지 파이썬(Python) 라이브러리(library)가 있습니다.
화려하지 않습니다. 과장된 장난감도 아닙니다. 이 라이브러리들은 당신이 이미 능숙해졌지만, 살짝 지쳐 있을 때 비로소 발견하게 될 도구들입니다.

### 1. ydata-profiling (이전 이름: pandas-profiling)

**추측 없이 탐색하기**

이런 과정을 잘 아실 겁니다:
`df.head()`, `df.describe()`, 결측치(missing values) 확인, `dtype` 혼란.

이 라이브러리는 그 모든 루틴을 한 줄로 압축하면서도, 당신이 미처 생각지 못했던 것들까지 보여줍니다.

```python
import pandas as pd
from ydata_profiling import ProfileReport

df = pd.read_csv("data.csv")
profile = ProfileReport(df, title="Dataset Report", explorative=True)
profile.to_file("report.html")
```

**사람이 놓치기 쉬운 점:**

*   제거하는 것을 잊은 상수(constant) 열(column)
*   ID인 척하는 고(高)카디널리티(high-cardinality) '범주형(categorical)' 열
*   나중에 모델(model)을 조용히 망가뜨리는 왜도(skewness)

**왜 중요할까요?**
저는 이 라이브러리가 단 하나의 모델도 훈련하기 전에 데이터 유출(data leakage)을 잡아내는 것을 본 적이 있습니다. 그것 하나만으로도 이 라이브러리의 존재 가치는 충분합니다.

### 2. Sweetviz

**EDA가 의례적인 절차가 아닌 시각적으로 느껴질 때**

Sweetviz는 40줄짜리 Matplotlib(맷플롯립) '접착(glue)' 코드를 작성하지 않고도 나란히 비교하는 보고서(report)를 생성합니다.

```python
import sweetviz as sv
import pandas as pd

df = pd.read_csv("train.csv")
report = sv.analyze(df)
report.show_html("sweetviz.html")
```

**어떤 점이 다를까요?**

*   자동 대상(target) 분석
*   실질적으로 의미 있는 특성(feature)별 분석
*   복잡한 설정의 늪이 없음

**개인적인 의견:**
회의 5분 전에 누군가 데이터셋을 넘겨줄 때 사용합니다. 빠르게 신뢰를 얻을 수 있죠.

### 3. missingno

**결측치(missing data)는 절대 무작위가 아니기 때문입니다 (사람들이 그렇다고 맹세하더라도요)**

결측치는 이야기를 들려줍니다. 불쾌한 이야기들을요.

```python
import missingno as msno
import pandas as pd

df = pd.read_csv("data.csv")
msno.matrix(df)
msno.heatmap(df)
```

**이를 통해 알 수 있는 것:**

*   결측치가 상관관계(correlated)가 있는지
*   전체 데이터 블록(block)이 함께 사라졌는지
*   시간 기반의 공백(gap)이 존재하는지

**냉정한 진실:**
결측치를 시각화(visualize)하지 않으면, 당신은 추측하는 것입니다. 그리고 추측은 확장이 어렵습니다(scales badly).

---

### 4. lux

**호기심에 반응하는 EDA**

Lux는 당신의 작업을 관찰하고 다음에 무엇을 살펴봐야 할지 제안합니다.

```python
import lux
import pandas as pd

df = pd.read_csv("data.csv")
df # 네, 그냥 이렇게만 표시하면 됩니다.
```

**갑자기 다음과 같은 것들을 얻게 됩니다:**

*   “흥미로운” 관계
*   자동 시각화(visualization) 추천
*   명시적으로 요구하지 않아도 비교 가능

**왜 드물까요?**
대부분의 라이브러리는 지시를 기다립니다. Lux는 마치 당신 옆에 앉아 있는 호기심 많은 분석가처럼 행동합니다.

### 5. dtale

**판다스(pandas)의 대화형(interactive)이고 정직한 버전**

거짓말하지 않는 스프레드시트(spreadsheet)라고 생각해보세요.

```python
import dtale
import pandas as pd

df = pd.read_csv("data.csv")
dtale.show(df)
```

**다음과 같은 작업을 할 수 있습니다:**

*   실시간으로 필터링(filter), 정렬(sort) 및 편집
*   UI(사용자 인터페이스)에서 판다스(pandas) 작업 실행
*   출력물(prints)을 찌푸리며 볼 필요 없이 이상점(anomalies)을 시각적으로 파악

**사용 사례:**
이해관계자들이 "잠깐만 보게 해달라"고 할 때, 엑셀(Excel)로 내보내어 영혼을 잃고 싶지 않을 때 유용합니다.

### 6. pyjanitor

**부끄러움 없는 데이터 클리닝(data cleaning)**

대부분의 데이터셋은 탐색되지 않습니다. 오히려 사과해야 할 대상이 됩니다.

```python
import pandas as pd
import janitor

df = (
    pd.read_csv("data.csv")
      .clean_names()
      .remove_empty()
      .filter_on("age", lambda x: x > 0)
)
```

**이 라이브러리가 해결하는 문제:**

*   열 이름의 혼란
*   반복적인 상용구(boilerplate) 코드
*   가독성을 해치는 문제

**전문가들이 이 라이브러리를 좋아하는 이유:**
깔끔한 코드는 패턴을 명확하게 보여줍니다. 지저분한 파이프라인(pipeline)은 프로덕션(production) 환경에 이르러서야 버그(bug)를 드러냅니다.

---

### 7. skimpy

**당신의 시간을 존중해 주는 `describe()` 함수**

```python
from skimpy import skim
import pandas as pd

df = pd.read_csv("data.csv")
skim(df)
```

**다음과 같은 것을 얻을 수 있습니다:**

*   사람이 읽기 쉬운 요약
*   타입(type)을 인지하는 통계
*   숫자형(numeric), 범주형(categorical), 날짜/시간(datetime) 특성(feature)의 명확한 구분

**사실:**
Skimpy는 `df.describe()`가 혼합 타입(mixed-type) 데이터셋에서 아무런 경고 없이 실패한다는 문제 때문에 특별히 개발되었습니다. 이는 결코 작은 문제가 아닙니다.

---

### 8. whylogs

**어제를 기억하는 데이터 탐색**

대부분의 EDA(탐색적 데이터 분석)는 일회성입니다. 이것이 문제입니다.

```python
import whylogs as why
import pandas as pd

df = pd.read_csv("data.csv")
profile = why.log_dataframe(df)
profile.view()
```

**Whylogs는 다음을 추적합니다:**

*   데이터 드리프트(data drift)
*   스키마(schema) 변경
*   시간 경과에 따른 통계적 변화

**이것이 바로 비약적인 발전입니다:**
데이터를 한 번 탐색하는 것으로 그치는 것이 아니라, 그 동작을 지속적으로 모니터링(monitoring)하기 시작하는 것이죠.

저는 '작은 상위 변경(upstream change)'이 2주 후에 프로덕션 모델(production model)을 망가뜨리기 전까지는 이 가치를 제대로 알지 못했습니다.
