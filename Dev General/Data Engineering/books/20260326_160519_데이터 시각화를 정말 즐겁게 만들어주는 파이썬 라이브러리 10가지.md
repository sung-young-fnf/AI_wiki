# 데이터 시각화를 정말 즐겁게 만들어주는 파이썬 라이브러리 10가지

사람들이 끊임없이 제게 물어보는 차트들을 만드는 라이브러리입니다.

저는 독자분들에 대해 두 가지 중 하나를 가정하려 합니다.

첫째, 파이썬(Python)으로 수년간 데이터를 시각화해왔지만, 여전히 차트가 마치 엑셀(Excel) 2007에서 튀어나온 것처럼 보인다는 것입니다.
둘째, 맷플롯립(Matplotlib)을 너무 잘 알아서 괜찮은 레이블(label)이나 색상 같은 기본적인 것들을 설정하느라 씨름하는 데 지쳤다는 것입니다.

저도 오랫동안 그런 경험을 했습니다.

데이터 시각화는 통찰력을 드러내기 위한 것이지, 깔끔한 차트를 만들려 노력하는 사람에게 벌을 주기 위한 것이 아닙니다. 지난 4년 넘게 파이썬을 전문적으로 사용하면서, 저는 인정하고 싶지 않을 만큼 많은 플로팅(plotting) 라이브러리를 테스트했습니다. 대부분은 버려지거나, 미완성이거나, 뭔가 하나를 사용자 정의(customize)하려고 하면 '멋짐'이 사라지는 것들이었습니다.

하지만 이 10가지는 다릅니다.

제가 다음과 같은 차트를 만들고 싶을 때 사용하는 라이브러리들입니다.

*   진정으로 스토리를 전달하고
*   50줄의 스타일링 코드 없이도 멋지게 보이며
*   사람들이 "잠깐만요… 저 라이브러리가 뭐죠?"라고 묻게 만드는 차트들

이제 본론으로 들어가 봅시다.

### 1. Seaborn (네, 여전히 — 하지만 제대로 사용해야 합니다)

모두가 시본(Seaborn)을 알고 있지만, 제대로 사용하는 사람은 극히 드뭅니다.

시본은 맷플롯립(Matplotlib) 기반으로 구축되었지만, 통계적 의도를 API에 인코딩(encoding)합니다. 이를 단순히 '예쁜 맷플롯립'으로 여기지 않고 통계 시각화 도구로 다루기 시작하면, 그 진가가 발휘됩니다.

```python
import seaborn as sns
import matplotlib.pyplot as plt

tips = sns.load_dataset("tips")

sns.violinplot(
    data=tips,
    x="day",
    y="total_bill",
    inner="quartile",
    cut=0
)

plt.title("Bill Distribution by Day (With Quartiles)")
plt.show()
```

**왜 중요할까요?**
이 차트는 분포, 확산(spread), 그리고 이상치(outlier)를 한눈에 보여줍니다. 대부분의 대시보드(dashboard)는 여전히 여기에 막대 차트(bar chart)를 사용하는데, 이는 잘못된 관행입니다.

**전문가 팁:** 플롯(plot)이 분포, 불확실성, 또는 비교를 포함한다면 — 시본이 가장 먼저 떠올라야 합니다.

### 2. Plotly (상호작용성(Interactivity)이 실제로 가치를 더할 때)

플로틀리(Plotly)는 단순히 '호버 툴팁(hover tooltip)'을 넘어섭니다. 이는 완전한 선언형 시각화 시스템(declarative visualization system)입니다.

**무엇이 다를까요?**
당신의 차트는 정적인 이미지가 아닌 대화의 장이 됩니다.

```python
import plotly.express as px
df = px.data.gapminder().query("year == 2007")

fig = px.scatter(
    df,
    x="gdpPercap",
    y="lifeExp",
    size="pop",
    color="continent",
    hover_name="country",
    log_x=True,
    title="GDP vs Life Expectancy (2007)"
)

fig.show()
```

사람들이 이 차트에 대해 묻는 것은 단지 색상이 화려해서가 아닙니다.
그들은 차트를 직접 탐색할 수 있기 때문에 묻습니다.

**사실:** 플로틀리 차트는 백엔드(backend) 없이도 직접 HTML로 내보내고 임베드(embed)할 수 있습니다. 이는 엄청나게 저평가된 기능입니다.

---

### 3. Altair (제대로 된 그래픽 문법(Grammar of Graphics))

알테어(Altair)는 시각화를 함수 더미가 아닌 언어처럼 다룰 때 비로소 드러나는 진면목입니다.

어떻게 그릴지(how to draw)가 아니라 무엇을 원하는지(what you want)를 설명합니다.

```python
import altair as alt
from vega_datasets import data

source = data.cars()

chart = alt.Chart(source).mark_circle(size=60).encode(
    x='Horsepower',
    y='Miles_per_Gallon',
    color='Origin',
    tooltip=['Name', 'Horsepower', 'Miles_per_Gallon']
).interactive()

chart
```

**알테어가 특별한 이유:**
알테어는 설계 단계부터 좋은 시각화 관행을 강제합니다. 통계적으로 말이 안 되는 부분이 있다면, 알테어는 종종 차트를 그리는 것을 거부합니다.

**그 마찰이요?**
그것이 당신을 더 나은 시각화 전문가로 만듭니다.

### 4. Bokeh (진정한 웹 우선(Web-First) 시각화를 위해)

플로틀리가 '대화형 차트(interactive chart)'라면, 보케(Bokeh)는 대화형 시스템(interactive system)입니다.

슬라이더(slider). 필터(filter). 연결된 플롯. 실시간 업데이트(live update).

```python
from bokeh.plotting import figure, show
from bokeh.io import output_notebook

output_notebook()

p = figure(title="Simple Line Example", width=600)
p.line([1, 2, 3, 4], [6, 7, 2, 4], line_width=3)

show(p)
```

보케는 다음과 같은 경우에 탁월합니다.

*   대시보드가 사용자 정의 동작을 필요로 할 때
*   상호작용성(interactivity)이 선택 사항이 아닐 때
*   성능이 중요할 때

저는 페이지 새로 고침이 용납되지 않는 내부 도구에 보케를 사용해봤습니다.

### 5. PyDeck (장난감처럼 보이지 않는 지도)

대부분의 지도 시각화는 보기에 좋지 않거나 오해를 불러일으킵니다. 파이덱(PyDeck)은 이 두 가지 모두를 피합니다.

우버(Uber)의 deck.gl을 기반으로 구축되어 수백만 개의 포인트를 부드럽게 처리합니다.

```python
import pydeck as pdk
import pandas as pd

df = pd.DataFrame({
    "lat": [37.7749, 34.0522],
    "lon": [-122.4194, -118.2437],
    "value": [100, 200]
})

layer = pdk.Layer(
    "ScatterplotLayer",
    df,
    get_position='[lon, lat]',
    get_radius=50000,
    get_fill_color='[200, 30, 0, 160]',
)

view_state = pdk.ViewState(latitude=36, longitude=-120, zoom=5)

deck = pdk.Deck(layers=[layer], initial_view_state=view_state)
deck
```

**솔직히 말해서:** 지리공간 데이터(geospatial data)를 시각화하면서 아직도 정적인 지도(static map)를 사용하고 있다면, 통찰력을 버리고 있는 것이나 다름없습니다.

### 6. Missingno (가장 정직한 시각화 라이브러리)

무엇이든 플로팅하기 전에, 무엇이 누락되었는지 알아야 합니다.

미싱노(Missingno)는 제가 본 어떤 것보다도 널(null) 값을 훌륭하게 시각화합니다.

```python
import missingno as msno
import pandas as pd

df = pd.read_csv("data.csv")

msno.matrix(df)
```

이 하나의 차트 덕분에 저는 며칠간의 디버깅(debugging) 시간을 절약할 수 있었습니다.

**왜 잘 사용되지 않을까요?**
대부분의 사람들은 모델링(modeling)으로 곧장 넘어갑니다. 미싱노는 당신이 현실을 먼저 직시하도록 만듭니다.

### 7. Matplotlib (하지만 고통스럽지 않은 스타일로)

순수한 맷플롯립(Matplotlib)은 사용하기 힘들지만, 스타일이 적용된 맷플롯립은 강력합니다.

```python
import matplotlib.pyplot as plt
import numpy as np

plt.style.use("seaborn-v0_8-darkgrid")

x = np.linspace(0, 10, 100)
y = np.sin(x)

plt.plot(x, y)
plt.title("Styled Matplotlib Without the Pain")
plt.show()
```

맷플롯립은 여전히 파이썬 시각화의 중추입니다.
이를 피하기보다 길들이는 방법을 배우는 것이 장기적으로 큰 도움이 됩니다.

### 8. Squarify (의미 있는 계층 구조를 위해)

트리맵(Treemap)은 지나치게 저평가되어 있습니다.

```python
import squarify
import matplotlib.pyplot as plt

sizes = [40, 30, 20, 10]
labels = ["A", "B", "C", "D"]

squarify.plot(sizes=sizes, label=labels)
plt.axis("off")
plt.show()
```

다음 경우에 완벽합니다.

*   디스크 사용량
*   예산 세부 내역
*   특성 중요도(feature importance)

데이터가 전체의 부분을 구성한다면, 다른 파이 차트(pie chart)보다 이것을 고려해 보세요.

### 9. PyVis (눈물 없이 만드는 네트워크 그래프)

네트워크 그래프(network graph)는 어렵습니다. 파이비스(PyVis)는 이를… 견딜 만하게 만듭니다.

```python
from pyvis.network import Network
net = Network(notebook=True)

net.add_node(1, label="Python")
net.add_node(2, label="Data")
net.add_edge(1, 2)

net
```

이것은 다음 용도에 매우 좋습니다.

*   의존성 그래프(dependency graph)
*   추천 시스템(recommendation system)
*   소셜 네트워크(social network)

그리고 물론 — 즉시 상호작용이 가능합니다.

### 10. Rich (터미널 시각화가 저평가된 이유)

모든 것이 브라우저(browser)를 필요로 하는 것은 아닙니다.

```python
from rich.console import Console
from rich.progress import track
import time

console = Console()

for step in track(range(20), description="Processing data..."):
    time.sleep(0.1)
```

리치(Rich)는 로그(log), 테이블(table), 진행률 표시줄(progress bar)을 실제로 보고 싶어 하는 형태로 바꿔줍니다.

저는 이것을 매일 사용합니다.
