파이썬(Python)으로 데이터를 시각화할 때, 대부분의 사람들은 늘 사용하는 몇 가지 차트 유형에 의존합니다. 산점도(scatter plot), 히스토그램(histogram), 막대 차트(bar chart), 라인 플롯(line plot)은 많은 경우에 유용하지만, 때로는 이러한 익숙한 차트로는 데이터가 담고 있는 이야기를 명확하게 전달하기 어려울 때가 있습니다.

저를 오랫동안 팔로우하셨다면, 제가 데이터 시각화를 다루는 것을 즐겨한다는 것을 아실 겁니다. 때로는 지루하고 평범한 시각화를 유용하고 훨씬 보기 좋게 바꾸곤 합니다.

이 글에서는 더 많은 주목을 받을 가치가 있다고 생각하는 세 가지 차트 유형을 소개하고자 합니다. 각 차트는 우리가 일반적으로 의존하는 기본 차트보다 특정 시각화 문제를 더 효과적으로 해결합니다. 각 차트에 대해 합성 데이터(synthetic data)를 사용하여 "이전"에는 표준 차트를, "이후"에는 대체 차트를 보여드리며 설명할 것입니다.

다룰 세 가지 차트는 다음과 같습니다.

*   시간이 지남에 따라 순위가 어떻게 변하는지 추적하기 위한 **범프 차트(Bump Charts)**
*   여러 그룹 간의 분포를 비교하기 위한 **릿지라인 플롯(Ridgeline Plots)**
*   요약 통계가 숨기는 개별 데이터 포인트를 보여주기 위한 **비스웜 플롯(Beeswarm Plots)**

## 1. 범프 차트(Bump Charts): 시간 경과에 따른 순위 변화 추적

### 문제점

여러 시간대에 걸쳐 다양한 항목의 순위가 어떻게 매겨졌는지 보여주는 데이터가 있다고 상상해 보세요. 표준 라인 플롯(line plot)을 사용할 수도 있지만, 선들이 서로 교차하기 시작하면 상황이 빠르게 복잡해집니다. 개별 선을 따라가기 어려워지고 차트는 스파게티 접시처럼 보이게 됩니다.

범프 차트(Bump charts)는 원시 값(raw values)이 아닌 순위 위치에 초점을 맞춰 이 문제를 해결합니다. 부드러운 베지에 곡선(Bézier curves)은 각 항목이 순위에서 오르내리는 움직임을 쉽게 따라갈 수 있도록 해줍니다.

### 합성 데이터(Synthetic Data) 예시

이 예시에서는 5년 동안 일련의 유정(well)들이 서로 어떻게 순위가 매겨졌는지 추적할 것입니다. 이는 생산량 또는 상대적 위치가 중요한 다른 어떤 측정 기준일 수도 있습니다. 각 유정에는 매년 순위가 할당되며, 1위가 가장 우수한 유정입니다.

```python
years = [2019, 2020, 2021, 2022, 2023]

measurements = {
    'Well A': [1, 2, 1, 1, 2],
    'Well B': [2, 1, 2, 3, 1],
    'Well C': [3, 3, 4, 2, 3],
    'Well D': [4, 4, 3, 4, 4],
    'Well E': [5, 6, 5, 5, 6],
    'Well F': [7, 5, 6, 7, 5],
    'Well G': [6, 7, 7, 6, 7],
}
```

### 표준 라인 플롯 (이전)

여기서 본능적으로 라인 플롯을 사용하게 됩니다.

```python
fig, ax = plt.subplots(figsize=(10, 6))
for measurement, ranks in measurements.items():
    ax.plot(years, ranks, marker='o', label=measurement,
            markerfacecolor='white', linewidth=2)
ax.set_ylabel('Rank')
ax.set_xlabel('Year')
ax.set_title('Well Ranking Over Time')
ax.invert_yaxis()
ax.legend(bbox_to_anchor=(1.05, 1), loc='upper left')
ax.set_yticks(range(1, 8))
plt.tight_layout()
plt.show()
```

이 방법은 작동하지만, 선들이 교차하면서 얼마나 빠르게 따라가기 어려워지는지 알 수 있습니다. 2020년에 유정 A와 유정 B가 위치를 바꾸고, 2021년에 유정 C가 다시 떨어질 때, 개별 유정의 움직임을 추적하는 데는 상당한 노력이 필요합니다.

범프 플롯을 흉내 낸 라인 플롯. 작동은 하지만, 더 보기 좋게 만들 수 있습니다.

### 범프 차트 (이후)

아래 범프 차트는 동일한 데이터를 사용하지만, 직선 세그먼트를 부드러운 베지에 곡선(Bézier curves)으로 대체하고 각 선에 레이블을 오른쪽 가장자리에 직접 표시합니다.

한 가지 유의할 점: `bumplot`은 데이터를 특정 형태, 즉 시간 축을 인덱스(index)로 하는 데이터프레임(DataFrame) 내의 순위 값으로 예상합니다. 만약 원시 데이터가 순위 형태가 아닌 값 형태인 경우, 아래의 `rank()` 호출이 변환을 처리합니다.

```python
from bumplot import bumplot

df_bump = pd.DataFrame(measurements, index=years)
df_bump.index.name = 'Year'

# bumplot ranks higher values as rank 1 (top), but our data stores ranks
# where 1=best (lowest number). Invert so bumplot computes the correct order.
n_wells = len(measurements)
df_plot = df_bump.apply(lambda col: n_wells + 1 - col)

fig, ax = plt.subplots(figsize=(12, 7))
bumplot(
    x='Year',
    y_columns=sorted(measurements.keys()),
    data=df_plot.reset_index(),
    curve_force=0.5,
    plot_kwargs={'lw': 3.5},
    scatter_kwargs={'s': 120, 'ec': 'white', 'lw': 2, 'zorder': 5},
)

# Force y-axis: rank 1 at top, rank 7 at bottom
ax.set_ylim(7.5, 0.5)
ax.set_yticks(range(1, 8))
ax.set_ylabel('Rank', fontsize=12)
ax.set_xlabel('')
ax.set_title('Production Well Oil Output Ranking Over Time',
             fontsize=14, fontweight='bold', pad=15)


for well in sorted(measurements.keys()):
    final_rank = df_bump.loc[2023, well]
    ax.text(2023.15, final_rank, well,
            va='center', fontsize=9, fontweight='bold')

ax.spines[['top', 'right']].set_visible(False)
ax.legend().set_visible(False)
plt.tight_layout()
plt.show()
```

이제 범례(legend)를 교차 참조할 필요 없이 5년 동안 개별 유정의 움직임을 따라갈 수 있습니다. 순위 상위권에서 유정 A와 유정 B가 주고받는 싸움이 즉시 파악됩니다.

시간이 지남에 따른 유정의 원유 생산량 순위를 보여주는 범프 플롯(Bumplot).

### 범프 차트(Bump Charts) 사용 시점

범프 차트는 절대적인 값보다 상대적인 위치에 관심이 있을 때 가장 효과적입니다. 리그 테이블(league tables), 설문조사 선호도 순위, 또는 우리의 예시처럼 다양한 유정의 생산 속도가 유전(field)이나 시간에 따라 어떻게 변화했는지 추적하는 데 이상적입니다. 항목이 10개 이상일 경우 차트가 복잡해질 수 있으므로, 상위 성과자들에게 초점을 맞추는 것을 고려해 보세요.

## 2. 릿지라인 플롯(Ridgeline Plots): 분포를 한눈에 비교하기

### 문제점

여러 그룹에 걸쳐 측정값의 분포를 비교하는 것은 암석 물리학(petrophysics)에서 끊임없이 수행하는 작업입니다. 일반적인 접근 방식은 중첩된 히스토그램(overlapping histograms)을 그리는 것이지만, 두세 그룹을 넘어서면 중첩 때문에 차트를 읽을 수 없게 됩니다.

릿지라인 플롯(Ridgeline plots) (때로는 조이 디비전(Joy Division) 앨범 커버에서 이름을 따와 조이 플롯(joy plots)이라고도 불림)은 부분적으로 중첩된 밀도 곡선(density curves)을 수직으로 쌓아 올립니다. 각 그룹은 자체 행을 가지지만, 동일한 수평 축을 공유하여 비교를 쉽게 만듭니다.

### 합성 데이터(Synthetic Data) 예시

우리는 여섯 가지 다른 지층(formation)에 걸쳐 감마선(gamma ray) 측정값을 시뮬레이션할 것입니다. 각 지층은 무작위적이고 다른 평균과 확산을 가집니다. 이는 단일 유정 내에서 암석학(lithology)을 특성화할 때 접할 수 있는 종류의 데이터를 반영합니다.

```python
np.random.seed(42)

formations = {
    'Balder Fm': np.random.normal(loc=45, scale=12, size=500),
    'Sele Fm': np.random.normal(loc=85, scale=15, size=500),
    'Lista Fm': np.random.normal(loc=70, scale=20, size=500),
    'Tor Fm': np.random.normal(loc=25, scale=8, size=500),
    'Draupne Fm': np.random.normal(loc=110, scale=18, size=500),
    'Heather Fm': np.random.normal(loc=65, scale=22, size=500),
}
df_ridge = pd.DataFrame({
    'GR': np.concatenate(list(formations.values())),
    'Formation': np.concatenate([[name] * len(values) 
                                  for name, values in formations.items()])
})
```

### 중첩된 히스토그램 (이전)

```python
fig, ax = plt.subplots(figsize=(10, 6))

for formation in formations:
    subset = df_ridge[df_ridge['Formation'] == formation]
    ax.hist(subset['GR'], bins=50, alpha=0.4, label=formation)
ax.set_xlabel('Gamma Ray (API)')
ax.set_ylabel('Frequency')
ax.set_title('Gamma Ray Distribution by Formation')
ax.legend()
plt.tight_layout()
plt.show()
```

여섯 가지 지층으로 인해 중첩된 히스토그램은 빠르게 복잡해집니다. 낮은 쪽의 Tor Fm과 높은 쪽의 Draupne Fm 같은 극단적인 값은 구별할 수 있지만, 중간에 있는 것은 모두 파악하기 어렵습니다. 투명도(alpha)를 줄이면 약간 도움이 되지만 근본적인 문제는 해결되지 않습니다.

다양한 지질 지층에 대한 감마선(Gamma Ray) 값의 중첩 막대 플롯. 여기 사용된 데이터는 합성 데이터입니다.

### 릿지라인 플롯 (이후)

아래 릿지라인 플롯은 x축을 공유하면서 각 지층에 자체 행을 부여합니다. `overlap` 매개변수는 각 행이 위에 있는 행으로 얼마나 번져 들어가는지를 제어하며, 0.4 값은 압축성과 가독성 사이의 좋은 균형을 제공합니다. 여기에서 `viridis` 컬러맵(colormap)은 각 지층을 시각적으로 구분하는 데 도움이 되지만, 선호도에 따라 어떤 순차적 또는 범주형 팔레트(palette)든 사용할 수 있습니다.

```python
import joypy

fig, axes = joypy.joyplot(
    df_ridge,
    by='Formation',
    column='GR',
    figsize=(10, 7),
    colormap=plt.cm.viridis,
    alpha=0.7,
    linewidth=1.2,
    overlap=0.4,
    x_range=(-20, 170),
)
plt.xlabel('Gamma Ray (API)', fontsize=12)
plt.title('Gamma Ray Distribution by Formation',
          fontsize=14, fontweight='bold', y=1.02)
plt.tight_layout()
plt.show()
```

여러 지질 지층에 대한 데이터 분포를 보여주는 릿지라인 플롯.

### 릿지라인 플롯(Ridgeline Plots) 사용 시점

릿지라인 플롯은 여섯 개 이상의 그룹이 있고 이들의 분포를 나란히 비교하고자 할 때 탁월한 성능을 발휘합니다. 이들은 간결하고, 시각적으로 독특하며, 확장이 용이합니다.

이 내용이 즐거우셨다면, 어떤 유형의 차트든 개선할 수 있는 네 가지 간단한 변경 사항에 대해 다룬 서브스택(Substack) 글을 제가 작성했습니다. 링크는 마지막에 있습니다.

## 3. 비스웜 플롯(Beeswarm Plots): 모든 데이터 포인트 표시하기

### 문제점

박스 플롯(Box plots)은 데이터 탐색의 주요 도구입니다. 중앙값(median), 사분위수(quartiles), 이상치(outliers)를 간결한 형태로 보여줍니다. 하지만 실제 데이터는 숨겨집니다. 15개의 데이터 포인트가 있는 박스 플롯은 1,500개의 데이터 포인트가 있는 박스 플롯과 동일하게 보이며, 개별 측정값들이 어디에 집중되어 있는지에 대한 감각을 모두 잃게 됩니다.

비스웜 플롯(Beeswarm plots)은 모든 개별 데이터 포인트가 겹치지 않도록 위치를 조정하여 표시합니다. 그 결과는 벌떼처럼 보이며, 단일 보기에서 분포 형태와 원시 데이터를 모두 제공합니다.

### 합성 데이터(Synthetic Data) 예시

우리는 네 가지 암석 유형(lithology)에 걸쳐 코어 공극률(core porosity) 측정값을 시뮬레이션할 것입니다. 코어 데이터는 표본 크기가 작고 모든 데이터 포인트가 실제 가치를 가지는 고전적인 경우입니다. 여기서는 각 측정값을 요약 통계에 흡수시키는 것이 아니라 개별적으로 보고 싶을 것입니다.

```python
np.random.seed(42)
lithologies = {
    'Sandstone': np.random.normal(loc=22, scale=4, size=30),
    'Limestone': np.random.normal(loc=12, scale=3, size=25),
    'Dolomite': np.random.normal(loc=8, scale=2.5, size=20),
    'Shale': np.random.normal(loc=5, scale=2, size=35),
}

df_swarm = pd.DataFrame({
    'Porosity (%)': np.concatenate(list(lithologies.values())),
    'Lithology': np.concatenate([[name] * len(values) 
                                  for name, values in lithologies.items()])
})

df_swarm['Porosity (%)'] = df_swarm['Porosity (%)'].clip(lower=0, upper=35)
```

### 박스 플롯 (이전)

```python
fig, ax = plt.subplots(figsize=(8, 6))
sns.boxplot(x='Lithology', y='Porosity (%)', data=df_swarm,
            palette='Set2', ax=ax)
ax.set_title('Core Porosity by Lithology')
plt.tight_layout()
plt.show()
```

아래 박스 플롯은 완벽하게 읽을 수 있지만, 무엇을 알려주지 않는지 주목하세요. 사암(sandstone) 측정값들이 중앙값(median) 주위에 밀집되어 있나요, 아니면 전체 사분위 범위(interquartile range)에 걸쳐 퍼져 있나요? 셰일(shale) 데이터에 숨겨진 이봉 클러스터(bimodal cluster)가 있나요? 박스 플롯은 이러한 질문에 답할 수 없습니다.

다양한 암석 유형(lithology)에 대한 공극률(porosity) 범위와 이상치(outlier)를 보여주는 박스 플롯.

### 비스웜 플롯 (이후)

```python
fig, ax = plt.subplots(figsize=(8, 6))
sns.swarmplot(x='Lithology', y='Porosity (%)', data=df_swarm,
              palette='Set2', size=7, edgecolor='black',
              linewidth=0.5, ax=ax)
ax.set_title('Core Porosity by Lithology',
             fontsize=14, fontweight='bold')
ax.spines[['top', 'right']].set_visible(False)
plt.tight_layout()
plt.show()
```

비스웜 플롯을 사용하면 110개의 모든 데이터 포인트가 보입니다. 사암(sandstone) 측정값은 범위에 걸쳐 적당히 퍼져 있는 반면, 백운암(dolomite)과 셰일(shale) 측정값은 밀집되어 있음을 즉시 알 수 있습니다. 이것이 박스 플롯이 숨기고 있던 정보입니다.

`size` 매개변수는 포인트의 직경을 제어합니다. 이 범위의 데이터셋에서는 7이 잘 작동하지만, 그룹당 포인트가 더 많아서 비스웜이 너무 넓어지는 것을 피하려면 값을 줄이는 것이 좋습니다.

다양한 암석 유형(lithology)에 대한 코어 공극률(core porosity) 값을 보여주는 비스웜 플롯.

### 두 가지 결합: 두 세계의 장점

박스 플롯의 요약 통계와 비스웜 플롯의 투명성을 한 번에 보고 싶다면, 이들을 직접 겹쳐서 사용할 수 있습니다. 핵심은 박스 플롯의 투명도(alpha)를 낮춰 시선을 끌기보다 배경 컨텍스트(background context)로 읽히도록 하는 것입니다. 중앙값(median) 선은 완전한 불투명도를 유지합니다.

```python
fig, ax = plt.subplots(figsize=(8, 6))

sns.boxplot(x='Lithology', y='Porosity (%)', data=df_swarm,
            palette='Set2', ax=ax, 
            boxprops=dict(alpha=0.3),
            whiskerprops=dict(alpha=0.5),
            capprops=dict(alpha=0.5),
            medianprops=dict(color='black', linewidth=2))
sns.swarmplot(x='Lithology', y='Porosity (%)', data=df_swarm,
              palette='Set2', size=6, edgecolor='black',
              linewidth=0.5, ax=ax)
ax.set_title('Core Porosity by Lithology\\n(Box Plot + Beeswarm Overlay)',
             fontsize=14, fontweight='bold')
ax.spines[['top', 'right']].set_visible(False)
plt.tight_layout()
plt.show()
```

다양한 암석 유형(lithology)에 대한 데이터 분포를 보여주기 위해 비스웜 플롯과 박스 플롯을 결합한 차트.

### 비스웜 플롯(Beeswarm Plots) 사용 시점

비스웜 플롯은 표본 크기가 작거나 중간 정도일 때(그룹당 약 200개 포인트까지) 가장 효과적입니다. 실험실 데이터, 코어 측정값, 또는 모든 데이터 포인트를 보여주는 것이 가치를 더하는 모든 상황에 완벽합니다.

## 요약

범프 차트(Bump charts)는 순위 변화를 쉽게 추적할 수 있게 하여, 복잡한 라인 플롯(line plots)을 부드럽고 가독성 있는 시각화로 만듭니다. `bumplot` 라이브러리를 사용하면 파이썬(Python)에서 이를 쉽게 생성할 수 있습니다.

릿지라인 플롯(Ridgeline plots)은 여러 분포 비교를 간결하고 시각적으로 인상적인 형식으로 압축합니다. 이는 지층(formation)이나 유정(well)에 걸쳐 로그(log) 반응을 정기적으로 비교해야 하는 암석 물리학(petrophysics) 분야에서 특히 유용합니다.

비스웜 플롯(Beeswarm plots)은 박스 플롯(box plots)이 숨기는 개별 데이터 포인트를 보여줍니다. 표본 크기가 작고 모든 측정값이 중요할 때, 요약 통계와 함께 원시 데이터를 보여주는 것은 훨씬 더 솔직한 그림을 제공합니다.
