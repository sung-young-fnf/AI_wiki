# Streamlit으로 30분 만에 데이터 과학 대시보드 구축하는 방법

복잡한 모델과 인사이트를 대화형 웹 앱으로 전환하기 위해 풀스택(Full-stack) 개발자가 될 필요는 없습니다.

데이터 과학 프로젝트에 깊이 몰두하여 지저분한 데이터를 다루고, 복잡한 모델을 미세 조정하며, 마침내 가치 높은 인사이트를 발견하는 데 수백 시간을 투자했을 것입니다. 이제 그 결과물을 보여줄 공간이 필요합니다.

문제는 주피터 노트북(Jupyter Notebook)을 CEO에게 바로 건넬 수 없다는 것입니다.

하지만 여전히 상호작용 가능한 대시보드가 필요하지만, 프런트엔드(Frontend) 개발(HTML, CSS, JS)은 데이터 과학자의 주 업무가 아닙니다.

이 때문에 Streamlit을 배우면 여러분의 노력이 빛을 발할 수 있습니다.

Streamlit은 순수 파이썬(Python)만으로 멋지고 상호작용적인 웹 애플리케이션(Web Application)을 구축할 수 있게 하여, 여러분의 노고에 걸맞은 플랫폼을 제공합니다.

이 글에서는 Streamlit을 사용하여 파이썬 코드를 누구나 쉽게 사용할 수 있는 세련되고 상호작용적인 웹 애플리케이션으로 전환하는 방법을 단계별로 설명합니다. HTML, CSS, JavaScript를 배울 필요가 없습니다.

시작해 봅시다!

## Streamlit이란 무엇인가요?

코드를 살펴보기 전에, 우리가 사용할 도구에 대해 더 잘 이해해 봅시다.

Streamlit은 데이터 과학자와 머신러닝(Machine Learning) 엔지니어가 파이썬 스크립트에서 직접 데이터를 위한 아름답고 맞춤형 웹 애플리케이션을 만들 수 있게 해주는 오픈 소스 파이썬 라이브러리입니다.

Streamlit을 주피터 노트북과 같지만 즉시 공유 가능하고 상호작용적이며, 사용자가 데이터를 입력하고 실시간으로 결과를 볼 수 있게 해주는 도구라고 생각할 수 있습니다.

가장 중요한 점은 주피터 노트북에는 여러분의 코드와 사고 과정이 담겨있다는 것입니다. 그러나 이해관계자들, 특히 비기술적인 측면의 사람들은 여러분의 코드에 관심이 없습니다.

그들이 원하는 것은 결과이며, Streamlit이 바로 그것을 제공할 수 있습니다.

---

## Streamlit이 데이터 과학자에게 완벽한 이유

*   **순수 파이썬(Pure Python):** 웹 개발 언어를 알 필요가 없습니다. 파이썬 스크립트를 작성할 수 있다면 Streamlit 앱을 만들 수 있습니다. 파이썬은 우리에게 가장 친숙한 언어입니다.
*   **빠른 프로토타이핑(Rapid Prototyping):** 속도를 위해 설계되었습니다. 따라서 간단한 대시보드는 며칠이 아닌 몇 분 만에 구축할 수 있습니다.
*   **자동 상호작용:** Streamlit은 슬라이더(Slider)나 버튼(Button)과 같은 구성 요소를 상호작용적으로 만드는 복잡성의 많은 부분을 처리하여 데이터 로직에 집중할 수 있게 해줍니다.

즉, 데이터 전문가는 레이아웃 디자인에 압도될 필요 없이, 본인이 잘하는 일에 계속 집중할 수 있습니다.

## 빠른 시작: Streamlit 설정

### 설치

시작은 간단합니다. 터미널(Terminal) 또는 명령 프롬프트(Command Prompt)를 열고 라이브러리를 설치하면 됩니다. 실제 프로젝트 환경을 시뮬레이션하기 위해 `pandas`와 `scikit-learn`도 함께 설치하겠습니다.

```bash
pip install streamlit pandas scikit-learn
```

### 첫 스크립트

`app.py`라는 새 파일을 생성하여 Streamlit 앱이 작동하는지 테스트합니다.

```python
import streamlit as st
import pandas as pd

# The main title of your app
st.title('Hello, Data Scientist Folks!')

# A simple text message
st.write("This is our first Streamlit app. Let's load some sample data.")

# Create and display a simple DataFrame
data = {'col1': [1, 2, 3], 'col2': ['A', 'B', 'C']}
df = pd.DataFrame(data)

st.dataframe(df)

st.success('App successfully deployed!')
```

### 실행하기!

터미널에서 `app.py` 파일이 있는 디렉토리로 이동하여 다음 명령을 실행하세요.

```bash
streamlit run app.py
```

Streamlit은 웹 브라우저에 새 탭(일반적으로 `http://localhost:8501`)을 자동으로 열어 라이브 애플리케이션을 표시합니다.

다음으로, 제 데이터 과학 프로젝트를 가능하게 하는 Streamlit의 강력한 다섯 가지 기능을 보여드리겠습니다. 분석 및 데이터 정제 과정은 Pandas를 사용할 때와 유사하므로, 여기서는 Streamlit이 제공하는 상호작용 기능에 중점을 둡니다.

## 문제점 1: 정적 차트와 접근하기 어려운 필터

때때로 사용자가 변수, 기간 또는 집계를 변경할 수 있는 차트를 공유하고 싶지만, 이는 복잡한 자바스크립트(JavaScript) 통합을 요구하는 경우가 많습니다.

**Streamlit 솔루션:** `st.sidebar`와 `st.selectbox`, `st.slider`와 같은 입력 위젯(Widget)을 사용하여 이를 달성할 수 있습니다.

**실제 상황:**

고객 활동 데이터셋이 있고, 비즈니스 사용자가 주피터 노트북을 다시 실행해달라고 요청하지 않고도 지역과 최소 구매 횟수(Minimum Number of Purchases)로 시각화를 동적으로 필터링하고 싶어 합니다.

`st.sidebar`를 통해 이를 달성하는 방법은 다음과 같습니다.

```python
import streamlit as st
import pandas as pd
import numpy as np

st.title("Dynamic Customer Activity Dashboard")

# 1. Simulate data
@st.cache_data
def load_data():
    df = pd.DataFrame({
        'region': np.random.choice(['East', 'West', 'North', 'South'], 100),
        'purchases': np.random.randint(1, 20, 100),
        'revenue': np.random.randn(100) * 100
    })
    return df

df = load_data()

# 2. Add the controls to the Sidebar
st.sidebar.header('Filter Controls')

# Selectbox for region
selected_region = st.sidebar.selectbox(
    'Select Region',
    options=['All'] + list(df['region'].unique())
)

# Slider for minimum purchases
min_purchases = st.sidebar.slider(
    'Minimum Purchases Threshold',
    min_value=1, max_value=20, value=5
)

# 3. Filter the data based on user input
if selected_region != 'All':
    df_filtered = df[df['region'] == selected_region]
else:
    df_filtered = df.copy()

df_filtered = df_filtered[df_filtered['purchases'] >= min_purchases]

# 4. Display the results
st.write(f"Displaying data for **{selected_region}** with **{min_purchases}** minimum purchases.")
st.dataframe(df_filtered.head())
st.line_chart(df_filtered.groupby('region')['revenue'].sum())
```

사용자가 정의한 변수(`selected_region`, `min_purchases`)는 파이썬 필터링 로직에 직접 사용됩니다. Streamlit은 입력이 변경될 때마다 전체 스크립트를 자동으로 다시 실행하여, 차트와 테이블을 원활하게 업데이트합니다.

## 문제점 2: 대규모 데이터셋에서의 성능 저하

방대한 데이터셋을 로드하거나 매번 비용이 많이 드는 특성 공학(Feature Engineering)을 다시 실행하면 애플리케이션이 느려지고 사용할 수 없게 됩니다. 이는 노트북(Notebook)에서 모델을 실행할 때의 어려움입니다.

이와 달리 Streamlit에서는 캐싱(Caching) 데코레이터(`@st.cache_data` 및 `@st.cache_resource`)가 함수의 결과값을 기억하도록 Streamlit에 지시하고, 입력이 변경되지 않는 한 함수를 다시 실행하지 않도록 합니다.

따라서 데이터베이스에서 데이터를 한 번만 호출하거나, 프런트엔드에서 매번 CSV 파일을 읽을 필요가 없습니다.

이를 달성하는 방법은 다음과 같습니다.

```python
import streamlit as st
import pandas as pd

st.header("The Power of Caching (Updated)")
my_path = 'path/to/your/large_file.csv'

# The @st.cache_data decorator tells Streamlit to store the result of this function.
@st.cache_data
def load_and_process_data(data_path):
 
    
    st.info(f"--- Data Processing Executed! Size: {data_size} rows ---")
    
    # Load your data
    df = pd.read_csv(data_path)
    return df

df = load_and_process_data(my_path)

st.write(f"DataFrame loaded with {len(df)} rows")

st.dataframe(df.head())
```

## 문제점 3: 분석에 사용자 파일 가져오기

많은 경우, 사용자는 새로운 엑셀(Excel) 시트나 PDF 파일과 같은 자신만의 데이터를 업로드해야 하지만, 안전한 파일 업로드 엔드포인트(Endpoint)를 설정하는 것은 복잡한 웹 작업입니다.

Streamlit에서는 `st.file_uploader`를 사용하여 사용자가 파일을 쉽게 업로드할 수 있도록 하며, 노트북에서 평소와 같이 `pandas` 또는 `polars`를 통해 데이터에 접근하고 데이터 변환을 처리할 수 있습니다.

이를 달성하는 방법은 다음과 같습니다.

```python
import streamlit as st
import pandas as pd

st.header("Batch Prediction File Uploader")
st.write("Upload a CSV file containing new customer data for prediction.")

# The file uploader widget
uploaded_file = st.file_uploader("Choose a CSV file", type=['csv'])

if uploaded_file is not None:
    # Read the file directly into a pandas DataFrame
    try:
        data = pd.read_csv(uploaded_file)
        st.success("File uploaded successfully!")
        
        # Display the first few rows of the uploaded data
        st.subheader("Preview of Uploaded Data")
        st.dataframe(data.head())
        
        # Model Prediction Logic 
      
        predictions = model.predict(data)
        st.dataframe(predictions)

    except Exception as e:
        st.error(f"Error reading file: {e}")
```

`uploaded_file` 변수는 파일 핸들(File Handle)처럼 작동하는 임시 객체입니다. 이를 `pd.read_csv()`에 직접 전달하면 사용자가 업로드한 파일을 시스템의 로컬(Local) 파일처럼 취급할 수 있어, 모든 수동 서버/클라이언트(Server/Client) 파일 처리 로직을 우회할 수 있습니다.

## 문제점 4: 부족한 레이아웃과 사용자 경험

발표 내용을 구성하기 위해 그리드 레이아웃(Grid Layout)에는 `st.columns`를, 다중 섹션 구성에는 `st.tabs`를 사용할 수 있습니다. 이는 CSS 스타일링과 동일한 시각적 결과를 제공하면서도 레이아웃 속성을 조정하는 데 드는 많은 시간을 절약해 줍니다.

**실제 상황:**

대시보드에 비교를 위한 두 개의 나란히 놓인 차트(예: 실제 값(Actual) 대 예측 값(Predicted))와 데이터 탐색 및 모델 지표(Model Metrics)를 위한 별도의 고유한 섹션이 필요합니다.

이를 쉽게 달성할 수 있습니다.

```python
import streamlit as st
import numpy as np

st.header("Structured, Professional Dashboard")

# 1. Using Tabs for major section separation
tab1, tab2 = st.tabs(["Data Visualization", "Model Metrics"])

with tab1:
    st.subheader("Actual vs. Predicted Comparison")
    
    # 2. Using Columns for side-by-side content
    col1, col2 = st.columns(2)

    with col1:
        st.metric(label="Actual Revenue", value="$850K", delta="12%")
        # Placeholder for Actual Revenue Chart
        data = np.random.randn(20, 1)
        st.line_chart(data)

    with col2:
        st.metric(label="Predicted Revenue", value="$880K", delta="15%")
        # Placeholder for Predicted Revenue Chart
        st.bar_chart(data)

with tab2:
    st.subheader("Key Model Performance Indicators")
    colA, colB, colC = st.columns(3)
    
    with colA:
        st.metric(label="Accuracy", value="92.5%")
    with colB:
        st.metric(label="F1 Score", value="0.88")
    with colC:
        st.metric(label="Training Time (s)", value="14.3")
```

`with col1:` 블록 안의 내용은 자동으로 첫 번째 열로 들어가고, `with tab1:` 안의 내용은 첫 번째 탭으로 들어갑니다. 이를 통해 파이썬 스크립트 내에서 바로 깔끔하고 잘 구조화된 레이아웃을 구축할 수 있습니다.

## 문제점 5: 상태 업데이트의 부재

모델 학습과 같이 긴 프로세스가 실행될 때와 작업이 성공적으로 완료될 때 사용자에게 피드백을 주는 것이 중요합니다. 화면상의 확인 메시지가 없으면 애플리케이션이 단순히 멈춘 것처럼 느껴질 수 있습니다.

**실제 상황:**

이해관계자에게 모델을 시연하고, 모델을 재학습(Retrain)하여 업데이트된 성능을 볼 수 있는 옵션을 제공하고 싶습니다.

이해관계자에게 모델을 보여주고 재학습하여 새로운 결과를 볼 수 있는 옵션을 포함한다고 상상해 보십시오. 이 경우, 그들이 "모델 재학습(Retrain Model)" 버튼을 클릭하면, 기다리는 동안 회전하는 로딩 애니메이션(Loader)이 표시된 후 성공 메시지가 나타나야 합니다.

다음 코드를 통해 이 기능을 달성할 수 있습니다.

```python
import streamlit as st
import time

st.header("Status and Feedback")

if st.button('Click to Start Model Retraining (10s)'):
    
    # Use st.spinner to show a loading animation while a long process runs
    with st.spinner('Model retraining in progress... Please wait ⏳'):
        
        # Simulate the 10-second training process
        time.sleep(10)
    
    # After the 'with st.spinner' block finishes, the spinner disappears
    st.success('✅ Model Retraining Complete! New model is live.')

# You can also use st.progress for processes with measurable steps
st.subheader("Progress Bar Example")
progress_bar = st.progress(0, text="Execution Progress")

for percent_complete in range(100):
    time.sleep(0.05)
    progress_bar.progress(percent_complete + 1, text=f"Executing Step {percent_complete + 1}%")

progress_bar.empty() # Clears the progress bar when done
st.info("Process finished!")
```

Streamlit은 `st.spinner` 또는 `st.success`와 같이 한 줄로 구성된 간단한 함수를 제공하여, 최신 애플리케이션에서 기대되는 즉각적이고 세련된 피드백을 제공합니다.

가장 중요하게, 이는 디버깅(Debugging) 및 오류 처리에도 유용하여 터미널 로그에서뿐만 아니라 프런트엔드 화면에서 문제를 식별할 수 있게 해줍니다.

모델 개발에 열심히 노력했지만, CSS나 자바스크립트 코드를 한 줄도 건드리지 않고도 전문적인 그리드 레이아웃, 깔끔한 구성, 원활한 사용자 피드백과 함께 결과를 제시할 공간이 필요합니다.
