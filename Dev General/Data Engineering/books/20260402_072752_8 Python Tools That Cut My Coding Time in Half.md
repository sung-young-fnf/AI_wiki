# 8 Python Tools That Cut My Coding Time in Half

단순히 타이핑을 많이 한다고 해서 개발 속도가 빨라지는 것은 아닙니다. 적절한 도구에 번거로운 작업을 맡길 때 진정한 효율이 발생합니다. 여기 소개하는 8가지 도구는 설치 후 "왜 진작 사용하지 않았을까"라는 후회가 들 정도로 유용한 것들입니다. 흔히 알려진 도구가 아닌, 길게만 느껴졌던 오후 업무를 짧은 풀 리퀘스트(Pull Request)로 바꿔줄 수 있는 숨은 보석 같은 유틸리티들을 소개합니다.

### 1) Hypothesis — 엣지 케이스 수동 작성 중단

*   **기능:** 속성 기반 테스트(Property-based testing). 입력값을 일일이 나열하는 대신 불변의 규칙(invariants)을 정의하면, Hypothesis가 그 규칙을 깨뜨리는 반례를 찾아냅니다.
*   **시간 절약 이유:** 개발자가 미처 생각하지 못한 비정상적인 케이스(edge cases)를 매우 빠르게 찾아냅니다. 덕분에 버그 재발이 줄어들고 "로컬에서는 잘 됐는데"라는 변명을 할 일이 사라집니다.
*   **전문가 팁:** `pytest -q`와 함께 실행하면 Hypothesis가 실패 사례를 최소한의 예시로 축약해 보여줍니다. 의심스러운 함수를 스트레스 테스트할 때는 `@settings(max_examples=200)`를 적절히 활용하세요.

### 2) MonkeyType — 번거로운 타입 타이핑에서 해방

*   **기능:** 런타임 추적(runtime traces)을 수집하여 실제 실행 데이터를 바탕으로 타입 스텁(type stubs)을 자동 생성합니다.
*   **시간 절약 이유:** 코드가 실제로 사용되는 방식에 맞춰 유의미한 타입 어노테이션(type annotations)을 추가해 줍니다. 수작업으로 일일이 입력하는 수고를 덜어줍니다.
*   **전문가 팁:** 지속적 통합(CI) 단계에서 대표적인 데이터셋으로 MonkeyType을 실행하여 실제 데이터 구조를 캡처한 뒤, 커밋 전 스텁(stubs)을 검토하는 워크플로우를 활용해 보세요.

### 3) Scalene — 정확한 수정 위치를 짚어주는 프로파일러

*   **기능:** CPU, 메모리, 라인별 프로파일러(profiler)로, 파이썬 코드 실행 시간과 네이티브(native) 코드 실행 시간을 구분하여 보여주며 라인별 메모리 사용 행태를 분석합니다.
*   **시간 절약 이유:** 막연히 "이 루프가 문제일 거야"라고 추측하는 대신, 정확히 어느 라인에서 병목이 발생하는지, C 확장 모듈 호출 뒤에 숨겨진 비용이 있는지 명확히 보여줍니다.
*   **전문가 팁:** 로컬 벤치마크 결과가 운영 환경과 다를 때 Scalene을 사용해 보세요. 리스트 컴프리헨션(list comprehensions)이나 `.values` 호출로 인해 발생하는 숨겨진 복사본 등 예기치 못한 메모리 할당 지점을 찾아내 줍니다.

### 4) Ruff — 속도와 기능을 모두 잡은 통합 도구

*   **기능:** 극도로 빠른 린터(linter) 겸 포매터(formatter)입니다. 러스트(Rust)로 작성되어 기존의 여러 린터보다 수십 배 이상 빠릅니다.
*   **시간 절약 이유:** 프리커밋(pre-commit) 단계에서 한 번만 실행하면 스타일 체크를 위해 몇 분씩 기다릴 필요가 없습니다.
*   **전문가 팁:** Ruff의 규칙 세트에는 f-string 변환이나 최신 문법 자동 적용 기능이 포함되어 있습니다. 가치가 낮은 단순 정리는 Ruff의 자동 수정(`--fix`) 기능에 맡기세요.

### 5) Polars — 기다림을 없애주는 데이터프레임 속도

*   **기능:** 러스트(Rust) 기반의 고성능 데이터프레임(DataFrame) 라이브러리로, 많은 ETL 단계에서 느린 판다스(Pandas) 파이프라인을 대체합니다.
*   **시간 절약 이유:** 작업 반복 주기가 빨라집니다. 예전에 몇 분씩 걸리던 작업이 몇 초 만에 완료되므로, 더 많은 실험을 하고 더 나은 데이터 변환 코드를 배포할 수 있습니다.
*   **전문가 팁:** 탐색적 분석 시에는 지연 실행(lazy mode, `df.lazy()`)을 사용하여 파이프라인을 구축한 뒤 `.collect()`를 호출하세요. 여러 번의 읽기/스캔 단계를 생략하여 성능을 극대화할 수 있습니다.

### 6) PEX / Shiv — 단일 실행 파일로 의존성 지옥 탈출

*   **기능:** 의존성이 포함된 독립형 파이썬 zipapp 또는 실행 파일을 빌드합니다.
*   **시간 절약 이유:** 배포할 때마다 가상환경(virtualenv)을 설정할 필요가 없습니다. 빌드된 파일은 어디서나 동일하게 작동합니다.
*   **전문가 팁:** 파이썬 환경이 구축되지 않은 팀에 내부 CLI 도구를 전달할 때 유용합니다. 설치 안내 없이 파일 하나만 전달하면 즉시 생산성을 높일 수 있습니다.

### 7) DoIt — 재현 가능한 작업을 위한 태스크 러너

*   **기능:** 파이썬으로 정의하는 가벼운 태스크 러너(task runner)입니다. `make`와 유사하지만 파이썬 문법을 사용합니다.
*   **시간 절약 이유:** 빌드, 린트, 테스트, 패키징 등 작고 재현 가능한 단계들을 셸 스크립트의 번거로움 없이 하나로 통합 관리할 수 있습니다.
*   **전문가 팁:** 실시간 개발 루프를 위해 감시(watch) 태스크를 추가하세요. `doit auto`를 사용하면 파일 변경 시 자동으로 태스크를 재실행하여 수동 실행보다 빠른 피드백을 받을 수 있습니다.

### 8) PyOxidizer — 빠른 실행을 위한 네이티브 바이너리 빌드

*   **기능:** 파이썬 인터프리터와 코드를 하나의 네이티브 실행 파일로 임베딩합니다. 빠른 시작 속도와 깔끔한 배포가 장점입니다.
*   **시간 절약 이유:** 배포된 도구의 런타임 버전 문제를 제거합니다. 파이썬이 설치되지 않은 동료들의 환경 설정을 도와주느라 소비하는 시간을 줄여줍니다.
*   **전문가 팁:** 여러 팀에서 사용하는 내부 CLI 도구에 PyOxidizer를 적용해 보세요. "제 컴퓨터에선 안 되는데요"라는 이슈 리포트를 획기적으로 줄여줍니다. 초기 학습 곡선은 있지만, 배포 관련 버그를 해결하는 시간을 생각하면 충분한 가치가 있습니다.
