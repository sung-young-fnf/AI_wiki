# 데이터 정제 속도를 5배 높여준 파이썬 라이브러리 6가지
깔끔한 데이터가 곧 명확한 인사이트로 이어지니까요.

데이터셋을 보면서 이런 생각을 해본 적이 있으신가요?

"이건 데이터가 아니야. 이건 CSV 옷을 입은 혼돈 그 자체야."

우리 모두 그런 경험이 있을 겁니다. 웹 스크래핑을 하든, 미심쩍은 API에서 데이터를 가져오든, 전임자의 끔찍한 스프레드시트 작업을 물려받든, 지저분한 데이터는 언제나 예상할 수 있는 부분이죠.

하지만 수년간 데이터 먼징(data munging) 작업에 깊이 몰두하면서, 저는 제대로 활용되지 않거나 과소평가된 파이썬(Python) 라이브러리 몇 가지를 발견했습니다. 이 라이브러리들은 저의 데이터 정제 워크플로우(workflow)를 '매번 되돌리고 싶은 혼돈'에서 '손쉬운 파이프라인(pipeline)의 축복'으로 탈바꿈시켰죠.

여기 데이터 정제 시간을 최소 5배 이상 단축시켜준 파이썬 라이브러리 6가지가 있습니다.
참고로 판다스(pandas)는 포함하지 않았습니다. 판다스는 이미 잘 알고 계실 테니, 다음 단계로 나아가 봅시다.

### 1. `datacompy`: 데이터프레임 자동 비교 (수동 검사 끝!)

두 엑셀 내보내기(Excel exports) 파일을 행(row) 단위로 비교해본 적이 있으신가요?
네. 저도요. 다시는 안 할 겁니다.

```python
import datacompy
import pandas as pd

df1 = pd.read_csv('data_old.csv')
df2 = pd.read_csv('data_new.csv')

compare = datacompy.Compare(df1, df2, join_columns='id')
print(compare.report())
```

**왜 혁신적인가:**
불일치를 감지하기 위해 사용자 지정 스크립트를 작성하는 일은 잊으세요. `datacompy`는 정확히 어떤 행이 다르고, 어떤 필드(field)가 변경되었는지 알려주며, 심지어 사람이 읽기 쉬운 보고서까지 출력합니다. 데이터프레임(dataframe)을 위한 `diff` 도구에 스테로이드를 맞춘 것과 같습니다.

**전문가 팁:** 다중 열 조인(multi-column joins)과 퍼지 매칭(fuzzy matching)을 지원합니다. 버전 관리되는 데이터셋(version-controlled datasets)과 함께 사용하면 막을 수 없습니다.

### 2. `pyjanitor`: 판다스, 하지만 MBA 학위 소지자

판다스(pandas)가 전동 공구라면, `pyjanitor`는 개인 비서가 딸린 전동 공구입니다. 이 라이브러리를 사용하면 데이터 정제 단계를 마치 이야기를 읽듯이 연결(chaining)할 수 있습니다.

```python
import pandas as pd
import janitor

df = (
    pd.read_csv('messy_data.csv')
    .clean_names()
    .remove_empty()
    .drop_duplicates(subset='email')
    .filter_on('age >= 18')
)
```

**왜 강력한가:**
스파게티 코드(spaghetti code)처럼 보이지 않는, 읽기 쉽고 연결 가능한 프로덕션 품질의 정제 코드를 얻을 수 있습니다. 또한, 도메인 특화(domain-specific) 정제 도구 모음으로 판다스(pandas)를 확장합니다.

**실제 사례:** 저는 `pyjanitor`를 사용하여 5백만 행 규모의 고객 지원 데이터셋을 15줄 미만의 코드로 정제했습니다. 순수 판다스(vanilla pandas)를 사용했다면 3배는 더 많은 시간이 걸렸을 겁니다.

### 3. `dirty_cat`: 모든 범주형 데이터가 똑같지는 않으니까

제품명, 직업명 또는 무작위 텍스트 항목과 같은 고유한 범주형 변수(high-cardinality categorical variables)를 다루고 있다면, 하던 일을 멈추고 다음을 설치하세요.

```bash
pip install dirty_cat
```

```python
from dirty_cat import SimilarityEncoder
import pandas as pd

X = pd.DataFrame({'job_title': ['data engineer', 'data scientist', 'ML engineer']})
encoder = SimilarityEncoder()
X_encoded = encoder.fit_transform(X[['job_title']])
```

**왜 천재적인가:**
전통적인 원-핫 인코딩(one-hot encoding)은 지저분한 문자열에는 제대로 작동하지 않습니다. `dirty_cat`은 문자열 유사도(string similarity)를 사용하여 범주를 클러스터링(cluster)하고 인코딩(encode)합니다. 이는 더 적은 차원(dimensions)과 ML 모델(ML models)을 위한 더 나은 신호를 의미합니다.

**재미있는 사실:** 사이킷런(scikit-learn) 개발팀이 만들었습니다. 신뢰할 수 있는 도구입니다.

### 4. `fuzzywuzzy`: 사람처럼 오타 정정하기

솔직히 말해봅시다. 사람은 철자법에 약합니다. "NYC" ≠ "New York" ≠ "N.Y." 이지만 모두 같은 것을 의미합니다.

여기서 `fuzzywuzzy`가 빛을 발합니다.

```python
from fuzzywuzzy import fuzz
from fuzzywuzzy import process

choices = ["New York", "Los Angeles", "Chicago"]
print(process.extract("N.Y.", choices, limit=2))
```

**왜 필수인가:**
이 라이브러리를 사용하여 열을 병합(merge columns)하거나, 문자열 중복을 제거(deduplicate strings)하거나, 일관성 없는 항목을 수정하거나, 유사성을 기반으로 열 값을 자동 수정(auto-correct)할 수 있습니다. 레벤슈타인 거리(Levenshtein distance)와 비슷하지만, 더 스마트한 점수 매기기 방식이 적용됩니다.

**주의 사항:** 대규모 데이터셋의 경우, 더 빠르고 드롭인 호환(drop-in compatible)되는 대안인 `RapidFuzz`와 함께 사용하세요.

### 5. `pandas_schema`: 데이터가 망가지기 전에 검증하세요

"salary"로 모델을 학습시켰는데, 절반의 값이 "N/A" 문자열이었다는 것을 깨달은 적이 있으신가요? 😐
`pandas_schema`는 데이터프레임(dataframe)에 스키마(schema) 규칙을 정의하고 적용할 수 있게 해줍니다.

```python
from pandas_schema import Column, Schema
from pandas_schema.validation import IsDtypeValidation, InRangeValidation

schema = Schema([
    Column('age', [IsDtypeValidation(int), InRangeValidation(0, 120)]),
    Column('email', [IsDtypeValidation(str)])
])

errors = schema.validate(df)
for error in errors:
    print(error)
```

**왜 중요한가:**
이것을 데이터의 단위 테스트(unit tests)라고 생각하세요. 쓰레기 데이터가 모델, 대시보드(dashboard) 또는 — 더 나쁘게는 — 상사에게 도달하기 전에 잡아낼 수 있습니다.

### 6. `missingno`: 결측치를 위한 X-레이

결측치(nulls)를 시각화하면 `df.isnull().sum()`을 20분간 스크롤하는 것보다 2초 만에 더 많은 것을 알 수 있습니다.

```python
import missingno as msno
msno.matrix(df)
```

또는 결측치 간의 상관관계(correlations) 히트맵(heatmap)을 얻을 수도 있습니다.

```python
msno.heatmap(df)
```

**왜 가치 있는가:**
가격 정보가 없는 모든 행에 할인 정보도 없는지, 또는 결측치가 특정 기간에 집중되어 있는지 즉시 파악할 수 있습니다. 이는 아무런 번거로움 없이 얻을 수 있는 실용적인 통찰력(actionable insight)입니다.
